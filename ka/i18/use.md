## Გამოყენება

### Დემო Პროექტი

`i18` იხილეთ დემო პროექტი თარგმანის კონფიგურაციის გასაგებად [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

მომხმარებლებს ჩინეთში შეუძლიათ კლონირება [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

კლონირების შემდეგ შედით დირექტორიაში და გაუშვით თარგმანის დასასრულებლად `i18`

### Დირექტორია Სტრუქტურა

შაბლონის საწყობის დირექტორია სტრუქტურა ასეთია

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` შეიცავს თარგმნილ დემო ფაილებს, რომლებიც მხოლოდ მაგალითია და შეიძლება წაიშალოს.

### Გაუშვით Თარგმანი

შეიყვანეთ დირექტორია და `i18` თარგმნა.

### Დაამატეთ Ფაილები Საცავში

თარგმანის გარდა, პროგრამა ასევე გამოიმუშავებს შემდეგ ფაილებს, გთხოვთ, დაამატოთ ისინი საცავში.

```
.i18n/hash
.i18n/cache/.gitignore
```

მათ : , შინაარსი ასეთია `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

ეს ნიშნავს, რომ უგულებელყოთ ყველა ფაილი `.i18n/cache/` (გარდა `.i18n/cache/.gitignore` ).

თუ თქვენი ვერსიის კონტროლის პროგრამა არ არის `git` გთხოვთ, უგულებელყოთ იგი ამ კონფიგურაციის მიხედვით.

## Კონფიგურაციის Ფაილი

ეს `.i18n/conf.yml` `i18` line თარგმანის ინსტრუმენტის კონფიგურაციის ფაილი

შინაარსი ასეთია:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Საწყისი Ენა &

კონფიგურაციის ფაილში `fromTo`

`en` არის საწყისი ენა, `zh ja ko de fr` არის თარგმანის სამიზნე ენა.

ენის კოდი იხილეთ [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

მაგალითად, თუ გსურთ თარგმნოთ ჩინური ინგლისურად, გადაწერეთ ეს ხაზი `zh: en`

თუ გსურთ თარგმნოთ ყველა მხარდაჭერილ ენაზე, გთხოვთ დატოვეთ ცარიელი შემდეგ `:` მაგალითად

```
i18n:
  fromTo:
    en:
```

### Ფაილის Იგნორირება

ნაგულისხმევად, ყველა ფაილი, `.yml` იწყება `.md`

თუ გსურთ უგულებელყოთ გარკვეული ფაილები და არ თარგმნოთ ისინი (როგორიცაა დაუმთავრებელი მონახაზები), შეგიძლიათ გამოიყენოთ `ignore` კონფიგურაცია.

იყენებს `.gitignore` `ignore` მსგავს სინტაქსს [globset](https://docs.rs/globset/latest/globset/#syntax)

მაგალითად, ზემოაღნიშნული კონფიგურაციის ფაილში ნიშნავს `_* ` რომ `_` -ით დაწყებული ფაილები არ ითარგმნება.

## Თარგმანის Წესები

### Თარგმანის Რედაქტორებმა Არ Უნდა Დაამატონ Ან Წაშალონ Ხაზები

თარგმანი რედაქტირებადია. შეცვალეთ ორიგინალი ტექსტი და თარგმნეთ იგი ხელახლა, თარგმანის მექანიკური ცვლილებები არ გადაიწერება (თუ ორიგინალური ტექსტის ეს პუნქტი არ არის შეცვლილი).

მაგრამ გთხოვთ გაითვალისწინოთ, რომ თარგმანისა და ორიგინალური ტექსტის სტრიქონები უნდა შეესაბამებოდეს ერთს. ანუ, თარგმანის შედგენისას არ დაამატოთ ან წაშალოთ სტრიქონები. წინააღმდეგ შემთხვევაში, ეს გამოიწვევს დაბნეულობას თარგმანის რედაქტირების ქეშიში.

თუ რამე არასწორედ წარიმართება, [გადაწყვეტილებისთვის მიმართეთ FAQ-ს.](/i18/qa#H1)

### `Yaml`

ბრძანების სტრიქონი იპოვის ყველა ფაილს `.yml` რომელიც მთავრდება საწყის ენაზე ფაილის დირექტორიაში და თარგმნის მათ.

* გაითვალისწინეთ `.yaml` რომ ფაილის სახელის სუფიქსი უნდა იყოს `.yml`

ინსტრუმენტი თარგმნის მხოლოდ ლექსიკონის მნიშვნელობებს `.yml` არა ლექსიკონის კლავიშებს.

მაგალითად `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

ითარგმნება როგორც `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` ასევე შეიძლება შეიცვალოს ხელით (მაგრამ არ დაამატოთ ან წაშალოთ კლავიშები ან ხაზები თარგმანში).

`YAML` საფუძველზე, თქვენ შეგიძლიათ მარტივად შექმნათ საერთაშორისო გადაწყვეტილებები სხვადასხვა პროგრამირების ენებისთვის.

## Გაფართოებული Გამოყენება

### Თარგმანის Ქვეცნობარი

სანამ თქვენ `.i18n/conf.yml` (არ არის საჭირო ყოველ ჯერზე დემო პროექტის შაბლონიდან დაწყება), `i18` კარგად იმუშავებს.

ბრძანების ხაზის ინსტრუმენტი იპოვის `.i18n/conf.yml`

პროექტებს, რომლებიც იყენებენ [monorepo](//monorepo.tools) შეუძლიათ ენობრივი ფაილების დაყოფა ქვედირექტორებად.

<img alt="" src="https://p.3ti.site/1719910016.avif">

### Მორგებული Ინსტალაციის Დირექტორია

ის დაინსტალირდება სტანდარტულად `/usr/local/bin`

`/usr/local/bin` არ აქვს ჩაწერის უფლება, ის დაინსტალირდება `~/.bin` -ზე.

დააყენეთ გარემოს ცვლადები `TO` მაგალითად, შეგიძლიათ განსაზღვროთ ინსტალაციის დირექტორია :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
