# التثبيت والاستخدام

## Windows يجب أولاً تثبيت Git Bash

الرجاء [الضغط هنا لتحميل وتثبيت `git bash` أولا](https://git-scm.com/download/win) على نظام Windows.

شغّل العمليات التالية داخل `git bash`.

## التثبيت

```sh
bash <(curl -sS https://i.i18n.site) i18
```

## الاستخدام

### مشروع تجريبي

يرجى الرجوع إلى مشروع [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18) لفهم تكوين الترجمة `i18`.

يمكن للمستخدمين في الصين استنساخ [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

بعد الاستنساخ، انتقل إلى الدليل وقم بتشغيل `i18` للترجمة.

### هيكل الدليل

هيكل دليل مستودع القالب كالتالي

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

الملفات التجريبية المترجمة في مجلد `en` هي مجرد أمثلة ويمكن حذفها.

### تشغيل الترجمة

انتقل إلى الدليل وقم بتشغيل `i18` للترجمة.

بالإضافة إلى الترجمة، سيقوم البرنامج بإنشاء مجلد `.i18n/data`، يرجى إضافته إلى المستودع.

بعد ترجمة الملف الجديد، سيتم إنشاء ملف بيانات جديد في هذا المجلد، تذكر إضافة `git add .` إلى سجل التزاماتك.

## ملف التكوين

`.i18n/conf.yml` هو ملف تكوين أداة الترجمة السطرية `i18`

المحتوى كالتالي:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### لغة المصدر و لغة الترجمة

في ملف التكوين،`tfromTo` يشير إلى:

`en` هي لغة المصدر، و `zh ja ko de fr` هي لغات الترجمة المستهدفة.

رموز اللغات راجع [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE).

على سبيل المثال، إذا كنت ترغب في ترجمة الصينية إلى الإنجليزية، قم بكتابة `zh: en`.

إذا كنت ترغب في الترجمة إلى جميع اللغات المدعومة، اترك الحقل فارغًا بعد `:`

```
i18n:
  fromTo:
    en:
```

يمكنك تكوين `fromTo` مختلف : فرعية مختلفة /

```
i18n:
  fromTo:
    en:
  path:
    blog:
      fromTo:
        zh:
    blog/your_file_name.md:
      fromTo:
        ja:
```

في هذا الجدول التكويني، اللغة المصدر لترجمة دليل `blog` هي `zh`، أما اللغة المصدر لترجمة ملف `blog/your_file_name.md` هي `ja`.

### تجاهل الملفات

افتراضيًا، سيتم ترجمة جميع الملفات التي تنتهي بـ `.md` و `.yml` في دليل لغة المصدر.

إذا كنت ترغب في تجاهل ملفات معينة وعدم ترجمتها (مثل المسودات غير المكتملة)، يمكنك استخدام حقل `ignore`.

`ignore` يستخدم بناءً مشابهًا لبناء [globset](https://docs.rs/globset/latest/globset/#syntax) مثل ملف `.gitignore`.

على سبيل المثال، `_*` في ملف التكوين يعني تجاهل جميع الملفات التي تبدأ بـ `_`.

## قواعد الترجمة

### لا يجب على محرري الترجمة إضافة أو حذف أسطر

الترجمة قابلة للتعديل. إذا قمت بتعديل النص الأصلي وترجمته مرة أخرى، فإن التعديلات اليدوية على الترجمة لن تُمحى (إذا لم يتم تعديل هذا الجزء من النص الأصلي).

لكن يرجى ملاحظة أن أسطر الترجمة والنص الأصلي يجب أن تتطابق أسطرًا بأسطر. لا تضيف أو احذف أسطرًا أثناء ترجمة النص، وإلا سيؤدي ذلك إلى فساد ذاكرة التخزين المؤقت للترجمة.

إذا حدث أي خطأ، راجع [الحلول في الأسئلة الشائعة](/i18/qa#H1)

### ترجمة `YAML`

أداة سطر الأوامر ستجد جميع الملفات التي تنتهي بـ `.yml` في دليل لغة المصدر وترجمتها.

* لاحظ أن لاحقة اسم الملف يجب أن تكون `.yml` (وليس `.yaml`).

الأداة تترجم فقط قيم القاموس في `.yml`، وليس مفاتيح القاموس.

على سبيل المثال، `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

سيتم ترجمتها إلى `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

يمكن تعديل ترجمة `YAML` يدويًا (لكن لا تقم بإضافة أو حذف مفاتيح أو أسطر في الترجمة).

استنادًا إلى `YAML` ترجمة، يمكنك بسهولة إنشاء حلول دولية لمختلف لغات البرمجة.

## الاستخدام المتقدم

### ترجمة الدلائل الفرعية

只要创建了 `.i18n/conf.yml` (بدون الحاجة إلى البدء من قالب المشروع التجريبي في كل مرة)، `i18` سيعمل بشكل طبيعي.

أداة سطر الأوامر ستجد جميع تكوينات `.i18n/conf.yml` في الدلائل الفرعية وترجمتها.

يمكن للمشاريع التي تستخدم بنية [monorepo](//monorepo.tools) توزيع ملفات اللغة على الدلائل الفرعية.

![](https://p.3ti.site/1719910016.avif)

### دليل التثبيت المخصص

سيتم تثبيته في `/usr/local/bin` بشكل افتراضي.

إذا لم يكن لديك إذن الكتابة في `/usr/local/bin`، سيتم تثبيته في `~/.bin`.

يمكنك تعيين متغير البيئة `TO` لتحديد دليل التثبيت، على سبيل المثال:

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
