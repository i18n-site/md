# البرنامج المساعد

يمكن تكوين المكونات الإضافية في ملف `.i18n/conf.yml`، مثل:

```yml
addon:
  - i18n.addon/toc
```

## المكونات الإضافية الرسمية

* `i18n.addon/toc` : فهرس المحتويات
  يتم إنشاء فهرس محتويات `json` بناءً على `TOC`، وهو مفعّل افتراضيًا

* `i18n.addon/mouse` : تأثيرات الفأرة

## تسمية الملفات

جميع المكونات الإضافية هي حزم `npm`.

الحزمة المقابلة لـ `i18n.addon/toc` هي [https://www.npmjs.com/package/@i18n.addon/toc](https://www.npmjs.com/package/@i18n.addon/toc).

يستخدم البرنامج المساعد أحدث إصدار افتراضيًا ويقوم بالتحقق من التحديثات بشكل أسبوعي.

إذا كنت ترغب في تثبيت إصدار محدد، يمكنك كتابة `i18n.addon/toc@0.1.3`.

سيقوم سطر الأوامر `i18n.site` بتثبيت ملفات الاتفاقية الخاصة بالمكون الإضافي ثم تنفيذها.

أسماء الملفات المتفق عليها هي كما يلي

### htmIndex.js

سيتم حقن `htmIndex.js` في نهاية ملف `.i18n/htm/index.js`.

حيث سيتم استبدال `__CONF__` باسم التكوين الحالي (مثل `dev` أو `ol`).

### afterTran.js

سيتم استدعاء هذا بعد اكتمال الترجمة، وسيتم تمرير المعلمات التالية.

* `lang_li` : قائمة اللغات، اللغة الأولى هي لغة المصدر
* `changed` : الملفات التي تم تعديلها
* `root` : دليل الجذر للمشروع

قيمة الإرجاع هي قاموس، مثل

```json
{
  file:{
    //  path: txt, for example :
    // "_.json": "[]"
  }
}
```

`file` هو قائمة ملفات الإخراج، `path` هو مسار الملف، و `txt` هو محتوى الملف.

## الدوال المدمجة

يعتمد وقت التشغيل `js` المدمج على التطوير الثانوي لـ [boa](https://github.com/boa-dev/boa)، والوظائف المضمنة هي كما يلي:

* `wPath(path, txt)` : كتابة النص في الملف
* `rTxt(path)` : قراءة النص من الملف
* `rBin(path)` : قراءة الملف الثنائي
* `rDir(dirpath)` : قراءة الدليل، القيمة المرجعة هي قائمة: قائمة الدلائل، قائمة الملفات
