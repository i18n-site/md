# ਸਰਵਰ ਸ਼ੁਰੂ ਕਰੋ (BTRFS ਭਾਗ, ਸਨੈਪਰ ਅਨੁਸੂਚਿਤ ਸਨੈਪਸ਼ਾਟ ਦੀ ਵਰਤੋਂ ਕਰੋ)

## contabo ਖਰੀਦ ਸਰਵਰ

contabo ਜਰਮਨੀ ਵਿੱਚ ਇੱਕ ਸਸਤਾ VPS ਪ੍ਰਦਾਤਾ ਹੈ ਜਿਸਦਾ ਲੰਬਾ ਇਤਿਹਾਸ (20 ਸਾਲਾਂ ਤੋਂ ਵੱਧ) ਅਤੇ ਬਹੁਤ ਸਾਰੇ ਉਪਭੋਗਤਾ (200,000 ਤੋਂ ਵੱਧ) ਹਨ, ਇਸ ਲਈ ਭੱਜਣ ਦੀ ਚਿੰਤਾ ਕਰਨ ਦੀ ਕੋਈ ਲੋੜ ਨਹੀਂ ਹੈ।

ਖਰੀਦਣ VPS [contabo.com](//contabo.com) ਯੂਰੋ ਕੀਮਤ ਦੀ ਚੋਣ ਕਰਨਾ ਯਾਦ ਰੱਖੋ, ਜੋ ਕਿ ਸਸਤਾ ਹੈ।

![](https://i-01.eu.org/1718285782.avif)

ਕੰਟੈਬੋ ਲਈ ਇੱਕ ਵਾਰ ਦੀ ਸਥਾਪਨਾ ਫੀਸ ਹੈ।
ਹਾਲਾਂਕਿ, ਹਰ ਮਹੀਨੇ ਕੁਝ ਦਿਨ ਹੁੰਦੇ ਹਨ ਜਿੱਥੇ ਇੰਸਟਾਲੇਸ਼ਨ ਫੀਸ ਮੁਫਤ ਹੁੰਦੀ ਹੈ, ਬੱਸ ਰਜਿਸਟਰ ਕਰੋ ਅਤੇ ਪ੍ਰਚਾਰ ਸੰਬੰਧੀ ਈਮੇਲਾਂ 'ਤੇ ਧਿਆਨ ਦਿਓ।

ਯੂਰਪ ਵਿੱਚ ਮੁਫ਼ਤ ਖੇਤਰ ਦੀ ਫੀਸ ਤੋਂ ਇਲਾਵਾ, ਹੋਰ ਖੇਤਰਾਂ ਵਿੱਚ ਕੀਮਤ ਵਾਧੇ ਦੇ ਵੱਖ-ਵੱਖ ਪੱਧਰ ਹਨ।

ਮੈਂ ਇੱਕ ਯੂਰੋਪੀਅਨ ਸਰਵਰ ਚੁਣਿਆ ਹੈ ਵੈਸੇ ਵੀ, ਕਲਾਉਡਫਲੇਅਰ ਦੀ ਵਰਤੋਂ ਕਰਨ ਤੋਂ ਬਾਅਦ, ਸਰਵਰਾਂ ਵਿੱਚ ਬਹੁਤ ਅੰਤਰ ਨਹੀਂ ਹੈ.

ਅੰਤਿਮ ਸੰਰਚਨਾ ਹੇਠ ਲਿਖੇ ਅਨੁਸਾਰ ਹੈ, 16GB ਮੈਮੋਰੀ, 6CPU, 200 GB NVMe, 9.50 ਯੂਰੋ ਪ੍ਰਤੀ ਮਹੀਨਾ, 74.43 RMB ਦੇ ਬਰਾਬਰ।

![](https://i-01.eu.org/1718286008.avif)

**ਇਹ ਕੀਮਤ ਅਲੀਬਾਬਾ ਕਲਾਊਡ ਅਤੇ ਐਮਾਜ਼ਾਨ ਦੀ ਕੀਮਤ ਦਾ ਲਗਭਗ ਦਸਵਾਂ ਹਿੱਸਾ ਹੈ** ।

![](https://i-01.eu.org/1718356956.avif)

ਮੈਂ ਪ੍ਰਦਰਸ਼ਨ ਅਤੇ ਪੈਰਾਮੀਟਰਾਂ ਨੂੰ ਚਲਾਉਣ ਲਈ [Yet-Another-Bench-Script](https://github.com/masonr/yet-another-bench-script) :

```
curl -sL yabs.sh | bash -s -- -s https://www.vpsbenchmarks.com/yabs/upload
```

ਮਾਡਲ `AMD EPYC 9224 24-Core Processor` ਦਿਖਾ ਰਿਹਾ CPU , ਸਿੰਗਲ-ਕੋਰ ਪ੍ਰਦਰਸ਼ਨ ਸਕੋਰ 1285
, ਮਲਟੀ-ਕੋਰ ਸਕੋਰ 4277 ਹੋਰ ਨਤੀਜੇ ਵੇਖੋ [vpsbenchmarks.com](https://www.vpsbenchmarks.com/yabs/contabo-6c-16gb-3fb7d8)

ਮੈਂ ਇੱਕ ਸਮੇਂ ਵਿੱਚ ਤਿੰਨ ਸਰਵਰ ਖਰੀਦੇ ਹਨ ਅਤੇ ਉਹਨਾਂ ਨੂੰ ਇੱਕੋ ਸਮੇਂ ਸਰਗਰਮ ਕੀਤਾ ਹੈ ਇਹ ਯਕੀਨੀ ਬਣਾਉਂਦਾ ਹੈ ਕਿ ਸਰਵਰ ਇੱਕੋ ਕੰਪਿਊਟਰ ਰੂਮ ਵਿੱਚ ਹਨ, ਜਿਸ ਨਾਲ ਭਵਿੱਖ ਵਿੱਚ ਉੱਚ ਉਪਲਬਧਤਾ ਬਣਾਉਣਾ ਆਸਾਨ ਹੋ ਜਾਂਦਾ ਹੈ।

ਇਸਦੇ ਸਰਵਰ ਕਦੇ-ਕਦਾਈਂ ਰੁਕ ਜਾਂਦੇ ਹਨ, ਉੱਚ ਉਪਲਬਧਤਾ ਦੁਆਰਾ ਵੈਬਸਾਈਟ ਦੀ ਪਹੁੰਚ ਨੂੰ ਯਕੀਨੀ ਬਣਾਉਂਦੇ ਹੋਏ।

ਨਵੇਂ ਉਪਭੋਗਤਾਵਾਂ ਨੂੰ ਖਰੀਦਣ ਵੇਲੇ ਅਸਲ-ਨਾਮ ਪ੍ਰਮਾਣਿਕਤਾ ਲਈ ਆਪਣੇ ਆਈਡੀ ਕਾਰਡ ਜਮ੍ਹਾ ਕਰਨ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ।

## ਪਾਸਵਰਡ-ਮੁਕਤ ਲੌਗਇਨ ਕੌਂਫਿਗਰ ਕਰੋ

ਕੁਝ ਮਿੰਟਾਂ ਬਾਅਦ, ਸਰਵਰ ਸਾਰੇ ਐਕਟੀਵੇਟ ਹੋ ਗਏ।

ਸਭ ਤੋਂ ਪਹਿਲਾਂ ਇੱਕ ਉਪਨਾਮ ਸੈੱਟ ਕਰਨ ਲਈ ਸਥਾਨਕ ਮਸ਼ੀਨ ਦੇ `~/.ssh/config` ਸੰਪਾਦਿਤ ਕਰਨਾ IP , ਅਤੇ ਫਿਰ ਪਾਸਵਰਡ-ਮੁਕਤ ਲਾਗਇਨ ਨੂੰ ਸੰਰਚਿਤ ਕਰਨ ਲਈ `ssh-copy-id root@c2` ਦੀ ਵਰਤੋਂ ਕਰਨਾ ਹੈ।

![](https://i-01.eu.org/1718287198.avif)

ਤੁਸੀਂ ਸ਼ੁਰੂਆਤੀ ਸੰਰਚਨਾ ਨੂੰ ਤੇਜ਼ੀ ਨਾਲ ਪੂਰਾ ਕਰਨ ਲਈ ਇਸ [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) ਸੋਧ ਸਕਦੇ ਹੋ।

ਪਾਸਵਰਡ-ਮੁਕਤ ਸੰਰਚਨਾ ਪੂਰੀ ਹੋਣ ਤੋਂ ਬਾਅਦ, ਤੁਸੀਂ ਇਸ ਨੂੰ ਬੈਚਾਂ ਵਿੱਚ ਚਲਾਇਆ ਜਾ ਸਕਦਾ ਹੈ, ਜੋ ਕਿ ਸੁਵਿਧਾਜਨਕ ਹੈ [pdsh](https://github.com/chaos/pdsh)

ਪਹਿਲਾਂ ipv6 ਅਤੇ ਟੈਸਟ ਨੂੰ ਹੱਥੀਂ ਸਮਰੱਥ ਕਰਨਾ ਹੈ

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

![](https://i-01.eu.org/1718287969.avif)

ਫਿਰ ਇੱਕ ਕਲਿੱਕ ਨਾਲ ਸਿਸਟਮ ਨੂੰ ubuntu 24 ਵਿੱਚ ਅੱਪਗਰੇਡ ਕਰੋ ਅਤੇ ਆਮ ਤੌਰ 'ਤੇ ਵਰਤੇ ਜਾਂਦੇ ਸੌਫਟਵੇਅਰ ਨੂੰ ਇੰਸਟਾਲ ਕਰੋ।

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 ਦਾ ਅਰਥ ਹੈ ਭਾਸ਼ਾ ਨੂੰ ਚੀਨੀ ਅਤੇ ਸਮਾਂ ਖੇਤਰ ਨੂੰ ਬੀਜਿੰਗ ਵਿੱਚ ਬਦਲਣਾ।

ਡਿਫੌਲਟ ਤੌਰ 'ਤੇ ਸਥਾਪਿਤ ਕੀਤੇ ਗਏ ਸੌਫਟਵੇਅਰ ਨੂੰ ਮੇਰੀ ਨਿੱਜੀ ਵਰਤੋਂ [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) ਆਦਤਾਂ ਦੇ ਅਨੁਸਾਰ ਸੰਰਚਿਤ ਕੀਤਾ ਗਿਆ ਹੈ, ਜਿਵੇਂ ਕਿ [mise](https://github.com/jdx/mise) , [direnv](https://direnv.net/) , ਆਦਿ।

ਇਸ ਕਦਮ ਵਿੱਚ ਲੰਬਾ ਸਮਾਂ ਲੱਗਦਾ ਹੈ ਅੱਧੇ ਘੰਟੇ ਦੀ ਉਡੀਕ ਕਰਨ ਤੋਂ ਬਾਅਦ, ਆਮ ਤੌਰ 'ਤੇ ਵਰਤੇ ਜਾਣ ਵਾਲੇ ਸੌਫਟਵੇਅਰ ਦਾ ਅੱਪਗਰੇਡ ਅਤੇ ਇੰਸਟਾਲੇਸ਼ਨ ਸਫਲ ਹੋ ਜਾਵੇਗਾ। ਤੁਸੀਂ ਹੇਠਾਂ ਦਿੱਤੀ ਕਮਾਂਡ ਨਾਲ ਇਸਦੀ ਪੁਸ਼ਟੀ ਕਰ ਸਕਦੇ ਹੋ:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## mosh ssh ਨੂੰ ਤੇਜ਼ ਕਰਦਾ ਹੈ

ਯੂਰਪ ਤੋਂ ਚੀਨ ਤੱਕ ਦਾ ਨੈੱਟਵਰਕ ਸਥਿਰ ਨਹੀਂ ਹੈ `ssh` ਤੇਜ਼ ਕਰਨ ਲਈ ਵਰਤਣ ਦੀ ਸਿਫਾਰਸ਼ ਕੀਤੀ ਜਾਂਦੀ ਹੈ [mosh](//mosh.org)

ਲੋਕਲ ਡਾਇਰੈਕਟਰੀ `~/.bin` ਵਿੱਚ `c0` ਸਕ੍ਰਿਪਟਾਂ ਬਣਾਓ (ਮੁੱਖ ਤੌਰ 'ਤੇ ਵਾਤਾਵਰਣ ਵੇਰੀਏਬਲ `PATH` ਵਿੱਚ ਜੋੜਿਆ ਗਿਆ) ਹੇਠਾਂ ਦਿੱਤੇ ਅਨੁਸਾਰ:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

ਬਣਾਉਣ ਤੋਂ ਬਾਅਦ, ਐਗਜ਼ੀਕਿਊਟੇਬਲ ਅਨੁਮਤੀਆਂ ਅਤੇ ਸਾਫਟ-ਲਿੰਕ `c1` ਅਤੇ `c2` ਦਿਓ।

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

ਭਵਿੱਖ ਵਿੱਚ, ਤੁਸੀਂ `mosh` ਹਾਈ-ਸਪੀਡ ਡਾਇਰੈਕਟ ਕਨੈਕਸ਼ਨ ਦੀ ਵਰਤੋਂ ਕਰਨ ਲਈ ਸਿੱਧੇ `c0` ਅਤੇ `c1` ਦਰਜ ਕਰ ਸਕਦੇ ਹੋ ਅਤੇ ਦਾਖਲ ਹੋ ਸਕਦੇ ਹੋ [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

## ਭਾਗ ਫਾਰਮੈਟ ਨੂੰ ext4 ਤੋਂ btrfs ਵਿੱਚ ਬਦਲੋ

ਆਸਾਨ ਬੈਕਅੱਪ ਲਈ ਡਿਸਕ ਸਨੈਪਸ਼ਾਟ ਲੈ ਸਕਦੇ ਹੋ [btrfs](https://wikipedia.org/wiki/Btrfs)

ਪਹਿਲਾਂ ਬਚਾਅ ਪ੍ਰਣਾਲੀ ਵਿੱਚ ਦਾਖਲ ਹੋਵੋ

![](https://i-01.eu.org/1718300448.avif)

ਡੇਬੀਅਨ ਬਚਾਅ ਦੀ ਚੋਣ ਕਰੋ

![](https://i-01.eu.org/1718300753.avif)

ਸਰਵਰ ਦੇ ਮੁੜ ਚਾਲੂ ਹੋਣ ਦੀ ਉਡੀਕ ਕਰੋ

ssh ਬਦਲਾਅ ਹੋਣਗੇ, ਇਸਨੂੰ ਸਾਫ਼ ਕਰਨ ਲਈ `ssh-keygen -R ip` ਵਰਤੋਂ ਕਰੋ

ਫਿਰ ਭਾਗ ਨੂੰ ਬਦਲਣ ਲਈ ਹੇਠ ਦਿੱਤੀ ਸਕ੍ਰਿਪਟ ਚਲਾਓ

```bash
nohup bash -c "bash <(curl -s https://raw.githubusercontent.com/i18n-ops/ops/main/contabo/ext4_btrfs.sh) >/tmp/nohup.out 2>&1 && reboot" &
sleep 1 && tail -f /tmp/nohup.out
```

ਪਰਿਵਰਤਨ ਸਫਲ ਹੋਣ ਤੋਂ ਬਾਅਦ ਸਕ੍ਰਿਪਟ ਆਪਣੇ ਆਪ ਸਿਸਟਮ ਨੂੰ ਮੁੜ ਚਾਲੂ ਕਰੇਗੀ।

### btrfs ਕੰਪਰੈਸ਼ਨ ਅਨੁਪਾਤ ਵੇਖੋ

ਇਹ ਡਿਫੌਲਟ ਰੂਪ ਵਿੱਚ ਸੰਕੁਚਿਤ ਹੋ ਜਾਵੇਗਾ:

```bash
if ! command -v compsize &>/dev/null; then
apt install -y  btrfs-compsize
fi
compsize -x /
```

### ਮੈਨੁਅਲ ਕੰਪਰੈਸ਼ਨ

ਜੇਕਰ ਤੁਸੀਂ ਇੱਕ ਸੰਪੂਰਨਤਾਵਾਦੀ ਹੋ, ਤਾਂ ਤੁਸੀਂ ਇੱਕ ਪੂਰੀ ਡਿਸਕ ਕੰਪਰੈਸ਼ਨ ਨੂੰ ਹੱਥੀਂ ਵੀ ਕਰ ਸਕਦੇ ਹੋ:

```bash
btrfs filesystem defragment -r -czstd /
```

ਜਿਵੇਂ ਕਿ ਹੇਠਾਂ ਚਿੱਤਰ ਵਿੱਚ ਦਿਖਾਇਆ ਗਿਆ ਹੈ, ਤੁਸੀਂ ਦੇਖ ਸਕਦੇ ਹੋ ਕਿ ਨਵੇਂ ਸਿਸਟਮ ਵਿੱਚ ਮੈਨੂਅਲ ਕੰਪਰੈਸ਼ਨ ਤੋਂ ਬਾਅਦ, ਲਗਭਗ 100MB ਹੋਰ ਸੰਕੁਚਿਤ ਕੀਤਾ ਗਿਆ ਸੀ।

![](https://i-01.eu.org/2023/12/8NWlr4c.webp)

### ਸਨੈਪਰ ਨਾਲ ਇੱਕ Btrfs ਸਨੈਪਸ਼ਾਟ ਬੈਕਅੱਪ ਬਣਾਓ

Btrfs ਬੈਕਅੱਪ ਲਈ ਸਭ ਤੋਂ ਵਧੀਆ ਸਾਥੀ ਹੈ [snapper](https://manual.siduction.org/sys-admin-btrfs-snapper_en.html)

ਸ਼ੁਰੂਆਤੀ ਸਕ੍ਰਿਪਟ `~/ubuntu` ਡਾਇਰੈਕਟਰੀ ਬਣਾਏਗੀ, ਬੱਸ ਹੇਠ ਦਿੱਤੀ ਕਮਾਂਡ ਚਲਾਓ:

```
~/ubuntu/snapper/init.sh
```

ਤੁਹਾਨੂੰ ਸਕ੍ਰਿਪਟ ਸਮੱਗਰੀ ਵਿੱਚ ਮਿਲਦੇ ਹਨ [github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh](https://github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh)

ਸਬ-ਵੋਲਿਊਮ ਦੇਖਣ ਲਈ `snapper list-configs` ਵਰਤੋਂ ਕਰੋ ਜਿਨ੍ਹਾਂ ਲਈ ਸਨੈਪਸ਼ਾਟ ਸੈੱਟ ਕੀਤੇ ਗਏ ਹਨ।

![](https://i-01.eu.org/1718302689.avif)

ਸਬ-ਵੋਲਿਊਮ ਦਾ ਸਨੈਪਸ਼ਾਟ ਦੇਖਣ ਲਈ `snapper -c mnt list` ਵਰਤੋਂ ਕਰੋ

`/etc/snapper/configs/mnt` ਇਸ ਸਬ-ਵੋਲਿਊਮ ਦੀਆਂ ਖਾਸ ਬੈਕਅੱਪ ਸੈਟਿੰਗਾਂ ਨੂੰ ਦੇਖ ਸਕਦਾ ਹੈ

ਹੋਰ ਹਵਾਲੇ:

[Btrfs ਦੇ ਸਨੈਪਸ਼ਾਟ ਫੰਕਸ਼ਨ ਨਾਲ ਆਸਾਨੀ ਨਾਲ ਖੇਡੋ snapper](https://zhuanlan.zhihu.com/p/31082518)

btrfs ਸਨੈਪਸ਼ਾਟ ਵਰਤਣਾ ਆਮ ਤੌਰ 'ਤੇ ਵਧੇਰੇ ਸੁਵਿਧਾਜਨਕ ਹੁੰਦਾ ਹੈ।

## ਬਚਾਅ ਪ੍ਰਣਾਲੀ ਦੇ ਵੇਰਵੇ

22 [tcping](https://github.com/paradiseduo/tcping) ਪਹੁੰਚਯੋਗ ਹੋਣ ਤੋਂ ਬਾਅਦ, ਤੁਸੀਂ 2-3 ਮਿੰਟ ਲਈ ਇਸ ਨੂੰ ਐਕਸੈਸ ਕਰਨ ਦੇ ਯੋਗ ਨਹੀਂ ਹੋਵੋਗੇ।

```
tcping ip 22 -c 9999
```

ਪੋਰਟ ਪਹੁੰਚ ਦੀ ਨਿਗਰਾਨੀ ਕੀਤੀ ਜਾ ਸਕਦੀ ਹੈ.

### ਰਿਸਟੋਰ ਸਨੈਪਸ਼ਾਟ ਵਿੱਚ ਦਾਖਲ ਹੋਣ ਤੋਂ ਬਾਅਦ ਹਾਰਡ ਡਰਾਈਵ ਨੂੰ ਕਿਵੇਂ ਐਕਸੈਸ ਕਰਨਾ ਹੈ &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

ਉਦਾਹਰਨ ਲਈ, ਤੁਸੀਂ ਸਨੈਪਸ਼ਾਟ ਦੇਖ ਸਕਦੇ ਹੋ

```
snapper -c etc list
```

ਸਨੈਪਸ਼ਾਟ ਫਾਈਲ ਬਦਲਾਅ ਵੇਖੋ

```
snapper -c etc status 5..6
```

ਖਾਸ ਅੰਤਰ ਟੈਕਸਟ ਵੇਖੋ

```
snapper -c etc diff 3..5
```

ਸਨੈਪਸ਼ਾਟ ਰੀਸਟੋਰ ਕਰੋ: ਉਦਾਹਰਨ ਲਈ, ਸਨੈਪਸ਼ਾਟ 5 ਤੋਂ ਬਾਅਦ ਸਾਰੀਆਂ ਤਬਦੀਲੀਆਂ ਨੂੰ ਅਣਡੂ ਕਰੋ

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### ਲੌਗ ਸਥਿਰਤਾ ਨੂੰ ਸਮਰੱਥ ਬਣਾਓ

ਇੱਕ ਸੰਕਟਕਾਲੀਨ ਸਿਸਟਮ ਦੇ ਅਧੀਨ ਲਾਗ ਸਥਿਰਤਾ ਨੂੰ ਯੋਗ ਕਰਨ ਲਈ ਤੁਹਾਨੂੰ ਸੰਬੰਧਿਤ ਡਾਇਰੈਕਟਰੀ ਨੂੰ ਦਸਤੀ ਬਣਾਉਣ ਅਤੇ ਕੁਝ ਵਾਧੂ ਕਦਮ ਚੁੱਕਣ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ, ਕਿਉਂਕਿ ਤੁਹਾਡੇ ਕੋਲ ਚੱਲ ਰਹੀ systemd ਸੇਵਾ ਨਹੀਂ ਹੈ। ਇੱਥੇ ਉਹ ਕਦਮ ਹਨ ਜੋ ਤੁਸੀਂ ਅਜ਼ਮਾ ਸਕਦੇ ਹੋ:

ਪਹਿਲਾਂ chroot, ਫਿਰ

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

ਆਪਣੇ ਉਬੰਟੂ ਸਿਸਟਮ ਵਿੱਚ ਰੀਬੂਟ ਕਰੋ: ਉਪਰੋਕਤ ਤਬਦੀਲੀਆਂ ਕਰਨ ਤੋਂ ਬਾਅਦ, ਤੁਹਾਨੂੰ ਬਚਾਅ ਸਿਸਟਮ ਦੀ ਬਜਾਏ ਆਪਣੀ ਮਸ਼ੀਨ ਨੂੰ ਆਮ ਤੌਰ 'ਤੇ ਅਤੇ ਆਪਣੇ ਉਬੰਟੂ ਸਿਸਟਮ ਵਿੱਚ ਰੀਬੂਟ ਕਰਨ ਦੀ ਲੋੜ ਹੈ।

ਇਸ ਤਰ੍ਹਾਂ, systemd-journald ਨਵੀਂ ਸੰਰਚਨਾ ਨਾਲ ਸ਼ੁਰੂ ਹੋ ਜਾਵੇਗਾ ਅਤੇ ਹਾਰਡ ਡਿਸਕ 'ਤੇ ਲਗਾਤਾਰ ਲਾਗਾਂ ਨੂੰ ਸਟੋਰ ਕਰਨਾ ਸ਼ੁਰੂ ਕਰ ਦੇਵੇਗਾ।

ਧਿਆਨ ਵਿੱਚ ਰੱਖੋ ਕਿ ਉਪਰੋਕਤ ਓਪਰੇਸ਼ਨ ਇੱਕ ਬਚਾਅ ਸਿਸਟਮ ਤੇ ਕੀਤੇ ਜਾਂਦੇ ਹਨ, ਇਸਲਈ ਉਹ ਤਬਦੀਲੀਆਂ ਉਦੋਂ ਤੱਕ ਪ੍ਰਭਾਵੀ ਨਹੀਂ ਹੋਣਗੀਆਂ ਜਦੋਂ ਤੱਕ ਸਿਸਟਮ ਰੀਬੂਟ ਨਹੀਂ ਹੁੰਦਾ ਅਤੇ ਹੁਣ ਬਚਾਅ ਮੋਡ ਵਿੱਚ ਨਹੀਂ ਹੁੰਦਾ।

ਇਸਦਾ ਮਤਲਬ ਹੈ ਕਿ ਤੁਸੀਂ ਮੌਜੂਦਾ ਲੌਗਸ ਵਿੱਚ ਕੋਈ ਵੀ ਬਦਲਾਅ ਨਹੀਂ ਦੇਖ ਸਕੋਗੇ ਜੋ ਸਥਿਰ ਬਣ ਜਾਂਦੇ ਹਨ, ਪਰ ਉਸ ਤੋਂ ਬਾਅਦ ਤਿਆਰ ਕੀਤੇ ਗਏ ਸਾਰੇ ਲੌਗ ਬਰਕਰਾਰ ਰਹਿਣਗੇ।

ਇਸ ਤਰ੍ਹਾਂ ਤੁਸੀਂ ssh ਸੇਵਾ ਦੇ ਲੌਗਸ ਨੂੰ ਦੇਖ ਸਕਦੇ ਹੋ

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## ਮੁਸ਼ਕਲ ਅਤੇ ਗੁੰਝਲਦਾਰ ਬਿਮਾਰੀਆਂ 'ਤੇ ਨੋਟਸ

### ਅਜੀਬ ਸਮੱਸਿਆ ਨੂੰ ਹੱਲ ਕਰੋ: ਸਰਵਰ ssh ਨਹੀਂ ਕਰ ਸਕਦਾ

ਮੈਂ ਪਹਿਲਾਂ ਇੱਕ ਆਟੋਮੈਟਿਕ ਕੰਪਾਈਲੇਸ਼ਨ ਅਤੇ ਡਿਪਲਾਇਮੈਂਟ ਸਕ੍ਰਿਪਟ ਬਣਾਉਣ ਲਈ ਗੀਥਬ ਐਕਸ਼ਨ ਦੀ ਵਰਤੋਂ ਕੀਤੀ, ਪਰ ਪਾਇਆ ਕਿ ਤੈਨਾਤੀ ਸੇਵਾ ssh ਨਹੀਂ ਹੋ ਸਕਦੀ.

ਬਚਾਅ ਪ੍ਰਣਾਲੀ ਵਿੱਚ ਦਾਖਲ ਹੋਣਾ ਅਤੇ ਲੌਗਸ ਨੂੰ ਦੇਖਣਾ ਇੱਕ ਅਨੁਮਤੀ ਦਾ ਮੁੱਦਾ ਹੈ:

`Missing privilege separation directory: /run/sshd`

ਤੁਸੀਂ ਹੇਠ ਦਿੱਤੀ ਕਮਾਂਡ ਨਾਲ ਇਹ ਡਾਇਰੈਕਟਰੀ ਬਣਾ ਸਕਦੇ ਹੋ

```
chown root:root /
systemd-tmpfiles --create
```

ਵਾਰ-ਵਾਰ ਜਾਂਚ ਤੋਂ ਬਾਅਦ, ਇਹ ਇਸ ਲਈ ਹੈ github action tar ਦਾ ਉਪਭੋਗਤਾ `uid` `1001` ਹੈ

ਜੇਕਰ ਕੋਈ ਪੈਰਾਮੀਟਰ `--no-same-owner` ਨਹੀਂ ਹੈ, ਤਾਂ ਇਸਨੂੰ `1001` ਉਪਭੋਗਤਾਵਾਂ ਲਈ `uid` ਤੱਕ ਡੀਕੰਪ੍ਰੈਸ ਕੀਤਾ ਜਾਵੇਗਾ tar

ਨਤੀਜੇ ਵਜੋਂ, rsync ਰੂਟ ਡਾਇਰੈਕਟਰੀ ਦੇ ਉਪਭੋਗਤਾ ਸਮੂਹ ਨੂੰ ਬਦਲ ਦੇਵੇਗਾ, ਅਤੇ ਫਿਰ ਤੁਸੀਂ ਰੀਸਟਾਰਟ ਕਰਨ ਤੋਂ ਬਾਅਦ ssh ਵਿੱਚ ਲਾਗਇਨ ਨਹੀਂ ਕਰ ਸਕੋਗੇ।

ਡੀਕੰਪ੍ਰੇਸ਼ਨ ਸਕ੍ਰਿਪਟ ਨੂੰ ਹੇਠਾਂ ਅਨੁਸਾਰ ਸੋਧੋ:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
