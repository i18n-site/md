# ਇੰਸਟਾਲ ਕਰੋ ਅਤੇ ਵਰਤੋ

## ਵਿੰਡੋਜ਼ ਨੂੰ ਪਹਿਲਾਂ ਗਿਟ ਬੈਸ਼ ਇੰਸਟਾਲ ਕਰੋ

windows , ਕਿਰਪਾ ਕਰਕੇ [ਪਹਿਲਾਂ ਡਾਊਨਲੋਡ ਅਤੇ ਸਥਾਪਿਤ ਕਰਨ ਲਈ ਇੱਥੇ ਕਲਿੱਕ ਕਰੋ `git bash`](https://git-scm.com/download/win)

ਵਿੱਚ ਅਗਲੇ ਓਪਰੇਸ਼ਨ ਚਲਾਓ `git bash`

## ਇੰਸਟਾਲ ਕਰੋ

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### ਅਨੁਵਾਦ ਟੋਕਨ ਕੌਂਫਿਗਰ ਕਰੋ

ਟੋਕਨ ਨੂੰ ਕਾਪੀ ਕਰਨ ਲਈ ਕਲਿੱਕ ਕਰੋ [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

ਬਣਾਓ `~/.config/i18n.site.yml` ਕਾਪੀ ਕੀਤੀ ਸਮੱਗਰੀ ਨੂੰ ਇਸ ਵਿੱਚ ਪੇਸਟ ਕਰੋ, ਸਮੱਗਰੀ ਇਸ ਤਰ੍ਹਾਂ ਹੈ:

```
token: YOUR_API_TOKEN
```

ਇਸ ਤੋਂ ਇਲਾਵਾ, ਤੁਹਾਨੂੰ ਭੁਗਤਾਨ ਲਈ ਇੱਕ ਕ੍ਰੈਡਿਟ ਕਾਰਡ ਬੰਨ੍ਹਣ [i18n.site/payBill](//i18n.site/payBill) ਲੋੜ ਹੈ (ਕੋਈ ਰੀਚਾਰਜ ਦੀ ਲੋੜ ਨਹੀਂ ਹੈ, ਵੈੱਬਸਾਈਟ ਵਰਤੋਂ ਦੇ ਅਨੁਸਾਰ ਆਪਣੇ ਆਪ ਫੀਸਾਂ ਕੱਟ ਲਵੇਗੀ, [ਕੀਮਤ ਲਈ ਹੋਮਪੇਜ ਦੇਖੋ](/#price) )।

## ਵਰਤੋ

### ਡੈਮੋ ਪ੍ਰੋਜੈਕਟ

`i18` ਦੀ ਸੰਰਚਨਾ ਸਿੱਖਣ ਲਈ ਕਿਰਪਾ ਕਰਕੇ ਡੈਮੋ ਪ੍ਰੋਜੈਕਟ ਵੇਖੋ [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

ਚੀਨ ਵਿੱਚ ਉਪਭੋਗਤਾ ਕਲੋਨ ਕਰ ਸਕਦੇ ਹਨ [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

ਕਲੋਨਿੰਗ ਤੋਂ ਬਾਅਦ, ਡਾਇਰੈਕਟਰੀ ਦਾਖਲ ਕਰੋ ਅਤੇ ਅਨੁਵਾਦ ਨੂੰ ਪੂਰਾ ਕਰਨ ਲਈ ਚਲਾਓ `i18`

### ਡਾਇਰੈਕਟਰੀ ਬਣਤਰ

ਟੈਂਪਲੇਟ ਵੇਅਰਹਾਊਸ ਡਾਇਰੈਕਟਰੀ ਬਣਤਰ ਹੇਠ ਲਿਖੇ ਅਨੁਸਾਰ ਹੈ

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਅਨੁਵਾਦਿਤ ਡੈਮੋ ਫਾਈਲਾਂ ਹਨ, ਜੋ ਸਿਰਫ ਇੱਕ ਉਦਾਹਰਣ ਹਨ ਅਤੇ ਮਿਟਾਈਆਂ ਜਾ ਸਕਦੀਆਂ ਹਨ।

### ਅਨੁਵਾਦ ਚਲਾਓ

ਡਾਇਰੈਕਟਰੀ ਦਰਜ ਕਰੋ ਅਤੇ ਅਨੁਵਾਦ ਕਰਨ ਲਈ ਚਲਾਓ `i18`

### ਰਿਪੋਜ਼ਟਰੀ ਵਿੱਚ ਫਾਈਲਾਂ ਸ਼ਾਮਲ ਕਰੋ

ਅਨੁਵਾਦ ਤੋਂ ਇਲਾਵਾ, ਪ੍ਰੋਗਰਾਮ ਹੇਠ ਲਿਖੀਆਂ ਫਾਈਲਾਂ ਵੀ ਤਿਆਰ ਕਰੇਗਾ, ਕਿਰਪਾ ਕਰਕੇ ਉਹਨਾਂ ਨੂੰ ਰਿਪੋਜ਼ਟਰੀ ਵਿੱਚ ਸ਼ਾਮਲ ਕਰੋ।

```
.i18n/hash
.i18n/cache/.gitignore
```

ਉਹਨਾਂ ਵਿੱਚੋਂ, `.i18n/cache/.gitignore` ਸਮੱਗਰੀ ਹੇਠ ਲਿਖੇ ਅਨੁਸਾਰ ਹੈ :

```
**/*
!**/.gitignore
```

ਇਸਦਾ ਮਤਲਬ ਹੈ ਕਿ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਸਾਰੀਆਂ ਫਾਈਲਾਂ ਨੂੰ ਅਣਡਿੱਠ ਕਰੋ ( `.i18n/cache/.gitignore` ਛੱਡ ਕੇ) `.i18n/cache/`

ਜੇਕਰ ਤੁਹਾਡਾ ਸੰਸਕਰਣ ਕੰਟਰੋਲ ਸਾਫਟਵੇਅਰ ਨਹੀਂ ਹੈ `git` ਤਾਂ ਕਿਰਪਾ ਕਰਕੇ ਇਸ ਸੰਰਚਨਾ ਦੇ ਅਨੁਸਾਰ ਇਸਨੂੰ ਅਣਡਿੱਠ ਕਰੋ।

## ਸੰਰਚਨਾ ਫਾਇਲ

ਇਹ `i18` ਲਾਈਨ ਅਨੁਵਾਦ ਟੂਲ ਦੀ ਸੰਰਚਨਾ ਫਾਇਲ `.i18n/conf.yml`

ਸਮੱਗਰੀ ਹੇਠ ਲਿਖੇ ਅਨੁਸਾਰ ਹੈ:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ਸਰੋਤ ਭਾਸ਼ਾ &

ਸੰਰਚਨਾ ਫਾਇਲ ਵਿੱਚ, ਦੇ ਅਧੀਨ ਹਨ `fromTo`

ਸਰੋਤ ਭਾਸ਼ਾ ਹੈ, `zh ja ko de fr` ਅਨੁਵਾਦ ਦੀ ਲਕਸ਼ ਭਾਸ਼ਾ ਹੈ `en`

ਭਾਸ਼ਾ ਕੋਡ ਵੇਖੋ [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

ਉਦਾਹਰਨ ਲਈ, ਜੇਕਰ ਤੁਸੀਂ ਚੀਨੀ ਨੂੰ ਅੰਗਰੇਜ਼ੀ ਵਿੱਚ ਅਨੁਵਾਦ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ ਇਸ ਲਾਈਨ ਨੂੰ ਦੁਬਾਰਾ ਲਿਖੋ `zh: en`

ਜੇਕਰ ਤੁਸੀਂ ਸਾਰੀਆਂ ਸਮਰਥਿਤ ਭਾਸ਼ਾਵਾਂ ਵਿੱਚ ਅਨੁਵਾਦ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ ਕਿਰਪਾ ਕਰਕੇ ਬਾਅਦ ਵਿੱਚ ਖਾਲੀ ਛੱਡੋ `:` ਉਦਾਹਰਣ ਲਈ

```
i18n:
  fromTo:
    en:
```

### ਫਾਈਲ ਨੂੰ ਅਣਡਿੱਠ ਕਰੋ

ਮੂਲ ਰੂਪ ਵਿੱਚ, ਸਰੋਤ ਭਾਸ਼ਾ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ `.md` ਅਤੇ `.yml` ਨਾਲ ਸ਼ੁਰੂ ਹੋਣ ਵਾਲੀਆਂ ਸਾਰੀਆਂ ਫਾਈਲਾਂ ਦਾ ਅਨੁਵਾਦ ਕੀਤਾ ਜਾਵੇਗਾ।

ਜੇ ਤੁਸੀਂ ਕੁਝ ਫਾਈਲਾਂ ਨੂੰ ਅਣਡਿੱਠ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ ਅਤੇ ਉਹਨਾਂ ਦਾ ਅਨੁਵਾਦ ਨਹੀਂ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ (ਜਿਵੇਂ ਕਿ ਅਧੂਰੇ ਡਰਾਫਟ), ਤੁਸੀਂ ਫੀਲਡ ਸੰਰਚਨਾ ਦੀ ਵਰਤੋਂ ਕਰ ਸਕਦੇ ਹੋ `ignore`

`.gitignore` ਵਰਗਾ ਸੰਟੈਕਸ ਵਰਤਦਾ [globset](https://docs.rs/globset/latest/globset/#syntax) `ignore`

ਉਦਾਹਰਨ ਲਈ, ਉਪਰੋਕਤ ਸੰਰਚਨਾ ਫਾਈਲ ਵਿੱਚ `_* ` ਦਾ ਮਤਲਬ ਹੈ ਕਿ `_` ਨਾਲ ਸ਼ੁਰੂ ਹੋਣ ਵਾਲੀਆਂ ਫਾਈਲਾਂ ਦਾ ਅਨੁਵਾਦ ਨਹੀਂ ਕੀਤਾ ਜਾਵੇਗਾ।

## ਅਨੁਵਾਦ ਦੇ ਨਿਯਮ

### ਅਨੁਵਾਦ ਸੰਪਾਦਕਾਂ ਨੂੰ ਲਾਈਨਾਂ ਨੂੰ ਜੋੜਨਾ ਜਾਂ ਮਿਟਾਉਣਾ ਨਹੀਂ ਚਾਹੀਦਾ

ਅਨੁਵਾਦ ਸੰਪਾਦਨਯੋਗ ਹੈ। ਮੂਲ ਪਾਠ ਨੂੰ ਸੋਧੋ ਅਤੇ ਮਸ਼ੀਨ-ਇਸ ਨੂੰ ਦੁਬਾਰਾ ਅਨੁਵਾਦ ਕਰੋ, ਅਨੁਵਾਦ ਲਈ ਦਸਤੀ ਸੋਧਾਂ ਨੂੰ ਓਵਰਰਾਈਟ ਨਹੀਂ ਕੀਤਾ ਜਾਵੇਗਾ (ਜੇਕਰ ਮੂਲ ਪਾਠ ਦੇ ਇਸ ਪੈਰੇ ਨੂੰ ਸੋਧਿਆ ਨਹੀਂ ਗਿਆ ਹੈ)।

ਪਰ ਕਿਰਪਾ ਕਰਕੇ ਧਿਆਨ ਦਿਓ ਕਿ ਅਨੁਵਾਦ ਅਤੇ ਮੂਲ ਪਾਠ ਦੀਆਂ ਲਾਈਨਾਂ ਇੱਕ ਤੋਂ ਇੱਕ ਹੋਣੀਆਂ ਚਾਹੀਦੀਆਂ ਹਨ। ਭਾਵ, ਅਨੁਵਾਦ ਨੂੰ ਕੰਪਾਇਲ ਕਰਦੇ ਸਮੇਂ ਲਾਈਨਾਂ ਨੂੰ ਨਾ ਜੋੜੋ ਅਤੇ ਨਾ ਹੀ ਮਿਟਾਓ। ਨਹੀਂ ਤਾਂ, ਇਹ ਅਨੁਵਾਦ ਸੰਪਾਦਨ ਕੈਸ਼ ਵਿੱਚ ਉਲਝਣ ਪੈਦਾ ਕਰੇਗਾ।

ਜੇਕਰ ਕੁਝ ਗਲਤ ਹੋ ਜਾਂਦਾ ਹੈ, ਤਾਂ ਕਿਰਪਾ ਕਰਕੇ [ਹੱਲ ਲਈ ਅਕਸਰ ਪੁੱਛੇ ਜਾਂਦੇ ਸਵਾਲ ਵੇਖੋ।](/i18/qa#H1)

### `YAML` ਅਨੁਵਾਦ

ਕਮਾਂਡ ਲਾਈਨ ਟੂਲ ਸਰੋਤ ਭਾਸ਼ਾ ਫਾਈਲ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਖਤਮ ਹੋਣ `.yml` ਸਾਰੀਆਂ ਫਾਈਲਾਂ ਨੂੰ ਲੱਭੇਗਾ ਅਤੇ ਉਹਨਾਂ ਦਾ ਅਨੁਵਾਦ ਕਰੇਗਾ।

* ਨੋਟ ਕਰੋ ਕਿ ਫਾਈਲ ਨਾਮ ਪਿਛੇਤਰ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ ( `.yaml` ਨਹੀਂ) `.yml`

ਟੂਲ ਸਿਰਫ਼ ਸ਼ਬਦਕੋਸ਼ ਮੁੱਲਾਂ ਦਾ ਅਨੁਵਾਦ ਕਰਦਾ ਹੈ, ਸ਼ਬਦਕੋਸ਼ ਦੀਆਂ ਕੁੰਜੀਆਂ ਦਾ ਨਹੀਂ `.yml`

ਉਦਾਹਰਣ ਲਈ `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

ਦੇ ਰੂਪ ਵਿੱਚ ਅਨੁਵਾਦ ਕੀਤਾ ਜਾਵੇਗਾ `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` ਅਨੁਵਾਦ ਹੱਥੀਂ ਵੀ ਸੰਸ਼ੋਧਿਤ ਕੀਤਾ ਜਾ ਸਕਦਾ ਹੈ (ਪਰ ਅਨੁਵਾਦ ਵਿੱਚ ਕੁੰਜੀਆਂ ਜਾਂ ਲਾਈਨਾਂ ਨਾ ਜੋੜੋ)।

ਅਨੁਵਾਦ `YAML` ਆਧਾਰ 'ਤੇ, ਤੁਸੀਂ ਵੱਖ-ਵੱਖ ਪ੍ਰੋਗਰਾਮਿੰਗ ਭਾਸ਼ਾਵਾਂ ਲਈ ਆਸਾਨੀ ਨਾਲ ਅੰਤਰਰਾਸ਼ਟਰੀ ਹੱਲ ਤਿਆਰ ਕਰ ਸਕਦੇ ਹੋ।

## ਉੱਨਤ ਵਰਤੋਂ

### ਅਨੁਵਾਦ ਉਪ-ਡਾਇਰੈਕਟਰੀ

ਜਿੰਨਾ ਚਿਰ ਤੁਸੀਂ ਬਣਾਉਂਦੇ ਹੋ `.i18n/conf.yml` (ਹਰ ਵਾਰ ਡੈਮੋ ਪ੍ਰੋਜੈਕਟ ਟੈਂਪਲੇਟ ਤੋਂ ਸ਼ੁਰੂ ਕਰਨ ਦੀ ਲੋੜ ਨਹੀਂ), `i18` ਵਧੀਆ ਕੰਮ ਕਰੇਗਾ।

ਕਮਾਂਡ ਲਾਈਨ ਟੂਲ ਸਾਰੀਆਂ ਸਬ-ਡਾਇਰੈਕਟਰੀਆਂ ਵਿੱਚ `.i18n/conf.yml` ਲੱਭੇਗਾ ਅਤੇ ਇਸਦਾ ਅਨੁਵਾਦ ਕਰੇਗਾ।

[monorepo](//monorepo.tools) ਦੀ ਵਰਤੋਂ ਕਰਨ ਵਾਲੇ ਪ੍ਰੋਜੈਕਟ ਭਾਸ਼ਾ ਦੀਆਂ ਫਾਈਲਾਂ ਨੂੰ ਸਬ-ਡਾਇਰੈਕਟਰੀਆਂ ਵਿੱਚ ਵੰਡ ਸਕਦੇ ਹਨ।

<img alt="" src="https://p.3ti.site/1719910016.avif">

### ਕਸਟਮ ਇੰਸਟਾਲੇਸ਼ਨ ਡਾਇਰੈਕਟਰੀ

ਇਸ ਨੂੰ ਮੂਲ ਰੂਪ ਵਿੱਚ ਇੰਸਟਾਲ ਕੀਤਾ ਜਾਵੇਗਾ `/usr/local/bin`

ਜੇਕਰ ਲਿਖਣ ਦੀ ਇਜਾਜ਼ਤ ਨਹੀਂ ਹੈ `/usr/local/bin` ਇਸ ਨੂੰ `~/.bin` 'ਤੇ ਇੰਸਟਾਲ ਕੀਤਾ ਜਾਵੇਗਾ।

ਵਾਤਾਵਰਣ ਵੇਰੀਏਬਲ ਸੈੱਟ `TO` ਤੁਸੀਂ ਇੰਸਟਾਲੇਸ਼ਨ ਡਾਇਰੈਕਟਰੀ ਨੂੰ ਪਰਿਭਾਸ਼ਿਤ ਕਰ ਸਕਦੇ ਹੋ, ਉਦਾਹਰਨ ਲਈ :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
