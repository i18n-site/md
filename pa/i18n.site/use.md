# ਇੰਸਟਾਲ ਕਰੋ &

## ਸੰਰਚਨਾ ਟੋਕਨ

`i18n.site` ਕੋਲ ਇੱਕ ਬਿਲਟ-ਇਨ `i18` ਅਨੁਵਾਦ ਟੂਲ ਹੈ ਕਿਰਪਾ ਕਰਕੇ [ਐਕਸੈਸ ਟੋਕਨ ਨੂੰ ਕੌਂਫਿਗਰ ਕਰਨ ਲਈ `i18` ਦਸਤਾਵੇਜ਼ ਦਾ ਹਵਾਲਾ ਦੇਣ ਲਈ ਇੱਥੇ ਕਲਿੱਕ ਕਰੋ](/i18/use) ।

## ਇੰਸਟਾਲ ਕਰੋ

```sh
bash <(curl -sS https://i.i18n.site) i18n.site
```

## ਡੈਮੋ ਪ੍ਰੋਜੈਕਟ

ਆਉ `i18n.site` ਵਰਤੋਂ ਕਰਨ ਬਾਰੇ ਸਿੱਖਣ ਲਈ ਇੱਕ ਡੈਮੋ ਪ੍ਰੋਜੈਕਟ ਨਾਲ ਸ਼ੁਰੂ ਕਰੀਏ।

ਅਸੀਂ ਪਹਿਲਾਂ ਡੈਮੋ ਰਿਪੋਜ਼ਟਰੀ ਨੂੰ ਕਲੋਨ ਕਰਦੇ ਹਾਂ ਅਤੇ ਹੇਠ ਲਿਖੇ ਅਨੁਸਾਰ ਕਮਾਂਡ ਚਲਾਉਂਦੇ ਹਾਂ:

```
git clone https://github.com/i18n-site/demo.i18n.site.git md
git clone https://github.com/i18n-site/demo.i18n.site.docker.git docker
```

ਮੁੱਖ ਭੂਮੀ ਚੀਨ ਵਿੱਚ ਉਪਭੋਗਤਾ ਇਹ ਕਰ ਸਕਦੇ ਹਨ:

```
git clone https://atomgit.com/i18n/demo.i18n.site.git md
git clone https://atomgit.com/i18n/demo.i18n.site.docker.git docker
```

`demo.i18n.site` ਕੋਡ ਬੇਸ ਕਲੋਨ ਦਾ ਡਾਇਰੈਕਟਰੀ ਨਾਮ `docker` ਨਾਲ ਸਥਾਨਕ ਪੂਰਵਦਰਸ਼ਨ ਦੀ ਸਹੂਲਤ ਲਈ `md` ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ।

### ਸਮੱਗਰੀ ਪੋਸਟ ਕਰੋ

`i18n.site` ਇੱਕ [ਸਿੰਗਲ-ਪੇਜ ਐਪਲੀਕੇਸ਼ਨ](https://developer.mozilla.org/docs/Glossary/SPA) ਆਰਕੀਟੈਕਚਰ ਨੂੰ ਅਪਣਾਉਂਦਾ ਹੈ, ਅਤੇ ਵੈਬਸਾਈਟ ਪ੍ਰਵੇਸ਼ ਪੰਨਾ ਅਤੇ ਵੈਬਸਾਈਟ ਸਮੱਗਰੀ ਸੁਤੰਤਰ ਤੌਰ 'ਤੇ ਤਾਇਨਾਤ ਕੀਤੀ ਜਾਂਦੀ ਹੈ।

ਉਪਰੋਕਤ ਅਨੁਵਾਦ ਨੂੰ ਚਲਾਉਣ ਤੋਂ ਬਾਅਦ, ਡਾਇਰੈਕਟਰੀਆਂ `htm` ਅਤੇ `v` `md/out/dev` ਡਾਇਰੈਕਟਰੀ ਦੇ ਅਧੀਨ ਤਿਆਰ ਕੀਤੀਆਂ ਜਾਣਗੀਆਂ।

ਇੱਥੇ, `dev` ਦਾ ਮਤਲਬ ਹੈ ਕਿ ਇਹ `.i18n/htm/dev.yml` ਸੰਰਚਨਾ ਫਾਈਲ ਦੇ ਅਧਾਰ ਤੇ ਬਣਾਇਆ ਗਿਆ ਹੈ।

`dev` ਡਾਇਰੈਕਟਰੀ :

`htm` ਡਾਇਰੈਕਟਰੀ ਵੈੱਬਸਾਈਟ ਦਾ ਪ੍ਰਵੇਸ਼ ਪੰਨਾ ਹੈ।

`v` ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਵਰਜਨ ਨੰਬਰਾਂ ਵਾਲੀ ਵੈੱਬਸਾਈਟ ਸਮੱਗਰੀ ਸ਼ਾਮਲ ਹੈ।

ਸਥਾਨਕ ਪ੍ਰੀਵਿਊ ਵਰਜਨ ਨੰਬਰ ਦੀ ਪਰਵਾਹ ਨਹੀਂ ਕਰਦਾ ਹੈ ਅਤੇ ਸਾਰੀਆਂ ਫਾਈਲਾਂ ਨੂੰ `out/dev/v/0.1.0` ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਕਾਪੀ ਕਰੇਗਾ।

ਅਧਿਕਾਰਤ ਰੀਲੀਜ਼ ਲਈ, ਬਦਲੀਆਂ ਗਈਆਂ ਫਾਈਲਾਂ ਨੂੰ ਨਵੇਂ ਸੰਸਕਰਣ ਨੰਬਰ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਕਾਪੀ ਕੀਤਾ ਜਾਵੇਗਾ।

#### ਸੰਰਚਨਾ ਫਾਇਲ ਦੇਣ ਲਈ -c ਦੀ ਵਰਤੋਂ ਕਰੋ

ਵੱਖ-ਵੱਖ ਸੰਰਚਨਾ ਫਾਈਲਾਂ `out` ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਸੰਬੰਧਿਤ ਡਾਇਰੈਕਟਰੀਆਂ ਬਣਾਉਣਗੀਆਂ।

ਉਦਾਹਰਨ ਲਈ, `.i18n/htm/ol.yml` `out/ol` ਡਾਇਰੈਕਟਰੀ ਬਣਾਏਗਾ।

`dev.yml` ਅਤੇ `ol.yml` ਡਿਫੌਲਟ ਸੰਰਚਨਾ ਹਨ।

`dev` `development` ਦਾ ਸੰਖੇਪ ਰੂਪ ਹੈ, ਵਿਕਾਸ ਵਾਤਾਵਰਣ ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ, ਸਥਾਨਕ ਪ੍ਰੀਵਿਊ ਲਈ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ, ਅਤੇ ਇਹ ਡਿਫੌਲਟ ਸੰਰਚਨਾ ਫਾਈਲ ਵੀ ਹੈ।
`ol` `online` ਦਾ ਸੰਖੇਪ ਰੂਪ ਹੈ, ਜੋ ਕਿ ਔਨਲਾਈਨ ਵਾਤਾਵਰਣ ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ, ਜੋ ਕਿ ਅਧਿਕਾਰਤ ਰੀਲੀਜ਼ ਲਈ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ, ਜਦੋਂ ਇਹ ਕਮਾਂਡ ਲਾਈਨ ਪੈਰਾਮੀਟਰ `-n` ਤੋਂ `npm` ਜਾਰੀ ਕਰਨ ਲਈ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ।

ਤੁਸੀਂ ਦੂਜੀਆਂ ਸੰਰਚਨਾ ਫਾਈਲਾਂ ਵੀ ਬਣਾ ਸਕਦੇ ਹੋ ਜਿਸਦੀ ਵਰਤੋਂ ਕਰਨ ਲਈ ਸੰਰਚਨਾ ਫਾਈਲ ਨਾਮ ਨਿਰਧਾਰਤ ਕਰਨ ਲਈ ਕਮਾਂਡ ਲਾਈਨ 'ਤੇ `--htm_conf` ਦੀ ਵਰਤੋਂ ਕਰੋ:

ਉਦਾਹਰਣ ਲਈ:
```
i18n.site --htm_conf yourConfig --save
```

ਇੱਥੇ `--save` ਅਪਡੇਟ ਰੀਲੀਜ਼ ਸੰਸਕਰਣ ਨੰਬਰ ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ।

#### <a rel=id href="#npm" id="npm"></a> ਸਮੱਗਰੀ ਨੂੰ npmjs.com 'ਤੇ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰੋ

[npmjs.com](//npmjs.com) ਤੇ ਸਮੱਗਰੀ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰਨਾ ਸਿਫ਼ਾਰਸ਼ ਕੀਤਾ ਡਿਫੌਲਟ ਹੱਲ ਹੈ (ਦੇਖੋ [ਫਰੰਟ-ਐਂਡ ਉੱਚ ਉਪਲਬਧਤਾ](/i18n.site/feature#ha) )।

##### ਜਾਰੀ npm login &

`nodejs` ਇੰਸਟਾਲ ਕਰੋ, `npm login` ਨਾਲ ਲਾਗਇਨ ਕਰੋ।

`md/.i18n/htm/ol.yml` ਸੰਪਾਦਿਤ ਕਰੋ ਅਤੇ ਆਪਣੇ ਖੁਦ ਦੇ `npm` ਪੈਕੇਜ ਨਾਮ ਵਿੱਚ `i18n.site` ਨੂੰ `v: //unpkg.com/i18n.site` ਵਿੱਚ ਬਦਲੋ।

ਬਸ 'ਤੇ ਖਾਲੀ ਪੈਕੇਜ ਨਾਮ ਦੀ ਵਰਤੋਂ ਕਰੋ [npmjs.com](//npmjs.com)

ਪੈਕੇਜ `npm` ਦੇ ਆਧਾਰ 'ਤੇ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰਦੇ ਸਮੇਂ, `v:` ਮੁੱਲ ਦੇ ਅਗੇਤਰ **ਲਈ `//unpkg.com/` ਵਰਤੋਂ ਕਰਨਾ ਯਕੀਨੀ ਬਣਾਓ** `i18n.site` ਨੇ ਨਵੇਂ ਰੀਲੀਜ਼ਾਂ ਨੂੰ ਸਮੇਂ ਸਿਰ ਦੇਖਣ ਲਈ ਇਸ ਅਗੇਤਰ ਮਾਰਗ ਦੇ ਅਧੀਨ `/.v` ਦੇ ਕੈਸ਼ ਸਮੇਂ ਨੂੰ ਵਿਸ਼ੇਸ਼ ਤੌਰ 'ਤੇ ਅਨੁਕੂਲਿਤ ਕੀਤਾ ਹੈ।

ਅਨੁਵਾਦ ਅਤੇ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰਨ ਲਈ `md` ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ `i18n.site --npm` ਜਾਂ `i18n.site -n` ਚਲਾਓ।

ਜੇਕਰ ਤੁਸੀਂ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰਨ ਲਈ ਇੱਕ ਨਿਰੰਤਰ ਏਕੀਕਰਣ ਵਾਤਾਵਰਣ ਦੀ ਵਰਤੋਂ ਕਰਦੇ ਹੋ, ਤਾਂ `nodejs` ਸਥਾਪਿਤ ਕਰਨ ਦੀ ਕੋਈ ਲੋੜ ਨਹੀਂ ਹੈ। ਬਸ ਲੌਗ-ਇਨ ਅਤੇ ਪ੍ਰਕਾਸ਼ਨ ਅਨੁਮਤੀਆਂ `~/.npmrc` ਨੂੰ ਵਾਤਾਵਰਣ ਵਿੱਚ ਕਾਪੀ ਕਰੋ।

ਜੇਕਰ ਤੁਸੀਂ `ol.yml` ਵਿੱਚ `v:` ਦੇ ਪੈਕੇਜ ਨਾਮ ਨੂੰ ਸੋਧਦੇ ਹੋ, ਤਾਂ ਕਿਰਪਾ ਕਰਕੇ **ਪਹਿਲਾਂ `.i18n/v/ol` ਮਿਟਾਉਣਾ ਯਕੀਨੀ ਬਣਾਓ** ਅਤੇ ਫਿਰ ਇਸਨੂੰ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰੋ।

##### npm ਦੁਆਰਾ ਪ੍ਰਕਾਸ਼ਿਤ ਪ੍ਰੌਕਸੀ ਸਰਵਰ

ਜੇਕਰ ਮੁੱਖ ਭੂਮੀ ਚੀਨ ਵਿੱਚ ਉਪਭੋਗਤਾਵਾਂ ਨੂੰ ਨੈਟਵਰਕ ਸਮੱਸਿਆਵਾਂ ਦਾ ਸਾਹਮਣਾ ਕਰਨਾ ਪੈਂਦਾ ਹੈ ਅਤੇ ਉਹ `npm` ਪੈਕੇਜ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰਨ ਵਿੱਚ ਅਸਮਰੱਥ ਹਨ, ਤਾਂ ਉਹ ਪ੍ਰੌਕਸੀ ਸਰਵਰ ਨੂੰ ਕੌਂਫਿਗਰ ਕਰਨ ਲਈ ਵਾਤਾਵਰਣ ਵੇਰੀਏਬਲ `https_proxy` ਸੈਟ ਕਰ ਸਕਦੇ ਹਨ।

ਇਹ ਮੰਨ ਕੇ ਕਿ ਤੁਹਾਡਾ ਪ੍ਰੌਕਸੀ ਸਰਵਰ ਪੋਰਟ `7890` ਹੈ, ਤੁਸੀਂ ਲਿਖ ਸਕਦੇ ਹੋ:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### ਸਵੈ-ਹੋਸਟ ਕੀਤੀ ਸਮੱਗਰੀ

ਜੇਕਰ ਤੁਸੀਂ ਸਮੱਗਰੀ ਨੂੰ ਸਵੈ-ਹੋਸਟ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ ਪਹਿਲਾਂ `md/.i18n/htm/ol.yml` ਸੰਪਾਦਿਤ ਕਰੋ ਅਤੇ `v: //unpkg.com/i18n.site` ਆਪਣੇ URL ਅਗੇਤਰ ਵਿੱਚ ਬਦਲੋ, ਜਿਵੇਂ ਕਿ `v: //i18n-v.xxx.com` ।

`md` ਡਾਇਰੈਕਟਰੀ ਦਰਜ ਕਰੋ ਅਤੇ ਚਲਾਓ

```
i18n.site --htm_conf ol --save
```

ਜਾਂ ਸੰਖੇਪ

```
i18n.site -c ol -s
```

ਫਿਰ, `md/out/ol/v` ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਸਮੱਗਰੀ ਨੂੰ `v:` ਵਿੱਚ ਸੈੱਟ ਕੀਤੇ URL ਪ੍ਰੀਫਿਕਸ ਮਾਰਗ ਲਈ ਸੰਰਚਿਤ ਕਰੋ।

ਅੰਤ ਵਿੱਚ, **`/.v` ਤੋਂ `1s` ਵਿੱਚ ਖਤਮ ਹੋਣ ਵਾਲੇ ਮਾਰਗ ਦਾ ਕੈਸ਼ ਸਮਾਂ ਕੌਂਫਿਗਰ ਕਰੋ** , ਨਹੀਂ ਤਾਂ ਨਵੀਂ ਜਾਰੀ ਕੀਤੀ ਸਮੱਗਰੀ ਨੂੰ ਤੁਰੰਤ ਐਕਸੈਸ ਨਹੀਂ ਕੀਤਾ ਜਾ ਸਕਦਾ ਹੈ।

ਬੇਲੋੜੀਆਂ ਬੇਨਤੀਆਂ ਨੂੰ ਘਟਾਉਣ ਲਈ ਹੋਰ ਮਾਰਗਾਂ ਲਈ ਕੈਸ਼ ਸਮਾਂ ਇੱਕ ਸਾਲ ਜਾਂ ਵੱਧ ਸੈੱਟ ਕੀਤਾ ਜਾ ਸਕਦਾ ਹੈ।

##### s3 ਲਈ ਸਮੱਗਰੀ ਦੀ ਮੇਜ਼ਬਾਨੀ ਕਰੋ

ਸਵੈ-ਹੋਸਟ ਸਮੱਗਰੀ ਲਈ, ਤੁਹਾਡੇ ਆਪਣੇ ਸਰਵਰ ਦੀ ਵਰਤੋਂ ਕਰਨ ਤੋਂ ਇਲਾਵਾ, `S3` `CDN` ਦੀ ਵਰਤੋਂ ਕਰਨਾ ਇੱਕ ਹੋਰ ਆਮ ਵਿਕਲਪ ਹੈ +

ਤੁਸੀਂ `S3` ਸਰਵਰ ਵਿੱਚ ਲੌਗਇਨ ਕਰਨ ਲਈ ਵਰਤ ਸਕਦੇ [rclone](https://rclone.org) , ਫਿਰ ਹੇਠਾਂ ਦਿੱਤੀ ਸਕ੍ਰਿਪਟ ਨੂੰ ਵੇਖੋ ਅਤੇ ਸੋਧੋ, ਅਤੇ ਹਰੇਕ ਰੀਲੀਜ਼ ਲਈ ਸਿਰਫ `S3` ਵਿੱਚ ਵਾਧੇ ਵਾਲੇ ਬਦਲਾਅ ਦੀ ਨਕਲ ਕਰੋ।

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

`CDN` ਕੌਂਫਿਗਰ ਕਰਨਾ ਯਾਦ ਰੱਖੋ ਤਾਂ ਕਿ `/.v` ਵਿੱਚ ਖਤਮ ਹੋਣ ਵਾਲੇ ਮਾਰਗ ਦਾ ਕੈਸ਼ ਸਮਾਂ `1s` ਹੋਵੇ, ਨਹੀਂ ਤਾਂ ਨਵੀਂ ਜਾਰੀ ਕੀਤੀ ਸਮੱਗਰੀ ਨੂੰ ਤੁਰੰਤ ਐਕਸੈਸ ਨਹੀਂ ਕੀਤਾ ਜਾ ਸਕਦਾ।

### ਵੈੱਬਸਾਈਟ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰੋ

ਵੈੱਬਸਾਈਟ ਨੂੰ ਕਿਤੇ ਵੀ ਲਗਾਇਆ ਜਾ ਸਕਦਾ ਹੈ [github page](https://pages.github.com) ਅਤੇ [cloudflare page](https://pages.cloudflare.com) ਵਧੀਆ ਵਿਕਲਪ ਹਨ।

ਕਿਉਂਕਿ ਵੈੱਬਸਾਈਟ ਇੱਕ [ਸਿੰਗਲ-ਪੰਨੇ ਐਪਲੀਕੇਸ਼ਨ](https://developer.mozilla.org/docs/Glossary/SPA) ਆਰਕੀਟੈਕਚਰ ਦੀ ਵਰਤੋਂ ਕਰਦੀ ਹੈ, ਯਾਦ ਰੱਖੋ ਕਿ URL ਪਾਥਾਂ ਨੂੰ ਮੁੜ ਲਿਖਣਾ ਹੈ ਜਿਸ ਵਿੱਚ `. ` ਤੋਂ `index.html` ਸ਼ਾਮਲ ਨਹੀਂ ਹਨ।

ਵੈੱਬਸਾਈਟ ਐਂਟਰੀ ਪੇਜ ਨੂੰ ਸਿਰਫ਼ ਇੱਕ ਵਾਰ ਤੈਨਾਤ ਕਰਨ ਦੀ ਲੋੜ ਹੈ, ਅਤੇ ਬਾਅਦ ਵਿੱਚ ਸਮੱਗਰੀ ਅੱਪਡੇਟ ਲਈ ਵੈੱਬਸਾਈਟ ਐਂਟਰੀ ਪੰਨੇ ਨੂੰ ਮੁੜ ਤੈਨਾਤ ਕਰਨ ਦੀ ਕੋਈ ਲੋੜ ਨਹੀਂ ਹੈ।

#### github ਪੰਨੇ 'ਤੇ ਤੈਨਾਤ ਕਰੋ

[ਇੱਕ ਸੰਗਠਨ ਬਣਾਉਣ ਲਈ ਪਹਿਲਾਂ ਇੱਥੇ ਕਲਿੱਕ github](https://github.com/account/organizations/new?plan=free) `i18n-demo`

ਫਿਰ ਇਸ ਸੰਗਠਨ ਦੇ ਅਧੀਨ ਵੇਅਰਹਾਊਸ `i18n-demo.github.io` ਬਣਾਓ (ਕਿਰਪਾ ਕਰਕੇ `i18n-demo` ਤੁਹਾਡੇ ਦੁਆਰਾ ਬਣਾਏ ਗਏ ਸੰਗਠਨ ਦੇ ਨਾਮ ਨਾਲ ਬਦਲੋ):

![](https://p.3ti.site/1721098657.avif)

ਪਿਛਲੇ ਲੇਖ ਵਿੱਚ ਸਮੱਗਰੀ ਨੂੰ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰਦੇ ਸਮੇਂ, `out/ol/htm` ਤਿਆਰ ਕੀਤਾ ਗਿਆ ਹੈ, ਕਿਰਪਾ ਕਰਕੇ ਇਸ ਡਾਇਰੈਕਟਰੀ ਨੂੰ ਦਾਖਲ ਕਰੋ ਅਤੇ ਚਲਾਓ :

```
ln -s index.html 404.html
```

ਕਿਉਂਕਿ `github page` URL ਪਾਥ ਰੀਰਾਈਟਿੰਗ ਦਾ ਸਮਰਥਨ ਨਹੀਂ ਕਰਦਾ, ਇਸਦੀ ਬਜਾਏ `404.html` ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ।

ਫਿਰ `htm` ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਹੇਠ ਲਿਖੀ ਕਮਾਂਡ ਚਲਾਓ ( `i18n-demo/i18n-demo.github.io.git` ਆਪਣੇ ਖੁਦ ਦੇ ਵੇਅਰਹਾਊਸ ਐਡਰੈੱਸ ਨਾਲ ਬਦਲਣਾ ਯਾਦ ਰੱਖੋ) :

ਉਪਰੋਕਤ ਤਸਵੀਰ ਵਿੱਚ ਨਿਯਮ ਹੇਠ ਲਿਖੇ ਅਨੁਸਾਰ ਹਨ ਕਿਰਪਾ ਕਰਕੇ ਹੇਠਾਂ ਦਿੱਤੀ ਪਹਿਲੀ ਲਾਈਨ ਵਿੱਚ `i18n.site` ਤੁਹਾਡੇ ਦੁਆਰਾ ਬੰਨ੍ਹੇ ਹੋਏ ਡੋਮੇਨ ਨਾਮ ਨਾਲ ਬਦਲੋ।

```
(http.host in {"i18n.site"}) and not (
substring(http.request.uri.path,-3) in {".js" ".gz"} or
substring(http.request.uri.path,-4) in {".htm" ".rss" ".css" ".svg" ".ico" ".png" ".xml" ".txt"} or
substring(http.request.uri.path,-5) in {".html" ".avif" ".json"} or
ends_with(http.request.uri.path,".webmanifest")
)
```

ਇਸ ਤੋਂ ਇਲਾਵਾ, ਕਿਰਪਾ ਕਰਕੇ ਕੈਸ਼ ਨਿਯਮਾਂ ਨੂੰ ਕੌਂਫਿਗਰ ਕਰੋ, ਜਿਵੇਂ ਕਿ ਹੇਠਾਂ ਦਿਖਾਇਆ ਗਿਆ ਹੈ, ਅਤੇ ਕੈਸ਼ ਦੀ ਮਿਆਦ ਨੂੰ ਇੱਕ ਮਹੀਨੇ 'ਤੇ ਸੈੱਟ ਕਰੋ।

ਕਿਰਪਾ ਕਰਕੇ ਉਪਰੋਕਤ ਤਸਵੀਰ ਵਿੱਚ ਦੂਜੇ ਪੜਾਅ ਵਿੱਚ ਮੇਲ ਖਾਂਦੇ ਡੋਮੇਨ ਨਾਮ ਨੂੰ ਤੁਹਾਡੇ ਦੁਆਰਾ ਬੰਨ੍ਹੇ ਹੋਏ ਡੋਮੇਨ ਨਾਮ ਵਿੱਚ ਬਦਲੋ।

#### ਮੁੱਖ ਭੂਮੀ ਚੀਨ ਵਿੱਚ ਵੈੱਬਸਾਈਟ ਤੈਨਾਤੀ ਨੂੰ ਅਨੁਕੂਲ ਬਣਾਉਣਾ

ਜੇਕਰ ਤੁਸੀਂ ਮੁੱਖ ਭੂਮੀ ਚੀਨ ਦੇ ਨੈੱਟਵਰਕ ਵਾਤਾਵਰਣ ਵਿੱਚ ਬਿਹਤਰ ਪਹੁੰਚਯੋਗਤਾ ਪ੍ਰਦਰਸ਼ਨ ਪ੍ਰਾਪਤ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ ਕਿਰਪਾ ਕਰਕੇ ਪਹਿਲਾਂ [ਇੱਕ ਡੋਮੇਨ ਨਾਮ ਰਜਿਸਟਰ ਕਰੋ](//beian.aliyun.com) ।

ਫਿਰ, ਮੇਨਲੈਂਡ ਚੀਨ ਵਿੱਚ ਕਲਾਉਡ ਵਿਕਰੇਤਾਵਾਂ ਦੇ ਆਬਜੈਕਟ ਸਟੋਰੇਜ ਦੀ ਵਰਤੋਂ ਕਰੋ `CDN` ਹੇਠਾਂ ਦਿੱਤੀ ਸਮੱਗਰੀ ਨੂੰ ਲਾਗੂ ਕਰੋ `out/ol/htm` +

ਤੁਸੀਂ ਏਜ ਕੰਪਿਊਟਿੰਗ ਦੀ ਵਰਤੋਂ ਸਿੰਗਲ-ਪੇਜ ਐਪਲੀਕੇਸ਼ਨਾਂ ਦੇ ਅਨੁਕੂਲ ਹੋਣ ਲਈ ਮਾਰਗ ਨੂੰ ਦੁਬਾਰਾ ਲਿਖਣ ਲਈ ਕਰ ਸਕਦੇ ਹੋ, ਉਦਾਹਰਨ ਲਈ, [Baidu Smart Cloud `CDN`](//cloud.baidu.com/product/cdn.html) ਨੂੰ ਇਸ ਤਰ੍ਹਾਂ ਸੰਰਚਿਤ ਕੀਤਾ ਜਾ ਸਕਦਾ ਹੈ:

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```
![](https://p.3ti.site/1721121273.avif)

ਕਿਉਂਕਿ ਰਿਕਾਰਡ `MX` ਅਤੇ ਰਿਕਾਰਡ `CNAME` ਇਕੱਠੇ ਨਹੀਂ ਹੋ ਸਕਦੇ, ਜੇਕਰ ਤੁਸੀਂ ਇੱਕੋ ਸਮੇਂ 'ਤੇ ਡੋਮੇਨ ਨਾਮ ਈਮੇਲਾਂ ਪ੍ਰਾਪਤ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ ਤੁਹਾਨੂੰ ਰਿਕਾਰਡ `A` ਵਿੱਚ ਲੈਵਲ `CNAME` ਲਈ ਸਕ੍ਰਿਪਟ ਨਾਲ ਸਹਿਯੋਗ ਕਰਨ ਦੀ ਲੋੜ ਹੈ [cname_flatten](https://github.com/i18n-site/lib/tree/main/cname_flatten)

ਇਸ ਤੋਂ ਇਲਾਵਾ, ਕਿਉਂਕਿ ਮੁੱਖ ਭੂਮੀ ਚੀਨ ਵਿੱਚ ਕਲਾਉਡ ਵਿਕਰੇਤਾਵਾਂ ਦੇ ਵਿਦੇਸ਼ੀ ਟ੍ਰੈਫਿਕ ਖਰਚੇ ਮੁਕਾਬਲਤਨ ਮਹਿੰਗੇ ਹਨ, ਜੇਕਰ ਤੁਸੀਂ ਲਾਗਤਾਂ ਨੂੰ ਅਨੁਕੂਲ ਬਣਾਉਣਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ ਤੁਸੀਂ ਪ੍ਰਾਪਤ ਕਰਨ ਲਈ [Huawei DNS ਦੇ ਮੁਫ਼ਤ ਭੂਗੋਲਿਕ ਰੈਜ਼ੋਲਿਊਸ਼ਨ](https://support.huaweicloud.com/usermanual-dns/dns_usermanual_0041.html) ਅਤੇ [Cloudflare for SaaS](https://developers.cloudflare.com/cloudflare-for-platforms/cloudflare-for-saas) ਦੇ ਕਸਟਮ ਡੋਮੇਨ ਨਾਮ ਦੀ ਵਰਤੋਂ ਕਰ ਸਕਦੇ ਹੋ ਟ੍ਰੈਫਿਕ ਡਾਇਵਰਸ਼ਨ──ਮੁੱਖ ਭੂਮੀ ਚੀਨ ਵਿੱਚ ਟ੍ਰੈਫਿਕ Baidu Cloud `CDN` , ਅੰਤਰਰਾਸ਼ਟਰੀ ਟ੍ਰੈਫਿਕ cloudflare ਜਾਂਦਾ ਹੈ।

![](https://p.3ti.site/1721119788.avif)

ਇਹ ਡਿਪਲਾਇਮੈਂਟ ਓਪਟੀਮਾਈਜੇਸ਼ਨ ਹੱਲ ਵਧੇਰੇ ਗੁੰਝਲਦਾਰ ਹਨ ਅਤੇ ਭਵਿੱਖ ਵਿੱਚ ਵੱਖਰੇ ਅਧਿਆਵਾਂ ਵਿੱਚ ਪੇਸ਼ ਕੀਤੇ ਜਾਣਗੇ।

#### ਆਮ ਡੋਮੇਨ ਨਾਮ ਰੀਡਾਇਰੈਕਸ਼ਨ

ਜੇਕਰ ਤੁਸੀਂ ਆਪਣੀ ਮੁੱਖ ਵੈੱਬਸਾਈਟ ਦੇ ਤੌਰ 'ਤੇ ਇੱਕ ਵੈੱਬਸਾਈਟ ਬਣਾਉਣ ਲਈ `i18n.site` ਵਰਤੋਂ ਕਰਦੇ ਹੋ, ਤਾਂ ਤੁਹਾਨੂੰ ਆਮ ਤੌਰ 'ਤੇ ਪੈਨ-ਡੋਮੇਨ ਰੀਡਾਇਰੈਕਸ਼ਨ ਨੂੰ ਕੌਂਫਿਗਰ ਕਰਨ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ, ਯਾਨੀ, `*.xxx.com` ( `www.xxx.com` ਸਮੇਤ) ਤੋਂ `xxx.com` ਤੱਕ ਪਹੁੰਚ ਨੂੰ ਰੀਡਾਇਰੈਕਟ ਕਰਨ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ।

ਇਹ ਲੋੜ ਅਲੀਬਾਬਾ ਕਲਾਊਡ `CDN` `EdgeScript` ( [ਅੰਗਰੇਜ਼ੀ ਦਸਤਾਵੇਜ਼](https://www.alibabacloud.com/help/en/cdn/developer-reference/how-edgescript-works) / [ਚੀਨੀ ਦਸਤਾਵੇਜ਼](https://help.aliyun.com/zh/cdn/developer-reference/edgescript) ) ਦੀ ਮਦਦ ਨਾਲ ਪ੍ਰਾਪਤ ਕੀਤੀ ਜਾ ਸਕਦੀ ਹੈ।

[ਅਲੀਬਾਬਾ CDN](https://cdn.console.aliyun.com/domain/list) ਵਿੱਚ ਡੋਮੇਨ ਨਾਮ ਸ਼ਾਮਲ ਕਰੋ ਅਤੇ ਅਲੀਬਾਬਾ ਕਲਾਉਡ `CDN` ਵਿੱਚ ਡੋਮੇਨ ਨਾਮ `*.xxx.com` ਤੋਂ `CNAME` ਪੁਆਇੰਟ ਕਰੋ।

![](https://p.3ti.site/1721122000.avif)

ਉਦਾਹਰਨ ਲਈ, ਉਪਰੋਕਤ ਤਸਵੀਰ ਵਿੱਚ `*.i18n.site` ਦੀ ਪੈਨ-ਡੋਮੇਨ ਨਾਮ ਰੀਡਾਇਰੈਕਸ਼ਨ ਕੌਂਫਿਗਰੇਸ਼ਨ ਇਸ ਤਰ੍ਹਾਂ ਹੈ:

```
rewrite(concat('https://i18n.site/',$uri), 'redirect',301)
```

#### nginx ਨਾਲ ਤੈਨਾਤ ਕਰੋ

ਕਿਰਪਾ ਕਰਕੇ ਆਪਣੇ ਖੁਦ ਦੇ ਪ੍ਰੋਜੈਕਟ `out/ol/htm` ਦੇ ਮਾਰਗ ਵਿੱਚ `server` `/root/i18n/md/out/ol/htm` ਬਦਲੋ nginx

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```

### ਡਾਇਰੈਕਟਰੀ ਬਣਤਰ

#### `public`

ਵੈੱਬਸਾਈਟ ਦੀਆਂ ਸਥਿਰ ਫਾਈਲਾਂ, ਜਿਵੇਂ ਕਿ `favicon.ico` , `robots.txt` , ਆਦਿ।

ਇੱਥੇ ਆਈਕਨ ਫਾਈਲਾਂ ਨੂੰ ਨਾਲ ਤਿਆਰ ਕੀਤਾ ਜਾ ਸਕਦਾ ਹੈ [realfavicongenerator.net](https://realfavicongenerator.net)

#### `.i18n`

`.i18n` ਡਾਇਰੈਕਟਰੀ ਦੇ ਹੇਠਾਂ `i18n.site` ਦੀਆਂ ਸੰਰਚਨਾ ਫਾਈਲਾਂ, ਅਨੁਵਾਦ ਕੈਸ਼ ਆਦਿ ਹਨ। ਵੇਰਵਿਆਂ ਲਈ ਅਗਲਾ ਅਧਿਆਇ ["ਸੰਰਚਨਾ"](/i18n.site/conf) ਵੇਖੋ।

#### `en`

ਸਰੋਤ ਭਾਸ਼ਾ ਡਾਇਰੈਕਟਰੀ, `.i18n/conf.yml` ਕੌਂਫਿਗਰੇਸ਼ਨ ਫਾਈਲ ਵਿੱਚ `fromTo` ਵਿੱਚੋਂ `en` ਦੇ ਅਨੁਸਾਰੀ

```yaml
i18n:
  fromTo:
    en: zh
```

ਕਿਰਪਾ ਕਰਕੇ ਅਨੁਵਾਦ ਦੀ ਸੰਰਚਨਾ ਨੂੰ ਵੇਖੋ [i18](/i18/use)
