# Mienskiplik Probleem

## Tafallich Wiske `.i18n/V` Wêrtroch't It `Npm` Pakket Net Publisearre Wurdt

De histoaryske ferzje fan `npm` pakketrelease is bewarre yn `.i18n/v/ol/v.hash`

As jo ​​per ûngelok wiskje `.i18n/v/ol` sil it `npm` pakket net publisearre wurde.

Fyn op dit stuit earst it ferzjenûmer fan de lêste releaseferzje fan jo projekt yn bygelyks [npmjs.com](//npmjs.com) `0.1.9`

Ferwize dan nei it `bash` oanmeitsje hjirûnder.

```bash
mkdir -p .i18n/v/ol
echo @0.1.9 > .i18n/v/ol/v.hash
```

Tink derom dat reparaasje op dizze manier de bestânskiednis ferliest, wêrtroch it ûnmooglik is om de folgjende release ynkrementeel te aktualisearjen, en alle ynhâld sil ien kear opnij ynpakt en upload wurde.
