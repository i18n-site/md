# प्लग-इन

प्लग-इन `.i18n/conf.yml` मध्ये कॉन्फिगर केले जाऊ शकतात, जसे की:

```yml
addon:
  - i18n.addon/toc
```

## अधिकृत प्लग-इन

* `i18n.addon/toc` : निर्देशिका निर्देशांक
  डीफॉल्टनुसार सक्षम केलेले `TOC` वर आधारित `json` ची निर्देशिका निर्देशांक तयार करा

* `i18n.addon/mouse` :

## फाइल नाव अधिवेशन

प्लग-इन सर्व `npm` पॅकेजेस आहेत.

वरील `i18n.addon/toc` शी संबंधित पॅकेज आहे [https://www.npmjs.com/package/@i18n.addon/toc](https://www.npmjs.com/package/@i18n.addon/toc)

प्लगइन डीफॉल्टनुसार नवीनतम आवृत्ती वापरते आणि साप्ताहिक अद्यतने तपासते.

आपण आवृत्तीचे निराकरण करू इच्छित असल्यास, आपण `i18n.addon/toc@0.1.3` लिहू शकता.

ट्रान्सलेशन कमांड लाइन `i18n.site` प्लग-इन पॅकेजची कन्व्हेन्शन फाइल स्थापित करेल आणि नंतर ती कार्यान्वित करेल.

मान्य केलेल्या फाईलची नावे खालीलप्रमाणे आहेत

### htmIndex.js

`.i18n/htm/index.js` च्या शेवटी `htmIndex.js` इंजेक्ट केले जाईल.

जेथे `__CONF__` वर्तमान कॉन्फिगरेशनच्या नावाने बदलले जाईल (जसे की `dev` किंवा `ol` ).

### afterTran.js

भाषांतर पूर्ण झाल्यानंतर ते कॉल केले जाईल, आणि पास केलेले पॅरामीटर्स खालीलप्रमाणे आहेत.

* `lang_li` : भाषा सूची, प्रथम भाषा ही स्त्रोत भाषा आहे
* `changed` : फायली
* `root` प्रोजेक्ट रूट :

परतीचे मूल्य एक शब्दकोश आहे, जसे की

```json
{
  file:{
    //  path: txt, for example :
    // "_.json": "[]"
  }
}
```

`file` ही आउटपुट फाइल सूची आहे, `path` फाइल पथ आहे आणि `txt` फाइल सामग्री आहे.

## अंगभूत कार्ये

बिल्ट-इन `js` रनटाइम [boa](https://github.com/boa-dev/boa) च्या दुय्यम विकासावर आधारित आहे आणि अंगभूत कार्ये खालीलप्रमाणे आहेत :

* : `wPath(path, txt)`
* `rTxt(path)` : फाइल वाचा
* `rBin(path)` : फाइल वाचा
* `rDir(dirpath)` : डिरेक्टरी वाचा, रिटर्न व्हॅल्यू ही एक ॲरे : list, file list आहे
