# පොදු ගැටළුවක්

## `npm` පැකේජය ප්‍රකාශනය කිරීමට අසමත් වීම හේතුවෙන් අහම්බෙන් මකා දමන `.i18n/v`

`npm` පැකේජ නිකුතුවේ ඓතිහාසික අනුවාදය සුරකින ලදි `.i18n/v/ol/v.hash`

ඔබ අහම්බෙන් `.i18n/v/ol` `npm` පැකේජය ප්‍රකාශනය නොකෙරේ.

මෙම අවස්ථාවේදී, පළමුව ඔබේ ව්‍යාපෘතියේ අවසන් නිකුතුවේ අනුවාද අංකය සොයා ගන්න [npmjs.com](//npmjs.com) උදාහරණයක් ලෙස `0.1.9`

ඉන්පසුව Create file `bash` බලන්න.

```bash
mkdir -p .i18n/v/ol
echo @0.1.9 > .i18n/v/ol/v.hash
```

මේ ආකාරයෙන් අලුත්වැඩියා කිරීමෙන් ගොනු ඉතිහාසය අහිමි වන අතර, ඊළඟ නිකුතුව වර්ධක ලෙස යාවත්කාලීන කිරීමට නොහැකි වන අතර, සියලු අන්තර්ගතයන් නැවත ඇසුරුම් කර එක් වරක් උඩුගත කරනු ලැබේ.
