# .i18n/conf.yml

වින්‍යාස ගොනුව `.i18n/conf.yml` වේ `i18n.site`

[`i18`](/i18) හි `ignore:` සහ `i18n:` සැකසුම් හැර, වින්‍යාස ගොනුව පහත පරිදි වේ:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

ඒවා අතර `upload` හි `ext:` වින්‍යාස අයිතමය ප්‍රකාශනය කිරීමේදී උඩුගත කරනු ලබන්නේ `.md` පමණක් බවයි.

## ඉහළ නාවික නාවික

`nav:` වින්‍යාස කිරීමේ විකල්ප, මුල් පිටුවේ ඉහළින් ඇති සංචාලන මෙනුවට අනුරූප වේ.

`en/i18n.yml` `zh/i18n.yml` බහු භාෂාවලට පරිවර්තනය කරනු ඇත.

පරිවර්තනය සම්පූර්ණ වූ පසු, ඔබට පරිවර්තනයේ `yml` වෙනස් කළ හැක, නමුත් `yml` පරිවර්තන යතුර එකතු කිරීම හෝ මකා දැමීම නොකරන්න.

### `use: Toc` තනි ගොනු අච්චුව (ලුහුඬු සටහන් සහිත)

වින්‍යාස : `nav`

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` යන්නෙන් අදහස් වන්නේ `Toc` විදැහුම්කරණය භාවිතා කිරීමයි, එය තනි `Markdown` අච්චුවක් විදැහුම්කරණය වේ.

`TOC` යනු `Table of Contents` හි කෙටි යෙදුමකි `Markdown`

`url:` `Markdown` ගොනු මාර්ගය නිරූපනය කරයි ( `/` මූල නාමාවලියට අනුරූප වේ `/README.md` , මෙම ගොනු නාමයට ලොකු අකුරු උපසර්ගයක් සහ කුඩා අකුරු උපසර්ගයක් අවශ්‍ය වේ).

### `use: Md` තනි ගොනු අච්චුව (දෘශ්‍ය සටහනක් නැත)

`Md` සැකිල්ල `Toc` සමාන වේ, දෙකම තනි `Markdown` ගොනුවක් ලබා දීමට භාවිතා කරයි. කෙසේ වෙතත් `Md` සැකිල්ල පැති තීරුවේ දළ සටහන නොපෙන්වයි.

ඔබට ඉහත වින්‍යාසය තුළ `use: Md` ලෙස `use: Toc` කළ හැක, `i18n.site` `md` නාමාවලියෙහි නැවත ධාවනය කරන්න, ඉන්පසු මුල් පිටුවේ වෙනස්කම් නිරීක්ෂණය කිරීමට සංවර්ධන පෙරදසුන URL වෙත යන්න.

### මානකරන මාර්ගයකින් තොරව පෙරනිමි පැටවීම

ප්‍රවේශ වන යම් මාර්ගයක මාර්ග උපසර්ගය `nav:` මාර්ගයට අනුරූප `MarkDown` ගොනුව පෙරනිමියෙන් පූරණය වන අතර `Md` අච්චුව භාවිතයෙන් ලබා දෙනු ඇත.

`/en/test.md` `en` `Md` `nav:` `/test`

මෙම ගොනුව නොපවතී `/en/test.md` , පෙරනිමි `404` පිටුව පෙන්වනු ඇත.

### `use: Doc` බහු ගොනු අච්චුව

වින්‍යාස ගොනුවේ:

```
  - i18n: blog
    use: Doc
```

අච්චු විදැහුම්කරණය සඳහා භාවිතා කරන බව දක්වයි `Doc`

`Doc` සැකිල්ල තනි හෝ බහු ව්‍යාපෘති සඳහා ලේඛන දළ සටහන් ජනනය කිරීමට බහු `MarkDown` ඒකාබද්ධ කිරීමට සහය දක්වයි.

#### තනි ව්‍යාපෘතිය (බහු ගොනු)

ඉහත `blog` `Doc` හි තනි ව්‍යාපෘති මාදිලියයි.

```
  - i18n: blog
    url: blog
    use: Doc
```

##### url හිස් වූ විට, එය i18n අගයට පෙරනිමි වේ

`url` `url` `i18n`

ඉහත ලිවීමේ ක්‍රමය තිබීමට සමාන වේ `url: blog` සහ එහි අනුරූප ගොනුව `en/blog/TOC` වේ.

#### බහු ව්යාපෘති

`.i18n/conf.yml` `i18n:doc` හි වින්‍යාසය බහු ව්‍යාපෘති ප්‍රකාරයයි.

```

    menu: NB demo1,demo2
    use: Doc
```

`NB` `Name Breif`

`NB` `demo1,demo2`
සටහන `demo1,demo2` කොමාව `,` ට පෙර ** නොතිබිය යුතුය : **

ඉහත පරාමිතීන් සඳහා, අනුරූප නාමාවලි දර්ශක ගොනුව වන්නේ:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### TOC පටුන දර්ශක

ඩිරෙක්ටරි දළ සටහනේ `json` ජනනය කිරීම සඳහා ඩිරෙක්ටරි දර්ශක ගොනුව `doc` සැකිල්ල වින්‍යාසයට අනුරූප `TOC` කියවීමට ආදර්ශන ගබඩාවේ `js` ප්ලගිනය `.i18n/hook/after.tran/TOC.js` ක්‍රියාත්මක කරයි `i18n.site`

ඔබ භාවිතා කරන්නේ නම් `doc`

ඔබ හිස් ෆෝල්ඩරයකින් `i18n.site` ආරම්භ කරන්නේ නම්, ආදර්ශන ව්‍යාපෘතියේ `.i18n` ඔබේ නාමාවලියට පිටපත් කිරීමට මතක තබා ගන්න.
