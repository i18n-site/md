ක්ලෝන කිරීමෙන් පසු, පරිවර්තන සම්පූර්ණ කිරීමට නාමාවලිය ඇතුළත් කර ධාවනය කරන්න `i18`

### නාමාවලි ව්යුහය

සැකිලි ගබඩා නාමාවලි ව්‍යුහය පහත පරිදි වේ

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` නාමාවලියෙහි පරිවර්තිත ආදර්ශන ගොනු අඩංගු වේ, ඒවා උදාහරණයක් පමණක් වන අතර මකා දැමිය හැක.

### පරිවර්තනය ධාවනය කරන්න

පරිවර්තන කිරීමට නාමාවලිය ඇතුල් කරන්න `i18`

### ගබඩාවට ගොනු එක් කරන්න

පරිවර්තනයට අමතරව, වැඩසටහන පහත ගොනු ද ජනනය කරනු ඇත, කරුණාකර ඒවා ගබඩාවට එක් කරන්න.

```
.i18n/hash
.i18n/cache/.gitignore
```

ඒවා අතර, අන්තර්ගතය පහත පරිදි : `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

මෙයින් අදහස් කරන්නේ `.i18n/cache/` ඇති සියලුම ගොනු නොසලකා හැරීමයි ( `.i18n/cache/.gitignore` හැර).

ඔබගේ අනුවාද පාලන මෘදුකාංග නොවේ නම් `git` කරුණාකර මෙම වින්‍යාසය අනුව එය නොසලකා හරින්න.

## මානකරන ගොනුව

එය `i18` line පරිවර්තන මෙවලමෙහි වින්‍යාස ගොනුව `.i18n/conf.yml`

අන්තර්ගතය පහත පරිදි වේ:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### මූලාශ්ර & පරිවර්තන භාෂාව

වින්‍යාස ගොනුවේ, `fromTo`

`en` මූලාශ්‍ර භාෂාව, `zh ja ko de fr` යනු පරිවර්තනයේ ඉලක්ක භාෂාවයි.

භාෂා කේතය බලන්න [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

උදාහරණයක් ලෙස, ඔබට චීන භාෂාව ඉංග්‍රීසි භාෂාවට පරිවර්තනය කිරීමට අවශ්‍ය නම්, මෙම පේළිය නැවත ලියන්න `zh: en`

ඔබට සහාය දක්වන සියලුම භාෂාවලට පරිවර්තනය කිරීමට අවශ්‍ය නම්, කරුණාකර පසුව හිස්ව තබන්න `:` උදාහරණ වශයෙන්

```
i18n:
  fromTo:
    en:
```

### ගොනුව නොසලකා හරින්න

පෙරනිමියෙන්, මූලාශ්‍ර භාෂා නාමාවලියෙහි `.md` සහ `.yml` සමඟින් ආරම්භ වන සියලුම ගොනු පරිවර්තනය කරනු ලැබේ.

ඔබට ඇතැම් ගොනු නොසලකා හැරීමට සහ ඒවා පරිවර්තනය නොකිරීමට අවශ්‍ය නම් (නිම නොකළ කෙටුම්පත් වැනි), ඔබට `ignore` වින්‍යාසය භාවිතා කළ හැක.

`ignore` `.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax)

උදාහරණයක් ලෙස, ඉහත වින්‍යාස ගොනුවෙහි අදහස් `_* ` `_` සමඟින් ආරම්භ වන ගොනු පරිවර්තනය නොකරන බවයි.

## පරිවර්තන නීති

### පරිවර්තන සංස්කාරකවරුන් රේඛා එකතු කිරීම හෝ මකා දැමීම නොකළ යුතුය

පරිවර්තනය සංස්කරණය කළ හැකි ය. මුල් පෙළ වෙනස් කර එය නැවත යන්ත්‍රයෙන් පරිවර්තනය කරන්න, පරිවර්තනයේ අතින් සිදු කරන ලද වෙනස් කිරීම් නැවත ලියන්නේ නැත (මුල් පෙළෙහි මෙම ඡේදය වෙනස් කර නොමැති නම්).

නමුත් පරිවර්තනයේ රේඛා සහ මුල් පිටපත එකකට අනුරූප විය යුතු බව කරුණාවෙන් සලකන්න. එනම්, පරිවර්තනය සම්පාදනය කිරීමේදී රේඛා එකතු කිරීම හෝ මකා දැමීම නොකළ යුතුය. එසේ නොමැති නම්, එය පරිවර්තන සංස්කරණ හැඹිලියේ ව්යාකූලත්වයට හේතු වනු ඇත.

යමක් වැරදී ඇත්නම්, කරුණාකර [විසඳුම් සඳහා නිතර අසන පැන වෙත යොමු වන්න.](/i18/qa#H1)

### පරිවර්තනය `YAML`

විධාන රේඛා මෙවලම මූලාශ්‍ර භාෂා ගොනු නාමාවලියෙන් අවසන් `.yml` සියලුම ගොනු සොයාගෙන ඒවා පරිවර්තනය කරයි.

* ගොනු නාමය ප්‍රත්‍යය විය යුතු `.yml` සලකන්න ( `.yaml` නොවේ).

මෙවලම ශබ්දකෝෂයේ අගයන් පමණක් පරිවර්තනය කරයි `.yml` ශබ්දකෝෂ යතුරු නොවේ.

උදාහරණ වශයෙන් `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

ලෙස පරිවර්තනය වනු ඇත `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` හි පරිවර්තනය අතින්ද වෙනස් කළ හැක (නමුත් පරිවර්තනයේ යතුරු හෝ රේඛා එකතු නොකරන්න).

`YAML` මත පදනම්ව, ඔබට විවිධ ක්‍රමලේඛන භාෂා සඳහා ජාත්‍යන්තර විසඳුම් පහසුවෙන් ගොඩනගා ගත හැකිය.

### පරිවර්තන උප බහලුම

ඔබ නිර්මාණය කරන තාක් `.i18n/conf.yml` (සෑම විටම ආදර්ශන ව්‍යාපෘති අච්චුවකින් ආරම්භ කිරීමට අවශ්‍ය නැත), `i18` හොඳින් ක්‍රියා කරයි.

විධාන රේඛා මෙවලම සියලුම උප බහලුම්වල `.i18n/conf.yml` සොයාගෙන එය පරිවර්තනය කරයි.

### අභිරුචි ස්ථාපන නාමාවලිය

එය පෙරනිමියෙන් ස්ථාපනය කෙරේ `/usr/local/bin`

ලිවීමේ අවසරය නොමැති `/usr/local/bin` එය `~/.bin` වෙත ස්ථාපනය කෙරේ

පරිසර විචල්‍යයන් `TO` ඔබට ස්ථාපන නාමාවලිය අර්ථ දැක්විය හැක, උදාහරණයක් ලෙස :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
