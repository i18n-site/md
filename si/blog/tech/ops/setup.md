## මුරපද රහිත පිවිසුම වින්යාස කරන්න

මිනිත්තු කිහිපයකට පසු, සේවාදායකයන් සියල්ලම සක්රිය විය.

පළමු දෙය IP අන්වර්ථයක් සැකසීමට දේශීය යන්ත්රයේ `~/.ssh/config` සංස්කරණය කිරීමයි, ඉන්පසු මුරපදයක් රහිත පිවිසුම වින්යාස කිරීම සඳහා `ssh-copy-id root@c2` භාවිතා කරන්න.

![](https://i-01.eu.org/1718287198.avif)

ආරම්භක වින්යාසය ඉක්මනින් සම්පූර්ණ කිරීමට ඔබට මෙම [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) වෙනස් කළ හැක.

මුරපද රහිත වින්යාසය අවසන් වූ පසු, ඔබට [pdsh](https://github.com/chaos/pdsh) භාවිතා කළ හැකිය කණ්ඩායම් වශයෙන්, එය පහසු වේ.

පළමුවැන්න නම් ipv6 අතින් සක්රීය කර පරීක්ෂා කිරීමයි

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

![](https://i-01.eu.org/1718287969.avif)

ඉන්පසු එක් ක්ලික් කිරීමකින් පද්ධතිය ubuntu 24 වෙත යාවත්කාලීන කර සාමාන්යයෙන් භාවිතා කරන මෘදුකාංග ස්ථාපනය කරන්න.

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 යනු භාෂාව චීන භාෂාවට සහ වේලා කලාපය බීජිං වෙත මාරු කිරීමයි.

පෙරනිමියෙන් ස්ථාපනය කර ඇති [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) මගේ පුද්ගලික භාවිත පුරුදු [direnv](https://direnv.net/) වින්යාස කර ඇත, එනම් [mise](https://github.com/jdx/mise)

මෙම පියවර පැය භාගයක් බලා සිටීමෙන් පසුව, සාමාන්යයෙන් භාවිතා කරන මෘදුකාංග වැඩිදියුණු කිරීම සහ ස්ථාපනය කිරීම සාර්ථක වනු ඇත. පහත දැක්වෙන විධානය සමඟ ඔබට මෙය තහවුරු කළ හැක:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## mosh ssh වේගවත් කරයි

යුරෝපයේ සිට චීනය දක්වා වූ ජාලය `ssh` වේගවත් කිරීම සඳහා භාවිතා කිරීම රෙකමදාරු කරනු ලැබේ [mosh](//mosh.org)

පහත දැක්වෙන පරිදි දේශීය නාමාවලිය `~/.bin` (පරිසර විචල්ය `PATH` වෙත ප්රධාන වශයෙන් එකතු කරන ලද) ස්ක්රිප්ට් `c0` සාදන්න:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

නිර්මාණය කිරීමෙන් පසු, ක්රියාත්මක කළ හැකි අවසර සහ මෘදු සබැඳිය `c1` සහ `c2` ලබා දෙන්න.

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

අනාගතයේදී, ඔබට අධිවේගී සෘජු සම්බන්ධතා `mosh` භාවිතා කිරීමට `c0` සහ `c1` සෘජුවම ඇතුළත් කර ඇතුල් කළ හැකිය [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

## කොටස් ආකෘතිය ext4 සිට btrfs දක්වා පරිවර්තනය කරන්න

[btrfs](https://wikipedia.org/wiki/Btrfs) පහසු උපස්ථ සඳහා තැටි ඡායාරූප ගත හැක.

මුලින්ම ගලවා ගැනීමේ පද්ධතියට ඇතුල් වන්න

![](https://i-01.eu.org/1718300448.avif)

Debian Rescue තෝරන්න

![](https://i-01.eu.org/1718300753.avif)

සේවාදායකය නැවත ආරම්භ වන තෙක් රැඳී සිටින්න

ssh ඇඟිලි සලකුණු වෙනස්කම් ඇති වේ, එය පිරිසිදු කිරීමට `ssh-keygen -R ip` භාවිතා කරන්න

ඉන්පසු කොටස පරිවර්තනය කිරීම සඳහා පහත ස්ක්රිප්ට් එක ධාවනය කරන්න

```bash
nohup bash -c "bash <(curl -s https://raw.githubusercontent.com/i18n-ops/ops/main/contabo/ext4_btrfs.sh) >/tmp/nohup.out 2>&1 && reboot" &
sleep 1 && tail -f /tmp/nohup.out
```

පරිවර්තනය සාර්ථක වූ පසු ස්ක්රිප්ට් ස්වයංක්රීයව පද්ධතිය නැවත ආරම්භ කරයි.

### btrfs සම්පීඩන අනුපාතය බලන්න

එය පෙරනිමියෙන් ස්වයංක්රීයව සම්පීඩනය වනු ඇත, සම්පීඩන අනුපාතය පරීක්ෂා කිරීමේ විධානය පහත පරිදි වේ.

```bash
if ! command -v compsize &>/dev/null; then
apt install -y  btrfs-compsize
fi
compsize -x /
```

## ගලවා ගැනීමේ පද්ධති විස්තර

ගලවා ගැනීමේ පද්ධතිය ඇතුල් 22 තොට [tcping](https://github.com/paradiseduo/tcping) ප්රවේශ විය හැකි වූ පසු, ඔබට විනාඩියකට 2-3 ට ප්රවේශ වීමට නොහැකි වනු ඇත, එය හොඳ වනු ඇත.

```
tcping ip 22 -c 9999
```

වරාය ප්රවේශ්යතාව නිරීක්ෂණය කළ හැක.

### Restore snapshot ඇතුළු කිරීමෙන් පසු දෘඪ තැටියට ප්රවේශ වන්නේ &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

උදාහරණයක් ලෙස, ඔබට ස්නැප්ෂොට් නැරඹිය හැකිය

```
snapper -c etc list
```

ස්නැප්ෂොට් ගොනු වෙනස්කම් බලන්න

```
snapper -c etc status 5..6
```

විශේෂිත වෙනස්කම් පෙළ බලන්න

```
snapper -c etc diff 3..5
```

ස්නැප්ෂොට් ප්රතිසාධනය කරන්න: උදාහරණයක් ලෙස, ස්නැප්ෂොට් 5 න් පසු සියලු වෙනස්කම් අහෝසි කරන්න

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### ලොග් පැවැත්ම සබල කරන්න

ගලවා ගැනීමේ පද්ධතියක් යටතේ ලොග් නොනැසී පැවතීම සක්රීය කිරීම සඳහා ඔබට ක්රියාත්මක වන systemd සේවාවක් නොතිබිය හැකි නිසා, ඔබට අදාල නාමාවලිය අතින් සාදා අමතර පියවර කිහිපයක් සිදු කිරීමට අවශ්ය වේ. ඔබට උත්සාහ කළ හැකි පියවර මෙන්න:

මුලින්ම chroot, පසුව

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

ඔබගේ උබුන්ටු පද්ධතියට නැවත පණ ගන්වන්න: ඉහත වෙනස්කම් සිදු කිරීමෙන් පසු, ඔබ ඔබේ යන්ත්රය සාමාන්ය ලෙස නැවත පණ ගැන්වීමට සහ ගලවා ගැනීමේ පද්ධතිය වෙනුවට ඔබේ උබුන්ටු පද්ධතියට නැවත ආරම්භ කිරීමට අවශ්ය වේ.

මේ ආකාරයෙන්, systemd-journald නව වින්යාසය සමඟ ආරම්භ වන අතර දෘඪ තැටියේ ලොග් අඛණ්ඩව ගබඩා කිරීම ආරම්භ කරයි.

ඉහත මෙහෙයුම් සිදු කරනු ලබන්නේ ගලවා ගැනීමේ පද්ධතියක් මත බව මතක තබා ගන්න, එම නිසා එම වෙනස්කම් පද්ධතිය නැවත පණගන්වන තෙක් සහ ගලවා ගැනීමේ ප්රකාරයේදී ක්රියාත්මක නොවනු ඇත.

මෙයින් අදහස් කරන්නේ පවතින ලඝු-සටහන් වල ස්ථීර වන කිසිදු වෙනසක් ඔබ වහාම නොපෙනෙන නමුත් ඉන් පසුව ජනනය කරන ලද සියලුම ලොග දිගටම පවතිනු ඇති බවයි.

මේ ආකාරයෙන් ඔබට ssh සේවාවේ ලඝු-සටහන් බැලිය හැක

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

මම කලින් ස්වයංක්රීය සම්පාදනය සහ යෙදවුම් ස්ක්රිප්ට් එකක් සෑදීමට github ක්රියාව භාවිතා කළ නමුත්, යෙදවුම් සේවාව ssh විය නොහැකි බව සොයා ගත්තා.

ගලවා ගැනීමේ පද්ධතියට ඇතුළු වීම සහ ලොග බැලීම අවසර ගැටලුවකි:

`Missing privilege separation directory: /run/sshd`

පහත දැක්වෙන විධානය මඟින් ඔබට මෙම නාමාවලිය සෑදිය හැක

```
chown root:root /
systemd-tmpfiles --create
```

නැවත නැවත විමර්ශනය කිරීමෙන් පසුව, tar පරිශීලක `uid` `1001` වන github action

`--no-same-owner` පරාමිතිය නොමැති නම් ඉවත් කිරීම විය යුතුය, එය පරිශීලකයින් `1001` සඳහා `uid` දක්වා විසන්ධි කරනු ලැබේ tar

එහි ප්රතිඵලයක් ලෙස, rsync විසින් root බහලුමේ පරිශීලක කණ්ඩායම වෙනස් කරනු ඇත, එවිට ඔබට නැවත ආරම්භ කිරීමෙන් පසු ssh වෙත ලොග් වීමට නොහැකි වනු ඇත.

පහත පරිදි විසංයෝජන ස්ක්රිප්ට් වෙනස් කරන්න:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
