## መጠቀም

### የማሳያ ፕሮጀክት

የትርጉም አወቃቀሩን ለማወቅ እባክዎን የማሳያ `i18` ይመልከቱ [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

በቻይና ያሉ ተጠቃሚዎች መዝለል ይችላሉ [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

ከክሎኒንግ በኋላ፣ ማውጫውን አስገባ እና ትርጉሙን ለማጠናቀቅ አሂድ `i18`

### ማውጫ መዋቅር

የአብነት መጋዘን ማውጫ መዋቅር እንደሚከተለው ነው

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` የተተረጎሙትን ማሳያ ፋይሎችን ይዟል፣ እነሱም ምሳሌ ብቻ ናቸው እና ሊሰረዙ ይችላሉ።

### ትርጉምን አሂድ

ማውጫውን ያስገቡ እና ለመተርጎም ያሂዱ `i18`

### ፋይሎችን ወደ ማከማቻው ያክሉ

ከትርጉም በተጨማሪ ፕሮግራሙ የሚከተሉትን ፋይሎች ያመነጫል, እባክዎን ወደ ማከማቻው ያክሏቸው.

```
.i18n/hash
.i18n/cache/.gitignore
```

ከነሱ መካከል : ይዘቱ እንደሚከተለው ነው `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

ይህ ማለት `.i18n/cache/` ውስጥ ያሉትን ሁሉንም ፋይሎች ችላ ይበሉ (ከ `.i18n/cache/.gitignore` በስተቀር)።

የእርስዎ ስሪት መቆጣጠሪያ ሶፍትዌር ካልሆነ `git` እባክዎ በዚህ ውቅር መሠረት ችላ ይበሉት።

## የማዋቀር ፋይል

`i18` መስመር የትርጉም መሳሪያ ውቅር ፋይል ነው `.i18n/conf.yml`

ይዘቱ እንደሚከተለው ነው።

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ምንጭ ቋንቋ &

በማዋቀር ፋይል ውስጥ፣ የ `fromTo`

ምንጭ ቋንቋ ነው `en` `zh ja ko de fr` የትርጉም ቋንቋ ነው።

የቋንቋ ኮድ ይመልከቱ [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

ለምሳሌ፣ ቻይንኛን ወደ እንግሊዝኛ ለመተርጎም ከፈለጉ፣ ይህን መስመር እንደገና ይፃፉ `zh: en`

ወደ ሁሉም የሚደገፉ ቋንቋዎች መተርጎም ከፈለጉ፣ እባክዎ ባዶ ይተውት `:` ለምሳሌ

```
i18n:
  fromTo:
    en:
```

### ፋይሉን ችላ በል

በነባሪነት በ `.md` እና `.yml` የሚጀምሩ ፋይሎች በሙሉ ይተረጎማሉ።

የተወሰኑ ፋይሎችን ችላ ለማለት እና ላለመተርጎም ከፈለጉ (እንደ ያልተጠናቀቁ ረቂቆች) የመስክ ውቅረትን መጠቀም ይችላሉ `ignore`

`.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax) የሚመሳሰል አገባብ ይጠቀማል `ignore`

ለምሳሌ `_* ` ባለው የውቅር ፋይል ውስጥ በ `_` የሚጀምሩ ፋይሎች አይተረጎሙም ማለት ነው።

## የትርጉም ደንቦች

### የትርጉም አዘጋጆች መስመሮችን ማከል ወይም መሰረዝ የለባቸውም

ትርጉሙ ሊስተካከል የሚችል ነው። ዋናውን ጽሑፍ አሻሽለው በማሽን-እንደገና ተርጉመው፣ በትርጉሙ ላይ የተደረጉ ማሻሻያዎች አይፃፉም (ይህ የዋናው ጽሑፍ አንቀጽ ካልተቀየረ)።

ግን እባክዎን የትርጉም መስመሮች እና የዋናው ጽሑፍ አንድ ወደ አንድ መዛመድ አለባቸው። ትርጉሙን በሚያጠናቅቅበት ጊዜ መስመሮችን አይጨምሩ ወይም አይሰርዙ ማለት ነው። አለበለዚያ በትርጉም ማረም መሸጎጫ ውስጥ ግራ መጋባት ይፈጥራል.

የሆነ ችግር ከተፈጠረ፣ እባክዎን [ለመፍትሄዎች የሚጠየቁ ጥያቄዎችን ይመልከቱ።](/i18/qa#H1)

### መተርጎም `YAML`

የትእዛዝ መስመር መሳሪያው በ `.yml` የሚያልቁ ፋይሎችን በምንጭ ቋንቋ ፋይል ማውጫ ውስጥ ፈልጎ ያገኛቸዋል።

* የፋይል ስም ቅጥያ ( `.yaml` ሳይሆን) መሆን እንዳለበት ልብ ይበሉ `.yml`

መሳሪያው የመዝገበ-ቃላትን እሴቶቹን የሚተረጉመው በ `.yml`

ለምሳሌ `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

ተብሎ ይተረጎማል `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

የ `YAML` ትርጉሙም በእጅ ሊስተካከል ይችላል (ነገር ግን በትርጉሙ ውስጥ ቁልፎችን ወይም መስመሮችን አይጨምሩ ወይም አይሰርዙ)።

በ `YAML` ትርጉም ላይ በመመስረት ለተለያዩ የፕሮግራም አወጣጥ ቋንቋዎች ዓለም አቀፍ መፍትሄዎችን በቀላሉ መገንባት ይችላሉ።

እስከፈጠርክ `.i18n/conf.yml` (ከማሳያ ፕሮጄክት አብነት በእያንዳንዱ ጊዜ መጀመር አያስፈልግም) `i18` ጥሩ ይሰራል።

### ብጁ የመጫኛ ማውጫ

በነባሪነት ወደ `/usr/local/bin`

የመጻፍ ፍቃድ `/usr/local/bin` ወደ `~/.bin` ይጫናል.

የአካባቢ ተለዋዋጮችን ያዘጋጁ `TO` ለምሳሌ የመጫኛ ማውጫውን መግለጽ ይችላሉ :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
