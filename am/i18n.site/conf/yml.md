# .i18n/conf.yml

የማዋቀሪያው ፋይል `.i18n/conf.yml` ነው `i18n.site`

[`i18`](/i18) `ignore:` እና `i18n:` ቅንጅቶች በስተቀር የማዋቀሪያው ፋይል እንደሚከተለው ነው።

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

ከነሱ መካከል የ `ext:` ማዋቀሪያ ንጥል ነገር ሲታተም `.md` `upload` ነው የሚሰቀለው ማለት ነው።

## ከፍተኛ አሰሳ ናቪ

በመነሻ ገጹ አናት ላይ ካለው የአሰሳ ምናሌ ጋር የሚዛመድ የውቅረት አማራጮች `nav:`

ከነሱ መካከል `i18n: home` ከ `en/i18n.yml`中`home: Home` ጋር ይዛመዳል።

እንደ `zh/i18n.yml` ባሉ ብዙ ቋንቋዎች ይተረጎማል `en/i18n.yml`

ትርጉሙ ከተጠናቀቀ በኋላ `yml` ውስጥ የ `yml`

### `use: Toc` ነጠላ ፋይል አብነት (ከዝርዝር ጋር)

: ላይ `nav`

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` ማለት `Toc` መስጠትን መጠቀም ነው፣ እሱም አንድ `Markdown` አብነት እያቀረበ ነው።

`Table of Contents` `TOC` ሲገለበጥ የ `Markdown` አህጽሮተ ቃል በጎን አሞሌው ላይ ይታያል።

`url:` የፋይሉን መንገድ ይወክላል `Markdown` ( `/` ከስር ማውጫው ጋር ይዛመዳል `/README.md` , ይህ የፋይል ስም አቢይ ሆሄ እና ትንሽ ቅጥያ ያስፈልገዋል).

### `use: Md` ነጠላ ፋይል አብነት (ምንም ንድፍ የለም)

`Md` ከ `Toc` ጋር አንድ ነው፣ ሁለቱም አንድ `Markdown` ፋይል ለመስራት ያገለግላሉ። ነገር ግን `Md` አብነቱ በጎን አሞሌው ውስጥ ያለውን ንድፍ አያሳይም።

ከላይ ባለው ውቅር ውስጥ ወደ `use: Md` `use: Toc` ፣ ማስኬድ `i18n.site` በ `md` ማውጫው ውስጥ እንደገና ማየት እና በመነሻ ገጹ ላይ ያሉትን ለውጦች ለመመልከት የገንቢ ቅድመ እይታ URLን ይጎብኙ።

### ያለ ውቅር ዱካ ያለ ነባሪ ጭነት

የሚደረስበት የተወሰነ መንገድ ቅድመ ቅጥያ በ `nav:` ውስጥ ካልተዋቀረ ከመንገዱ ጋር የሚዛመደው `MarkDown` ፋይል በነባሪ ተጭኖ `Md` አብነት በመጠቀም ይቀርባል።

ለምሳሌ ፣ `/test` እና `nav:` ያለዚህ መንገድ ከተዋቀረ እና የገጹ ቋንቋ እንግሊዝኛ (ኮድ `en` ) ከሆነ ፣ አብነቱ በነባሪ ይጫናል `/en/test.md` እና በመጠቀም `Md` ።

ይህ ፋይል `/en/test.md` ነባሪው `404` ገጽ ይታያል።

### `use: Doc` ባለብዙ-ፋይል አብነት

በማዋቀር ፋይል ውስጥ፡-

```
  - i18n: blog
    use: Doc
```

ለአብነት ስራ መጠቀሙን ያመለክታል `Doc`

`Doc` አብነት ለነጠላ ወይም ለብዙ ፕሮጀክቶች የሰነድ ዝርዝሮችን ለመፍጠር በርካታ `MarkDown` ማዋሃድ ይደግፋል።

#### ነጠላ ፕሮጀክት (በርካታ ፋይሎች)

ከላይ ያለው ነጠላ-ፕሮጀክት የ `Doc` ነው `blog`

```
  - i18n: blog
    url: blog
    use: Doc
```

##### ዩአርኤል ባዶ ሲሆን ወደ i18n እሴት ይቀይራል።

ካልፃፉ `url` በ `i18n` ዋጋ ላይ `url` ።

ከላይ ያለው የአጻጻፍ ዘዴ `url: blog` እና ተዛማጅ ፋይሉ `en/blog/TOC` ነው።

#### በርካታ ፕሮጀክቶች

ውቅር `i18n:doc` ባለብዙ ፕሮጄክት ሁነታ ነው `.i18n/conf.yml`

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

እዚህ `menu: NB demo1,demo2` ተቆልቋይ ሜኑ ለመስራት `NB` መጠቀም ማለት ነው።

`NB` የ `Name Breif` አህጽሮተ ቃል ነው, ይህም ተቆልቋይ ሜኑ የፕሮጀክቱን ስም እና መፈክር ያሳያል.

በመለኪያው `demo1,demo2` ተላልፏል `NB`
ከኮማ `,` በፊት እና በኋላ ** ክፍተቶች ሊኖሩ እንደማይገባ ልብ `demo1,demo2` : **

ከላይ ላሉት መመዘኛዎች፣ ተጓዳኝ የማውጫ ማውጫ ፋይል የሚከተለው ነው፡-

* `en/demo1/TOC`
* `en/demo2/TOC`

#### የTOC ማውጫ ማውጫ

የማውጫውን ማውጫ ፋይሉን ለማንበብ `doc` ከአብነት ውቅር ጋር የሚዛመድ `TOC` የማውጫውን ዝርዝር `json` ለማመንጨት የ `js` ተሰኪውን `.i18n/hook/after.tran/TOC.js` በማሳያ ማከማቻ ውስጥ ያስፈጽማል `i18n.site`

`doc` ከተጠቀሙ፣ ይህ ተሰኪ ሊኖርዎት ይገባል።

የ `i18n.site` ፕሮጀክቱን ከባዶ ፎልደር ከጀመሩት ፣በማሳያ ፕሮጄክቱ ውስጥ `.i18n` ወደ ማውጫዎ መቅዳትዎን ያስታውሱ።

`Doc` በተፈጠረው `json` ላይ በመመስረት የይዘቱን ዝርዝር ያቀርባል።

##### ዝርዝር ይዘት ማብራሪያ

##### ደረጃዎችን ለማመልከት መግባቱን ይጠቀሙ

በላይ `en/blog/TOC` በመጀመሪያው መስመር ላይ `README.md` ከታች በምስሉ ላይ ካለው `i18n.site` ጋር ይዛመዳል፣ እሱም የፕሮጀክት ስም ነው።

የሚቀጥሉት ሁለት መስመሮች ከታች ባለው ቅጽበታዊ ገጽ እይታ ላይ እንደሚታየው ናቸው.

##### የወላጅ ደረጃ የሚጽፈው ርዕሱን ብቻ እንጂ ይዘቱን አይደለም።
