# Встановити Та Використовувати

## windows Спочатку Встановлює git bash

windows Система, будь ласка, [спершу завантажте та встановіть `git bash`](https://git-scm.com/download/win)

Виконайте наступні операції в `git bash`

## Встановити

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### Налаштувати Маркер Перекладу

Відвідайте [i18n.site/token](//i18n.site/token) Натисніть, щоб скопіювати маркер

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

Створіть `~/.config/i18n.site.yml` вставте в нього скопійований вміст, вміст такий:

```
token: YOUR_API_TOKEN
```

Крім того, вам потрібно [i18n.site/payBill](//i18n.site/payBill) кредитну картку для оплати (поповнення не потрібне, веб-сайт автоматично стягуватиме комісію відповідно до використання, [дивіться ціни на домашній сторінці](/#price) ).

## Використовувати

### Демо Проект

Будь ласка `i18` зверніться до демонстраційного проекту [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

Користувачі в Китаї можуть клонувати [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

Після клонування увійдіть у каталог і запустіть `i18`

### Структура Каталогу

Структура каталогу складу шаблону виглядає наступним чином

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

Каталог `en` містить перекладені демонстраційні файли, які є лише прикладом і можуть бути видалені.

### Запустити Переклад

Увійдіть в каталог і `i18` переклад.

### Додати Файли До Сховища

Окрім перекладу, програма також згенерує наступні файли, додайте їх до репозиторію.

```
.i18n/hash
.i18n/cache/.gitignore
```

Серед : зміст `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

Це означає, що ігнорувати всі файли в `.i18n/cache/` (крім `.i18n/cache/.gitignore` ).

Якщо ваше програмне забезпечення для контролю версій не `git` , проігноруйте його відповідно до цієї конфігурації.

## Файл Конфігурації

`.i18n/conf.yml` Це файл конфігурації інструмента перекладу `i18` рядка

Зміст такий:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Мова Оригіналу & Мова Перекладу

У файлі конфігурації підлеглі `fromTo`

`en` є вихідною мовою, `zh ja ko de fr` є цільовою мовою перекладу.

Код мови див [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

Наприклад, якщо ви хочете перекласти китайську на англійську, перепишіть цей рядок `zh: en`

Якщо ви хочете перекладати на всі підтримувані мови, залиште порожнім після `:` наприклад

```
i18n:
  fromTo:
    en:
```

### Ігнорувати Файл

За замовчуванням усі файли, що починаються з `.md` і `.yml` у каталозі вихідної мови, будуть перекладені.

Якщо ви хочете ігнорувати певні файли та не перекладати їх (наприклад, незавершені чернетки), ви можете скористатися конфігурацією поля `ignore`

Використовує синтаксис `ignore` подібний до `.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax)

Наприклад, у наведеному вище файлі конфігурації `_* ` означає, що файли, які починаються з `_` , не будуть перекладені.

## Правила Перекладу

### Редактори Перекладу Не Повинні Додавати Або Видаляти Рядки

Переклад доступний для редагування. Змініть оригінальний текст і машинно перекладіть його ще раз, зміни перекладу, внесені вручну, не будуть перезаписані (якщо цей абзац оригінального тексту не було змінено).

Але зверніть увагу, що рядки перекладу та оригінального тексту повинні відповідати один одному. Тобто не додавати і не видаляти рядки при складанні перекладу. Інакше це призведе до плутанини в кеші редагування перекладу.

Якщо щось піде не так, зверніться до [розділу поширених запитань, щоб знайти рішення.](/i18/qa#H1)

### `YAML` Перекласти

Інструмент командного рядка знайде всі файли, що закінчуються на `.yml` у каталозі файлів вихідної мови та перекладе їх.

* Зауважте, що суфікс імені файлу має бути `.yml` (а не `.yaml` ).

Інструмент перекладає лише значення словника `.yml` а не ключі словника.

наприклад `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

буде перекладено як `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

Переклад `YAML` також можна змінити вручну (але не додавайте та не видаляйте ключі чи рядки в перекладі).

На основі `YAML` Translation ви можете легко створювати міжнародні рішення для різних мов програмування.

## Розширене Використання

### Підкаталог Перекладу

Поки ви створюєте `.i18n/conf.yml` (не потрібно кожного разу починати з шаблону демонстраційного проекту), `i18` працюватиме нормально.

Інструмент командного рядка знайде конфігурацію `.i18n/conf.yml` в усіх підкаталогах і переведе її.

Проекти, які використовують архітектуру [monorepo](//monorepo.tools) можуть розділяти мовні файли на підкаталоги.

<img alt="" src="https://p.3ti.site/1719910016.avif">

### Індивідуальний Каталог Встановлення

Він буде встановлено в `/usr/local/bin`

Якщо `/usr/local/bin` не має дозволу на запис, він буде встановлений на `~/.bin` .

Встановіть змінні середовища `TO` Ви можете визначити каталог встановлення, наприклад :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
