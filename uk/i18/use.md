# Встановлення та використання

## windows Спочатку встановіть git bash

windows Система, будь ласка, натисніть [тут](https://git-scm.com/download/win), щоб спочатку завантажити та встановити `git bash`.

Виконуйте подальші операції в `git bash`.

## Встановлення

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### Налаштування перекладацького токену

Відвідайте [i18n.site/token](//i18n.site/token) та натисніть, щоб скопіювати токен

<img src="https://p.3ti.site/1719911689.avif" style="width:400px">

Створіть `~/.config/i18n.site.yml`, вставте в нього скопійований вміст, наприклад:

```
token: YOUR_API_TOKEN
```

Додатково, вам потрібно прив'язати кредитну картку на [i18n.site/payBill](//i18n.site/payBill) (не потрібно поповнення, сайт автоматично стягуватиме плату відповідно до використання, [дивіться ціни на головній сторінці](/#price)).

## Використання

### Демонстраційний проект

Зверніться до демонстраційного проекту [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18), щоб дізнатися про налаштування `i18`.

Користувачі з Китаю можуть клонувати [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

Після клонування увійдіть до каталогу та запустіть `i18`, щоб завершити переклад.

### Структура каталогу

Структура каталогу шаблону виглядає так:

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

Файли в каталозі `en` є демонстраційними перекладеними файлами і можуть бути видалені.

### Запуск перекладу

Увійдіть до каталогу та запустіть `i18`, щоб виконати переклад.

Програма, крім перекладу, створить папку `.i18n/data`, яку потрібно додати до репозиторію.

Після перекладу нового файлу, у цьому каталозі буде створено новий файл даних. Не забудьте виконати `git add .`.

## Файл конфігурації

`.i18n/conf.yml` є файлом конфігурації для перекладацького інструменту командного рядка `i18`

Зміст файлу такий:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Джерелова мова та мова перекладу

У файлі конфігурації під `fromTo`:

`en` — джерелова мова, `zh ja ko de fr` — мови перекладу.

Коди мов дивіться на [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE).

Наприклад, щоб перекласти з китайської на англійську, змініть рядок на `zh: en`.

Якщо ви хочете перекладати на всі підтримувані мови, залиште поле після `:` порожнім, наприклад:

```
i18n:
  fromTo:
    en:
```

Для різних `fromTo` / записана так: :

```
i18n:
  fromTo:
    en:
  path:
    blog:
      fromTo:
        zh:
    blog/your_file_name.md:
      fromTo:
        ja:
```

У цій таблиці налаштувань мова джерела для перекладу каталогу `blog` є `zh`, а для файлу `blog/your_file_name.md` мова джерела є `ja`.

### Ігнорування файлів

За замовчуванням всі файли з розширеннями `.md` та `.yml` у каталозі джерелової мови будуть перекладені.

Якщо ви хочете ігнорувати деякі файли (наприклад, незавершені чернетки), використовуйте поле `ignore` для налаштування.

`ignore` використовує той самий синтаксис, що й `.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax).

Наприклад, `_*` у файлі конфігурації означає, що файли, які починаються з `_`, будуть ігноруватися.

## Правила перекладу

### Редактори перекладу не повинні додавати або видаляти рядки

Переклад можна редагувати. Якщо ви зміните текст оригіналу, машинний переклад не перезапише ваші手动 зміни (якщо оригінальний текст не був змінений).

> [!WARN]
> Рядки перекладу та оригінального тексту мають відповідати один одному. Тобто не додавати та не видаляти рядки під час складання перекладу. Інакше це може призвести до плутанини в кеші редагування перекладу.

Якщо виникнуть проблеми, зверніться до [розділу поширених запитань](/i18/qa#H1) для вирішення

### `YAML` перекладів

Інструмент командного рядка знайде всі файли з розширенням `.yml` у каталозі джерелової мови та перекладе їх.

* Зверніть увагу, що суфікс файлу повинен бути `.yml` (а не `.yaml`).

Інструмент перекладає лише значення словників у `.yml`, а не ключі.

Наприклад, `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

буде перекладено як `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

Переклад `YAML` також можна手动 змінювати (але не додавайте та не видаляйте ключі або рядки в перекладі).

На основі перекладу `YAML` ви можете легко створювати міжнародні рішення для різних мов програмування.

## Розширене використання

### Переклад підкаталогів

Поки створено `.i18n/conf.yml` (не потрібно кожного разу починати з шаблону демонстраційного проекту), `i18` буде працювати нормально.

Інструмент командного рядка знайде `.i18n/conf.yml` у всіх підкаталогах та виконає переклад.

Проекти, які використовують архітектуру [monorepo](//monorepo.tools), можуть розділяти мовні файли на підкаталоги.

![](https://p.3ti.site/1719910016.avif)

### Налаштування каталогу встановлення

За замовчуванням встановлюється в `/usr/local/bin`.

Якщо у `/usr/local/bin` немає прав на запис, встановлюється в `~/.bin`.

Ви можете визначити каталог встановлення, налаштувавши змінну середовища `TO`, наприклад:

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
