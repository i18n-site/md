
## Прив’язка CNAME до доменного імені

Після прив’язки доменного імені до Cloudflare Worker через `SSL/TLS` → налаштування спеціального імені хоста для CNAME-прив’язки, безпосередній доступ призводить до помилки `error code: 522`

У цьому випадку необхідно використовувати HTTP-маршрутизацію → маршрутизацію для Workers, щоб прив’язати маршрут, як показано на малюнку нижче
