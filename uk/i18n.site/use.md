### Опублікувати Вміст

`i18n.site` Приймає [односторінкову архітектуру програми](https://developer.mozilla.org/docs/Glossary/SPA) , а сторінка входу на веб-сайт і вміст веб-сайту розгортаються незалежно.

Після виконання наведеного вище перекладу в `md/out/dev` буде створено каталоги `htm` і `v` .

Тут `dev` вказує на те, що він створений на основі файлу `.i18n/htm/dev.yml` .

Під : `dev`

Під каталогом `htm` вхідна сторінка сайту.

`v` містить вміст веб-сайту з номером версії.

Локальний попередній перегляд скопіює всі файли в каталог `out/dev/v/0.1.0` незалежно від номера версії.

Для офіційного випуску змінені файли буде скопійовано до каталогу з новим номером версії.

#### Використовуйте -C, Щоб Вказати Файл Конфігурації

Різні конфігураційні файли створять відповідні каталоги в каталозі `out`

Наприклад, `.i18n/htm/ol.yml` створить каталог `out/ol` .

`dev.yml` і `ol.yml` є конфігураціями за замовчуванням.

`dev` — це абревіатура `development` , яка представляє середовище розробки, використовується для локального попереднього перегляду, а також є файлом конфігурації за замовчуванням.
`ol` Абревіатура `online` , яка представляє онлайн-середовище, використовується для офіційного випуску, а також є файлом конфігурації за замовчуванням у `npm` з використанням параметра командного рядка `-n` .

Ви також можете створити інші файли конфігурації `--htm_conf` щоб вказати назву файлу конфігурації.

наприклад:
```
i18n.site --htm_conf yourConfig --save
```

Тут `--save` вказує номер версії оновлення.

#### <a rel=id href="#npm" id="npm"></a> Публікуйте вміст на npmjs.com

Публікація вмісту [в](/i18n.site/feature#ha) [npmjs.com](//npmjs.com)

##### npm login & Реліз

Встановіть `nodejs` використовуйте `npm login` для входу.

Змініть `md/.i18n/htm/ol.yml` змініть `i18n.site` у `v: //unpkg.com/i18n.site` на власне `npm` назва пакета.

Просто використовуйте незайняте ім'я пакета на [npmjs.com](//npmjs.com)

Під час публікації на основі пакета `npm` **обов’язково використовуйте `//unpkg.com/`** як префікс значення `v:` `i18n.site` під цим префіксом `/.v` було спеціально оптимізовано для своєчасного перегляду нових випусків.

Запустіть `i18n.site --npm` або `i18n.site -n` у каталозі `md` , щоб перекласти та опублікувати.

Якщо ви використовуєте безперервне інтеграційне середовище для публікації, немає потреби встановлювати його `nodejs` Просто скопіюйте дозвіл `~/.npmrc` для входу в систему та опублікований у середовищі.

Якщо ви зміните назву пакета в `v:` в `ol.yml` , будь ласка **, спочатку видаліть `.i18n/v/ol`** , а потім опублікуйте його.

##### Проксі-Сервер, Опублікований npm

Якщо користувачі в материковому Китаї стикаються з проблемами мережі та не можуть опублікувати пакет `npm` , вони можуть встановити змінну середовища `https_proxy` для налаштування проксі-сервера.

Якщо припустити, що порт вашого проксі-сервера `7890` ви можете написати:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### Самостійний Вміст

Якщо ви хочете самостійно розмістити вміст, `md/.i18n/htm/ol.yml` змініть `v: //unpkg.com/i18n.site` на префікс URL-адреси, наприклад `v: //i18n-v.xxx.com` .

Увійдіть в каталог `md` і запустіть

```
i18n.site --htm_conf ol --save
```

або абревіатура

```
i18n.site -c ol -s
```

Потім налаштуйте вміст у `md/out/ol/v` на префікс URL-адреси, заданий у `v:` .

Нарешті, **налаштуйте час кешу шляху `/.v` що закінчується на `1s`** , інакше доступ до нещодавно опублікованого вмісту буде неможливим.

Час кешу для інших шляхів можна встановити на один рік або більше, щоб зменшити непотрібні запити.

##### Розмістити Вміст На s3

Для самостійного розміщення вмісту, окрім використання власного сервера, ще `CDN` поширеним варіантом є + `S3`

Ви можете скористатися сервером [rclone](https://rclone.org) Login `S3` , потім перейти до сценарію нижче та скопіювати лише додаткові зміни `S3` час кожної публікації.

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

Не забудьте налаштувати `CDN` щоб час кешу шляху, який закінчується на `/.v` , становив `1s` , інакше ви не зможете негайно отримати доступ до нещодавно опублікованого вмісту.

#### Оптимізація Розгортання Веб-Сайту В Китаї

Якщо ви хочете отримати кращу доступність у мережевому середовищі материкового Китаю, спершу [зареєструйте доменне ім’я](//beian.aliyun.com) .

Потім використовуйте `out/ol/htm` сховище постачальників у материковому `CDN` +

Ви можете використовувати периферійні обчислення, щоб переписати шлях для адаптації до односторінкових програм, таких як [Baidu Smart Cloud `CDN`](//cloud.baidu.com/product/cdn.html) Його можна налаштувати так:

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```
<img alt="" src="https://p.3ti.site/1721121273.avif">

Оскільки записи `MX` і `CNAME` записи не можуть співіснувати, якщо ви хочете отримувати електронні листи з іменами домену одночасно, вам потрібно співпрацювати зі сценарієм [cname_flatten](https://github.com/i18n-site/lib/tree/main/cname_flatten) , щоб звести `CNAME` в `A` запис.

Крім того, оскільки плата за закордонний трафік постачальників хмарних технологій у материковому Китаї є відносно високою, якщо ви хочете оптимізувати витрати, ви можете скористатися [безкоштовною географічною роздільною здатністю Huawei DNS](https://support.huaweicloud.com/usermanual-dns/dns_usermanual_0041.html) і спеціальним доменним іменем [Cloudflare for SaaS](https://developers.cloudflare.com/cloudflare-for-platforms/cloudflare-for-saas) (як показано нижче). перенаправлення трафіку──Трафік у материковому Китаї Baidu Cloud `CDN` міжнародний трафік становить cloudflare .

#### Загальне Переспрямування Доменного Імені

Якщо ви використовуєте `i18n.site` генерувати веб-сайт як свій основний веб-сайт, зазвичай потрібно налаштувати переспрямування на весь домен, тобто перенаправити `*.xxx.com` (включаючи `www.xxx.com` ) доступ до `xxx.com` .

Ця вимога може бути виконана за допомогою Alibaba Cloud `CDN` `EdgeScript` ( [англомовний документ](https://www.alibabacloud.com/help/en/cdn/developer-reference/how-edgescript-works) / [китайський документ](https://help.aliyun.com/zh/cdn/developer-reference/edgescript) )

Додайте доменне ім'я в [Alibaba CDN](https://cdn.console.aliyun.com/domain/list) і вкажіть `*.xxx.com` ім'я на `CNAME` Alibaba Cloud `CDN` .

```
rewrite(concat('https://i18n.site/',$uri), 'redirect',301)
```

#### Розгорнути За Допомогою nginx

Будь ласка, додайте конфігурацію, подібну до наведеної `server` nginx де `/root/i18n/md/out/ol/htm` змініть її на шлях вашого власного проекту `out/ol/htm` :

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```

### Структура Каталогу

#### Громадськість

Статичні файли веб-сайту, такі як `favicon.ico` `robots.txt` тощо.

Файли значків тут можна створити за допомогою [realfavicongenerator.net](https://realfavicongenerator.net)

#### .i18n

`.i18n` У каталозі знаходяться конфігураційні файли `i18n.site` , кеш перекладу тощо. Подробиці див. у наступному розділі [«Конфігурація»](/i18n.site/conf) .

#### en

Каталог вихідної мови, що `.i18n/conf.yml` `fromTo` `en` у файлі конфігурації

```yaml
i18n:
  fromTo:
    en: zh
```

Будь ласка, зверніться до конфігурації перекладу [i18](/i18/use)
