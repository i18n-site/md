# .i18n/conf.yml

Конфігураційний файл для `i18n.site` — `.i18n/conf.yml`.

За винятком налаштувань [`i18`](/i18) , `ignore:` і `i18n:` , файл конфігурації виглядає наступним чином:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

Там, де `upload` має налаштування `ext:`, це означає, що під час публікації буде завантажено лише файли з розширенням `.md`.

## Верхнє навігаційне меню

Налаштування `nav:` відповідають верхньому навігаційному меню на головній сторінці.

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

Серед них `i18n: home` відповідає `home: Home` в `en/i18n.yml`.

`en/i18n.yml` буде перекладено на кілька мов, наприклад `zh/i18n.yml`.

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

Після завершення перекладу ви можете змінити значення перекладу в `yml`, але не додавайте та не видаляйте ключі перекладу `yml`.

### `use: Toc`, шаблон для одного файлу (з контентом)

У налаштуваннях `nav`:

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` означає використання шаблону `Toc` для відображення, тобто відображення одного шаблону `Markdown`.

`TOC` — це скорочення від `Table of Contents`. Коли цей шаблон відображається, він показує структуру змісту файлу `Markdown` на бічній панелі.

`url:` вказує на шлях до файлу `Markdown` ( `/` відповідає кореневому каталогу `/README.md`, ім'я файлу має префікс у верхньому регістрі та суфікс у нижньому регістрі).

### `use: Md`, шаблон для одного файлу (без контенту)

Шаблон `Md` і шаблон `Toc` ідентичні і обидва використовуються для відображення одного файлу `Markdown`. Але шаблон `Md` не відображає контент на бічній панелі.

Ви можете змінити `use: Toc` на `use: Md` у наведених вище налаштуваннях, потім знову запустити `i18n.site` у каталозі `md` і відвідати URL для попереднього перегляду, щоб спостерігати за змінами на головній сторінці.

### Завантаження за умовчанням без налаштуваного шляху

Якщо ви намагаєтеся отримати доступ до певного шляху, і його префікс не налаштований у `nav:`, тоді за умовчанням буде завантажено файл `Markdown`, що відповідає цьому шляху, і відображено за допомогою шаблону `Md`.

Наприклад, якщо ви намагаєтеся отримати доступ до `/test`, і цей шлях не налаштований у `nav:`, а мова сторінки — англійська (код `en`), тоді буде завантажено `/en/test.md` за умовчанням і відображено за допомогою шаблону `Md`.

Якщо файл `/en/test.md` не існує, буде відображено стандартну сторінку `404`.

### `use: Doc`, багатофайловий шаблон

У файлі конфігурації:

```
  - i18n: blog
    use: Doc
```

Вказує на використання `Doc` для відображення шаблону.

Шаблон `Doc` підтримує інтеграцію кількох файлів `Markdown` для створення єдиного або кількох планів документів для проектів.

#### Один проект (кілька файлів)

`blog` вище є режимом одного проекту `Doc`.

```
  - i18n: blog
    url: blog
    use: Doc
```

##### Якщо URL-Адреса Порожня, За Умовчанням Використовується Значення i18n

Якщо `url` не вказано, `url` за умовчанням дорівнює `i18n`. Це правило діє для всіх інших шаблонів.

Наведений вище спосіб запису еквівалентний `url: blog`, і відповідний файл — `en/blog/TOC`.

#### Кілька проектів

Конфігурація `i18n:doc` у `.i18n/conf.yml` є мультипроектним режимом.

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

Тут `menu: NB demo1,demo2` означає використання шаблону `NB` для відображення спадного меню.

`NB` — це скорочення від `Name Brief`, що означає, що спадне меню може відображати назву та слоган проекту.

`NB` супроводжується параметром `demo1,demo2`.
Зверніть увагу: **не повинно бути пробілів** перед і після коми `,` у `demo1,demo2`.

Відповідний файл індексу каталогу для наведених вище параметрів:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### Індекс таблиці змісту

`i18n.site` виконує плагін `js` `.i18n/hook/after.tran/TOC.js` у демонстраційному сховищі для читання файлу індексу каталогу `doc`, що відповідає конфігурації шаблону `TOC`, для створення структури `json` каталогу.

Якщо ви використовуєте шаблон `doc`, вам потрібно мати цей плагін.

Якщо ви ініціалізуєте проект `i18n.site` з порожньої папки, не забудьте скопіювати демонстраційний проект `.i18n` у ваш каталог.

Шаблон `Doc` відображає структуру змісту на основі згенерованого `json`.

##### Детальний опис змісту

`en/blog/TOC` виглядає так:

```
README.md

news/README.md
  news/begin.md
```

##### Використовуйте відступи для позначення рівнів

Перший рядок `README.md` у `en/blog/TOC` відповідає `i18n.site` на малюнку нижче, тобто назві проекту.

Наступні два рядки, як показано на знімку екрана нижче.

<img src="https://p.3ti.site/1721097381.avif" style="width:320px">

`news/README.md` відповідає `News`
`news/begin.md` відповідає `Our Product is Online !`

Файл `TOC` використовує відступи для позначення ієрархічних рівнів структури, підтримуючи багаторівневі відступи.

##### Батьківський рівень записує лише заголовок, не вміст

У разі багаторівневих відступів батьківський рівень записує лише заголовок, а не вміст, щоб уникнути розладу типографії.

##### Проект README.md

У файлі `README.md` проекту, наприклад `en/demo2/README.md`, можна записати вміст.

Зверніть увагу, що вміст цього файлу не відображає структуру змісту, тому рекомендується обмежити його довжину та написати короткий вступ.

###### Слоган проекту

Ви можете побачити, що `Deme Two` має свій слоган під спадним меню та назву проекту `Your Project slogan`.

![](https://p.3ti.site/1721276842.avif)

Це відповідає першому рядку `en/demo2/README.md`:

```
# Demo Two : Your Project slogan
```

Вміст після першої двокрапки `:` у заголовку першого рівня `README.md` проекту буде розглядатися як слоган проекту.

Користувачі з Китаю, Японії та Кореї, зверніть увагу, що ви повинні використовувати половинну двокрапку `:` замість повної.

##### Як Масово Перемістити TOC?

Файли `TOC` потрібно розмістити в каталозі вихідної мови.

Наприклад, якщо вихідною мовою є китайська, то `TOC` вище означає `zh/blog/TOC`.

Якщо вихідну мову змінено, вам потрібно масово перемістити `TOC` файли певної мови в проекті на іншу мову.

Ви можете звернутися до таких команд:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

Будь ласка, змініть `en/` і `zh/` у наведеній вище команді на код вашої мови.
