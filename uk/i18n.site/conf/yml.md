# .i18n/conf.yml

Профіль для `i18n.site` дорівнює `.i18n/conf.yml`.

-Za винятком налаштувань [`i18`](/i18) `ignore:` та `i18n:` configuration, файл конфігурації виглядає наступним чином:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
addon:
  - i18n.addon/toc
```

Де `upload` з `ext:` означає, що під час публікації буде завантажено лише файли з розширенням `.md`.

## Верхня навігаційна панель

`nav:` параметри конфігурації відповідають верхньому навігаційному меню на головній сторінці.

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

Серед них `i18n: home` відповідає `home: Home` в `en/i18n.yml`.

`en/i18n.yml` буде перекладено на кілька мов, наприклад `zh/i18n.yml`.

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

Після завершення перекладу ви можете змінити значення перекладу в `yml`, але не додавайте та не видаляйте ключі перекладу `yml`.

### `use: Toc`, шаблон одного файлу (з контентом)

У `nav` конфігурації:

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` означає використання шаблону `Toc` для відображення одного файлу `Markdown`.

`TOC` — це скорочення від `Table of Contents`. Коли цей шаблон буде відтворено, контент цього `Markdown` файлу буде відображено у бічній панелі.

`url:` позначає шлях до `Markdown` файлу ( `/` відповідає кореневому каталогу `/README.md`, ім'я файлу повинно починатися з великої літери та закінчуватися малою).

### `use: Md`, шаблон одного файлу (без контенту)

Шаблони `Md` та `Toc` ідентичні та використовуються для відображення одного `Markdown` файлу. Однак шаблон `Md` не відображає контент у бічній панелі.

Ви можете змінити `use: Toc` на `use: Md` у наведеній вище конфігурації, перезапустити `i18n.site` у каталозі `md`, а потім відвідати URL попереднього перегляду для спостереження за змінами на головній сторінці.

### Автоматичне завантаження за відсутності конфігураційного шляху

Якщо доступ до певного шляху здійснюється, і його префікс не налаштовано в `nav:`, за замовчуванням буде завантажено відповідний `Markdown` файл і відтворено за допомогою шаблону `Md`.

Наприклад, при доступі до `/test`, якщо `nav:` не налаштовано для цього шляху, і мова сторінки англійська (код `en`), буде завантажено `/en/test.md` і відтворено за допомогою шаблону `Md`.

Якщо файл `/en/test.md` не існує, буде відображено стандартну `404` сторінку.

### `use: Doc`, багатофайловий шаблон

У конфігураційному файлі:

```
  - i18n: blog
    use: Doc
```

Вказує на використання `Doc` для відображення шаблону.

Шаблон `Doc` підтримує інтеграцію кількох `Markdown` файлів для генерації єдиного або кількох проектних документів.

#### Один проект (кілька файлів)

`blog` вище є режимом одного проекту `Doc`.

```
  - i18n: blog
    url: blog
    use: Doc
```

##### Якщо URL-адреса порожня, за умовчанням використовується значення i18n

Якщо `url` не вказано, `url` за замовчуванням дорівнює `i18n`. Це правило діє для всіх інших шаблонів.

Наведений вище спосіб запису еквівалентний `url: blog`, відповідний файл — `en/blog/TOC`.

#### Кілька проектів

Конфігурація `i18n:doc` у `.i18n/conf.yml` є мультипроектним режимом.

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

Тут `menu: NB demo1,demo2` означає використання шаблону `NB` для відображення спадного меню.

`NB` — це скорочення від `Name Brief`, що означає, що спадне меню може відображати назву та слоган проекту.

`NB` супроводжується параметром `demo1,demo2`.
Зверніть увагу: ** між `demo1,demo2` не повинно бути пробілів перед і після коми `,`.**

Для наведених вище параметрів відповідний файл індексу каталогу:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### Індекс таблиці вмісту

`i18n.site` виконує плагін `js` `.i18n/hook/after.tran/TOC.js` у демонстраційному сховищі для читання файлу індексу каталогу `doc`, що відповідає конфігурації шаблону `TOC`, для генерації структури `json` каталогу.

Якщо ви використовуєте шаблон `doc`, вам обов'язково потрібен цей плагін.

Якщо ви ініціалізуєте проект `i18n.site` з порожньої папки, не забудьте скопіювати демонстраційний `.i18n` до вашого каталогу.

Шаблон `Doc` відображає структуру змісту на основі згенерованого `json`.

##### Детальний опис вмісту

`en/blog/TOC` виглядає так:

```
README.md

news/README.md
  news/begin.md
```

##### Використовуйте відступи для позначення рівнів

Перший рядок `README.md` у `en/blog/TOC` відповідає `i18n.site` на малюнку нижче, що є назвою проекту.

Наступні два рядки, як показано на знімку екрана нижче.

<img src="https://p.3ti.site/1721097381.avif" style="width:320px">

`news/README.md` відповідає `News`
`news/begin.md` відповідає `Our Product is Online !`

Файли `TOC` використовують відступи для позначення ієрархічних рівнів структури, підтримують багаторівневі відступи та рядкові коментарі, що починаються з `#`.

##### Батьківський рівень записує лише заголовок, не вміст

При наявності кількох рівнів відступів, батьківський рівень записує лише заголовок, не вміст, щоб уникнути помилок у форматуванні.

##### Проект README.md

Вміст можна записати в `README.md` файлі, наприклад `en/demo2/README.md`.

Зверніть увагу, що вміст цього файлу не включає план змісту, тому раджу обмежити його довжину та написати короткий вступ.

###### Слоган проекту

Ви можете побачити, що `Deme Two` має свій слоган під спадним меню та назву проекту `Your Project slogan`.

![](https://p.3ti.site/1721276842.avif)

Це відповідає першому рядку `en/demo2/README.md`:

```
# Demo Two : Your Project slogan
```

Значення після першої двокрапки `:` у заголовку першого рівня `README.md` файлу буде розглядатися як слоган проекту.

Користувачі з Китаю, Японії та Кореї, зверніть увагу, що ви повинні використовувати половинну двокрапку `:` замість повної.

##### Як масово перемістити TOC?

`TOC` файли повинні бути розміщені в каталозі вихідної мови.

Наприклад, якщо вихідною мовою є китайська, то `TOC` вище означає `zh/blog/TOC`.

Якщо вихідну мову змінено, вам потрібно масово перемістити `TOC` файли певної мови в проекті на іншу мову.

Ви можете звернутися до таких команд:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

Будь ласка, змініть `en/` та `zh/` у наведеній вище команді на код вашої мови.
