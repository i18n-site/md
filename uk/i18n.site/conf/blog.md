# Шаблон для блогу

`i18n/conf.yml` з `use: Blog` означає використання шаблону блогу для рендерингу.

Файл `markdown` статті блогу потребує налаштування метаданих.

Метадані мають бути на початку файлу, починаючи з `---` і закінчуючи `---`. Формат конфігураційної інформації між ними — `YAML`.

Приклад налаштування файлу:

```yml
---

brief: |
  this is a demo brief
  you can write multiline

---

# title

… …
```

`brief` вказує на короткий огляд змісту, який буде відображений на сторінці індексу блогу.

За допомогою `YMAL` '|`Синтаксис, ви можете створювати багаторядкові резюме.

Налаштування бічного меню блогу також здійснюється через файл `TOC` (див. попередній розділ). Лише статті, перелічені в `TOC`, будуть відображені в індексі головної сторінки блогу.

Статті без метаданих не будуть відображені на головній сторінці блогу, але можуть бути присутні в бічному меню.

Якщо потрібно підняти статтю вгору, виконайте `i18n.site` і відредагуйте файл `xxx.yml` у директорії `.i18n/data/blog`, змінивши часовий штамп на від'ємне число (кілька від'ємних чисел будуть відсортовані за зростанням їх абсолютних значень).
