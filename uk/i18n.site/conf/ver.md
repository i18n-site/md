# Версія проекту

Розглянемо приклад демонстраційного проекту:

`en/demo2/v` — це поточний номер версії проекту, який буде відображатися праворуч від назви проекту на бічній панелі.

<img src="https://p.3ti.site/1721290486.avif" width="320px">

Тут `en/` — код мови, що відповідає мові джерела перекладу, налаштованому в `.i18n/conf.yml`.

Якщо ваша мова джерела не англійська, файл `v` слід розмістити в каталозі вашого проекту з мовою джерела.

Функція перегляду історичних версій документів зараз розробляється.

Рекомендується змінювати версію документа лише при випуску значних оновлень (наприклад, `v1`, `v2`), щоб уникнути великої кількості версій, що можуть створити безлад у індексах пошукових систем.

## Використовуйте порожні файли `v` для розділення індексів файлів різних проектів

У демонстраційному проекті, крім `en/demo2/v`, ви також побачите порожні файли `v` у каталогах `en/blog` та `en/demo1`.

Порожній `v` не буде відображатися у бічній панелі, але наявність файлу `v` дозволить створити незалежний індекс для файлів у цьому каталозі та його підкаталогах.

Розділяючи індекси різних проектів, можна уникнути повільного завантаження через одночасне завантаження індексу всіх файлів на сайті.

Наприклад, файл індексу для `blog` у демонстраційному проекті: [https://unpkg.com/i18n.site/en/blog.json](https://unpkg.com/i18n.site/en/blog.json)
