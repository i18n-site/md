# ဘုံပြဿနာ

## မတော်တဆ ဖျက်လိုက်သဖြင့် `npm` ပက်ကေ့ခ်ျကို ထုတ်ဝေရန် ပျက်ကွက်ခဲ့သည် `.i18n/v`

`.i18n/v/ol/v.hash` ပက်ကေ့ဂျ် `npm` ထုတ်ဝေမှု၏ သမိုင်းဝင်ဗားရှင်းကို သိမ်းဆည်းထားသည်။

မတော်တဆ ဖျက်လိုက်လျှင် `npm` ပက်ကေ့ဂျ်ကို ထုတ်ပြန်မည်မဟုတ်ပါ `.i18n/v/ol`

ဤအချိန်တွင်၊ ဥပမာ `0.1.9` သင့်ပရောဂျက်၏ နောက်ဆုံးထုတ်ဗားရှင်း၏ ဗားရှင်းနံပါတ်ကို ဦးစွာရှာပါ [npmjs.com](//npmjs.com)

ထို့နောက် အောက်ပါ `bash` Create ဖိုင်ကို ကိုးကားပါ။

```bash
mkdir -p .i18n/v/ol
echo @0.1.9 > .i18n/v/ol/v.hash
```

ဤနည်းဖြင့် ပြုပြင်ခြင်းသည် ဖိုင်မှတ်တမ်းကို ဆုံးရှုံးစေပြီး နောက်ထွက်ရှိမှုကို တိုးမြှင့်၍ အပ်ဒိတ်လုပ်ရန် မဖြစ်နိုင်တော့ဘဲ အကြောင်းအရာအားလုံးကို ပြန်လည်ထုပ်ပိုးပြီး တစ်ကြိမ် အပ်လုဒ်လုပ်ပါမည်။
