# ထည့်သွင်းပါ &

## ဖွဲ့စည်းမှုသင်္ကေတ

`i18n.site` `i18` ဘာသာပြန်ကိရိယာကို ထည့်သွင်းထားသည်၊ [`i18` ခွင့် တိုကင်ကို ပြင်ဆင်သတ်မှတ်ရန် ဤနေရာကို နှိပ်ပါ](/i18/use) ။

## တပ်ဆင်ပါ။

```sh
bash <(curl -sS https://i.i18n.site) i18n.site
```

## ဒီမိုပရောဂျက်

သရုပ်ပြပရောဂျက်တစ်ခုဖြင့် စတင်ပြီး အသုံးပြုနည်းကို လေ့လာကြပါစို့ `i18n.site`

ကျွန်ုပ်တို့ ပထမဦးစွာ သရုပ်ပြသိုလှောင်ခန်းကို ပွားပြီး အောက်ပါအတိုင်း command ကို run ပါ။

```
git clone https://github.com/i18n-site/demo.i18n.site.git md
git clone https://github.com/i18n-site/demo.i18n.site.docker.git docker
```

တရုတ်ပြည်မကြီးရှိ အသုံးပြုသူများ လုပ်နိုင်သည်-

```
git clone https://atomgit.com/i18n/demo.i18n.site.git md
git clone https://atomgit.com/i18n/demo.i18n.site.docker.git docker
```

! `docker` ဖြင့် ဒေသတွင်း အစမ်းကြည့်ရှုမှုကို လွယ်ကူချောမွေ့စေရန် ကုဒ်အခြေခံကိုယ်ပွား၏ လမ်းညွှန်အမည်သည် `md` ဖြစ်ရပါမည် `demo.i18n.site`

### အကြောင်းအရာကို တင်ပါ။

`i18n.site` [စာမျက်နှာတစ်ခုတည်း အပလီကေးရှင်းတည်ဆောက်ပုံ](https://developer.mozilla.org/docs/Glossary/SPA) တည်ဆောက်မှုကို လက်ခံပြီး ဝဘ်ဆိုက်ဝင်ပေါက် စာမျက်နှာနှင့် ဝဘ်ဆိုက်အကြောင်းအရာကို လွတ်လပ်စွာ အသုံးချသည်။

အထက်ဖော်ပြပါ ဘာသာပြန်ခြင်းကို လုပ်ဆောင်ပြီးနောက်၊ `htm` နှင့် `v` directory များကို `md/out/dev` တွင် ထုတ်ပေးပါမည်။

`.i18n/htm/dev.yml` `dev`

`dev` :

`htm` လမ်းညွှန်အောက်တွင် ဝဘ်ဆိုဒ်ဝင်ပေါက် စာမျက်နှာ။

`v` လမ်းညွှန်တွင် ဗားရှင်းနံပါတ်ပါသော ဝဘ်ဆိုက်အကြောင်းအရာပါရှိသည်။

Local preview သည် ဗားရှင်းနံပါတ်မခွဲခြားဘဲ ဖိုင်အားလုံးကို ဖိုင်လမ်းညွှန်သို့ ကူးယူမည်ဖြစ်သည် `out/dev/v/0.1.0`

တရားဝင်ထုတ်ဝေမှုအတွက်၊ ပြောင်းလဲထားသောဖိုင်များကို ဗားရှင်းနံပါတ်လမ်းညွှန်သို့ ကူးယူပါမည်။

#### configuration file ကို သတ်မှတ်ရန် -c ကိုသုံးပါ။

မတူညီသောဖွဲ့စည်းပုံဖိုင်များသည် `out`

ဥပမာ၊ `.i18n/htm/ol.yml` သည် `out/ol` လမ်းညွှန်တစ်ခု ဖန်တီးလိမ့်မည်။

`dev.yml` နှင့် `ol.yml` တို့သည် မူရင်းပုံစံများဖြစ်သည်။

`dev` သည် ဖွံ့ဖြိုးတိုးတက်မှုပတ်ဝန်းကျင်ကိုကိုယ်စားပြုသည့် `development` ၏အတိုကောက်ဖြစ်ပြီး၊ ဒေသန္တရအကြိုကြည့်ရှုမှုအတွက်အသုံးပြုပြီး မူရင်းဖွဲ့စည်းပုံဖိုင်လည်းဖြစ်သည်။
`ol` သည် အွန်လိုင်းပတ်ဝန်းကျင်ကိုကိုယ်စားပြုသည့် `online` ၏အတိုကောက်ဖြစ်ပြီး တရားဝင်ထုတ်ဝေမှုအတွက်အသုံးပြုပြီး `npm` command line parameter `-n` ကိုအသုံးပြု၍ ထုတ်ဝေသည့်အခါ ပုံသေဖွဲ့စည်းပုံဖိုင်လည်းဖြစ်သည်။

configuration file name ကို သတ်မှတ်ရန် အခြား configuration files များကိုလည်း ဖန်တီးနိုင်သည် `--htm_conf`

ဥပမာအားဖြင့်:
```
i18n.site --htm_conf yourConfig --save
```

ဤတွင် `--save` အပ်ဒိတ်ထုတ်ဝေမှုဗားရှင်းနံပါတ်ကို ညွှန်ပြသည်။

#### <a rel=id href="#npm" id="npm"></a> အကြောင်းအရာကို npmjs.com သို့ လွှင့်တင်ပါ။

[npmjs.com](//npmjs.com) ထုတ်ဝေခြင်းသည် အကြံပြုထားသော မူရင်းဖြေရှင်းချက်ဖြစ်သည် ( [Front-end High Availability](/i18n.site/feature#ha) ကိုကြည့်ပါ)။

##### npm login &

ထည့်သွင်းရန် `nodejs` အကောင့်ဝင်ရန် `npm login` အသုံးပြုပါ။

တည်းဖြတ်ပါ `md/.i18n/htm/ol.yml` `v: //unpkg.com/i18n.site` တွင် `i18n.site` သင့်ကိုယ်ပိုင် `npm` ပက်ကေ့ဂျ်အမည်သို့ ပြောင်းပါ။

ပက်ကေ့ဂျ်အမည်အဖြစ် ဝဘ်ဆိုဒ်ဒိုမိန်းအမည်ကို အသုံးပြုခြင်းသည် ကောင်းသောရွေးချယ်မှုဖြစ်သည် [npmjs.com](//npmjs.com)

`npm` ပက်ကေ့ဂျ်ကို အခြေခံ၍ ထုတ်ဝေသည့်အခါ၊ ထုတ်ဝေမှုအသစ်များကို `v:` နှင့်တပြေးညီကြည့်ရှုနိုင်စေရန်အတွက် ကက်ရှ်အချိန် `i18n.site` cache time `/.v` ၏ရှေ့ဆက်အဖြစ် **`//unpkg.com/` အသုံးပြုပါ** ။

ဘာသာပြန်ရန်နှင့် ထုတ်ဝေရန် `i18n.site --npm` သို့မဟုတ် `i18n.site -n` `md`

ထုတ်ဝေရန် စဉ်ဆက်မပြတ် ပေါင်းစပ် `~/.npmrc` အသုံးပြုပါက၊ ၎င်းကို ထည့်သွင်းရန် မလိုအပ်ပါ `nodejs`

`v:` `ol.yml` တွင် ပက်ကေ့ဂျ်အမည်ကို မွမ်းမံ **ပါက `.i18n/v/ol` ဦးစွာဖျက်ပြီး** ထုတ်ဝေပါ။

##### npm မှထုတ်ဝေသော ပရောက်စီဆာဗာ

တရုတ်ပြည်မကြီးရှိ သုံးစွဲသူများသည် ကွန်ရက်ပြဿနာများနှင့် ကြုံတွေ့ရပြီး `npm` ပက်ကေ့ဂျ်ကို မထုတ်ဝေနိုင်ပါက၊ ၎င်းတို့သည် ပရောက်စီဆာဗာကို configure လုပ်ရန် ပတ်၀န်းကျင်ပြောင်းလဲနိုင်သော `https_proxy` သတ်မှတ်နိုင်သည်။

သင်၏ ပရောက်စီဆာဗာ ပို့တ်သည် `7890` သင်သည် ရေးနိုင်သည်-

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### ကိုယ်တိုင်လက်ခံထားသော အကြောင်းအရာ

အကြောင်းအရာကို ကိုယ်တိုင်လက်ခံကျင်းပလိုပါက ပထမဦးစွာ တည်းဖြတ် `md/.i18n/htm/ol.yml` `v: //i18n-v.xxx.com` ကဲ့သို့သော သင့် URL ရှေ့ဆက်သို့ `v: //unpkg.com/i18n.site` ပြောင်းပါ။

`md` လမ်းညွှန်ကိုရိုက်ထည့်ပြီး run ပါ။

```
i18n.site --htm_conf ol --save
```

သို့မဟုတ် အတိုကောက်

```
i18n.site -c ol -s
```

ထို့နောက်၊ `v:` တွင် သတ်မှတ်ထားသော URL ရှေ့ဆက်လမ်းကြောင်းသို့ `md/out/ol/v` လမ်းညွှန်ရှိ အကြောင်းအရာကို သတ်မှတ်ပါ။

နောက်ဆုံးတွင်၊ **လမ်းကြောင်း၏ ကက်ရှ်အချိန်ကို `/.v` `1s` သတ်မှတ်သတ်မှတ်ပါ** ၊ သို့မဟုတ်ပါက အသစ်ထွက်ရှိထားသော အကြောင်းအရာကို ချက်ချင်းဝင်ရောက်နိုင်မည်မဟုတ်ပါ။

မလိုအပ်သော တောင်းဆိုမှုများကို လျှော့ချရန်အတွက် အခြားလမ်းကြောင်းများအတွက် ကက်ရှ်အချိန်ကို တစ်နှစ် သို့မဟုတ် ထို့ထက်ပို၍ သတ်မှတ်နိုင်သည်။

##### အကြောင်းအရာကို s3 သို့ လက်ခံပါ။

ကိုယ်တိုင်လက်ခံကျင်းပသည့်အကြောင်းအရာအတွက်၊ သင်၏ကိုယ်ပိုင်ဆာဗာကိုအသုံးပြုခြင်းအပြင် အခြားဘုံရွေးချယ်မှုမှာ + `CDN` အသုံးပြုရန်ဖြစ်သည် `S3`

သင်သည် [rclone](https://rclone.org) လော့ဂ်အင် `S3` ဆာဗာကို သုံးနိုင်ပြီး၊ ထို့နောက် အောက်ပါ script ကို ကိုးကားပြီး ပြုပြင်မွမ်းမံကာ သင်ထုတ်ဝေသည့်အကြိမ်တိုင်းတွင် တိုးမြင့်ပြောင်းလဲမှုများကိုသာ `S3` ကူးယူပါ။

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

! သို့မှသာ `/.v` ဖြင့်အဆုံးသတ်သည့်လမ်းကြောင်း၏ cache အချိန်သည် `1s` ဖြစ်သည်၊ သို့မှသာ အသစ်ထွက်ရှိထားသောအကြောင်းအရာကို သင်ချက်ချင်းဝင်ရောက်နိုင်မည်မဟုတ်ကြောင်းကို မှတ်သားထားရန် `CDN`

### ဝဘ်ဆိုဒ်ကိုလွှင့်တင်ပါ။

ဝဘ်ဆိုဒ်ကို နေရာတိုင်းတွင် ဖြန့်ကျက်အသုံးပြုနိုင်ပြီး [cloudflare page](https://pages.cloudflare.com) သည် ရွေးချယ်မှုကောင်းများဖြစ်သည် [github page](https://pages.github.com)

ဝဘ်ဆိုက် [သည် စာမျက်နှာတစ်ခုတည်း အပလီကေးရှင်းတစ်ခု၏](https://developer.mozilla.org/docs/Glossary/SPA) ဗိသုကာလက်ရာကို လက်ခံထားသောကြောင့်၊ `index.html` သို့မပါဝင်သည့် URL လမ်းကြောင်းကို ပြန်လည်ရေးသားရန် မမေ့ပါနှင့် `. `

ဝဘ်ဆိုက် ဝင်ရောက်မှု စာမျက်နှာကို တစ်ကြိမ်သာ အသုံးပြုရန် လိုအပ်ပြီး နောက်ဆက်တွဲ အကြောင်းအရာ အပ်ဒိတ်များအတွက် ဝဘ်ဆိုက် ဝင်ရောက်မှု စာမျက်နှာကို ပြန်လည် အသုံးပြုရန် မလိုအပ်ပါ။

#### တရုတ်ပြည်မကြီးတွင် ဝဘ်ဆိုဒ်ဖြန့်ကျက်မှုကို အကောင်းဆုံးဖြစ်အောင်လုပ်ခြင်း။

တရုတ်ပြည်မကြီး၏ ကွန်ရက်ပတ်ဝန်းကျင်တွင် ပိုမိုကောင်းမွန်သော ဝင်ရောက်နိုင်မှုစွမ်းဆောင်ရည်ကို ရယူလိုပါက၊ [ဒိုမိန်းအမည်ကို ဦးစွာမှတ်ပုံတင်](//beian.aliyun.com) ပါ။

ထို့နောက် တရုတ်ပြည် `out/ol/htm` ကြီးရှိ cloud ရောင်းချသူများ၏ အရာဝတ္ထုသိုလှောင်မှုကို အသုံးပြုပါ + `CDN`

[Baidu Smart `CDN`](//cloud.baidu.com/product/cdn.html) ကဲ့သို့သော စာမျက်နှာတစ်ခုတည်း အပလီကေးရှင်းများနှင့် လိုက်လျောညီထွေဖြစ်အောင် လမ်းကြောင်းကို ပြန်လည်ရေးသားရန် edge computing ကို အသုံးပြု၍ ၎င်းကို ဤကဲ့သို့ configure လုပ်နိုင်ပါသည်။

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```

`MX` မှတ်တမ်းများနှင့် `CNAME` မှတ်တမ်းများသည် အတူမတည်ရှိနိုင်သောကြောင့်၊ သင်သည် ဒိုမိန်းအမည် အီးမေးလ်များကို တစ်ချိန်တည်း လက်ခံရယူလိုပါက၊ `A` record `CNAME` နှင့် ပူးပေါင်းဆောင်ရွက်ရန် လိုအပ်ပါသည် [cname_flatten](https://github.com/i18n-site/lib/tree/main/cname_flatten)

ထို့အပြင်၊ တရုတ်ပြည်မကြီးရှိ cloud ရောင်းချသူများ၏ နိုင်ငံရပ်ခြားအသွားအလာ ကောက်ခံမှုများသည် အတော်လေးစျေးကြီးသောကြောင့် ကုန်ကျစရိတ်များကို ပိုမိုကောင်းမွန်အောင်ပြုလုပ်လိုပါက၊ သင်ရရှိရန် [Huawei Cloud ၏ အခမဲ့ပထဝီဝင် DNS](https://support.huaweicloud.com/usermanual-dns/dns_usermanual_0041.html) နှင့် [Cloudflare for SaaS](https://developers.cloudflare.com/cloudflare-for-platforms/cloudflare-for-saas) ၏ စိတ်ကြိုက်ဒိုမိန်းအမည် (အောက်တွင်ဖော်ပြထားသည့်အတိုင်း) ကို အသုံးပြုနိုင်ပါသည်။ ယာဉ်အသွားအလာလွှဲခြင်း── တရုတ်ပြည်မကြီးရှိ Baidu Cloud တွင် `CDN` နိုင်ငံတကာ အသွားအလာ cloudflare ။

#### .i18n

`.i18n` လမ်းညွှန်အောက်တွင် `i18n.site` 's configuration files, translation cache, etc. အသေးစိတ်အတွက် နောက်အခန်း ["Configuration" ကို](/i18n.site/conf) ကြည့်ပါ။

#### en

စီစဉ်သတ်မှတ်မှုဖိုင်ရှိ `fromTo` `en` `.i18n/conf.yml` သက်ဆိုင်သော အရင်းအမြစ်ဘာသာစကားလမ်းညွှန်

```yaml
i18n:
  fromTo:
    en: zh
```
