# .i18n/conf.yml

`i18n.site` ဖွဲ့စည်းမှုဖိုင်သည် `.i18n/conf.yml` ဖြစ်သည်။

[`i18`](/i18) `ignore:` `i18n:`

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

`.md` အထဲမှ `ext:` ဖွဲ့စည်းမှုပုံစံ `upload`

## ထိပ်တန်းလမ်းညွှန် navigation

`nav:` ပင်မစာမျက်နှာ၏ထိပ်ရှိ လမ်းညွှန်ချက်မီနူးနှင့် သက်ဆိုင်သော ဖွဲ့စည်းမှုရွေးချယ်စရာများ။

`en/i18n.yml` `home: Home` `i18n: home`

`zh/i18n.yml` ကဲ့သို့သော ဘာသာစကားများစွာဖြင့် ဘာသာပြန်ပါမည် `en/i18n.yml`

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

ဘာသာပြန်ဆိုမှုပြီးသွားသောအခါ၊ ဘာသာပြန်မှုတွင် `yml` ၏တန်ဖိုးကို သင်မွမ်းမံနိုင်သော်လည်း ဘာသာပြန်ခြင်း၏ `yml` ကို ထည့်ခြင်း သို့မဟုတ် မဖျက်ပါနှင့်။

### `use: Toc` ဖိုင်ပုံစံတစ်ခုတည်း (ကောက်ကြောင်းပါရှိ)

: `nav`

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` ဆိုသည်မှာ `Markdown` template တစ်ခုတည်းကို rendering ပြုလုပ်သည့် `Toc` rendering ကို အသုံးပြုခြင်း ဖြစ်သည်။

`TOC` သည် `Table of Contents` `Markdown` အတိုကောက်ဖြစ်သည်။

`url:` သည် `Markdown` ၏ ဖိုင်လမ်းကြောင်းကို ကိုယ်စားပြုသည် ( `/` root directory နှင့် သက်ဆိုင်သည် `/README.md` ၊ ဤဖိုင်အမည်သည် စာလုံးအကြီးနှင့် နောက်ဆက်တွဲတစ်ခု လိုအပ်သည်)။

### `use: Md` ဖိုင်ပုံစံတစ်ခုတည်း (ကောက်ကြောင်းမပါ)

`Md` Template သည် `Toc` နှင့် အတူတူပင်ဖြစ်သည်၊ နှစ်ခုလုံးကို `Markdown` ဖိုင်တခုတည်းကို တင်ဆက်ရန်အတွက် အသုံးပြုပါသည်။ သို့သော်လည်း `Md` ပုံစံပလိတ်သည် ဘေးဘားရှိ ကောက်ကြောင်းကို မပြပါ။

`use: Toc` အထက်ဖော်ပြပါဖွဲ့စည်းမှုပုံစံတွင် `use: Md` ၊ run `i18n.site` `md` directory တွင် နောက်တစ်ကြိမ်၊ ပင်မစာမျက်နှာရှိ ပြောင်းလဲမှုများကို စောင့်ကြည့်ရန် ဖွံ့ဖြိုးတိုးတက်မှုအကြိုကြည့်ရှုခြင်း URL သို့ သွားပါ။

### ဖွဲ့စည်းမှုလမ်းကြောင်းမပါဘဲ ပုံသေဖွင့်ခြင်း။

ဝင်ရောက်ကြည့်ရှုနေသည့် လမ်းကြောင်းတစ်ခု၏ လမ်းကြောင်းရှေ့ဆက်ကို `nav:` လမ်းကြောင်းနှင့်သက်ဆိုင်သည့် `MarkDown` ဖိုင်ကို မူရင်းအတိုင်း တင်ပြီး `Md` template ကို အသုံးပြု၍ ပြန်ဆိုပါမည်။

ဥပမာအားဖြင့်၊ အကယ်၍ သင်သည် `/test` နှင့် `nav:` ဤလမ်းကြောင်းမပါဘဲ စီစဉ်သတ်မှတ်ထားပြီး စာမျက်နှာဘာသာစကားမှာ အင်္ဂလိပ် (ကုဒ် `en` ) ၊ ပုံစံပလိတ်ကို မူရင်းအတိုင်း `/en/test.md` ဖွင့်ပြီး ပြန်ဆိုထားသည် `Md` ။

`/en/test.md` ဤဖိုင်မရှိပါက၊ မူရင်း `404` စာမျက်နှာကို ပြသပါမည်။

### `use: Doc` ၊ ဖိုင်အစုံ

ဖွဲ့စည်းမှုဖိုင်တွင်-

```
  - i18n: blog
    use: Doc
```

ပုံစံထုတ်ခြင်းအတွက် `Doc`

`Doc` Template သည် တစ်ခုတည်း သို့မဟုတ် ပရောဂျက်များစွာအတွက် စာရွက်စာတမ်းအကောက်များကို ဖန်တီးရန် `MarkDown` အများအပြားကို ပေါင်းစပ်မှုကို ပံ့ပိုးပေးသည်။

#### ပရောဂျက်တစ်ခု (ဖိုင်များစွာ)

`blog` အထက်တွင်ဖော်ပြထားသည်မှာ `Doc` ပရောဂျက်တစ်ခုတည်းဖြစ်သည်။

```
  - i18n: blog
    url: blog
    use: Doc
```

##### url သည် ဗလာဖြစ်နေသောအခါ၊ ၎င်းသည် i18n တန်ဖိုးသို့ ပုံသေသတ်မှတ်သည်။

အကယ်၍ သင်မရေးပါ `url` , `url` `i18n` ၏တန်ဖိုးဖြစ်သည်။

အထက်ဖော်ပြပါ စာရေးနည်းသည် `url: blog` ၎င်း၏ သက်ဆိုင်ရာ ဖိုင်မှာ `en/blog/TOC` ဖြစ်သည်။

#### TOC အကြောင်းအရာ အညွှန်းကိန်း

လမ်းညွှန်အညွှန်းဖိုင် `doc` ပုံစံပုံစံဖွဲ့စည်းမှုပုံစံနှင့်သက်ဆိုင်သော `TOC` `json` ဖတ်ရန် သရုပ်ပြဂိုဒေါင်ရှိ `js` plugin `.i18n/hook/after.tran/TOC.js` လုပ်ဆောင်ပါမည် `i18n.site`

`doc` template ကိုသုံးပါက၊ ဤ plug-in ရှိရပါမည်။

အလွတ်ဖိုဒါမှ `i18n.site` ပရောဂျက်ကို စတင်ပါက၊ သရုပ်ပြပရောဂျက်ရှိ `.i18n` သင့်လမ်းညွှန်သို့ ကူးယူရန် မမေ့ပါနှင့်။

`Doc` ပုံစံခွက်သည် ထုတ်လုပ်ထားသော `json` ကို အခြေခံ၍ အကြောင်းအရာဇယားကို တင်ဆက်ပေးပါမည်။

##### အစုလိုက် TOC ကို ဘယ်လိုရွှေ့မလဲ။
