# ထည့်သွင်းပြီး အသုံးပြုပါ။

## Windows ကို အရင်ဆုံး git bash ကို install လုပ်ပါ။

windows စနစ်၊ ကျေးဇူးပြု၍ [ဒေါင်းလုဒ်လုပ်ပြီး ထည့်သွင်းရန် ဤနေရာကို နှိပ်ပါ `git bash`](https://git-scm.com/download/win)

နောက်ဆက်တွဲလုပ်ဆောင်မှုများကို လုပ်ဆောင်ပါ `git bash`

## တပ်ဆင်ပါ။

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### ဘာသာပြန်သင်္ကေတကို စီစဉ်သတ်မှတ်ပါ။

တိုကင်ကူးရန် နှိပ်ပါ [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

Create `~/.config/i18n.site.yml` ကူးယူထားသော အကြောင်းအရာကို ၎င်းထဲသို့ ကူးထည့်ပါ၊ အကြောင်းအရာမှာ အောက်ပါအတိုင်းဖြစ်သည်-

```
token: YOUR_API_TOKEN
```

ထို့အပြင်၊ သင်သည် ငွေပေးချေရန်အတွက် ခရက်ဒစ်ကတ်ကို စည်းထား [i18n.site/payBill](//i18n.site/payBill) လိုအပ်သည် (ငွေပြန်သွင်းရန် မလိုအပ်ပါ၊ အသုံးပြုမှုအရ ဝဘ်ဆိုဒ်သည် အခကြေးငွေများကို အလိုအလျောက် နုတ်ယူမည်ဖြစ်ပြီး၊ [ဈေးနှုန်းများအတွက် ပင်မစာမျက်နှာကို ကြည့်ပါ](/#price) )။

## အသုံးပြု

### ဒီမိုပရောဂျက်

ဘာသာပြန်ခြင်းပုံစံ `i18` လေ့လာရန် သရုပ်ပြပရောဂျက်ကို ကြည့်ပါ [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

တရုတ်ရှိအသုံးပြုသူများသည်ကိုယ်ပွားလုပ်နိုင်သည် [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

ပုံတူပွားပြီးနောက်၊ ဘာသာပြန်ခြင်းကို အပြီးသတ်ရန် လမ်းညွှန်ကို ထည့်သွင်းပြီး လုပ်ဆောင်ပါ `i18`

### လမ်းညွှန်ဖွဲ့စည်းပုံ

template warehouse directory structure မှာ အောက်ပါအတိုင်းဖြစ်သည်။

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` လမ်းညွှန်တွင် ဥပမာတစ်ခုသာဖြစ်ပြီး ဖျက်ပစ်နိုင်သည့် ဘာသာပြန်သရုပ်ပြဖိုင်များပါရှိသည်။

### ဘာသာပြန်ခြင်းကို လုပ်ဆောင်ပါ။

ဘာသာပြန်ရန် လမ်းညွှန်ကို ထည့်သွင်းပြီး လုပ်ဆောင်ပါ `i18`

### သိုလှောင်မှုသို့ ဖိုင်များထည့်ပါ။

ဘာသာပြန်ခြင်းအပြင်၊ ပရိုဂရမ်သည် အောက်ပါဖိုင်များကို ထုတ်ပေးမည်ဖြစ်ပြီး၊ ကျေးဇူးပြု၍ ၎င်းတို့ကို သိုလှောင်ရာသို့ ထည့်ပါ။

```
.i18n/hash
.i18n/cache/.gitignore
```

၎င်းတို့အနက်မှ အကြောင်းအရာများ : အောက်ပါအတိုင်းဖြစ်သည် `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

ဆိုလိုသည်မှာ `.i18n/cache/` directory ( `.i18n/cache/.gitignore` မှလွဲ၍) အတွင်းရှိ ဖိုင်အားလုံးကို လျစ်လျူရှုပါ။

သင့်ဗားရှင်းထိန်းချုပ်သည့်ဆော့ဖ်ဝဲမဟုတ်ပါက `git` ဤဖွဲ့စည်းပုံအရ ၎င်းကိုလျစ်လျူရှုပါ။

## ဖွဲ့စည်းမှုဖိုင်

`.i18n/conf.yml` ၎င်းသည် `i18` line ဘာသာပြန်ကိရိယာ၏ ဖွဲ့စည်းမှုဖိုင်ဖြစ်သည်။

အကြောင်းအရာမှာ အောက်ပါအတိုင်းဖြစ်သည်။

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### အရင်းအမြစ်ဘာသာစကား & ဘာသာပြန်ဘာသာစကား

configuration file တွင်၊ လက်အောက်ငယ်သားများ `fromTo`

`en` သည် မူရင်းဘာသာစကားဖြစ်ပြီး `zh ja ko de fr` သည် ဘာသာပြန်ခြင်း၏ ပစ်မှတ်ဖြစ်သည်။

ဘာသာစကားကုဒ်ကိုကြည့်ပါ [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

ဥပမာ၊ တရုတ်ကို အင်္ဂလိပ်လို ဘာသာပြန်ချင်ရင် ဒီစာကြောင်းကို ပြန်ရေးပါ `zh: en`

ပံ့ပိုးပေးထားသော ဘာသာစကားအားလုံးကို ဘာသာပြန်လိုပါက ကျေးဇူးပြု၍ ကွက်လပ်ထားခဲ့ပါ `:` ဥပမာအားဖြင့်

```
i18n:
  fromTo:
    en:
```

### ဖိုင်ကို လျစ်လျူရှုပါ။

မူရင်းအားဖြင့်၊ အရင်းအမြစ်ဘာသာစကားလမ်းညွှန်တွင် `.yml` ဖြင့်စသော ဖိုင်အားလုံးကို ဘာသာပြန်ပါမည် `.md`

အချို့သောဖိုင်များကို လျစ်လျူရှုကာ ၎င်းတို့ကို ဘာသာပြန်ခြင်းမပြုလိုပါက (မပြီးဆုံးသေးသော မူကြမ်းများကဲ့သို့သော) အကွက်ပုံစံကို အသုံးပြုနိုင်သည် `ignore`

`ignore` `.gitignore` နှင့်ဆင်တူသော syntax ကိုအသုံးပြုသည် [globset](https://docs.rs/globset/latest/globset/#syntax)

ဥပမာအားဖြင့်၊ `_` ဖိုင်ဖွဲ့စည်းပုံ `_* `

## ဘာသာပြန်စည်းမျဉ်းများ

### ဘာသာပြန်တည်းဖြတ်သူများသည် စာကြောင်းများထည့်ခြင်း သို့မဟုတ် ဖျက်ခြင်းမပြုသင့်ပါ။

ဘာသာပြန်သည် ပြင်နိုင်သည်။ မူရင်းစာသားကို မွမ်းမံပြီး စက်ဖြင့်ပြန်ဆိုပါက၊ ဘာသာပြန်ရန် လက်စွဲစာအုပ်ကို ပြုပြင်မွမ်းမံမှုများသည် ထပ်ရေးမည်မဟုတ်ပါ (မူရင်းစာသား၏ ဤစာပိုဒ်ကို မပြုပြင်ရသေးပါက)။

သို့သော် ဘာသာပြန်သည့်စာကြောင်းများနှင့် မူရင်းစာသားများသည် တစ်ခုနှင့်တစ်ခု ဆက်စပ်နေရမည်ကို ကျေးဇူးပြု၍ သတိပြုပါ။ ဆိုလိုသည်မှာ ဘာသာပြန်ခြင်းကို ပြုစုရာတွင် စာကြောင်းများထည့်ခြင်း သို့မဟုတ် မဖျက်ပါနှင့်။ မဟုတ်ပါက၊ ၎င်းသည် ဘာသာပြန်တည်းဖြတ်သည့် ကက်ရှ်တွင် ရှုပ်ထွေးမှုများ ဖြစ်စေသည်။

တစ်ခုခုမှားယွင်းနေပါက [ဖြေရှင်းချက်များအတွက် FAQ ကို ကိုးကားပါ။](/i18/qa#H1)

### `YAML`

command line tool သည် source language file directory တွင် `.yml` ဆုံးရှိသော ဖိုင်အားလုံးကို ရှာဖွေပြီး ၎င်းတို့ကို ဘာသာပြန်ပေးပါမည်။

* ဖိုင်အမည်၏ နောက်ဆက်သည် `.yml` ( `.yaml` မဟုတ်ပါ) သတိပြုပါ။

ကိရိယာသည် အဘိဓာန်တန်ဖိုးများကိုသာ ဘာသာပြန်ပေးသည် `.yml` အဘိဓာန်သော့များ မဟုတ်ပါ။

ဥပမာအားဖြင့် `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

ဟု ဘာသာပြန်မည် `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` ၏ ဘာသာပြန်ကို ကိုယ်တိုင်လည်း ပြုပြင်နိုင်ပါသည်။

`YAML` ဘာသာပြန်ခြင်းကို အခြေခံ၍ သင်သည် ပရိုဂရမ်းမင်းဘာသာစကားအမျိုးမျိုးအတွက် နိုင်ငံတကာဖြေရှင်းချက်များကို အလွယ်တကူ ဖန်တီးနိုင်သည်။

## အဆင့်မြင့်အသုံးပြုမှု

### ဘာသာပြန်ဆိုမှု လမ်းကြောင်းခွဲ

သင် `.i18n/conf.yml` နေသမျှကာလပတ်လုံး (အချိန်တိုင်းသရုပ်ပြပရောဂျက်ပုံစံပုံစံမှစတင်ရန်မလိုအပ်ပါ) `i18` သည်ကောင်းမွန်လိမ့်မည်။

command line tool သည် subdirectories အားလုံးတွင် `.i18n/conf.yml` ကိုတွေ့ပြီး ၎င်းကို ဘာသာပြန်ပေးလိမ့်မည်။

[monorepo](//monorepo.tools) ဗိသုကာကို အသုံးပြုသည့် ပရောဂျက်များသည် ဘာသာစကားဖိုင်များကို ဖိုင်တွဲများအဖြစ် ခွဲနိုင်သည်။

<img alt="" src="https://p.3ti.site/1719910016.avif">

### စိတ်ကြိုက်တပ်ဆင်မှုလမ်းညွှန်

၎င်းကို မူရင်းအတိုင်း တပ်ဆင်မည်ဖြစ်သည် `/usr/local/bin`

`/usr/local/bin` ရေးသားခွင့်မရှိပါက ၎င်းကို `~/.bin` တွင် ထည့်သွင်းမည်ဖြစ်သည်။

ပတ်ဝန်းကျင် အပြောင်းအလဲများကို သတ်မှတ်ပါ `TO` ဥပမာအားဖြင့် တပ်ဆင်မှုလမ်းညွှန်ကို သင်သတ်မှတ်နိုင်သည် :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
