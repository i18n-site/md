## Command line ဆန်ဆန်

ဘာသာပြန်စာတမ်းများကို စီမံခန့်ခွဲရန်အတွက် [လေးလံသောကိရိယာများ](https://www.capterra.com/translation-management-software) စွာရှိသည်။

`git` ရင်းနှီးသော ပရိုဂရမ်မာများအတွက် , `vim` , `vscode` ၊ စာရွက်စာတမ်းများကို တည်းဖြတ်ရန်နှင့် ဗားရှင်းများကို စီမံခန့်ခွဲရန် ဤကိရိယာများကို အသုံးပြုခြင်းသည် သင်ယူမှုကုန်ကျစရိတ်ကို သံသယဝင်စရာပင်။

`KISS` ( `Keep It Simple, Stupid` ) နိယာမကို ယုံကြည်သူများကြားတွင်၊ လုပ်ငန်းအဆင့်ဖြေရှင်းချက်များသည် ခက်ခဲခြင်း၊ ထိရောက်မှုမရှိခြင်းနှင့် အသုံးပြုရခက်ခဲခြင်းတို့နှင့် တူညီသည်။

ညွှန်ကြားချက်များကို ထည့်သွင်းပြီး ကလစ်တစ်ချက်နှိပ်ရုံဖြင့် ရှုပ်ထွေးသော ပတ်၀န်းကျင် မှီခိုမှုများ မရှိစေရပါ။

မလိုအပ်ပါက အရာများကို မထည့်ပါနှင့်။

## ပြုပြင်မွမ်းမံခြင်း၊ ဘာသာပြန်ခြင်း မရှိပါ။

[translate-shell](https://github.com/soimort/translate-shell) ကဲ့သို့သော command line ဘာသာပြန်ကိရိယာအချို့လည်း ရှိပါသည်။

သို့ရာတွင်၊ ၎င်းတို့သည် ဖိုင်ပြုပြင်မွမ်းမံမှုများကို ခွဲခြားသတ်မှတ်ခြင်းကို မပံ့ပိုးဘဲ ကုန်ကျစရိတ်လျှော့ချရန်နှင့် ထိရောက်မှုတိုးစေရန်အတွက် ပြုပြင်ထားသောဖိုင်များကိုသာ ဘာသာပြန်ဆိုခြင်းမပြုပါ။

## ဘာသာပြန်ခြင်းကို ပံ့ပိုးပါ `YAML`

ကိရိယာသည် အဘိဓာန်တန်ဖိုးများကိုသာ ဘာသာပြန်ပေးသည် `YAML` အဘိဓာန်သော့များ မဟုတ်ပါ။

`YAML` ဘာသာပြန်ခြင်းကို အခြေခံ၍ သင်သည် ပရိုဂရမ်းမင်းဘာသာစကားအမျိုးမျိုးအတွက် နိုင်ငံတကာဖြေရှင်းချက်များကို အလွယ်တကူ ဖန်တီးနိုင်သည်။

## ဘာသာပြန်ခြင်းကို ပံ့ပိုးပါ `HOGO`

[HOGO](https://github.com/gohugoio/hugo) ငြိမ်ဘလော့ဂ်၏ ခေါင်းစီးဖွဲ့စည်းပုံကို ပံ့ပိုးသည်၊ `title` ၊ `summary` နှင့် `description` အကွက်များကိုသာ ဘာသာပြန်ပါသည်။

## ပြောင်းလဲနိုင်သောအမည်များကို ဘာသာပြန်မထားပါ။

`Markdown` အီးမေးလ်ပုံစံတစ်ခုအနေဖြင့်၊ `YAML` ဘာသာစကားဖိုင်ဖွဲ့စည်းပုံတစ်ခုအနေဖြင့်၊ ပြောင်းလဲနိုင်သော ကန့်သတ်ဘောင်များ ရှိလိမ့်မည် (ဥပမာ- ငွေပြန်သွင်းရန် `${amount}` လက်ခံရရှိထားသည်)။

`${varname}` ဤပြောင်းလဲနိုင်သောအမည်မျိုးကို အင်္ဂလိပ်ဘာသာပြန်အဖြစ် မှတ်ယူမည်မဟုတ်ပါ။

## တရုတ်၊ ဂျပန်နှင့် ကိုရီးယားတို့အတွက် ဘာသာပြန်ဆိုခြင်းအား ပိုမိုကောင်းမွန်အောင်ပြုလုပ်ခြင်း။

### အင်္ဂလိပ်ဘာသာသို့ ပြန်ဆိုသောအခါ၊ ခေါင်းစဉ်၏ ပထမအက္ခရာကို အလိုအလျောက် စာလုံးကြီးဖြင့် ရေးသည်။

တရုတ်၊ ဂျပန်နှင့် ကိုရီးယားတို့သည် စာလုံးအကြီး သို့မဟုတ် စာလုံးသေးများ မရှိသော်လည်း အင်္ဂလိပ် ခေါင်းစဉ်များသည် ယေဘုယျအားဖြင့် ပထမစာလုံးကို စာလုံးကြီးသည်။

`i18` သည် `MarkDown` တွင် ခေါင်းစဉ်များကို မှတ်မိနိုင်ပြီး စာလုံးအကြီးအသေးသို့ ဘာသာပြန်သည့်အခါ ပထမစာလုံးကို အလိုအလျောက် စာလုံးကြီးလုပ်ပါမည်။

ဥပမာ `为阅读体验而优化` `Optimized for Reading Experience` ဟု ဘာသာပြန်ပါမည်။

### တရုတ်၊ ဂျပန်၊ ကိုးရီးယားနှင့် တရုတ်တို့တွင် အင်္ဂလိပ်အသုံးအနှုန်းများကို ဘာသာပြန်ထားခြင်းမရှိပါ။

တရုတ်၊ ဂျပန်နှင့် ကိုရီးယားနိုင်ငံတို့မှ စာရွက်စာတမ်းများတွင် အင်္ဂလိပ်အသုံးအနှုန်းများစွာ ပါဝင်လေ့ရှိသည်။

တရုတ်၊ ဂျပန်နှင့် ကိုရီးယားဘာသာစကားများကို စက်ဖြင့်ဘာသာပြန်ခြင်းသည် အင်္ဂလိပ်မဟုတ်သောဘာသာစကားတစ်ခုဖြစ်လာပြီး ဝေါဟာရများကို အောက်ပါတရုတ်ဝါကျများကဲ့သို့ မကြာခဏ တွဲဆိုထားပါသည်။

`i18` ဘာသာစကားပေါင်းစုံ ဝဘ်ဆိုဒ်တည်ဆောက်ခြင်း [`i18n.site`](/i18n.site) အတွက် command line tool တွင် ပေါင်းစပ်ထားသည်။
