# Установка и использование

## Для Windows сначала установите Git Bash

Для системы Windows, пожалуйста, [нажмите здесь, чтобы сначала загрузить и установить `git bash`](https://git-scm.com/download/win).

Запустите все последующие операции в `git bash`.

## Установить

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### Настройка токена для перевода

Перейдите на [i18n.site/token](//i18n.site/token) и скопируйте токен

<img src="https://p.3ti.site/1719911689.avif" style="width:400px">

Создайте файл `~/.config/i18n.site.yml` и вставьте в него скопированный контент, вот пример:

```
token: YOUR_API_TOKEN
```

Кроме того, вам необходимо [на i18n.site/payBill](//i18n.site/payBill) привязать кредитную карту для оплаты (нет необходимости пополнять счет, сайт будет автоматически списывать средства в зависимости от использования, [тарифы см. на главной странице](/#price)).

## Использование

### Демонстрационный проект

Рекомендуется ознакомиться с демонстрационным проектом [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18) для изучения конфигурации перевода `i18`.

Пользователи из Китая могут клонировать [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

После克隆ирования перейдите в каталог и запустите `i18` для завершения перевода.

### Структура каталогов

Структура каталогов шаблона хранилища следующая:

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

Файлы для демонстрации перевода в каталоге `en` являются merely примером и могут быть удалены.

### Выполнение перевода

Перейдите в каталог и выполните команду `i18` для перевода.

### Добавление файлов в репозиторий

Кроме перевода, программа также generates следующие файлы, добавьте их в репозиторий.

```
.i18n/hash
.i18n/cache/.gitignore
```

Среди них содержимое файла `.i18n/cache/.gitignore` следующее:

```
**/*
!**/.gitignore
```

Это означает, что все файлы в каталоге `.i18n/cache/` (кроме файла `.i18n/cache/.gitignore`) будут ignored.

Если ваше программное обеспечение для управления версиями не `git`, настройте его аналогичным образом.

## Конфигурационный файл

Файл `.i18n/conf.yml` является файлом конфигурации для командного инструмента перевода `i18`

Содержание файла следующее:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Исходный язык и язык перевода

В файле конфигурации `fromTo` включает:

`en` — это исходный язык, `zh ja ko de fr` — языки перевода.

Коды языков см. на [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE).

Например, если вы хотите перевести с китайского на английский, измените строку на `zh: en`.

Если вы хотите перевести на все поддерживаемые языки, оставьте поле после `:` пустым, например:

```
i18n:
  fromTo:
    en:
```

Вы можете настроить различные `fromTo` для разных подкаталогов/файлов, вот пример записи:

```
i18n:
  fromTo:
    en:
  path:
    blog:
      fromTo:
        zh:
    blog/your_file_name.md:
      fromTo:
        ja:
```

В этой таблице конфигураций исходный язык перевода для каталога `blog` — `zh`, а для файла `blog/your_file_name.md` — `ja`.

### Игнорирование файлов

По умолчанию будут переведены все файлы с расширениями `.md` и `.yml` в каталоге исходного языка.

Если вы хотите игнорировать某些 файлы и неtranslate их (например, незаконченные черновики), настройте это с помощью поля `ignore`.

`ignore` использует синтаксис [globset](https://docs.rs/globset/latest/globset/#syntax), аналогичный файлу `.gitignore`.

Например, `_*` в файле конфигурации означает, что файлы, начинающиеся с `_`, не будут переведены.

## Правила перевода

### Редакторы переводов не должны добавлять или удалять строки

Перевод можно редактировать. Если вы измените исходный текст и переведете его снова, ручные изменения в переводе не будут перезаписаны (если исходный текст этого абзаца не был изменен).

Но обратите внимание, что строки перевода и исходного текста должны соответствовать друг другу. То есть не добавляйте и не удаляйте строки при переводе,否则 это может привести к путанице в кэше редактирования переводов.

Если出现了 путаница, обратитесь к [решениям в часто задаваемых вопросах](/i18/qa#H1)

### `YAML` Переводы

Инструмент командной строки найдет все файлы с расширением `.yml` в каталоге исходного языка и переведет их.

* Обратите внимание, что расширение файла должно быть `.yml` (а не `.yaml`).

Инструмент translate только значения словарей в `.yml`, ключи словарей не будут переведены.

Например, `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

будет переведено в `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

Перевод `YAML` также можно вносить ручным способом (но не добавляйте и не удаляйте ключи или строки в переводе).

На основе `YAML` перевода вы можете легко создавать международные решения для различных языков программирования.

## Продвинутые методы

### Перевод подкаталогов

Пока создан файл `.i18n/conf.yml` (нет необходимости начинать с шаблона демонстрационного проекта), `i18` будет работать корректно.

Инструмент командной строки найдет все файлы конфигурации `.i18n/conf.yml` во всех подкаталогах и переведет их.

Проекты, использующие архитектуру [monorepo](//monorepo.tools), могут разбивать языковые файлы по подкаталогам.

![](https://p.3ti.site/1719910016.avif)

### Пользовательский каталог установки

По умолчанию устанавливается в `/usr/local/bin`.

Если у `/usr/local/bin` нет прав на запись, он будет установлен в `~/.bin`.

Установка переменной окружения `TO` может определить каталог установки, например:

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
