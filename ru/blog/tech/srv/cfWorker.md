
## Привязка доменного имени через CNAME

После привязки доменного имени для Cloudflare Worker через `SSL/TLS` → «Настройка的主人» → привязка доменного имени CNAME, при прямом доступе会出现 ошибка `error code: 522`

В этом случае необходимо использовать HTTP-маршрутизацию → маршрутизация для Workers, чтобы привязать маршрут, как показано на рисунке ниже
