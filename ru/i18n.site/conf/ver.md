# Версия Проекта

В качестве примера возьмем демо-проект:

`en/demo2/v` — номер текущей версии проекта, который будет отображаться справа от имени проекта в структуре боковой панели.

<img src="https://p.3ti.site/1721290486.avif" width="320px">

Здесь `en/` — код языка, соответствующий `.i18n/conf.yml` исходному языку перевода.

Если ваш исходный язык не английский, файл `v` следует поместить в каталог проекта на вашем исходном языке. 

Возможность просмотра исторических версий документов находится в стадии разработки.

Рекомендуется изменять номер версии документа только при выпуске крупных обновлений (например `v1` , `v2` ), чтобы не загромождать страницы, индексируемые поисковыми системами, из-за слишком большого количества номеров версий.

## Используйте Пустые Файлы `v` Для Разделения Индексов Файлов Для Разных Проектов.

В демонстрационном проекте помимо `en/demo2/v` вы также можете увидеть, что содержимое каталогов `en/blog` и `en/demo1` пусто `v` files.

Пустой `v` не будет отображаться в контуре боковой панели, но пока файл `v` существует, для файлов в каталоге и подкаталогах будет создан независимый индекс.

Разделив индексы разных проектов, вы сможете избежать медленного доступа, вызванного одновременной загрузкой индекса всех файлов всего сайта.

Например, в демонстрационном проекте соответствующий индексный [https://unpkg.com/i18n.site/en/blog.json](https://unpkg.com/i18n.site/en/blog.json) : `blog`
