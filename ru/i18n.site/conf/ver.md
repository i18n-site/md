# Версия проекта

На примере демонстрационного проекта:

`en/demo2/v` — текущая версия проекта, которая будет отображаться справа от названия проекта в панели оглавления.

<img src="https://p.3ti.site/1721290486.avif" width="320px">

Здесь `en/` — это код языка, соответствующий исходному языку перевода, указанному в файле `.i18n/conf.yml`.

Если ваш исходный язык не английский, файл `v` должен быть помещен в директорию проекта, соответствующую вашему исходному языку.

Функция просмотра истории версий документов находится в стадии разработки.

Рекомендуется изменять номер версии документов только при выпуске значительных обновлений (например, `v1`, `v2`), чтобы избежать чрезмерного количества версий, что может привести к混乱у в индексе поисковых систем.

## Используйте пустые файлы `v` для разделения индексов файлов различных проектов

В демонстрационном проекте, помимо `en/demo2/v`, можно увидеть, что в директориях `en/blog` и `en/demo1` также присутствуют пустые файлы `v`.

Пустой файл `v` не будет отображаться в панели оглавления, но при наличии файла `v` для соответствующей директории и ее подкаталогов будет создан отдельный индекс.

Разделяя индексы для различных проектов, можно избежать замедления доступа, вызванного одновременной загрузкой индекса всех файлов сайта.

Например, файл индекса для раздела `blog` в демонстрационном проекте: [https://unpkg.com/i18n.site/en/blog.json](https://unpkg.com/i18n.site/en/blog.json)
