# .i18n/conf.yml

Файл конфигурации для `i18n.site` — это `.i18n/conf.yml`.

Кроме настроек `ignore:` и `i18n:` для [`i18`](/i18), файл конфигурации имеет следующий вид:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

Где `upload` с параметром `ext:` означает, что при публикации будут загружены только файлы с расширением `.md`.

## Верхняя навигация

Опции конфигурации `nav:` соответствуют навигационному меню в верхней части главной страницы.

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

Здесь `i18n: home` соответствует `home: Home` в `en/i18n.yml`.

`en/i18n.yml` будет переведен на несколько языков, таких как `zh/i18n.yml`.

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

После завершения перевода вы можете изменить значения в `yml`, но не добавлять и не удалять ключи из `yml`.

### `use: Toc`, шаблон для одного файла (с оглавлением)

В конфигурации `nav`:

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` означает использование шаблона `Toc` для рендеринга одного файла `Markdown`.

`TOC` — это аббревиатура от `Table of Contents`. При рендеринге этого шаблона оглавление файла `Markdown` будет отображаться в боковой панели.

`url:` означает путь к файлу `Markdown` ( `/` соответствует корневому каталогу `/README.md`, и имя файла должно начинаться с большой буквы, заканчиваться малой).

### `use: Md`, шаблон для одного файла (без оглавления)

Шаблоны `Md` и `Toc` одинаковы и оба используются для рендеринга одного файла `Markdown`, но шаблон `Md` не отображает оглавление в боковой панели.

Вы можете изменить `use: Toc` на `use: Md` в вышеуказанной конфигурации, затем перезапустить `i18n.site` в директории `md` и посетить URL-адрес для предварительного просмотра, чтобы увидеть изменения на главной странице.

### implicit path loading by default

Если доступ к определенному пути не настроен в `nav:`, то по умолчанию будет загружен соответствующий файл `Markdown` и отрендерен с использованием шаблона `Md`.

Например, при доступе к `/test`, если путь не настроен в `nav:`, а язык страницы — английский (код `en`), то по умолчанию будет загружен `/en/test.md` и отрендерен с использованием шаблона `Md`.

Если файл `/en/test.md` не существует, будет отображена страница `404` по умолчанию.

### `use: Doc`, многофайловый шаблон

В файле конфигурации:

```
  - i18n: blog
    use: Doc
```

Значит, используется `Doc` для рендеринга шаблона.

Шаблон `Doc` поддерживает интеграцию нескольких файлов `Markdown` для создания оглавлений для одного или нескольких проектов.

#### Одиночный проект (множество файлов)

`blog` в предыдущем примере — это режим одиночного проекта `Doc`.

```
  - i18n: blog
    url: blog
    use: Doc
```

##### Когда URL-адрес пуст, по умолчанию используется значение i18n

Если `url` не указан, `url` по умолчанию принимает значение `i18n`. Это правило действует и для других шаблонов.

Приведенный выше метод записи эквивалентен `url: blog`, и соответствующий файл — `en/blog/TOC`.

#### Много проектов

Конфигурация `i18n:doc` в `.i18n/conf.yml` — это режим множества проектов.

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

Здесь `menu: NB demo1,demo2` означает использование шаблона `NB` для отображения раскрывающегося меню.

`NB` — это аббревиатура от `Name Brief`, что означает, что раскрывающееся меню может отображать имя и слоган проекта.

За `NB` следуют параметры `demo1,demo2`.
Заметка: ** между `,` в `demo1,demo2` не должно быть пробелов **.

Соответствующие файлы индекса каталога для вышеперечисленных параметров:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### Указатель Оглавления TOC

`i18n.site` выполнит плагин `js` `.i18n/hook/after.tran/TOC.js` из демонстрационного репозитория для чтения индексного файла каталога `doc`, соответствующего конфигурации шаблона `TOC`, и создания структуры оглавления в формате `json`.

Если вы используете шаблон `doc`, у вас должен быть этот плагин.

Если вы инициируете проект `i18n.site` из пустой директории, не забудьте скопировать демонстрационный `.i18n` в вашу директорию.

Шаблон `Doc` будет рендерить структуру оглавления на основе сгенерированного `json`.

##### Подробное объяснение содержимого

Содержимое `en/blog/TOC` следующее:

```
README.md

news/README.md
  news/begin.md
```

##### Используйте отступы для обозначения уровней

`README.md` в первой строке `en/blog/TOC` соответствует `i18n.site` на рисунке ниже, то есть названию проекта.

Следующие две строки показаны на скриншоте ниже.

<img src="https://p.3ti.site/1721097381.avif" style="width:320px">

`news/README.md` соответствует `News`
`news/begin.md` соответствует `Our Product is Online !`

`TOC` с отступами обозначают иерархическую структуру, и поддерживают многоуровневые отступы.

##### Родительский уровень записывает только заголовок, не содержимое

При наличии нескольких уровней отступов родительский уровень записывает только заголовок, чтобы избежать混乱 в верстке.

##### Документ README.md проекта

Содержимое можно записать в файл `README.md`, например `en/demo2/README.md`.

Обратите внимание, что содержимое этого файла не включает оглавление, поэтому рекомендуется ограничить его длину и написать краткое введение.

###### Слоган проекта

Вы можете видеть, что `Deme Two` имеет слоган проекта под раскрывающимся меню и в структуре каталога `Your Project slogan`.

![](https://p.3ti.site/1721276842.avif)

Это соответствует первой строке `en/demo2/README.md`

```
# Demo Two : Your Project slogan
```

Содержание после первого двоеточия `:` в заголовке первого уровня файла `README.md` будет рассматриваться как слоган проекта.

Пользователям из Китая, Японии и Кореи обратите внимание, что следует использовать полужирное двоеточие `:` вместо全媒体ного.

##### Как Переместить TOC Оптом?

`TOC` должен быть в директории исходного языка.

Например, если исходный язык — китайский, то `TOC` будет `zh/blog/TOC`.

Если исходный язык изменен, вам нужно будет массово переместить `TOC` для определенного языка в проекте в другой язык.

Вы можете использовать следующие команды:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

Пожалуйста, измените `en/` и `zh/` в вышеуказанной команде на код вашего языка.
