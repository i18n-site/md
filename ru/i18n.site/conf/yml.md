# .i18n/conf.yml

Профиль для `i18n.site` — это `.i18n/conf.yml`.

За исключением настроек [`i18`](/i18) , `ignore:` и `i18n:` , файл конфигурации выглядит следующим образом:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
addon:
  - i18n.addon/toc
```

在那里 `upload` с `ext:` указывает, что только `.md` файлы будут загружены при публикации.

## Верхняя навигация

Опции конфигурации `nav:` соответствуют верхнему навигационному меню на главной странице.

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

Там `i18n: home` соответствует `home: Home` в `en/i18n.yml`.

`en/i18n.yml` будет переведен на несколько языков, таких как `zh/i18n.yml`.

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

После завершения перевода вы можете изменить значения в `yml`, но не добавлять или удалять ключи `yml`.

### `use: Toc`, шаблон для одного файла (с оглавлением)

В конфигурации `nav`:

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` означает использование шаблона `Toc` для рендеринга одного `Markdown` шаблона.

`TOC` — это сокращение от `Table of Contents`. Этот шаблон при рендеринге显示侧边栏 с оглавлением файла `Markdown`.

`url:` означает путь к `Markdown` файлу ( `/` соответствует корневому каталогу `/README.md`, имя файла должно быть с заглавной буквы в начале и строчными буквами в конце).

### `use: Md`, шаблон для одного файла (без оглавления)

Шаблоны `Md` и `Toc` одинаковы и оба используются для рендеринга одного `Markdown` файла, но шаблон `Md` не отображает оглавление в侧边ной панели.

Вы можете изменить `use: Toc` на `use: Md` в вышеуказанной конфигурации, затем перезапустить `i18n.site` в директории `md` и посетить URL-адрес для предварительного просмотра разработок, чтобы увидеть изменения на главной странице.

### По умолчанию загрузка без пути конфигурации

Если доступ к определенному пути осуществляется, и префикс этого пути не настроен в `nav:`, то по умолчанию будет загружен соответствующий пути `Markdown` файл и отрендерен с использованием шаблона `Md`.

Например, если доступ осуществляется к `/test`, и путь не настроен в `nav:`, а язык страницы — английский (код `en`), то по умолчанию будет загружен `/en/test.md` и отрендерен с использованием шаблона `Md`.

Если `/en/test.md` файл не существует, будет отображена стандартная `404` страница.

### `use: Doc`, многофайловый шаблон

В файле конфигурации:

```
  - i18n: blog
    use: Doc
```

Значит используется `Doc` для рендеринга шаблона.

Шаблон `Doc` поддерживает интеграцию нескольких `Markdown` файлов для создания оглавлений для одного или нескольких проектов.

#### Одиночный проект (множество файлов)

`blog` в предыдущем тексте — это одиночный режим `Doc`.

```
  - i18n: blog
    url: blog
    use: Doc
```

##### Когда URL-Адрес Пуст, По Умолчанию Используется Значение i18n

Если `url` не указан, `url` по умолчанию принимает значение `i18n`. Это правило действует и для других шаблонов.

Приведенный выше метод записи эквивалентен `url: blog`, соответствующий файл — `en/blog/TOC`.

#### Множественные проекты

Конфигурация `i18n:doc` в `.i18n/conf.yml` — это режим множественных проектов.

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

Здесь `menu: NB demo1,demo2` означает использование шаблона `NB` для рендеринга выпадающего меню.

`NB` — это сокращение от `Name Brief`, что означает, что выпадающее меню может отображать имя и слоган проекта.

`NB` следует за параметром `demo1,demo2`.
Заметка: ** Между `demo1,demo2` не должно быть пробелов до и после запятой `,`**.

Для вышеуказанных параметров соответствующие файлы каталога:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### Указатель оглавления TOC

`i18n.site` выполнит демонстрационное хранилище `js` плагина `.i18n/hook/after.tran/TOC.js` для чтения индексного файла каталога `doc`, соответствующего шаблону конфигурации `TOC`, чтобы создать структуру оглавления в `json`.

Если вы используете шаблон `doc`, у вас должен быть этот плагин.

Если вы инициализируете `i18n.site` из пустой директории, не забудьте скопировать демонстрационный проект `.i18n` в свою директорию.

Шаблон `Doc` будет рендерить структуру оглавления на основе сгенерированного `json`.

##### Подробное объяснение содержимого

Содержание `en/blog/TOC` следующее:

```
README.md

news/README.md
  news/begin.md
```

##### Используйте отступы для обозначения уровней

Первая строка `README.md` в `en/blog/TOC` соответствует `i18n.site` на рисунке ниже, то есть названию проекта.

Следующие две строки показаны на рисунке ниже.

<img src="https://p.3ti.site/1721097381.avif" style="width:320px">

`news/README.md` соответствует `News`
`news/begin.md` соответствует `Our Product is Online !`

Файлы `TOC` имеют отступы, обозначающие иерархическую структуру, поддерживают многоуровневые отступы и комментарии к строкам, начинающимся с `#`.

##### Родительский уровень записывает только заголовок, не содержимое

При наличии нескольких уровней отступов родительский уровень записывает только заголовок, а не содержимое, чтобы избежать нарушения форматирования.

##### Проект README.md

Содержание можно записать в `README.md` файлах, например `en/demo2/README.md`.

Обратите внимание, что содержимое этого файла не отображается в оглавлении, поэтому рекомендуется ограничить его размер и написать краткое введение.

###### Слоган проекта

Вы можете видеть, что `Deme Two` имеет слоган проекта под выпадающим меню и в структуре каталога `Your Project slogan`.

![](https://p.3ti.site/1721276842.avif)

Это соответствует первой строке `en/demo2/README.md`

```
# Demo Two : Your Project slogan
```

Содержание после первого двоеточия `:` в заголовке первого уровня `README.md` файла проекта будет рассматриваться как слоган проекта.

Пользователям из Китая, Японии и Кореи обратите внимание, что следует использовать полужирное двоеточие `:` вместо全媒体ширинного.

##### Как Переместить TOC Оптом?

`TOC` должен быть в директории исходного языка.

Например, если исходный язык — китайский, то `TOC` будет `zh/blog/TOC`.

Если исходный язык изменен, вам нужно оптом переместить `TOC` файлы определенного языка в другой язык в проекте.

Вы можете использовать следующие команды:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

Пожалуйста, измените `en/` и `zh/` в приведенной выше команде на код вашего языка.
