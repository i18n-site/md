# نصب او کارول

## وینډوز لومړی د git bash نصب کړئ

windows ، [لومړی د ډاونلوډ او نصبولو لپاره دلته کلیک وکړئ `git bash`](https://git-scm.com/download/win)

راتلونکی عملیات په کې پرمخ وړئ `git bash`

## نصب کړئ

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### د ژباړې نښه ترتیب کړئ

د نښه کولو لپاره کلیک وکړئ [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

جوړ کړئ `~/.config/i18n.site.yml` کاپي شوي مواد په کې پیسټ کړئ، مینځپانګه په لاندې ډول ده:

```
token: YOUR_API_TOKEN
```

سربیره پردې ، تاسو اړتیا لرئ د تادیې لپاره کریډیټ کارت [i18n.site/payBill](//i18n.site/payBill) (د ریچارج ته اړتیا نشته ، ویب پا به په اوتومات ډول د کارونې سره سم فیسونه کموي ، [د نرخ لپاره کورپاڼه وګورئ](/#price) ).

## کارول

### د ډیمو پروژه

`i18` ژباړې ترتیب زده کولو لپاره د ډیمو پروژې ته مراجعه وکړئ [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

په چین کې کاروونکي کولی شي کلون کړي [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

د کلون کولو وروسته، ډایرکټر داخل کړئ او د ژباړې بشپړولو لپاره چل کړئ `i18`

### د لارښود جوړښت

د ټیمپلیټ ګودام لارښود جوړښت په لاندې ډول دی

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` لارښود کې ژباړل شوي ډیمو فایلونه شامل دي، کوم چې یوازې یو مثال دی او حذف کیدی شي.

### ژباړه پرمخ وړئ

ډایرکټر داخل کړئ او د ژباړې لپاره لاړ شئ `i18`

### په ذخیره کې فایلونه اضافه کړئ

د ژباړې سربیره، برنامه به لاندې فایلونه هم تولید کړي، مهرباني وکړئ په ذخیره کې یې اضافه کړئ.

```
.i18n/hash
.i18n/cache/.gitignore
```

د `.i18n/cache/.gitignore` منځپانګه په لاندې ډول ده :

```
**/*
!**/.gitignore
```

دا پدې مانا ده چې په `.i18n/cache/` کې ټولې فایلونه له پامه غورځول کیږي (پرته له `.i18n/cache/.gitignore` ).

که ستاسو نسخه کنټرول سافټویر نه وي `git` مهرباني وکړئ د دې ترتیب سره سم یې له پامه غورځوئ.

## د ترتیب کولو فایل

دا د `i18` لاین ژباړې وسیلې تشکیلاتو فایل `.i18n/conf.yml`

منځپانګه په لاندې ډول ده:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### د ژباړې ژبه &

د تشکیلاتو په فایل کې، د لاندې `fromTo`

سرچینه ژبه ده `en` `zh ja ko de fr` د ژباړې هدف ژبه ده.

د ژبې کوډ وګورئ [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

د مثال په توګه، که تاسو غواړئ چینایي په انګلیسي کې وژباړئ، دا کرښه بیا ولیکئ `zh: en`

که تاسو غواړئ چې ټولو ملاتړو ژبو ته وژباړئ، مهرباني وکړئ وروسته خالي پریږدئ `:` د مثال په ډول

```
i18n:
  fromTo:
    en:
```

### فایل له پامه غورځول

په ډیفالټ کې، ټولې فایلونه `.md` د سرچینې ژبې لارښود کې پیل کیږي او `.yml` سره به ژباړل شي.

که تاسو غواړئ چې ځینې فایلونه له پامه غورځوئ او ژباړئ (لکه نامکمل مسودې)، تاسو کولی شئ `ignore` ساحې ترتیب وکاروئ.

[globset](https://docs.rs/globset/latest/globset/#syntax) `.gitignore` په څیر یو ترکیب کاروي `ignore`

د مثال په توګه، د پورته ترتیب `_* ` کې معنی دا ده چې `_` سره پیل شوي فایلونه به ژباړل نشي.

## د ژباړې قواعد

### د ژباړې مدیران باید کرښې اضافه یا حذف نکړي

ژباړه د تدوین وړ ده. اصلي متن تعدیل کړئ او ماشین یې بیا وژباړئ، په ژباړې کې لاسي بدلونونه به له سره نه لیکل کیږي (که د اصلي متن دا پراګراف نه وي بدل شوی).

مګر مهرباني وکړئ په یاد ولرئ چې د ژباړې کرښې او اصلي متن باید یو له یو سره مطابقت ولري. دا دی، د ژباړې د راټولولو په وخت کې لیکې مه اضافه یا حذف کړئ. که نه نو، دا به د ژباړې د ترمیم زیرمه کې ګډوډي رامنځته کړي.

که یو څه غلط شي، مهرباني وکړئ [د حل لپاره FAQ ته مراجعه وکړئ.](/i18/qa#H1)

### ژباړه `YAML`

د کمانډ لاین وسیله به ټولې فایلونه د سرچینې ژبې فایل لارښود کې `.yml` ته ورسوي او ژباړې یې کړي.

* په یاد ولرئ چې د فایل نوم ضمیمه باید `.yml` (نه `.yaml` ).

دا وسیله یوازې د لغت ارزښتونه ژباړي `.yml` نه د لغت کیلي.

د مثال په ډول `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

په توګه به ژباړل شي `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` هم په لاسي ډول تعدیل کیدی شي (مګر په ژباړه کې کیلي یا لینونه مه اضافه کړئ).

`YAML` ژباړې پر بنسټ، تاسو کولی شئ په اسانۍ سره د مختلفو پروګرامونو ژبو لپاره نړیوال حلونه جوړ کړئ.

## پرمختللی کارول

### د ژباړې فرعي لارښود

تر هغه چې تاسو جوړ `.i18n/conf.yml` (هر ځل د ډیمو پروژې ټیمپلیټ څخه پیل کولو ته اړتیا نلري)، `i18` به ښه کار وکړي.

د کمانډ لاین وسیله به په ټولو فرعي ډایرکټرونو کې `.i18n/conf.yml` ومومي او وژباړي.

هغه پروژې چې [monorepo](//monorepo.tools) کاروي کولی شي د ژبې فایلونه په فرعي لارښودونو ویشي.

<img alt="" src="https://p.3ti.site/1719910016.avif">

### د ګمرک نصبولو لارښود

دا به په ډیفالټ کې نصب شي `/usr/local/bin`

`/usr/local/bin` د لیکلو اجازه نه وي نو دا به `~/.bin` ته نصب شي.

د چاپیریال متغیرات تنظیم `TO` تاسو کولی شئ د نصبولو لارښود تعریف کړئ، د بیلګې په توګه :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
