## Користити

### Демо Пројекат

Молимо погледајте демо пројекат да научите конфигурацију `i18` [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

Корисници у Кини могу да клонирају [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

Након клонирања, уђите у директоријум и покрените `i18`

### Структура Именика

Структура директоријума складишта шаблона је следећа

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

Директоријум `en` садржи преведене демо датотеке, које су само пример и могу се избрисати.

### Покрени Превод

Унесите именик и покрените `i18`

### Додајте Датотеке У Спремиште

Поред превода, програм ће такође генерисати следеће датотеке, додајте их у спремиште.

```
.i18n/hash
.i18n/cache/.gitignore
```

: њима, садржај `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

То значи да занемарите све датотеке у `.i18n/cache/` (осим `.i18n/cache/.gitignore` ).

Ако ваш софтвер за контролу верзија није `git` занемарите га у складу са овом конфигурацијом.

## Конфигурациони Фајл

`.i18n/conf.yml` је конфигурациони фајл алата за превођење `i18` линије

Садржај је следећи:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Изворни Језик &

У конфигурационој датотеци, подређени `fromTo`

`en` је изворни језик, `zh ja ko de fr` је циљни језик превода.

Шифра језика види [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

На пример, ако желите да преведете кинески на енглески, препишите овај ред `zh: en`

Ако желите да преведете на све подржане језике, оставите празно након `:` на пример

```
i18n:
  fromTo:
    en:
```

### Игнорисати Датотеку

Подразумевано, све датотеке које почињу са `.md` и `.yml` у директоријуму изворног језика биће преведене.

Ако желите да занемарите одређене датотеке и да их не преведете (као што су недовршени нацрти), можете користити конфигурацију поља `ignore`

Користи синтаксу сличну `.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax) `ignore`

На пример, у горњој конфигурационој датотеци `_* ` да датотеке које почињу са `_` неће бити преведене.

## Правила Превођења

### Уредници Превода Не Би Требало Да Додају Или Бришу Редове

Превод се може уређивати. Измените оригинални текст и поново га машински преведите, ручне измене превода неће бити преписане (ако овај параграф оригиналног текста није измењен).

Али имајте на уму да редови превода и оригиналног текста морају одговарати један према један. То јест, немојте додавати или брисати редове приликом састављања превода. У супротном, то ће изазвати забуну у кешу за уређивање превода.

Ако нешто крене наопако, погледајте [ФАК за решења.](/i18/qa#H1)

### `YAML` Превести

Алат командне линије ће пронаћи све датотеке које се завршавају са `.yml`

* Имајте `.yaml` уму да суфикс имена датотеке мора бити `.yml`

Алат преводи само вредности речника у `.yml` а не кључеве речника.

на пример `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

биће преведено као `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

Превод `YAML` се такође може мењати ручно (али немојте додавати или брисати кључеве или редове у преводу).

На `YAML` превода, можете лако да направите међународна решења за различите програмске језике.

## Напредно Коришћење

### Поддиректориј Превода

Све док креирате `.i18n/conf.yml` нема потребе да сваки пут почињете од шаблона демо пројекта), `i18` ће радити добро!

Алат командне линије ће пронаћи конфигурацију `.i18n/conf.yml`

Пројекти који користе архитектуру [monorepo](//monorepo.tools) могу поделити језичке датотеке у поддиректоријуме.

### Прилагођени Директоријум За Инсталацију

Подразумевано ће бити инсталиран на `/usr/local/bin`

Ако нема дозволу за писање, биће инсталиран на `~/.bin` `/usr/local/bin`

Подесите променљиве `TO` Можете дефинисати директоријум за инсталацију, на пример :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
