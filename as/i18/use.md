## ব্যৱহাৰ

### ডেমো প্ৰজেক্ট

অনুবাদৰ বিন্যাস শিকিবলৈ অনুগ্ৰহ কৰি ডেমো প্ৰকল্প `i18` [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

চীনত ব্যৱহাৰকাৰীয়ে ক্ল'ন কৰিব পাৰে [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

ক্ল'নিং কৰাৰ পিছত, ডাইৰেকটৰীত প্ৰৱেশ কৰক আৰু অনুবাদ সম্পূৰ্ণ কৰিবলৈ `i18`

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
```

### অনুবাদ চলাওক

ডাইৰেকটৰীত প্ৰৱেশ কৰক আৰু অনুবাদ কৰিবলৈ `i18`

### ভঁৰাললৈ ফাইলসমূহ যোগ কৰক

অনুবাদৰ উপৰিও প্ৰগ্ৰামটোৱে নিম্নলিখিত ফাইলসমূহো সৃষ্টি কৰিব, অনুগ্ৰহ কৰি ভঁৰালত যোগ কৰক।

```
.i18n/hash
.i18n/cache/.gitignore
```

: ভিতৰত `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

ইয়াৰ অৰ্থ হৈছে `.i18n/cache/` পঞ্জিকাৰ সকলো নথিপত্ৰ আওকাণ কৰক ( `.i18n/cache/.gitignore` বাহিৰে) ।

যদি আপোনাৰ সংস্কৰণ নিয়ন্ত্ৰণ চফ্টৱেৰ নহয় `git` অনুগ্ৰহ কৰি এই বিন্যাস অনুসৰি ইয়াক আওকাণ কৰক ।

## বিন্যাস নথিপত্ৰ

`.i18n/conf.yml` ই `i18` শাৰী অনুবাদ সঁজুলিৰ বিন্যাস নথিপত্ৰ

বিষয়বস্তু তলত দিয়া ধৰণৰ:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### উৎস ভাষা অনুবাদ ভাষা &

বিন্যাস নথিপত্ৰত, `fromTo`

`en` `zh ja ko de fr`

ভাষাৰ ক'ড চাওক [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

উদাহৰণস্বৰূপে, যদি আপুনি চীনা ভাষা ইংৰাজীলৈ অনুবাদ কৰিব বিচাৰে তেন্তে এই শাৰীটো পুনৰ লিখক `zh: en`

যদি আপুনি সকলো সমৰ্থিত ভাষালৈ অনুবাদ কৰিব বিচাৰে, অনুগ্ৰহ কৰি `:` উদাহৰণ স্বৰূপে

```
i18n:
  fromTo:
    en:
```

### ফাইলটো আওকাণ কৰক

অবিকল্পিতভাৱে, উৎস ভাষা পঞ্জিকাত `.md` আৰু `.yml` ৰে আৰম্ভ হোৱা সকলো নথিপত্ৰ অনুবাদ কৰা হ'ব ।

যদি আপুনি কিছুমান নথিপত্ৰ আওকাণ কৰিব বিচাৰে আৰু সিহতক অনুবাদ কৰিব বিচাৰে (যেনে অসমাপ্ত খচৰা), আপুনি `ignore`

`.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax) দৰে এটা বাক্যবিন্যাস ব্যৱহাৰ কৰে `ignore`

উদাহৰণস্বৰূপ, ওপৰৰ বিন্যাস নথিপত্ৰত `_* ` ৰ অৰ্থ হ'ল `_` ৰে আৰম্ভ হোৱা নথিপত্ৰসমূহ অনুবাদ কৰা নহ'ব ।

## অনুবাদৰ নিয়ম

### অনুবাদ সম্পাদকসকলে শাৰী যোগ বা মচি পেলাব নালাগে

অনুবাদ সম্পাদনাযোগ্য। মূল লিখনী পৰিবৰ্তন কৰক আৰু ইয়াক পুনৰ মেচিন-অনুবাদ কৰক, অনুবাদৰ হস্তচালিত পৰিবৰ্তনসমূহ অভাৰৰাইট কৰা নহ'ব (যদি মূল লিখনীৰ এই অনুচ্ছেদ পৰিবৰ্তন কৰা হোৱা নাই)।

কিন্তু মন কৰিব যে অনুবাদৰ শাৰী আৰু মূল লিখনী এটাৰ লগত এটাৰ লগত মিল থাকিব লাগিব। অৰ্থাৎ অনুবাদ কম্পাইল কৰাৰ সময়ত শাৰী যোগ বা মচি নিদিব। অন্যথা অনুবাদ সম্পাদনা কেচত বিভ্ৰান্তিৰ সৃষ্টি কৰিব।

যদি কিবা ভুল হয়, অনুগ্ৰহ কৰি [সমাধানৰ বাবে FAQ চাওক।](/i18/qa#H1)

### `YAML` অনুবাদ কৰা

আদেশ শাৰী সঁজুলিয়ে উৎস ভাষা নথিপত্ৰ পঞ্জিকাত `.yml`

* মন কৰিব যে ফাইলৰ নামৰ প্ৰত্যয়টো হ'ব লাগিব `.yml` ( `.yaml` নহয়) ।

সঁজুলিটোৱে কেৱল অভিধানৰ মানসমূহ `.yml` ত অনুবাদ কৰে, অভিধানৰ কি'সমূহ নহয়।

উদাহৰণ স্বৰূপে `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

অনুবাদ কৰা হ'ব `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML`

`YAML` ভিত্তিত আপুনি বিভিন্ন প্ৰগ্ৰেমিং ভাষাৰ বাবে আন্তঃৰাষ্ট্ৰীয় সমাধান সহজে নিৰ্মাণ কৰিব পাৰে।

যেতিয়ালৈকে আপুনি সৃষ্টি `.i18n/conf.yml` (প্ৰতিবাৰেই ডেমো প্ৰজেক্ট টেমপ্লেটৰ পৰা আৰম্ভ কৰাৰ প্ৰয়োজন নাই), `i18` ভালদৰে কাম কৰিব।

আদেশ শাৰী সঁজুলিয়ে সকলো উপডাইৰেকটৰীত `.i18n/conf.yml` বিচাৰি পাব আৰু ইয়াক অনুবাদ কৰিব ।

### স্বনিৰ্বাচিত সংস্থাপন পঞ্জিকা

ইয়াক অবিকল্পিতভাৱে `/usr/local/bin`

`~/.bin` `/usr/local/bin`

আপুনি সংস্থাপন পঞ্জিকা সংজ্ঞায়িত কৰিব পাৰে `TO` উদাহৰণস্বৰূপ :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
