# .i18n/conf.yml

বিন্যাস নথিপত্ৰ হৈছে `.i18n/conf.yml` `i18n.site`

[`i18`](/i18) ৰ `ignore:` আৰু `i18n:` সংহতিসমূহৰ বাহিৰে, বিন্যাস নথিপত্ৰ নিম্নলিখিত:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

ইয়াৰ ভিতৰত, `upload` ৰ `ext:` বিন্যাস বস্তুৰ অৰ্থ হ'ল প্ৰকাশ কৰাৰ সময়ত কেৱল `.md` আপলোড কৰা হ'ব।

### `use: Md` একক ফাইল সাঁচ (কোনো ৰূপৰেখা নাই)

`Md` `Toc` ৰ সৈতে একে, দুয়োটাকে এটা `Markdown` ফাইল ৰেণ্ডাৰ কৰিবলৈ ব্যৱহাৰ কৰা হয়। কিন্তু `Md` সাঁচটোৱে কাষৰ বাৰত ৰূপৰেখা দেখুৱাব নোৱাৰে।

আপুনি ওপৰৰ বিন্যাসত `use: Toc` `use: Md` পৰিবৰ্তন কৰিব পাৰে, চলাব পাৰে `i18n.site` `md` পঞ্জিকাত পুনৰায়, আৰু তাৰ পিছত হোমপেজত পৰিবৰ্তনসমূহ পৰ্যবেক্ষণ কৰিবলে বিকাশ পূৰ্বদৰ্শন URL চাওক ।
