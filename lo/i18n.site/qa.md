# ບັນຫາທົ່ວໄປ

## ຖືກ `npm` ໂດຍບັງເອີນ `.i18n/v`

`.i18n/v/ol/v.hash` ສະບັບປະຫວັດສາດຂອງ `npm` ຊຸດປ່ອຍຖືກບັນທຶກໄວ້ໃນ !

ຖ້າເຈົ້າລຶບໂດຍບັງເອີນ `.i18n/v/ol` ຊຸດ `npm` ຈະບໍ່ຖືກເຜີຍແຜ່.

ໃນ​ເວ​ລາ​ນີ້, ທໍາ​ອິດ​ຊອກ​ຫາ​ຈໍາ​ນວນ​ຂອງ​ສະ​ບັບ​ຫລ້າ​ສຸດ​ຂອງ​ໂຄງ​ການ​ຂອງ `0.1.9` ​ໃນ [npmjs.com](//npmjs.com)

ຫຼັງ​ຈາກ​ນັ້ນ​, ອີງ​ໃສ່​ການ​ສ້າງ​ໄຟລ​໌​ຂ້າງ​ລຸ່ມ `bash` ​.

```bash
mkdir -p .i18n/v/ol
echo @0.1.9 > .i18n/v/ol/v.hash
```

ຈົ່ງຈື່ໄວ້ວ່າການສ້ອມແປງດ້ວຍວິທີນີ້ຈະສູນເສຍປະຫວັດໄຟລ໌, ເຮັດໃຫ້ມັນເປັນໄປບໍ່ໄດ້ທີ່ຈະເພີ່ມການປັບປຸງການປ່ອຍຕໍ່ໄປ, ແລະເນື້ອຫາທັງຫມົດຈະຖືກຫຸ້ມຫໍ່ຄືນໃຫມ່ແລະອັບໂຫລດຄັ້ງດຽວ.
