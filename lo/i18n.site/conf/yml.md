# .i18n/conf.yml

`i18n.site` ໄຟລ໌ການຕັ້ງຄ່າແມ່ນ `.i18n/conf.yml` .

ຍົກເວັ້ນການຕັ້ງຄ່າ `ignore:` ແລະ [`i18`](/i18) `i18n:` ໄຟລ໌ການຕັ້ງຄ່າມີດັ່ງນີ້:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

ໃນບັນດາພວກເຂົາ, `ext:` ລາຍການການຕັ້ງຄ່າຂອງ `upload` ຫມາຍຄວາມວ່າພຽງແຕ່ `.md` ຈະຖືກອັບໂຫລດເມື່ອເຜີຍແຜ່.

`en/i18n.yml` ຈະຖືກແປເປັນຫຼາຍພາສາເຊັ່ນ: `zh/i18n.yml` .

### `use: Toc` , ແມ່ແບບໄຟລ໌ດຽວ (ມີໂຄງຮ່າງ)

: ​ຕັ້ງ​ຄ່າ `nav`

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` ຫມາຍຄວາມວ່າໃຊ້ `Toc` rendering, ເຊິ່ງແມ່ນ rendering `Markdown` ດຽວ.

`TOC` ແມ່ນຕົວຫຍໍ້ຂອງ `Table of Contents` `Markdown`

`url:` `Markdown` `/` `/README.md`

### `use: Md` , ແມ່ແບບໄຟລ໌ດຽວ (ບໍ່ມີໂຄງຮ່າງ)

`Md` Template ແມ່ນຄືກັນກັບ `Toc` , ທັງສອງແມ່ນໃຊ້ເພື່ອສະແດງໄຟລ໌ `Markdown` ດຽວ. ແນວໃດກໍ່ຕາມ `Md` ແມ່ແບບບໍ່ໄດ້ສະແດງໂຄງຮ່າງໃນແຖບດ້ານຂ້າງ.

ທ່ານ `md` ມາດປັບປຸງ `i18n.site` `use: Md` `use: Toc`

### ການໂຫຼດເລີ່ມຕົ້ນໂດຍບໍ່ມີເສັ້ນທາງການຕັ້ງຄ່າ

ຖ້າຄໍານໍາຫນ້າເສັ້ນທາງຂອງເສັ້ນທາງສະເພາະໃດຫນຶ່ງທີ່ກໍາລັງເຂົ້າເຖິງບໍ່ໄດ້ຖືກຕັ້ງຄ່າໃນ `nav:` `MarkDown` ໄຟລ໌ທີ່ສອດຄ້ອງກັບເສັ້ນທາງຈະຖືກໂຫລດໂດຍຄ່າເລີ່ມຕົ້ນແລະສະແດງໂດຍໃຊ້ `Md` template.

ຕົວຢ່າງ, ຖ້າທ່ານໄປຢ້ຽມຢາມ `/test` ແລະ `nav:` ຖືກຕັ້ງຄ່າໂດຍບໍ່ມີເສັ້ນທາງນີ້, ແລະພາສາຫນ້າແມ່ນພາສາອັງກິດ (ລະຫັດ `en` ), ແມ່ແບບຈະຖືກໂຫລດ `/en/test.md` ແລະສະແດງໂດຍໃຊ້ `Md` ໂດຍຄ່າເລີ່ມຕົ້ນ.

ຖ້າ `/en/test.md` ໄຟລ໌ນີ້ບໍ່ມີ, ຄ່າເລີ່ມຕົ້ນ `404` ໜ້າຈະສະແດງ

### `use: Doc` , ແມ່ແບບຫຼາຍໄຟລ໌

ໃນ​ໄຟລ​໌​ການ​ຕັ້ງ​ຄ່າ​:

```
  - i18n: blog
    use: Doc
```

ຊີ້ບອກການໃຊ້ `Doc`

`Doc` ແມ່ແບບຮອງຮັບການລວມຫຼາຍ `MarkDown` ເພື່ອສ້າງໂຄງຮ່າງເອກະສານສໍາລັບໂຄງການດຽວ ຫຼືຫຼາຍໂຄງການ.

#### ໂຄງການດຽວ (ຫຼາຍໄຟລ໌)

`blog` ໃນຂ້າງເທິງນີ້ແມ່ນຮູບແບບໂຄງການດຽວຂອງ `Doc` .

##### ວິທີການຍ້າຍ TOC ຈໍານວນຫລາຍ?

`TOC` ໄຟລ໌ຕ້ອງຖືກວາງໄວ້ໃນໄດເລກະທໍລີຂອງພາສາຕົ້ນສະບັບ.

ຕົວຢ່າງ, ຖ້າພາສາຕົ້ນສະບັບເປັນພາສາຈີນ, `TOC` ເທິງແມ່ນ `zh/blog/TOC` .

ຖ້າພາສາຕົ້ນສະບັບຖືກດັດແກ້, ທ່ານຈໍາເປັນຕ້ອງໄດ້ຍ້າຍ `TOC` ໃນບາງພາສາໃນໂຄງການໄປຫາພາສາອື່ນ.

ທ່ານສາມາດອ້າງອີງເຖິງຄໍາສັ່ງຕໍ່ໄປນີ້:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```
