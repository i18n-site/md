  It is convenient to directly reference in the Markdown initialization script and also be used in the packaged website simultaneously, so as to avoid duplicate code.

## Back End

The construction of the documentation site only requires the front end, not the back end.

This back end is the back end of the i18n.site website itself, including systems such as users, payments, and message push.

* Service [github](https://github.com/i18n-api/srv) / [atomgit](https://atomgit.com/i18n-api/srv)
* Plugin [github](https://github.com/i18n-api/pub) / [atomgit](https://atomgit.com/i18n-api/pub)
* Basic Library [github](https://github.com/i18n-site/rust) / [atomgit](https://atomgit.com/i18n/rust)
