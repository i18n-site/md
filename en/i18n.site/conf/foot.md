# Footer

Still taking the demo project as an example, `.i18n/htm/foot.pug` in the `md` directory defines the footer of the website.

![](https://p.3ti.site/1721286077.avif)

[`pug`](https://pugjs.org) is a template language for generating `HTML`.

[➔ Click here to learn pug's grammar](https://pugjs.org)

**Never write `css` or `js` in `foot.pug`**, or errors will occur.

Please refer to the following. Write the styles into the corresponding `.css` files, and implement interactions by creating web components.

## Footer Styles

In the demo project, there are three `css` files under `md/.i18n/htm`.

* `foot.css`: Footer styles
* `import.css`: The default styles for the entire `i18n.site`
* `conf.css`: Icons and fonts for the footer

### Icon Font

The footer icons are generated by creating a font named `F` on iconfont.cn ( [English version](https://www.iconfont.cn/?lang=en-us) / [Chinese version](https://www.iconfont.cn/?lang=zh)).

Please create your own icon fonts as required and replace the following configurations in `conf.css`:

```
@font-face {
  font-family: "F";
  src: url("//at.alicdn.com/t/c/font_4281991_urfar2m2zce.woff2?t=1716188208767") format("woff2");
}
```

## Web Components

`js` cannot be written in `foot.pug`. If interaction is required, please customize web components.

You can define [web components](https://www.freecodecamp.org/news/build-your-first-web-component/) in `md/.i18n/htm/index.js` and then use them in `foot.pug`.

Creating web components is easy, such as customizing the `<x - img>` tag.

```js
customElements.define(
  'x-img',
  class extends HTMLElement {
    constructor() {
      super();
      var img = document.createElement('img');
      img.src = '//p.3ti.site/i18n.svg';
      img.style = "height:99px;width:99px;";
      this.append(img);
    }
  }
)
```

## Multilingual Footer

The following is the code in `.i18n/htm/foot.pug`:

```
#Ft
  b
    a.site(href="/")
    b ${I18N.C}
```

Here, `${I18N.C}` corresponds to the following in `en/i18n.yml`:

```
C: Power By <a class="a" href="https://i18n.site">i18n.site</a>
```

By using expressions like `${I18N.xxx}` in combination with `i18n.yml`, multilingual internationalization of the footer can be achieved.

Adding `class = "a"` to the link is to prevent the link from being converted into `MarkDown` format. See:
 [➔ `YAML`: How to prevent `HTML` links from being converted to `Markdown`](/i18/qa#H2).
