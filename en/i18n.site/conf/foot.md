# Footer

Still taking the demo project as an example, the footer of the website `.i18n/htm/foot.pug` defined in the `md` directory.

<img alt="" src="https://p.3ti.site/1721286077.avif">

[`pug`](https://pugjs.org) is a template language that generates `HTML` .

[➔ Click here to learn the grammar of pug](https://pugjs.org)

**Don't write `css` and `js` in `foot.pug`** , otherwise there will be errors.

Please refer to the following, the style is written in the corresponding `.css` and the interaction is achieved by creating web components.

## Footer Style

In the demo project `md/.i18n/htm` There are 3 `css` files below.

* `foot.css` : Footer style
* `import.css` : `i18n.site` style for the entire site
* `conf.css` : Footer icons and fonts

### Icon Font

The footer icon is generated by iconfont.cn ( [English version](https://www.iconfont.cn/?lang=en-us) /[中文版](https://www.iconfont.cn/?lang=zh)) creating a font named `F`

Please create your own icon font as needed and replace : following configuration in `conf.css`

```
@font-face {
  font-family: "F";
  src: url("//at.alicdn.com/t/c/font_4281991_urfar2m2zce.woff2?t=1716188208767") format("woff2");
}
```

## Web Components

`foot.pug` cannot write `js` in it. If interaction is required, please customize the web page component.

You can define [a web component](https://www.freecodecamp.org/news/build-your-first-web-component/) in `md/.i18n/htm/index.js` and then use the component in `foot.pug` .

It's easy to create web components, like custom tags `<x-img>`

```js
customElements.define(
  'x-img',
  class extends HTMLElement {
    constructor() {
      super();
      var img = document.createElement('img');
      img.src = '//p.3ti.site/i18n.svg';
      img.style = "height:99px;width:99px;";
      this.append(img);
    }
  }
)
```

## Multilingual Footer

`.i18n/htm/foot.pug` The code in it is as follows :

```
#Ft
  b
    a.site(href="/")
    b ${I18N.C}
```

Here `${I18N.C}` The corresponding one is in `en/i18n.yml` :

```
C: Power By <a class="a" href="https://i18n.site">i18n.site</a>
```

Using `${I18N.xxx}` similar to this writing method, combined with `i18n.yml` , you can achieve multi-language internationalization of the footer.

Add `class="a"` to the link to avoid the link being converted to `MarkDown` , see :
 [: `YAML` to prevent `HTML` of the link from being converted to `Markdown`](/i18/qa#H2) .
