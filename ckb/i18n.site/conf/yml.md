# .i18n/conf.yml

فایلە ڕێکخستنەکە بریتییە لە `.i18n/conf.yml` `i18n.site`

جگە لە ڕێکخستنەکانی `ignore:` و `i18n:` [`i18`](/i18)

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

لە نێوانیاندا بابەتی `ext:` ڕێکخستنی `upload` واتە تەنها `.md` لە کاتی بڵاوکردنەوەدا باردەکرێت.

## سەرەوە گەشتیاری nav

هەڵبژاردەکانی ڕێکخستن، `nav:` هاوتایە لەگەڵ مینیوی گەشتکردن لە سەرەوەی لاپەڕەی سەرەکی.

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

لە `en/i18n.yml` `home: Home` `i18n: home`

وەرگێڕانی بۆ چەند زمانێک، وەک `zh/i18n.yml` `en/i18n.yml`

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

دوای تەواوبوونی `yml` دەتوانیت دەستکاری بەهای `yml`

### `use: Toc` , قاڵبی تاکە پەڕگە (بە هێڵکاری)

: `nav`

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` بە واتای بەکارهێنانی `Toc` rendering کە ڕەندەرکردنی تاکە `Markdown` template ە.

`TOC` `Table of Contents` `Markdown`

`url:` ڕێڕەوی پەڕگەی `Markdown` ( `/` هاوتایە لەگەڵ بەڕێوەبەرایەتی ڕەگ `/README.md` , ئەم ناوی پەڕگەیە پێویستی بە پێشگرێکی گەورە و پاشگری پیتی بچووکە).

### `use: Md` , قاڵبی تاکە پەڕگە (بێ هێڵکاری)

`Md` `Toc` `Markdown` بەڵام `Md` قاڵبەکە هێڵکارییەکە لە پەڕەی لایەنیدا نیشان نادات.

دەتوانیت `i18n.site` ڕێکخستنی `md` `use: Md` `use: Toc`

### بارکردنی پێشوەختە بەبێ ڕێڕەوی ڕێکخستن

ئەگەر پێشگرەکەی ڕێڕەوی ڕێڕەوێکی دیاریکراو کە دەستی پێدەگات لە `nav:` پەڕگەی `MarkDown` کە هاوتایە لەگەڵ ڕێڕەوەکە بە شێوازی پێشوەختە باردەکرێت و بە بەکارهێنانی قاڵبی `Md` ڕەندەر دەکرێت.

`en` `Md` `nav:` `/en/test.md` `/test`

ئەگەر `/en/test.md` ئەم پەڕگەیە بوونی نەبێت، ئەوا `404` لاپەڕەی پێشوەختە پیشان دەدرێت.

<img src="https://p.3ti.site/1721184299.avif" style="width:360px">

### `use: Doc` , قاڵبی فرە فایل

لە فایلە ڕێکخستنەکەدا:

```
  - i18n: blog
    use: Doc
```

ئاماژەیە بۆ بەکارهێنانی `Doc` بۆ ڕەندەری قاڵب.

`Doc` `MarkDown`

#### تاکە پڕۆژە (چەند فایلێک)

لە سەرەوە دۆخی تاکە پرۆژەی `Doc` `blog`

```
  - i18n: blog
    url: blog
    use: Doc
```

##### کاتێک url بەتاڵ دەبێت، بە پێشوەختە بەهای i18n دەبێت

`url` `url` `i18n`

شێوازی نووسینەکەی سەرەوە یەکسانە بە هەبوونی `url: blog` و فایلە هاوبەشەکەی `en/blog/TOC` .

#### چەندین پڕۆژە

`.i18n/conf.yml` ڕێکخستنەکە لە `i18n:doc` دۆخی فرە پڕۆژەیە.

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

لێرەدا، `menu: NB demo1,demo2` واتە بەکارهێنانی `NB` بۆ ڕەندەرکردنی مینیوی دابەزین.

`NB` `Name Breif`

`NB` `demo1,demo2`
تێبینی: نابێت هیچ بۆشایی ** پێش `demo1,demo2` دوای کۆما `,` لە : **

بۆ ئەو پارامێتەرانەی سەرەوە، پەڕگەی ئیندێکسی بەڕێوەبەرایەتی هاوبەش بریتییە لە:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### ئیندێکسی خشتەی ناوەڕۆکی TOC

`i18n.site` `js` `.i18n/hook/after.tran/TOC.js` لە کۆگای دیمۆدا جێبەجێ دەکات بۆ خوێندنەوەی پەڕگەی ئیندێکسی بەڕێوەبەرایەتی `doc` هاوتایە لەگەڵ ڕێکخستنی قاڵبەکە `TOC` بۆ دروستکردنی `json` هێڵکاری بەڕێوەبەرایەتی.

ئەگەر `doc`

ئەگەر لە فۆڵدەرێکی بەتاڵەوە `.i18n` `i18n.site`

`Doc` `json`

دوو دێڕی داهاتوو وەک لە وێنەی شاشەی خوارەوەدا دیارە.

##### ئاستی دایک و باوک تەنها ناونیشان دەنووسێت نەک ناوەڕۆک.

کاتێک چەندین ئاستی ئیندێنتکردن هەبێت، ئاستی دایک و باوک تەنها ناونیشانەکە دەنووسێت و ناوەڕۆکەکە نانووسێت. ئەگەرنا تایپۆگرافی تێکدەچێت.

##### پڕۆژەی README.md 

`README.md` پڕۆژەکە، بۆ نموونە دەتوانیت ناوەڕۆک بە `en/demo2/README.md` بنووسیت.

تێبینی بکە ناوەڕۆکی ئەم پەڕگەیە هێڵکاری خشتەی ناوەڕۆک نیشان نادات، بۆیە باشترە درێژییەکەی سنووردار بکرێت و پێشەکییەکی کورت بنووسرێت.

وەک `Deme Two` لە خوارەوەی مینیوی دابەزین و ناوی پڕۆژەکە هێڵکاری کەتەلۆگەکە، دروشمی پڕۆژەکەی `Your Project slogan` :

: هاوتایە لەگەڵ دێڕی یەکەم `en/demo2/README.md`

`README.md` ناوەڕۆکی دوای کۆلۆنی یەکەم `:` لە ناونیشانی ئاستی یەکەمدا وەک دروشمی پڕۆژەکە سەیر دەکرێت.

بەکارهێنەرانی چین و ژاپۆن و کۆریا، تکایە ئاگاداربن کە دەبێت کۆلۆنی نیو پانایی بەکاربهێنن `:`

##### چۆن TOC بە کۆ بگوازینەوە؟

`TOC` پێویستە فایلەکە لە ناوەڕۆکی زمانی سەرچاوەدا دابنرێت.

بۆ نموونە ئەگەر زمانی سەرچاوە `zh/blog/TOC` بێت، ئەوا `TOC`

ئەگەر زمانی سەرچاوە دەستکاری کرا، پێویستە `TOC` بە زمانێکی دیاریکراو لە پرۆژەکەدا بگوازیتەوە بۆ زمانێکی تر.

دەتوانیت ئاماژە بەم فرمانە بکەیت:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

تکایە دەستکاری `en/` و `zh/` لە فەرمانەکەی سەرەوە بکە بۆ کۆدی زمانەکەت.
