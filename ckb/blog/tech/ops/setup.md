## سێرڤەری کڕینی contabo

contabo دابینکەرێکی VPS هەرزانە لە ئەڵمانیا کە مێژوویەکی دوور و درێژی هەیە (زیاتر لە 20 ساڵ) و بەکارهێنەرێکی زۆری هەیە (زیاتر لە 200,000)، بۆیە پێویست ناکات خەمی ڕاکردنت هەبێت.

VPS کڕین [contabo.com](//contabo.com) لەبیرت بێت نرخی یۆرۆ هەڵبژێریت، کە هەرزانترە.

<img alt="" src="https://i-01.eu.org/1718285782.avif">

پارەی دانانی یەکجارەکی هەیە بۆ کۆنتابۆ.
بەڵام هەموو مانگێک چەند ڕۆژێک هەیە کە کرێی دامەزراندنەکە بەخۆڕاییە تەنها ناوت تۆمار بکە و گرنگی بە ئیمەیڵە بانگەشەییەکان بدە.

جگە لە کرێی ناوچەی ئازاد لە ئەوروپا، ناوچەکانی دیکە ئاستی جیاوازی بەرزبوونەوەی نرخیان هەیە.

من سێرڤەرێکی ئەوروپیم هەڵبژارد بەهەر حاڵ دوای بەکارهێنانی cloudflare جیاوازییەکی زۆر لە نێوان سێرڤەرەکاندا نییە.

بەکارهێنەرانی نوێ پێویستە ناسنامەیان پێشکەش بکەن بۆ دڵنیابوون لە ناوی ڕاستەقینە لە کاتی کڕیندا.

## ڕێکخستنی چوونەژوورەوەی بێ وشەی نهێنی

دوای چەند خولەکێک سێرڤەرەکان هەموویان چالاک بوون.

یەکەم `ssh-copy-id root@c2` IP `~/.ssh/config`

<img alt="" src="https://i-01.eu.org/1718287198.avif">

دەتوانیت دەستکاری ئەم [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) بکەیت بۆ ئەوەی ڕێکخستنی دەستپێکردن خێراتر تەواو بکەیت.

دوای تەواوبوونی ڕێکخستنی بێ وشەی نهێنی، دەتوانیت بەکاری [pdsh](https://github.com/chaos/pdsh) دەتوانرێت بە وەجبە کارپێبکرێت، کە ئەمەش گونجاوە.

یەکەمیان ئەوەیە کە بە دەستی ipv6 چالاک بکەیت و تاقیکردنەوە بکەیت

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

<img alt="" src="https://i-01.eu.org/1718287969.avif">

پاشان بە یەک کلیک سیستەمەکە بەرز بکەرەوە بۆ ubuntu 24 و نەرمەکاڵا بەکارهێنراوەکانی باو دابمەزرێنە.

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 واتە گۆڕینی زمان بۆ چینی و ناوچەی کاتی بۆ پەکین.

ئەو نەرمەکاڵایەی کە بە شێوازی پێشوەختە دامەزراوە بەپێی [direnv](https://direnv.net/) [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) کەسی من ڕێکخراوە، وەک [mise](https://github.com/jdx/mise)

ئەم هەنگاوە کاتێکی زۆری دەوێت دوای چاوەڕوانی نیو کاتژمێر، نوێکردنەوە و دامەزراندنی نەرمەکاڵا بەکارهێنراوەکان سەرکەوتوو دەبێت. دەتوانیت بەم فرمانە پشتڕاستی بکەیتەوە:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## mosh خێراتر دەکات ssh

تۆڕەکە لە ئەوروپاوە بۆ چین جێگیر `ssh` پێشنیار دەکرێت بەکاری بهێنیت [mosh](//mosh.org)

دروستکردنی `c0` سکریپت لە `~/.bin` ئامێری ناوخۆیی (بە شێوەیەکی سەرەکی زیادکراوە بۆ گۆڕاوەی ژینگە `PATH` ) بەم شێوەیە:

```bash
#!/usr/bin/env باش

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

دوای دروستکردن، مۆڵەتی جێبەجێکراو و سۆفت-لینک `c2` `c1`

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

لە داهاتوودا `c1` `mosh` بچیتە [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html) `c0`

چاوەڕێ بکە تا سێرڤەرەکە دووبارە دەستپێبکاتەوە

گۆڕینی پەنجەمۆر کاتێک ڕوودەدات ssh بەکاری `ssh-keygen -R ip` بۆ پاککردنەوەی

پاشان ئەم سکریپتەى خوارەوە جێبەجێ بکە بۆ گۆڕینی پارتیشنەکە

```bash
sleep 1 && tail -f /tmp/nohup.out
```

سکریپتەکە بە شێوەیەکی ئۆتۆماتیکی سیستەمەکە دەستپێدەکاتەوە دوای ئەوەی گۆڕانکارییەکە سەرکەوتوو بوو.

### بینینی ڕێژەی پاڵەپەستۆی btrfs

بە شێوازی پێشوەختە بە شێوەیەکی ئۆتۆماتیکی پاڵپێوەنانی دەبێت فرمان بۆ پشکنینی ڕێژەی پاڵەپەستۆ بەم شێوەیە:

```bash
if ! command -v compsize &>/dev/null; then
apt install -y  btrfs-compsize
fi
compsize -x /
```

### پاڵەپەستۆی دەستی

ئەگەر تۆ کەسێکی تەواوکاریت، دەتوانیت بە دەستی پاڵپێوەنانی دیسکێکی تەواو ئەنجام بدەیت فرمانەکە بەم شێوەیە

```bash
btrfs filesystem defragment -r -czstd /
```

وەک لە وێنەی خوارەوەدا دیارە، دەتوانن بزانن کە دوای پاڵپێوەنانی دەستی لە سیستەمی نوێدا، نزیکەی ١٠٠ مێگابایت زیاتر پاڵەپەستۆ کراوە.

### دروستکردنی پاشەکەوتی وێنەی خێرا Btrfs بە snapper

باشترین [snapper](https://manual.siduction.org/sys-admin-btrfs-snapper_en.html) بۆ Btrfs backup.

سکریپتی سەرەتاییکردن دایرێکتۆری `~/ubuntu` دروست دەکات، تەنها ئەم فرمانەی خوارەوە جێبەجێ بکە:

```
~/ubuntu/snapper/init.sh
```

لە ناوەڕۆکی سیناریۆکەدا دەتانبینمەوە [github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh](https://github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh)

`snapper list-configs` بەکاربهێنە بۆ بینینی ئەو ژێر بەرگانەی کە وێنەی خێرایان بۆ دانراوە.

`/etc/snapper/configs/mnt` دەتوانیت ڕێکخستنە تایبەتەکانی پاشەکەوتی ئەم ژێر بەرگە ببینیت

## وردەکارییەکانی سیستەمی ڕزگارکردن

داخڵ بکە بۆ سیستەمی ڕزگارکردن دوای ئەوەی کە دەروازەکە [tcping](https://github.com/paradiseduo/tcping) دەستڕاگەیشتن بوو 22 بۆ ماوەی 2-3 خولەک ناتوانیت بە وشەی نهێنی دانراو دەستی پێ بگات تەنها ساتێک چاوەڕێ بکە و باش دەبێت.

```
tcping ip 22 -c 9999
```

دەتوانرێت چاودێری دەستڕاگەیشتن بە بەندەرەکان بکرێت.

### چۆنێتی دەستگەیشتن بە هارد دیسک دوای چوونە ژوورەوەی rescue &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

دەتوانیت بۆ نموونە وێنەی خێراکە ببینیت

```
snapper -c etc list
```

بینینی گۆڕانکارییەکانی پەڕگەی وێنەی خێرا

```
snapper -c etc status 5..6
```

بینینی دەقی جیاوازی تایبەت

```
snapper -c etc diff 3..5
```

گەڕاندنەوەی وێنەی خێرا: بۆ نموونە، هەموو گۆڕانکارییەکان دوای وێنەی خێرا 5 پووچەڵ بکەرەوە

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### بەردەوامبوونی لۆگ چالاک بکە

چالاککردنی بەردەوامیی لۆگ لە ژێر سیستەمی ڕزگارکردندا پێویستی بەوەیە کە بە دەستی بەڕێوەبەرایەتیی گونجاو دروست بکەیت و هەندێک هەنگاوی زیادە ئەنجام بدەیت، چونکە لەوانەیە خزمەتگوزاری systemd ی کارپێکراوت نەبێت. لێرەدا ئەو هەنگاوانە دەخەینەڕوو کە دەتوانیت تاقی بکەیتەوە:

سەرەتا chroot، پاشان

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

دووبارە هەڵکردنەوە بۆ ناو سیستەمی ئوبونتوەکەت: دوای ئەنجامدانی ئەو گۆڕانکاریانەی سەرەوە، پێویستە ئامێرەکەت بە شێوەیەکی ئاسایی و بۆ ناو سیستەمی ئوبونتوەکەت دووبارە هەڵبکەیتەوە لەبری سیستەمی ڕزگارکردن.

بەم شێوەیە systemd-journald بە ڕێکخستنە نوێیەکە دەست پێدەکات و دەست دەکات بە هەڵگرتنی لۆگەکان بە بەردەوامی لەسەر هارد دیسکەکە.

لەبیرت بێت کە ئەو کارانەی سەرەوە لەسەر سیستەمی ڕزگارکردن ئەنجام دەدرێن، بۆیە ئەو گۆڕانکاریانە تەنها کاتێک کاریگەر دەبن کە سیستەمەکە دووبارە هەڵکرایەوە و چیتر لە دۆخی ڕزگارکردندا نەبێت.

ئەمەش واتە یەکسەر نابینیت هیچ گۆڕانکارییەک لە لۆگەکانی ئێستادا بەردەوام بێت، بەڵام هەموو ئەو لۆگانەی دوای ئەمە دروست دەبن بەردەوام دەبن.

بەم شێوەیە دەتوانیت لۆگەکانی خزمەتگوزاری ssh ببینیت

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## تێبینی لەسەر نەخۆشییە سەخت و ئاڵۆزەکان

### چارەسەری کێشە سەیرەکە بکە: سێرڤەرەکە ناتوانێت ssh بکات

پێشتر github actionم بەکارهێنا بۆ دروستکردنی سکریپتی کۆمپایڵکردن و بڵاوکردنەوەی ئۆتۆماتیکی، بەڵام بۆم دەرکەوت کە خزمەتگوزاری بڵاوکردنەوەکە ناتوانێت ssh بێت.

چوونە ناو سیستەمی ڕزگارکردن و سەیرکردنی لۆگەکان پرسێکی مۆڵەتە:

`Missing privilege separation directory: /run/sshd`

دەتوانیت بەم فرمانەی خوارەوە ئەم دایرێکتۆرییە دروست بکەیت

```
chown root:root /
systemd-tmpfiles --create
```

دوای بەدواداچوونە دووبارەکان، لەبەر ئەوەیە github action ئەو بەکارهێنەرەی tar ئەپلۆد `1001` `uid`

tar Unpacking دەبێت بەبێ پارامێتەر ئەنجام بدرێت `--no-same-owner` پەستان دەخرێتە سەر `uid` بۆ `1001` بەکارهێنەر

لە ئەنجامدا rsync گروپی بەکارهێنەری بەڕێوەبەرایەتی ڕەگ دەگۆڕێت، و پاشان ناتوانیت بچیتە ناو ssh دوای دووبارە دەستپێکردنەوە.

دەستکاری سکریپتی داماڵینی پاڵەپەستۆ بکە بەم شێوەیە:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
