### פּאָסטן אינהאַלט

`i18n.site` אַדאַפּץ אַ [איין-בלאַט אַפּלאַקיישאַן](https://developer.mozilla.org/docs/Glossary/SPA) אַרקאַטעקטשער, און די וועבזייטל אַרייַנגאַנג בלאַט און וועבזייטל אינהאַלט זענען דיפּלויד ינדיפּענדאַנטלי.

נאָך פליסנדיק די אויבן איבערזעצונג, `htm` און `v` Directory וועט זיין דזשענערייטאַד אין די `md/out/dev`

דאָ, `dev` ינדיקייץ אַז עס איז געבויט באזירט אויף די `.i18n/htm/dev.yml` טעקע.

אונטער : `dev`

`htm` דער וועגווייַזער איז די אַרייַנגאַנג בלאַט.

`v` דער וועגווייַזער כּולל די וועבזייטל אינהאַלט מיט ווערסיע נומער.

לאקאלע פאָרויסיקע ווייַזונג וועט נאָכמאַכן אַלע טעקעס צו די `out/dev/v/0.1.0` ראַגאַרדלאַס פון די ווערסיע נומער.

פֿאַר באַאַמטער מעלדונג, טשיינדזשד טעקעס וועט זיין קאַפּיד צו די נייַע ווערסיע נומער וועגווייַזער.

#### ניצן -c צו ספּעציפיצירן די קאַנפיגיעריישאַן טעקע

פאַרשידענע קאַנפיגיעריישאַן טעקעס וועט מאַכן קאָראַספּאַנדינג דיירעקטעריז אונטער די `out` .

פֿאַר `out/ol` , `.i18n/htm/ol.yml`

`dev.yml` און `ol.yml` זענען די פעליקייַט קאַנפיגיעריישאַנז.

`dev` איז די אַבריווייישאַן פון `development` , וואָס רעפּראַזענץ די אַנטוויקלונג סוויווע, איז געניצט פֿאַר היגע פאָרויסיקע ווייַזונג, און איז אויך די פעליקייַט קאַנפיגיעריישאַן טעקע.
`ol` איז די אַבריווייישאַן פון `online` , וואָס רעפּראַזענץ די אָנליין סוויווע, איז געניצט פֿאַר באַאַמטער מעלדונג, און איז אויך די פעליקייַט קאַנפיגיעריישאַן טעקע ווען ארויסגעבן צו `npm` ניצן די באַפֿעל שורה פּאַראַמעטער `-n` .

איר קענען אויך שאַפֿן אנדערע קאַנפיגיעריישאַן טעקעס ניצן `--htm_conf`

למשל:
```
i18n.site --htm_conf yourConfig --save
```

`--save` ינדיקייץ די דערהייַנטיקן מעלדונג ווערסיע.

#### <a rel=id href="#npm" id="npm"></a> אַרויסגעבן אינהאַלט צו npmjs.com

ארויסגעבן אינהאַלט [צו](/i18n.site/feature#ha) [npmjs.com](//npmjs.com)

##### npm login &

ינסטאַלירן `nodejs` נוצן `npm login` צו קלאָץ אין.

רעדאַגירן `md/.i18n/htm/ol.yml` טוישן `i18n.site` אין `v: //unpkg.com/i18n.site` צו דיין אייגענע `npm` פּעקל נאָמען.

נאָר נוצן דעם נאָמען פון דעם פּעקל אויף [npmjs.com](//npmjs.com)

ווען `i18n.site` באזירט אויף די `npm` `/.v` , **זיין זיכער צו נוצן `//unpkg.com/`** ווי די פּרעפיקס פון די `v:` ווערט.

לויפן `i18n.site --npm` אָדער `i18n.site -n` אין די `md` צו איבערזעצן און אַרויסגעבן.

אויב איר נוצן אַ קעסיידערדיק ינטאַגריישאַן סוויווע צו אַרויסגעבן, עס איז ניט דאַרפֿן צו ינסטאַלירן `nodejs` נאָר נאָכמאַכן די לאָגד-אין און ארויס דערלויבעניש `~/.npmrc` צו די סוויווע.

אויב איר מאָדיפיצירן די פּעקל נאָמען אין `v:` `ol.yml` , ביטע **זיין זיכער צו ויסמעקן `.i18n/v/ol` ערשטער** און דאַן אַרויסגעבן עס.

##### פּראָקסי סערווער ארויס דורך npm

אויב יוזערז אין יאַבאָשע טשיינאַ טרעפן נעץ פּראָבלעמס און קענען `https_proxy` אַרויסגעבן די `npm`

אַסומינג אַז דיין פראקסי סערווער פּאָרט איז `7890` איר קענען שרייַבן:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### זיך-כאָוסטיד אינהאַלט

אויב איר ווילט זיך-באַהאַלטן דעם אינהאַלט, ערשטער, `md/.i18n/htm/ol.yml` און טוישן `v: //unpkg.com/i18n.site` צו דיין URL פּרעפיקס, אַזאַ ווי `v: //i18n-v.xxx.com` !

אַרייַן די `md` און לויפן

```
i18n.site --htm_conf ol --save
```

אָדער אַבריווייישאַן

```
i18n.site -c ol -s
```

דערנאָך, קאַנפיגיער די אינהאַלט אין `md/out/ol/v` וועגווייַזער צו די URL פּרעפיקס דרך שטעלן אין `v:` !

צום סוף, **קאַנפיגיער די קאַש צייט פון דעם דרך וואָס ענדיקט `/.v` מיט `1s`** , אַנדערש איר קענען נישט אַקסעס די ניי פריי אינהאַלט.

די קאַש צייט פֿאַר אנדערע פּאַטס קענען זיין באַשטימט צו איין יאָר אָדער מער צו רעדוצירן ומנייטיק ריקוועס.

##### האָסט אינהאַלט צו s3

צו זיך-האָסט אינהאַלט, אין אַדישאַן צו ניצן דיין אייגענע סערווער, + אנדער `CDN` אָפּציע איז צו נוצן `S3`

איר קענען נוצן [rclone](https://rclone.org) `S3` , דאַן אָפּשיקן צו און מאָדיפיצירן די שריפט אונטן, און נאָר נאָכמאַכן ינקראַמענטאַל ענדערונגען צו `S3` יעדער מאָל איר אַרויסגעבן.

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

געדענקט צו קאַנפיגיער `CDN` אַזוי אַז די קאַש צייט פון דעם דרך וואָס ענדיקט זיך אין `/.v` איז `1s` , אַנדערש איר וועט נישט קענען צו אַקסעס די ניי-פריי אינהאַלט גלייך.

#### אָפּטימיזינג וועבזייטל דיפּלוימאַנט אין יאַבאָשע טשיינאַ

#### ניצן nginx

ביטע לייג אַ קאַנפיגיעריישאַן ענלעך צו די פאלגענדע אין `server` nginx ווו `/root/i18n/md/out/ol/htm` ביטע טוישן עס צו די דרך פון דיין אייגן פּרויעקט `out/ol/htm` :

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```
