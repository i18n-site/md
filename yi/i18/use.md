דער `en` כּולל די איבערגעזעצט דעמאָ טעקעס, וואָס זענען בלויז אַ ביישפּיל און קענען ווערן אויסגעמעקט.

### לויפן איבערזעצונג

### לייג טעקעס צו די ריפּאַזאַטאָרי

אין אַדישאַן צו איבערזעצונג, די פּראָגראַם וועט אויך דזשענערייט די פאלגענדע טעקעס, ביטע לייגן זיי צו די ריפּאַזאַטאָרי.

```

```

צווישן זיי, דער אינהאַלט : `.i18n/cache/.gitignore`

```

```

דעם מיטל איגנאָרירן אַלע טעקעס אין די `.i18n/cache/` (חוץ `.i18n/cache/.gitignore` ).

אויב דיין ווערסיע קאָנטראָל ווייכווארג איז נישט `git` ביטע איגנאָרירן עס לויט דעם קאַנפיגיעריישאַן.

## קאָנפיגוראַטיאָן טעקע

`.i18n/conf.yml` עס איז די קאַנפיגיעריישאַן טעקע פון ​​די `i18` שורה איבערזעצונג געצייַג

דער אינהאַלט איז ווי גייט:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### איבערזעצונג &

אין די קאַנפיגיעריישאַן טעקע, די סאַבאָרדאַנייץ פון `fromTo`

`en` איז די מקור שפּראַך, `zh ja ko de fr` איז די ציל שפּראַך פון איבערזעצונג.

שפּראַך קאָד זען [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

פֿאַר בייַשפּיל, אויב איר ווילן צו איבערזעצן כינעזיש אין ענגליש, רירייט די שורה `zh: en`

אויב איר ווילן צו איבערזעצן צו אַלע געשטיצט שפּראַכן, ביטע לאָזן ליידיק נאָך `:` פֿאַר בייַשפּיל

```
i18n:
  fromTo:
    en:
```

### איגנאָרירן טעקע

דורך פעליקייַט, אַלע טעקעס סטאַרטינג מיט `.md` `.yml` אין די מקור שפּראַך וועגווייַזער וועט זיין איבערגעזעצט.

אויב איר ווילן צו איגנאָרירן זיכער טעקעס און נישט איבערזעצן זיי (אַזאַ ווי אַנפינישט דראַפץ), איר קענען נוצן די `ignore` קאַנפיגיעריישאַן.

`ignore` ניצט אַ סינטאַקס ענלעך צו `.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax)

פֿאַר בייַשפּיל, אין די אויבן קאַנפיגיעריישאַן `_* ` מיטל אַז טעקעס סטאַרטינג מיט `_` וועט נישט זיין איבערגעזעצט.

## איבערזעצונג כּללים

### איבערזעצונג רעדאקציע זאָל נישט לייגן אָדער ויסמעקן שורות

די איבערזעצונג איז עדיטאַבלע. מאָדיפיצירן דעם אָריגינעל טעקסט און מאַשין איבערזעצן עס ווידער, די מאַנואַל מאָדיפיקאַטיאָנס צו די איבערזעצונג וועט נישט זיין אָוווערריטאַן (אויב דער פּאַראַגראַף פון דער אָריגינעל טעקסט איז נישט מאַדאַפייד).

אָבער ביטע טאָן אַז די שורות פון דער איבערזעצונג און דער אָריגינעל טעקסט מוזן שטימען איינער צו איין. דאָס איז, טאָן ניט לייגן אָדער ויסמעקן שורות ווען קאַמפּיילינג די איבערזעצונג. אַנדערש, עס וועט פאַרשאַפן צעמישונג אין די איבערזעצונג עדיטינג קאַש.

אויב עפּעס גייט פאַלש, ביטע אָפּשיקן צו [די FAQ פֿאַר סאַלושאַנז.](/i18/qa#H1)

### `YAML`

די באַפֿעלן שורה געצייַג וועט געפֿינען אַלע טעקעס וואָס ענדיקן מיט `.yml`

* באַמערקונג אַז דער טעקע נאָמען סאַפיקס מוזן `.yml` (נישט `.yaml` ).

די געצייַג איבערזעצט בלויז די ווערטערבוך וואַלועס אין `.yml` נישט די ווערטערבוך שליסלען.

למשל `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

וועט איבערגעזעצט ווערן ווי `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

די איבערזעצונג פון `YAML` קענען אויך זיין מאַדאַפייד מאַניואַלי (אָבער טאָן ניט לייגן אָדער ויסמעקן שליסלען אָדער שורות אין דער איבערזעצונג).

באזירט אויף `YAML` , איר קענען לייכט בויען אינטערנאַציאָנאַלע סאַלושאַנז פֿאַר פאַרשידן פּראָגראַממינג שפּראַכן.

### מנהג ינסטאַלירונג וועגווייַזער

עס וועט זיין `/usr/local/bin` דורך פעליקייַט.

`/usr/local/bin` האט נישט שרייַבן דערלויבעניש עס וועט זיין אינסטאַלירן צו `~/.bin` .

באַשטעטיק סוויווע `TO` איר קענען דעפינירן די ינסטאַלירונג וועגווייַזער, למשל :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
