## เซิร์ฟเวอร์การซื้อ contabo

contabo เป็นผู้ให้บริการ VPS ราคาถูกในเยอรมนีที่มีประวัติยาวนาน (มากกว่า 20 ปี) และมีผู้ใช้งานจำนวนมาก (มากกว่า 200,000 ราย) ดังนั้นจึงไม่จำเป็นต้องกังวลเรื่องการหลบหนี

VPS ซื้อ [contabo.com](//contabo.com) อย่าลืมเลือกราคายูโรซึ่งถูกกว่า

![](https://i-01.eu.org/1718285782.avif)

มีค่าธรรมเนียมการติดตั้งเพียงครั้งเดียวสำหรับ contabo
อย่างไรก็ตาม มีสองสามวันทุกเดือนที่จะเสียค่าธรรมเนียมการติดตั้ง เพียงลงทะเบียนและให้ความสนใจกับอีเมลส่งเสริมการขาย

นอกจากค่าธรรมเนียมพื้นที่ฟรีในยุโรปแล้ว ภูมิภาคอื่นๆ ยังมีระดับการขึ้นราคาที่แตกต่างกันอีกด้วย

ฉันเลือกเซิร์ฟเวอร์ยุโรป อย่างไรก็ตาม หลังจากใช้ cloudflare แล้ว ก็ไม่มีความแตกต่างระหว่างเซิร์ฟเวอร์มากนัก

การกำหนดค่าสุดท้ายมีดังนี้ หน่วยความจำ 16GB, 6CPU, NVMe 200 GB, 9.50 ยูโรต่อเดือน หรือเท่ากับ 74.43 RMB

## กำหนดค่าการเข้าสู่ระบบโดยไม่ต้องใช้รหัสผ่าน

หลังจากนั้นไม่กี่นาที เซิร์ฟเวอร์ทั้งหมดก็เปิดใช้งาน

สิ่งแรก IP แก้ไข `~/.ssh/config` ของเครื่องท้องถิ่นเพื่อตั้งค่านามแฝง จากนั้นใช้ `ssh-copy-id root@c2` เพื่อกำหนดค่าการเข้าสู่ระบบโดยไม่ต้องใช้รหัสผ่าน

![](https://i-01.eu.org/1718287198.avif)

คุณสามารถแก้ไข [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) นี้เพื่อให้การกำหนดค่าเริ่มต้นเสร็จสมบูรณ์เร็วขึ้น

หลังจากเสร็จสิ้นการกำหนดค่าโดยไม่ต้องใช้รหัสผ่าน คุณสามารถใช้งานได้ [pdsh](https://github.com/chaos/pdsh) สามารถใช้งานได้เป็นชุดซึ่งสะดวก

สิ่งแรกคือการเปิดใช้งาน ipv6 และทดสอบด้วยตนเอง

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

![](https://i-01.eu.org/1718287969.avif)

จากนั้นอัพเกรดระบบเป็น Ubuntu 24 ได้ในคลิกเดียวและติดตั้งซอฟต์แวร์ที่ใช้กันทั่วไป

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 หมายถึงการเปลี่ยนภาษาเป็นภาษาจีนและเขตเวลาเป็นปักกิ่ง

ซอฟต์แวร์ที่ติดตั้งโดยค่าเริ่มต้นได้รับการกำหนดค่าตามพฤติกรรมการใช้งานส่วนตัวของฉัน เช่น [mise](https://github.com/jdx/mise) , [direnv](https://direnv.net/) ฯลฯ หากจำเป็น คุณสามารถคัดลอกคลัง [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) และแก้ไขได้ตามต้องการ

ขั้นตอนนี้ใช้เวลานาน หลังจากรอประมาณครึ่งชั่วโมง การอัพเกรดและติดตั้งซอฟต์แวร์ที่ใช้กันทั่วไปจะสำเร็จ คุณสามารถยืนยันสิ่งนี้ได้ด้วยคำสั่งต่อไปนี้:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## mosh เพิ่มความเร็ว ssh

เครือข่ายจากยุโรปไปยังจีนไม่เสถียร ขอ `ssh` ให้ใช้ [mosh](//mosh.org)

สร้าง `c0` สคริปต์ในโลคัลไดเร็กทอรี `~/.bin` (เพิ่มให้กับตัวแปรสภาพแวดล้อมเป็นหลัก `PATH` ) ดังนี้:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

หลังจากสร้าง ให้อนุญาตสิทธิ์ปฏิบัติการและซอฟต์ลิงก์ `c1` และ `c2`

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

ในอนาคตคุณสามารถป้อน `c0` และ `c1` ได้โดยตรงเพื่อใช้การเชื่อมต่อโดยตรงความเร็วสูง `mosh` แล้วป้อน [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

## แปลงรูปแบบพาร์ติชันจาก ext4 เป็น btrfs

[btrfs](https://wikipedia.org/wiki/Btrfs) สามารถบันทึกสแน็ปช็อตของดิสก์เพื่อการสำรองข้อมูลที่ง่ายดาย

ขั้นแรกให้เข้าสู่ระบบช่วยเหลือ

![](https://i-01.eu.org/1718300448.avif)

เลือก Debian Rescue

![](https://i-01.eu.org/1718300753.avif)

รอให้เซิร์ฟเวอร์รีสตาร์ท

ssh จะมีการเปลี่ยนแปลงลายนิ้วมือใช้ `ssh-keygen -R ip` เพื่อทำความสะอาด

จากนั้นรันสคริปต์ต่อไปนี้เพื่อแปลงพาร์ติชัน

```bash
nohup bash -c "bash <(curl -s https://raw.githubusercontent.com/i18n-ops/ops/main/contabo/ext4_btrfs.sh) >/tmp/nohup.out 2>&1 && reboot" &
sleep 1 && tail -f /tmp/nohup.out
```

สคริปต์จะรีสตาร์ทระบบโดยอัตโนมัติหลังจากการแปลงสำเร็จ

### ดูอัตราส่วนการบีบอัด btrfs

มันจะบีบอัดโดยอัตโนมัติตามค่าเริ่มต้นคำสั่งในการตรวจสอบอัตราส่วนการบีบอัดมีดังนี้:

```bash
if ! command -v compsize &>/dev/null; then
apt install -y  btrfs-compsize
fi
compsize -x /
```

### การบีบอัดแบบแมนนวล

หากคุณเป็นพวกชอบความสมบูรณ์แบบ คุณสามารถทำการบีบอัดดิสก์แบบเต็มด้วยตนเองได้ คำสั่งจะเป็นดังนี้:

```bash
btrfs filesystem defragment -r -czstd /
```

ดังแสดงในรูปด้านล่าง คุณจะเห็นว่าหลังจากการบีบอัดด้วยตนเองในระบบใหม่ จะมีการบีบอัดเพิ่มขึ้นเกือบ 100MB

![](https://i-01.eu.org/2023/12/8NWlr4c.webp)

### สร้างการสำรองข้อมูลสแน็ปช็อต Btrfs ด้วย snapper

[snapper](https://manual.siduction.org/sys-admin-btrfs-snapper_en.html) เป็นพันธมิตรที่ดีที่สุดสำหรับ #การสำรองข้อมูล Btrfs

สคริปต์การเริ่มต้นจะสร้างไดเร็กทอรี `~/ubuntu` เพียงรันคำสั่งต่อไปนี้:

```
~/ubuntu/snapper/init.sh
```

เจอกันในเนื้อหาสคริปต์ [github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh](https://github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh)

ใช้ `snapper list-configs` เพื่อดูไดรฟ์ข้อมูลย่อยที่มีการตั้งค่าสแนปช็อต

![](https://i-01.eu.org/1718302689.avif)

ใช้ `snapper -c mnt list` เพื่อดูภาพรวมของไดรฟ์ข้อมูลย่อย

`/etc/snapper/configs/mnt` สามารถดูการตั้งค่าการสำรองข้อมูลเฉพาะของวอลุ่มย่อยนี้ได้

ข้อมูลอ้างอิงเพิ่มเติม:

[เล่นได้อย่างง่ายดายด้วยฟังก์ชั่นสแน็ปช็อตของ Btrfs snapper](https://zhuanlan.zhihu.com/p/31082518)

## รายละเอียดระบบกู้ภัย

เข้าสู่ระบบ 22 หลังจากที่พอร์ต [tcping](https://github.com/paradiseduo/tcping) เข้าถึงได้ คุณจะไม่สามารถเข้าถึงได้ด้วยรหัสผ่านที่ตั้งไว้เป็นเวลา 2-3 นาที เพียงรอสักครู่ก็จะไม่เป็นไร

```
tcping ip 22 -c 9999
```

สามารถตรวจสอบการเข้าถึงพอร์ตได้

### วิธีเข้าถึงฮาร์ดไดรฟ์หลังจากเข้าสู่การช่วยเหลือ &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

คุณสามารถดูภาพรวมได้ เป็นต้น

```
snapper -c etc list
```

ดูการเปลี่ยนแปลงไฟล์สแน็ปช็อต

```
snapper -c etc status 5..6
```

ดูข้อความส่วนต่างเฉพาะ

```
snapper -c etc diff 3..5
```

คืนค่าสแนปช็อต: ตัวอย่างเช่น เลิกทำการเปลี่ยนแปลงทั้งหมดหลังจากสแนปชอต 5

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### เปิดใช้งานการคงอยู่ของบันทึก

ในการเปิดใช้งานการคงอยู่ของบันทึกภายใต้ระบบช่วยเหลือ คุณจะต้องสร้างไดเร็กทอรีที่เกี่ยวข้องด้วยตนเอง และดำเนินการขั้นตอนเพิ่มเติมบางส่วน เนื่องจากคุณอาจไม่มีเซอร์วิส systemd ที่กำลังรันอยู่ ต่อไปนี้เป็นขั้นตอนที่คุณสามารถลองได้:

อันดับแรกเลย

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

รีบูตเข้าสู่ระบบ Ubuntu ของคุณ: หลังจากทำการเปลี่ยนแปลงข้างต้นแล้ว คุณจะต้องรีบูตเครื่องตามปกติและเข้าสู่ระบบ Ubuntu แทนระบบช่วยเหลือ

ด้วยวิธีนี้ systemd-journald จะเริ่มต้นด้วยการกำหนดค่าใหม่ และเริ่มจัดเก็บบันทึกอย่างถาวรบนฮาร์ดดิสก์

โปรดทราบว่าการดำเนินการข้างต้นจะดำเนินการบนระบบช่วยเหลือ ดังนั้นการเปลี่ยนแปลงเหล่านั้นจะไม่มีผลจนกว่าระบบจะรีบูตและไม่ได้อยู่ในโหมดช่วยเหลืออีกต่อไป

ซึ่งหมายความว่าคุณจะไม่เห็นการเปลี่ยนแปลงใดๆ ในบันทึกที่มีอยู่ซึ่งคงอยู่ถาวรในทันที แต่บันทึกทั้งหมดที่สร้างขึ้นหลังจากนั้นจะยังคงอยู่

วิธีนี้ทำให้คุณสามารถดูบันทึกของบริการ ssh ได้

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## หมายเหตุโรคที่ยากและซับซ้อน

### แก้ไขปัญหาแปลก ๆ : เซิร์ฟเวอร์ไม่สามารถ ssh ได้

ฉันเคยใช้การกระทำของ GitHub เพื่อสร้างสคริปต์การคอมไพล์และการปรับใช้อัตโนมัติมาก่อน แต่พบว่าบริการการปรับใช้นั้นไม่สามารถเป็น ssh ได้

การเข้าสู่ระบบช่วยเหลือและดูบันทึกเป็นปัญหาเรื่องการอนุญาต:

`Missing privilege separation directory: /run/sshd`

คุณสามารถสร้างไดเร็กทอรีนี้ได้ด้วยคำสั่งต่อไปนี้

```
chown root:root /
systemd-tmpfiles --create
```

หลังจากตรวจสอบซ้ำแล้วซ้ำอีก เป็นเพราะ github action User `uid` ของ tar คือ `1001`

tar การแกะจะต้องเป็นหากไม่มีพารามิเตอร์ `--no-same-owner` มันจะถูกแตกออกเป็น `uid` สำหรับผู้ใช้ `1001` คน

ด้วยเหตุนี้ rsync จะเปลี่ยนกลุ่มผู้ใช้ของไดเรกทอรีราก จากนั้นคุณจะไม่สามารถเข้าสู่ระบบ ssh ได้หลังจากรีสตาร์ท

แก้ไขสคริปต์การคลายการบีบอัดดังต่อไปนี้:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
