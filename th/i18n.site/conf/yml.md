# .i18n/conf.yml

`i18n.site` ไฟล์คอนฟิกูเรชันคือ `.i18n/conf.yml` .

ยกเว้นการตั้งค่า `ignore:` และ `i18n:` ของ [`i18`](/i18)

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

ในบรรดารายการการกำหนดค่า `ext:` ของ `upload` หมายความว่าจะมีการอัปโหลดเพียง `.md` เท่านั้นเมื่อเผยแพร่

## ระบบนำทางด้านบน

ตัวเลือกการกำหนด `nav:` ที่สอดคล้องกับเมนูการนำทางที่ด้านบนของหน้าแรก

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

ในหมู่พวกเขา `i18n: home` สอดคล้องกับ `en/i18n.yml`中`home: Home`

`en/i18n.yml` จะถูกแปลเป็นหลายภาษา เช่น `zh/i18n.yml`

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

หลังจากการแปลเสร็จสิ้น คุณสามารถแก้ไขค่าของ `yml` ในการแปลได้ แต่อย่าเพิ่มหรือลบคีย์ของการแปล `yml`

### `use: Toc` เทมเพลตไฟล์เดียว (พร้อมโครงร่าง)

`nav` การกำหนดค่า :

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` หมายถึงการใช้ `Toc` rendering ซึ่งกำลังเรนเดอร์เทมเพลต `Markdown` เดียว

`TOC` เป็นตัวย่อของ `Table of Contents` เมื่อแสดงผลเทมเพลตนี้ โครงร่างของไฟล์ `Markdown` นี้จะแสดงในแถบด้านข้าง

`url:` แสดงถึงเส้นทางไฟล์ของ `Markdown` ( `/` สอดคล้องกับไดเร็กทอรีราก `/README.md` ชื่อไฟล์นี้ต้องมีคำนำหน้าตัวพิมพ์ใหญ่และส่วนต่อท้ายตัวพิมพ์เล็ก)

### `use: Md` , เทมเพลตไฟล์เดียว (ไม่มีโครงร่าง)

`Md` เทมเพลตเหมือนกับ `Toc` ทั้งสองใช้เพื่อเรนเดอร์ไฟล์ `Markdown` ไฟล์เดียว อย่างไรก็ตาม `Md` เทมเพลตไม่แสดงโครงร่างในแถบด้านข้าง

คุณสามารถแก้ไข `use: Toc` ในการกำหนดค่าด้านบนเป็น `use: Md` เรียกใช้ `i18n.site` ในไดเรกทอรี `md` อีกครั้ง จากนั้นไปที่ URL ตัวอย่างการพัฒนาเพื่อสังเกตการเปลี่ยนแปลงในหน้าแรก

### การโหลดเริ่มต้นโดยไม่มีเส้นทางการกำหนดค่า

หากคำนำหน้าเส้นทางของเส้นทางบางเส้นทางที่กำลังเข้าถึงไม่ได้รับการกำหนดค่าใน `nav:` ไฟล์ `MarkDown` ที่สอดคล้องกับเส้นทางจะถูกโหลดตามค่าเริ่มต้นและแสดงผลโดยใช้เทมเพลต `Md`

ตัวอย่างเช่น หากคุณไปที่ `/test` และ `nav:` ได้รับการกำหนดค่าโดยไม่มีเส้นทางนี้ และภาษาของหน้าเป็นภาษาอังกฤษ (รหัส `en` ) เทมเพลตจะถูกโหลด `/en/test.md` และแสดงผลโดยใช้ `Md` ตามค่าเริ่มต้น

`/en/test.md` ไม่มีไฟล์นี้ ระบบจะแสดง `404` เพจเริ่มต้น

### `use: Doc` , เทมเพลตหลายไฟล์

ในไฟล์กำหนดค่า:

```
  - i18n: blog
    use: Doc
```

บ่งชี้ถึงการใช้ `Doc` สำหรับการแสดงผลเทมเพลต

`Doc` รองรับการรวม `MarkDown` หลายรายการเพื่อสร้างโครงร่างเอกสารสำหรับโครงการเดียวหรือหลายโครงการ

#### โครงการเดียว (หลายไฟล์)

`blog` ด้านบนเป็นโหมดโปรเจ็กต์เดียวของ `Doc`

```
  - i18n: blog
    url: blog
    use: Doc
```

##### เมื่อ url ว่างเปล่า จะมีค่าเริ่มต้นเป็นค่า i18n

หากคุณไม่ได้เขียน `url` , `url` เป็นค่าของ `i18n` กฎนี้ยังมีผลกับเทมเพลตอื่นด้วย

วิธีการเขียนข้างต้นเทียบเท่ากับการมี `url: blog` และไฟล์ที่เกี่ยวข้องคือ `en/blog/TOC`

#### หลายโครงการ

`.i18n/conf.yml` การกำหนดค่าใน `i18n:doc` เป็นโหมดหลายโครงการ

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

ในที่นี้ `menu: NB demo1,demo2` หมายถึงการใช้ `NB` เพื่อแสดงเมนูแบบเลื่อนลง

`NB` เป็นตัวย่อของ `Name Breif` ระบุว่าเมนูแบบเลื่อนลงสามารถแสดงชื่อและสโลแกนของโครงการได้

`NB` ตามด้วยพารามิเตอร์ `demo1,demo2` ที่ส่งไป
`demo1,demo2` ทราบว่าไม่ควรมีช่องว่าง ** ก่อนและหลังเครื่องหมายจุลภาค `,` ใน : **

สำหรับพารามิเตอร์ข้างต้น ไฟล์ดัชนีไดเร็กทอรีที่เกี่ยวข้องคือ:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### ดัชนีสารบัญ TOC

`i18n.site` จะดำเนินการ `js` ปลั๊กอิน `.i18n/hook/after.tran/TOC.js` ในคลังสาธิตเพื่ออ่านไฟล์ดัชนีไดเร็กทอรี `doc` สอดคล้องกับการกำหนดค่าเทมเพลต `TOC` เพื่อสร้าง `json` ของโครงร่างไดเร็กทอรี

หากคุณใช้เทมเพลต `doc` คุณต้องมีปลั๊กอินนี้

หากคุณเริ่มต้นโปรเจ็กต์ `i18n.site` จากโฟลเดอร์ว่าง อย่าลืมคัดลอก `.i18n` ในโปรเจ็กต์สาธิตไปยังไดเร็กทอรีของคุณ

`Doc` จะแสดงโครงร่างสารบัญตาม `json` ที่สร้างขึ้น

##### คำอธิบายเนื้อหาโดยละเอียด

เนื้อหา : ดังนี้ `en/blog/TOC`

```
README.md

news/README.md
  news/begin.md
```

##### ใช้การเยื้องเพื่อระบุระดับ

ด้านบน `en/blog/TOC` `README.md` ในบรรทัดแรกตรงกับ `i18n.site` ในภาพด้านล่างซึ่งเป็นชื่อโครงการ

สองบรรทัดถัดไปดังที่แสดงในภาพหน้าจอด้านล่าง

<img src="https://p.3ti.site/1721097381.avif" style="width:320px">

ตรงกับ `News` `news/README.md`
`news/begin.md` สอดคล้องกับ `Our Product is Online !`

`TOC` ไฟล์ถูกเยื้องเพื่อระบุความสัมพันธ์แบบลำดับชั้นของโครงร่างและรองรับการเยื้องหลายระดับ

##### ระดับผู้ปกครองเขียนเฉพาะชื่อเรื่องเท่านั้น ไม่ใช่เนื้อหา

เมื่อมีการเยื้องหลายระดับ ระดับพาเรนต์จะเขียนเฉพาะชื่อเรื่องเท่านั้น ไม่ใช่เนื้อหา ไม่อย่างนั้นตัวอักษรจะเลอะเทอะ

##### โครงการ README.md 

`README.md` โครงการ เช่น คุณสามารถเขียนเนื้อหาใน `en/demo2/README.md`

โปรดทราบว่าเนื้อหาของไฟล์นี้ไม่แสดงโครงร่างสารบัญ ดังนั้นจึงแนะนำให้จำกัดความยาวและเขียนคำแนะนำสั้นๆ

<img alt="" src="https://p.3ti.site/1721276842.avif">

: กับบรรทัดแรก `en/demo2/README.md`


โครงการ `README.md` เนื้อหาที่อยู่หลังเครื่องหมายทวิภาคแรก `:` ในชื่อระดับแรกจะถือเป็นสโลแกนของโครงการ

ผู้ใช้จากประเทศจีน ญี่ปุ่น และเกาหลี โปรดทราบว่าคุณต้องใช้โคลอนแบบครึ่งความกว้าง `:`

##### จะย้าย TOC จำนวนมากได้อย่างไร

`TOC` ไฟล์จะต้องอยู่ในไดเร็กทอรีของภาษาต้นฉบับ

ตัวอย่างเช่น หากภาษาต้นฉบับเป็นภาษาจีน ดังนั้น `TOC` ด้านบนจะเป็น `zh/blog/TOC`

หากมีการแก้ไขภาษาต้นฉบับ คุณจะต้องย้าย `TOC` ในภาษาใดภาษาหนึ่งในโปรเจ็กต์เป็นชุดไปยังภาษาอื่น

คุณสามารถอ้างถึงคำสั่งต่อไปนี้:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

โปรดแก้ไข `en/` และ `zh/` ในคำสั่งข้างต้นเป็นรหัสภาษาของคุณ
