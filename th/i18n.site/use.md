# ติดตั้ง &

## โทเค็นการกำหนดค่า

`i18n.site` เครื่องมือแปล `i18` ถูกฝังไว้ โปรด [คลิกที่นี่เพื่อดูเอกสาร `i18` เพื่อกำหนดค่าโทเค็นการเข้าถึง](/i18/use)

## ติดตั้ง

```sh
bash <(curl -sS https://i.i18n.site) i18n.site
```

## โครงการสาธิต

เริ่มต้นด้วยโครงการสาธิตและเรียนรู้วิธีใช้ `i18n.site`

ก่อนอื่นเราจะโคลนพื้นที่เก็บข้อมูลสาธิตและรันคำสั่งดังต่อไปนี้:

```
git clone https://github.com/i18n-site/demo.i18n.site.git md
git clone https://github.com/i18n-site/demo.i18n.site.docker.git docker
```

ผู้ใช้ในจีนแผ่นดินใหญ่สามารถ:

```
git clone https://atomgit.com/i18n/demo.i18n.site.git md
git clone https://atomgit.com/i18n/demo.i18n.site.docker.git docker
```

`demo.i18n.site` ชื่อไดเร็กทอรีของโค้ดฐานโคลนต้องเป็น `md` เพื่ออำนวยความสะดวกในการดูตัวอย่างในเครื่องด้วย `docker`

### โพสต์เนื้อหา

`i18n.site` สถาปัตยกรรม [แอปพลิเคชันหน้าเดียว](https://developer.mozilla.org/docs/Glossary/SPA) และหน้าทางเข้าเว็บไซต์และเนื้อหาเว็บไซต์ถูกปรับใช้อย่างอิสระ

หลังจากรันการแปลข้างต้น ไดเร็กทอรี `htm` และ `v` จะถูกสร้างขึ้นในไดเร็กทอรี `md/out/dev`

ในที่นี้ `dev` บ่งบอกว่ามันถูกสร้างขึ้นตามไฟล์ `.i18n/htm/dev.yml`

`dev` ตามเนื้อหา :

`htm` ใต้ไดเร็กทอรีคือหน้าทางเข้าเว็บไซต์

`v` ไดเร็กทอรีประกอบด้วยเนื้อหาเว็บไซต์พร้อมหมายเลขเวอร์ชัน

การแสดงตัวอย่างในเครื่องจะคัดลอกไฟล์ทั้งหมดไปยังไดเร็กทอรี `out/dev/v/0.1.0` โดยไม่คำนึงถึงหมายเลขเวอร์ชัน

สำหรับการเปิดตัวอย่างเป็นทางการ ไฟล์ที่เปลี่ยนแปลงจะถูกคัดลอกไปยังไดเร็กทอรีหมายเลขเวอร์ชันใหม่

#### ใช้ -c เพื่อระบุไฟล์การกำหนดค่า

ไฟล์คอนฟิกูเรชันที่แตกต่างกันจะสร้างไดเร็กทอรีที่เกี่ยวข้องภายใต้ไดเร็กทอรี `out`

ตัวอย่างเช่น `.i18n/htm/ol.yml` จะสร้าง `out/ol` ไดเร็กทอรี

`dev.yml` และ `ol.yml` เป็นการกำหนดค่าเริ่มต้น

`dev` เป็นตัวย่อของ `development` ซึ่งแสดงถึงสภาพแวดล้อมการพัฒนา ใช้สำหรับการดูตัวอย่างในเครื่อง และยังเป็นไฟล์การกำหนดค่าเริ่มต้นอีกด้วย
`ol` เป็นตัวย่อของ `online` ซึ่งแสดงถึงสภาพแวดล้อมออนไลน์ ใช้สำหรับการเปิดตัวอย่างเป็นทางการ และยังเป็นไฟล์การกำหนดค่าเริ่มต้นเมื่อเผยแพร่ไปยัง `npm` โดยใช้พารามิเตอร์บรรทัดคำสั่ง `-n` .

คุณยังสามารถสร้างไฟล์คอนฟิกูเรชันอื่นๆ ได้ ใช้ `--htm_conf` บนบรรทัดคำสั่งเพื่อระบุชื่อไฟล์คอนฟิกูเรชัน:

ตัวอย่างเช่น:
```
i18n.site --htm_conf yourConfig --save
```

ที่นี่ `--save` ระบุหมายเลขเวอร์ชันการอัพเดต

#### <a rel=id href="#npm" id="npm"></a> เผยแพร่เนื้อหาไปยัง npmjs.com

การเผยแพร่เนื้อหาไปที่ [npmjs.com](//npmjs.com) เป็นวิธีแก้ปัญหาเริ่มต้นที่แนะนำ (ดู [Front-end High Availability](/i18n.site/feature#ha) )

##### npm login & ปล่อย

ติดตั้ง `nodejs` ใช้ `npm login` เพื่อเข้าสู่ระบบ

แก้ไข `md/.i18n/htm/ol.yml` เปลี่ยน `i18n.site` ใน `v: //unpkg.com/i18n.site` เป็นของคุณเอง `npm` ชื่อแพ็คเกจ

เพียงใช้ชื่อแพ็คเกจว่างใน [npmjs.com](//npmjs.com) ​​การใช้ชื่อโดเมนเว็บไซต์เป็นชื่อแพ็คเกจก็เป็นทางเลือกที่ดี

เมื่อเผยแพร่ตามแพ็คเกจ `npm` **ต้องแน่ใจว่าใช้ `//unpkg.com/`** เป็นคำนำหน้าของค่า `v:` เวลาแคช `i18n.site` ภายใต้พาธคำนำหน้านี้ `/.v` ได้รับการปรับให้เหมาะสมเป็นพิเศษเพื่อให้สามารถดูรีลีสใหม่ได้ทันเวลา

รัน `i18n.site --npm` หรือ `i18n.site -n` ในไดเร็กทอรี `md`

หากคุณใช้สภาพแวดล้อมการรวมอย่างต่อเนื่องเพื่อเผยแพร่ ไม่จำเป็นต้องติดตั้ง `nodejs` คัดลอก `~/.npmrc` สิทธิ์ในการเข้าสู่ระบบและเผยแพร่ไปยังสภาพแวดล้อม

หากคุณแก้ไขชื่อแพ็คเกจใน `v:` in `ol.yml` โปรด **อย่าลืมลบ `.i18n/v/ol` ก่อน** แล้วจึงเผยแพร่

##### พร็อกซีเซิร์ฟเวอร์เผยแพร่โดย npm

หากผู้ใช้ในจีนแผ่นดินใหญ่ประสบปัญหาเครือข่ายและไม่สามารถเผยแพร่แพ็คเกจ `npm` ได้ พวกเขาสามารถตั้งค่าตัวแปรสภาพแวดล้อม `https_proxy` เพื่อกำหนดค่าพร็อกซีเซิร์ฟเวอร์ได้

สมมติว่าพอร์ตพร็อกซีเซิร์ฟเวอร์ของคุณคือ `7890` คุณสามารถเขียนได้:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### เนื้อหาที่โฮสต์ด้วยตนเอง

หากคุณต้องการโฮสต์เนื้อหาด้วยตนเอง ขั้นแรกให้แก้ไข `md/.i18n/htm/ol.yml` และเปลี่ยน `v: //unpkg.com/i18n.site` เป็นคำนำหน้า URL ของคุณ เช่น `v: //i18n-v.xxx.com`

ป้อนไดเร็กทอรี `md` และเรียกใช้

```
i18n.site --htm_conf ol --save
```

หรือตัวย่อ

```
i18n.site -c ol -s
```

จากนั้น กำหนดค่าเนื้อหาในไดเร็กทอรี `md/out/ol/v` เป็นพาธคำนำหน้า URL ที่ตั้งค่าไว้ใน `v:`

สุดท้าย **กำหนดค่าเวลาแคชของเส้นทางที่ลงท้ายด้วย `/.v` ถึง `1s`** มิฉะนั้นจะไม่สามารถเข้าถึงเนื้อหาที่ออกใหม่ได้ทันที

เวลาแคชสำหรับเส้นทางอื่นสามารถตั้งค่าเป็นหนึ่งปีหรือมากกว่านั้นเพื่อลดคำขอที่ไม่จำเป็น

##### โฮสต์เนื้อหาไปที่ s3

หากต้องการโฮสต์เนื้อหาด้วยตนเอง นอกเหนือจากการใช้เซิร์ฟเวอร์ของคุณเอง `CDN` อีกทางเลือกหนึ่งคือการใช้ `S3` + .

คุณสามารถใช้ [rclone](https://rclone.org) Login `S3` จากนั้นอ้างอิงและแก้ไขสคริปต์ด้านล่าง และคัดลอกเฉพาะการเปลี่ยนแปลงที่เพิ่มขึ้นเป็น `S3` ทุกครั้งที่คุณเผยแพร่

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

อย่าลืมกำหนดค่า `CDN` เพื่อให้เวลาแคชของเส้นทางที่ลงท้ายด้วย `/.v` คือ `1s` มิฉะนั้นคุณจะไม่สามารถเข้าถึงเนื้อหาที่เพิ่งเปิดตัวได้ทันที

### เผยแพร่เว็บไซต์

เว็บไซต์สามารถติดตั้งได้ทุกที่ [github page](https://pages.github.com) และ [cloudflare page](https://pages.cloudflare.com) เป็นตัวเลือกที่ดี

เนื่องจากเว็บไซต์ใช้สถาปัตยกรรมของ [แอปพลิเคชันหน้าเดียว](https://developer.mozilla.org/docs/Glossary/SPA) อย่าลืมเขียนเส้นทาง URL ที่ไม่มี `. ` ถึง `index.html`

#### การเพิ่มประสิทธิภาพการใช้งานเว็บไซต์ในจีนแผ่นดินใหญ่

หากคุณต้องการได้รับประสิทธิภาพการเข้าถึงที่ดีขึ้นในสภาพแวดล้อมเครือข่ายของจีนแผ่นดินใหญ่ โปรด [จดทะเบียนชื่อโดเมน](//beian.aliyun.com) ก่อน

`out/ol/htm` นั้นใช้ที่เก็บข้อมูลออบเจ็กต์ของผู้จำหน่ายระบบคลาวด์ใน `CDN` แผ่นดินใหญ่ +

คุณสามารถใช้การประมวลผลแบบ Edge เพื่อเขียนเส้นทางใหม่เพื่อปรับให้เข้ากับแอปพลิเคชันหน้าเดียว เช่น [Baidu Smart Cloud `CDN`](//cloud.baidu.com/product/cdn.html)

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```

#### การเปลี่ยนเส้นทางชื่อโดเมนทั่วไป

หากคุณใช้ `i18n.site` เพื่อสร้างเว็บไซต์เป็นเว็บไซต์หลักของคุณ โดยปกติคุณจะต้องกำหนดค่าการเปลี่ยนเส้นทางแพนโดเมน นั่นคือ เปลี่ยนเส้นทาง `*.xxx.com` (รวมถึง `www.xxx.com` ) เข้าถึง `xxx.com` .

ข้อกำหนดนี้สามารถบรรลุได้ด้วยความช่วยเหลือ `CDN` `EdgeScript` ( [เอกสารภาษาอังกฤษ](https://www.alibabacloud.com/help/en/cdn/developer-reference/how-edgescript-works) / [เอกสารภาษาจีน](https://help.aliyun.com/zh/cdn/developer-reference/edgescript) )

เพิ่มชื่อโดเมนใน [Alibaba CDN](https://cdn.console.aliyun.com/domain/list) และชี้ `*.xxx.com` โดเมนไปที่ `CNAME` ของ Alibaba Cloud `CDN`

#### ปรับใช้กับ nginx

โปรดเพิ่มการกำหนดค่าที่คล้ายกับข้อความต่อไปนี้ใน `server` nginx ที่ `/root/i18n/md/out/ol/htm` โปรดเปลี่ยนเป็นเส้นทางของโครงการของคุณเอง `out/ol/htm` :

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```

### โครงสร้างไดเร็กทอรี

#### สาธารณะ

ไฟล์คงที่ของเว็บไซต์ เช่น `favicon.ico` `robots.txt` เป็นต้น

ไฟล์ไอคอนที่นี่สามารถสร้างได้ด้วย [realfavicongenerator.net](https://realfavicongenerator.net)

#### .i18n

`.i18n` ภายใต้ไดเร็กทอรีคือไฟล์การกำหนดค่าของ `i18n.site` แคชการแปล ฯลฯ โปรดดูรายละเอียดในบทถัดไป ["การกำหนดค่า"](/i18n.site/conf)

#### ห้องน้ำในตัว

ไดเร็กทอรีภาษาต้นทางซึ่งสอดคล้อง `.i18n/conf.yml` `fromTo` `en` ในไฟล์กำหนดค่า

```yaml
i18n:
  fromTo:
    en: zh
```

โปรดดูการกำหนดค่าการแปล [i18](/i18/use)
