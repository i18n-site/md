# ติดตั้งและใช้งาน

## Windows ติดตั้ง git bash ก่อน

windows ระบบ กรุณา [คลิกที่นี่เพื่อดาวน์โหลดและติดตั้งก่อน `git bash`](https://git-scm.com/download/win)

ดำเนินการต่อไปใน `git bash`

## ติดตั้ง

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### กำหนดค่าโทเค็นการแปล

เยี่ยมชม [i18n.site/token](//i18n.site/token) คลิกเพื่อคัดลอกโทเค็น

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

สร้าง `~/.config/i18n.site.yml` วางเนื้อหาที่คัดลอกไว้ลงไป โดยมีเนื้อหาดังนี้:

```
token: YOUR_API_TOKEN
```

นอกจากนี้คุณต้อง [i18n.site/payBill](//i18n.site/payBill) บัตรเครดิตในการชำระเงิน (ไม่ต้องเติมเงิน เว็บไซต์จะหักค่าธรรมเนียมตามการใช้งานโดยอัตโนมัติ [ดูราคาที่หน้าแรก](/#price) )

## ใช้

### โครงการสาธิต

โปรดดูโครงการสาธิต [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18) เพื่อเรียนรู้การกำหนดค่าการ `i18`

ผู้ใช้ในประเทศจีนสามารถโคลนได้ [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

หลังจากการโคลน ให้เข้าสู่ไดเร็กทอรีและรัน `i18` เพื่อทำการแปลให้เสร็จสิ้น

### โครงสร้างไดเร็กทอรี

โครงสร้างไดเร็กทอรีคลังสินค้าเทมเพลตมีดังนี้

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

ไดเร็กทอรี `en` มีไฟล์สาธิตที่แปลแล้ว ซึ่งเป็นเพียงตัวอย่างและสามารถลบได้

### เรียกใช้การแปล

เข้าสู่ไดเร็กทอรีและรัน `i18`

### เพิ่มไฟล์ไปยังที่เก็บ

นอกจากการแปลแล้ว โปรแกรมจะสร้างไฟล์ต่อไปนี้ด้วย โปรดเพิ่มไฟล์เหล่านี้ลงในพื้นที่เก็บข้อมูล

```
.i18n/hash
.i18n/cache/.gitignore
```

โดยมีเนื้อหา : `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

ซึ่งหมายความว่าละเว้นไฟล์ทั้งหมดในไดเร็กทอรี `.i18n/cache/` (ยกเว้น `.i18n/cache/.gitignore` )

หากซอฟต์แวร์ควบคุมเวอร์ชันของคุณไม่ใช่ `git` โปรดละเว้นตามการกำหนดค่านี้

## ไฟล์การกำหนดค่า

`.i18n/conf.yml` เป็นไฟล์กำหนดค่าของเครื่องมือแปลบรรทัด `i18`

เนื้อหามีดังนี้:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ภาษาต้นฉบับ &

ในไฟล์คอนฟิกูเรชัน ผู้ใต้บังคับบัญชาของ `fromTo`

`en` เป็นภาษาต้นทาง `zh ja ko de fr` คือภาษาเป้าหมายของการแปล

รหัสภาษาดู [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

เช่น หากคุณต้องการแปลภาษาจีนเป็นภาษาอังกฤษ ให้เขียนบรรทัดนี้ใหม่ `zh: en`

หากคุณต้องการแปลเป็นภาษาที่รองรับทั้งหมด โปรดเว้นว่างไว้หลังจาก `:` ตัวอย่างเช่น

```
i18n:
  fromTo:
    en:
```

### ละเว้นไฟล์

ตามค่าเริ่มต้น ไฟล์ทั้งหมดที่ขึ้นต้นด้วย `.md` และ `.yml` ในไดเร็กทอรีภาษาต้นฉบับจะถูกแปล

หากคุณต้องการละเว้นไฟล์บางไฟล์และไม่แปลไฟล์เหล่านั้น (เช่น ร่างที่ยังเขียนไม่เสร็จ) คุณสามารถใช้การกำหนดค่าฟิลด์ `ignore`

`ignore` ใช้ไวยากรณ์คล้าย [globset](https://docs.rs/globset/latest/globset/#syntax) `.gitignore`

ตัวอย่างเช่น ในไฟล์การกำหนดค่าด้านบน `_* ` หมายความว่าไฟล์ที่ขึ้นต้นด้วย `_` จะไม่ถูกแปล

## กฎการแปล

### ผู้แก้ไขคำแปลไม่ควรเพิ่มหรือลบบรรทัด

การแปลสามารถแก้ไขได้ แก้ไขข้อความต้นฉบับและแปลด้วยเครื่องอีกครั้ง การแก้ไขการแปลด้วยตนเองจะไม่ถูกเขียนทับ (หากย่อหน้านี้ของข้อความต้นฉบับไม่ได้รับการแก้ไข)

แต่โปรดทราบว่าบรรทัดของการแปลและข้อความต้นฉบับจะต้องสอดคล้องกัน กล่าวคือ ห้ามเพิ่มหรือลบบรรทัดเมื่อรวบรวมการแปล มิฉะนั้นจะทำให้เกิดความสับสนในแคชการแก้ไขการแปล

หากมีสิ่งผิดปกติเกิดขึ้น โปรดดู [คำถามที่พบบ่อยเพื่อดูแนวทางแก้ไข](/i18/qa#H1)

### `YAML` แปลภาษา

เครื่องมือบรรทัดคำสั่งจะค้นหาไฟล์ทั้งหมดที่ลงท้ายด้วย `.yml` ในไดเร็กทอรีไฟล์ภาษาต้นฉบับและแปลไฟล์เหล่านั้น

* โปรดทราบว่าส่วนต่อท้ายชื่อไฟล์ต้องเป็น `.yml` (ไม่ใช่ `.yaml` )

เครื่องมือจะแปลเฉพาะค่าพจนานุกรมเป็น `.yml` ไม่ใช่คีย์พจนานุกรม

ตัวอย่างเช่น `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

จะแปลว่า `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

การแปล `YAML` สามารถแก้ไขได้ด้วยตนเอง (แต่อย่าเพิ่มหรือลบคีย์หรือบรรทัดในการแปล)

จากการแปล `YAML` คุณสามารถสร้างโซลูชันระดับสากลสำหรับภาษาการเขียนโปรแกรมต่างๆ ได้อย่างง่ายดาย

## การใช้งานขั้นสูง

### ไดเรกทอรีย่อยการแปล

ตราบใดที่คุณสร้าง `.i18n/conf.yml` (ไม่จำเป็นต้องเริ่มจากเทมเพลตโครงการสาธิตทุกครั้ง) `i18` จะทำงานได้ดี

เครื่องมือบรรทัดคำสั่งจะค้นหาการกำหนดค่า `.i18n/conf.yml` ในไดเรกทอรีย่อยทั้งหมดแล้วแปล

โปรเจ็กต์ที่ใช้สถาปัตยกรรม [monorepo](//monorepo.tools) สามารถแบ่งไฟล์ภาษาออกเป็นไดเร็กทอรีย่อยได้

<img alt="" src="https://p.3ti.site/1719910016.avif">

### ไดเร็กทอรีการติดตั้งแบบกำหนดเอง

มันจะถูกติดตั้งเป็น `/usr/local/bin`

ถ้า `/usr/local/bin` ไม่มีสิทธิ์ในการเขียน ก็จะถูกติดตั้งไปที่ `~/.bin`

ตั้งค่าตัวแปรสภาพแวดล้อม `TO` คุณสามารถกำหนดไดเร็กทอรีการติดตั้งได้ เช่น :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
