# ఇన్‌స్టాల్ చేయండి &

## కాన్ఫిగరేషన్ టోకెన్

`i18n.site` `i18` అనువాద సాధనం పొందుపరచబడింది, [యాక్సెస్ టోకెన్‌ను కాన్ఫిగర్ చేయడానికి `i18` పత్రాన్ని సూచించడానికి దయచేసి ఇక్కడ క్లిక్ చేయండి](/i18/use) .

## ఇన్‌స్టాల్ చేయండి

```sh
bash <(curl -sS https://i.i18n.site) i18n.site
```

## డెమో ప్రాజెక్ట్

డెమో ప్రాజెక్ట్‌తో ప్రారంభించి, ఎలా ఉపయోగించాలో నేర్చుకుందాం `i18n.site`

మేము మొదట డెమో రిపోజిటరీని క్లోన్ చేస్తాము మరియు కింది విధంగా ఆదేశాన్ని అమలు చేస్తాము:

```
git clone https://github.com/i18n-site/demo.i18n.site.git md
git clone https://github.com/i18n-site/demo.i18n.site.docker.git docker
```

చైనా ప్రధాన భూభాగంలోని వినియోగదారులు వీటిని చేయగలరు:

```
git clone https://atomgit.com/i18n/demo.i18n.site.git md
git clone https://atomgit.com/i18n/demo.i18n.site.docker.git docker
```

`docker` తో స్థానిక ప్రివ్యూను సులభతరం చేయడానికి కోడ్ బేస్ క్లోన్ యొక్క డైరెక్టరీ పేరు తప్పనిసరిగా `md` ఉండాలి `demo.i18n.site`

### కంటెంట్‌ను పోస్ట్ చేయండి

`i18n.site` [ఒకే పేజీ అప్లికేషన్](https://developer.mozilla.org/docs/Glossary/SPA) ఆర్కిటెక్చర్‌ని స్వీకరిస్తుంది మరియు వెబ్‌సైట్ ప్రవేశ పేజీ మరియు వెబ్‌సైట్ కంటెంట్ స్వతంత్రంగా అమలు చేయబడతాయి.

పై అనువాదాన్ని అమలు చేసిన తర్వాత, `htm` మరియు `v` డైరెక్టరీలు `md/out/dev` ఉత్పత్తి చేయబడతాయి.

ఇక్కడ, `dev` ఇది `.i18n/htm/dev.yml` ఫైల్ ఆధారంగా నిర్మించబడిందని సూచిస్తుంది.

: కింద `dev`

`htm` కింద వెబ్‌సైట్ ప్రవేశ పేజీ ఉంది.

`v` డైరెక్టరీ వెబ్‌సైట్ కంటెంట్‌ను వెర్షన్ నంబర్‌తో కలిగి ఉంది.

స్థానిక ప్రివ్యూ సంస్కరణ సంఖ్యతో సంబంధం లేకుండా అన్ని ఫైల్‌లను `out/dev/v/0.1.0` కాపీ చేస్తుంది.

అధికారిక విడుదల కోసం, మార్చబడిన ఫైల్‌లు కొత్త వెర్షన్ నంబర్ డైరెక్టరీకి కాపీ చేయబడతాయి.

#### కాన్ఫిగరేషన్ ఫైల్‌ను పేర్కొనడానికి -c ఉపయోగించండి

వివిధ కాన్ఫిగరేషన్ ఫైల్‌లు `out` క్రింద సంబంధిత డైరెక్టరీలను సృష్టిస్తాయి.

ఉదాహరణకు, `.i18n/htm/ol.yml` `out/ol` డైరెక్టరీని సృష్టిస్తుంది.

`dev.yml` మరియు `ol.yml` డిఫాల్ట్ కాన్ఫిగరేషన్‌లు.

`dev` అనేది `development` యొక్క సంక్షిప్తీకరణ, ఇది అభివృద్ధి వాతావరణాన్ని సూచిస్తుంది, ఇది స్థానిక ప్రివ్యూ కోసం ఉపయోగించబడుతుంది మరియు ఇది డిఫాల్ట్ కాన్ఫిగరేషన్ ఫైల్ కూడా.
`ol` అనేది `online` యొక్క సంక్షిప్తీకరణ, ఇది ఆన్‌లైన్ వాతావరణాన్ని సూచిస్తుంది, ఇది అధికారిక విడుదల కోసం ఉపయోగించబడుతుంది మరియు `npm` కమాండ్ లైన్ పరామితిని ఉపయోగించి `-n` కు ప్రచురించేటప్పుడు ఇది డిఫాల్ట్ కాన్ఫిగరేషన్ ఫైల్.

కాన్ఫిగరేషన్ ఫైల్ పేరును పేర్కొనడానికి `--htm_conf` కమాండ్ లైన్‌లో ఇతర కాన్ఫిగరేషన్ ఫైల్‌లను కూడా సృష్టించవచ్చు.

ఉదాహరణకి:
```
i18n.site --htm_conf yourConfig --save
```

`--save` నవీకరణ విడుదల సంస్కరణ సంఖ్యను సూచిస్తుంది.

#### <a rel=id href="#npm" id="npm"></a> కంటెంట్‌ను npmjs.comకు ప్రచురించండి

[npmjs.com](//npmjs.com) కంటెంట్‌ని ప్రచురించడం అనేది సిఫార్సు చేయబడిన డిఫాల్ట్ పరిష్కారం ( [ముందు-ముగింపు అధిక లభ్యత](/i18n.site/feature#ha) చూడండి).

##### npm login & విడుదల

ఇన్‌స్టాల్ చేయండి `nodejs` లాగిన్ చేయడానికి `npm login` ఉపయోగించండి.

సవరించు `md/.i18n/htm/ol.yml` `i18n.site` లో `v: //unpkg.com/i18n.site` ని మీ స్వంతంగా మార్చుకోండి `npm` ప్యాకేజీ పేరు.

కేవలం ఆక్రమించని ప్యాకేజీ పేరుని ఉపయోగించండి [npmjs.com](//npmjs.com)

`npm` ప్యాకేజీ ఆధారంగా ప్రచురించేటప్పుడు, `v:` విలువ యొక్క ఉపసర్గగా **`//unpkg.com/` ఉపయోగించాలని నిర్ధారించుకోండి** `i18n.site` ఈ ఉపసర్గ మార్గం క్రింద `/.v` కొత్త విడుదలలను సకాలంలో చూడడానికి ప్రత్యేకంగా ఆప్టిమైజ్ చేయబడింది.

అనువదించడానికి మరియు ప్రచురించడానికి `md` `i18n.site --npm` లేదా `i18n.site -n` అమలు చేయండి.

మీరు ప్రచురించడానికి నిరంతర ఏకీకరణ వాతావరణాన్ని ఉపయోగిస్తే, దాన్ని ఇన్‌స్టాల్ చేయాల్సిన అవసరం `nodejs` లాగ్ ఇన్ చేసి ప్రచురించిన అనుమతి `~/.npmrc` ని పర్యావరణానికి కాపీ చేయండి.

మీరు `v:` `ol.yml` లో ప్యాకేజీ పేరును సవరించినట్లయితే, **దయచేసి ముందుగా `.i18n/v/ol` తొలగించి** , ఆపై దానిని ప్రచురించండి.

##### npm ద్వారా ప్రాక్సీ సర్వర్ ప్రచురించబడింది

చైనా ప్రధాన భూభాగంలోని వినియోగదారులు నెట్‌వర్క్ సమస్యలను ఎదుర్కొంటే మరియు `npm` ప్రచురించలేకపోతే, వారు ప్రాక్సీ సర్వర్‌ను కాన్ఫిగర్ చేయడానికి ఎన్విరాన్‌మెంట్ వేరియబుల్ `https_proxy` సెట్ చేయవచ్చు.

మీ ప్రాక్సీ సర్వర్ పోర్ట్ `7890` మీరు వ్రాయవచ్చు:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### స్వీయ-హోస్ట్ కంటెంట్

మీరు కంటెంట్‌ను స్వీయ-హోస్ట్ చేయాలనుకుంటే, ముందుగా `v: //i18n-v.xxx.com` వంటి మీ URL ఉపసర్గకు `v: //unpkg.com/i18n.site` మరియు మార్చండి `md/.i18n/htm/ol.yml`

`md` నమోదు చేసి, అమలు చేయండి

```
i18n.site --htm_conf ol --save
```

లేదా సంక్షిప్తీకరణ

```
i18n.site -c ol -s
```

ఆపై, `v:` లో సెట్ చేసిన URL ఉపసర్గ పాత్‌కు `md/out/ol/v` డైరెక్టరీలోని కంటెంట్‌ను కాన్ఫిగర్ చేయండి.

చివరగా, **పాత్ యొక్క కాష్ సమయాన్ని కాన్ఫిగర్ `1s` `/.v`**

అనవసరమైన అభ్యర్థనలను తగ్గించడానికి ఇతర మార్గాల కోసం కాష్ సమయాన్ని ఒక సంవత్సరం లేదా అంతకంటే ఎక్కువ కాలానికి సెట్ చేయవచ్చు.

##### కంటెంట్‌ని s3కి హోస్ట్ చేయండి

`CDN` -హోస్ట్ కంటెంట్‌కి, మీ స్వంత సర్వర్‌ని ఉపయోగించడంతో పాటు, + సాధారణ ఎంపిక `S3`

`S3` `S3` [rclone](https://rclone.org)

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

కాన్ఫిగర్ చేయాలని గుర్తుంచుకోండి `CDN` తద్వారా `/.v` తో ముగిసే మార్గం యొక్క కాష్ సమయం `1s` , లేకుంటే మీరు వెంటనే కొత్తగా విడుదల చేసిన కంటెంట్‌ను యాక్సెస్ చేయలేరు.

#### చైనా ప్రధాన భూభాగంలో వెబ్‌సైట్ విస్తరణను ఆప్టిమైజ్ చేయడం

మీరు మెయిన్‌ల్యాండ్ చైనా యొక్క నెట్‌వర్క్ వాతావరణంలో మెరుగైన ప్రాప్యత పనితీరును పొందాలనుకుంటే, దయచేసి ముందుగా [డొమైన్ పేరును నమోదు చేయండి](//beian.aliyun.com) .

`out/ol/htm` `CDN` ప్రధాన భూభాగంలో క్లౌడ్ విక్రేతల ఆబ్జెక్ట్ స్టోరేజ్‌ని ఉపయోగించండి +

[బైడు స్మార్ట్ `CDN`](//cloud.baidu.com/product/cdn.html) వంటి సింగిల్-పేజీ అప్లికేషన్‌లకు అనుగుణంగా మార్గాన్ని తిరిగి వ్రాయడానికి మీరు ఎడ్జ్ కంప్యూటింగ్‌ని ఉపయోగించవచ్చు:

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```

ఎందుకంటే `MX` రికార్డ్‌లు మరియు `CNAME` రికార్డ్‌లు కలిసి ఉండవు, మీరు ఒకే సమయంలో డొమైన్ పేరు ఇమెయిల్‌లను స్వీకరించాలనుకుంటే, మీరు [cname_flatten](https://github.com/i18n-site/lib/tree/main/cname_flatten) `CNAME` చదును `A` స్క్రిప్ట్‌తో సహకరించాలి.

అదనంగా, చైనా ప్రధాన భూభాగంలో క్లౌడ్ విక్రేతల విదేశీ ట్రాఫిక్ ఛార్జీలు సాపేక్షంగా ఖరీదైనవి, మీరు ఖర్చులను ఆప్టిమైజ్ చేయాలనుకుంటే, మీరు సాధించడానికి [Huawei DNS యొక్క ఉచిత భౌగోళిక రిజల్యూషన్](https://support.huaweicloud.com/usermanual-dns/dns_usermanual_0041.html) మరియు [Cloudflare for SaaS](https://developers.cloudflare.com/cloudflare-for-platforms/cloudflare-for-saas) యొక్క అనుకూల డొమైన్ పేరు (క్రింద చూపిన విధంగా) ఉపయోగించవచ్చు ట్రాఫిక్ మళ్లింపు──ప్రధాన భూభాగంలో ట్రాఫిక్ చైనా బైడు క్లౌడ్ `CDN` అంతర్జాతీయ ట్రాఫిక్ cloudflare .

<img alt="" src="https://p.3ti.site/1721119788.avif">

ఈ విస్తరణ ఆప్టిమైజేషన్ సొల్యూషన్‌లు మరింత క్లిష్టంగా ఉంటాయి మరియు భవిష్యత్తులో ప్రత్యేక అధ్యాయాలలో పరిచయం చేయబడతాయి.

#### సాధారణ డొమైన్ పేరు దారి మళ్లింపు

మీరు మీ ప్రధాన వెబ్‌సైట్‌గా వెబ్‌సైట్‌ను రూపొందించడానికి `i18n.site` , మీరు సాధారణంగా పాన్-డొమైన్ దారి మళ్లింపును కాన్ఫిగర్ చేయాలి, అంటే, ( `www.xxx.com` తో సహా) `xxx.com` `*.xxx.com` .
