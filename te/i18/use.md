# ఇన్‌స్టాల్ చేసి ఉపయోగించుకోండి

## విండోస్ మొదట గిట్ బాష్‌ను ఇన్‌స్టాల్ చేయండి

windows సిస్టమ్, దయచేసి [ముందుగా డౌన్‌లోడ్ చేసి ఇన్‌స్టాల్ చేయడానికి ఇక్కడ క్లిక్ చేయండి `git bash`](https://git-scm.com/download/win)

లో తదుపరి కార్యకలాపాలను అమలు చేయండి `git bash`

## ఇన్‌స్టాల్ చేయండి

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### అనువాద టోకెన్‌ని కాన్ఫిగర్ చేయండి

టోకెన్‌ని కాపీ చేయడానికి క్లిక్ [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

సృష్టించు `~/.config/i18n.site.yml` కాపీ చేసిన కంటెంట్‌ను అందులో అతికించండి, కంటెంట్ క్రింది విధంగా ఉంటుంది:

```
token: YOUR_API_TOKEN
```

అదనంగా, మీరు చెల్లింపు కోసం క్రెడిట్ కార్డ్‌ని బైండ్ [i18n.site/payBill](//i18n.site/payBill) (రీఛార్జ్ అవసరం లేదు, వినియోగానికి అనుగుణంగా వెబ్‌సైట్ ఆటోమేటిక్‌గా ఫీజులను తీసివేస్తుంది, [ధర కోసం హోమ్‌పేజీని చూడండి](/#price) ).

## వా డు

### డెమో ప్రాజెక్ట్

అనువాదం కాన్ఫిగరేషన్ తెలుసుకోవడానికి దయచేసి డెమో ప్రాజెక్ట్‌ని `i18` [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

చైనాలోని వినియోగదారులు క్లోన్ చేయవచ్చు [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

క్లోనింగ్ తర్వాత, అనువాదాన్ని పూర్తి చేయడానికి డైరెక్టరీని నమోదు చేయండి `i18`

### డైరెక్టరీ నిర్మాణం

టెంప్లేట్ గిడ్డంగి డైరెక్టరీ నిర్మాణం క్రింది విధంగా ఉంది

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` డైరెక్టరీ అనువదించబడిన డెమో ఫైల్‌లను కలిగి ఉంది, అవి కేవలం ఒక ఉదాహరణ మాత్రమే మరియు తొలగించబడతాయి.

### అనువాదాన్ని అమలు చేయండి

అనువదించడానికి డైరెక్టరీని నమోదు చేయండి మరియు అమలు చేయండి `i18`

### రిపోజిటరీకి ఫైల్‌లను జోడించండి

అనువాదంతో పాటు, ప్రోగ్రామ్ కింది ఫైల్‌లను కూడా రూపొందిస్తుంది, దయచేసి వాటిని రిపోజిటరీకి జోడించండి.

```
.i18n/hash
.i18n/cache/.gitignore
```

వాటిలో, `.i18n/cache/.gitignore` కంటెంట్ క్రింది విధంగా ఉంది :

```
**/*
!**/.gitignore
```

`.i18n/cache/` అన్ని ఫైల్‌లను విస్మరించండి ( `.i18n/cache/.gitignore` తప్ప).

మీ సంస్కరణ నియంత్రణ సాఫ్ట్‌వేర్ కాకపోతే `git` దయచేసి ఈ కాన్ఫిగరేషన్ ప్రకారం దాన్ని విస్మరించండి.

## కాన్ఫిగరేషన్ ఫైల్

`.i18n/conf.yml` ఇది `i18` లైన్ అనువాద సాధనం యొక్క కాన్ఫిగరేషన్ ఫైల్

కంటెంట్ క్రింది విధంగా ఉంది:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### మూల భాష &

కాన్ఫిగరేషన్ ఫైల్‌లో, `fromTo`

`en` అనేది మూల భాష, `zh ja ko de fr` అనువాదానికి లక్ష్య భాష.

భాషా కోడ్ చూడండి [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

ఉదాహరణకు, మీరు చైనీస్‌ని ఆంగ్లంలోకి అనువదించాలనుకుంటే, ఈ పంక్తిని తిరిగి వ్రాయండి `zh: en`

మీరు మద్దతు ఉన్న అన్ని భాషలకు అనువదించాలనుకుంటే, దయచేసి తర్వాత ఖాళీగా ఉంచండి `:` ఉదాహరణకి

```
i18n:
  fromTo:
    en:
```

### ఫైల్‌ను విస్మరించండి

డిఫాల్ట్‌గా, సోర్స్ లాంగ్వేజ్ డైరెక్టరీలో `.md` మరియు `.yml` తో మొదలయ్యే అన్ని ఫైల్‌లు అనువదించబడతాయి.

మీరు నిర్దిష్ట ఫైల్‌లను విస్మరించి, వాటిని అనువదించకూడదనుకుంటే (అసంపూర్తిగా ఉన్న చిత్తుప్రతులు వంటివి), మీరు `ignore` కాన్ఫిగరేషన్‌ని ఉపయోగించవచ్చు.

`ignore` `.gitignore` వంటి వాక్యనిర్మాణాన్ని ఉపయోగిస్తుంది [globset](https://docs.rs/globset/latest/globset/#syntax)

ఉదాహరణకు, పై కాన్ఫిగరేషన్ ఫైల్‌లో `_* ` `_` ప్రారంభమయ్యే ఫైల్‌లు అనువదించబడవు.

## అనువాద నియమాలు

### అనువాద సంపాదకులు పంక్తులను జోడించకూడదు లేదా తొలగించకూడదు

అనువాదం సవరించదగినది. అసలు వచనాన్ని సవరించండి మరియు దాన్ని మళ్లీ మెషీన్-అనువదించండి, అనువాదానికి మాన్యువల్ సవరణలు భర్తీ చేయబడవు (అసలు టెక్స్ట్ యొక్క ఈ పేరా సవరించబడకపోతే).

కానీ దయచేసి అనువాదం యొక్క పంక్తులు మరియు అసలు వచనం తప్పనిసరిగా ఒకదానికి అనుగుణంగా ఉండాలి. అంటే, అనువాదాన్ని కంపైల్ చేసేటప్పుడు పంక్తులను జోడించవద్దు లేదా తొలగించవద్దు. లేకపోతే, ఇది అనువాద సవరణ కాష్‌లో గందరగోళాన్ని కలిగిస్తుంది.

ఏదైనా తప్పు జరిగితే, దయచేసి [పరిష్కారాల కోసం తరచుగా అడిగే ప్రశ్నలను చూడండి.](/i18/qa#H1)

### `YAML`

కమాండ్ లైన్ సాధనం `.yml` తో ముగిసే అన్ని ఫైళ్ళను సోర్స్ లాంగ్వేజ్ ఫైల్ డైరెక్టరీలో కనుగొని వాటిని అనువదిస్తుంది.

* ఫైల్ పేరు ప్రత్యయం తప్పనిసరిగా `.yml` ( `.yaml` కాదు).

సాధనం నిఘంటువు విలువలను మాత్రమే అనువదిస్తుంది `.yml` నిఘంటువు కీలు కాదు.

ఉదాహరణకి `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

గా అనువదించబడుతుంది `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` అనువాదం మాన్యువల్‌గా కూడా సవరించబడుతుంది (కానీ అనువాదంలో కీలు లేదా పంక్తులను జోడించవద్దు లేదా తొలగించవద్దు).

`YAML` ఆధారంగా, మీరు వివిధ ప్రోగ్రామింగ్ భాషల కోసం సులభంగా అంతర్జాతీయ పరిష్కారాలను రూపొందించవచ్చు.

## అధునాతన వినియోగం

### అనువాద ఉప డైరెక్టరీ

మీరు సృష్టించినంత `.i18n/conf.yml` (ప్రతిసారీ డెమో ప్రాజెక్ట్ టెంప్లేట్ నుండి ప్రారంభించాల్సిన అవసరం లేదు), `i18` బాగా పని చేస్తుంది.

కమాండ్ లైన్ సాధనం అన్ని ఉప డైరెక్టరీలలో `.i18n/conf.yml` కనుగొని దానిని అనువదిస్తుంది.

[monorepo](//monorepo.tools) ఉపయోగించే ప్రాజెక్ట్‌లు భాషా ఫైల్‌లను ఉప డైరెక్టరీలుగా విభజించగలవు

<img alt="" src="https://p.3ti.site/1719910016.avif">

### కస్టమ్ ఇన్‌స్టాలేషన్ డైరెక్టరీ

ఇది డిఫాల్ట్‌గా ఇన్‌స్టాల్ చేయబడుతుంది `/usr/local/bin`

వ్రాయడానికి అనుమతి `/usr/local/bin` అది `~/.bin` కి ఇన్‌స్టాల్ చేయబడుతుంది.

ఎన్విరాన్మెంట్ వేరియబుల్స్ సెట్ `TO` మీరు ఇన్‌స్టాలేషన్ డైరెక్టరీని నిర్వచించవచ్చు, ఉదాహరణకు :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
