## Օգտագործել

### Դեմո Նախագիծ

Խնդրում ենք դիմել ցուցադրական նախագծին [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18) որպեսզի սովորեք թարգմանության կոնֆիգուրացիան `i18`

Չինաստանում օգտագործողները կարող են կլոնավորել [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

Կլոնավորումից հետո մուտքագրեք գրացուցակը և գործարկեք թարգմանությունը ավարտելու համար `i18`

### Տեղեկատուի Կառուցվածքը

Կաղապարի պահեստի գրացուցակի կառուցվածքը հետևյալն է

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` գրացուցակը պարունակում է թարգմանված ցուցադրական ֆայլեր, որոնք ընդամենը օրինակ են և կարող են ջնջվել:

### Գործարկել Թարգմանությունը

Մուտքագրեք գրացուցակը և գործարկեք թարգմանելու համար `i18`

### Ֆայլեր Ավելացրեք Պահեստում

Բացի թարգմանությունից, ծրագիրը կստեղծի նաև հետևյալ ֆայլերը, խնդրում ենք դրանք ավելացնել պահեստում:

```
.i18n/hash
.i18n/cache/.gitignore
```

Դրանց թվում `.i18n/cache/.gitignore` -ի բովանդակությունը հետևյալն է :

```
**/*
!**/.gitignore
```

Սա նշանակում է, որ անտեսեք `.i18n/cache/` բոլոր ֆայլերը (բացի `.i18n/cache/.gitignore` ից):

Եթե ​​ձեր տարբերակի վերահսկման ծրագրակազմն այն չէ `git` խնդրում ենք անտեսել այն ըստ այս կազմաձևման:

## Կազմաձևման Ֆայլ

Դա `i18` line թարգմանության գործիքի կազմաձևման ֆայլն `.i18n/conf.yml`

Բովանդակությունը հետևյալն է.

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Թարգմանության &

Կազմաձևման ֆայլում `fromTo` -ի ենթակաները.

`en` սկզբնաղբյուր լեզուն է, `zh ja ko de fr` թարգմանության նպատակային լեզուն է:

Լեզվի կոդը տես [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

Օրինակ, եթե ցանկանում եք չինարեն թարգմանել անգլերեն, վերագրեք այս տողը `zh: en`

Եթե ​​ցանկանում եք թարգմանել բոլոր աջակցվող լեզուներով, ապա թողեք դատարկ `:` օրինակ

```
i18n:
  fromTo:
    en:
```

### Անտեսել Ֆայլը

Լռելյայնորեն, բոլոր ֆայլերը, որոնք սկսվում են `.md` -ով և `.yml` , սկզբնաղբյուր լեզվի գրացուցակում կթարգմանվեն:

Եթե ​​ցանկանում եք անտեսել որոշ ֆայլեր և չթարգմանել դրանք (օրինակ՝ անավարտ սևագրեր), կարող եք օգտագործել `ignore` կազմաձևումը:

Օգտագործում `ignore` շարահյուսություն, որը նման է `.gitignore` ին [globset](https://docs.rs/globset/latest/globset/#syntax)

Օրինակ, վերը նշված կոնֆիգուրացիայի ֆայլում նշանակում է `_* ` որ `_` -ով սկսվող ֆայլերը չեն թարգմանվի:

## Թարգմանության Կանոններ

### Թարգմանության Խմբագիրները Չպետք Է Տողեր Ավելացնեն Կամ Ջնջեն

Թարգմանությունը խմբագրելի է։ Փոփոխեք բնօրինակ տեքստը և այն նորից մեքենայական թարգմանեք, թարգմանության ձեռքով կատարված փոփոխությունները չեն վերագրվի (եթե բնօրինակ տեքստի այս պարբերությունը փոփոխված չէ):

Բայց նկատի ունեցեք, որ թարգմանության և բնագրի տողերը պետք է համապատասխանեն մեկին: Այսինքն՝ թարգմանությունը կազմելիս տողեր մի ավելացրեք կամ ջնջեք։ Հակառակ դեպքում դա շփոթություն կառաջացնի թարգմանության խմբագրման քեշում:

Եթե ​​ինչ-որ բան սխալ է, [լուծումների համար դիմեք ՀՏՀ-ին:](/i18/qa#H1)

### `Yaml`

Հրամանի տող գործիքը կգտնի `.yml`

* Նկատի `.yaml` , որ ֆայլի անվան վերջածանցը պետք է լինի `.yml`

Գործիքը թարգմանում է միայն բառարանի արժեքները `.yml` այլ ոչ թե բառարանի ստեղները:

օրինակ `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

կթարգմանվի որպես `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` կարող է նաև փոփոխվել ձեռքով (բայց մի ավելացրեք կամ ջնջեք բանալիներ կամ տողեր թարգմանության մեջ):

`YAML` հիման վրա դուք կարող եք հեշտությամբ ստեղծել միջազգային լուծումներ տարբեր ծրագրավորման լեզուների համար:

## Ընդլայնված Օգտագործումը

### Թարգմանությունների Ենթացուցակ

Քանի դեռ դուք ստեղծում `.i18n/conf.yml` (կարիք չկա ամեն անգամ սկսել ցուցադրական նախագծի ձևանմուշից), `i18` լավ կաշխատի:

Հրամանի տող գործիքը կգտնի `.i18n/conf.yml` -ի կոնֆիգուրացիան բոլոր ենթագրքերում և կթարգմանի այն:

Նախագծերը, որոնք օգտագործում են [monorepo](//monorepo.tools) -ի ճարտարապետությունը, կարող են բաժանել լեզվական ֆայլերը ենթատեղեկատուների:

### Պատվերով Տեղադրման Գրացուցակ

Այն կտեղադրվի լռելյայն `/usr/local/bin`

Եթե ​​գրելու թույլտվություն չունի `/usr/local/bin` այն կտեղադրվի `~/.bin` -ում:

Սահմանել շրջակա միջավայրի փոփոխականներ `TO` կարող եք սահմանել տեղադրման գրացուցակը, օրինակ :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
