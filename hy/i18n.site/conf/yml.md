# .i18n/conf.yml

Կազմաձևման ֆայլը `.i18n/conf.yml` է `i18n.site`

Բացառությամբ `ignore:` և `i18n:` [`i18`](/i18) կարգավորումների, կազմաձևման ֆայլը հետևյալն է.

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

Դրանցից `ext:` կազմաձևման տարրը նշանակում է `upload` որ հրապարակման ժամանակ կվերբեռնվի միայն `.md` :

կթարգմանվի մի քանի լեզուներով, օրինակ՝ `zh/i18n.yml` `en/i18n.yml`

Թարգմանության ավարտից հետո `yml` մեջ կարող եք փոփոխել `yml`

### `use: Toc` Մեկ Ֆայլի Ձևանմուշ (Ուրվագծով)

: `nav`

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` նշանակում է օգտագործել `Toc` rendering, որը ներկայացնում է մեկ `Markdown` կաղապար:

Երբ այս `Table of Contents` ներկայացվի `TOC` այս `Markdown` ֆայլի ուրվագիծը կցուցադրվի կողային տողում:

`url:` ներկայացնում է ֆայլի ուղին `Markdown` ( `/` համապատասխանում է արմատային գրացուցակին `/README.md` , այս ֆայլի անունը պահանջում է մեծատառ նախածանց և փոքրատառ ածանց):

### `use: Md` Մեկ Ֆայլի Ձևանմուշ (Առանց Ուրվագծի)

Կաղապարը նույնն է `Md` ինչ `Toc` , երկուսն էլ օգտագործվում են մեկ `Markdown` ֆայլի մատուցման համար: Այնուամենայնիվ `Md` ձևանմուշը չի ցուցադրում ուրվագիծը կողագոտում:

Դուք կարող եք վերը նշված կազմաձևում `use: Toc` `use: Md` , գործարկել `i18n.site` կրկին `md` գրացուցակում և այնուհետև այցելեք զարգացման նախադիտման URL ՝ հիմնական էջի փոփոխությունները դիտելու համար:

### Կանխադրված Բեռնում Առանց Կազմաձևման Ուղու

Եթե ​​մուտք գործած որոշակի ուղու ուղու նախածանցը կազմաձևված չէ `nav:` ճանապարհին համապատասխանող `MarkDown` ֆայլը կբեռնվի լռելյայն և կարտացոլվի `Md` կաղապարի միջոցով:

Օրինակ, եթե այցելեք `/test` և `nav:` կազմաձևված է առանց այս ուղու, իսկ էջի լեզուն անգլերենն է ( `en` կոդ), ձևանմուշը կբեռնվի `/en/test.md` և կարտացոլվի օգտագործելով `Md` :

`/en/test.md` ​​այս ֆայլը գոյություն չունի, կցուցադրվի լռելյայն `404` էջը:

### `use: Doc` Բազմաֆայլ Կաղապար

Կազմաձևման ֆայլում.

```
  - i18n: blog
    use: Doc
```

Ցույց է տալիս `Doc` որ օգտագործում է կաղապարը:

`Doc` աջակցում է մի քանի `MarkDown` ինտեգրմանը մեկ կամ մի քանի նախագծերի համար փաստաթղթերի ուրվագծեր ստեղծելու համար:

#### Մեկ Նախագիծ (Բազմաթիվ Ֆայլեր)

`blog` `Doc` -ի մեկ նախագծի ռեժիմն է:

```
  - i18n: blog
    url: blog
    use: Doc
```

##### Երբ Url-Ը Դատարկ Է, Այն Լռելյայն Սահմանում Է i18n Արժեքը

Եթե ​​դուք չեք գրում `url` `url` `i18n` արժեքն է:

Գրելու վերը նշված մեթոդը համարժեք է `url: blog` իսկ դրա համապատասխան ֆայլը `en/blog/TOC` է:

#### Բազմաթիվ Նախագծեր

`.i18n/conf.yml` `i18n:doc` կոնֆիգուրացիան բազմանախագծային ռեժիմ է:

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

Այստեղ, `menu: NB demo1,demo2` նշանակում է օգտագործել `NB` ՝ բացվող ընտրացանկը ցուցադրելու համար:

`NB` -ը `Name Breif` -ի հապավումն է, որը ցույց է տալիս, որ բացվող ընտրացանկը կարող է ցուցադրել նախագծի անվանումը և կարգախոսը:

`NB` -ին հաջորդում է դրան փոխանցված `demo1,demo2` պարամետրը:
Նկատի ունեցեք `demo1,demo2` որ `,` ստորակետից առաջ և հետո բացատներ ** է լինեն : **

Վերոնշյալ պարամետրերի համար համապատասխան գրացուցակի ինդեքսային ֆայլը հետևյալն է.

* `en/demo1/TOC`
* `en/demo2/TOC`

#### TOC Բովանդակության Ինդեքս

`i18n.site` `js` plugin `.i18n/hook/after.tran/TOC.js` ցուցադրական պահեստում՝ կարդալու գրացուցակի ինդեքսի ֆայլը `doc` համապատասխան ձևանմուշի կազմաձևին `TOC` ՝ գրացուցակի `json` ստեղծելու համար:

Եթե ​​դուք օգտագործում `doc` կաղապարը, դուք պետք է ունենաք այս plug-in-ը:

Եթե `i18n.site` նախաստորագրեք դատարկ թղթապանակից, հիշեք, որ պատճենեք `.i18n` ցուցադրական նախագծում ձեր գրացուցակում:

`Doc` կներկայացնի բովանդակության ուրվագիծը՝ հիմնվելով ստեղծված `json` վրա:

Հաջորդ երկու տողերը, ինչպես ցույց է տրված ստորև ներկայացված սքրինշոթում:

##### Ծնողական Մակարդակը Գրում Է Միայն Վերնագիրը, Ոչ Թե Բովանդակությունը:

##### Ինչպե՞Ս Մեծաքանակ Տեղափոխել TOC-Ը:

Ֆայլը պետք `TOC` տեղադրվի սկզբնաղբյուր լեզվի գրացուցակում:

Օրինակ, եթե սկզբնաղբյուրը չինարենն է, `TOC` վերը նշված է `zh/blog/TOC` :

Եթե ​​սկզբնաղբյուր լեզուն փոփոխված է, դուք պետք է նախագծում որոշակի լեզվով `TOC` խմբաքանակ տեղափոխեք այլ լեզու:

Կարող եք դիմել հետևյալ հրամաններին.

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

Խնդրում `zh/` վերը նշված հրամանում փոփոխել `en/`
