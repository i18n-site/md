# MarkDown Format Conventionem

## Ancora Punctum

Traditional `MarkDown` Ancora generantur in textu contento.

`i18n.site` Solutio pacta anchora est inserere textum similem `<a rel=id href="#xxx" id="xxx"></a>` in `MarkDown` ad ancoras positionis manually creare.

`<a rel=id href="#xxx" id="xxx"></a>` hic `rel=id` modum paginae ancorae punctum definit, pro ancora tua Anglica repone `xxx` .

Ancorae titulo adiectae sunt, ut:

```md
### <a rel=id href="#i18" id="i18"></a>i18 : MarkDown Command Line Translation Tool
```

## Scribe `HTML` in `MarkDown`

`pug` `HTML` inseri potest in codice.

Contentus intra `<pre>` elementum non transferetur.

Coniungens haec duo puncta, facile `HTML` scribere potes `MarkDown`

Exsecutionem [github.com/i18n-site/md/en/README.md](https://raw.githubusercontent.com/i18n-site/md/main/en/README.md) codice referre potes ut sequitur :

```html
<h1 style="display:flex;justify-content:space-between">i18n.site ⋅ International Solutions<img src="//p.3ti.site/logo.svg" style="user-select:none;margin-top:-1px;width:42px"></h1>
<pre class="langli" style="display:flex;flex-wrap:wrap;background:transparent;border:1px solid #eee;font-size:12px;box-shadow:0 0 3px inset #eee;padding:12px 5px 4px 12px;justify-content:space-between;"><style>pre.langli i{font-weight:300;font-family:s;margin-right:2px;margin-bottom:8px;font-style:normal;color:#666;border-bottom:1px dashed #ccc;}</style><i>English</i><i>简体中文</i><i>Deutsch</i> … …</pre>
```

Nota quod `<style>` etiam `<pre>` definitum est.
