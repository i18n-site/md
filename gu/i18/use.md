# ઇન્સ્ટોલ કરો અને ઉપયોગ કરો

## વિન્ડોઝ પહેલા git bash ઇન્સ્ટોલ કરો

windows , [ડાઉનલોડ અને ઇન્સ્ટોલ કરવા માટે કૃપા કરીને અહીં ક્લિક કરો `git bash`](https://git-scm.com/download/win)

માં અનુગામી કામગીરી ચલાવો `git bash`

## ઇન્સ્ટોલ કરો

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### અનુવાદ ટોકન ગોઠવો

ટોકન કોપી કરવા માટે ક્લિક [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

બનાવો `~/.config/i18n.site.yml` તેમાં કોપી કરેલી સામગ્રી પેસ્ટ કરો, સામગ્રી નીચે મુજબ છે:

```
token: YOUR_API_TOKEN
```

વધુમાં, તમારે ચુકવણી માટે ક્રેડિટ કાર્ડ [i18n.site/payBill](//i18n.site/payBill) જરૂર છે (કોઈ રિચાર્જની જરૂર નથી, વેબસાઈટ આપમેળે વપરાશ અનુસાર ફી કપાત કરશે, [કિંમત માટે હોમપેજ જુઓ](/#price) ).

## વાપરવુ

### ડેમો પ્રોજેક્ટ

અનુવાદની ગોઠવણી શીખવા માટે કૃપા કરીને ડેમો પ્રોજેક્ટનો સંદર્ભ `i18` [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

ચીનમાં વપરાશકર્તાઓ ક્લોન કરી શકે છે [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

ક્લોનિંગ પછી, ડિરેક્ટરી દાખલ કરો અને અનુવાદ પૂર્ણ કરવા માટે ચલાવો `i18`

### ડિરેક્ટરી માળખું

ટેમ્પલેટ વેરહાઉસ ડિરેક્ટરીનું માળખું નીચે મુજબ છે

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` અનુવાદ કરાયેલ ડેમો ફાઇલો છે, જે માત્ર એક ઉદાહરણ છે અને કાઢી શકાય છે.

### અનુવાદ ચલાવો

ડિરેક્ટરી દાખલ કરો અને અનુવાદ કરવા માટે ચલાવો `i18`

### રીપોઝીટરીમાં ફાઇલો ઉમેરો

અનુવાદ ઉપરાંત, પ્રોગ્રામ નીચેની ફાઇલો પણ જનરેટ કરશે, કૃપા કરીને તેને ભંડારમાં ઉમેરો.

```
.i18n/hash
.i18n/cache/.gitignore
```

તેમાંથી, `.i18n/cache/.gitignore` સામગ્રી નીચે મુજબ છે :

```
**/*
!**/.gitignore
```

આનો અર્થ એ છે કે `.i18n/cache/` બધી ફાઇલોને અવગણો ( `.i18n/cache/.gitignore` સિવાય).

જો તમારું સંસ્કરણ નિયંત્રણ સોફ્ટવેર નથી `git` તો કૃપા કરીને આ ગોઠવણી અનુસાર તેને અવગણો.

## રૂપરેખાંકન ફાઇલ

તે `i18` વાક્ય અનુવાદ સાધનની રૂપરેખાંકન ફાઇલ `.i18n/conf.yml`

સામગ્રી નીચે મુજબ છે:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### સ્ત્રોત ભાષા &

રૂપરેખાંકન ફાઈલમાં, નીચેના `fromTo`

સ્ત્રોત ભાષા `en` , `zh ja ko de fr` અનુવાદની લક્ષ્ય ભાષા છે.

ભાષા કોડ જુઓ [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

ઉદાહરણ તરીકે, જો તમે ચાઇનીઝનું અંગ્રેજીમાં ભાષાંતર કરવા માંગતા હો, તો આ લાઇન ફરીથી લખો `zh: en`

જો તમે બધી સમર્થિત ભાષાઓમાં અનુવાદ કરવા માંગતા હો, તો કૃપા કરીને પછી ખાલી છોડી દો `:` દાખ્લા તરીકે

```
i18n:
  fromTo:
    en:
```

### ફાઇલને અવગણો

મૂળભૂત રીતે, સ્ત્રોત ભાષા નિર્દેશિકામાં `.md` અને `.yml` થી શરૂ થતી તમામ ફાઇલોનું અનુવાદ કરવામાં આવશે.

જો તમે અમુક ફાઇલોને અવગણવા માંગતા હોવ અને તેનો અનુવાદ ન કરો (જેમ કે અપૂર્ણ ડ્રાફ્ટ), તો તમે `ignore` રૂપરેખાંકનનો ઉપયોગ કરી શકો છો.

`.gitignore` ના સમાન વાક્યરચનાનો ઉપયોગ કરે [globset](https://docs.rs/globset/latest/globset/#syntax) `ignore`

ઉદાહરણ તરીકે, ઉપરોક્ત રૂપરેખાંકન ફાઇલનો અર્થ `_* ` છે કે `_` થી શરૂ થતી ફાઇલોનું ભાષાંતર કરવામાં આવશે નહીં.

## અનુવાદ નિયમો

### અનુવાદ સંપાદકોએ લીટીઓ ઉમેરવા અથવા કાઢી નાખવા જોઈએ નહીં

અનુવાદ સંપાદનયોગ્ય છે. મૂળ લખાણમાં ફેરફાર કરો અને મશીન-તેનો ફરીથી અનુવાદ કરો, અનુવાદમાં મેન્યુઅલ ફેરફારો ફરીથી લખવામાં આવશે નહીં (જો મૂળ ટેક્સ્ટના આ ફકરામાં ફેરફાર કરવામાં આવ્યો નથી).

પરંતુ મહેરબાની કરીને નોંધ કરો કે અનુવાદની રેખાઓ અને મૂળ લખાણ એકથી એકને અનુરૂપ હોવા જોઈએ. એટલે કે, અનુવાદનું સંકલન કરતી વખતે રેખાઓ ઉમેરો કે કાઢી નાખશો નહીં. નહિંતર, તે અનુવાદ સંપાદન કેશમાં મૂંઝવણ પેદા કરશે.

જો કંઈક ખોટું થાય, તો કૃપા કરીને [ઉકેલો માટે FAQ નો સંદર્ભ લો.](/i18/qa#H1)

### `YAML`

કમાન્ડ લાઇન ટૂલ સોર્સ લેંગ્વેજ ફાઇલ ડિરેક્ટરીમાં `.yml` સાથે સમાપ્ત થતી બધી ફાઇલો શોધી કાઢશે અને તેનો અનુવાદ કરશે.

* નોંધ કરો કે ફાઈલ નામનો પ્રત્યય હોવો `.yml` ( `.yaml` નહીં).

ટૂલ માત્ર ડિક્શનરીના મૂલ્યોનું ભાષાંતર કરે છે `.yml` ડિક્શનરી કીનો નહીં.

દાખ્લા તરીકે `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

તરીકે ભાષાંતર કરવામાં આવશે `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` ભાષાંતર મેન્યુઅલી પણ સુધારી શકાય છે (પરંતુ અનુવાદમાં કી અથવા લીટીઓ ઉમેરો કે કાઢી નાખશો નહીં).

`YAML` આધારે, તમે વિવિધ પ્રોગ્રામિંગ ભાષાઓ માટે સરળતાથી આંતરરાષ્ટ્રીય ઉકેલો બનાવી શકો છો.

## અદ્યતન ઉપયોગ

### અનુવાદ સબડિરેક્ટરી

જ્યાં સુધી તમે બનાવો `.i18n/conf.yml` (દર વખતે ડેમો પ્રોજેક્ટ ટેમ્પલેટથી શરૂ કરવાની જરૂર નથી), `i18` સારું કામ કરશે.

કમાન્ડ લાઇન ટૂલ બધી સબડિરેક્ટરીઝમાં `.i18n/conf.yml` શોધશે અને તેનું ભાષાંતર કરશે.

પ્રોજેક્ટ્સ કે જે આર્કિટેક્ચરનો ઉપયોગ કરે [monorepo](//monorepo.tools) તે ભાષાની ફાઇલોને સબડિરેક્ટરીઝમાં વિભાજિત કરી શકે છે.

### કસ્ટમ ઇન્સ્ટોલેશન ડિરેક્ટરી

તે મૂળભૂત રીતે સ્થાપિત થશે `/usr/local/bin`

જો લખવાની પરવાનગી ન હોય `/usr/local/bin` તે `~/.bin` પર ઇન્સ્ટોલ થશે.

પર્યાવરણ ચલો સેટ `TO` તમે સ્થાપન નિર્દેશિકાને વ્યાખ્યાયિત કરી શકો છો, ઉદાહરણ તરીકે :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
