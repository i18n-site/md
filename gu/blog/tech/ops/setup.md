# સર્વર શરૂ કરો (BTRFS પાર્ટીશનનો ઉપયોગ કરો, સ્નેપર સુનિશ્ચિત સ્નેપશોટ)

## કોન્ટાબો ખરીદી સર્વર

કોન્ટાબો જર્મનીમાં લાંબો ઇતિહાસ (20 વર્ષથી વધુ) અને ઘણા વપરાશકર્તાઓ (200,000 થી વધુ) સાથે સસ્તા VPS પ્રદાતા છે, તેથી ભાગી જવાની ચિંતા કરવાની જરૂર નથી.

ખરીદી કરતી VPS [contabo.com](//contabo.com) યુરો કિંમત પસંદ કરવાનું યાદ રાખો, જે સસ્તું છે.

<img alt="" src="https://i-01.eu.org/1718285782.avif">

કોન્ટાબો માટે એક વખતની ઇન્સ્ટોલેશન ફી છે.
જો કે, દર મહિને કેટલાક દિવસો હોય છે જ્યાં ઇન્સ્ટોલેશન ફી મફત હોય છે, ફક્ત નોંધણી કરો અને પ્રમોશનલ ઇમેઇલ્સ પર ધ્યાન આપો.

યુરોપમાં ફ્રી એરિયા ફી ઉપરાંત, અન્ય પ્રદેશોમાં ભાવ વધારાના વિવિધ સ્તરો છે.

મેં યુરોપિયન સર્વર પસંદ કર્યું, કોઈપણ રીતે, ક્લાઉડફ્લેરનો ઉપયોગ કર્યા પછી, સર્વર્સ વચ્ચે બહુ તફાવત નથી.

અંતિમ રૂપરેખાંકન નીચે મુજબ છે, 16GB મેમરી, 6CPU, 200 GB NVMe, 9.50 યુરો પ્રતિ માસ, 74.43 RMB ની સમકક્ષ.

**આ કિંમત અલીબાબા ક્લાઉડ અને એમેઝોનની કિંમતના લગભગ દસમા ભાગની છે** .

મેં પ્રદર્શન અને પરિમાણો ચલાવવા માટે ઉપયોગ [Yet-Another-Bench-Script](https://github.com/masonr/yet-another-bench-script) :

```

```

ડિસ્પ્લે મોડલ CPU સિંગલ-કોર પરફોર્મન્સ સ્કોર 1285 `AMD EPYC 9224 24-Core Processor`
, મલ્ટિ-કોર સ્કોર 4277 વધુ પરિણામો જુઓ [vpsbenchmarks.com](https://www.vpsbenchmarks.com/yabs/contabo-6c-16gb-3fb7d8)

મેં એક સમયે ત્રણ સર્વર ખરીદ્યા અને તેમને એક જ સમયે સક્રિય કર્યા આ ખાતરી કરે છે કે સર્વર્સ એક જ કમ્પ્યુટર રૂમમાં છે, જે ભવિષ્યમાં ઉચ્ચ ઉપલબ્ધતા બનાવવાનું સરળ બનાવે છે.

ઉચ્ચ ઉપલબ્ધતા દ્વારા વેબસાઇટની સુલભતા સુનિશ્ચિત કરીને તેના સર્વર્સ પ્રસંગોપાત અટકી જાય છે.

નવા વપરાશકર્તાઓએ ખરીદી કરતી વખતે વાસ્તવિક નામ પ્રમાણીકરણ માટે તેમના ID કાર્ડ સબમિટ કરવાની જરૂર છે.

## પાસવર્ડ-મુક્ત લૉગિન ગોઠવો

થોડીવાર પછી, સર્વર બધા સક્રિય થઈ ગયા.

પ્રથમ વસ્તુ એ છે કે સ્થાનિક મશીનને સંપાદિત કરો `~/.ssh/config` માટે ઉપનામ સેટ કરો IP અને પછી પાસવર્ડ-મુક્ત લૉગિન ગોઠવવા માટે `ssh-copy-id root@c2` ઉપયોગ કરો.

<img alt="" src="https://i-01.eu.org/1718287198.avif">

પ્રારંભિક રૂપરેખાંકન ઝડપથી પૂર્ણ કરવા માટે તમે આ [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) ફેરફાર કરી શકો છો.

પાસવર્ડ-મુક્ત રૂપરેખાંકન પૂર્ણ થયા પછી, તમે તેનો ઉપયોગ કરી શકો છો [pdsh](https://github.com/chaos/pdsh) તે બેચમાં ચલાવી શકાય છે, જે અનુકૂળ છે.

પ્રથમ ipv6 અને પરીક્ષણને મેન્યુઅલી સક્ષમ કરવાનું છે

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

<img alt="" src="https://i-01.eu.org/1718287969.avif">

પછી એક ક્લિક સાથે સિસ્ટમને ઉબુન્ટુ 24 પર અપગ્રેડ કરો અને સામાન્ય રીતે ઉપયોગમાં લેવાતા સોફ્ટવેરને ઇન્સ્ટોલ કરો.

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 એટલે ભાષાને ચાઇનીઝ અને ટાઇમ ઝોનને બેઇજિંગમાં સ્વિચ કરવું.

ડિફૉલ્ટ રૂપે ઇન્સ્ટોલ કરેલું સોફ્ટવેર મારી વ્યક્તિગત ઉપયોગની આદતો અનુસાર ગોઠવાયેલું છે, જેમ કે [mise](https://github.com/jdx/mise) , [direnv](https://direnv.net/) , વગેરે. જો જરૂરી હોય, તો તમે [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) વેરહાઉસની નકલ કરી શકો છો અને તેને જરૂર મુજબ સુધારી શકો છો.

આ પગલું લાંબો સમય લે છે અડધા કલાકની રાહ જોયા પછી, સામાન્ય રીતે ઉપયોગમાં લેવાતા સોફ્ટવેરનું અપગ્રેડ અને ઇન્સ્ટોલેશન સફળ થશે. તમે નીચેના આદેશ સાથે તેની પુષ્ટિ કરી શકો છો:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## mosh ssh ને ઝડપી બનાવે છે

યુરોપથી ચીન સુધીનું નેટવર્ક સ્થિર `ssh` તેનો ઉપયોગ કરવાની ભલામણ કરવામાં આવે છે [mosh](//mosh.org)

નીચે પ્રમાણે સ્થાનિક `~/.bin` (મુખ્યત્વે પર્યાવરણ ચલ `PATH` માં ઉમેરવામાં આવે છે) માં `c0` સ્ક્રિપ્ટ બનાવો:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

બનાવટ પછી, એક્ઝિક્યુટેબલ પરવાનગીઓ અને સોફ્ટ-લિંક આપો `c1` `c2` .

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

ભવિષ્યમાં, તમે સીધા જ દાખલ કરી શકો છો `c0` `c1` અને તમે `mosh` હાઇ-સ્પીડ ડાયરેક્ટ કનેક્શનનો ઉપયોગ કરી શકો છો અને દાખલ કરી શકો છો [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

## પાર્ટીશન ફોર્મેટને ext4 થી btrfs માં કન્વર્ટ કરો

સરળ બેકઅપ માટે ડિસ્ક સ્નેપશોટ લઈ શકો [btrfs](https://wikipedia.org/wiki/Btrfs) .

પ્રથમ રેસ્ક્યૂ સિસ્ટમ દાખલ કરો

<img alt="" src="https://i-01.eu.org/1718300448.avif">

ડેબિયન બચાવ પસંદ કરો

<img alt="" src="https://i-01.eu.org/1718300753.avif">

સર્વર પુનઃપ્રારંભ થાય ત્યાં સુધી રાહ જુઓ

ફિંગરપ્રિન્ટમાં ફેરફાર થશે ssh ઉપયોગ કરો `ssh-keygen -R ip`

પછી પાર્ટીશનને કન્વર્ટ કરવા માટે નીચેની સ્ક્રિપ્ટ ચલાવો

```bash
nohup bash -c "bash <(curl -s https://raw.githubusercontent.com/i18n-ops/ops/main/contabo/ext4_btrfs.sh) >/tmp/nohup.out 2>&1 && reboot" &
sleep 1 && tail -f /tmp/nohup.out
```

રૂપાંતરણ સફળ થયા પછી સ્ક્રિપ્ટ આપમેળે સિસ્ટમને પુનઃપ્રારંભ કરશે.

### btrfs કમ્પ્રેશન રેશિયો જુઓ

તે ડિફૉલ્ટ રૂપે આપમેળે સંકુચિત થશે કમ્પ્રેશન રેશિયો તપાસવા માટેનો આદેશ નીચે મુજબ છે

```bash
if ! command -v compsize &>/dev/null; then
apt install -y  btrfs-compsize
fi
compsize -x /
```

### મેન્યુઅલ કમ્પ્રેશન

જો તમે પરફેક્શનિસ્ટ છો, તો તમે મેન્યુઅલી સંપૂર્ણ ડિસ્ક કમ્પ્રેશન પણ કરી શકો છો

```bash
btrfs filesystem defragment -r -czstd /
```

નીચેની આકૃતિમાં બતાવ્યા પ્રમાણે, તમે જોઈ શકો છો કે નવી સિસ્ટમમાં મેન્યુઅલ કમ્પ્રેશન પછી, લગભગ 100MB વધુ સંકુચિત થયું હતું.

<img alt="" src="https://i-01.eu.org/2023/12/8NWlr4c.webp">

### સ્નેપર સાથે Btrfs સ્નેપશોટ બેકઅપ બનાવો

Btrfs બેકઅપ માટે શ્રેષ્ઠ ભાગીદાર [snapper](https://manual.siduction.org/sys-admin-btrfs-snapper_en.html) .

પ્રારંભિક સ્ક્રિપ્ટ `~/ubuntu` બનાવશે, ફક્ત નીચેનો આદેશ ચલાવો:

```
~/ubuntu/snapper/init.sh
```

તમને સ્ક્રિપ્ટ સામગ્રીમાં મળીશું [github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh](https://github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh)

સબવોલ્યુમ્સ જોવા માટે ઉપયોગ `snapper list-configs` કે જેના માટે સ્નેપશોટ સેટ કરેલ છે.

<img alt="" src="https://i-01.eu.org/1718302689.avif">

સબવોલ્યુમનો સ્નેપશોટ જોવા માટે ઉપયોગ `snapper -c mnt list`

તમે આ સબવોલ્યુમના ચોક્કસ બેકઅપ સેટિંગ્સ જોઈ શકો `/etc/snapper/configs/mnt`

[Btrfs ના સ્નેપશોટ ફંક્શન સાથે સરળતાથી રમો snapper](https://zhuanlan.zhihu.com/p/31082518)

## રેસ્ક્યુ સિસ્ટમ વિગતો

રેસ્ક્યૂ સિસ્ટમ દાખલ કરો 22 પોર્ટ સુલભ થઈ [tcping](https://github.com/paradiseduo/tcping) પછી, તમે 2-3 મિનિટ માટે તેને એક્સેસ કરી શકશો નહીં અને થોડીવાર રાહ જુઓ.

```
tcping ip 22 -c 9999
```

પોર્ટ એક્સેસિબિલિટી પર નજર રાખી શકાય છે.

### & રિસ્ટોર સ્નેપશોટ દાખલ કર્યા પછી હાર્ડ ડ્રાઈવને કેવી રીતે એક્સેસ કરવી

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

ઉદાહરણ તરીકે, તમે સ્નેપશોટ જોઈ શકો છો

```
snapper -c etc list
```

સ્નેપશોટ ફાઇલ ફેરફારો જુઓ

```
snapper -c etc status 5..6
```

ચોક્કસ તફાવત ટેક્સ્ટ જુઓ

```
snapper -c etc diff 3..5
```

સ્નેપશોટ પુનઃસ્થાપિત કરો: ઉદાહરણ તરીકે, સ્નેપશોટ 5 પછી બધા ફેરફારો પૂર્વવત્ કરો

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### લોગ દ્રઢતા સક્ષમ કરો

રેસ્ક્યુ સિસ્ટમ હેઠળ લોગ પર્સિસ્ટન્સને સક્રિય કરવા માટે તમારે મેન્યુઅલી અનુરૂપ ડિરેક્ટરી બનાવવાની અને કેટલાક વધારાના પગલાં ભરવાની જરૂર છે, કારણ કે તમારી પાસે ચાલી રહેલ systemd સેવા નથી. તમે અજમાવી શકો તે પગલાં અહીં છે:

પ્રથમ chroot, પછી

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

તમારી ઉબુન્ટુ સિસ્ટમમાં રીબૂટ કરો: ઉપરોક્ત ફેરફારો કર્યા પછી, તમારે તમારા મશીનને સામાન્ય રીતે અને રેસ્ક્યૂ સિસ્ટમને બદલે તમારી ઉબુન્ટુ સિસ્ટમમાં રીબૂટ કરવાની જરૂર છે.

આ રીતે, systemd-journald નવા રૂપરેખાંકન સાથે શરૂ થશે અને હાર્ડ ડિસ્ક પર લોગને સતત સંગ્રહિત કરવાનું શરૂ કરશે.

ધ્યાનમાં રાખો કે ઉપરોક્ત કામગીરીઓ રેસ્ક્યુ સિસ્ટમ પર કરવામાં આવે છે, તેથી તે ફેરફારો માત્ર એકવાર સિસ્ટમ રીબૂટ થઈ જાય અને બચાવ મોડમાં નહીં હોય ત્યારે જ પ્રભાવી થશે.

આનો અર્થ એ છે કે તમે હાલના લૉગ્સમાં કોઈ ફેરફાર તરત જ નિરંતર બનતા જોઈ શકશો નહીં, પરંતુ આ પછી જનરેટ થયેલા તમામ લૉગ્સ ચાલુ રહેશે.

આ રીતે તમે ssh સેવાના લોગ જોઈ શકો છો

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## મુશ્કેલ અને જટિલ રોગો પર નોંધો

### વિચિત્ર સમસ્યા હલ કરો: સર્વર ssh કરી શકતું નથી

મેં પહેલાં સ્વચાલિત સંકલન અને ડિપ્લોયમેન્ટ સ્ક્રિપ્ટ બનાવવા માટે ગીથબ ક્રિયાનો ઉપયોગ કર્યો, પરંતુ જાણવા મળ્યું કે જમાવટ સેવા ssh કરી શકતી નથી.

રેસ્ક્યૂ સિસ્ટમમાં પ્રવેશવું અને લોગ જોવું એ પરવાનગીની સમસ્યા છે:

`Missing privilege separation directory: /run/sshd`

તમે નીચેના આદેશ સાથે આ ડિરેક્ટરી બનાવી શકો છો

```
chown root:root /
systemd-tmpfiles --create
```

વારંવાર તપાસ કર્યા પછી, કારણ કે `1001` tar અપલોડ કરનાર વપરાશકર્તા `uid` github action

tar `--no-same-owner` `uid` `1001`

પરિણામે, rsync રૂટ ડિરેક્ટરીના વપરાશકર્તા જૂથને બદલશે, અને પછી તમે પુનઃપ્રારંભ કર્યા પછી ssh માં લૉગ ઇન કરી શકશો નહીં.

નીચે પ્રમાણે ડીકોમ્પ્રેસન સ્ક્રિપ્ટમાં ફેરફાર કરો:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
