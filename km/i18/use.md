## ប្រើ

### គម្រោងសាកល្បង

សូមមើលគម្រោង `i18` [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

អ្នកប្រើប្រាស់នៅប្រទេសចិនអាចក្លូនបាន [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

បន្ទាប់​ពី​ការ​ក្លូន​ សូម​ចូល​ក្នុង​ថត​ ហើយ​រត់ `i18`

### រចនាសម្ព័ន្ធថត

រចនាសម្ព័ន្ធថតឃ្លាំងគំរូមានដូចខាងក្រោម

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

ថត `en` មានឯកសារសាកល្បងដែលបានបកប្រែ ដែលគ្រាន់តែជាឧទាហរណ៍ ហើយអាចលុបបាន។

### ដំណើរការការបកប្រែ

បញ្ចូល​ថត​ហើយ​រត់ `i18`

### បន្ថែមឯកសារទៅឃ្លាំង

បន្ថែមពីលើការបកប្រែ កម្មវិធីនេះក៏នឹងបង្កើតឯកសារខាងក្រោមផងដែរ សូមបន្ថែមពួកវាទៅក្នុងឃ្លាំង។

```
.i18n/hash
.i18n/cache/.gitignore
```

ក្នុង​ចំណោម​នោះ ខ្លឹមសារ : `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

នេះមានន័យថាមិនអើពើឯកសារទាំងអស់នៅក្នុងថត `.i18n/cache/.gitignore` `.i18n/cache/`

ប្រសិនបើកម្មវិធីគ្រប់គ្រងកំណែរបស់អ្នកមិនមែនជា `git`

## ឯកសារកំណត់រចនាសម្ព័ន្ធ

`.i18n/conf.yml` វាគឺជាឯកសារកំណត់រចនាសម្ព័ន្ធរបស់ឧបករណ៍បកប្រែបន្ទាត់ `i18`

ខ្លឹមសារមានដូចខាងក្រោម៖

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ភាសាប្រភព &

នៅក្នុងឯកសារកំណត់រចនាសម្ព័ន្ធអ្នកក្រោមបង្គាប់របស់ `fromTo`

`en` ជាភាសាប្រភព `zh ja ko de fr` ជាភាសាគោលដៅនៃការបកប្រែ។

លេខកូដភាសាសូមមើល [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

ឧទាហរណ៍ បើ​អ្នក​ចង់​បក​ប្រែ​ភាសា​ចិន​ទៅ​ជា​ភាសា​អង់គ្លេស សូម​សរសេរ​បន្ទាត់​នេះ​ឡើង​វិញ `zh: en`

ប្រសិន​បើ​អ្នក​ចង់​បក​ប្រែ​ទៅ​គ្រប់​ភាសា​ដែល​អាច​ប្រើ​បាន សូម​ទុក​ឲ្យ​នៅ​ទទេ​បន្ទាប់​ពី `:` ឧទាហរណ៍

```
i18n:
  fromTo:
    en:
```

### មិនអើពើឯកសារ

តាមលំនាំដើម ឯកសារទាំងអស់ដែលចាប់ផ្តើមដោយ `.md` និង `.yml` នៅក្នុងថតភាសាប្រភពនឹងត្រូវបានបកប្រែ។

ប្រសិនបើ​អ្នក​ចង់​មិន​អើពើ​ឯកសារ​ជាក់លាក់ ហើយ​មិន​បកប្រែ​វា (ដូចជា​សេចក្តីព្រាង​ដែល​មិន​ទាន់​បញ្ចប់) អ្នក​អាច​ប្រើ​ការ​កំណត់​រចនាសម្ព័ន្ធ​វាល `ignore`

`ignore` ប្រើវាក្យសម្ព័ន្ធស្រដៀងនឹង `.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax)

ឧទាហរណ៍ ក្នុងឯកសារកំណត់រចនាសម្ព័ន្ធខាងលើ `_* ` មានន័យថាឯកសារដែលចាប់ផ្តើមដោយ `_` នឹងមិនត្រូវបានបកប្រែទេ។

## ច្បាប់នៃការបកប្រែ

### អ្នកកែសម្រួលការបកប្រែមិនគួរបន្ថែម ឬលុបបន្ទាត់ទេ។

ការបកប្រែអាចកែសម្រួលបាន។ កែប្រែអត្ថបទដើម ហើយបកប្រែដោយម៉ាស៊ីនម្តងទៀត ការកែប្រែដោយដៃចំពោះការបកប្រែនឹងមិនត្រូវបានសរសេរជាន់ពីលើទេ (ប្រសិនបើកថាខណ្ឌនៃអត្ថបទដើមនេះមិនត្រូវបានកែប្រែ)។

ប៉ុន្តែ​សូម​ចំណាំ​ថា បន្ទាត់​នៃ​ការ​បកប្រែ​និង​អត្ថបទ​ដើម​ត្រូវ​តែ​ត្រូវ​គ្នា​មួយ​ទៅ​មួយ។ នោះគឺកុំបន្ថែម ឬលុបបន្ទាត់នៅពេលចងក្រងការបកប្រែ។ បើមិនដូច្នោះទេ វានឹងធ្វើឱ្យមានការភ័ន្តច្រឡំនៅក្នុងឃ្លាំងសម្ងាត់កែសម្រួលការបកប្រែ។

ប្រសិនបើមានអ្វីមួយខុស សូមយោងទៅ [FAQ សម្រាប់ដំណោះស្រាយ។](/i18/qa#H1)

### `YAML`

ឧបករណ៍បន្ទាត់ពាក្យបញ្ជានឹងស្វែងរកឯកសារទាំងអស់ដែលបញ្ចប់ដោយ `.yml` នៅក្នុងថតឯកសារភាសាប្រភព ហើយបកប្រែពួកវា។

* ចំណាំថា បច្ច័យឈ្មោះឯកសារត្រូវតែជា `.yml` (មិនមែន `.yaml` )។

ឧបករណ៍នេះគ្រាន់តែបកប្រែតម្លៃវចនានុក្រមនៅក្នុង `.yml`

ឧទាហរណ៍ `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

នឹងប្រែថា `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

ការបកប្រែរបស់ `YAML` ក៏អាចត្រូវបានកែប្រែដោយដៃ (ប៉ុន្តែកុំបន្ថែម ឬលុបគ្រាប់ចុច ឬបន្ទាត់ក្នុងការបកប្រែ)។

ដោយផ្អែកលើ `YAML` ការបកប្រែ អ្នកអាចបង្កើតដំណោះស្រាយអន្តរជាតិយ៉ាងងាយស្រួលសម្រាប់ភាសាសរសេរកម្មវិធីផ្សេងៗ។

## ការប្រើប្រាស់កម្រិតខ្ពស់

### ថតរងការបកប្រែ

ដរាបណាអ្នកបង្កើត `.i18n/conf.yml` (មិនចាំបាច់ចាប់ផ្តើមពីគំរូគម្រោងសាកល្បងរាល់ពេលនោះទេ) `i18` នឹងដំណើរការល្អ។

ឧបករណ៍បន្ទាត់ពាក្យបញ្ជានឹងរកឃើញការកំណត់រចនាសម្ព័ន្ធ `.i18n/conf.yml` នៅក្នុងថតរងទាំងអស់ ហើយបកប្រែវា។

គម្រោងដែលប្រើស្ថាបត្យកម្ម [monorepo](//monorepo.tools) អាចបំបែកឯកសារភាសាទៅជាថតរង។

### ថតដំឡើងផ្ទាល់ខ្លួន

វា​នឹង​ត្រូវ​បាន​ដំឡើង​ទៅ `/usr/local/bin`

ប្រសិនបើ `/usr/local/bin` មិនមានការអនុញ្ញាតក្នុងការសរសេរទេ វានឹងត្រូវបានដំឡើងទៅ `~/.bin` ។

កំណត់អថេរ : `TO`

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
