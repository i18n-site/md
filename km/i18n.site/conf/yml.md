# .i18n/conf.yml

`i18n.site` ឯកសារកំណត់រចនាសម្ព័ន្ធគឺ `.i18n/conf.yml` ។

លើកលែងតែការកំណត់ `ignore:` និង `i18n:` នៃ [`i18`](/i18) ឯកសារកំណត់រចនាសម្ព័ន្ធមានដូចខាងក្រោម៖

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

ក្នុង​ចំណោម​នោះ ធាតុ​កំណត់​រចនាសម្ព័ន្ធ `ext:` របស់ `upload` មាន​ន័យ​ថា​មាន​តែ `.md` ប៉ុណ្ណោះ​នឹង​ត្រូវ​បាន​ផ្ទុក​ឡើង​នៅ​ពេល​បោះពុម្ព​ផ្សាយ។

### `use: Toc` គំរូឯកសារតែមួយ (មានគ្រោង)

កំណត់ : `nav`

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` មានន័យ​ថា​ប្រើ `Toc` rendering ដែល​កំពុង​ render មួយ `Markdown` template។

`TOC` គឺជាអក្សរកាត់នៃ `Table of Contents` `Markdown`

`url:` តំណាងឱ្យផ្លូវឯកសាររបស់ `Markdown` ( `/` ត្រូវគ្នាទៅនឹងថតឫស `/README.md` ឈ្មោះឯកសារនេះទាមទារបុព្វបទអក្សរធំ និងបច្ច័យអក្សរតូច)។

### `use: Md` គំរូឯកសារតែមួយ (គ្មានគ្រោង)

`Md` Template គឺដូចគ្នាទៅនឹង `Toc` ទាំងពីរត្រូវបានប្រើដើម្បីបង្ហាញឯកសារ `Markdown` តែមួយ។ ទោះយ៉ាងណាក៏ដោយ `Md` គំរូមិនបង្ហាញគ្រោងនៅក្នុងរបារចំហៀងទេ។

អ្នកអាចកែប្រែ `use: Toc` នៅក្នុងការកំណត់ខាងលើទៅជា `use: Md` ដំណើរការ `i18n.site` ក្នុងថត `md` ម្តងទៀត ហើយបន្ទាប់មកចូលទៅកាន់ URL មើលជាមុននៃការអភិវឌ្ឍន៍ ដើម្បីសង្កេតមើលការផ្លាស់ប្តូរនៅលើគេហទំព័រ។

### ការផ្ទុកលំនាំដើមដោយគ្មានផ្លូវកំណត់រចនាសម្ព័ន្ធ

ប្រសិនបើបុព្វបទផ្លូវនៃផ្លូវជាក់លាក់មួយដែលត្រូវបានចូលប្រើមិនត្រូវបានកំណត់រចនាសម្ព័ន្ធនៅក្នុង `nav:` ឯកសារ `MarkDown` ដែលទាក់ទងទៅនឹងផ្លូវនឹងត្រូវបានផ្ទុកតាមលំនាំដើម និងបង្ហាញដោយប្រើ `Md` template ។

ឧទាហរណ៍ ប្រសិនបើអ្នកចូលមើល `/test` និង `nav:` ត្រូវបានកំណត់រចនាសម្ព័ន្ធដោយគ្មានផ្លូវនេះ ហើយភាសាទំព័រគឺភាសាអង់គ្លេស (កូដ `en` ) គំរូនឹងត្រូវបានផ្ទុក `/en/test.md` និងបង្ហាញដោយប្រើ `Md` តាមលំនាំដើម។

ប្រសិនបើ `/en/test.md` ឯកសារនេះមិនមានទេ ទំព័រ `404` នឹងត្រូវបានបង្ហាញ។

### `use: Doc` , ពុម្ពឯកសារច្រើន។

នៅក្នុងឯកសារកំណត់រចនាសម្ព័ន្ធ៖

```
  - i18n: blog
    use: Doc
```

ចង្អុលបង្ហាញការប្រើប្រាស់ `Doc`

`Doc` Template គាំទ្រការរួមបញ្ចូល `MarkDown` ច្រើនដើម្បីបង្កើតគ្រោងឯកសារសម្រាប់គម្រោងតែមួយ ឬច្រើន។

#### គម្រោងតែមួយ (ឯកសារច្រើន)

`blog` នៅក្នុងខាងលើគឺជារបៀបគម្រោងតែមួយនៃ `Doc` ។
