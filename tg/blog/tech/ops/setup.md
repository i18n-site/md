## Воридшавӣ Бидуни Паролро Танзим Кунед

Пас аз чанд дақиқа, ҳама серверҳо фаъол шуданд.

Аввалин IP ки таҳрир кардани `~/.ssh/config` и мошини маҳаллӣ барои гузоштани тахаллус ва сипас `ssh-copy-id root@c2` барои танзими вуруди бидуни парол истифода кунед.

![](https://i-01.eu.org/1718287198.avif)

Шумо метавонед ин [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) тағир диҳед, то конфигуратсияи оғозёбиро зудтар анҷом диҳед.

Пас аз [pdsh](https://github.com/chaos/pdsh) итмом расонидани конфигуратсияи бидуни парол, шумо метавонед онро дар гурӯҳҳо истифода баред, ки ин қулай аст.

Аввал ин аст, ки ба таври дастӣ фаъол кардани ipv6 ва санҷиш

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

![](https://i-01.eu.org/1718287969.avif)

Пас бо як клик системаро ба Ubuntu 24 навсозӣ кунед ва нармафзори маъмулан истифодашавандаро насб кунед.

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 маънои гузаштани забон ба чинӣ ва минтақаи вақт ба Пекинро дорад.

Нармафзоре, ки бо нобаёнӣ насб шудааст, мувофиқи одатҳои истифодаи шахсии ман, ба монанди [mise](https://github.com/jdx/mise) [direnv](https://direnv.net/) , ва ғайра танзим карда мешавад. Агар лозим бошад, шумо метавонед анбори [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) нусхабардорӣ кунед ва дар ҳолати зарурӣ онро тағир диҳед.

Ин қадам пас аз интизории ним соат вақти зиёдро мегирад, навсозӣ ва насби нармафзори маъмулан истифодашаванда муваффақ хоҳад шуд. Шумо метавонед инро бо фармони зерин тасдиқ кунед:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## mosh Ssh-Ро Суръат Мебахшад

Шабака аз Аврупо ба `ssh` [mosh](//mosh.org) нест.

Дар директорияи маҳаллӣ `~/.bin` `c0` скрипт эҷод кунед (асосан ба тағирёбандаи муҳити зист `PATH` илова карда мешавад) ба таври зерин:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

Пас аз эҷод, иҷозатҳои иҷрошаванда ва пайванди нармафзори `c1` ва `c2` диҳед.

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

Дар оянда, шумо метавонед мустақиман `c0` ва `c1` ворид кунед, то `mosh` пайвасти мустақими баландсуръатро истифода баред ва ворид кунед [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

Агар шумо идеалист бошед, шумо инчунин метавонед фишурдани дискро дастӣ иҷро кунед Фармон чунин аст:

## Тафсилоти Системаи Наҷот

Ба системаи наҷот ворид шавед 22 пас аз [tcping](https://github.com/paradiseduo/tcping) дастрас будани бандар, шумо наметавонед ба он бо пароли муқарраршуда барои 2-3 дақиқа интизор шавед ва ин хуб хоҳад буд.

```
tcping ip 22 -c 9999
```

Дастрасии портро метавон назорат кард.

### Чӣ Тавр & Диски Сахт Пас Аз Ворид Кардани Наҷотдиҳӣ Дастрасӣ Пайдо Кунед

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

Масалан, шумо метавонед аксро бинед

```
snapper -c etc list
```

Тағироти файли аксҳоро бинед

```
snapper -c etc status 5..6
```

Дидани матни мушаххаси фарқият

```
snapper -c etc diff 3..5
```

Барқарор кардани акс: Масалан, ҳама тағиротро пас аз акси 5 бекор кунед

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### Истиқлолияти Сабтро Фаъол Созед

Даргиронидани устувории сабт дар системаи наҷотдиҳӣ аз шумо талаб мекунад, ки директорияи мувофиқро дастӣ созед ва чанд қадами иловагиро иҷро кунед, зеро шумо шояд хидмати системавии коркунанда надошта бошед. Инҳо қадамҳое ҳастанд, ки шумо метавонед кӯшиш кунед:

Аввал chroot, баъд

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

Ба системаи Ubuntu-и худ аз нав оғоз кунед: Пас аз ворид кардани тағиротҳои дар боло зикршуда, шумо бояд мошини худро ба таври муқаррарӣ ва ба системаи Ubuntu ба ҷои системаи наҷот аз нав оғоз кунед.

Бо ин роҳ, systemd-journald бо конфигуратсияи нав оғоз мекунад ва сабтҳоро дар диски сахт доимӣ нигоҳ медорад.

Дар хотир доред, ки амалиётҳои дар боло зикршуда дар системаи наҷотдиҳӣ иҷро карда мешаванд, бинобар ин, ин тағиротҳо то он даме, ки система бозоғоз нашавад ва дигар дар ҳолати наҷотдиҳӣ набошад, эътибор пайдо намекунанд.

Ин маънои онро дорад, ки шумо дарҳол ягон тағиротро дар гузоришҳои мавҷуда, ки доимӣ мешаванд, нахоҳед дид, аммо ҳамаи гузоришҳои баъд аз он тавлидшуда боқӣ хоҳанд монд.

Бо ин роҳ шумо метавонед сабтҳои хидмати ssh-ро бубинед

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## Қайдҳо Дар Бораи Бемориҳои Душвор Ва Мураккаб

### Масъалаи Аҷибро Ҳал Кунед: Сервер Ssh-Ро Иҷро Карда Наметавонад

Ман қаблан амали github-ро барои сохтани скрипти худкор ва таҳрир истифода бурдам, аммо дарёфтам, ки хидмати ҷойгиркунӣ ssh буда наметавонад.

Ворид шудан ба системаи наҷотдиҳӣ ва дидани гузоришҳо як масъалаи иҷозат аст:

`Missing privilege separation directory: /run/sshd`

Шумо метавонед ин директорияро бо фармони зерин эҷод кунед

```
chown root:root /
systemd-tmpfiles --create
```

Пас аз таҳқиқоти такрорӣ, ин аст github action ки корбари `uid` аз tar `1001` аст

Агар параметри `--no-same-owner` мавҷуд набошад, он ба `uid` барои `1001` корбар кушода мешавад tar

Дар натиҷа, rsync гурӯҳи корбарони директорияи решаро тағир медиҳад ва пас аз бозоғозӣ шумо наметавонед ба ssh ворид шавед.

Скрипти декомпрессиро ба таври зерин тағир диҳед:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
