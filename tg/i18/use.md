## Истифода Бурдан

### Лоиҳаи Намоишӣ

Лутфан ба лоиҳаи намоишӣ муроҷиат кунед [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18) то конфигуратсияи тарҷумаро омӯзед `i18`

Истифодабарандагон дар Чин метавонанд клон кунанд [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

Пас аз клонкунӣ, директорияро ворид кунед ва барои анҷом додани тарҷума иҷро кунед `i18`

### Сохтори Директория

Сохтори феҳристи анбори шаблон чунин аст

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

Феҳристи `en` файлҳои намоишии тарҷумашударо дар бар мегирад, ки онҳо танҳо намунаанд ва метавонанд нест карда шаванд.

### Тарҷумаро Иҷро Кунед

Директорияро ворид кунед ва барои тарҷума иҷро `i18` .

### Файлҳоро Ба Анбор Илова Кунед

Илова ба тарҷума, барнома инчунин файлҳои зеринро тавлид мекунад, лутфан онҳоро ба анбор илова кунед.

```
.i18n/hash
.i18n/cache/.gitignore
```

: байни онхо мазмуни `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

Ин маънои `.i18n/cache/.gitignore` дорад, ки ҳамаи файлҳои дар феҳристи `.i18n/cache/`

Агар нармафзори идоракунии версияи шумо набошад `git` лутфан онро мувофиқи ин конфигуратсия сарфи назар кунед.

## Файли Конфигуратсия

Ин файли конфигуратсияи воситаи тарҷумаи сатри `i18` `.i18n/conf.yml`

Мазмуни он чунин аст:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Забони Тарҷума &

Дар файли конфигуратсия тобеъони `fromTo`

`en` забони сарчашма, `zh ja ko de fr` забони ҳадафи тарҷума аст.

Рамзи забонро бубинед [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

Масалан, агар шумо хоҳед, ки хитоиро ба англисӣ тарҷума кунед, ин сатрро аз нав нависед `zh: en`

Агар шумо хоҳед, ки ба ҳамаи забонҳои дастгирӣшаванда тарҷума кунед, лутфан пас аз `:` барои намуна

```
i18n:
  fromTo:
    en:
```

### Файлро Нодида Гиред

Бо нобаёнӣ, ҳама файлҳое, ки бо `.md` ва `.yml` сар карда мешаванд, тарҷума карда мешаванд.

Агар шумо хоҳед, ки баъзе файлҳоро сарфи назар кунед ва онҳоро тарҷума накунед (масалан, лоиҳаҳои нотамом), шумо метавонед конфигуратсияи майдони `ignore`

Синтаксиси шабеҳи `.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax) мебарад `ignore`

Масалан, дар файли конфигуратсияи дар боло зикршуда маънои онро `_* ` , ки файлҳои бо `_` саршуда тарҷума карда намешаванд.

## Қоидаҳои Тарҷума

### Муҳаррирони Тарҷума Набояд Сатрҳоро Илова Ё Нест Кунанд

Тарҷума таҳриршаванда аст. Матни аслиро тағир диҳед ва онро дубора бо мошин тарҷума кунед, тағиротҳои дастӣ ба тарҷума дубора навишта намешаванд (агар ин параграфи матни аслӣ тағир дода нашуда бошад).

Аммо лутфан қайд кунед, ки сатрҳои тарҷума ва матни аслӣ бояд як ба як мувофиқат кунанд. Яъне ҳангоми тартиб додани тарҷума сатрҳоро илова ва нест накунед. Дар акси ҳол, он боиси нофаҳмиҳо дар кэши таҳрири тарҷума мегардад.

Агар чизе хато кунад, лутфан ба [FAQ барои ҳалли мушкилот муроҷиат кунед.](/i18/qa#H1)

### `Yaml`

Воситаи сатри фармон ҳамаи файлҳоро дар феҳристи файли забони аслӣ пайдо `.yml` ва онҳоро тарҷума мекунад.

* Дар хотир доред, ки суффикси номи файл бояд `.yml` (на `.yaml` ).

Восита танҳо арзишҳои луғатро дар `.yml` тарҷума мекунад, на калидҳои луғат.

барои намуна `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

ҳамчун тарҷума карда мешавад `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

Тарҷумаи `YAML` -ро низ дастӣ тағир додан мумкин аст (аммо калидҳо ё сатрҳоро дар тарҷума илова ё нест накунед).

Дар асоси `YAML` , шумо метавонед ба осонӣ қарорҳои байналмилалӣ барои забонҳои гуногуни барномасозӣ созед.

## Истифодаи Пешрафта

### Зеркаталоги Тарҷума

То он даме, ки шумо эҷод `.i18n/conf.yml` (ҳар вақт аз қолаби лоиҳаи намоишӣ оғоз кардан лозим нест), `i18` хуб кор мекунад.

Асбоби сатри фармон `.i18n/conf.yml` дар ҳама зеркаталогҳо пайдо мекунад ва онро тарҷума мекунад.

Лоиҳаҳое, ки меъмории [monorepo](//monorepo.tools) -ро истифода мебаранд, метавонанд файлҳои забонро ба зеркаталогҳо тақсим кунанд.

<img alt="" src="https://p.3ti.site/1719910016.avif">

### Феҳристи Насби Фармоишӣ

Он ба таври нобаёнӣ насб карда мешавад `/usr/local/bin`

Агар иҷозати навиштан надошта бошад `/usr/local/bin` он ба `~/.bin` насб карда мешавад.

Тағйирёбандаҳои муҳити зистро таъин кунед `TO` Шумо метавонед директорияи насбкуниро муайян кунед, масалан :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
