# .i18n/conf.yml

Файли конфигуратсия `.i18n/conf.yml` аст `i18n.site`

Ба истиснои танзимоти `ignore:` ва `i18n:` [`i18`](/i18) файли конфигуратсия чунин аст:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

Дар байни онҳо, `ext:` конфигуратсияи `upload` маънои онро дорад, ки ҳангоми нашр танҳо `.md` бор карда мешавад.

## Навигарии Болоӣ

имконоти конфигуратсия, `nav:` ба менюи паймоиш дар болои саҳифаи асосӣ мувофиқ аст.

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

Дар байни онхо ба `en/i18n.yml`中`home: Home` `i18n: home`

`en/i18n.yml` ба бисёр забонҳо тарҷума карда мешавад, ба монанди `zh/i18n.yml` .

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

Пас аз анҷоми тарҷума, шумо метавонед арзиши `yml` -ро дар тарҷума тағир диҳед, аммо калиди тарҷума `yml` -ро илова кунед ё нест накунед.

### `use: Toc` Қолаби Як Файл (Бо Контур)

: `nav`

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` маънои истифодаи `Toc` -ро дорад, ки як қолаби `Markdown` медиҳад.

`TOC` ихтисораи `Table of Contents` аст. Вақте ки ин қолаб намоиш дода мешавад, контури `Markdown` файл дар панели паҳлӯ нишон дода мешавад.

`url:` роҳи файли `Markdown` -ро ифода мекунад ( `/` ба директорияи реша мувофиқ аст `/README.md` , ин номи файл префикси калон ва суффикси хурдро талаб мекунад).

### `use: Md` , Қолаби Файли Ягона (Нақша Нест)

`Md` якхела бо `Toc` аст, ҳарду барои намоиши як файли `Markdown` истифода мешаванд. Аммо `Md` шаблон контурро дар панели паҳлӯ нишон намедиҳад.

Шумо метавонед `use: Toc` дар конфигуратсияи дар боло овардашуда ба `use: Md` тағир диҳед, `i18n.site` дар феҳристи `md` бори дигар кор кунед ва сипас ба URL-и пешнамоиши рушд ворид шавед, то тағиротро дар сафҳаи асосӣ мушоҳида кунед.

### Боркунии Пешфарз Бе Роҳи Конфигуратсия

Агар префикси роҳи роҳи муайяне, ки дастрас аст, дар `nav:` файли `MarkDown` мувофиқ ба роҳ бо нобаёнӣ бор карда мешавад ва бо истифода аз қолаби `Md` намоиш дода мешавад.

Масалан, агар шумо ба `/test` ташриф оваред ва `nav:` бе ин роҳ танзим шуда бошад ва забони саҳифа англисӣ (код `en` ) бошад, қолаб ба таври пешфарз бор карда мешавад `/en/test.md` ва бо истифода аз `Md` .

`/en/test.md` ин файл мавҷуд набошад, саҳифаи пешфарз `404` намоиш дода мешавад.

### `use: Doc` , Қолаби Бисёрфайл

Дар файли конфигуратсия:

```
  - i18n: blog
    use: Doc
```

Истифодаи `Doc` -ро барои намоиши қолаб нишон медиҳад.

`Doc` ҳамгиро кардани якчанд `MarkDown` барои тавлиди контурҳои ҳуҷҷат барои лоиҳаҳои як ё якчанд дастгирӣ мекунад.

#### Лоиҳаи Ягона (Файлҳои Сершумор)

`blog` дар боло реҷаи як-лоиҳаи `Doc` аст.

```
  - i18n: blog
    url: blog
    use: Doc
```

##### Вақте Ки url Холӣ Аст, Он Ба Арзиши i18n Муқаррар Карда Мешавад

Агар шумо нанависед `url` `url` ба арзиши `i18n` .

Усули навиштани боло ба доштани `url: blog` баробар аст ва файли мувофиқи он `en/blog/TOC` аст.

#### Бисёр Лоиҳаҳо

`.i18n/conf.yml` Конфигуратсия дар `i18n:doc` реҷаи бисёрҷониба аст.

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

Дар ин ҷо, `menu: NB demo1,demo2` маънои истифодаи `NB` барои намоиш додани менюи афтанда.

`NB` , ихтисораи `Name Breif` аст, ки нишон медиҳад, ки менюи афтанда метавонад ном ва шиори лоиҳаро нишон диҳад.

`NB` аз паи параметр `demo1,demo2` ба он дода мешавад.
** : ** `,` `demo1,demo2`

Барои параметрҳои дар боло зикршуда, файли индекси директорияи мувофиқ ин аст:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### Индекси Ҷадвали Мундариҷаи TOC

`i18n.site` Васлкунаки `js` `.i18n/hook/after.tran/TOC.js` дар анбори намоишӣ иҷро мекунад `json` то файли индекси директорияро `doc` мувофиқ ба конфигуратсияи шаблон `TOC` эҷод кунад

Агар шумо `doc` истифода баред, шумо бояд ин плагинро дошта бошед.

Агар шумо `i18n.site` аз ҷузвдони холӣ оғоз кунед, фаромӯш накунед, ки `.i18n` дар лоиҳаи намоишӣ ба феҳристи худ нусхабардорӣ кунед.

`Doc` ҷадвали мундариҷаро дар асоси `json` тавлидшуда пешкаш мекунад.

##### Сатҳи Волидайн Танҳо Унвонро Менависад, На Мундариҷа.

Ҳангоме ки сатҳҳои сершумор вуҷуд доранд, сатҳи волидайн танҳо унвонро менависад, на мундариҷа. Дар акси ҳол, матбаа вайрон мешавад.

##### Лоиҳаи README.md 

`README.md` -и лоиҳа, масалан, шумо метавонед мундариҷаро дар `en/demo2/README.md` нависед.

Дар хотир доред, ки мундариҷаи ин файл ҷадвали мундариҷаро нишон намедиҳад, аз ин рӯ тавсия дода мешавад, ки дарозиро маҳдуд кунед ва сарсухани кӯтоҳ нависед.

##### Чӣ Тавр TOC-Ро Ба Таври Оммавӣ Интиқол Додан Мумкин Аст?

`TOC` Файл бояд дар феҳристи забони сарчашма ҷойгир карда шавад.

Масалан, агар забони аслӣ чинӣ бошад, дар боло `zh/blog/TOC` `TOC`

Агар забони ибтидоӣ тағир дода шавад, шумо бояд `TOC` бо забони муайяни лоиҳа ба забони дигар интиқол диҳед.

Шумо метавонед ба фармонҳои зерин муроҷиат кунед:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

Лутфан `en/` ва `zh/` дар фармони боло ба рамзи забони худ тағир диҳед.
