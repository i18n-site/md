# স্থাপন করুন &

## কনফিগারেশন টোকেন

`i18` অনুবাদ টুলটি এম্বেড করা হয়েছে, অনুগ্রহ `i18n.site` [এখানে ক্লিক করুন এক্সেস টোকেন কনফিগার করতে `i18` নথিটি দেখুন](/i18/use) ।

## ইনস্টল করুন

```sh
bash <(curl -sS https://i.i18n.site) i18n.site
```

## ডেমো প্রকল্প

আসুন একটি ডেমো প্রকল্প দিয়ে শুরু করি এবং কীভাবে ব্যবহার করতে হয় তা শিখি `i18n.site`

আমরা প্রথমে ডেমো সংগ্রহস্থল ক্লোন করি এবং নিম্নরূপ কমান্ডটি চালাই:

```
git clone https://github.com/i18n-site/demo.i18n.site.git md
git clone https://github.com/i18n-site/demo.i18n.site.docker.git docker
```

চীনের মূল ভূখন্ডের ব্যবহারকারীরা করতে পারেন:

```
git clone https://atomgit.com/i18n/demo.i18n.site.git md
git clone https://atomgit.com/i18n/demo.i18n.site.docker.git docker
```

কোড বেস ক্লোনের ডিরেক্টরির নাম অবশ্যই `docker` সাথে স্থানীয় পূর্বরূপের সুবিধার্থে `md` হতে হবে `demo.i18n.site`

### বিষয়বস্তু পোস্ট করুন

একটি [একক-পৃষ্ঠা অ্যাপ্লিকেশন](https://developer.mozilla.org/docs/Glossary/SPA) আর্কিটেকচার গ্রহণ করে, এবং ওয়েবসাইট প্রবেশদ্বার পৃষ্ঠা এবং ওয়েবসাইটের বিষয়বস্তু স্বাধীনভাবে স্থাপন করা হয় `i18n.site`

উপরের অনুবাদটি চালানোর পরে, ডিরেক্টরিতে `htm` এবং `v` ডিরেক্টরি তৈরি হবে `md/out/dev`

এখানে `dev` নির্দেশ করে যে এটি `.i18n/htm/dev.yml` কনফিগারেশন ফাইলের উপর ভিত্তি করে তৈরি করা হয়েছে।

বিষয়বস্তু : `dev`

ডাইরেক্টরির অধীনে `htm` ওয়েবসাইট এন্ট্রান্স পেজ।

ডিরেক্টরীতে ভার্সন নম্বর সহ ওয়েবসাইটের বিষয়বস্তু রয়েছে `v`

সংস্করণ নম্বর নির্বিশেষে স্থানীয় পূর্বরূপ `out/dev/v/0.1.0`

অফিসিয়াল রিলিজের জন্য, পরিবর্তিত ফাইলগুলি নতুন সংস্করণ নম্বর ডিরেক্টরিতে অনুলিপি করা হবে।

#### কনফিগারেশন ফাইলটি নির্দিষ্ট করতে -c ব্যবহার করুন

বিভিন্ন কনফিগারেশন ফাইল ডিরেক্টরির অধীনে সংশ্লিষ্ট ডিরেক্টরি তৈরি করবে `out`

উদাহরণস্বরূপ `.i18n/htm/ol.yml` একটি `out/ol` ডিরেক্টরি তৈরি করবে।

এবং `ol.yml` ডিফল্ট কনফিগারেশন `dev.yml`

`dev` হল `development` এর সংক্ষিপ্ত রূপ, যা ডেভেলপমেন্ট এনভায়রনমেন্টের প্রতিনিধিত্ব করে, এটি স্থানীয় পূর্বরূপের জন্য ব্যবহৃত হয় এবং এটি ডিফল্ট কনফিগারেশন ফাইল।
`online` এর সংক্ষিপ্ত রূপ, যা অনলাইন পরিবেশের প্রতিনিধিত্ব করে, অফিসিয়াল রিলিজের জন্য ব্যবহৃত হয় এবং এটি `npm` কমান্ড লাইন প্যারামিটার ব্যবহার করে `-n` প্রকাশ করার সময় ডিফল্ট কনফিগারেশন ফাইল `ol`

কনফিগারেশন ফাইলের নাম উল্লেখ করতে `--htm_conf` কমান্ড লাইনে অন্যান্য কনফিগারেশন ফাইলও তৈরি করতে পারেন:

উদাহরণ স্বরূপ:
```
i18n.site --htm_conf yourConfig --save
```

এখানে আপডেট রিলিজ সংখ্যা নির্দেশ করে `--save`

#### <a rel=id href="#npm" id="npm"></a> npmjs.com এ বিষয়বস্তু প্রকাশ করুন

কন্টেন্ট প্রকাশ [npmjs.com](//npmjs.com) হল প্রস্তাবিত ডিফল্ট সমাধান (দেখুন [ফ্রন্ট-এন্ড হাই অ্যাভেলেবিলিটি](/i18n.site/feature#ha) )।

##### npm login & মুক্তি

ইনস্টল করুন `nodejs` লগ ইন করতে `npm login` ব্যবহার করুন।

`i18n.site` করুন `md/.i18n/htm/ol.yml` আপনার নিজের `npm` প্যাকেজের নাম `v: //unpkg.com/i18n.site`

প্যাকেজ নাম হিসাবে ওয়েবসাইট ডোমেইন নাম ব্যবহার করা একটি ভাল পছন্দ [npmjs.com](//npmjs.com)

`v:` `npm` **`//unpkg.com/`** `i18n.site` `/.v`

অনুবাদ এবং প্রকাশ করার জন্য ডিরেক্টরিতে `i18n.site --npm` বা `i18n.site -n` চালান `md`

আপনি যদি প্রকাশ করার জন্য একটি অবিচ্ছিন্ন ইন্টিগ্রেশন এনভায়রনমেন্ট ব্যবহার করেন, `nodejs` এটিকে ইন্সটল করার দরকার নেই শুধু লগ-ইন করা এবং প্রকাশিত অনুমতি `~/.npmrc` পরিবেশে কপি করুন।

আপনি যদি প্যাকেজের নামটি `v:` in `ol.yml` এ পরিবর্তন করেন, **অনুগ্রহ করে প্রথমে `.i18n/v/ol` মুছে ফেলুন** এবং তারপরে প্রকাশ করুন।

##### npm দ্বারা প্রকাশিত প্রক্সি সার্ভার

যদি চীনের মূল ভূখন্ডের ব্যবহারকারীরা নেটওয়ার্ক সমস্যার সম্মুখীন হয় এবং `npm` প্রকাশ করতে অক্ষম হয়, তাহলে তারা প্রক্সি সার্ভার কনফিগার করতে পরিবেশ পরিবর্তনশীল `https_proxy` সেট করতে পারে।

ধরে নিচ্ছি যে আপনার প্রক্সি সার্ভার পোর্ট `7890` আপনি লিখতে পারেন:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### স্ব-হোস্ট করা সামগ্রী

আপনি যদি বিষয়বস্তু স্ব-হোস্ট করতে চান, প্রথমে সম্পাদনা `md/.i18n/htm/ol.yml` এবং `v: //unpkg.com/i18n.site` আপনার URL উপসর্গে পরিবর্তন করুন, যেমন `v: //i18n-v.xxx.com` ।

ডিরেক্টরি লিখুন এবং চালান `md`

```
i18n.site --htm_conf ol --save
```

বা সংক্ষেপণ

```
i18n.site -c ol -s
```

তারপর, `v:` এ সেট করা ইউআরএল প্রিফিক্স পাথে `md/out/ol/v` ডিরেক্টরির বিষয়বস্তু কনফিগার করুন।

অবশেষে, **`1s` `/.v` শেষ হওয়া পাথের ক্যাশে সময় কনফিগার করুন** , অন্যথায় নতুন প্রকাশিত বিষয়বস্তু অবিলম্বে অ্যাক্সেস করা যাবে না।

অপ্রয়োজনীয় অনুরোধ কমাতে অন্যান্য পাথের জন্য ক্যাশে সময় এক বছর বা তার বেশি সেট করা যেতে পারে।

##### s3 তে কন্টেন্ট হোস্ট করুন

স্ব-হোস্টের জন্য, আপনার নিজের সার্ভার ব্যবহার করার জন্য, `CDN` সাধারণ বিকল্প হল `S3` +

আপনি [rclone](https://rclone.org) `S3` ব্যবহার করতে পারেন, তারপরে নীচের স্ক্রিপ্টটি দেখুন এবং পরিবর্তন করুন এবং প্রতিবার প্রকাশ করার সময় শুধুমাত্র ক্রমবর্ধমান পরিবর্তনগুলি `S3` করুন৷

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

কনফিগার করতে মনে রাখবেন `CDN` যাতে `/.v` এ শেষ হওয়া পাথের ক্যাশে সময় `1s` হয়, অন্যথায় আপনি অবিলম্বে নতুন প্রকাশিত সামগ্রী অ্যাক্সেস করতে পারবেন না।

### ওয়েবসাইট প্রকাশ করুন

ওয়েবসাইটটি যে কোনো জায়গায় স্থাপন করা যেতে পারে [github page](https://pages.github.com) এবং [cloudflare page](https://pages.cloudflare.com) ভালো পছন্দ।

যেহেতু ওয়েবসাইটটি [একটি একক-পৃষ্ঠার অ্যাপ্লিকেশনের](https://developer.mozilla.org/docs/Glossary/SPA) আর্কিটেকচার গ্রহণ করে, মনে রাখবেন যে URL পাথটি `index.html` এ নেই `. `

ওয়েবসাইট এন্ট্রি পৃষ্ঠাটি শুধুমাত্র একবার স্থাপন করা প্রয়োজন, এবং পরবর্তী বিষয়বস্তু আপডেটের জন্য ওয়েবসাইট এন্ট্রি পৃষ্ঠাটি পুনরায় স্থাপন করার প্রয়োজন নেই।

উপরের ছবির নিয়মগুলি নীচের প্রথম লাইনে আপনার সাথে আবদ্ধ করুন `i18n.site`

```
(http.host in {"i18n.site"}) and not (
ends_with(http.request.uri.path,".html") or
ends_with(http.request.uri.path,".htm") or
ends_with(http.request.uri.path,".ico") or
ends_with(http.request.uri.path,".js") or
ends_with(http.request.uri.path,".avif") or
ends_with(http.request.uri.path,".css") or
ends_with(http.request.uri.path,".json") or
ends_with(http.request.uri.path,".png") or
ends_with(http.request.uri.path,".svg") or
ends_with(http.request.uri.path,".txt") or
ends_with(http.request.uri.path,".webmanifest") or
ends_with(http.request.uri.path,".xml")
)
```

#### মূল ভূখণ্ড চীনে ওয়েবসাইট স্থাপনা অপ্টিমাইজ করা

আপনি যদি চীনের মূল ভূখন্ডের নেটওয়ার্ক পরিবেশে আরও ভাল অ্যাক্সেসিবিলিটি পারফরম্যান্স পেতে চান, অনুগ্রহ করে প্রথমে [একটি ডোমেন নাম নিবন্ধন করুন](//beian.aliyun.com) ৷

তারপর, `out/ol/htm` ভূখণ্ড চীনে ক্লাউড বিক্রেতাদের অবজেক্ট স্টোরেজ ব্যবহার `CDN` +

আপনি এজ কম্পিউটিং ব্যবহার করে একক-পৃষ্ঠার অ্যাপ্লিকেশানগুলির সাথে খাপ খাইয়ে নিতে পাথটি পুনর্লিখন করতে পারেন, যেমন [Baidu স্মার্ট `CDN`](//cloud.baidu.com/product/cdn.html) এটি এইভাবে কনফিগার করা যেতে পারে:

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```

`MX` রেকর্ড এবং `CNAME` রেকর্ড একসাথে থাকতে পারে না, যদি আপনি একই সময়ে ডোমেন নাম ইমেল পেতে চান, তাহলে আপনাকে `CNAME` `A` রেকর্ডে সমতল করতে স্ক্রিপ্টের সাথে সহযোগিতা করতে হবে [cname_flatten](https://github.com/i18n-site/lib/tree/main/cname_flatten)

উপরন্তু, মূল ভূখণ্ডের চীনে ক্লাউড বিক্রেতাদের বিদেশী ট্রাফিক চার্জ তুলনামূলকভাবে ব্যয়বহুল, আপনি যদি খরচ অপ্টিমাইজ করতে চান, তাহলে আপনি [Huawei DNS বিনামূল্যের ভৌগলিক রেজোলিউশন](https://support.huaweicloud.com/usermanual-dns/dns_usermanual_0041.html) এবং [Cloudflare for SaaS](https://developers.cloudflare.com/cloudflare-for-platforms/cloudflare-for-saas) এর কাস্টম ডোমেইন নাম (নিচে দেখানো হয়েছে) ব্যবহার করতে পারেন ট্রাফিক ডাইভারশন──প্রধান ভূখন্ডে ট্রাফিক চীন Baidu ক্লাউড `CDN` আন্তর্জাতিক ট্রাফিক cloudflare যায়।

#### জেনেরিক ডোমেইন নাম পুনর্নির্দেশ

আপনি যদি আপনার প্রধান ওয়েবসাইট হিসাবে একটি ওয়েবসাইট তৈরি করতে ব্যবহার `i18n.site` তবে আপনাকে সাধারণত প্যান-ডোমেন পুনঃনির্দেশ কনফিগার করতে হবে, অর্থাৎ, ( `www.xxx.com` সহ) অ্যাক্সেস করতে `*.xxx.com` `xxx.com`

আলিবাবা ক্লাউডের `EdgeScript` ( [ইংরেজি ডকুমেন্ট](https://www.alibabacloud.com/help/en/cdn/developer-reference/how-edgescript-works) / [চাইনিজ ডকুমেন্ট](https://help.aliyun.com/zh/cdn/developer-reference/edgescript) ) এর সাহায্যে এই প্রয়োজনীয়তা অর্জন করা যেতে পারে `CDN`

[আলিবাবা CDN](https://cdn.console.aliyun.com/domain/list) একটি ডোমেন নাম যোগ করুন এবং ডোমেন `*.xxx.com` আলিবাবা ক্লাউড `CDN` এর `CNAME` এ নির্দেশ করুন।

#### nginx দিয়ে স্থাপন করুন

`server` অনুরূপ একটি কনফিগারেশন যোগ করুন nginx যেখানে `/root/i18n/md/out/ol/htm` অনুগ্রহ করে এটিকে আপনার নিজের প্রকল্পের পথে পরিবর্তন করুন `out/ol/htm`

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```

### ডিরেক্টরি কাঠামো

#### সর্বজনীন

ওয়েবসাইটের স্ট্যাটিক ফাইল, যেমন `favicon.ico` `robots.txt` , ইত্যাদি।

এখানে আইকন ফাইল তৈরি করা যেতে পারে [realfavicongenerator.net](https://realfavicongenerator.net)

#### .i18n

ডাইরেক্টরিটির অধীনে `i18n.site` এর কনফিগারেশন ফাইল, অনুবাদ ক্যাশে ইত্যাদি রয়েছে `.i18n` বিস্তারিত জানার জন্য পরবর্তী অধ্যায় ["কনফিগারেশন"](/i18n.site/conf) দেখুন।

#### en

উৎস ভাষা ডিরেক্টরি, কনফিগারেশন ফাইলে `fromTo` `en` `.i18n/conf.yml` সম্পর্কিত

```yaml
i18n:
  fromTo:
    en: zh
```

অনুগ্রহ করে অনুবাদের কনফিগারেশন পড়ুন [i18](/i18/use)
