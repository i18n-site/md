# সার্ভার আরম্ভ করুন (BTRFS পার্টিশন ব্যবহার করুন, স্ন্যাপার নির্ধারিত স্ন্যাপশট)

## কনটাবো ক্রয় সার্ভার

contabo হল জার্মানির একটি সস্তা VPS প্রদানকারী যার একটি দীর্ঘ ইতিহাস (20 বছরেরও বেশি) এবং অনেক ব্যবহারকারী (200,000 এর বেশি), তাই পালিয়ে যাওয়ার বিষয়ে চিন্তা করার দরকার নেই৷

কেনার VPS [contabo.com](//contabo.com) ইউরো মূল্য চয়ন করতে ভুলবেন না, যা সস্তা।

![](https://i-01.eu.org/1718285782.avif)

কনটাবোর জন্য এককালীন ইনস্টলেশন ফি আছে।
যাইহোক, প্রতি মাসে কিছু দিন আছে যেখানে ইনস্টলেশন ফি বিনামূল্যে শুধুমাত্র নিবন্ধন করুন এবং প্রচারমূলক ইমেলগুলিতে মনোযোগ দিন।

ইউরোপে ফ্রি এরিয়া ফি ছাড়াও অন্যান্য অঞ্চলে দাম বৃদ্ধির বিভিন্ন স্তর রয়েছে।

আমি একটি ইউরোপীয় সার্ভার বেছে নিয়েছি, ক্লাউডফ্লেয়ার ব্যবহার করার পরে, সার্ভারের মধ্যে খুব বেশি পার্থক্য নেই।

চূড়ান্ত কনফিগারেশন নিম্নরূপ, 16GB মেমরি, 6CPU, 200 GB NVMe, মাসে 9.50 ইউরো, 74.43 RMB এর সমতুল্য।

![](https://i-01.eu.org/1718286008.avif)

**এই দাম আলিবাবা ক্লাউড এবং অ্যামাজনের দামের প্রায় দশমাংশ** ।

![](https://i-01.eu.org/1718356956.avif)

আমি কর্মক্ষমতা এবং পরামিতি চালানোর জন্য ব্যবহার [Yet-Another-Bench-Script](https://github.com/masonr/yet-another-bench-script) :

```
curl -sL yabs.sh | bash -s -- -s https://www.vpsbenchmarks.com/yabs/upload
```

মডেল `AMD EPYC 9224 24-Core Processor` দেখানো CPU , একক-কোর পারফরম্যান্স স্কোর 1285
, মাল্টি-কোর স্কোর 4277 আরও ফলাফল দেখুন [vpsbenchmarks.com](https://www.vpsbenchmarks.com/yabs/contabo-6c-16gb-3fb7d8)

আমি একই সময়ে তিনটি সার্ভার কিনেছি এবং সেগুলিকে একই সময়ে সক্রিয় করেছি এটি নিশ্চিত করে যে সার্ভারগুলি একই কম্পিউটার রুমে রয়েছে, যাতে ভবিষ্যতে উচ্চ উপলব্ধতা তৈরি করা সহজ হয়৷

এর সার্ভারগুলি মাঝে মাঝে হ্যাং আপ হয়, উচ্চ প্রাপ্যতার মাধ্যমে ওয়েবসাইট অ্যাক্সেসযোগ্যতা নিশ্চিত করে।

নতুন ব্যবহারকারীদের ক্রয় করার সময় আসল-নাম প্রমাণীকরণের জন্য তাদের আইডি কার্ড জমা দিতে হবে।

## পাসওয়ার্ড-মুক্ত লগইন কনফিগার করুন

কয়েক মিনিট পরে, সার্ভারগুলি সমস্ত সক্রিয় করা হয়েছিল।

প্রথমে একটি উপনাম সেট করতে স্থানীয় মেশিনের `~/.ssh/config` সম্পাদনা IP এবং তারপর পাসওয়ার্ড-মুক্ত লগইন কনফিগার করতে `ssh-copy-id root@c2` ব্যবহার করা।

![](https://i-01.eu.org/1718287198.avif)

আরম্ভ করার কনফিগারেশন দ্রুত সম্পন্ন করতে আপনি এই [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) পরিবর্তন করতে পারেন।

পাসওয়ার্ড-মুক্ত কনফিগারেশন সম্পন্ন হওয়ার পরে, আপনি এটি ব্যবহার করতে পারেন এটি ব্যাচগুলিতে পরিচালনা করা যেতে পারে, যা সুবিধাজনক [pdsh](https://github.com/chaos/pdsh)

প্রথমটি ম্যানুয়ালি ipv6 এবং পরীক্ষা সক্ষম করা

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

![](https://i-01.eu.org/1718287969.avif)

তারপর এক ক্লিকে সিস্টেমটিকে উবুন্টু 24-এ আপগ্রেড করুন এবং সাধারণত ব্যবহৃত সফ্টওয়্যার ইনস্টল করুন।

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 মানে ভাষাকে চাইনিজ এবং টাইম জোনকে বেইজিং-এ পরিবর্তন করা।

ডিফল্টভাবে ইনস্টল করা সফ্টওয়্যারটি আমার ব্যক্তিগত ব্যবহারের অভ্যাস অনুযায়ী কনফিগার করা হয়েছে, যেমন [mise](https://github.com/jdx/mise) , [direnv](https://direnv.net/) , ইত্যাদি। প্রয়োজনে আপনি [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) গুদামটি অনুলিপি করতে পারেন এবং প্রয়োজন অনুসারে এটি পরিবর্তন করতে পারেন।

এই পদক্ষেপটি দীর্ঘ সময় নেয় আধা ঘন্টা অপেক্ষা করার পরে, সাধারণত ব্যবহৃত সফ্টওয়্যারটির আপগ্রেড এবং ইনস্টলেশন সফল হবে৷ আপনি নিম্নলিখিত কমান্ড দিয়ে এটি নিশ্চিত করতে পারেন:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## mosh ssh গতি বাড়ায়

ইউরোপ থেকে চীন পর্যন্ত স্থিতিশীল নয় এটি `ssh` গতি বাড়াতে সুপারিশ করা হয় [mosh](//mosh.org)

স্থানীয় ডিরেক্টরি `~/.bin` এ `c0` স্ক্রিপ্ট তৈরি করুন (প্রধানত এনভায়রনমেন্ট ভেরিয়েবল `PATH` এ যোগ করা হয়েছে) নিম্নরূপ:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

তৈরি করার পরে, এক্সিকিউটেবল অনুমতি এবং সফট-লিঙ্ক `c1` এবং `c2` মঞ্জুর করুন।

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

ভবিষ্যতে, আপনি সরাসরি `c0` এবং `c1` লিখতে পারেন `mosh` উচ্চ-গতির সরাসরি সংযোগ ব্যবহার করতে এবং প্রবেশ করতে পারেন [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

## পার্টিশন ফরম্যাটকে ext4 থেকে btrfs এ রূপান্তর করুন

সহজ ব্যাকআপের জন্য ডিস্ক স্ন্যাপশট নিতে [btrfs](https://wikipedia.org/wiki/Btrfs) .

প্রথমে রেসকিউ সিস্টেমে প্রবেশ করুন

![](https://i-01.eu.org/1718300448.avif)

ডেবিয়ান রেসকিউ বেছে নিন

![](https://i-01.eu.org/1718300753.avif)

সার্ভার পুনরায় চালু হওয়ার জন্য অপেক্ষা করুন

ssh পরিবর্তন হবে, এটি পরিষ্কার করতে `ssh-keygen -R ip` ব্যবহার করুন

তারপর পার্টিশনটি রূপান্তর করতে নিম্নলিখিত স্ক্রিপ্টটি চালান

```bash
nohup bash -c "bash <(curl -s https://raw.githubusercontent.com/i18n-ops/ops/main/contabo/ext4_btrfs.sh) >/tmp/nohup.out 2>&1 && reboot" &
sleep 1 && tail -f /tmp/nohup.out
```

রূপান্তর সফল হওয়ার পরে স্ক্রিপ্টটি স্বয়ংক্রিয়ভাবে সিস্টেমটি পুনরায় চালু করবে।

### btrfs কম্প্রেশন অনুপাত দেখুন

এটি স্বয়ংক্রিয়ভাবে ডিফল্টভাবে সংকুচিত হবে কম্প্রেশন অনুপাত পরীক্ষা করার জন্য নিম্নরূপ:

```bash
if ! command -v compsize &>/dev/null; then
apt install -y  btrfs-compsize
fi
compsize -x /
```

### ম্যানুয়াল কম্প্রেশন

আপনি যদি একজন পারফেকশনিস্ট হন তবে আপনি ম্যানুয়ালি একটি সম্পূর্ণ ডিস্ক কম্প্রেশন করতে পারেন:

```bash
btrfs filesystem defragment -r -czstd /
```

নীচের চিত্রে যেমন দেখানো হয়েছে, আপনি দেখতে পাচ্ছেন যে নতুন সিস্টেমে ম্যানুয়াল কম্প্রেশনের পরে, প্রায় 100MB বেশি সংকুচিত হয়েছিল।

![](https://i-01.eu.org/2023/12/8NWlr4c.webp)

### স্ন্যাপারের সাথে একটি Btrfs স্ন্যাপশট ব্যাকআপ তৈরি করুন

Btrfs ব্যাকআপের জন্য সেরা অংশীদার [snapper](https://manual.siduction.org/sys-admin-btrfs-snapper_en.html)

আরম্ভ করার স্ক্রিপ্টটি `~/ubuntu` ডিরেক্টরি তৈরি করবে, শুধুমাত্র নিম্নলিখিত কমান্ডটি চালান:

```
~/ubuntu/snapper/init.sh
```

স্ক্রিপ্টের বিষয়বস্তুতে দেখা হবে [github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh](https://github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh)

সাবভলিউম দেখতে `snapper list-configs` ব্যবহার করুন যার জন্য স্ন্যাপশট সেট করা হয়েছে।

![](https://i-01.eu.org/1718302689.avif)

একটি সাবভলিউমের একটি স্ন্যাপশট দেখতে `snapper -c mnt list` ব্যবহার করুন

`/etc/snapper/configs/mnt` এই সাবভলিউমের নির্দিষ্ট ব্যাকআপ সেটিংস দেখতে পারে

## উদ্ধার ব্যবস্থার বিশদ বিবরণ

রেসকিউ সিস্টেম লিখুন 22 পোর্টটি অ্যাক্সেসযোগ্য [tcping](https://github.com/paradiseduo/tcping) পরে, আপনি 2-3 মিনিটের জন্য এটিকে অ্যাক্সেস করতে পারবেন না এবং এটি ঠিক হয়ে যাবে।

```
tcping ip 22 -c 9999
```

পোর্ট অ্যাক্সেসিবিলিটি পর্যবেক্ষণ করা যেতে পারে।

### রেসকিউ রিস্টোর করার পর হার্ড ড্রাইভ অ্যাক্সেস করবেন &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

আপনি স্ন্যাপশট দেখতে পারেন, উদাহরণস্বরূপ

```
snapper -c etc list
```

স্ন্যাপশট ফাইল পরিবর্তন দেখুন

```
snapper -c etc status 5..6
```

নির্দিষ্ট পার্থক্য পাঠ্য দেখুন

```
snapper -c etc diff 3..5
```

স্ন্যাপশট পুনরুদ্ধার করুন: উদাহরণস্বরূপ, স্ন্যাপশট 5 এর পরে সমস্ত পরিবর্তন পূর্বাবস্থায় ফেরান৷

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### লগ অধ্যবসায় সক্ষম করুন

একটি রেসকিউ সিস্টেমের অধীনে লগ পারসিস্টেন্স সক্রিয় করার জন্য আপনাকে ম্যানুয়ালি সংশ্লিষ্ট ডিরেক্টরি তৈরি করতে হবে এবং কিছু অতিরিক্ত পদক্ষেপ করতে হবে, কারণ আপনার সিস্টেমড পরিষেবা চালু নাও থাকতে পারে। এখানে আপনি চেষ্টা করতে পারেন পদক্ষেপ আছে:

আগে chroot, তারপর

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

আপনার উবুন্টু সিস্টেমে রিবুট করুন: উপরের পরিবর্তনগুলি করার পরে, আপনাকে রেসকিউ সিস্টেমের পরিবর্তে আপনার মেশিনটি স্বাভাবিকভাবে এবং আপনার উবুন্টু সিস্টেমে পুনরায় বুট করতে হবে।

এইভাবে, systemd-journald নতুন কনফিগারেশনের সাথে শুরু হবে এবং হার্ড ডিস্কে ক্রমাগত লগ সংরক্ষণ করা শুরু করবে।

মনে রাখবেন যে উপরের ক্রিয়াকলাপগুলি একটি রেসকিউ সিস্টেমে সঞ্চালিত হয়, তাই এই পরিবর্তনগুলি কার্যকর হবে না যতক্ষণ না সিস্টেম রিবুট করা হয় এবং আর রেসকিউ মোডে না থাকে।

এর মানে হল যে আপনি অবিলম্বে বিদ্যমান লগগুলিতে কোনও পরিবর্তন দেখতে পাবেন না যা স্থায়ী হয়ে যায়, তবে এর পরে তৈরি হওয়া সমস্ত লগগুলি বজায় থাকবে৷

এইভাবে আপনি ssh পরিষেবার লগগুলি দেখতে পারেন

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## কঠিন এবং জটিল রোগের নোট

### অদ্ভুত সমস্যা সমাধান করুন: সার্ভার ssh করতে পারে না

আমি আগে একটি স্বয়ংক্রিয় সংকলন এবং স্থাপনার স্ক্রিপ্ট তৈরি করতে গিথুব অ্যাকশন ব্যবহার করেছি, কিন্তু দেখা গেছে যে স্থাপনার পরিষেবাটি ssh হতে পারে না।

রেসকিউ সিস্টেমে প্রবেশ করা এবং লগগুলি দেখা একটি অনুমতি সমস্যা:

`Missing privilege separation directory: /run/sshd`

আপনি নিম্নলিখিত কমান্ড দিয়ে এই ডিরেক্টরি তৈরি করতে পারেন

```
chown root:root /
systemd-tmpfiles --create
```

বারবার তদন্তের পরে, github action tar ব্যবহারকারী `uid` হল `1001`

প্যারামিটার `--no-same-owner` না থাকলে আনপ্যাকিং হতে tar , এটি `1001` ব্যবহারকারীর জন্য `uid` এ ডিকম্প্রেস করা হবে

ফলস্বরূপ, rsync রুট ডিরেক্টরির ব্যবহারকারী গোষ্ঠী পরিবর্তন করবে, এবং তারপরে আপনি পুনরায় চালু করার পরে ssh-এ লগ ইন করতে পারবেন না।

নিম্নরূপ ডিকম্প্রেশন স্ক্রিপ্ট পরিবর্তন করুন:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
