### অনুবাদ টোকেন কনফিগার করুন

টোকেন কপি করতে ক্লিক করুন [i18n.site/token](//i18n.site/token)

তৈরি করুন `~/.config/i18n.site.yml` কপি করা সামগ্রী এতে পেস্ট করুন, সামগ্রীটি নিম্নরূপ:


## ব্যবহার

### ডেমো প্রকল্প

`i18` কনফিগারেশন শিখতে ডেমো প্রকল্প দেখুন [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

চীনে ব্যবহারকারীরা ক্লোন করতে পারেন [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

ক্লোনিং করার পরে, ডাইরেক্টরিটি লিখুন এবং অনুবাদটি সম্পূর্ণ করতে চালান `i18`

### ডিরেক্টরি কাঠামো

টেমপ্লেট গুদাম ডিরেক্টরি গঠন নিম্নরূপ

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` অনূদিত ডেমো ফাইল রয়েছে, যেগুলো শুধুমাত্র একটি উদাহরণ এবং মুছে ফেলা যেতে পারে।

### অনুবাদ চালান

ডিরেক্টরি লিখুন এবং অনুবাদ করতে `i18`

### সংগ্রহস্থলে ফাইল যোগ করুন

অনুবাদ ছাড়াও, প্রোগ্রামটি নিম্নলিখিত ফাইলগুলিও তৈরি করবে, অনুগ্রহ করে সেগুলিকে সংগ্রহস্থলে যুক্ত করুন৷

```
.i18n/hash
.i18n/cache/.gitignore
```

তাদের মধ্যে : বিষয়বস্তু নিম্নরূপ `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

এর মানে হল `.i18n/cache/` সমস্ত ফাইল উপেক্ষা করুন ( `.i18n/cache/.gitignore` বাদে)।

আপনার সংস্করণ নিয়ন্ত্রণ সফ্টওয়্যার না হলে `git` এই কনফিগারেশন অনুযায়ী এটি উপেক্ষা করুন!

## কনফিগারেশন ফাইল

`.i18n/conf.yml` `i18` লাইন অনুবাদ টুলের কনফিগারেশন ফাইল

বিষয়বস্তু নিম্নরূপ:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### উৎস ভাষা &

কনফিগারেশন ফাইলে, এর অধীনস্থরা `fromTo`

উৎস ভাষা, `zh ja ko de fr` হল অনুবাদের লক্ষ্য ভাষা `en`

ভাষা কোড দেখুন [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

উদাহরণস্বরূপ, আপনি যদি চীনাকে ইংরেজিতে অনুবাদ করতে চান তবে এই লাইনটি পুনরায় লিখুন `zh: en`

আপনি যদি সমস্ত সমর্থিত ভাষায় অনুবাদ করতে চান, তাহলে অনুগ্রহ করে এর পরে খালি রাখুন `:` উদাহরণ স্বরূপ

```
i18n:
  fromTo:
    en:
```

### ফাইল উপেক্ষা করুন

ডিফল্টরূপে, সোর্স ল্যাঙ্গুয়েজ ডিরেক্টরিতে `.md` এবং `.yml` দিয়ে শুরু হওয়া সমস্ত ফাইল অনুবাদ করা হবে।

আপনি যদি নির্দিষ্ট ফাইলগুলিকে উপেক্ষা করতে চান এবং তাদের অনুবাদ না করতে চান (যেমন অসমাপ্ত খসড়া), আপনি ক্ষেত্র কনফিগারেশন ব্যবহার করতে পারেন `ignore`

`.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax) মত একটি সিনট্যাক্স ব্যবহার করে `ignore`

উদাহরণস্বরূপ, উপরের কনফিগারেশন ফাইলের মানে `_* ` যে `_` দিয়ে শুরু হওয়া ফাইলগুলি অনুবাদ করা হবে না।

## অনুবাদের নিয়ম

### অনুবাদ সম্পাদকদের লাইন যোগ বা মুছে দেওয়া উচিত নয়

অনুবাদ সম্পাদনাযোগ্য। মূল পাঠ্যটি পরিবর্তন করুন এবং মেশিন-এটি আবার অনুবাদ করুন, অনুবাদের ম্যানুয়াল পরিবর্তনগুলি ওভাররাইট করা হবে না (যদি মূল পাঠের এই অনুচ্ছেদটি সংশোধন করা না হয়)।

কিন্তু অনুগ্রহ করে মনে রাখবেন যে অনুবাদের লাইন এবং মূল পাঠ্য একের সাথে এক হতে হবে। অর্থাৎ অনুবাদ কম্পাইল করার সময় লাইন যোগ বা মুছে ফেলবেন না। অন্যথায়, এটি অনুবাদ সম্পাদনা ক্যাশে বিভ্রান্তি সৃষ্টি করবে।

কিছু ভুল হলে, [সমাধানের জন্য FAQ পড়ুন।](/i18/qa#H1)

### `YAML` অনুবাদ করা

কমান্ড লাইন টুলটি সোর্স ল্যাঙ্গুয়েজ ফাইল ডিরেক্টরিতে শেষ `.yml` সমস্ত ফাইল খুঁজে পাবে এবং সেগুলিকে অনুবাদ করবে।

* মনে রাখবেন যে ফাইলের নামের প্রত্যয়টি অবশ্যই হবে ( `.yaml` না) `.yml`

টুলটি শুধুমাত্র অভিধানের মানগুলিকে অনুবাদ করে `.yml` অভিধান কীগুলি নয়।

উদাহরণ স্বরূপ `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

হিসাবে অনুবাদ করা হবে `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` অনুবাদটি ম্যানুয়ালিও পরিবর্তন করা যেতে পারে (তবে অনুবাদে কী বা লাইনগুলি যোগ করবেন না)।

`YAML` উপর ভিত্তি করে, আপনি সহজেই বিভিন্ন প্রোগ্রামিং ভাষার জন্য আন্তর্জাতিক সমাধান তৈরি করতে পারেন।

## উন্নত ব্যবহার

### অনুবাদ সাবডিরেক্টরি

যতক্ষণ আপনি তৈরি `.i18n/conf.yml` (প্রতিবার ডেমো প্রকল্প টেমপ্লেট থেকে শুরু করার দরকার নেই), `i18` ঠিক কাজ করবে।

কমান্ড লাইন টুলটি সমস্ত সাবডিরেক্টরিতে `.i18n/conf.yml` খুঁজে পাবে এবং এটি অনুবাদ করবে।

যে প্রকল্পগুলি আর্কিটেকচার ব্যবহার করে [monorepo](//monorepo.tools) ভাষা ফাইলগুলিকে সাবডিরেক্টরিতে বিভক্ত করতে পারে।

### কাস্টম ইনস্টলেশন ডিরেক্টরি

এটি ডিফল্টরূপে ইনস্টল করা হবে `/usr/local/bin`

লেখার অনুমতি না `/usr/local/bin` এটি `~/.bin` এ ইনস্টল করা হবে।

এনভায়রনমেন্ট ভেরিয়েবল সেট `TO` আপনি ইন্সটলেশন ডিরেক্টরি নির্ধারণ করতে পারেন, উদাহরণস্বরূপ :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
