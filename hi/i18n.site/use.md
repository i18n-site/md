# स्थापित करें &

## कॉन्फ़िगरेशन टोकन

`i18n.site` `i18` अनुवाद उपकरण एम्बेडेड है, [एक्सेस टोकन को कॉन्फ़िगर करने के लिए `i18` दस्तावेज़ को देखने के लिए कृपया यहां क्लिक करें](/i18/use) ।

## स्थापित करना

```sh
bash <(curl -sS https://i.i18n.site) i18n.site
```

## डेमो प्रोजेक्ट

आइए एक डेमो प्रोजेक्ट से शुरुआत करें और सीखें कि इसका उपयोग कैसे करें `i18n.site`

हम पहले डेमो रिपॉजिटरी को क्लोन करते हैं और निम्नानुसार कमांड चलाते हैं:

```
git clone https://github.com/i18n-site/demo.i18n.site.git md
git clone https://github.com/i18n-site/demo.i18n.site.docker.git docker
```

मुख्य भूमि चीन में उपयोगकर्ता ये कर सकते हैं:

```
git clone https://atomgit.com/i18n/demo.i18n.site.git md
git clone https://atomgit.com/i18n/demo.i18n.site.docker.git docker
```

`docker` के साथ स्थानीय पूर्वावलोकन की सुविधा के लिए कोड बेस क्लोन का निर्देशिका नाम `md` होना चाहिए `demo.i18n.site`

### पोस्ट सामग्री

[एकल-पृष्ठ एप्लिकेशन](https://developer.mozilla.org/docs/Glossary/SPA) आर्किटेक्चर को अपनाता है, और वेबसाइट प्रवेश पृष्ठ और वेबसाइट सामग्री स्वतंत्र रूप से तैनात की जाती है `i18n.site`

उपरोक्त अनुवाद चलाने के बाद, `md/out/dev` निर्देशिका में `htm` और `v` निर्देशिकाएँ उत्पन्न हो जाएंगी।

यहाँ, `dev` इंगित करता है कि यह `.i18n/htm/dev.yml` फ़ाइल के आधार पर बनाया गया है।

`dev` सामग्री के अंतर्गत :

`htm` निर्देशिका के अंतर्गत वेबसाइट प्रवेश पृष्ठ है।

`v` निर्देशिका में संस्करण संख्या के साथ वेबसाइट सामग्री शामिल है।

स्थानीय पूर्वावलोकन संस्करण संख्या की परवाह किए बिना सभी फ़ाइलों को `out/dev/v/0.1.0` निर्देशिका में कॉपी कर देगा।

आधिकारिक रिलीज़ के लिए, परिवर्तित फ़ाइलों को नई संस्करण संख्या निर्देशिका में कॉपी किया जाएगा।

#### कॉन्फ़िगरेशन फ़ाइल निर्दिष्ट करने के लिए -c का उपयोग करें

विभिन्न कॉन्फ़िगरेशन फ़ाइलें `out` निर्देशिका के अंतर्गत संबंधित निर्देशिकाएँ बनाएंगी।

उदाहरण के लिए, `.i18n/htm/ol.yml` एक `out/ol` निर्देशिका बनाएगा.

`dev.yml` और `ol.yml` डिफ़ॉल्ट कॉन्फ़िगरेशन हैं.

`dev` `development` का संक्षिप्त नाम है, जो विकास परिवेश का प्रतिनिधित्व करता है, स्थानीय पूर्वावलोकन के लिए उपयोग किया जाता है, और यह डिफ़ॉल्ट कॉन्फ़िगरेशन फ़ाइल भी है।
`ol` `online` का संक्षिप्त नाम है, जो ऑनलाइन वातावरण का प्रतिनिधित्व करता है, आधिकारिक रिलीज के लिए उपयोग किया जाता है, और `npm` कमांड लाइन पैरामीटर `-n` का उपयोग करके) प्रकाशित करते समय डिफ़ॉल्ट कॉन्फ़िगरेशन फ़ाइल भी है।

आप कॉन्फ़िगरेशन फ़ाइल नाम निर्दिष्ट करने के लिए कमांड लाइन पर अन्य कॉन्फ़िगरेशन फ़ाइलें भी बना सकते हैं `--htm_conf`

उदाहरण के लिए:
```
i18n.site --htm_conf yourConfig --save
```

यहाँ `--save` अद्यतन रिलीज़ संस्करण संख्या दर्शाता है।

#### <a rel=id href="#npm" id="npm"></a> npmjs.com पर सामग्री प्रकाशित करें

[npmjs.com](//npmjs.com) सामग्री प्रकाशित करना अनुशंसित डिफ़ॉल्ट समाधान है ( [फ़्रंट-एंड उच्च उपलब्धता](/i18n.site/feature#ha) देखें)।

##### npm login & मुक्त करना

इंस्टॉल करें `nodejs` लॉग इन करने के लिए `npm login` उपयोग करें।

संपादित करें `md/.i18n/htm/ol.yml` `v: //unpkg.com/i18n.site` में `i18n.site` अपने `npm` पैकेज नाम) में बदलें।

बस खाली पैकेज नाम का उपयोग करें [npmjs.com](//npmjs.com) पैकेज नाम के रूप में वेबसाइट डोमेन नाम का उपयोग करना एक अच्छा विकल्प है।

`npm` पैकेज के आधार पर प्रकाशित करते समय, `v:` मान के उपसर्ग के रूप में **`//unpkg.com/` का उपयोग करना सुनिश्चित करें** । नई रिलीज़ को समय पर देखने में सक्षम करने के लिए कैश समय `i18n.site` इस उपसर्ग पथ के तहत `/.v` को विशेष रूप से अनुकूलित किया गया है।

अनुवाद और प्रकाशन के लिए `md` निर्देशिका में `i18n.site --npm` या `i18n.site -n` चलाएँ।

यदि आप प्रकाशित करने के लिए निरंतर एकीकरण वातावरण का उपयोग करते हैं, तो इसे स्थापित करने की कोई आवश्यकता नहीं है `nodejs` बस लॉग-इन और प्रकाशित अनुमति `~/.npmrc` पर्यावरण में कॉपी करें।

यदि आप पैकेज नाम को `ol.yml` में संशोधित करते हैं, तो कृपया पहले $ `v:` **`.i18n/v/ol` सुनिश्चित करें** और फिर इसे प्रकाशित करें।

##### एनपीएम द्वारा प्रकाशित प्रॉक्सी सर्वर

यदि मुख्य भूमि चीन में उपयोगकर्ता नेटवर्क समस्याओं का सामना करते हैं और `npm` पैकेज प्रकाशित करने में असमर्थ हैं, तो वे प्रॉक्सी सर्वर को कॉन्फ़िगर करने के लिए पर्यावरण चर `https_proxy` सेट कर सकते हैं।

यह मानते हुए कि आपका प्रॉक्सी सर्वर पोर्ट है `7890` आप लिख सकते हैं:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### स्व-होस्ट की गई सामग्री

यदि आप सामग्री को स्वयं-होस्ट करना चाहते हैं, `md/.i18n/htm/ol.yml` पहले, संपादित करें और `v: //unpkg.com/i18n.site` अपने यूआरएल उपसर्ग में बदलें, जैसे कि `v: //i18n-v.xxx.com`

`md` निर्देशिका दर्ज करें और चलाएँ

```
i18n.site --htm_conf ol --save
```

या संक्षिप्तीकरण

```
i18n.site -c ol -s
```

फिर, `md/out/ol/v` निर्देशिका में सामग्री को `v:` में सेट URL उपसर्ग पथ पर कॉन्फ़िगर करें।

अंत में, **`/.v` से समाप्त होने वाले पथ के कैश समय को `1s` पर कॉन्फ़िगर करें** , अन्यथा नई रिलीज़ की गई सामग्री को तुरंत एक्सेस नहीं किया जा सकता है।

अनावश्यक अनुरोधों को कम करने के लिए अन्य पथों के लिए कैश समय को एक वर्ष या उससे अधिक पर सेट किया जा सकता है।

##### सामग्री को s3 पर होस्ट करें

सामग्री को स्व-होस्ट `CDN` के लिए + अपने स्वयं के सर्वर का उपयोग करने के अलावा, एक अन्य सामान्य विकल्प का उपयोग करना है `S3`

आप [rclone](https://rclone.org) लॉगिन `S3` सर्वर का उपयोग कर सकते हैं, फिर नीचे दी गई स्क्रिप्ट को देखें और संशोधित करें, और हर बार प्रकाशित होने पर केवल वृद्धिशील परिवर्तनों `S3` कॉपी करें।

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

कॉन्फ़िगर करना याद रखें `CDN` ताकि `/.v` में समाप्त होने वाले पथ का कैश समय `1s` हो, अन्यथा आप नई जारी सामग्री तक तुरंत पहुंच नहीं पाएंगे।

### वेबसाइट प्रकाशित करें

वेबसाइट को कहीं भी तैनात किया जा सकता है, और [cloudflare page](https://pages.cloudflare.com) अच्छे विकल्प हैं [github page](https://pages.github.com)

चूँकि वेबसाइट [एकल-पृष्ठ एप्लिकेशन](https://developer.mozilla.org/docs/Glossary/SPA) के आर्किटेक्चर को अपनाती है, इसलिए उस URL पथ को फिर से लिखना याद रखें जिसमें `index.html` शामिल नहीं है `. `

वेबसाइट प्रविष्टि पृष्ठ को केवल एक बार तैनात करने की आवश्यकता है, और बाद के सामग्री अपडेट के लिए वेबसाइट प्रविष्टि पृष्ठ को फिर से तैनात करने की कोई आवश्यकता नहीं है।

ends_with(http.request.uri.path,".png") or
ends_with(http.request.uri.path,".svg") or
ends_with(http.request.uri.path,".txt") or
ends_with(http.request.uri.path,".webmanifest") or
ends_with(http.request.uri.path,".xml")

इसके अलावा, कृपया कैश नियमों को कॉन्फ़िगर करें, जैसा कि नीचे दिखाया गया है, और कैश अवधि को एक महीने पर सेट करें।

कृपया ऊपर चित्र में दूसरे चरण में मेल खाने वाले डोमेन नाम को आपके द्वारा बंधे डोमेन नाम से बदलें।

#### मुख्य भूमि चीन में वेबसाइट परिनियोजन का अनुकूलन

यदि आप मुख्य भूमि चीन के नेटवर्क वातावरण में बेहतर पहुंच प्रदर्शन प्राप्त करना चाहते हैं, तो कृपया पहले [एक डोमेन नाम पंजीकृत करें](//beian.aliyun.com) ।

`out/ol/htm` , मुख्य भूमि चीन में क्लाउड विक्रेताओं के ऑब्जेक्ट स्टोरेज `CDN` उपयोग करें +

आप [Baidu स्मार्ट `CDN`](//cloud.baidu.com/product/cdn.html) जैसे सिंगल-पेज एप्लिकेशन के अनुकूल पथ को फिर से लिखने के लिए एज कंप्यूटिंग का उपयोग कर सकते हैं, इसे इस तरह कॉन्फ़िगर किया जा सकता है:

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```
<img alt="" src="https://p.3ti.site/1721121273.avif">

क्योंकि `MX` रिकॉर्ड और `CNAME` रिकॉर्ड एक साथ नहीं रह सकते हैं, यदि आप एक ही समय में डोमेन नाम ईमेल प्राप्त करना चाहते हैं, तो आपको `CNAME` `A` रिकॉर्ड) में फ़्लैट करने के लिए [cname_flatten](https://github.com/i18n-site/lib/tree/main/cname_flatten) स्क्रिप्ट के साथ सहयोग करने की आवश्यकता है।

इसके अलावा, क्योंकि मुख्य भूमि चीन में क्लाउड विक्रेताओं के विदेशी ट्रैफ़िक शुल्क अपेक्षाकृत महंगे हैं, यदि आप लागतों को अनुकूलित करना चाहते हैं, तो आप [Huawei DNS के मुफ़्त भौगोलिक रिज़ॉल्यूशन](https://support.huaweicloud.com/usermanual-dns/dns_usermanual_0041.html) और [Cloudflare for SaaS](https://developers.cloudflare.com/cloudflare-for-platforms/cloudflare-for-saas) के कस्टम डोमेन नाम (जैसा कि नीचे दिखाया गया है) का उपयोग कर सकते हैं ट्रैफ़िक डायवर्जन──मुख्य भूमि चीन Baidu क्लाउड में `CDN` अंतर्राष्ट्रीय ट्रैफ़िक cloudflare हो जाता है।

<img alt="" src="https://p.3ti.site/1721119788.avif">

ये परिनियोजन अनुकूलन समाधान अधिक जटिल हैं और इन्हें भविष्य में अलग-अलग अध्यायों में पेश किया जाएगा।

#### सामान्य डोमेन नाम पुनर्निर्देशन

यदि आप अपनी मुख्य वेबसाइट के रूप में एक वेबसाइट बनाने के लिए `i18n.site` का उपयोग करते हैं, तो आपको आमतौर पर पैन-डोमेन पुनर्निर्देशन को कॉन्फ़िगर करने की आवश्यकता होती है, यानी `*.xxx.com` ( `www.xxx.com` सहित) पहुंच को रीडायरेक्ट करें `xxx.com`

यह आवश्यकता अलीबाबा क्लाउड के `EdgeScript` ( [अंग्रेजी दस्तावेज़](https://www.alibabacloud.com/help/en/cdn/developer-reference/how-edgescript-works) / [चीनी दस्तावेज़](https://help.aliyun.com/zh/cdn/developer-reference/edgescript) ) की सहायता से प्राप्त की जा सकती है `CDN`

[अलीबाबा CDN](https://cdn.console.aliyun.com/domain/list) में एक डोमेन नाम जोड़ें और `*.xxx.com` नाम को अलीबाबा क्लाउड `CDN` के `CNAME` पर इंगित करें।

<img alt="" src="https://p.3ti.site/1721122000.avif">

उदाहरण के लिए, ऊपर चित्र में पैन-डोमेन नाम पुनर्निर्देशन कॉन्फ़िगरेशन इस `*.i18n.site` है:

```
rewrite(concat('https://i18n.site/',$uri), 'redirect',301)
```

#### nginx के साथ परिनियोजन करें

कृपया `server` में निम्नलिखित के समान एक कॉन्फ़िगरेशन जोड़ें nginx जहां `/root/i18n/md/out/ol/htm` कृपया इसे अपने प्रोजेक्ट `out/ol/htm` के पथ में बदलें:

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```

### निर्देशिका संरचना

#### जनता

वेबसाइट की स्थिर फ़ाइलें, जैसे `favicon.ico` `robots.txt` , आदि।

यहां आइकन फ़ाइलें [realfavicongenerator.net](https://realfavicongenerator.net) के साथ उत्पन्न की जा सकती हैं।

#### .i18n

`.i18n` निर्देशिका के अंतर्गत `i18n.site` की कॉन्फ़िगरेशन फ़ाइलें, अनुवाद कैश आदि हैं। विवरण के लिए अगला अध्याय ["कॉन्फ़िगरेशन"](/i18n.site/conf) देखें।

#### एन

स्रोत भाषा निर्देशिका, कॉन्फ़िगरेशन फ़ाइल में `fromTo` `en` `.i18n/conf.yml` अनुरूप

```yaml
i18n:
  fromTo:
    en: zh
```

कृपया अनुवाद की कॉन्फ़िगरेशन देखें [i18](/i18/use)
