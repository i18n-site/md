## Конфигурирајте Најава Без Лозинка

По неколку минути, сите сервери беа активирани.

Првата `ssh-copy-id root@c2` е да IP уредите локалната машина `~/.ssh/config`

<img alt="" src="https://i-01.eu.org/1718287198.avif">

Можете да ја измените оваа [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) за побрзо да ја завршите конфигурацијата за иницијализација.

Откако ќе заврши конфигурацијата без лозинка, можете да ја користите [pdsh](https://github.com/chaos/pdsh) Може да се ракува во серии, што е погодно.

Првата е рачно да се овозможи ipv6 и да се тестира

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

<img alt="" src="https://i-01.eu.org/1718287969.avif">

Потоа надградете го системот на ubuntu 24 со еден клик и инсталирајте го најчесто користениот софтвер.

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 значи префрлување јазик на кинески и временска зона во Пекинг.

Стандардно инсталираниот софтвер е конфигуриран според моите лични навики за користење, како што се [mise](https://github.com/jdx/mise) [direnv](https://direnv.net/) , итн. Доколку е потребно, можете да го копирате складиштето [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) и да го менувате по потреба.

Овој чекор трае долго По чекање половина час, надградбата и инсталирањето на најчесто користениот софтвер ќе биде успешна. Можете да го потврдите со следнава команда:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## Мош Го Забрзува Сш

Мрежата од Европа `ssh` Кина не е стабилна Се препорачува да се користи [mosh](//mosh.org)

Создадете скрипта `c0` во `~/.bin` на локалната машина (главно додадена во променливата на животната средина `PATH` ) на следниов начин:

```bash
#!/usr/bin/env баш

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

По креирањето, доделете извршни дозволи и мека врска `c1` `c2` .

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

Во иднина, можете директно да внесете `c0` `c1` и можете да користите директна врска со голема брзина `mosh` и да внесете [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

Може [btrfs](https://wikipedia.org/wiki/Btrfs) прави снимки од дискот за лесна резервна копија.

<img alt="" src="https://i-01.eu.org/1718300753.avif">

Почекајте серверот да се рестартира

Промените `ssh-keygen -R ip` отпечатокот од прст ќе се појават кога ssh

Потоа извршете ја следната скрипта за да ја конвертирате партицијата

### Погледнете Го Односот На Компресија btrfs

Стандардно автоматски ќе се компресира Командата за проверка на односот на компресија е како што следува:

```bash

```

## Детали За Системот За Спасување

Внесете го системот за спасување 22 Откако пристаништето е [tcping](https://github.com/paradiseduo/tcping) пристапно, нема да можете да пристапите до него со поставената лозинка за 2-3 минути.

```
tcping ip 22 -c 9999
```

Може да се следи пристапноста на пристаништето.

### Како Да Пристапите До Хард Дискот Откако Ќе Влезете Во Спасување &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

Можете да ја видите снимката, на пример

```
snapper -c etc list
```

Прикажи ги промените во датотеката со слики

```
snapper -c etc status 5..6
```

Прикажи специфичен различен текст

```
snapper -c etc diff 3..5
```

Враќање на снимката: на пример, вратете ги сите промени по слика 5

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### Овозможи Упорност На Дневникот

Овозможувањето на упорност на дневникот во системот за спасување бара од вас рачно да го креирате соодветниот директориум и да извршите некои дополнителни чекори, бидејќи можеби немате вклучена systemd услуга. Еве чекори што можете да ги испробате:

Прво chroot, тогаш

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

Рестартирајте се во вашиот Ubuntu систем: Откако ќе ги направите горенаведените промени, треба да ја рестартирате машината нормално и во вашиот Ubuntu систем наместо во системот за спасување.

На овој начин, systemd-journald ќе започне со новата конфигурација и ќе започне постојано да складира дневници на тврдиот диск.

Имајте на ум дека горенаведените операции се вршат на систем за спасување, така што тие промени ќе стапат на сила само штом системот ќе се рестартира и повеќе не е во режим за спасување.

Ова значи дека нема веднаш да видите какви било промени во постојните дневници да станат постојани, но сите дневници генерирани после ова ќе останат.

На овој начин можете да ги прегледате дневниците на услугата ssh

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## Белешки За Тешки И Комплицирани Болести

### Решете Го Чудниот Проблем: Серверот Не Може Да ssh

Претходно користев акција github за да направам скрипта за автоматска компилација и распоредување, но открив дека услугата за распоредување не може да биде ssh.

Влегувањето во системот за спасување и гледањето на дневниците е проблем со дозволата:

`Missing privilege separation directory: /run/sshd`

Можете да го креирате овој директориум со следнава команда

```
chown root:root /
systemd-tmpfiles --create
```

По повторените истраги, тоа е затоа github action корисникот кој го поставил tar е `1001` `uid`

Отпакувањето `1001` `--no-same-owner` `uid` направи без параметри tar

Како резултат на тоа, rsync ќе ја промени корисничката група на root директориумот, а потоа нема да можете да се најавите на ssh по рестартирањето.

Изменете ја скриптата за декомпресија на следниов начин:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
