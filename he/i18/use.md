```sh
bash <(curl -sS https://i.i18n.site) i18
```

## להשתמש

### פרויקט הדגמה

אנא עיין בפרויקט ההדגמה כדי ללמוד את תצורת `i18` [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

משתמשים בסין יכולים לשכפל [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

לאחר השיבוט, היכנס לספרייה `i18` כדי להשלים את התרגום.

### מבנה ספריות

מבנה ספריות המחסן של התבניות הוא כדלקמן

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

ספריית `en` מכילה את קבצי ההדגמה המתורגמים, שהם רק דוגמה וניתן למחוק אותם.

### הפעל תרגום

היכנס לספרייה `i18` כדי לתרגם.

### הוסף קבצים למאגר

בנוסף לתרגום, התוכנה תייצר גם את הקבצים הבאים, אנא הוסף אותם למאגר.

```
.i18n/hash
.i18n/cache/.gitignore
```

ביניהם, התוכן : `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

זה אומר להתעלם מכל הקבצים `.i18n/cache/` (למעט `.i18n/cache/.gitignore` ).

אם תוכנת בקרת הגירסה שלך אינה `git` , אנא התעלם ממנה בהתאם לתצורה זו.

## קובץ תצורה

`.i18n/conf.yml` זהו קובץ התצורה של כלי התרגום `i18` שורת הפקודה

התוכן הוא כדלקמן:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### שפת המקור &

בקובץ התצורה, הכפופים של `fromTo`

`en` היא שפת המקור, `zh ja ko de fr` היא שפת היעד של התרגום.

קוד שפה ראה [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

לדוגמה, אם אתה רוצה לתרגם סינית לאנגלית, כתוב מחדש את השורה הזו `zh: en`

אם אתה רוצה לתרגם לכל השפות הנתמכות, אנא השאר ריק לאחר `:` לדוגמה

```
i18n:
  fromTo:
    en:
```

### להתעלם מהקובץ

כברירת מחדל, כל הקבצים המתחילים `.md` ו- `.yml` בספריית שפת המקור יתורגמו.

אם אתה רוצה להתעלם מקבצים מסוימים ולא לתרגם אותם (כגון טיוטות לא גמורות), אתה יכול להשתמש `ignore` השדה.

`ignore` משתמש בתחביר דומה ל- `.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax)

לדוגמה, בקובץ התצורה לעיל `_* ` שקבצים המתחילים ב- `_` לא יתורגמו.

## כללי תרגום

### עורכי תרגום אינם צריכים להוסיף או למחוק שורות

התרגום ניתן לעריכה. שנה את הטקסט המקורי ותרגם אותו שוב במכונה, השינויים הידניים בתרגום לא יוחלפו (אם פסקה זו של הטקסט המקורי לא שונתה).

אך שימו לב כי שורות התרגום והטקסט המקורי חייבות להתאים אחת לאחד. כלומר, אין להוסיף או למחוק שורות בעת הידור התרגום. אחרת, זה יגרום לבלבול במטמון עריכת התרגום.

אם משהו משתבש, עיין [בשאלות הנפוצות לקבלת פתרונות.](/i18/qa#H1)

### `YAML` לתרגם

כלי שורת הפקודה ימצא את כל הקבצים המסתיימים עם `.yml` בספריית הקבצים של שפת המקור ויתרגם אותם.

* שים לב שסיומת שם הקובץ חייבת `.yml` (לא `.yaml` ).

הכלי מתרגם רק את ערכי המילון `.yml` , לא את מפתחות המילון.

לדוגמה `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

יתורגם כ `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

ניתן לשנות את התרגום של `YAML` גם באופן ידני (אך אין להוסיף או למחוק מפתחות או שורות בתרגום).

בהתבסס על `YAML` , אתה יכול בקלות לבנות פתרונות בינלאומיים לשפות תכנות שונות.

## שימוש מתקדם

### ספריית משנה לתרגום

כל עוד אתה `.i18n/conf.yml` (אין צורך להתחיל מתבנית פרויקט הדגמה בכל פעם), `i18` יעבוד בסדר.

כלי שורת הפקודה ימצא `.i18n/conf.yml` התצורה בכל ספריות המשנה ויתרגם אותה.

פרויקטים המשתמשים בארכיטקטורת [monorepo](//monorepo.tools) יכולים לפצל קבצי שפה לספריות משנה.

<img alt="" src="https://p.3ti.site/1719910016.avif">

### ספריית התקנה מותאמת אישית

זה יותקן כברירת מחדל `/usr/local/bin`

`/usr/local/bin` אין הרשאת כתיבה הוא יותקן ב- `~/.bin` .

הגדר משתני סביבה : יכול להגדיר את ספריית ההתקנה, למשל `TO`

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
