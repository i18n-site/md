# நிறுவி பயன்படுத்தவும்

## விண்டோஸ் முதலில் git bash ஐ நிறுவவும்

windows சிஸ்டம், [முதலில் பதிவிறக்கி நிறுவ இங்கே கிளிக் `git bash`](https://git-scm.com/download/win) .

இல் அடுத்தடுத்த செயல்பாடுகளை இயக்கவும் `git bash`

## நிறுவு

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### மொழிபெயர்ப்பு டோக்கனை உள்ளமைக்கவும்

டோக்கனை நகலெடுக்க பார்வையிடவும் [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

உருவாக்கவும் `~/.config/i18n.site.yml` நகலெடுக்கப்பட்ட உள்ளடக்கத்தை அதில் ஒட்டவும், உள்ளடக்கம் பின்வருமாறு:

```
token: YOUR_API_TOKEN
```

கூடுதலாக, நீங்கள் பணம் செலுத்துவதற்கு ஒரு கிரெடிட் கார்டை பிணைக்க [i18n.site/payBill](//i18n.site/payBill) (ரீசார்ஜ் தேவையில்லை, இணையதளம் தானாகவே பயன்பாட்டிற்கு ஏற்ப கட்டணங்களைக் கழிக்கும், [விலை நிர்ணயத்திற்கான முகப்புப் பக்கத்தைப் பார்க்கவும்](/#price) ).

## பயன்படுத்த

### டெமோ திட்டம்

`i18` உள்ளமைவை அறிய டெமோ திட்டத்தைப் பார்க்கவும் [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

சீனாவில் உள்ள பயனர்கள் குளோன் செய்யலாம் [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

குளோனிங் செய்த பிறகு, கோப்பகத்தை உள்ளிட்டு, மொழிபெயர்ப்பை முடிக்க இயக்கவும் `i18`

### அடைவு அமைப்பு

டெம்ப்ளேட் கிடங்கு அடைவு அமைப்பு பின்வருமாறு

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` கோப்பகத்தில் மொழிபெயர்க்கப்பட்ட டெமோ கோப்புகள் உள்ளன, அவை ஒரு எடுத்துக்காட்டு மற்றும் நீக்கப்படலாம்.

### மொழிபெயர்ப்பை இயக்கவும்

கோப்பகத்தை உள்ளிட்டு மொழிபெயர்க்க இயக்கவும் `i18`

### களஞ்சியத்தில் கோப்புகளைச் சேர்க்கவும்

மொழிபெயர்ப்புடன் கூடுதலாக, நிரல் பின்வரும் கோப்புகளையும் உருவாக்கும், தயவுசெய்து அவற்றை களஞ்சியத்தில் சேர்க்கவும்.

```
.i18n/hash
.i18n/cache/.gitignore
```

அவற்றில் : உள்ளடக்கம் பின்வருமாறு `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

இதன் பொருள் `.i18n/cache/` உள்ள அனைத்து கோப்புகளையும் புறக்கணிக்கவும் ( `.i18n/cache/.gitignore` தவிர).

உங்கள் பதிப்பு கட்டுப்பாட்டு மென்பொருள் இல்லை என்றால் `git` இந்த உள்ளமைவின் படி அதை புறக்கணிக்கவும்.

## கட்டமைப்பு கோப்பு

`.i18n/conf.yml` `i18` வரி மொழிபெயர்ப்பு கருவியின் உள்ளமைவு கோப்பு

உள்ளடக்கம் பின்வருமாறு:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### மூல மொழி &

உள்ளமைவு கோப்பில், `fromTo`

`en` மூல மொழி, `zh ja ko de fr` மொழிபெயர்ப்பின் இலக்கு மொழி.

மொழி குறியீடு பார்க்கவும் [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

எடுத்துக்காட்டாக, நீங்கள் சீன மொழியை ஆங்கிலத்தில் மொழிபெயர்க்க விரும்பினால், இந்த வரியை மீண்டும் எழுதவும் `zh: en`

ஆதரிக்கப்படும் அனைத்து மொழிகளுக்கும் நீங்கள் மொழிபெயர்க்க விரும்பினால், பின்னர் காலியாக விடவும் `:` உதாரணத்திற்கு

```
i18n:
  fromTo:
    en:
```

### கோப்பை புறக்கணிக்கவும்

முன்னிருப்பாக, மூல மொழி கோப்பகத்தில் `.md` மற்றும் `.yml` இல் தொடங்கும் அனைத்து கோப்புகளும் மொழிபெயர்க்கப்படும்.

சில கோப்புகளை நீங்கள் புறக்கணித்து அவற்றை மொழிபெயர்க்காமல் இருக்க விரும்பினால் (முடிக்கப்படாத வரைவுகள் போன்றவை), நீங்கள் `ignore` உள்ளமைவைப் பயன்படுத்தலாம்.

`.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax) தொடரியல் பயன்படுத்துகிறது `ignore`

எடுத்துக்காட்டாக, மேலே உள்ள உள்ளமைவு கோப்பில் `_* ` `_` தொடங்கும் கோப்புகள் மொழிபெயர்க்கப்படாது.

## மொழிபெயர்ப்பு விதிகள்

### மொழிபெயர்ப்பு ஆசிரியர்கள் வரிகளைச் சேர்க்கவோ நீக்கவோ கூடாது

மொழிபெயர்ப்பு திருத்தக்கூடியது. அசல் உரையை மாற்றியமைத்து, அதை மீண்டும் இயந்திரத்தில் மொழிபெயர்த்தால், மொழிபெயர்ப்பில் கைமுறை மாற்றங்கள் மேலெழுதப்படாது (அசல் உரையின் இந்தப் பத்தி மாற்றப்படாவிட்டால்).

ஆனால் மொழிபெயர்ப்பின் வரிகளும் அசல் உரையும் ஒன்றோடு ஒன்று ஒத்திருக்க வேண்டும் என்பதை நினைவில் கொள்ளவும். அதாவது, மொழிபெயர்ப்பைத் தொகுக்கும்போது வரிகளைச் சேர்க்கவோ நீக்கவோ கூடாது. இல்லையெனில், அது மொழிபெயர்ப்பு எடிட்டிங் தற்காலிக சேமிப்பில் குழப்பத்தை ஏற்படுத்தும்.

ஏதேனும் தவறு நடந்தால், [தீர்வுகளுக்கு அடிக்கடி கேட்கப்படும் கேள்விகளைப் பார்க்கவும்.](/i18/qa#H1)

### `YAML` மொழிபெயர்

கட்டளை வரி கருவியானது மூல மொழி கோப்பு கோப்பகத்தில் உள்ள அனைத்து கோப்புகளையும் `.yml` அவற்றை மொழிபெயர்க்கும்.

* கோப்பு பெயர் பின்னொட்டு ( `.yaml` அல்ல) இருக்க வேண்டும் என்பதை நினைவில் கொள்ளவும் `.yml`

கருவி அகராதி மதிப்புகளை மட்டுமே மொழிபெயர்க்கிறது `.yml` அகராதி விசைகளை அல்ல.

உதாரணத்திற்கு `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

என மொழிபெயர்க்கப்படும் `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` இன் மொழிபெயர்ப்பை கைமுறையாக மாற்றலாம் (ஆனால் மொழிபெயர்ப்பில் விசைகள் அல்லது வரிகளை சேர்க்கவோ அல்லது நீக்கவோ வேண்டாம்).

`YAML` அடிப்படையில், பல்வேறு நிரலாக்க மொழிகளுக்கான சர்வதேச தீர்வுகளை நீங்கள் எளிதாக உருவாக்கலாம்.

## மேம்பட்ட பயன்பாடு

### மொழிபெயர்ப்பு துணை அடைவு

நீங்கள் `.i18n/conf.yml` வரை (ஒவ்வொரு முறையும் டெமோ திட்ட டெம்ப்ளேட்டிலிருந்து தொடங்க வேண்டிய அவசியமில்லை), `i18` நன்றாக வேலை செய்யும்.

கட்டளை வரி கருவி அனைத்து துணை அடைவுகளிலும் `.i18n/conf.yml` கண்டுபிடித்து மொழிபெயர்க்கும்.

### தனிப்பயன் நிறுவல் அடைவு

இது முன்னிருப்பாக நிறுவப்படும் `/usr/local/bin`

எழுத அனுமதி இல்லை `/usr/local/bin` அது `~/.bin` க்கு நிறுவப்படும்.

சூழல் மாறிகளை அமைக்கவும் `TO` எடுத்துக்காட்டாக, நிறுவல் கோப்பகத்தை நீங்கள் வரையறுக்கலாம் :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
