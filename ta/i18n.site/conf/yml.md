# .i18n/conf.yml

உள்ளமைவு கோப்பு `.i18n/conf.yml` `i18n.site`

[`i18`](/i18) இன் `ignore:` மற்றும் `i18n:` அமைப்புகளைத் தவிர, கட்டமைப்பு கோப்பு பின்வருமாறு:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

அவற்றில், `upload` இன் `ext:` உருப்படியை வெளியிடும் போது `.md` மட்டுமே பதிவேற்றப்படும்.

## மேல் வழிசெலுத்தல் nav

`nav:` உள்ளமைவு விருப்பங்கள், முகப்புப் பக்கத்தின் மேலே உள்ள வழிசெலுத்தல் மெனுவுடன் தொடர்புடையது.

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

அவற்றில் `i18n: home` `en/i18n.yml`中`home: Home` க்கு ஒத்திருக்கிறது.

`en/i18n.yml` `zh/i18n.yml` பல மொழிகளில் மொழிபெயர்க்கப்படும்.

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

மொழிபெயர்ப்பு முடிந்ததும், நீங்கள் மொழிபெயர்ப்பில் உள்ள `yml` மாற்றலாம், ஆனால் `yml` விசையைச் சேர்க்கவோ நீக்கவோ வேண்டாம்.

### `use: Toc` ஒற்றை கோப்பு டெம்ப்ளேட் (அவுட்லைனுடன்)

: `nav`

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` `Toc` ரெண்டரிங்கைப் பயன்படுத்துகிறது, இது ஒரு `Markdown` டெம்ப்ளேட்டை ரெண்டரிங் செய்கிறது.

`TOC` என்பது `Table of Contents` `Markdown` சுருக்கமாகும்.

`url:` `Markdown` கோப்பு பாதையை குறிக்கிறது ( `/` ரூட் கோப்பகத்திற்கு ஒத்திருக்கிறது `/README.md` , இந்த கோப்பு பெயருக்கு ஒரு பெரிய முன்னொட்டு மற்றும் ஒரு சிறிய பின்னொட்டு தேவை).

### `use: Md` ஒற்றை கோப்பு டெம்ப்ளேட் (அவுட்லைன் இல்லை)

`Md` `Toc` `Markdown` இருப்பினும் `Md` டெம்ப்ளேட் பக்கப்பட்டியில் அவுட்லைனைக் காட்டவில்லை.

நீங்கள் மேலே உள்ள கட்டமைப்பில் `use: Md` க்கு `use: Toc` `i18n.site` `md` கோப்பகத்தில் மீண்டும் இயக்கவும், பின்னர் முகப்புப்பக்கத்தில் உள்ள மாற்றங்களைக் காண மேம்பாட்டு முன்னோட்ட URL ஐப் பார்வையிடவும்.

### உள்ளமைவு பாதை இல்லாமல் இயல்புநிலை ஏற்றுதல்

அணுகப்படும் குறிப்பிட்ட பாதையின் `MarkDown` `Md` `nav:`

`nav:` `en` `/test` `/en/test.md` `Md`

இந்த கோப்பு இல்லை `/en/test.md` , இயல்புநிலை `404` பக்கம் காட்டப்படும்.

<img src="https://p.3ti.site/1721184299.avif" style="width:360px">

### `use: Doc` , பல கோப்பு டெம்ப்ளேட்

உள்ளமைவு கோப்பில்:

```
  - i18n: blog
    use: Doc
```

டெம்ப்ளேட் ரெண்டரிங் பயன்படுத்துவதைக் குறிக்கிறது `Doc`

`Doc` டெம்ப்ளேட் ஒற்றை அல்லது பல திட்டங்களுக்கு ஆவண அவுட்லைன்களை உருவாக்க பல `MarkDown` ஒருங்கிணைப்பதை ஆதரிக்கிறது.

#### ஒற்றை திட்டம் (பல கோப்புகள்)

`blog` உள்ளவை `Doc` இன் ஒற்றைத் திட்ட பயன்முறையாகும்.

```
  - i18n: blog
    url: blog
    use: Doc
```

##### url காலியாக இருக்கும்போது, ​​அது i18n இன் மதிப்புக்கு இயல்புநிலையாக இருக்கும்

`url` `url` `i18n`

மேலே எழுதும் முறை `url: blog` அதனுடன் தொடர்புடைய கோப்பு `en/blog/TOC` .

#### பல திட்டங்கள்

`.i18n/conf.yml` `i18n:doc` இல் உள்ள கட்டமைப்பு பல திட்ட பயன்முறையாகும்.

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

இங்கே, `menu: NB demo1,demo2` என்பது கீழ்தோன்றும் மெனுவை வழங்குவதற்கு `NB` பயன்படுத்துகிறது.

`NB` `Name Breif`

`NB` `demo1,demo2`
`demo1,demo2` : ** `,` **

மேலே உள்ள அளவுருக்களுக்கு, தொடர்புடைய அடைவு குறியீட்டு கோப்பு:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### TOC உள்ளடக்க அட்டவணை

`i18n.site` டெமோ கிடங்கில் `json` `js` `TOC` `.i18n/hook/after.tran/TOC.js` இயக்கும் `doc`

நீங்கள் `doc` பயன்படுத்தினால், இந்த செருகுநிரலை நீங்கள் கொண்டிருக்க வேண்டும்

`i18n.site` `.i18n`

`Doc` உருவாக்கப்படும் `json` இன் அடிப்படையில் உள்ளடக்க அட்டவணையை வழங்கும்.

`i18n.site` `README.md` `en/blog/TOC`

##### TOC ஐ மொத்தமாக நகர்த்துவது எப்படி?

`TOC` கோப்பு மூல மொழியின் கோப்பகத்தில் வைக்கப்பட வேண்டும்.

எடுத்துக்காட்டாக, மூல மொழி சீனமாக இருந்தால், மேலே `zh/blog/TOC` `TOC`

மூல மொழி மாற்றியமைக்கப்பட்டால், திட்டத்தில் உள்ள ஒரு குறிப்பிட்ட மொழியில் உள்ள `TOC` வேறு மொழிக்கு மாற்ற வேண்டும்.

பின்வரும் கட்டளைகளை நீங்கள் குறிப்பிடலாம்:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

மேலே உள்ள கட்டளையில் `en/` மற்றும் `zh/` உங்கள் மொழிக் குறியீட்டில் மாற்றவும்.
