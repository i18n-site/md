### உள்ளடக்கத்தை இடுகையிடவும்

`i18n.site` [ஒற்றை-பக்க பயன்பாட்டு](https://developer.mozilla.org/docs/Glossary/SPA) கட்டமைப்பை ஏற்றுக்கொள்கிறது, மேலும் இணையதளத்தின் நுழைவுப் பக்கமும் இணையதள உள்ளடக்கமும் தனித்தனியாக பயன்படுத்தப்படுகின்றன.

மேலே உள்ள மொழிபெயர்ப்பை இயக்கிய பிறகு, `htm` மற்றும் `v` கோப்பகங்கள் `md/out/dev` உருவாக்கப்படும்.

இங்கே, `dev` இது `.i18n/htm/dev.yml` கோப்பின் அடிப்படையில் கட்டப்பட்டது என்பதைக் குறிக்கிறது.

உள்ளடக்கத்தின் : `dev`

`htm` கீழ் இணையதள நுழைவுப் பக்கம் உள்ளது.

`v` கோப்பகத்தில் பதிப்பு எண்ணுடன் கூடிய இணையதள உள்ளடக்கம் உள்ளது.

பதிப்பு எண்ணைப் பொருட்படுத்தாமல் அனைத்து கோப்புகளையும் `out/dev/v/0.1.0`

அதிகாரப்பூர்வ வெளியீட்டிற்கு, மாற்றப்பட்ட கோப்புகள் புதிய பதிப்பு எண் கோப்பகத்தில் நகலெடுக்கப்படும்.

#### உள்ளமைவு கோப்பைக் குறிப்பிட -c ஐப் பயன்படுத்தவும்

வெவ்வேறு உள்ளமைவு கோப்புகள் `out` கீழ் தொடர்புடைய கோப்பகங்களை உருவாக்கும்.

எடுத்துக்காட்டாக, `.i18n/htm/ol.yml` ஒரு `out/ol` கோப்பகத்தை உருவாக்கும்.

`dev.yml` மற்றும் `ol.yml` ஆகியவை இயல்புநிலை உள்ளமைவுகள்.

`dev` என்பது `development` இன் சுருக்கமாகும், இது வளர்ச்சி சூழலைக் குறிக்கிறது, இது உள்ளூர் மாதிரிக்காட்சிக்கு பயன்படுத்தப்படுகிறது, மேலும் இது இயல்புநிலை உள்ளமைவு கோப்பாகும்.
`ol` என்பது `online` இன் சுருக்கமாகும், இது ஆன்லைன் சூழலைக் குறிக்கிறது, இது அதிகாரப்பூர்வ வெளியீட்டிற்குப் பயன்படுத்தப்படுகிறது, மேலும் இது `npm` `-n` கட்டளை வரி அளவுருவைப் பயன்படுத்தி) வெளியிடும் போது இயல்புநிலை உள்ளமைவுக் கோப்பாகும்.

உள்ளமைவு கோப்பின் பெயரைக் குறிப்பிட `--htm_conf` மற்ற கட்டமைப்பு கோப்புகளை உருவாக்கலாம்.

உதாரணத்திற்கு:
```
i18n.site --htm_conf yourConfig --save
```

`--save` புதுப்பிப்பு வெளியீட்டு எண்ணைக் குறிக்கிறது.

`npm` தொகுப்பை அடிப்படையாகக் கொண்டு வெளியிடும் போது, `v:` மதிப்பின் முன்னொட்டாக **`//unpkg.com/` பயன்படுத்துவதை உறுதிப்படுத்திக் கொள்ளுங்கள்** `i18n.site` இந்த முன்னொட்டு பாதையின் கீழ் `/.v` புதிய வெளியீடுகளை சரியான நேரத்தில் பார்க்கச் சிறப்பாகச் செய்யப்பட்டுள்ளது.

மொழிபெயர்த்து வெளியிட `md` `i18n.site --npm` அல்லது `i18n.site -n` இயக்கவும்.

நீங்கள் வெளியிடுவதற்கு தொடர்ச்சியான ஒருங்கிணைப்பு சூழலைப் பயன்படுத்தினால், அதை நிறுவ `nodejs` அவசியமில்லை உள்நுழைந்த மற்றும் வெளியிடப்பட்ட அனுமதியை `~/.npmrc` சூழலுக்கு நகலெடுக்கவும்.

நீங்கள் `ol.yml` `v:` தொகுப்பின் பெயரை மாற்றினால், **முதலில் `.i18n/v/ol` நீக்கிவிட்டு பின்னர் வெளியிடுவதை உறுதி செய்யவும்** .

இறுதியாக, **`1s` `/.v` முடிவடையும் பாதையின் தற்காலிக சேமிப்பு நேரத்தை உள்ளமைக்கவும்** , இல்லையெனில் புதிதாக வெளியிடப்பட்ட உள்ளடக்கத்தை உடனடியாக அணுக முடியாது.

தேவையற்ற கோரிக்கைகளை குறைக்க மற்ற பாதைகளுக்கான தற்காலிக சேமிப்பு நேரத்தை ஒரு வருடம் அல்லது அதற்கும் அதிகமாக அமைக்கலாம்.

##### உள்ளடக்கத்தை s3க்கு ஹோஸ்ட் செய்யவும்

```

ends_with(http.request.uri.path,".js") or

ends_with(http.request.uri.path,".css") or

)
```

#### சீனாவின் பிரதான நிலப்பரப்பில் இணையதள வரிசைப்படுத்தலை மேம்படுத்துதல்

சீனாவின் மெயின்லேண்ட் நெட்வொர்க் சூழலில் சிறந்த அணுகல்தன்மை செயல்திறனைப் பெற விரும்பினால், முதலில் [ஒரு டொமைன் பெயரைப் பதிவு செய்யவும்](//beian.aliyun.com) .

`out/ol/htm` , சீனாவில் `CDN` கிளவுட் விற்பனையாளர்களின் பொருள் சேமிப்பகத்தைப் பயன்படுத்தவும் +

[பைடு ஸ்மார்ட் கிளவுட் போன்ற ஒற்றைப் பக்க பயன்பாடுகளுக்கு ஏற்ப பாதையை மீண்டும் எழுத நீங்கள் எட்ஜ் கம்ப்யூட்டிங்கைப் பயன்படுத்தலாம் `CDN`](//cloud.baidu.com/product/cdn.html)

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```

### அடைவு அமைப்பு

#### பொது

`favicon.ico` `robots.txt` , போன்ற வலைத்தளத்தின் நிலையான கோப்புகள்.

ஐகான் கோப்புகளை உருவாக்க முடியும் [realfavicongenerator.net](https://realfavicongenerator.net)

#### .i18n

`.i18n` கோப்பகத்தின் கீழ் `i18n.site` இன் உள்ளமைவு கோப்புகள், மொழிபெயர்ப்பு தற்காலிக சேமிப்பு போன்றவை உள்ளன. விவரங்களுக்கு அடுத்த அத்தியாயமான ["உள்ளமைவு" ஐப்](/i18n.site/conf) பார்க்கவும்.

#### en

உள்ளமைவு கோப்பில் உள்ள `fromTo` `en` `.i18n/conf.yml` தொடர்புடைய மூல மொழி அடைவு

```yaml
i18n:
  fromTo:
    en: zh
```

மொழிபெயர்ப்பின் உள்ளமைவைப் பார்க்கவும் [i18](/i18/use)
