# சேவையகத்தைத் துவக்கவும் (BTRFS பகிர்வு, ஸ்னாப்பர் திட்டமிடப்பட்ட ஸ்னாப்ஷாட்டைப் பயன்படுத்தவும்)

## contabo கொள்முதல் சேவையகம்

contabo என்பது ஜேர்மனியில் ஒரு மலிவான VPS வழங்குநராகும், இது நீண்ட வரலாறு (20 ஆண்டுகளுக்கும் மேலாக) மற்றும் பல பயனர்கள் (200,000 க்கும் அதிகமானவர்கள்) கொண்டதாகும், எனவே ஓடிப்போவதைப் பற்றி கவலைப்படத் தேவையில்லை.

வாங்கும் VPS [contabo.com](//contabo.com) யூரோ விலையை தேர்வு செய்ய நினைவில் கொள்ளுங்கள், இது மலிவானது.

![](https://i-01.eu.org/1718285782.avif)

கான்டாபோவிற்கு ஒரு முறை நிறுவல் கட்டணம் உள்ளது.
இருப்பினும், ஒவ்வொரு மாதமும் சில நாட்கள் நிறுவல் கட்டணம் இலவசம், பதிவுசெய்து விளம்பர மின்னஞ்சல்களுக்கு கவனம் செலுத்துங்கள்.

ஐரோப்பாவில் இலவச பகுதி கட்டணத்துடன் கூடுதலாக, பிற பிராந்தியங்களில் விலை உயர்வு வெவ்வேறு நிலைகளில் உள்ளது.

நான் ஒரு ஐரோப்பிய சேவையகத்தைத் தேர்ந்தெடுத்தேன், கிளவுட்ஃப்ளேரைப் பயன்படுத்திய பிறகு, சேவையகங்களுக்கு இடையே அதிக வித்தியாசம் இல்லை.

இறுதி கட்டமைப்பு பின்வருமாறு, 16GB நினைவகம், 6CPU, 200 GB NVMe, ஒரு மாதத்திற்கு 9.50 யூரோக்கள், 74.43 RMBக்கு சமம்.

![](https://i-01.eu.org/1718286008.avif)

**இந்த விலை அலிபாபா கிளவுட் மற்றும் அமேசான் விலையில் கிட்டத்தட்ட பத்தில் ஒரு பங்கு ஆகும்** .

![](https://i-01.eu.org/1718356956.avif)

செயல்திறன் மற்றும் அளவுருக்களை இயக்க நான் [Yet-Another-Bench-Script](https://github.com/masonr/yet-another-bench-script) :

```
curl -sL yabs.sh | bash -s -- -s https://www.vpsbenchmarks.com/yabs/upload
```

மாடல் `AMD EPYC 9224 24-Core Processor` , ஒற்றை மைய செயல்திறன் மதிப்பெண் 1285 ஐக் காட்டுகிறது CPU
, பல மைய மதிப்பெண் 4277 மேலும் முடிவுகளைக் காண்க [vpsbenchmarks.com](https://www.vpsbenchmarks.com/yabs/contabo-6c-16gb-3fb7d8)

நான் ஒரே நேரத்தில் மூன்று சேவையகங்களை வாங்கினேன் மற்றும் ஒரே நேரத்தில் அவற்றை செயல்படுத்தினேன், இது சேவையகங்கள் ஒரே கணினி அறையில் இருப்பதை உறுதிசெய்கிறது, இது எதிர்காலத்தில் அதிக கிடைக்கும் தன்மையை உருவாக்குகிறது.

அதன் சேவையகங்கள் எப்போதாவது செயலிழந்து, அதிக கிடைக்கும் தன்மை மூலம் இணையதள அணுகலை உறுதி செய்கிறது.

புதிய பயனர்கள் தங்கள் அடையாள அட்டைகளை வாங்கும் போது உண்மையான பெயர் அங்கீகாரத்திற்காக சமர்ப்பிக்க வேண்டும்.

## கடவுச்சொல் இல்லாத உள்நுழைவை உள்ளமைக்கவும்

சில நிமிடங்களுக்குப் பிறகு, சர்வர்கள் அனைத்தும் செயல்படுத்தப்பட்டன.

முதல் விஷயம், மாற்றுப்பெயரை அமைக்க உள்ளூர் இயந்திரத்தின் `~/.ssh/config` திருத்த IP , பின்னர் கடவுச்சொல் இல்லாத உள்நுழைவை உள்ளமைக்க `ssh-copy-id root@c2` பயன்படுத்தவும்.

![](https://i-01.eu.org/1718287198.avif)

துவக்க உள்ளமைவை விரைவாக முடிக்க இந்த [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) மாற்றலாம்.

கடவுச்சொல் இல்லாத உள்ளமைவு முடிந்ததும், நீங்கள் அதைப் பயன்படுத்தலாம், இது வசதியானது [pdsh](https://github.com/chaos/pdsh)

முதலாவது ipv6 ஐ கைமுறையாக இயக்கி சோதனை செய்வது

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

![](https://i-01.eu.org/1718287969.avif)

பின்னர் ஒரே கிளிக்கில் கணினியை ubuntu 24 க்கு மேம்படுத்தி, பொதுவாக பயன்படுத்தப்படும் மென்பொருளை நிறுவவும்.

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 என்பது மொழியை சீன மொழிக்கும், நேர மண்டலத்தை பெய்ஜிங்கிற்கும் மாற்றுவதாகும்.

[direnv](https://direnv.net/) நிறுவப்பட்ட [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) , [mise](https://github.com/jdx/mise)

இந்த நடவடிக்கை அரை மணி நேரம் காத்திருந்த பிறகு, பொதுவாகப் பயன்படுத்தப்படும் மென்பொருளின் மேம்படுத்தல் மற்றும் நிறுவல் வெற்றிகரமாக இருக்கும். பின்வரும் கட்டளையுடன் இதை உறுதிப்படுத்தலாம்:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## மோஷ் ssh ஐ வேகப்படுத்துகிறது

ஐரோப்பாவிலிருந்து சீனா வரையிலான பிணையம் `ssh` வேகப்படுத்தப் பயன்படுத்த பரிந்துரைக்கப்படுகிறது [mosh](//mosh.org)

உள்ளூர் அடைவு `~/.bin` இல் `c0` ஸ்கிரிப்ட்களை உருவாக்கவும் (முக்கியமாக சூழல் மாறி `PATH` இல் சேர்க்கப்பட்டது) பின்வருமாறு:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

உருவாக்கிய பிறகு, இயங்கக்கூடிய அனுமதிகள் மற்றும் மென்மையான இணைப்பு `c1` மற்றும் `c2` வழங்கவும்.

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

எதிர்காலத்தில், `mosh` அதிவேக நேரடி இணைப்பைப் பயன்படுத்த நீங்கள் நேரடியாக `c0` மற்றும் `c1` உள்ளிட்டு [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

## பகிர்வு வடிவமைப்பை ext4 இலிருந்து btrfs ஆக மாற்றவும்

எளிதாக காப்புப் பிரதி எடுக்க வட்டு ஸ்னாப்ஷாட்களை [btrfs](https://wikipedia.org/wiki/Btrfs) .

முதலில் மீட்பு அமைப்பை உள்ளிடவும்

![](https://i-01.eu.org/1718300448.avif)

டெபியன் மீட்பு என்பதைத் தேர்ந்தெடுக்கவும்

![](https://i-01.eu.org/1718300753.avif)

சேவையகம் மறுதொடக்கம் செய்ய காத்திருக்கவும்

ssh மாற்றங்கள் இருக்கும், அதை சுத்தம் செய்ய `ssh-keygen -R ip` பயன்படுத்தவும்

பகிர்வை மாற்ற பின்வரும் ஸ்கிரிப்டை இயக்கவும்

```bash
nohup bash -c "bash <(curl -s https://raw.githubusercontent.com/i18n-ops/ops/main/contabo/ext4_btrfs.sh) >/tmp/nohup.out 2>&1 && reboot" &
sleep 1 && tail -f /tmp/nohup.out
```

ஸ்கிரிப்ட் வெற்றிகரமாக மாற்றப்பட்ட பிறகு கணினியை தானாகவே மறுதொடக்கம் செய்யும்.

### btrfs சுருக்க விகிதத்தைக் காண்க

இது தானாகவே சுருக்க விகிதத்தை சரிபார்க்கும் கட்டளை பின்வருமாறு:

```bash
if ! command -v compsize &>/dev/null; then
apt install -y  btrfs-compsize
fi
compsize -x /
```

### கைமுறை சுருக்கம்

நீங்கள் ஒரு பரிபூரணவாதியாக இருந்தால், நீங்கள் ஒரு முழு வட்டு சுருக்கத்தை கைமுறையாக செய்யலாம்:

```bash
btrfs filesystem defragment -r -czstd /
```

கீழே உள்ள படத்தில் காட்டப்பட்டுள்ளபடி, புதிய அமைப்பில் கைமுறையாக அழுத்திய பிறகு, கிட்டத்தட்ட 100MB அதிகமாக சுருக்கப்பட்டதைக் காணலாம்.

![](https://i-01.eu.org/2023/12/8NWlr4c.webp)

### ஸ்னாப்பருடன் Btrfs ஸ்னாப்ஷாட் காப்புப்பிரதியை உருவாக்கவும்

[snapper](https://manual.siduction.org/sys-admin-btrfs-snapper_en.html) Btrfs

துவக்க ஸ்கிரிப்ட் `~/ubuntu` கோப்பகத்தை உருவாக்கும், பின்வரும் கட்டளையை இயக்கவும்:

```
~/ubuntu/snapper/init.sh
```

ஸ்கிரிப்ட் உள்ளடக்கத்தில் சந்திப்போம் [github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh](https://github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh)

ஸ்னாப்ஷாட்கள் அமைக்கப்பட்டுள்ள துணைத் தொகுதிகளைக் காண `snapper list-configs` பயன்படுத்தவும்.

![](https://i-01.eu.org/1718302689.avif)

துணைத்தொகுதியின் ஸ்னாப்ஷாட்டைக் காண `snapper -c mnt list` பயன்படுத்தவும்

இந்த துணைத்தொகுதியின் குறிப்பிட்ட காப்பு அமைப்புகளை `/etc/snapper/configs/mnt` பார்க்க முடியும்

## மீட்பு அமைப்பு விவரங்கள்

மீட்பு அமைப்பை உள்ளிடவும் போர்ட் [tcping](https://github.com/paradiseduo/tcping) அணுகக்கூடியது 22 2-3 நிமிடங்களுக்கு நீங்கள் அதை அணுக முடியாது.

```
tcping ip 22 -c 9999
```

துறைமுக அணுகலை கண்காணிக்க முடியும்.

### மீட்பு ஸ்னாப்ஷாட்டை உள்ளிட்ட பிறகு ஹார்ட் டிரைவை அணுகுவது &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

உதாரணமாக, நீங்கள் ஸ்னாப்ஷாட்டைப் பார்க்கலாம்

```
snapper -c etc list
```

ஸ்னாப்ஷாட் கோப்பு மாற்றங்களைக் காண்க

```
snapper -c etc status 5..6
```

குறிப்பிட்ட வேறுபாடு உரையைக் காண்க

```
snapper -c etc diff 3..5
```

ஸ்னாப்ஷாட்டை மீட்டமை: எடுத்துக்காட்டாக, ஸ்னாப்ஷாட் 5க்குப் பிறகு எல்லா மாற்றங்களையும் செயல்தவிர்க்கவும்

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### பதிவு நிலைத்தன்மையை இயக்கு

மீட்பு முறைமையின் கீழ் பதிவு நிலைத்தன்மையை இயக்குவதற்கு நீங்கள் கைமுறையாக தொடர்புடைய கோப்பகத்தை உருவாக்கி சில கூடுதல் படிகளைச் செய்ய வேண்டும், ஏனெனில் உங்களிடம் இயங்கும் systemd சேவை இல்லை. நீங்கள் முயற்சி செய்யக்கூடிய படிகள் இங்கே:

முதலில் chroot, பின்னர்

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

உங்கள் உபுண்டு கணினியில் மறுதொடக்கம் செய்யுங்கள்: மேலே உள்ள மாற்றங்களைச் செய்த பிறகு, உங்கள் கணினியை சாதாரணமாக மறுதொடக்கம் செய்ய வேண்டும் மற்றும் மீட்பு அமைப்புக்கு பதிலாக உபுண்டு கணினியில் மீண்டும் துவக்க வேண்டும்.

இந்த வழியில், systemd-journald புதிய உள்ளமைவுடன் தொடங்கும் மற்றும் வன் வட்டில் பதிவுகளை தொடர்ந்து சேமிக்கத் தொடங்கும்.

மேலே உள்ள செயல்பாடுகள் மீட்பு அமைப்பில் செய்யப்படுகின்றன என்பதை நினைவில் கொள்ளவும், எனவே கணினி மறுதொடக்கம் செய்யப்படும் வரை மற்றும் மீட்பு பயன்முறையில் இல்லாத வரை அந்த மாற்றங்கள் செயல்படாது.

இதன் பொருள், ஏற்கனவே உள்ள பதிவுகளில் நிரந்தரமாக இருக்கும் எந்த மாற்றங்களையும் நீங்கள் உடனடியாகக் காண மாட்டீர்கள், ஆனால் அதன் பிறகு உருவாக்கப்பட்ட அனைத்து பதிவுகளும் தொடரும்.

இந்த வழியில் நீங்கள் ssh சேவையின் பதிவுகளைப் பார்க்கலாம்

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## கடினமான மற்றும் சிக்கலான நோய்கள் பற்றிய குறிப்புகள்

### வித்தியாசமான சிக்கலைத் தீர்க்கவும்: சேவையகம் ssh செய்ய முடியாது

நான் முன்பு ஒரு தானியங்கி தொகுத்தல் மற்றும் வரிசைப்படுத்தல் ஸ்கிரிப்டை உருவாக்க கிதுப் செயலைப் பயன்படுத்தினேன், ஆனால் வரிசைப்படுத்தல் சேவை ssh ஆக இருக்க முடியாது என்பதைக் கண்டறிந்தேன்.

மீட்பு அமைப்பில் நுழைந்து பதிவுகளைப் பார்ப்பது அனுமதிச் சிக்கலாகும்:

`Missing privilege separation directory: /run/sshd`

பின்வரும் கட்டளையுடன் இந்த கோப்பகத்தை உருவாக்கலாம்

```
chown root:root /
systemd-tmpfiles --create
```

பலமுறை விசாரணைக்குப் பிறகு, tar இன் github action `uid` `1001` ஆகும்

`--no-same-owner` அளவுரு இல்லாவிட்டால், `1001` பயனர்களுக்கு `uid` ஆகக் குறைக்கப்படும் tar

இதன் விளைவாக, ரூட் கோப்பகத்தின் பயனர் குழுவை rsync மாற்றும், பின்னர் நீங்கள் மறுதொடக்கம் செய்த பிறகு ssh இல் உள்நுழைய முடியாது.

டிகம்ப்ரஷன் ஸ்கிரிப்டை பின்வருமாறு மாற்றவும்:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
