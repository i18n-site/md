# インストールおよび使用

## WindowsではまずGit Bashをインストールする

windowsシステム。まず[ここをクリックして`git bash`をダウンロードし、インストールしてください](https://git-scm.com/download/win)。

後続の操作は`git bash`を使用して実行してください。

## インストール

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### 翻訳トークンの設定

[i18n.site/token](//i18n.site/token)にアクセスしてトークンをコピーしてください

<img src="https://p.3ti.site/1719911689.avif" style="width:400px">

`~/.config/i18n.site.yml`を作成し、コピーした内容を貼り付けます。内容は以下の通りです:

```
token: YOUR_API_TOKEN
```

また、[i18n.site/payBill](//i18n.site/payBill)で支払いカードを紐付ける必要があります(充電は不要で、サイトは使用量に応じて自動的に請求します。[価格はホームページを参照してください](/#price))。

## 使用方法

### デモプロジェクト

`i18`の翻訳設定については、デモプロジェクト[github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)を参照してください。

中国エリアのユーザーは[atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)をクローンできます

クローンを作成した後、ディレクトリに入り、`i18`を実行することで翻訳を完了します。

### ディレクトリ構造

テンプレートリポジトリのディレクトリ構造は以下の通りです

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en`ディレクトリ内の翻訳デモファイルは単なる例ですので、削除しても問題ありません。

### 翻訳を実行する

ディレクトリに入り、`i18`を実行して翻訳を行います。

プログラムは翻訳に加えて、`.i18n/data`ディレクトリも生成します。このディレクトリをリポジトリに追加してください。

新しいファイルを翻訳した後、このディレクトリ配下に新しいデータファイルが生成されます。忘れずに`git add .`を実行して追加してください。

## 設定ファイル

`.i18n/conf.yml`は`i18`コマンドライン翻訳ツールの設定ファイルです

内容は以下の通りです:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ソース言語と翻訳言語

設定ファイルでは、`fromTo`の下位は以下のようになります:

`en`はソース言語であり、`zh ja ko de fr`は翻訳のターゲット言語です。

言語コードは[i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)を参照してください。

たとえば、中国語を英語に翻訳したい場合は、以下の行を`zh: en`に変更します。

全てのサポート言語に翻訳したい場合は、`:`の後ろを空白のままにします。例えば:

```
i18n:
  fromTo:
    en:
```

異なるサブディレクトリ/ファイルに対して異なる`fromTo`を設定することができます。デモの記述例は以下の通りです:

```
i18n:
  fromTo:
    en:
  path:
    blog:
      fromTo:
        zh:
    blog/your_file_name.md:
      fromTo:
        ja:
```

この設定表では、`blog`ディレクトリのソース言語は`zh`であり、`blog/your_file_name.md`のソース言語は`ja`です。

### ファイルを無視する

デフォルトでは、ソース言語ディレクトリ内の`.md`および`.yml`で始まるすべてのファイルが翻訳されます。

特定のファイル(未完成の草稿など)を無視して翻訳しないように、`ignore`フィールドを設定します。

`ignore`は`.gitignore`ファイルと同様の[globset](https://docs.rs/globset/latest/globset/#syntax)構文を使用します。

たとえば、設定ファイルの`_*`は`_`で始まるファイルを無視することを意味します。

## 翻訳規則

### 翻訳の編集では行を追加または削除しないでください

翻訳は編集可能です。元のテキストを変更し、再度機械翻訳を行うと、翻訳に対する手動の修正は上書きされません(元のテキストのこの段落が変更されていない場合)。

ただし、翻訳と原文の行は1対1で対応させる必要があります。つまり、翻訳を編集する際には行を追加または削除しないでください。そうしないと、翻訳編集キャッシュが混乱する可能性があります。

問題が発生した場合は、[よくある質問の解決策を参照してください](/i18/qa#H1)

### `YAML`の翻訳

コマンドラインツールは、ソース言語ファイルディレクトリ内の`.yml`で終わるすべてのファイルを検索し翻訳します。

* ファイル名の拡張子は`.yml`でなければなりません(`.yaml`ではありません)ことに注意してください。

このツールは`.yml`の辞書の値のみを翻訳し、辞書の键は翻訳しません。

たとえば、`i18n/en/i18n.yml`は

```
apiToken: API Token
defaultToken: Default Token
```

`i18n/zh/i18n.yml`に翻訳されます

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML`の変換は手動で変更することもできます (ただし、変換内のキーや行を追加または削除しないでください)。

`YAML`の翻訳に基づいて、さまざまなプログラミング言語の国際化ソリューションを簡単に構築できます。

## 高度な使い方

### サブディレクトリの翻訳

`.i18n/conf.yml`が存在する限り(デモプロジェクトのテンプレートから毎回始める必要はありません)、`i18`は正常に動作します。

コマンドラインツールは、すべてのサブディレクトリ内の`.i18n/conf.yml`設定を見つけて翻訳します。

[monorepo](//monorepo.tools)アーキテクチャを使用するプロジェクトでは、言語ファイルをサブディレクトリに分割して配置できます。

![](https://p.3ti.site/1719910016.avif)

### カスタムインストールディレクトリ

デフォルトでは`/usr/local/bin`にインストールされます。

`/usr/local/bin`への書き込み権限がない場合、インストールは`~/.bin`に移行されます。

環境変数`TO`を設定することで、インストールディレクトリを定義できます。例えば:

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
