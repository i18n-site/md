# インストールして使用する

## Windows ではまず git bash をインストールします

windowsシステム。[まずここをクリックしてダウンロードしてインストールしてください`git bash`](https://git-scm.com/download/win)

`git bash`で後続の操作を実行します。

## インストール

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### 翻訳トークンを構成する

クリックしてトークンをコピーして[i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

`~/.config/i18n.site.yml`コピーした内容を貼り付けます。内容は次のとおりです。

```
token: YOUR_API_TOKEN
```

さらに、支払いにはクレジット カードをバインド[i18n.site/payBill](//i18n.site/payBill)必要があります (リチャージは必要ありません。Web サイトは使用量に応じて料金を自動的に差し引きます。[価格についてはホームページを参照してください](/#price))。

## 使用

### デモプロジェクト

翻訳の構成については`i18`デモプロジェクトを参照してください[github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

中国のユーザーはクローンを作成できます[atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

クローン作成後、ディレクトリに入り、 `i18`を実行して変換を完了します。

### ディレクトリ構造

テンプレートウェアハウスのディレクトリ構造は次のとおりです。

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en`ディレクトリには翻訳されたデモ ファイルが含まれていますが、これは単なる例なので、削除しても構いません。

### 翻訳を実行する

ディレクトリに入り、 `i18`を実行して翻訳します。

### リポジトリにファイルを追加する

翻訳に加えて、プログラムは次のファイルも生成します。これらをリポジトリに追加してください。

```
.i18n/hash
.i18n/cache/.gitignore
```

その内、 `.i18n/cache/.gitignore`の内容は以下の通りです:

```
**/*
!**/.gitignore
```

これは、 `.i18n/cache/`ディレクトリ内のすべてのファイルを無視することを意味します ( `.i18n/cache/.gitignore`を除く)。

バージョン管理ソフトウェアが`git`でない場合は、この設定に従って無視してください。

## 設定ファイル

`.i18n/conf.yml` `i18`ライン変換ツールの設定ファイルです

内容は以下の通りです。

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ソース&翻訳言語

設定ファイル内の`fromTo`の配下は次のとおりです。

`en`はソース言語、 `zh ja ko de fr`は翻訳先の言語です。

言語コードを参照してください[i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

たとえば、中国語を英語に翻訳したい場合は、この行を`zh: en`

サポートされているすべての言語に翻訳したい場合は、 `:`の後ろを空白のままにしてください。例えば

```
i18n:
  fromTo:
    en:
```

### ファイルを無視する

デフォルトでは、ソース言語ディレクトリ内の`.md`と`.yml`で始まるすべてのファイルが翻訳されます。

特定のファイル (未完成の下書きなど) を無視して翻訳しない場合は、 `ignore`フィールド設定を使用できます。

`ignore` `.gitignore`に似た構文を使用します[globset](https://docs.rs/globset/latest/globset/#syntax)

たとえば、上記の設定ファイルの`_* `は、 `_`で始まるファイルは翻訳されないことを意味します。

## 翻訳ルール

### 翻訳編集者は行を追加または削除してはなりません

翻訳は編集可能です。元のテキストを変更して再度機械翻訳すると、翻訳に対する手動の変更は上書きされません (元のテキストのこの段落が変更されていない場合)。

ただし、翻訳文と原文の行は 1 対 1 に対応している必要があることに注意してください。つまり、翻訳をコンパイルするときに行を追加または削除しないでください。そうしないと、翻訳編集キャッシュで混乱が生じる可能性があります。

何か問題が発生した場合は、 [FAQ で解決策を参照してください。](/i18/qa#H1)

### `YAML`翻訳する

コマンド ライン ツールは、ソース言語ファイル ディレクトリ内で`.yml`で終わるすべてのファイルを検索し、翻訳します。

* ファイル名の接尾辞は ( `.yaml`ではなく) でなければならないことに注意してください`.yml`

このツールは、 `.yml`の辞書値のみを変換し、辞書キーは変換しません。

例えば`i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

`i18n/zh/i18n.yml`として翻訳されます。

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML`の翻訳は手動で変更することもできます (ただし、翻訳内のキーや行を追加または削除しないでください)。

`YAML` Translation に基づいて、さまざまなプログラミング言語用の国際的なソリューションを簡単に構築できます。

## 高度な使用法

### 翻訳サブディレクトリ

`.i18n/conf.yml`を作成する限り (毎回デモ プロジェクト テンプレートから開始する必要はありません)、 `i18`正常に動作します。

コマンド ライン ツールは、すべてのサブディレクトリ内の`.i18n/conf.yml`設定を検索して変換します。

[monorepo](//monorepo.tools)アーキテクチャを使用するプロジェクトでは、言語ファイルをサブディレクトリに分割できます。

<img alt="" src="https://p.3ti.site/1719910016.avif">

### カスタムインストールディレクトリ

デフォルトでは`/usr/local/bin`にインストールされます。

`/usr/local/bin`に書き込み権限がない場合は、 `~/.bin`にインストールされます。

環境変数を設定します`TO`たとえば、インストール ディレクトリを定義できます:

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
