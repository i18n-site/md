# インストールして使用する

## Windowsではまずgit bashをインストールします

Windowsシステム。[まずここをクリックして`git bash`をダウンロードしてインストールしてください。](https://git-scm.com/download/win)

以降の操作は`git bash`で実行してください。

## インストール

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### 翻訳トークンの設定

クリックしてトークンをコピーする[i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="width:400px">

`~/.config/i18n.site.yml`を作成し、コピーした内容を貼り付けてください。内容は以下の通りです

```
token: YOUR_API_TOKEN
```

また、[i18n.site/payBill](//i18n.site/payBill)で支払い信用卡を绑定する必要があります(充值は不要で、サイトは使用量に応じて自動的に請求します。[価格はホームページを参照してください](/#price))。

## 使用

### デモプロジェクト

`i18`の翻訳設定を学ぶには、デモプロジェクトを参照してください。[github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

中国エリアのユーザーは、[atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)をクローンできます

クローンを作成した後、ディレクトリに移動し、`i18`を実行して翻訳を完了します。

### ディレクトリ構造

テンプレートリポジトリのディレクトリ構造は以下の通りです

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en`ディレクトリ内の翻訳デモファイルは単なる例です。必要に応じて削除してください。

### 翻訳を実行する

ディレクトリに移動し、`i18`を実行して翻訳します。

プログラムは`.i18n/data`フォルダーも生成します。これをリポジトリに追加してください。

新しいファイルを翻訳した後、このディレクトリに新しいデータファイルが生成されるので、`git add .`を忘れずに追加してください。

## 設定ファイル

`.i18n/conf.yml`は`i18`コマンドライン翻訳ツールの設定ファイルです

内容は以下の通りです

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ソース言語と翻訳言語

設定ファイルでは、`fromTo`の設定は以下の通りです

`en`はソース言語で、`zh ja ko de fr`は翻訳のターゲット言語です。

言語コードは[i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)を参照してください。

例えば、中国語を英語に翻訳する場合は、`zh: en`と書き換えます。

サポートしているすべての言語に翻訳する場合は、`:`の後ろを空白のままにします。例えば

```
i18n:
  fromTo:
    en:
```

異なるサブディレクトリに対して異なる`fromTo`を設定できます/デモは次のように記述されます:

```
i18n:
  fromTo:
    en:
  path:
    blog:
      fromTo:
        zh:
    blog/your_file_name.md:
      fromTo:
        ja:
```

この設定では、`blog`ディレクトリのソース言語は`zh`で、`blog/your_file_name.md`のソース言語は`ja`です。

### ファイルの無視

デフォルトでは、ソース言語ディレクトリ内の`.md`および`.yml`ファイルすべてを翻訳します。

特定のファイル(未完成の草稿など)を無視して翻訳しないようにするには、`ignore`フィールドを設定します。

`ignore`は`.gitignore`ファイルと同様の[globset](https://docs.rs/globset/latest/globset/#syntax)構文を使用します。

例えば、設定ファイルの`_*`は`_`で始まるファイルを翻訳しないことを意味します。

## 翻訳のルール

### 翻訳の編集では行を追加または削除しないでください

翻訳は編集可能です。元のテキストを修正し、再度機械翻訳を行うと、既に手動で修正された翻訳は上書きされません(元のテキストのこの段落が変更されていない場合を除きます)。

ただし、翻訳文と原文の行は1対1で対応する必要があります。つまり、翻訳を編集する際には行を追加または削除しないでください。そうしないと、翻訳の編集キャッシュが混乱する可能性があります。

問題が発生した場合は、[よくある質問の解決策を参照してください](/i18/qa#H1)

### `YAML`の翻訳

コマンドラインツールは、ソース言語ファイルディレクトリ内で`.yml`で終わるすべてのファイルを検索し、翻訳します。

* ファイル名の接尾辞は`.yml`にする必要があります(`.yaml`ではありません)。

ツールは`.yml`の辞書の値のみを翻訳し、辞書のキーは翻訳しません。

例えば、`i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

は`i18n/zh/i18n.yml`に翻訳されます

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML`の変換は手動で変更することもできます (ただし、変換内のキーや行を追加または削除しないでください)。

`YAML`の翻訳を基に、さまざまなプログラミング言語の国際化ソリューションを簡単に構築できます。

## 高度な使い方

### サブディレクトリの翻訳

`.i18n/conf.yml`が存在する限り、`i18`は正常に動作します(デモプロジェクトのテンプレートから始める必要はありません)。

コマンドラインツールは、すべてのサブディレクトリ内の`.i18n/conf.yml`設定を検索し、それに基づいて翻訳を行います。

[monorepo](//monorepo.tools)アーキテクチャのプロジェクトでは、言語ファイルをサブディレクトリに分割して配置できます。

![](https://p.3ti.site/1719910016.avif)

### カスタムインストールディレクトリ

デフォルトでは`/usr/local/bin`にインストールされます。

`/usr/local/bin`に書き込み権限がない場合は、`~/.bin`にインストールされます。

環境変数`TO`を設定することで、インストール先ディレクトリを指定できます。例えば、以下のようになります:

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
