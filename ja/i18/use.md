# インストールして使用する

## Windowsではまずgit bashをインストールします

Windowsシステム。[まずここをクリックして`git bash`をダウンロードしてインストールしてください。](https://git-scm.com/download/win)

`git bash`中进行后续操作。

## インストール

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### 翻訳トークンの設定

クリックしてトークンをコピーする[i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="width:400px">

`~/.config/i18n.site.yml`を作成し、コピーした内容を貼り付けます。内容は以下の通りです:

```
token: YOUR_API_TOKEN
```

また、[i18n.site/payBill](//i18n.site/payBill)で支払い用のクレジットカードを紐づける必要があります(充電は不要で、サイトは使用量に応じて自動的に請求します。[価格はこちら](/#price)を参照してください)。

## 使用

### デモプロジェクト

`i18`の翻訳設定を学ぶには、デモプロジェクト[github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)を参照してください。

中国エリアのユーザーは、[atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)をクローンできます

クローン後、ディレクトリに移動し、`i18`を実行することで翻訳が完了します。

### ディレクトリ構造

テンプレートリポジトリのディレクトリ構造は以下の通りです

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en`ディレクトリ内の翻訳デモファイルは、単なる例ですので、削除しても問題ありません。

### 翻訳を実行する

ディレクトリに移動し、`i18`を実行して翻訳を行います。

プログラムは`.i18n/data`フォルダーも生成します。これをリポジトリに追加してください。

新しいファイルを翻訳した後、このディレクトリに新しいデータファイルが生成されますので、`git add .` を追加することを忘れないでください。

## 設定ファイル

`.i18n/conf.yml`は`i18`コマンドライン翻訳ツールの設定ファイルです

内容は以下の通りです:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ソース言語と翻訳言語

設定ファイルでは、`fromTo`の設定は以下の通りです:

`en`はソース言語で、`zh ja ko de fr`は翻訳先言語です。

言語コードは[i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)を参照してください。

例えば、中国語を英語に翻訳したい場合は、`zh: en`と書き換えます。

サポートしているすべての言語に翻訳したい場合は、`:`の後ろを空白のままにします。例えば:

```
i18n:
  fromTo:
    en:
```

異なるサブディレクトリに対して異なる`fromTo`設定できます/デモは次のように記述されます:

```
i18n:
  fromTo:
    en:
  path:
    blog:
      fromTo:
        zh:
    blog/your_file_name.md:
      fromTo:
        ja:
```

この設定表中、`blog` ディレクトリの翻訳元言語は `zh` であり、`blog/your_file_name.md` の翻訳元言語は `ja` です。

### ファイルの無視

デフォルトでは、ソース言語ディレクトリ内の`.md`および`.yml`で始まるすべてのファイルが翻訳されます。

特定のファイル(例えば、未完成の草稿)を翻訳から除外したい場合は、`ignore`フィールドを設定します。

`ignore`は`.gitignore`ファイルと同様の[globset](https://docs.rs/globset/latest/globset/#syntax)構文を使用します。

例えば、上記の設定ファイルでは`_*`は`_`で始まるファイルを翻訳しないことを意味します。

## 翻訳のルール

### 翻訳の編集では行を追加または削除しないでください

翻訳は編集可能です。元のテキストを変更し、再度機械翻訳を行うと、手動で行った翻訳の変更は上書きされません(元のテキストのこの段落が変更されていない場合を除きます)。

> [!WARN]
> 润色后的译文:

問題が発生した場合は、[よくある質問の解決策](/i18/qa#H1)を参照してください

### `YAML` 翻訳

コマンドラインツールは、ソース言語ファイルディレクトリ内で`.yml`で終わるすべてのファイルを検索し、翻訳します。

* ファイル名の接尾辞は`.yml`である必要があります(`.yaml`ではありません)。

ツールは`.yml`の辞書の値のみを翻訳し、辞書のキーは翻訳しません。

例えば、`i18n/en/i18n.yml`は以下のように翻訳されます:

```
apiToken: API Token
defaultToken: Default Token
```

`i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML`の変換は手動で変更することもできます (ただし、変換内のキーや行を追加または削除しないでください)。

`YAML`の翻訳を基に、さまざまなプログラミング言語に対応する国際化ソリューションを簡単に構築できます。

## 高度な使い方

### サブディレクトリの翻訳

`.i18n/conf.yml`が存在する限り(デモプロジェクトのテンプレートから毎回始める必要はありません)、`i18`は正常に動作します。

コマンドラインツールは、すべてのサブディレクトリで`.i18n/conf.yml`の設定を見つけて翻訳します。

[monorepo](//monorepo.tools)アーキテクチャを使用するプロジェクトでは、言語ファイルをサブディレクトリに分割できます。

![](https://p.3ti.site/1719910016.avif)

### カスタムインストールディレクトリ

デフォルトでは`/usr/local/bin`にインストールされます。

`/usr/local/bin`に書き込み権限がない場合は、`~/.bin`にインストールされます。

環境変数 `TO` を設定することで、インストールディレクトリを定義することができます。例えば、以下のようになります:

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
