# コマンドラインパラメータの詳細な説明

## `-p`でファイルをクリーンアップ

`-p`または`--purge`を使用すると、各翻訳ディレクトリに存在するがソース言語ディレクトリに存在しないファイルが削除されます。

ドキュメント作成の際、Markdownファイル名が頻繁に変更されるため、翻訳ディレクトリ内に不要な古いファイルが多く残る場合があります。

このパラメータを使用すると、他の言語ディレクトリ内で削除すべきファイルをクリーンアップすることができます。

## `-d`で翻訳ディレクトリを指定

翻訳ディレクトリのデフォルトは、現在のファイルがあるディレクトリです。

`-d`または`--workdir`を使用して、以下のように翻訳ディレクトリを指定できます

```
i18 -d ~/i18n/md
```

## `-h`でヘルプを表示

`-h`または`--help`を使用すると、コマンドラインのヘルプ情報が表示されます。
