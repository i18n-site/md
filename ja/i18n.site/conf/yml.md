# .i18n/conf.yml

`i18n.site`設定ファイルは`.i18n/conf.yml`です。

[`i18`](/i18)の`ignore:`と`i18n:`の設定を除いた構成ファイルは次のとおりです。

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

このうち、 `ext:`の設定項目は`upload`公開時に`.md`のみをアップロードすることを意味します。

### `use: Toc` 、単一ファイルのテンプレート (アウトライン付き)

`nav`設定中:

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` 、単一の`Markdown`テンプレートをレンダリングする`Toc`レンダリングを使用することを意味します。

`TOC`は`Table of Contents`の略です。このテンプレートがレンダリングされると、この`Markdown`ファイルのアウトラインがサイドバーに表示されます。

`url:`は`Markdown`のファイル パスを表します ( `/`ルート ディレクトリに対応します`/README.md`このファイル名には大文字の接頭辞と小文字の接尾辞が必要です)。

### `use: Md` 、単一ファイルのテンプレート (アウトラインなし)

`Md` Template は`Toc`と同じで、どちらも単一の`Markdown`ファイルをレンダリングするために使用されます。ただし`Md`テンプレートではサイドバーにアウトラインが表示されません。

上記の設定の`use: Toc`を`use: Md`に変更し、 `md`ディレクトリで`i18n.site`を再度実行し、開発プレビュー URL にアクセスしてホームページ上の変更を確認します。

### 設定パスなしのデフォルトのロード

アクセスされている特定のパスのパス接頭辞が`nav:`そのパスに対応する`MarkDown`ファイルがデフォルトでロードされ、 `Md` template を使用してレンダリングされます。

たとえば、 `/test`にアクセスし、 `nav:`このパスなしで設定されており、ページ言語が英語 (コード`en` ) の場合、デフォルトでテンプレートが読み込まれます`/en/test.md`および を使用してレンダリングされます`Md` 。

`/en/test.md`このファイルが存在しない場合は、デフォルトの`404`ページが表示されます。

### `use: Doc` 、複数ファイルのテンプレート

構成ファイル内:

```
  - i18n: blog
    use: Doc
```

テンプレートのレンダリングに`Doc`を使用することを示します。

`Doc`テンプレートは、単一または複数のプロジェクトのドキュメント アウトラインを生成するための複数の`MarkDown`統合をサポートしています。

#### 単一プロジェクト (複数ファイル)

上記の`blog`は`Doc`の単一プロジェクト モードです。

```
  - i18n: blog
    url: blog
    use: Doc
```

##### URL が空の場合、デフォルトの値は i18n になります。

`url`記述しない場合、 `url` `i18n`の値になります。このルールは他のテンプレートにも有効です。

上記の書き方は`url: blog`対応するファイルは`en/blog/TOC`です。

#### 複数のプロジェクト

`.i18n/conf.yml` `i18n:doc`の構成はマルチプロジェクト モードです。

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

ここで、 `menu: NB demo1,demo2`は、 `NB`を使用してドロップダウン メニューをレンダリングすることを意味します。

`NB`は`Name Breif`の省略形で、ドロップダウン メニューにプロジェクトの名前とスローガンを表示できることを示します。

`NB`の後にパラメータ`demo1,demo2`渡されます。
: **のカンマ`,`の前後にスペース**があってはいけないことに注意してください`demo1,demo2`

上記のパラメータに対応するディレクトリ インデックス ファイルは次のとおりです。

* `en/demo1/TOC`
* `en/demo2/TOC`

#### 目次索引

`i18n.site`デモ ウェアハウスで`js` plugin `.i18n/hook/after.tran/TOC.js`を実行し、 `doc`テンプレート設定に対応する`TOC`ディレクトリ インデックス ファイルを読み取り、ディレクトリ アウトラインの`json`を生成します。

`doc`テンプレートを使用する場合は、このプラグインが必要です。

空のフォルダーから`i18n.site`プロジェクトを初期化する場合は、デモ プロジェクトの`.i18n`を忘れずにディレクトリにコピーしてください。

`Doc`テンプレートは、生成された`json`に基づいて目次のアウトラインをレンダリングします。

##### 詳しい内容説明

内容:以下の通りです`en/blog/TOC`

```
README.md

news/README.md
  news/begin.md
```

##### インデントを使用してレベルを示します

上`en/blog/TOC`最初の行の`README.md`下の図の`i18n.site`に対応しており、これがプロジェクト名です。

次の 2 行は、以下のスクリーンショットに示されているとおりです。

<img src="https://p.3ti.site/1721097381.avif" style="width:320px">

`news/README.md`に対応`News` 、
`news/begin.md`に対応`Our Product is Online !`

`TOC`ファイルはアウトラインの階層関係を示すためにインデントされており、複数レベルのインデントをサポートしています。

##### 親レベルではタイトルのみが書き込まれ、内容は書き込まれません。

複数のレベルのインデントがある場合、親レベルではタイトルのみが書き込まれ、コンテンツは書き込まれません。そうしないと、タイポグラフィが崩れてしまいます。

##### プロジェクトの README.md 

たとえば、プロジェクトの`README.md` `en/demo2/README.md`コンテンツを記述できます。

このファイルの内容には目次の概要が示されていないため、長さを制限して短い紹介文を書くことをお勧めします。

##### 目次を一括で移動するにはどうすればよいですか?

`TOC`ファイルはソース言語のディレクトリに配置する必要があります。

たとえば、ソース言語が中国語の場合、上記の`TOC`は`zh/blog/TOC`です。

ソース言語が変更された場合は、プロジェクト内の特定の言語の`TOC`別の言語にバッチ移動する必要があります。

次のコマンドを参照できます。

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

上記のコマンドの`en/`と`zh/`言語コードに変更してください。
