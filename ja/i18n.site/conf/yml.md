# .i18n/conf.yml

`i18n.site`の設定ファイルは`.i18n/conf.yml`です。

[`i18`](/i18)の`ignore:`および`i18n:`の設定を除いた、設定ファイルは以下の通りです

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
addon:
  - i18n.addon/toc
```

その中で、`upload`の`ext:`設定項目は、公開時に`.md`ファイルのみをアップロードすることを意味します。

## 顶部导航 ナビ

`nav:`設定オプションは、ホームページの顶部ナビゲーションメニューに対応します。

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

その中で、`i18n: home`は`en/i18n.yml`の`home: Home`に相当します。

`en/i18n.yml`は`zh/i18n.yml`など、複数の言語に翻訳されます。

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

翻訳が完了した後、翻訳文の`yml`の値を修正することができますが、`yml`のキーを追加または削除しないでください。

### `use: Toc`、単一ファイルテンプレート(アウトライン付き)

`nav`設定では:

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc`は、`Toc`テンプレートを使用してレンダリングすることを意味し、単一の`Markdown`テンプレートをレンダリングします。

`TOC`は`Table of Contents`の略で、このテンプレートがレンダリングされると、サイドバーにこの`Markdown`ファイルのアウトラインが表示されます。

`url:`は`Markdown`のファイルパスを示します(`/`はルートディレクトリを表し、`/README.md`に対応します。ファイル名には大文字の接頭辞と小文字の接尾辞が必要です)。

### `use: Md`、単一ファイルテンプレート(アウトラインなし)

`Md`テンプレートと`Toc`テンプレートは同一であり、どちらも単一の`Markdown`ファイルのレンダリングに用いられますが、`Md`テンプレートではサイドバーに目次が表示されません。

上記の設定で`use: Toc`を`use: Md`に変更し、`md`ディレクトリで`i18n.site`を再度実行し、開発プレビューURLにアクセスして、ホームページの変更を確認してください。

### 設定パスなしのデフォルトロード

特定のパスにアクセスし、そのパスのプレフィックスが`nav:`に設定されていない場合、そのパスに対応する`Markdown`ファイルがデフォルトでロードされ、`Md`テンプレートでレンダリングされます。

例えば、`/test`にアクセスし、このパスが`nav:`設定にない場合、ページの言語が英語(コード`en`)の場合、デフォルトで`/en/test.md`がロードされ、`Md`テンプレートでレンダリングされます。

`/en/test.md`が存在しない場合、デフォルトの`404`ページが表示されます。

### `use: Doc`、複数ファイルテンプレート

設定ファイルでは:

```
  - i18n: blog
    use: Doc
```

`Doc`テンプレートを使用してレンダリングすることを示します。

`Doc`テンプレートは、複数の`Markdown`を統合して、単一または複数のプロジェクトのドキュメントアウトラインを生成することをサポートします。

#### 単一プロジェクト(複数ファイル)

前述の`blog`は`Doc`の単一プロジェクトモードです。

```
  - i18n: blog
    url: blog
    use: Doc
```

##### URLが空の場合、デフォルトはi18nの値です

`url`を記述しない場合、デフォルトの`url`は`i18n`の値となります。この規則は他のテンプレートにも適用されます。

上記の記述方法は`url: blog`に相当し、対応するファイルは`en/blog/TOC`です。

#### 複数プロジェクト

`.i18n/conf.yml`の`i18n:doc`設定は、マルチプロジェクトモードです。

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

ここで、`menu: NB demo1,demo2`は、`NB`テンプレートを使用してドロップダウンメニューをレンダリングすることを意味します。

`NB`は`Name Brief`の略で、ドロップダウンメニューにプロジェクトの名前とスローガンを表示できることを意味します。

`NB`の後にパラメータ`demo1,demo2`が渡されます。
注意:**`demo1,demo2`のカンマ`,`の前後にスペースを入れると、エラーが発生します**。

上記のパラメータに対応するディレクトリインデックスファイルは以下の通りです

* `en/demo1/TOC`
* `en/demo2/TOC`

#### TOCディレクトリインデックス

`i18n.site`は、デモリポジトリ内の`js`プラグイン`.i18n/hook/after.tran/TOC.js`を実行して、`TOC`テンプレート設定に対応する`doc`ディレクトリインデックスファイルを読み取り、`json`のディレクトリアウトラインを生成します。

`doc`テンプレートを使用する場合は、このプラグインが必要です。

空のフォルダーから`i18n.site`プロジェクトを初期化する際は、デモプロジェクトの`.i18n`をディレクトリにコピーしてください。

`Doc`テンプレートは、生成された`json`に基づいてディレクトリアウトラインをレンダリングします。

##### 内容の詳細な説明

`en/blog/TOC`の内容は以下の通りです:

```
README.md

news/README.md
  news/begin.md
```

##### インデントを使用してレベルを示します

上記の`en/blog/TOC`の最初の行の`README.md`は、下图の`i18n.site`に対応しており、プロジェクト名を示します。

次の2行は以下のスクリーンショットに示されています。

<img src="https://p.3ti.site/1721097381.avif" style="width:320px">

`news/README.md`は`News`に対応し、
`news/begin.md`は`Our Product is Online !`に対応します

`TOC`ファイルはインデントを使用してアウトラインの階層関係を示し、複数のインデントレベルをサポートし、`#`で始まる行はコメントとしてサポートされます。

##### 親レベルではタイトルのみが記入され、内容は記入されません

複数のインデントレベルがある場合、親レベルではタイトルのみが記入され、内容は記入されません。そうしないと、レイアウトが崩れる可能性があります。

##### プロジェクトのREADME.md

プロジェクトの`README.md`には、例えば`en/demo2/README.md`に内容を記述します。

このファイルの内容には目次の概要が含まれていないため、短い紹介文を記述することを推奨します。

###### プロジェクトのスローガン

`Deme Two`はドロップダウンメニューの下にプロジェクトのスローガンが表示され、カタログのアウトラインのプロジェクト名の下に`Your Project slogan`が表示されます。

![](https://p.3ti.site/1721276842.avif)

これは`en/demo2/README.md`の最初の行に対応します:

```
# Demo Two : Your Project slogan
```

プロジェクトの`README.md`の1レベル目のタイトルの最初のコロン`:`以降の内容がプロジェクトのスローガンと見なされます。

中国、日本、韓国のユーザーは、全角コロンではなく半角コロン`:`を使用する必要があります。

##### TOCを一括で移動する方法は？

ソース言語のディレクトリに配置する必要があるファイルは`TOC`です。

例えば、ソース言語が中国語の場合、上記の`TOC`は`zh/blog/TOC`です。

ソース言語を変更した場合、プロジェクト内の特定の言語の`TOC`ファイルを別の言語に一括移動する必要があります。

以下のコマンドを参照してください

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

上記のコマンドの`en/`と`zh/`を言語コードに変更してください。
