# .i18n/conf.yml

`i18n.site`の設定ファイルは`.i18n/conf.yml`で、以下の内容です:

```yaml
i18n:
  fromTo:
    en:
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Blog
addon:
  - i18n.addon/toc
```

ここで、`upload`の`ext:`設定項目は、公開時に`.md`ファイルのみをアップロードすることを意味します。

## 顶部导航 ナビ

`nav:`設定オプションは、ホームページの顶部ナビゲーションメニューに対応します。

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

その中で、`i18n: home`は`en/i18n.yml`の`home: Home`に対応します(`en`はプロジェクトの翻訳元言語です)。

`en/i18n.yml`の内容はナビゲーションメニューに表示されるテキストで、設定の`fromTo`に基づいて翻訳されます。例えば、`zh/i18n.yml`に翻訳されます。

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

翻訳が完了したら、翻訳された`yml`の値を修正することはできますが、`yml`のキーを追加または削除しないでください。

### `use: Toc`、アウトライン付きの単一ファイルテンプレート

`nav`設定では:

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc`は、`Toc`テンプレートを使用してレンダリングすることを意味し、単一の`Markdown`ファイルをレンダリングします。

`TOC`は`Table of Contents`の略で、このテンプレートがレンダリングされると、サイドバーにこの`Markdown`ファイルのアウトラインが表示されます。

`url:` `Markdown`のファイルパスを示します(`/`はルートディレクトリを指し、`/README.md`に対応します。このファイル名には大文字の接頭辞と小文字の接尾辞が必要です)。

### `use: Md`、アウトラインのない単一ファイルテンプレート

`Md`テンプレートと`Toc`テンプレートは同じで、どちらも単一の`Markdown`ファイルをレンダリングするために使用されますが、`Md`テンプレートはサイドバーにアウトラインを表示しません。

上記の設定を`use: Toc`から`use: Md`に変更し、`md`ディレクトリで`i18n.site`を再実行し、開発プレビューURLにアクセスしてホームページの変更を確認します。

### `use: Blog`、ブログテンプレート

ブログテンプレートは、公開時間順に記事のリスト(タイトルと要約)を表示します。

[→ 具体的な設定についてはここをクリックしてください](/i18n.site/conf/blog)

### `use: Doc`、複数ファイルのドキュメントテンプレート

設定ファイルでは:

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

`Doc`を使用してテンプレートをレンダリングすることを示します。

`Doc`テンプレートは、複数の`Markdown`を統合して、単一または複数のプロジェクトのドキュメントアウトラインを生成することをサポートします。

#### 複数プロジェクト・複数ファイル

`.i18n/conf.yml`の`i18n:doc`設定は、マルチプロジェクト・マルチファイルレンダリングモードです。

ここで、`menu: NB demo1,demo2`は、`NB`テンプレートを使用してドロップダウンメニューをレンダリングすることを意味します。

<img src="//p.3ti.site/1721275191.avif" width="320px">

`NB`は`Name Brief`の略で、ドロップダウンメニューにプロジェクトの名前とスローガンを表示できることを意味します。

`NB`の後にパラメータ`demo1,demo2`が渡されます。

注意:**`demo1,demo2`のカンマ`,`の前後にスペースを入れてはなりません**。

上記のパラメータに対応するディレクトリインデックスファイルは以下の通りです

* `en/demo1/TOC`
* `en/demo2/TOC`

#### 単一プロジェクト・複数ファイル

プロジェクトが1つしかない場合は、以下のように設定できます。

```
  - i18n: xxx
    use: Doc
```

##### URLが空の場合、デフォルト値はi18nに設定されます

`url`が書かれていない場合、デフォルトの`url`は`i18n`の値です。この規則は他のテンプレートにも適用されます。

上記の書き方と`url: xxx`は同等で、対応するファイルは`en/xxx/TOC`です。

#### TOCディレクトリインデックス

`i18n.site`は、デモリポジトリ内の`js`プラグイン`.i18n/hook/after.tran/TOC.js`を実行して、`doc`テンプレート設定に対応する`TOC`ディレクトリインデックスファイルを読み取り、`json`のディレクトリアウトラインを生成します。

`doc`テンプレートを使用する場合は、このプラグインが必要です。

空のフォルダーから`i18n.site`プロジェクトを初期化する場合は、デモプロジェクトの`.i18n`ディレクトリをコピーしてください。

`Doc`テンプレートは、生成された`json`に基づいてディレクトリアウトラインをレンダリングします。

##### 詳細な内容説明を以下に示します

`en/blog/TOC`の内容は以下の通りです:

```
README.md

news/README.md
  news/begin.md
```

##### インデントを使用してレベルを示します

上記の`en/blog/TOC`の最初の行の`README.md`は、下图の`i18n.site`に対応しており、これはプロジェクト名です。

次の2行は、以下のスクリーンショットに示されています。

<img src="https://p.3ti.site/1721097381.avif" style="width:320px">

`news/README.md`は`News`に対応し、
`news/begin.md`は`Our Product is Online !`に対応します

`TOC`ファイルはインデントを使用してディレクトリの階層関係を示し、複数レベルのインデントをサポートし、`#`で始まる行はコメントとしてサポートされます。

##### 親レベルではタイトルのみが書き込まれ、内容は書き込まれません

複数レベルのインデントがある場合、親レベルではタイトルのみが書き込まれ、内容は書き込まれません。そうしないと、レイアウトが崩れてしまいます。

##### プロジェクトのREADME.md

プロジェクトの`README.md`、例えば`en/demo2/README.md`には以下の内容を記述できます。

このファイルの内容には目次の概要が示されていないため、短い紹介文を書くことをお勧めします。

###### プロジェクトスローガン

`Deme Two`はドロップダウンメニューとカタログアウトラインのプロジェクト名の下に、プロジェクトのスローガン`Your Project slogan`が表示されます。

![](https://p.3ti.site/1721276842.avif)

これは`en/demo2/README.md`の最初の行に対応します:

```
# Demo Two : Your Project slogan
```

プロジェクトの`README.md`の1レベル目のタイトルの最初のコロン`:`の後の内容が、プロジェクトのスローガンと見なされます。

中国語、日本語、韓国語のユーザーは、全角コロンではなく半角コロン`:`を使用する必要があります。

##### TOCを一括で移動するにはどうすればよいですか？

`TOC`ファイルはソース言語のディレクトリに配置する必要があります。

例えば、ソース言語が中国語の場合、上記の`TOC`は`zh/blog/TOC`です。

ソース言語を変更した場合、プロジェクト内の特定言語の`TOC`ファイルを他の言語に一括移動する必要があります。

以下のコマンドを参照してください

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

上記のコマンドの`en/`と`zh/`を自分の言語コードに変更してください。

### 設定パスなしのデフォルトロード

特定のパスにアクセスされた場合、パスのプレフィックスが`nav:`に設定されていない場合、そのパスに対応する`Markdown`ファイルがデフォルトでロードされ、`Md`テンプレートを使用してレンダリングされます。

例えば、`/test`にアクセスされ、このパスのプレフィックスが`nav:`に設定されていない場合、現在の閲覧言語が英語(コード`en`)の場合、デフォルトで`/en/test.md`がロードされ、`Md`テンプレートを使用してレンダリングされます。

`/en/test.md`が存在しない場合、デフォルトの`404`ページが表示されます。
