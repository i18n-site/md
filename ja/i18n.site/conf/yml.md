# .i18n/conf.yml

`i18n.site`の設定ファイルは`.i18n/conf.yml`です。

[`i18`](/i18)の`ignore:`と`i18n:`の設定を除いた構成ファイルは以下の通りです

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

ここで、`upload`の`ext:`設定項目は、公開時に`.md`ファイルのみをアップロードすることを意味します。

## 顶部导航 ナビ

`nav:`の設定オプションは、ホームページの上部ナビゲーションメニューに対応します。

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

ここで、`i18n: home`は`en/i18n.yml`の`home: Home`に相当します。

`en/i18n.yml`は`zh/i18n.yml`など、複数の言語に翻訳されます。

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

翻訳が完了したら、翻訳文の`yml`の値を変更することはできますが、`yml`のキーを追加または削除しないでください。

### `use: Toc` 単一ファイルテンプレート (アウトライン付き)

`nav`の設定では:

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc`は、`Toc`テンプレートを使用してレンダリングすることを意味し、単一の`Markdown`テンプレートをレンダリングします。

`TOC`は`Table of Contents`の略で、このテンプレートがレンダリングされると、サイドバーにこの`Markdown`ファイルのアウトラインが表示されます。

`url:`は`Markdown`のファイルパスを示します(`/`はルートディレクトリを指し、`/README.md`に対応します。ファイル名には大文字の接頭辞と小文字の接尾辞が必要です)。

### `use: Md` 単一ファイルテンプレート (アウトラインなし)

`Md`テンプレートと`Toc`テンプレートは同一であり、どちらも単一の`Markdown`ファイルのレンダリングに用いられますが、`Md`テンプレートではサイドバーに目次が表示されません。

上記の設定で`use: Toc`を`use: Md`に変更し、`md`ディレクトリで`i18n.site`を再度実行し、開発プレビューのURLにアクセスしてホームページの変更を確認します。

### 無設定パスのデフォルトロード

特定のパスにアクセスし、そのパスのプレフィックスが`nav:`に設定されていない場合、そのパスに対応する`Markdown`ファイルがデフォルトでロードされ、`Md`テンプレートでレンダリングされます。

例えば、`/test`にアクセスし、`nav:`設定にこのパスが含まれていない場合、ページの言語が英語(コード`en`)の場合、デフォルトで`/en/test.md`がロードされ、`Md`テンプレートでレンダリングされます。

`/en/test.md`が存在しない場合、デフォルトの`404`ページが表示されます。

### `use: Doc` 複数ファイルテンプレート

構成ファイルでは:

```
  - i18n: blog
    use: Doc
```

`Doc`テンプレートを使用してレンダリングすることを示します。

`Doc`テンプレートは、複数の`Markdown`を統合して、単一または複数のプロジェクトのドキュメントアウトラインを生成することをサポートします。

#### 単一プロジェクト (複数ファイル)

前述の`blog`は`Doc`の単一プロジェクトモードです。

```
  - i18n: blog
    url: blog
    use: Doc
```

##### URLが空の場合、デフォルト値はi18nに設定されます

`url`が指定されていない場合、デフォルトの`url`は`i18n`の値です。この規則は他のテンプレートにも適用されます。

上記の設定は`url: blog`に相当し、対応するファイルは`en/blog/TOC`です。

#### 複数プロジェクト

`.i18n/conf.yml`の`i18n:doc`設定は複数プロジェクトモードです。

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

ここで、`menu: NB demo1,demo2`は`NB`テンプレートを使用してドロップダウンメニューをレンダリングすることを意味します。

`NB`は`Name Brief`の略で、ドロップダウンメニューにプロジェクトの名前とスローガンを表示できることを意味します。

`NB`の後にパラメータ`demo1,demo2`が渡されます。
注意:**`demo1,demo2`のカンマ`,`の前後にはスペースを入れないでください**。

上記のパラメータに対応するディレクトリインデックスファイルは以下の通りです

* `en/demo1/TOC`
* `en/demo2/TOC`

#### 目次索引

`i18n.site`はデモリポジトリ内の`js`プラグイン`.i18n/hook/after.tran/TOC.js`を実行して、`TOC`テンプレート構成に対応する`doc`ディレクトリインデックスファイルを読み取り、`json`形式の目次アウトラインを生成します。

`doc`テンプレートを使用する場合は、このプラグインが必要です。

空のフォルダーから`i18n.site`プロジェクトを初期化する場合は、デモプロジェクトの`.i18n`をディレクトリにコピーしてください。

`Doc`テンプレートは生成された`json`に基づいて目次のアウトラインをレンダリングします。

##### 内容詳細

`en/blog/TOC`の内容は以下の通りです:

```
README.md

news/README.md
  news/begin.md
```

##### インデントでレベルを示します

上記の`en/blog/TOC`の最初の行`README.md`は、下图の`i18n.site`に対応しており、プロジェクト名です。

次の2行は以下のスクリーンショットに示されています。

<img src="https://p.3ti.site/1721097381.avif" style="width:320px">

`news/README.md`は`News`に対応します
`news/begin.md`は`Our Product is Online !`に対応します

`TOC`ファイルはインデントを使用してアウトラインの階層関係を示し、複数のインデントをサポートします。

##### 親レベルではタイトルのみを記入し、内容は記入しないでください

複数のインデントレベルがある場合、親レベルではタイトルのみを記入し、内容は記入しないでください。そうしないと、レイアウトが崩れます。

##### プロジェクトのREADME.md

プロジェクトの`README.md`、例えば`en/demo2/README.md`には内容を記述できます。

このファイルの内容は目次の概要を示さないため、短い紹介文を書くことをお勧めします。

###### プロジェクトのスローガン

`Deme Two`はドロップダウンメニューとカタログアウトラインのプロジェクト名の下に、プロジェクトのスローガン`Your Project slogan`が表示されます。

![](https://p.3ti.site/1721276842.avif)

これは`en/demo2/README.md`の最初の行に対応します

```
# Demo Two : Your Project slogan
```

プロジェクトの`README.md`の1レベル目のタイトルの最初のコロン`:`の後の内容がプロジェクトのスローガンと見なされます。

中国、日本、韓国のユーザーは、全角コロンではなく半角コロン`:`を使用してください。

##### 目次を一括で移動する方法

`TOC`ファイルはソース言語のディレクトリに配置する必要があります。

例えば、ソース言語が中国語の場合、上記の`TOC`は`zh/blog/TOC`です。

ソース言語を変更した場合、プロジェクト内の特定の言語の`TOC`ファイルを別の言語に一括移動する必要があります。

以下のコマンドを参照してください

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

上記のコマンドの`en/`と`zh/`を言語コードに変更してください。
