## प्रयोग गर्नुहोस्

### डेमो परियोजना

`i18` कन्फिगरेसन जान्नको लागि कृपया डेमो प्रोजेक्टलाई सन्दर्भ गर्नुहोस् [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

चीनमा प्रयोगकर्ताहरूले क्लोन गर्न सक्छन् [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

क्लोनिङ पछि, निर्देशिका प्रविष्ट गर्नुहोस् र अनुवाद पूरा गर्न चलाउनुहोस् `i18`

### निर्देशिका संरचना

टेम्प्लेट गोदाम निर्देशिका संरचना निम्नानुसार छ

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` अनुवादित डेमो फाइलहरू समावेश गर्दछ, जुन केवल एक उदाहरण हो र मेटाउन सकिन्छ।

### अनुवाद चलाउनुहोस्

निर्देशिका प्रविष्ट गर्नुहोस् र अनुवाद गर्न चलाउनुहोस् `i18`

### भण्डारमा फाइलहरू थप्नुहोस्

अनुवादको अतिरिक्त, कार्यक्रमले निम्न फाइलहरू पनि उत्पन्न गर्नेछ, कृपया तिनीहरूलाई भण्डारमा थप्नुहोस्।

```
.i18n/hash
.i18n/cache/.gitignore
```

ती मध्ये, `.i18n/cache/.gitignore` सामग्री निम्नानुसार छ :

```
**/*
!**/.gitignore
```

यसको मतलब `.i18n/cache/` रहेका सबै फाइलहरूलाई बेवास्ता गर्नुहोस् ( `.i18n/cache/.gitignore` बाहेक)।

यदि तपाइँको संस्करण नियन्त्रण सफ्टवेयर छैन भने `git` कृपया यो कन्फिगरेसन अनुसार बेवास्ता गर्नुहोस्।

## कन्फिगरेसन फाइल

यो `i18` लाइन अनुवाद उपकरणको कन्फिगरेसन फाइल `.i18n/conf.yml`

सामग्री निम्नानुसार छ:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### स्रोत भाषा &

कन्फिगरेसन फाइलमा, को अधीनस्थहरू `fromTo`

स्रोत भाषा `en` , `zh ja ko de fr` अनुवादको लक्ष्य भाषा हो।

भाषा कोड हेर्नुहोस् [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

उदाहरणका लागि, यदि तपाइँ चिनियाँलाई अंग्रेजीमा अनुवाद गर्न चाहनुहुन्छ भने, यो रेखा पुन: लेख्नुहोस् `zh: en`

यदि तपाइँ सबै समर्थित भाषाहरूमा अनुवाद गर्न चाहनुहुन्छ भने, कृपया खाली छोड्नुहोस् `:` उदाहरण को लागी

```
i18n:
  fromTo:
    en:
```

### फाइल बेवास्ता गर्नुहोस्

पूर्वनिर्धारित रूपमा, स्रोत भाषा डाइरेक्टरीमा `.md` र `.yml` बाट सुरु हुने सबै फाइलहरू अनुवाद हुनेछन्।

यदि तपाइँ निश्चित फाइलहरूलाई बेवास्ता गर्न चाहनुहुन्छ र तिनीहरूलाई अनुवाद गर्नुहुन्न (जस्तै अधूरा ड्राफ्टहरू), तपाइँ फिल्ड कन्फिगरेसन प्रयोग गर्न सक्नुहुन्छ `ignore`

`.gitignore` जस्तै सिन्ट्याक्स प्रयोग [globset](https://docs.rs/globset/latest/globset/#syntax) `ignore`

उदाहरणका लागि, माथिको कन्फिगरेसन फाइलको अर्थ `_* ` `_` सुरु हुने फाइलहरू अनुवाद गरिने छैन।

## अनुवाद नियम

### अनुवाद सम्पादकहरूले रेखाहरू थप्नु वा मेट्नु हुँदैन

अनुवाद सम्पादन योग्य छ। मूल पाठ परिमार्जन गर्नुहोस् र मेसिन-यसलाई फेरि अनुवाद गर्नुहोस्, अनुवादमा म्यानुअल परिमार्जनहरू ओभरराइट हुने छैनन् (यदि मूल पाठको यो अनुच्छेद परिमार्जन गरिएको छैन)।

तर कृपया ध्यान दिनुहोस् कि अनुवादका रेखाहरू र मूल पाठ एक-एक अनुरूप हुनुपर्छ। त्यो हो, अनुवाद कम्पाइल गर्दा लाइनहरू थप्नुहोस् वा मेटाउनुहोस्। अन्यथा, यसले अनुवाद सम्पादन क्यासमा भ्रम पैदा गर्नेछ।

यदि केहि गलत भयो भने, कृपया [समाधानका लागि FAQ लाई सन्दर्भ गर्नुहोस्।](/i18/qa#H1)

### अनुवाद `YAML`

कमाण्ड लाइन उपकरणले स्रोत भाषा फाइल डाइरेक्टरीमा अन्त्य `.yml` सबै फाइलहरू फेला पार्नेछ र तिनीहरूलाई अनुवाद गर्नेछ।

* नोट गर्नुहोस् कि फाइल नाम प्रत्यय `.yml` ( `.yaml` होइन)।

उपकरणले केवल शब्दकोश मानहरू अनुवाद गर्दछ `.yml` शब्दकोश कुञ्जीहरू होइन।

उदाहरण को लागी `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

को रूपमा अनुवाद गरिनेछ `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` अनुवाद म्यानुअल रूपमा पनि परिमार्जन गर्न सकिन्छ (तर अनुवादमा कुञ्जी वा रेखाहरू थप्नुहोस् वा नमेट्नुहोस्)।

`YAML` आधारमा, तपाइँ सजिलैसँग विभिन्न प्रोग्रामिङ भाषाहरूको लागि अन्तर्राष्ट्रिय समाधानहरू निर्माण गर्न सक्नुहुन्छ।

## उन्नत उपयोग

### अनुवाद उपनिर्देशिका

जब सम्म तपाइँ सिर्जना `.i18n/conf.yml` (प्रत्येक पटक डेमो परियोजना टेम्प्लेटबाट सुरु गर्न आवश्यक छैन), `i18` राम्रो काम गर्नेछ।

कमाण्ड लाइन उपकरणले सबै उपनिर्देशिकाहरूमा `.i18n/conf.yml` फेला पार्नेछ र यसलाई अनुवाद गर्नेछ।

[monorepo](//monorepo.tools) प्रयोग गर्ने परियोजनाहरूले भाषा फाइलहरूलाई उपनिर्देशिकाहरूमा विभाजन गर्न सक्छन्।

<img alt="" src="https://p.3ti.site/1719910016.avif">

### अनुकूलन स्थापना निर्देशिका

यो पूर्वनिर्धारित रूपमा स्थापित हुनेछ `/usr/local/bin`

यदि लेख्ने अनुमति छैन `/usr/local/bin` यो `~/.bin` मा स्थापित हुनेछ।

वातावरण चर सेट गर्नुहोस् `TO` तपाईंले स्थापना डाइरेक्टरी परिभाषित गर्न सक्नुहुन्छ, उदाहरणका लागि :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
