# انسٹال کریں اور استعمال کریں۔

## ونڈوز پہلے گٹ باش انسٹال کریں۔

windows ، [پہلے ڈاؤن لوڈ اور انسٹال کرنے کے لیے یہاں کلک کریں `git bash`](https://git-scm.com/download/win)

میں بعد کی کارروائیوں کو چلائیں `git bash`

## انسٹال کریں۔

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### ترجمہ ٹوکن کنفیگر کریں۔

ٹوکن کاپی کرنے کے لیے کلک کریں [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

تخلیق کریں `~/.config/i18n.site.yml` اس میں کاپی شدہ مواد پیسٹ کریں، مواد درج ذیل ہے:

```
token: YOUR_API_TOKEN
```

اس کے علاوہ، آپ کو ادائیگی کے لیے ایک کریڈٹ کارڈ کا پابند [i18n.site/payBill](//i18n.site/payBill) ہوگا (کوئی ریچارج کی ضرورت نہیں ہے، ویب سائٹ خود بخود استعمال کے مطابق فیس کاٹ لے گی، [قیمت کے لیے ہوم پیج دیکھیں](/#price) )۔

## استعمال کریں

### ڈیمو پروجیکٹ

`i18` کی ترتیب سیکھنے کے لیے براہ کرم ڈیمو پروجیکٹ کو دیکھیں [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

چین میں صارفین کلون کر سکتے ہیں [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

کلوننگ کے بعد، ڈائرکٹری درج کریں اور ترجمہ مکمل کرنے کے لیے چلائیں `i18`

### ڈائرکٹری کا ڈھانچہ

ٹیمپلیٹ گودام ڈائریکٹری ڈھانچہ مندرجہ ذیل ہے۔

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` میں ترجمہ شدہ ڈیمو فائلیں ہیں، جو صرف ایک مثال ہیں اور حذف کی جا سکتی ہیں۔

### ترجمہ چلائیں۔

ڈائرکٹری درج کریں اور ترجمہ کرنے کے لیے چلائیں `i18`

### ریپوزٹری میں فائلیں شامل کریں۔

ترجمہ کے علاوہ، یہ پروگرام درج ذیل فائلیں بھی تیار کرے گا، براہ کرم انہیں ذخیرہ میں شامل کریں۔

```
.i18n/hash
.i18n/cache/.gitignore
```

ان میں سے `.i18n/cache/.gitignore` مندرجہ ذیل کے طور پر ہے :

```
**/*
!**/.gitignore
```

اس کا مطلب ہے کہ `.i18n/cache/` میں موجود تمام فائلوں کو نظر انداز کریں (سوائے `.i18n/cache/.gitignore` )۔

اگر آپ کا ورژن کنٹرول سافٹ ویئر نہیں ہے `git` براہ کرم اس ترتیب کے مطابق اسے نظر انداز کریں۔

## کنفیگریشن فائل

یہ `i18` لائن ٹرانسلیشن ٹول کی کنفیگریشن فائل ہے `.i18n/conf.yml`

مندرجہ ذیل مواد ہے:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ماخذ کی زبان &

کنفیگریشن فائل میں، کے ماتحت `fromTo`

ماخذ زبان ہے `en` `zh ja ko de fr` ترجمہ کی ہدف کی زبان ہے۔

زبان کا کوڈ دیکھیں [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

مثال کے طور پر، اگر آپ چینی کا انگریزی میں ترجمہ کرنا چاہتے ہیں، تو اس لائن کو دوبارہ لکھیں `zh: en`

اگر آپ تمام معاون زبانوں میں ترجمہ کرنا چاہتے ہیں، تو براہ کرم اس کے بعد خالی چھوڑ دیں `:` مثال کے طور پر

```
i18n:
  fromTo:
    en:
```

### فائل کو نظر انداز کریں

بطور ڈیفالٹ، سورس لینگویج ڈائرکٹری میں `.md` اور `.yml` سے شروع ہونے والی تمام فائلوں کا ترجمہ کیا جائے گا۔

اگر آپ کچھ فائلوں کو نظر انداز کرنا چاہتے ہیں اور ان کا ترجمہ نہیں کرنا چاہتے ہیں (جیسے نامکمل مسودے)، تو آپ فیلڈ کنفیگریشن استعمال کر سکتے ہیں `ignore`

`.gitignore` سے ملتا جلتا نحو استعمال کرتا [globset](https://docs.rs/globset/latest/globset/#syntax) `ignore`

مثال کے طور پر، اوپر کنفیگریشن فائل `_* ` مطلب ہے کہ `_` سے شروع ہونے والی فائلوں کا ترجمہ نہیں کیا جائے گا۔

## ترجمے کے قواعد

### ترجمہ ایڈیٹرز کو لائنوں کو شامل یا حذف نہیں کرنا چاہئے۔

ترجمہ قابل تدوین ہے۔ اصل متن میں ترمیم کریں اور مشین سے اس کا دوبارہ ترجمہ کریں، ترجمہ میں دستی ترمیم کو اوور رائٹ نہیں کیا جائے گا (اگر اصل متن کے اس پیراگراف میں ترمیم نہیں کی گئی ہے)۔

لیکن براہ کرم نوٹ کریں کہ ترجمہ اور اصل متن کی لائنیں ایک سے ایک کے مطابق ہونی چاہئیں۔ یعنی ترجمہ مرتب کرتے وقت سطریں شامل یا حذف نہ کریں۔ بصورت دیگر، یہ ترجمے کی تدوین کیش میں الجھن پیدا کرے گا۔

اگر کچھ غلط ہو جاتا ہے تو، براہ کرم [حل کے لیے اکثر پوچھے گئے سوالات سے رجوع کریں۔](/i18/qa#H1)

### ترجمہ `YAML`

کمانڈ لائن ٹول سورس لینگویج فائل ڈائرکٹری میں ختم ہونے `.yml` تمام فائلوں کو تلاش کرے گا اور ان کا ترجمہ کرے گا۔

* نوٹ کریں کہ فائل کے نام کا لاحقہ ہونا چاہیے ( `.yaml` نہیں) `.yml`

ٹول صرف لغت کی قدروں کا ترجمہ کرتا ہے `.yml` لغت کی کلیدوں کا نہیں۔

مثال کے طور پر `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

کے طور پر ترجمہ کیا جائے گا `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` ترجمہ کو دستی طور پر بھی تبدیل کیا جا سکتا ہے (لیکن ترجمے میں کلیدیں یا لائنیں شامل نہ کریں)۔

ترجمہ `YAML` بنیاد پر، آپ آسانی سے مختلف پروگرامنگ زبانوں کے لیے بین الاقوامی حل تیار کر سکتے ہیں۔

## اعلی درجے کا استعمال

### ترجمہ ذیلی ڈائرکٹری

جب تک آپ تخلیق کرتے `.i18n/conf.yml` (ہر بار ڈیمو پروجیکٹ ٹیمپلیٹ سے شروع کرنے کی ضرورت نہیں ہے)، `i18` ٹھیک کام کرے گا۔

کمانڈ لائن ٹول تمام ذیلی ڈائریکٹریوں میں `.i18n/conf.yml` تلاش کرے گا اور اس کا ترجمہ کرے گا۔

آرکیٹیکچر [monorepo](//monorepo.tools) استعمال کرنے والے پروجیکٹس زبان کی فائلوں کو ذیلی ڈائریکٹریوں میں تقسیم کر سکتے ہیں۔

<img alt="" src="https://p.3ti.site/1719910016.avif">

### کسٹم انسٹالیشن ڈائرکٹری

یہ ڈیفالٹ کے طور پر نصب کیا جائے گا `/usr/local/bin`

اگر لکھنے کی اجازت نہیں ہے `/usr/local/bin` اسے `~/.bin` پر انسٹال کر دیا جائے گا۔

ماحول کے متغیرات کو سیٹ `TO` آپ انسٹالیشن ڈائرکٹری کی وضاحت کر سکتے ہیں، مثال کے طور پر :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
