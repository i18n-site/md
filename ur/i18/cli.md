# کمانڈ لائن پیرامیٹرز کی تفصیلی وضاحت

## فائلوں کو صاف کریں `-p`

`-p` `--purge` ان فائلوں کو صاف کرے گا جو ہر ترجمہ ڈائرکٹری میں موجود ہیں لیکن ماخذ زبان کی ڈائرکٹری میں موجود نہیں ہیں۔

کیونکہ دستاویزات لکھتے وقت، مارک ڈاؤن فائل کے ناموں کو اکثر ایڈجسٹ کیا جاتا ہے، جس کے نتیجے میں ترجمے کی ڈائرکٹری میں بہت سی پرانی اور لاوارث فائلیں ہوتی ہیں۔

اس پیرامیٹر کو ان فائلوں کو صاف کرنے کے لیے استعمال کریں جنہیں دوسری زبان کی ڈائریکٹریوں میں حذف کیا جانا چاہیے۔

## ترجمہ ڈائریکٹری کی وضاحت کریں `-d`

ترجمہ شدہ ڈائرکٹری پہلے سے طے شدہ ڈائرکٹری میں ہے جہاں موجودہ فائل واقع ہے۔

`-d` `--workdir` ترجمہ ڈائریکٹری کی وضاحت کر سکتا ہے، جیسے:

```
i18 -d ~/i18n/md
```

## مدد `-h`

`-h` `--help` کمانڈ لائن مدد دیکھ سکتے ہیں۔
