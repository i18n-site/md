चीनदेशे उपयोक्तारः क्लोन् कर्तुं शक्नुवन्ति [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

क्लोनिङ्ग् कृत्वा अनुवादं पूर्णं कर्तुं निर्देशिकां प्रविश्य चालयन्तु `i18`

### निर्देशिका संरचना

टेम्पलेट् गोदामनिर्देशिकासंरचना निम्नलिखितरूपेण अस्ति

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` निर्देशिकायां अनुवादिताः डेमो सञ्चिकाः सन्ति, ये केवलं उदाहरणं भवन्ति, तेषां विलोपनं कर्तुं शक्यते ।

### अनुवादं चालयन्तु

निर्देशिकां प्रविश्य अनुवादं कर्तुं `i18`

### भण्डारे सञ्चिकाः योजयन्तु

अनुवादस्य अतिरिक्तं कार्यक्रमः निम्नलिखितसञ्चिकाः अपि जनयिष्यति, कृपया तान् भण्डारे योजयन्तु ।

```
.i18n/hash
.i18n/cache/.gitignore
```

: `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

अस्य अर्थः अस्ति यत् `.i18n/cache/` निर्देशिकायां सर्वाणि सञ्चिकाः अवहेलयन्तु ( `.i18n/cache/.gitignore` विहाय ).

यदि भवतः संस्करणनियन्त्रणसॉफ्टवेयरं नास्ति `git` तर्हि कृपया अस्य विन्यासस्य अनुसारं तस्य अवहेलनां कुर्वन्तु ।

## विन्याससञ्चिका

`.i18n/conf.yml` `i18`

सामग्री यथा- १.

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### स्रोत भाषा अनुवाद भाषा &

विन्याससञ्चिकायां `fromTo`

`en` `zh ja ko de fr`

भाषासंहिता पश्यन्तु [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

यथा, यदि भवान् चीनीभाषायाः आङ्ग्लभाषायां अनुवादं कर्तुम् इच्छति तर्हि एतां पङ्क्तिं पुनः लिखतु `zh: en`

यदि भवान् सर्वासु समर्थितभाषासु अनुवादं कर्तुम् इच्छति तर्हि कृपया `:` उदाहरणतया

```
i18n:
  fromTo:
    en:
```

### सञ्चिकां अवहेलयन्तु

पूर्वनिर्धारितरूपेण, स्रोतभाषानिर्देशिकायां `.md` , `.yml` इत्यनेन आरभ्यमाणाः सर्वाणि सञ्चिकाः अनुवादिताः भविष्यन्ति ।

यदि भवान् कतिपयानां सञ्चिकानां अवहेलनां कर्तुम् इच्छति तथा च तान् अनुवादयितुम् इच्छति (यथा असमाप्तमसौदा), तर्हि भवान् `ignore`

`.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax) सदृशं वाक्यविन्यासं प्रयुङ्क्ते `ignore`

यथा, उपरिष्टाद् विन्याससञ्चिकायां `_* ` इत्यस्य अर्थः अस्ति यत् `_` इत्यनेन आरभ्यमाणाः सञ्चिकाः अनुवादिताः न भविष्यन्ति ।

## अनुवाद नियमाः

### अनुवादसम्पादकाः पङ्क्तयः न योजयेयुः, न लोपयेयुः

अनुवादः सम्पादनीयः अस्ति। मूलपाठं परिवर्त्य पुनः यन्त्रानुवादं कुर्वन्तु, अनुवादस्य हस्तचलितसंशोधनं अधिलिखितं न भविष्यति (यदि मूलपाठस्य एषः अनुच्छेदः परिवर्तितः नास्ति)

परन्तु अनुवादस्य मूलग्रन्थस्य च पङ्क्तयः एकैकस्य अनुरूपं भवितुमर्हति इति ज्ञातव्यम् । अर्थात् अनुवादसंकलनकाले पङ्क्तयः न योजयन्तु, न लोपयन्तु वा। अन्यथा अनुवादसम्पादनसञ्चये भ्रमः उत्पद्यते ।

यदि किमपि भ्रष्टं भवति तर्हि [समाधानार्थं कृपया FAQ पश्यन्तु ।](/i18/qa#H1)

### `Yaml` अनुवदति

आदेशपङ्क्तिसाधनं स्रोतभाषासञ्चिकानिर्देशिकायां `.yml`

* ध्यानं कुर्वन्तु यत् सञ्चिकानामप्रत्ययः `.yml` ( `.yaml` न ) भवितुमर्हति ।

साधनं केवलं शब्दकोशमूल्यानां अनुवादं करोति `.yml`

उदाहरणतया `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

इति अनुवादः भविष्यति `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML`

`YAML` आधारेण भवान् विविधप्रोग्रामिंगभाषायाः अन्तर्राष्ट्रीयसमाधानं सहजतया निर्मातुम् अर्हति ।
