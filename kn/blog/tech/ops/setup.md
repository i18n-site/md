# ಸರ್ವರ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸಿ (BTRFS ವಿಭಾಗವನ್ನು ಬಳಸಿ, ಸ್ನ್ಯಾಪರ್ ನಿಗದಿತ ಸ್ನ್ಯಾಪ್‌ಶಾಟ್)

## contabo ಖರೀದಿ ಸರ್ವರ್

contabo ದೀರ್ಘ ಇತಿಹಾಸದೊಂದಿಗೆ (20 ವರ್ಷಗಳಿಗಿಂತ ಹೆಚ್ಚು) ಮತ್ತು ಅನೇಕ ಬಳಕೆದಾರರು (200,000 ಕ್ಕಿಂತ ಹೆಚ್ಚು) ಜರ್ಮನಿಯಲ್ಲಿ ಅಗ್ಗದ VPS ಪೂರೈಕೆದಾರರಾಗಿದ್ದಾರೆ, ಆದ್ದರಿಂದ ಓಡಿಹೋಗುವ ಬಗ್ಗೆ ಚಿಂತಿಸಬೇಕಾಗಿಲ್ಲ.

VPS [contabo.com](//contabo.com) ಯುರೋ ಬೆಲೆಯನ್ನು ಆಯ್ಕೆ ಮಾಡಲು ಮರೆಯದಿರಿ, ಅದು ಅಗ್ಗವಾಗಿದೆ.

<img alt="" src="https://i-01.eu.org/1718285782.avif">

ಕಾಂಟಾಬೊಗೆ ಒಂದು-ಬಾರಿ ಅನುಸ್ಥಾಪನಾ ಶುಲ್ಕವಿದೆ.
ಆದಾಗ್ಯೂ, ಪ್ರತಿ ತಿಂಗಳು ಕೆಲವು ದಿನಗಳು ಅನುಸ್ಥಾಪನಾ ಶುಲ್ಕವು ಉಚಿತವಾಗಿದೆ ಮತ್ತು ಪ್ರಚಾರದ ಇಮೇಲ್‌ಗಳಿಗೆ ಗಮನ ಕೊಡಿ.

ಯುರೋಪ್ನಲ್ಲಿ ಉಚಿತ ಪ್ರದೇಶದ ಶುಲ್ಕದ ಜೊತೆಗೆ, ಇತರ ಪ್ರದೇಶಗಳು ಬೆಲೆ ಹೆಚ್ಚಳದ ವಿವಿಧ ಹಂತಗಳನ್ನು ಹೊಂದಿವೆ.

ನಾನು ಯುರೋಪಿಯನ್ ಸರ್ವರ್ ಅನ್ನು ಆಯ್ಕೆ ಮಾಡಿದೆ, ಕ್ಲೌಡ್‌ಫ್ಲೇರ್ ಅನ್ನು ಬಳಸಿದ ನಂತರ, ಸರ್ವರ್‌ಗಳ ನಡುವೆ ಹೆಚ್ಚಿನ ವ್ಯತ್ಯಾಸವಿಲ್ಲ.

ಅಂತಿಮ ಸಂರಚನೆಯು ಈ ಕೆಳಗಿನಂತಿರುತ್ತದೆ, 16GB ಮೆಮೊರಿ, 6CPU, 200 GB NVMe, ತಿಂಗಳಿಗೆ 9.50 ಯುರೋಗಳು, 74.43 RMB ಗೆ ಸಮನಾಗಿರುತ್ತದೆ.

<img alt="" src="https://i-01.eu.org/1718286008.avif">

**ಈ ಬೆಲೆಯು ಅಲಿಬಾಬಾ ಕ್ಲೌಡ್ ಮತ್ತು ಅಮೆಜಾನ್ ಬೆಲೆಯ ಹತ್ತನೇ ಒಂದು ಭಾಗವಾಗಿದೆ** .

<img alt="" src="https://i-01.eu.org/1718356956.avif">

ಕಾರ್ಯಕ್ಷಮತೆ ಮತ್ತು ನಿಯತಾಂಕಗಳನ್ನು ಚಲಾಯಿಸಲು ನಾನು [Yet-Another-Bench-Script](https://github.com/masonr/yet-another-bench-script) :

```
curl -sL yabs.sh | bash -s -- -s https://www.vpsbenchmarks.com/yabs/upload
```

ಡಿಸ್‌ಪ್ಲೇ CPU `AMD EPYC 9224 24-Core Processor` ಸಿಂಗಲ್-ಕೋರ್ ಕಾರ್ಯಕ್ಷಮತೆಯ ಸ್ಕೋರ್ 1285
, ಮಲ್ಟಿ-ಕೋರ್ ಸ್ಕೋರ್ 4277 ಹೆಚ್ಚಿನ ಫಲಿತಾಂಶಗಳನ್ನು ನೋಡಿ [vpsbenchmarks.com](https://www.vpsbenchmarks.com/yabs/contabo-6c-16gb-3fb7d8)

ನಾನು ಒಂದೇ ಬಾರಿಗೆ ಮೂರು ಸರ್ವರ್‌ಗಳನ್ನು ಖರೀದಿಸಿದೆ ಮತ್ತು ಅದೇ ಸಮಯದಲ್ಲಿ ಅವುಗಳನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿದೆ, ಇದು ಸರ್ವರ್‌ಗಳು ಒಂದೇ ಕಂಪ್ಯೂಟರ್ ಕೋಣೆಯಲ್ಲಿದೆ ಎಂದು ಖಚಿತಪಡಿಸುತ್ತದೆ, ಇದು ಭವಿಷ್ಯದಲ್ಲಿ ಹೆಚ್ಚಿನ ಲಭ್ಯತೆಯನ್ನು ನಿರ್ಮಿಸಲು ಸುಲಭವಾಗುತ್ತದೆ.

ಇದರ ಸರ್ವರ್‌ಗಳು ಸಾಂದರ್ಭಿಕವಾಗಿ ಸ್ಥಗಿತಗೊಳ್ಳುತ್ತವೆ, ಹೆಚ್ಚಿನ ಲಭ್ಯತೆಯ ಮೂಲಕ ವೆಬ್‌ಸೈಟ್ ಪ್ರವೇಶವನ್ನು ಖಾತ್ರಿಪಡಿಸುತ್ತದೆ.

ಹೊಸ ಬಳಕೆದಾರರು ಖರೀದಿಸುವಾಗ ನೈಜ-ಹೆಸರಿನ ದೃಢೀಕರಣಕ್ಕಾಗಿ ತಮ್ಮ ID ಕಾರ್ಡ್‌ಗಳನ್ನು ಸಲ್ಲಿಸಬೇಕಾಗುತ್ತದೆ.

## ಪಾಸ್ವರ್ಡ್-ಮುಕ್ತ ಲಾಗಿನ್ ಅನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ

ಕೆಲವು ನಿಮಿಷಗಳ ನಂತರ, ಸರ್ವರ್‌ಗಳು ಎಲ್ಲಾ ಸಕ್ರಿಯಗೊಂಡವು.

ಮೊದಲ ವಿಷಯವೆಂದರೆ ಸ್ಥಳೀಯ ಯಂತ್ರವನ್ನು ಸಂಪಾದಿಸುವುದು `~/.ssh/config` ಗಾಗಿ ಅಲಿಯಾಸ್ ಅನ್ನು ಹೊಂದಿಸಿ IP ತದನಂತರ ಪಾಸ್‌ವರ್ಡ್-ಮುಕ್ತ ಲಾಗಿನ್ ಅನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಲು `ssh-copy-id root@c2` ಬಳಸಿ.

<img alt="" src="https://i-01.eu.org/1718287198.avif">

ಪ್ರಾರಂಭಿಕ ಸಂರಚನೆಯನ್ನು ವೇಗವಾಗಿ ಪೂರ್ಣಗೊಳಿಸಲು ನೀವು ಈ [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) ಮಾರ್ಪಡಿಸಬಹುದು.

ಪಾಸ್ವರ್ಡ್-ಮುಕ್ತ ಸಂರಚನೆಯನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದ ನಂತರ, ನೀವು ಅದನ್ನು ಬ್ಯಾಚ್ಗಳಲ್ಲಿ ನಿರ್ವಹಿಸಬಹುದು, ಇದು ಅನುಕೂಲಕರವಾಗಿರುತ್ತದೆ [pdsh](https://github.com/chaos/pdsh)

ಮೊದಲನೆಯದು ipv6 ಮತ್ತು ಪರೀಕ್ಷೆಯನ್ನು ಹಸ್ತಚಾಲಿತವಾಗಿ ಸಕ್ರಿಯಗೊಳಿಸುವುದು

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

<img alt="" src="https://i-01.eu.org/1718287969.avif">

ನಂತರ ಒಂದೇ ಕ್ಲಿಕ್‌ನಲ್ಲಿ ಸಿಸ್ಟಮ್ ಅನ್ನು ಉಬುಂಟು 24 ಗೆ ಅಪ್‌ಗ್ರೇಡ್ ಮಾಡಿ ಮತ್ತು ಸಾಮಾನ್ಯವಾಗಿ ಬಳಸುವ ಸಾಫ್ಟ್‌ವೇರ್ ಅನ್ನು ಸ್ಥಾಪಿಸಿ.

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 ಎಂದರೆ ಭಾಷೆಯನ್ನು ಚೈನೀಸ್‌ಗೆ ಮತ್ತು ಸಮಯ ವಲಯವನ್ನು ಬೀಜಿಂಗ್‌ಗೆ ಬದಲಾಯಿಸುವುದು ಎಂದರ್ಥ.

ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ ಸ್ಥಾಪಿಸಲಾದ ಸಾಫ್ಟ್‌ವೇರ್ [direnv](https://direnv.net/) ನನ್ನ [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) ಬಳಕೆಯ ಅಭ್ಯಾಸಗಳಿಗೆ ಅನುಗುಣವಾಗಿ ಕಾನ್ಫಿಗರ್ ಮಾಡಲಾಗಿದೆ, ಉದಾಹರಣೆಗೆ [mise](https://github.com/jdx/mise)

ಈ ಹಂತವು ಅರ್ಧ ಘಂಟೆಯವರೆಗೆ ಕಾಯುವ ನಂತರ, ಸಾಮಾನ್ಯವಾಗಿ ಬಳಸುವ ಸಾಫ್ಟ್‌ವೇರ್ ಅನ್ನು ನವೀಕರಿಸುವುದು ಮತ್ತು ಸ್ಥಾಪಿಸುವುದು ಯಶಸ್ವಿಯಾಗುತ್ತದೆ. ಕೆಳಗಿನ ಆಜ್ಞೆಯೊಂದಿಗೆ ನೀವು ಅದನ್ನು ದೃಢೀಕರಿಸಬಹುದು:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## mosh ssh ಅನ್ನು ವೇಗಗೊಳಿಸುತ್ತದೆ

ಯುರೋಪ್‌ನಿಂದ ಚೀನಾಕ್ಕೆ ನೆಟ್‌ವರ್ಕ್ ಸ್ಥಿರವಾಗಿಲ್ಲ `ssh` ಇದನ್ನು ಬಳಸಲು ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ [mosh](//mosh.org)

ಸ್ಥಳೀಯ `~/.bin` (ಮುಖ್ಯವಾಗಿ ಪರಿಸರ ವೇರಿಯಬಲ್ `PATH` ಗೆ ಸೇರಿಸಲಾಗಿದೆ) ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ `c0` ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು ಈ ಕೆಳಗಿನಂತೆ ರಚಿಸಿ:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

ರಚನೆಯ ನಂತರ, ಕಾರ್ಯಗತಗೊಳಿಸಬಹುದಾದ ಅನುಮತಿಗಳು ಮತ್ತು ಸಾಫ್ಟ್-ಲಿಂಕ್ , `c2` `c1`

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

ಭವಿಷ್ಯದಲ್ಲಿ, ನೀವು ನೇರವಾಗಿ ನಮೂದಿಸಬಹುದು `c0` `c1` ಮತ್ತು ನೀವು `mosh` ಹೆಚ್ಚಿನ ವೇಗದ ನೇರ ಸಂಪರ್ಕವನ್ನು ಬಳಸಬಹುದು ಮತ್ತು ನಮೂದಿಸಬಹುದು [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

## ವಿಭಜನಾ ಸ್ವರೂಪವನ್ನು ext4 ನಿಂದ btrfs ಗೆ ಪರಿವರ್ತಿಸಿ

ಸುಲಭ ಬ್ಯಾಕಪ್‌ಗಾಗಿ ಡಿಸ್ಕ್ ಸ್ನ್ಯಾಪ್‌ಶಾಟ್‌ಗಳನ್ನು ತೆಗೆದುಕೊಳ್ಳಬಹುದು [btrfs](https://wikipedia.org/wiki/Btrfs)

ಮೊದಲು ಪಾರುಗಾಣಿಕಾ ವ್ಯವಸ್ಥೆಯನ್ನು ನಮೂದಿಸಿ

<img alt="" src="https://i-01.eu.org/1718300448.avif">

ಡೆಬಿಯನ್ ಪಾರುಗಾಣಿಕಾ ಆಯ್ಕೆಮಾಡಿ

<img alt="" src="https://i-01.eu.org/1718300753.avif">

ಸರ್ವರ್ ಮರುಪ್ರಾರಂಭಿಸಲು ನಿರೀಕ್ಷಿಸಿ

ಫಿಂಗರ್‌ಪ್ರಿಂಟ್ ಬದಲಾವಣೆಗಳು ಸಂಭವಿಸಿದಾಗ ssh ಅದನ್ನು ಸ್ವಚ್ಛಗೊಳಿಸಲು `ssh-keygen -R ip`

ನಂತರ ವಿಭಾಗವನ್ನು ಪರಿವರ್ತಿಸಲು ಕೆಳಗಿನ ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು ರನ್ ಮಾಡಿ

```bash
nohup bash -c "bash <(curl -s https://raw.githubusercontent.com/i18n-ops/ops/main/contabo/ext4_btrfs.sh) >/tmp/nohup.out 2>&1 && reboot" &
sleep 1 && tail -f /tmp/nohup.out
```

ಪರಿವರ್ತನೆ ಯಶಸ್ವಿಯಾದ ನಂತರ ಸ್ಕ್ರಿಪ್ಟ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸಿಸ್ಟಮ್ ಅನ್ನು ಮರುಪ್ರಾರಂಭಿಸುತ್ತದೆ.

### btrfs ಕಂಪ್ರೆಷನ್ ಅನುಪಾತವನ್ನು ವೀಕ್ಷಿಸಿ

ಇದು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ ಸಂಕುಚಿತಗೊಳ್ಳುತ್ತದೆ ಸಂಕೋಚನ ಅನುಪಾತವನ್ನು ಪರಿಶೀಲಿಸುವ ಆಜ್ಞೆಯು ಈ ಕೆಳಗಿನಂತಿರುತ್ತದೆ

```bash
if ! command -v compsize &>/dev/null; then
apt install -y  btrfs-compsize
fi
compsize -x /
```

### ಹಸ್ತಚಾಲಿತ ಸಂಕೋಚನ

ನೀವು ಪರಿಪೂರ್ಣತಾವಾದಿಯಾಗಿದ್ದರೆ, ನೀವು ಸಂಪೂರ್ಣ ಡಿಸ್ಕ್ ಕಂಪ್ರೆಷನ್ ಅನ್ನು ಹಸ್ತಚಾಲಿತವಾಗಿ ನಿರ್ವಹಿಸಬಹುದು

```bash
btrfs filesystem defragment -r -czstd /
```

ಕೆಳಗಿನ ಚಿತ್ರದಲ್ಲಿ ತೋರಿಸಿರುವಂತೆ, ಹೊಸ ವ್ಯವಸ್ಥೆಯಲ್ಲಿ ಹಸ್ತಚಾಲಿತ ಸಂಕೋಚನದ ನಂತರ, ಸುಮಾರು 100MB ಹೆಚ್ಚು ಸಂಕುಚಿತಗೊಂಡಿದೆ ಎಂದು ನೀವು ನೋಡಬಹುದು.

### ಸ್ನ್ಯಾಪರ್‌ನೊಂದಿಗೆ Btrfs ಸ್ನ್ಯಾಪ್‌ಶಾಟ್ ಬ್ಯಾಕಪ್ ಅನ್ನು ರಚಿಸಿ

[snapper](https://manual.siduction.org/sys-admin-btrfs-snapper_en.html) Btrfs ಬ್ಯಾಕಪ್‌ಗೆ ಅತ್ಯುತ್ತಮ ಪಾಲುದಾರ.

ಪ್ರಾರಂಭಿಕ ಸ್ಕ್ರಿಪ್ಟ್ `~/ubuntu` ರಚಿಸುತ್ತದೆ, ಈ ಕೆಳಗಿನ ಆಜ್ಞೆಯನ್ನು ಚಲಾಯಿಸಿ:

```
~/ubuntu/snapper/init.sh
```

ಸ್ಕ್ರಿಪ್ಟ್ ವಿಷಯದಲ್ಲಿ ನಿಮ್ಮನ್ನು ಭೇಟಿ ಮಾಡುತ್ತೇವೆ [github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh](https://github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh)

ಸ್ನ್ಯಾಪ್‌ಶಾಟ್‌ಗಳನ್ನು ಹೊಂದಿಸಿರುವ ಉಪಸಂಪುಟಗಳನ್ನು ವೀಕ್ಷಿಸಲು `snapper list-configs`

<img alt="" src="https://i-01.eu.org/1718302689.avif">

ಉಪಸಂಪುಟದ ಸ್ನ್ಯಾಪ್‌ಶಾಟ್ ವೀಕ್ಷಿಸಲು `snapper -c mnt list`

`/etc/snapper/configs/mnt` ಈ ಉಪಸಂಪುಟದ ನಿರ್ದಿಷ್ಟ ಬ್ಯಾಕಪ್ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ನೀವು ವೀಕ್ಷಿಸಬಹುದು

ಹೆಚ್ಚಿನ ಉಲ್ಲೇಖಗಳು:

## contabo ಸರ್ವರ್ ಸ್ನ್ಯಾಪ್‌ಶಾಟ್

## ಪಾರುಗಾಣಿಕಾ ವ್ಯವಸ್ಥೆಯ ವಿವರಗಳು

ಪಾರುಗಾಣಿಕಾ 22 ನಮೂದಿಸಿ ಪೋರ್ಟ್ [tcping](https://github.com/paradiseduo/tcping) ಪ್ರವೇಶಿಸಬಹುದಾದ ನಂತರ, ನೀವು 2-3 ನಿಮಿಷಗಳವರೆಗೆ ಅದನ್ನು ಪ್ರವೇಶಿಸಲು ಸಾಧ್ಯವಾಗುವುದಿಲ್ಲ ಮತ್ತು ಅದು ಉತ್ತಮವಾಗಿರುತ್ತದೆ.

```
tcping ip 22 -c 9999
```

ಪೋರ್ಟ್ ಪ್ರವೇಶವನ್ನು ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಬಹುದು.

### & ಸ್ನ್ಯಾಪ್‌ಶಾಟ್ ಅನ್ನು ನಮೂದಿಸಿದ ನಂತರ ಹಾರ್ಡ್ ಡ್ರೈವ್ ಅನ್ನು ಹೇಗೆ ಪ್ರವೇಶಿಸುವುದು

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

ನೀವು ಸ್ನ್ಯಾಪ್‌ಶಾಟ್ ಅನ್ನು ವೀಕ್ಷಿಸಬಹುದು, ಉದಾಹರಣೆಗೆ

```
snapper -c etc list
```

ಸ್ನ್ಯಾಪ್‌ಶಾಟ್ ಫೈಲ್ ಬದಲಾವಣೆಗಳನ್ನು ವೀಕ್ಷಿಸಿ

```
snapper -c etc status 5..6
```

ನಿರ್ದಿಷ್ಟ ವ್ಯತ್ಯಾಸ ಪಠ್ಯವನ್ನು ವೀಕ್ಷಿಸಿ

```
snapper -c etc diff 3..5
```

ಸ್ನ್ಯಾಪ್‌ಶಾಟ್ ಅನ್ನು ಮರುಸ್ಥಾಪಿಸಿ: ಉದಾಹರಣೆಗೆ, ಸ್ನ್ಯಾಪ್‌ಶಾಟ್ 5 ರ ನಂತರ ಎಲ್ಲಾ ಬದಲಾವಣೆಗಳನ್ನು ರದ್ದುಗೊಳಿಸಿ

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### ಲಾಗ್ ನಿರಂತರತೆಯನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ

ಪಾರುಗಾಣಿಕಾ ವ್ಯವಸ್ಥೆಯ ಅಡಿಯಲ್ಲಿ ಲಾಗ್ ಪರ್ಸಿಸ್ಟೆನ್ಸ್ ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲು ನೀವು ಅನುಗುಣವಾದ ಡೈರೆಕ್ಟರಿಯನ್ನು ಹಸ್ತಚಾಲಿತವಾಗಿ ರಚಿಸುವ ಅಗತ್ಯವಿದೆ ಮತ್ತು ಕೆಲವು ಹೆಚ್ಚುವರಿ ಹಂತಗಳನ್ನು ನಿರ್ವಹಿಸಬೇಕಾಗುತ್ತದೆ, ಏಕೆಂದರೆ ನೀವು ಚಾಲನೆಯಲ್ಲಿರುವ systemd ಸೇವೆಯನ್ನು ಹೊಂದಿಲ್ಲದಿರಬಹುದು. ನೀವು ಪ್ರಯತ್ನಿಸಬಹುದಾದ ಹಂತಗಳು ಇಲ್ಲಿವೆ:

ಮೊದಲು ಕ್ರೂಟ್, ನಂತರ

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

ನಿಮ್ಮ ಉಬುಂಟು ಸಿಸ್ಟಮ್‌ಗೆ ರೀಬೂಟ್ ಮಾಡಿ: ಮೇಲಿನ ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡಿದ ನಂತರ, ನೀವು ನಿಮ್ಮ ಗಣಕವನ್ನು ಸಾಮಾನ್ಯವಾಗಿ ಮತ್ತು ಪಾರುಗಾಣಿಕಾ ವ್ಯವಸ್ಥೆಯ ಬದಲಿಗೆ ನಿಮ್ಮ ಉಬುಂಟು ಸಿಸ್ಟಮ್‌ಗೆ ರೀಬೂಟ್ ಮಾಡಬೇಕಾಗುತ್ತದೆ.

ಈ ರೀತಿಯಾಗಿ, systemd-journald ಹೊಸ ಸಂರಚನೆಯೊಂದಿಗೆ ಪ್ರಾರಂಭವಾಗುತ್ತದೆ ಮತ್ತು ಹಾರ್ಡ್ ಡಿಸ್ಕ್‌ನಲ್ಲಿ ಲಾಗ್‌ಗಳನ್ನು ನಿರಂತರವಾಗಿ ಸಂಗ್ರಹಿಸಲು ಪ್ರಾರಂಭಿಸುತ್ತದೆ.

ಮೇಲಿನ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು ಪಾರುಗಾಣಿಕಾ ವ್ಯವಸ್ಥೆಯಲ್ಲಿ ನಿರ್ವಹಿಸಲಾಗುತ್ತದೆ ಎಂಬುದನ್ನು ನೆನಪಿನಲ್ಲಿಡಿ, ಆದ್ದರಿಂದ ಸಿಸ್ಟಮ್ ಅನ್ನು ರೀಬೂಟ್ ಮಾಡಿದ ನಂತರ ಮಾತ್ರ ಆ ಬದಲಾವಣೆಗಳು ಕಾರ್ಯಗತಗೊಳ್ಳುತ್ತವೆ ಮತ್ತು ಇನ್ನು ಮುಂದೆ ಪಾರುಗಾಣಿಕಾ ಕ್ರಮದಲ್ಲಿ ಇರುವುದಿಲ್ಲ.

ಇದರರ್ಥ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಲಾಗ್‌ಗಳಿಗೆ ಯಾವುದೇ ಬದಲಾವಣೆಗಳು ನಿರಂತರವಾಗುವುದನ್ನು ನೀವು ತಕ್ಷಣ ನೋಡುವುದಿಲ್ಲ, ಆದರೆ ಇದರ ನಂತರ ರಚಿಸಲಾದ ಎಲ್ಲಾ ಲಾಗ್‌ಗಳು ನಿರಂತರವಾಗಿರುತ್ತವೆ.

ಈ ರೀತಿಯಲ್ಲಿ ನೀವು ssh ಸೇವೆಯ ಲಾಗ್‌ಗಳನ್ನು ವೀಕ್ಷಿಸಬಹುದು

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## ಕಷ್ಟಕರ ಮತ್ತು ಸಂಕೀರ್ಣ ರೋಗಗಳ ಟಿಪ್ಪಣಿಗಳು

### ವಿಲಕ್ಷಣ ಸಮಸ್ಯೆಯನ್ನು ಪರಿಹರಿಸಿ: ಸರ್ವರ್ ssh ಸಾಧ್ಯವಿಲ್ಲ

ನಾನು ಮೊದಲು ಸ್ವಯಂಚಾಲಿತ ಸಂಕಲನ ಮತ್ತು ನಿಯೋಜನೆ ಸ್ಕ್ರಿಪ್ಟ್ ಮಾಡಲು ಗಿಥಬ್ ಕ್ರಿಯೆಯನ್ನು ಬಳಸಿದ್ದೇನೆ, ಆದರೆ ನಿಯೋಜನೆ ಸೇವೆಯು ssh ಸಾಧ್ಯವಿಲ್ಲ ಎಂದು ಕಂಡುಕೊಂಡೆ.

ಪಾರುಗಾಣಿಕಾ ವ್ಯವಸ್ಥೆಯನ್ನು ಪ್ರವೇಶಿಸುವುದು ಮತ್ತು ಲಾಗ್‌ಗಳನ್ನು ನೋಡುವುದು ಅನುಮತಿ ಸಮಸ್ಯೆಯಾಗಿದೆ:

`Missing privilege separation directory: /run/sshd`

ಕೆಳಗಿನ ಆಜ್ಞೆಯೊಂದಿಗೆ ನೀವು ಈ ಡೈರೆಕ್ಟರಿಯನ್ನು ರಚಿಸಬಹುದು

```
chown root:root /
systemd-tmpfiles --create
```

ಪುನರಾವರ್ತಿತ ತನಿಖೆಯ ನಂತರ, tar `uid` `1001` ಬಳಕೆದಾರರು github action

tar ಅನ್ಪ್ಯಾಕ್ ಮಾಡುವುದನ್ನು ನಿಯತಾಂಕಗಳಿಲ್ಲದೆ ಮಾಡಬೇಕು `--no-same-owner` ಇದು `1001` ಬಳಕೆದಾರರಿಗೆ `uid` ಆಗಿ ಸಂಕುಚಿತಗೊಳ್ಳುತ್ತದೆ

ಪರಿಣಾಮವಾಗಿ, rsync ರೂಟ್ ಡೈರೆಕ್ಟರಿಯ ಬಳಕೆದಾರರ ಗುಂಪನ್ನು ಬದಲಾಯಿಸುತ್ತದೆ, ಮತ್ತು ಮರುಪ್ರಾರಂಭಿಸಿದ ನಂತರ ನೀವು ssh ಗೆ ಲಾಗ್ ಇನ್ ಮಾಡಲು ಸಾಧ್ಯವಾಗುವುದಿಲ್ಲ.

ಡಿಕಂಪ್ರೆಷನ್ ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು ಈ ಕೆಳಗಿನಂತೆ ಮಾರ್ಪಡಿಸಿ:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
