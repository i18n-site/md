# ಸ್ಥಾಪಿಸಿ ಮತ್ತು ಬಳಸಿ

## ವಿಂಡೋಸ್ ಮೊದಲು git bash ಅನ್ನು ಸ್ಥಾಪಿಸಿ

windows ಸಿಸ್ಟಮ್, ದಯವಿಟ್ಟು [ಡೌನ್‌ಲೋಡ್ ಮಾಡಲು ಮತ್ತು ಮೊದಲು ಸ್ಥಾಪಿಸಲು ಇಲ್ಲಿ ಕ್ಲಿಕ್ ಮಾಡಿ `git bash`](https://git-scm.com/download/win)

ರಲ್ಲಿ ನಂತರದ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು ರನ್ ಮಾಡಿ `git bash`

## ಸ್ಥಾಪಿಸಿ

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### ಅನುವಾದ ಟೋಕನ್ ಅನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ

ಟೋಕನ್ ನಕಲಿಸಲು ಭೇಟಿ ನೀಡಿ [i18n.site/token](//i18n.site/token)

```
token: YOUR_API_TOKEN
```

ಹೆಚ್ಚುವರಿಯಾಗಿ, ಪಾವತಿಗಾಗಿ ನೀವು ಕ್ರೆಡಿಟ್ ಕಾರ್ಡ್ ಅನ್ನು ಬೈಂಡ್ [i18n.site/payBill](//i18n.site/payBill) (ಯಾವುದೇ ರೀಚಾರ್ಜ್ ಅಗತ್ಯವಿಲ್ಲ, ವೆಬ್‌ಸೈಟ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬಳಕೆಯ ಪ್ರಕಾರ ಶುಲ್ಕವನ್ನು ಕಡಿತಗೊಳಿಸುತ್ತದೆ, [ಬೆಲೆಗಾಗಿ ಮುಖಪುಟವನ್ನು ನೋಡಿ](/#price) ).

## ಬಳಸಿ

### ಡೆಮೊ ಯೋಜನೆ

`i18` ಸಂರಚನೆಯನ್ನು ತಿಳಿಯಲು ದಯವಿಟ್ಟು ಡೆಮೊ ಯೋಜನೆಯನ್ನು ನೋಡಿ [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

ಚೀನಾದಲ್ಲಿ ಬಳಕೆದಾರರು ಕ್ಲೋನ್ ಮಾಡಬಹುದು [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

ಅಬೀಜ ಸಂತಾನೋತ್ಪತ್ತಿಯ ನಂತರ, ಡೈರೆಕ್ಟರಿಯನ್ನು ನಮೂದಿಸಿ ಮತ್ತು ಅನುವಾದವನ್ನು ಪೂರ್ಣಗೊಳಿಸಲು ರನ್ ಮಾಡಿ `i18`

### ಡೈರೆಕ್ಟರಿ ರಚನೆ

ಟೆಂಪ್ಲೇಟ್ ಗೋದಾಮಿನ ಡೈರೆಕ್ಟರಿ ರಚನೆಯು ಈ ಕೆಳಗಿನಂತಿರುತ್ತದೆ

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` ಡೈರೆಕ್ಟರಿಯು ಭಾಷಾಂತರಿಸಿದ ಡೆಮೊ ಫೈಲ್‌ಗಳನ್ನು ಹೊಂದಿದೆ, ಇದು ಕೇವಲ ಉದಾಹರಣೆಯಾಗಿದೆ ಮತ್ತು ಅಳಿಸಬಹುದು.

### ಅನುವಾದವನ್ನು ರನ್ ಮಾಡಿ

ಡೈರೆಕ್ಟರಿಯನ್ನು ನಮೂದಿಸಿ ಮತ್ತು ಅನುವಾದಿಸಲು ರನ್ ಮಾಡಿ `i18`

### ರೆಪೊಸಿಟರಿಗೆ ಫೈಲ್‌ಗಳನ್ನು ಸೇರಿಸಿ

ಅನುವಾದದ ಜೊತೆಗೆ, ಪ್ರೋಗ್ರಾಂ ಈ ಕೆಳಗಿನ ಫೈಲ್‌ಗಳನ್ನು ಸಹ ರಚಿಸುತ್ತದೆ, ದಯವಿಟ್ಟು ಅವುಗಳನ್ನು ರೆಪೊಸಿಟರಿಗೆ ಸೇರಿಸಿ.

```
.i18n/hash
.i18n/cache/.gitignore
```

ಅವುಗಳಲ್ಲಿ, ವಿಷಯವು : ಕೆಳಗಿನಂತಿರುತ್ತದೆ `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

ಇದರರ್ಥ `.i18n/cache/` ಎಲ್ಲಾ ಫೈಲ್‌ಗಳನ್ನು ನಿರ್ಲಕ್ಷಿಸಿ ( `.i18n/cache/.gitignore` ಹೊರತುಪಡಿಸಿ).

ನಿಮ್ಮ ಆವೃತ್ತಿ ನಿಯಂತ್ರಣ ಸಾಫ್ಟ್‌ವೇರ್ ಇಲ್ಲದಿದ್ದರೆ `git` ದಯವಿಟ್ಟು ಈ ಕಾನ್ಫಿಗರೇಶನ್ ಪ್ರಕಾರ ಅದನ್ನು ನಿರ್ಲಕ್ಷಿಸಿ.

## ಕಾನ್ಫಿಗರೇಶನ್ ಫೈಲ್

`.i18n/conf.yml` ಇದು `i18` ಲೈನ್ ಅನುವಾದ ಉಪಕರಣದ ಸಂರಚನಾ ಕಡತವಾಗಿದೆ

ವಿಷಯ ಹೀಗಿದೆ:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ಮೂಲ ಭಾಷೆ &

ಸಂರಚನಾ ಕಡತದಲ್ಲಿ, `fromTo`

`en` ಮೂಲ ಭಾಷೆ, `zh ja ko de fr` ಅನುವಾದದ ಗುರಿ ಭಾಷೆ.

ಭಾಷಾ ಕೋಡ್ ನೋಡಿ [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

ಉದಾಹರಣೆಗೆ, ನೀವು ಚೈನೀಸ್ ಅನ್ನು ಇಂಗ್ಲಿಷ್‌ಗೆ ಭಾಷಾಂತರಿಸಲು ಬಯಸಿದರೆ, ಈ ಸಾಲನ್ನು ಪುನಃ ಬರೆಯಿರಿ `zh: en`

ನೀವು ಎಲ್ಲಾ ಬೆಂಬಲಿತ ಭಾಷೆಗಳಿಗೆ ಅನುವಾದಿಸಲು ಬಯಸಿದರೆ, ದಯವಿಟ್ಟು ನಂತರ ಖಾಲಿ ಬಿಡಿ `:` ಉದಾಹರಣೆಗೆ

```
i18n:
  fromTo:
    en:
```

### ಫೈಲ್ ನಿರ್ಲಕ್ಷಿಸಿ

ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ, ಮೂಲ ಭಾಷೆಯ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ `.md` ಮತ್ತು `.yml` ನಿಂದ ಪ್ರಾರಂಭವಾಗುವ ಎಲ್ಲಾ ಫೈಲ್‌ಗಳನ್ನು ಅನುವಾದಿಸಲಾಗುತ್ತದೆ.

ನೀವು ಕೆಲವು ಫೈಲ್‌ಗಳನ್ನು ನಿರ್ಲಕ್ಷಿಸಲು ಬಯಸಿದರೆ ಮತ್ತು ಅವುಗಳನ್ನು ಅನುವಾದಿಸದಿದ್ದರೆ (ಉದಾಹರಣೆಗೆ ಪೂರ್ಣಗೊಳಿಸದ ಡ್ರಾಫ್ಟ್‌ಗಳು), ನೀವು `ignore` ಕಾನ್ಫಿಗರೇಶನ್ ಅನ್ನು ಬಳಸಬಹುದು.

`ignore` `.gitignore` ಗೆ ಸಮಾನವಾದ ಸಿಂಟ್ಯಾಕ್ಸ್ ಅನ್ನು ಬಳಸುತ್ತದೆ [globset](https://docs.rs/globset/latest/globset/#syntax)

ಉದಾಹರಣೆಗೆ, ಮೇಲಿನ ಕಾನ್ಫಿಗರೇಶನ್ ಫೈಲ್‌ನಲ್ಲಿ `_* ` `_` ಪ್ರಾರಂಭವಾಗುವ ಫೈಲ್‌ಗಳನ್ನು ಅನುವಾದಿಸಲಾಗುವುದಿಲ್ಲ.

## ಅನುವಾದ ನಿಯಮಗಳು

### ಅನುವಾದ ಸಂಪಾದಕರು ಸಾಲುಗಳನ್ನು ಸೇರಿಸಬಾರದು ಅಥವಾ ಅಳಿಸಬಾರದು

ಅನುವಾದವನ್ನು ಸಂಪಾದಿಸಬಹುದಾಗಿದೆ. ಮೂಲ ಪಠ್ಯವನ್ನು ಮಾರ್ಪಡಿಸಿ ಮತ್ತು ಅದನ್ನು ಮತ್ತೆ ಯಂತ್ರ-ಭಾಷಾಂತರ ಮಾಡಿ, ಅನುವಾದಕ್ಕೆ ಹಸ್ತಚಾಲಿತ ಮಾರ್ಪಾಡುಗಳನ್ನು ತಿದ್ದಿ ಬರೆಯಲಾಗುವುದಿಲ್ಲ (ಮೂಲ ಪಠ್ಯದ ಈ ಪ್ಯಾರಾಗ್ರಾಫ್ ಅನ್ನು ಮಾರ್ಪಡಿಸದಿದ್ದರೆ).

ಆದರೆ ಅನುವಾದದ ಸಾಲುಗಳು ಮತ್ತು ಮೂಲ ಪಠ್ಯವು ಒಂದಕ್ಕೆ ಹೊಂದಿಕೆಯಾಗಬೇಕು ಎಂಬುದನ್ನು ದಯವಿಟ್ಟು ಗಮನಿಸಿ. ಅಂದರೆ, ಅನುವಾದವನ್ನು ಕಂಪೈಲ್ ಮಾಡುವಾಗ ಸಾಲುಗಳನ್ನು ಸೇರಿಸಬೇಡಿ ಅಥವಾ ಅಳಿಸಬೇಡಿ. ಇಲ್ಲದಿದ್ದರೆ, ಇದು ಅನುವಾದ ಸಂಪಾದನೆ ಸಂಗ್ರಹದಲ್ಲಿ ಗೊಂದಲವನ್ನು ಉಂಟುಮಾಡುತ್ತದೆ.

ಏನಾದರೂ ತಪ್ಪಾದಲ್ಲಿ, ದಯವಿಟ್ಟು [ಪರಿಹಾರಗಳಿಗಾಗಿ FAQ ಅನ್ನು ಉಲ್ಲೇಖಿಸಿ.](/i18/qa#H1)

### `YAML` ಅನುವಾದಿಸು

ಕಮಾಂಡ್ ಲೈನ್ ಪರಿಕರವು ಮೂಲ ಭಾಷೆಯ ಫೈಲ್ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ `.yml` ಎಲ್ಲಾ ಫೈಲ್‌ಗಳನ್ನು ಹುಡುಕುತ್ತದೆ ಮತ್ತು ಅವುಗಳನ್ನು ಅನುವಾದಿಸುತ್ತದೆ.

* ಫೈಲ್ ಹೆಸರು ಪ್ರತ್ಯಯ `.yml` ( `.yaml` ಅಲ್ಲ).

ಸಾಧನವು ನಿಘಂಟು ಮೌಲ್ಯಗಳನ್ನು ಮಾತ್ರ ಅನುವಾದಿಸುತ್ತದೆ `.yml` ನಿಘಂಟು ಕೀಗಳನ್ನು ಅಲ್ಲ.

ಉದಾಹರಣೆಗೆ `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

ಎಂದು ಅನುವಾದಿಸಲಾಗುತ್ತದೆ `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` ನ ಅನುವಾದವನ್ನು ಹಸ್ತಚಾಲಿತವಾಗಿ ಮಾರ್ಪಡಿಸಬಹುದು (ಆದರೆ ಅನುವಾದದಲ್ಲಿ ಕೀಲಿಗಳು ಅಥವಾ ಸಾಲುಗಳನ್ನು ಸೇರಿಸಬೇಡಿ ಅಥವಾ ಅಳಿಸಬೇಡಿ).

`YAML` ಆಧರಿಸಿ, ನೀವು ವಿವಿಧ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಿಗೆ ಸುಲಭವಾಗಿ ಅಂತರರಾಷ್ಟ್ರೀಯ ಪರಿಹಾರಗಳನ್ನು ರಚಿಸಬಹುದು.

### ಅನುವಾದ ಉಪಕೋಶ

ನೀವು `.i18n/conf.yml` (ಪ್ರತಿ ಬಾರಿ ಡೆಮೊ ಪ್ರಾಜೆಕ್ಟ್ ಟೆಂಪ್ಲೇಟ್‌ನಿಂದ ಪ್ರಾರಂಭಿಸುವ ಅಗತ್ಯವಿಲ್ಲ), `i18` ಉತ್ತಮವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ.

ಕಮಾಂಡ್ ಲೈನ್ ಉಪಕರಣವು ಎಲ್ಲಾ ಉಪ ಡೈರೆಕ್ಟರಿಗಳಲ್ಲಿ `.i18n/conf.yml` ಕಂಡುಕೊಳ್ಳುತ್ತದೆ ಮತ್ತು ಅದನ್ನು ಅನುವಾದಿಸುತ್ತದೆ.

### ಕಸ್ಟಮ್ ಅನುಸ್ಥಾಪನ ಡೈರೆಕ್ಟರಿ

ಇದನ್ನು ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ ಸ್ಥಾಪಿಸಲಾಗುವುದು `/usr/local/bin`

ಬರೆಯಲು ಅನುಮತಿಯನ್ನು `/usr/local/bin` ಅದನ್ನು `~/.bin` ಗೆ ಸ್ಥಾಪಿಸಲಾಗುತ್ತದೆ.

ಪರಿಸರ ವೇರಿಯಬಲ್‌ಗಳನ್ನು ಹೊಂದಿಸಿ ನೀವು ಅನುಸ್ಥಾಪನಾ ಡೈರೆಕ್ಟರಿಯನ್ನು ವ್ಯಾಖ್ಯಾನಿಸಬಹುದು, `TO` :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
