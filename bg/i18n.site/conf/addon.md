# Плъгин

Плагините могат да бъдат конфигурирани в `.i18n/conf.yml`, като например:

```yml
addon:
  - i18n.addon/toc
```

## Официални плъгини

* `i18n.addon/toc` : Индекс на съдържанието
  Генериране на индекс на съдържанието от `json`, базиран на `TOC`, активиран по подразбиране

* `i18n.addon/mouse` : Ефекти с мишката

## Конвенция за именуване на файлове

Плъгините са всички `npm` пакети.

Пакетът, съответстващ на `i18n.addon/toc` по-горе, е [https://www.npmjs.com/package/@i18n.addon/toc](https://www.npmjs.com/package/@i18n.addon/toc).

Плъгините използват най-новата версия по подразбиране и проверяват за актуализации всяка седмица.

Ако искате да зададете固定на версия, можете да напишете `i18n.addon/toc@0.1.3`.

Командата за превод `i18n.site` ще инсталира договорения файл на добавката и след това ще го изпълни.

Договорените имена на файлове са както следва

### htmIndex.js

`htmIndex.js` ще бъде инжектиран в края на `.i18n/htm/index.js`.

Където `__CONF__` ще бъде заменено с името на текущата конфигурация (например `dev` или `ol`).

### afterTran.js

Ще бъде извикан след завършване на превода, като следните параметри ще бъдат предадени.

* `lang_li` : Списък с езици, първият език е изходният
* `changed` : Променени файлове
* `root` : Коренова директория на проекта

Връщаната стойност е речник, например

```json
{
  file:{
    //  path: txt, for example :
    // "_.json": "[]"
  }
}
```

`file` е списъкът с изходни файлове, `path` е пътят до файла, `txt` е съдържанието на файла.

## Вградени функции

Вграденият `js` RuntimeObject е разработен на базата на [boa](https://github.com/boa-dev/boa) чрез вторична разработка, като вградените функции са както следва:

* `wPath(path, txt)` : Записва в файл
* `rTxt(path)` : Чете текстов файл
* `rBin(path)` : Чете двоичен файл
* `rDir(dirpath)` : Чете директорията, връща масив: списък на директории, списък на файлове
