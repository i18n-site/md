## Пайдалану

### Демо Жоба

`i18` конфигурациясын білу үшін демонстрациялық жобаны қараңыз [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

Қытайдағы пайдаланушылар клондай алады [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

Клондағаннан кейін, аударманы аяқтау үшін каталогты енгізіңіз және іске `i18` .

### Каталог Құрылымы

Үлгі қоймасының каталог құрылымы келесідей

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` каталогында тек мысал болып табылатын және жойылуы мүмкін аударылған демонстрациялық файлдар бар.

### Аударманы Іске Қосыңыз

Каталогты енгізіп, аудару үшін іске қосыңыз `i18`

### Файлдарды Репозиторийге Қосыңыз

Аудармаға қоса, бағдарлама келесі файлдарды да жасайды, оларды репозиторийге қосыңыз.

```
.i18n/hash
.i18n/cache/.gitignore
```

Олардың : мазмұны төмендегідей `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

Бұл `.i18n/cache/` барлық файлдарды елемеу дегенді білдіреді ( `.i18n/cache/.gitignore` қоспағанда).

Егер сіздің нұсқаңызды басқару бағдарламалық құралы `git` , оны осы конфигурацияға сәйкес елемеуіңізді өтінеміз.

## Конфигурация Файлы

`.i18n/conf.yml` Бұл `i18` жолды аудару құралының конфигурация файлы

Мазмұны келесідей:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Бастапқы Тіл Аударма &

Конфигурация файлында `fromTo`

`en` - бастапқы тіл, `zh ja ko de fr` - аударма тілі.

Тіл кодын қараңыз [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

Мысалы, қытай тілін ағылшын тіліне аударғыңыз келсе, осы жолды қайта жазыңыз `zh: en`

Барлық қолдау көрсетілетін тілдерге аударғыңыз келсе, кейін бос қалдырыңыз `:` мысалы

```
i18n:
  fromTo:
    en:
```

### Файлды Елемеу

Әдепкі бойынша, бастапқы тіл каталогындағы `.md` және `.yml` әрпінен басталатын барлық файлдар аударылады.

Белгілі бір файлдарды елемей, оларды аудармақыңыз келсе (мысалы, аяқталмаған жобалар), `ignore` конфигурациясын пайдалануға болады.

`ignore` `.gitignore` файлға ұқсас синтаксисті пайдаланады [globset](https://docs.rs/globset/latest/globset/#syntax)

Мысалы, жоғарыдағы конфигурация `_* ` `_` әрпінен басталатын файлдар аударылмайды дегенді білдіреді.

## Аударма Ережелері

### Аударма Редакторлары Жолдарды Қоспауы Немесе Жоймауы Керек

Аударма өңдеуге болады. Түпнұсқа мәтінді өзгертіңіз және оны машинамен қайта аударыңыз, аудармаға қолмен енгізілген өзгертулер қайта жазылмайды (егер түпнұсқа мәтіннің осы абзацы өзгертілмеген болса).

Бірақ аударма жолдары мен түпнұсқа мәтін бір-біріне сәйкес келуі керек екенін ескеріңіз. Яғни, аударманы құрастырған кезде жолдарды қосуға немесе жоюға болмайды. Әйтпесе, бұл аударманы өңдеу кэшінде шатасуға әкеледі.

Егер бірдеңе дұрыс болмаса, [шешімдер үшін жиі қойылатын сұрақтар бөлімін қараңыз.](/i18/qa#H1)

### `YAML` Аудару

Пәрмен жолы құралы бастапқы тілдің файлдар каталогында `.yml` деп аяқталатын барлық файлдарды табады және оларды аударады.

* Файл атауының жұрнағы `.yml` ( `.yaml` емес) болуы керек екенін ескеріңіз.

Құрал сөздік пернелерін емес, сөздік мәндерін `.yml` аударады.

мысалы `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

ретінде аударылады `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` аудармасын қолмен де өзгертуге болады (бірақ аудармаға кілттерді немесе жолдарды қоспаңыз немесе жоймаңыз).

`YAML` негізінде әртүрлі бағдарламалау тілдеріне арналған халықаралық шешімдерді оңай құрастыруға болады.

## Жетілдірілген Пайдалану

### Аударма Ішкі Каталогы

Сіз `.i18n/conf.yml` кезде (әр уақытта демонстрациялық жоба үлгісінен бастау қажет емес), `i18` жақсы жұмыс істейді.

Пәрмен жолы құралы `.i18n/conf.yml` барлық ішкі каталогтардан табады және оны аударады.

[monorepo](//monorepo.tools) архитектурасын пайдаланатын жобалар тілдік файлдарды ішкі каталогтарға бөле алады.

<img alt="" src="https://p.3ti.site/1719910016.avif">

### Пайдаланушы Орнату Каталогы

Ол әдепкі бойынша орнатылады `/usr/local/bin`

Жазу рұқсаты `/usr/local/bin` ол `~/.bin` үшін орнатылады.

Орташа айнымалы мәндерді орнату `TO` Мысалы, орнату каталогын анықтауға болады :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
