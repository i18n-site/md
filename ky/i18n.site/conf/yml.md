# .i18n/conf.yml

Конфигурация файлы `.i18n/conf.yml` `i18n.site`

[`i18`](/i18) `ignore:` жана `i18n:` жөндөөлөрүнөн башкасы, конфигурация файлы төмөнкүдөй:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

Алардын ичинен `ext:` конфигурация элементи жарыялоодо `.md` `upload` жүктөлөт дегенди билдирет.

## Жогорку Навигация Навигациясы

Башкы беттин жогору жагындагы навигация менюсуна туура келген `nav:` параметрлери.

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

Алардын ичинен `en/i18n.yml`中`home: Home` туура `i18n: home` .

`en/i18n.yml` бир нече тилге которулат, мисалы `zh/i18n.yml` .

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

Котормо аяктагандан кийин, котормодогу `yml` өзгөртө аласыз, бирок `yml` котормо ачкычын кошпоңуз же жок кылбаңыз.

### `use: Toc` Бир Файл Шаблону (Контур Менен)

: `nav`

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` бир `Markdown` шаблонду көрсөткөн `Toc` рендерингди колдонууну билдирет.

`TOC` бул `Table of Contents` дын аббревиатурасы `Markdown`

`url:` `Markdown` жолун билдирет ( `/` түпкү каталогуна туура келет `/README.md` , бул файлдын аталышы чоң жана кичине суффиксти талап кылат).

### `use: Md` Бир Файл Шаблону (Контур Жок)

`Md` Калып `Toc` менен бирдей, экөө тең бир `Markdown` файлын көрсөтүү үчүн колдонулат. Бирок `Md` шаблон каптал тилкесинде контурду көрсөтпөйт.

Сиз жогорудагы конфигурацияда `use: Md` дегенге `use: Toc` аласыз, `i18n.site` кайра `md` каталогунда иштетип, анан башкы бетте өзгөрүүлөрдү көрүү үчүн иштеп чыгууну алдын ала көрүү URL дарегине баш багыңыз.

### Конфигурация Жолу Жок Демейки Жүктөө

Эгер жеткиликтүү болгон белгилүү бир жолдун жол префикси `nav:` жолго тиешелүү `MarkDown` файл демейки боюнча жүктөлөт жана `Md` шаблону аркылуу көрсөтүлөт.

Мисалы, эгер сиз `/test` дегенге кирсеңиз жана `nav:` бул жолсуз конфигурацияланган болсо, жана баракчанын тили англисче (коду `en` ), демейки боюнча шаблон жүктөлөт `/en/test.md` жана аркылуу көрсөтүлөт `Md` .

Бул файл жок `/en/test.md` , демейки `404` бет көрсөтүлөт.

### `use: Doc` Көп Файл Шаблону

Конфигурация файлында:

```
  - i18n: blog
    use: Doc
```

шаблонду көрсөтүү үчүн колдонууну көрсөтөт `Doc`

`Doc` Калып бир же бир нече долбоорлор үчүн документ схемаларын түзүү үчүн бир нече `MarkDown` интеграциясын колдойт.

#### Жалгыз Долбоор (Бир Нече Файл)

`blog` бир долбоор режими болуп `Doc` .

```
  - i18n: blog
    url: blog
    use: Doc
```

##### url Бош Болгондо, Ал Демейки i18n Маанисине Келет

`url` сиз жазбасаңыз `url` бул эреже `i18n` калыптар үчүн да күчүнө кирет.

Жогорудагы жазуу ыкмасы бар `url: blog` ге барабар жана анын тиешелүү файлы `en/blog/TOC` .

#### Бир Нече Долбоорлор

`.i18n/conf.yml` конфигурациясы `i18n:doc` долбоор режими.

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

Бул жерде `menu: NB demo1,demo2` ачылуучу менюну көрсөтүү үчүн `NB` колдонууну билдирет.

`NB` , ылдый түшүүчү менюда долбоордун атын жана ураанын көрсөтө ала турган `Name Breif` дын аббревиатурасы.

`NB` ден кийин ага `demo1,demo2` берилген параметр келет.
`demo1,demo2` : ** `,` **

Жогорудагы параметрлер үчүн, тиешелүү каталогдун индекси файлы болуп саналат:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### TOC Мазмунунун Индекси

`i18n.site` Каталогдун `json` түзүү үчүн каталогдун индексинин файлын `doc` калыптын конфигурациясына ылайыктуу `TOC` окуу үчүн `.i18n/hook/after.tran/TOC.js` `js` демо кампада ишке ашырат.

Эгер сиз `doc` колдонсоңуз, сизде бул плагин болушу керек.

Эгер сиз `i18n.site` бош папкадан инициализацияласаңыз, демо долбоордогу `.i18n` каталогуңузга көчүрүүнү унутпаңыз.

`Doc` түзүлгөн `json` негизинде мазмундун контурун көрсөтөт.

##### TOC Кантип Жапырт Көчүрүү Керек?

`TOC` Файлды баштапкы тилдин каталогуна жайгаштыруу керек.

Мисалы, эгер баштапкы тил кытай болсо, анда жогоруда `zh/blog/TOC` `TOC`

Эгерде баштапкы тил өзгөртүлсө, сиз проекттеги белгилүү бир тилдеги `TOC` башка тилге көчүрүүңүз керек.

Сиз төмөнкү буйруктарга кайрыла аласыз:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

`zh/` , тил кодуңузга жогорудагы буйруктагы `en/`
