## Колдонуу

### Демо Долбоор

`i18` конфигурациясын билүү үчүн демо долбоорго кайрылыңыз [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

Кытайдагы колдонуучулар клондой алышат [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

Клондоодон кийин, каталогду киргизип, котормону бүтүрүү үчүн иштетиңиз `i18`

### Каталог Структурасы

Калып кампасынын каталогунун түзүлүшү төмөнкүдөй

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` которулган демо файлдарды камтыйт, алар жөн гана мисал жана жок кылынышы мүмкүн.

### Котормосун Иштетүү

Каталогду киргизиңиз жана которуу үчүн чуркаңыз `i18`

### Репозиторийге Файлдарды Кошуңуз

Котормодон тышкары, программа төмөнкү файлдарды да жаратат, аларды репозиторийге кошуңуз.

```
.i18n/hash
.i18n/cache/.gitignore
```

Алардын : мазмуну `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

Бул `.i18n/cache/` бардык файлдарды этибарга албай коюуну билдирет ( `.i18n/cache/.gitignore` дан башка).

Эгерде сиздин версияны башкаруу программалык камсыздооңуз жок болсо `git` бул конфигурацияга ылайык аны этибарга албаңыз!

## Тарам Билэ

`.i18n/conf.yml` Бул `i18` командалык котормо куралынын конфигурация файлы

мазмуну төмөнкүдөй:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Котормо &

Конфигурация файлында `fromTo`

`en` булагы тил, `zh ja ko de fr` котормонун максаттуу тили.

Тил коду караңыз [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

Мисалы, эгер сиз кытайчаны англис тилине которгуңуз келсе, бул сапты кайра жазыңыз `zh: en`

Эгерде сиз бардык колдоого алынган тилдерге которууну кааласаңыз, андан кийин бош калтырыңыз `:` мисалы

```
i18n:
  fromTo:
    en:
```

### Файлды Этибарга Алба

Демейки боюнча, баштапкы тил каталогундагы `.md` жана `.yml` менен башталган бардык файлдар которулат.

Эгер сиз айрым файлдарды этибарга албай, аларды которбоону кааласаңыз (мисалы, бүтө элек долбоорлор), `ignore` конфигурациясын колдонсоңуз болот.

`ignore` `.gitignore` синтаксисин колдонот [globset](https://docs.rs/globset/latest/globset/#syntax)

Мисалы, жогорудагы конфигурация файлында `_* ` `_` башталган файлдар которулбайт дегенди билдирет.

## Котормо Эрежелери

### Котормо Редакторлору Саптарды Кошуп Же Жок Кылбашы Керек

Котормо түзөтүүгө болот. Түпнуска текстти өзгөртүңүз жана аны кайра машина менен которуңуз, котормого кол менен жасалган өзгөртүүлөр кайра жазылбайт (эгерде түпнуска тексттин бул абзацы өзгөртүлбөсө).

Бирок котормонун саптары менен түпнуска тексттин саптары бири-бирине дал келиши керек экенин эске алыңыз. Башкача айтканда, котормону түзүүдө саптарды кошуп же жок кылбаңыз. Болбосо, котормонун түзөтүү кэшинде башаламандыкка алып келет.

Эгер бир нерсе туура эмес болуп калса, [чечүү жолдорун FAQ бөлүмүнө караңыз.](/i18/qa#H1)

### `Yaml`

Буйрук сабы куралы булак тилинин файл каталогунан `.yml` менен аяктаган бардык файлдарды таап, аларды которот.

* Файлдын аталышынын суффикси болушу керек `.yml` эске алыңыз ( `.yaml` эмес).

Курал сөздүк баскычтарын эмес, сөздүк маанилерин гана которот `.yml`

мисалы `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

катары которулат `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` кол менен да өзгөртүүгө болот (бирок котормодогу баскычтарды же саптарды кошпоңуз же жок кылбаңыз).

`YAML` негизинде ар кандай программалоо тилдери үчүн эл аралык чечимдерди оңой түзө аласыз.

## Өркүндөтүлгөн Колдонуу

### Котормочу Подкаталог

Сиз `.i18n/conf.yml` (ар бир жолу демо долбоордун шаблонунан баштоонун кереги жок), `i18` жакшы иштейт.

Буйрук сабы куралы бардык подкаталогдордон `.i18n/conf.yml` таап, аны которот.

[monorepo](//monorepo.tools) колдонгон долбоорлор тил файлдарын подкаталогдорго бөлө алат.

<img alt="" src="https://p.3ti.site/1719910016.avif">

### Ыңгайлаштырылган Орнотуу Каталогу

Ал демейки боюнча орнотулат `/usr/local/bin`

Эгер жазууга уруксаты жок болсо `/usr/local/bin` ал `~/.bin` орнотулат.

Айлана-чөйрө өзгөрмөлөрүн орнотуңуз `TO` мисалы, орнотуу каталогун аныктай аласыз :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
