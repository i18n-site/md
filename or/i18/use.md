# ସଂସ୍ଥାପନ ଏବଂ ବ୍ୟବହାର କରନ୍ତୁ |

## ୱିଣ୍ଡୋଜ୍ ପ୍ରଥମେ git bash କୁ ସଂସ୍ଥାପନ କରେ |

windows , ଦୟାକରି [ପ୍ରଥମେ ଡାଉନଲୋଡ୍ ଏବଂ ସଂସ୍ଥାପନ କରିବାକୁ ଏଠାରେ କ୍ଲିକ୍ କରନ୍ତୁ `git bash`](https://git-scm.com/download/win)

ପରବର୍ତ୍ତୀ ଅପରେସନ୍ ଚଲାନ୍ତୁ `git bash`

## ସଂସ୍ଥାପନ କରନ୍ତୁ |

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### ଅନୁବାଦ ଟୋକେନ୍ ବିନ୍ୟାସ କରନ୍ତୁ |

ଟୋକନ୍ କପି କରିବାକୁ କ୍ଲିକ୍ କରନ୍ତୁ [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

ସୃଷ୍ଟି କରନ୍ତୁ `~/.config/i18n.site.yml` ଏଥିରେ କପି ହୋଇଥିବା ବିଷୟବସ୍ତୁକୁ ଲେପନ କରନ୍ତୁ, ବିଷୟବସ୍ତୁ ନିମ୍ନଲିଖିତ ଅଟେ:

```
token: YOUR_API_TOKEN
```

ଏହା ସହିତ, ଆପଣଙ୍କୁ ଦେୟ ପାଇଁ ଏକ କ୍ରେଡିଟ୍ କାର୍ଡ [i18n.site/payBill](//i18n.site/payBill) ପଡିବ (କ rec ଣସି ରିଚାର୍ଜ ଆବଶ୍ୟକ ନାହିଁ, ୱେବସାଇଟ୍ ବ୍ୟବହାର ଅନୁଯାୟୀ ସ୍ୱୟଂଚାଳିତ ଭାବରେ ଫି କାଟିବ, [ମୂଲ୍ୟ ପାଇଁ ମୂଳପୃଷ୍ଠା ଦେଖନ୍ତୁ](/#price) ) |

## ବ୍ୟବହାର କରନ୍ତୁ |

### ଡେମୋ ପ୍ରୋଜେକ୍ଟ |

ଅନୁବାଦର ସଂରଚନା ଜାଣିବା ପାଇଁ ଦୟାକରି ଡେମୋ ପ୍ରୋଜେକ୍ଟକୁ ଅନୁସରଣ `i18` [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

ଚାଇନାରେ ଉପଭୋକ୍ତାମାନେ କ୍ଲୋନ କରିପାରିବେ [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

କ୍ଲୋନିଂ ପରେ, ଡିରେକ୍ଟୋରୀକୁ ପ୍ରବେଶ କର ଏବଂ ଅନୁବାଦ ସଂପୂର୍ଣ୍ଣ କର `i18`

### ଡିରେକ୍ଟୋରୀ ସଂରଚନା

ଟେମ୍ପଲେଟ୍ ଗୋଦାମ ଡିରେକ୍ଟୋରୀ ସଂରଚନା ନିମ୍ନଲିଖିତ ଅଟେ |

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` ଅନୁବାଦିତ ଡେମୋ ଫାଇଲଗୁଡ଼ିକୁ ଧାରଣ କରିଥାଏ, ଯାହା କେବଳ ଏକ ଉଦାହରଣ ଏବଂ ବିଲୋପ ହୋଇପାରେ |

### ଅନୁବାଦ ଚଲାନ୍ତୁ |

ଡିରେକ୍ଟୋରୀ ପ୍ରବେଶ କରନ୍ତୁ ଏବଂ ଅନୁବାଦ କରନ୍ତୁ `i18`

### ସଂଗ୍ରହାଳୟରେ ଫାଇଲ୍ ଯୋଡନ୍ତୁ |

ଅନୁବାଦ ସହିତ, ପ୍ରୋଗ୍ରାମ୍ ନିମ୍ନଲିଖିତ ଫାଇଲଗୁଡିକ ମଧ୍ୟ ସୃଷ୍ଟି କରିବ, ଦୟାକରି ସେଗୁଡ଼ିକୁ ସଂଗ୍ରହାଳୟରେ ଯୋଡନ୍ତୁ |

```
.i18n/hash
.i18n/cache/.gitignore
```

ସେମାନଙ୍କ ମଧ୍ୟରେ, ବିଷୟବସ୍ତୁ ନିମ୍ନଲିଖିତ : `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

ଏହାର ଅର୍ଥ ହେଉଛି `.i18n/cache/.gitignore` ଡିରେକ୍ଟୋରୀରେ ଥିବା ସମସ୍ତ ଫାଇଲଗୁଡ଼ିକୁ ଅଣଦେଖା କରନ୍ତୁ `.i18n/cache/`

ଯଦି ଆପଣଙ୍କର ସଂସ୍କରଣ ନିୟନ୍ତ୍ରଣ ସଫ୍ଟୱେର୍ ନୁହେଁ `git` ଦୟାକରି ଏହାକୁ ଏହି ବିନ୍ୟାସ ଅନୁଯାୟୀ ଅଣଦେଖା କରନ୍ତୁ!

## ବିନ୍ୟାସ ଫାଇଲ୍ |

ଏହା ହେଉଛି `i18` ଲାଇନ୍ ଅନୁବାଦ ଉପକରଣର ବିନ୍ୟାସ ଫାଇଲ୍ `.i18n/conf.yml`

ବିଷୟବସ୍ତୁ ନିମ୍ନଲିଖିତ ଅଟେ:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ଉତ୍ସ ଭାଷା ଅନୁବାଦ ଭାଷା &

ବିନ୍ୟାସ ଫାଇଲରେ, ଅଧସ୍ତନମାନେ `fromTo`

`en` ଉତ୍ସ ଭାଷା, `zh ja ko de fr` ହେଉଛି ଅନୁବାଦର ଲକ୍ଷ୍ୟ ଭାଷା |

ଭାଷା କୋଡ୍ ଦେଖନ୍ତୁ [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

ଉଦାହରଣ ସ୍ୱରୂପ, ଯଦି ଆପଣ ଚାଇନିଜ୍ କୁ ଇଂରାଜୀରେ ଅନୁବାଦ କରିବାକୁ ଚାହାଁନ୍ତି, ତେବେ ଏହି ରେଖା ପୁନ r ଲିଖନ କରନ୍ତୁ `zh: en`

ଯଦି ଆପଣ ସମସ୍ତ ସମର୍ଥିତ ଭାଷାକୁ ଅନୁବାଦ କରିବାକୁ ଚାହାଁନ୍ତି, ଦୟାକରି ଖାଲି ଛାଡିଦିଅନ୍ତୁ `:` ଉଦାହରଣ ସ୍ଵରୁପ

```
i18n:
  fromTo:
    en:
```

### ଫାଇଲକୁ ଅଣଦେଖା କରନ୍ତୁ |

ଡିଫଲ୍ଟ ଭାବରେ, ଉତ୍ସ ଫାଇଲ ଡିରେକ୍ଟୋରୀରେ ଆରମ୍ଭ `.yml` ସମସ୍ତ ଫାଇଲଗୁଡ଼ିକ ଅନୁବାଦ ହେବ `.md`

ଯଦି ଆପଣ କିଛି ଫାଇଲ୍ କୁ ଅଣଦେଖା କରିବାକୁ ଚାହାଁନ୍ତି ଏବଂ ସେଗୁଡିକୁ ଅନୁବାଦ ନକରନ୍ତି (ଯେପରିକି ଅସମାପ୍ତ ଡ୍ରାଫ୍ଟଗୁଡିକ), ଆପଣ ଫିଲ୍ଡ ଫିଲ୍ଡ ବିନ୍ୟାସକରଣ ବ୍ୟବହାର କରିପାରିବେ `ignore`

`.gitignore` [globset](https://docs.rs/globset/latest/globset/#syntax) ଏକ ବାକ୍ୟବିନ୍ୟାସ ବ୍ୟବହାର କରେ `ignore`

ଉଦାହରଣ ସ୍ୱରୂପ, ଉପରୋକ୍ତ ବିନ୍ୟାସ ଫାଇଲରେ ଅର୍ଥ ହେଉଛି `_` ରୁ ଆରମ୍ଭ ହୋଇଥିବା ଫାଇଲଗୁଡ଼ିକ ଅନୁବାଦ ହେବ ନାହିଁ `_* `

## ଅନୁବାଦ ନିୟମ |

### ଅନୁବାଦ ସମ୍ପାଦକମାନେ ରେଖା ଯୋଡିବା କିମ୍ବା ବିଲୋପ କରିବା ଉଚିତ୍ ନୁହେଁ |

ଅନୁବାଦ ସମ୍ପାଦନାଯୋଗ୍ୟ | ମୂଳ ପାଠ୍ୟକୁ ପରିବର୍ତ୍ତନ କରନ୍ତୁ ଏବଂ ଏହାକୁ ପୁନର୍ବାର ମେସିନ୍-ଅନୁବାଦ କରନ୍ତୁ, ଅନୁବାଦରେ ମାନୁଆଲ୍ ପରିବର୍ତ୍ତନଗୁଡ଼ିକ ଓଭର୍ ରାଇଟ୍ ହେବ ନାହିଁ (ଯଦି ମୂଳ ପାଠ୍ୟର ଏହି ଅନୁଚ୍ଛେଦଟି ପରିବର୍ତ୍ତନ ହୋଇନାହିଁ) |

କିନ୍ତୁ ଦୟାକରି ଧ୍ୟାନ ଦିଅନ୍ତୁ ଯେ ଅନୁବାଦର ରେଖା ଏବଂ ମୂଳ ପାଠ୍ୟ ନିଶ୍ଚିତ ଭାବରେ ଗୋଟିଏ ସହିତ ଅନୁରୂପ ହେବା ଆବଶ୍ୟକ | ତାହା ହେଉଛି, ଅନୁବାଦ ସଂକଳନ କରିବା ସମୟରେ ରେଖା ଯୋଡନ୍ତୁ କିମ୍ବା ବିଲୋପ କରନ୍ତୁ ନାହିଁ | ଅନ୍ୟଥା, ଏହା ଅନୁବାଦ ଏଡିଟିଂ କ୍ୟାଶରେ ଦ୍ୱନ୍ଦ୍ୱ ସୃଷ୍ଟି କରିବ |

ଯଦି କିଛି ଭୁଲ୍ ହୁଏ, ଦୟାକରି [ସମାଧାନ ପାଇଁ FAQ କୁ ଅନୁସରଣ କରନ୍ତୁ |](/i18/qa#H1)

### ଅନୁବାଦ କର `YAML`

କମାଣ୍ଡ୍ ଲାଇନ୍ ଟୁଲ୍ ଉତ୍ସ ଭାଷା ଫାଇଲ୍ ଡିରେକ୍ଟୋରୀରେ ସମାପ୍ତ ହେଉଥିବା ସମସ୍ତ ଫାଇଲ୍ ଖୋଜିବ ଏବଂ ଅନୁବାଦ କରିବ `.yml`

* ଧ୍ୟାନ ଦିଅନ୍ତୁ ଯେ ଫାଇଲ୍ ନାମ ସଫିକ୍ସ ନିଶ୍ଚିତ ଭାବରେ ହେବ ( `.yaml` ନୁହେଁ) `.yml`

ସାଧନ କେବଳ ଅଭିଧାନ ମୂଲ୍ୟଗୁଡ଼ିକୁ ଅନୁବାଦ କରେ, ଅଭିଧାନ ଚାବି ନୁହେଁ `.yml`

ଉଦାହରଣ ସ୍ଵରୁପ `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

ଯେପରି ଅନୁବାଦ ହେବ `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` ଅନୁବାଦ ମଧ୍ୟ ମାନୁଆଲରେ ରୂପାନ୍ତରିତ ହୋଇପାରେ (କିନ୍ତୁ ଅନୁବାଦରେ ଚାବି କିମ୍ବା ରେଖା ଯୋଡନ୍ତୁ କିମ୍ବା ବିଲୋପ କରନ୍ତୁ ନାହିଁ) |

`YAML` ଉପରେ ଆଧାର କରି, ଆପଣ ବିଭିନ୍ନ ପ୍ରୋଗ୍ରାମିଂ ଭାଷା ପାଇଁ ସହଜରେ ଆନ୍ତର୍ଜାତୀୟ ସମାଧାନ ଗଠନ କରିପାରିବେ |

## ଉନ୍ନତ ବ୍ୟବହାର

### ଅନୁବାଦ ଉପଡିରେକ୍ଟୋରୀ |

ଯେପର୍ଯ୍ୟନ୍ତ ଆପଣ ସୃଷ୍ଟି `.i18n/conf.yml` (ପ୍ରତ୍ୟେକ ଥର ଡେମୋ ପ୍ରୋଜେକ୍ଟ ଟେମ୍ପଲେଟରୁ ଆରମ୍ଭ କରିବାର ଆବଶ୍ୟକତା ନାହିଁ), `i18` ଭଲ କାମ କରିବ |

କମାଣ୍ଡ୍ ଲାଇନ୍ ଟୁଲ୍ ସମସ୍ତ ସବ୍ ଡିରେକ୍ଟୋରୀରେ ବିନ୍ୟାସ ଖୋଜିବ ଏବଂ ଏହାକୁ ଅନୁବାଦ କରିବ `.i18n/conf.yml`

[monorepo](//monorepo.tools) ବ୍ୟବହାର କରୁଥିବା ପ୍ରୋଜେକ୍ଟଗୁଡ଼ିକ ଭାଷା ଫାଇଲଗୁଡ଼ିକୁ ଉପଡିରେକ୍ଟୋରୀରେ ବିଭକ୍ତ କରିପାରେ |

### କଷ୍ଟମ୍ ସଂସ୍ଥାପନ ଡିରେକ୍ଟୋରୀ |

ଡିଫଲ୍ଟ ଭାବରେ ଏହା ସଂସ୍ଥାପିତ ହେବ `/usr/local/bin`

ଯଦି ଲେଖିବାର ଅନୁମତି ନାହିଁ `/usr/local/bin` ଏହା `~/.bin` କୁ ସଂସ୍ଥାପିତ ହେବ |

ପରିବେଶ ଭେରିଏବଲ୍ ସେଟ୍ କରନ୍ତୁ `TO` ଉଦାହରଣ ସ୍ୱରୂପ ଆପଣ ସଂସ୍ଥାପନ ଡିରେକ୍ଟୋରୀକୁ ବ୍ୟାଖ୍ୟା କରିପାରିବେ :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
