## ଡେମୋ ପ୍ରୋଜେକ୍ଟ |

ଚାଲ ଏକ ଡେମୋ ପ୍ରୋଜେକ୍ଟରୁ ଆରମ୍ଭ କରିବା ଏବଂ କିପରି ବ୍ୟବହାର କରିବାକୁ ଶିଖିବା `i18n.site`

ଆମେ ପ୍ରଥମେ ଡେମୋ ରେପୋଜିଟୋରୀକୁ କ୍ଲୋନ୍ କରି ନିମ୍ନଲିଖିତ ଭାବରେ କମାଣ୍ଡ୍ ଚଲାଇଥାଉ:

```
git clone https://github.com/i18n-site/demo.i18n.site.git md
git clone https://github.com/i18n-site/demo.i18n.site.docker.git docker
```

ଚୀନ୍ ମଧ୍ୟପ୍ରଦେଶର ଉପଭୋକ୍ତାମାନେ କରିପାରିବେ:

```
git clone https://atomgit.com/i18n/demo.i18n.site.git md
git clone https://atomgit.com/i18n/demo.i18n.site.docker.git docker
```

`docker` ସହିତ ସ୍ଥାନୀୟ ପୂର୍ବାବଲୋକନକୁ ସୁଗମ କରିବା ପାଇଁ କୋଡ୍ ବେସ୍ କ୍ଲୋନ୍ର ଡିରେକ୍ଟୋରୀ ନାମ `md` ହେବା ଆବଶ୍ୟକ `demo.i18n.site`

### ବିଷୟବସ୍ତୁ ପୋଷ୍ଟ କରନ୍ତୁ |

ଏକ [ପୃଷ୍ଠା ପ୍ରୟୋଗ](https://developer.mozilla.org/docs/Glossary/SPA) ସ୍ଥାପତ୍ୟକୁ ଗ୍ରହଣ କରେ, ଏବଂ ୱେବସାଇଟ୍ ପ୍ରବେଶ ପୃଷ୍ଠା ଏବଂ ୱେବସାଇଟ୍ ବିଷୟବସ୍ତୁ ସ୍ independ ାଧୀନ ଭାବରେ ନିୟୋଜିତ ହୁଏ `i18n.site`

ଉପରୋକ୍ତ ଅନୁବାଦ ଚଲାଇବା ପରେ, `htm` ଏବଂ `v` ଡିରେକ୍ଟୋରୀଗୁଡିକ ଡିରେକ୍ଟୋରୀରେ ସୃଷ୍ଟି ହେବ `md/out/dev`

ଏଠାରେ `dev` ସୂଚିତ କରେ ଯେ ଏହା `.i18n/htm/dev.yml` ଫାଇଲ୍ ଉପରେ ଆଧାରିତ |

ବିଷୟବସ୍ତୁ : `dev`

ଡିରେକ୍ଟୋରୀ ତଳେ ୱେବସାଇଟ୍ ପ୍ରବେଶ ପୃଷ୍ଠା ଅଛି `htm`

ଡିରେକ୍ଟୋରୀଟି ସଂସ୍କରଣ ସଂଖ୍ୟା ସହିତ ୱେବସାଇଟ୍ ବିଷୟବସ୍ତୁ ଧାରଣ କରେ `v`

ସ୍ଥାନୀୟ ପୂର୍ବାବଲୋକନ ସଂସ୍କରଣ ନମ୍ବରକୁ ଖାତିର ନକରି ସମସ୍ତ ଡିରେକ୍ଟୋରୀକୁ କପି କରିବ `out/dev/v/0.1.0`

ଅଫିସିଆଲ୍ ରିଲିଜ୍ ପାଇଁ, ପରିବର୍ତ୍ତିତ ଫାଇଲଗୁଡ଼ିକୁ ନୂତନ ସଂସ୍କରଣ ନମ୍ବର ଡିରେକ୍ଟୋରୀରେ କପି କରାଯିବ |

#### ବିନ୍ୟାସ ଫାଇଲ ନିର୍ଦ୍ଦିଷ୍ଟ କରିବାକୁ -c ବ୍ୟବହାର କରନ୍ତୁ |

ବିଭିନ୍ନ ବିନ୍ୟାସ ଫାଇଲଗୁଡ଼ିକ ଡିରେକ୍ଟୋରୀରେ ଅନୁରୂପ ଡିରେକ୍ଟୋରୀ ସୃଷ୍ଟି କରିବ `out`

ଉଦାହରଣ ସ୍ୱରୂପ, ଏକ `out/ol` ଡିରେକ୍ଟୋରୀ ସୃଷ୍ଟି କରିବ `.i18n/htm/ol.yml`

ଏବଂ `ol.yml` ହେଉଛି ଡିଫଲ୍ଟ ବିନ୍ୟାସକରଣ `dev.yml`

`dev` `development` ର ସଂକ୍ଷିପ୍ତକରଣ, ଯାହା ବିକାଶ ପରିବେଶକୁ ପ୍ରତିନିଧିତ୍ୱ କରେ, ସ୍ଥାନୀୟ ପୂର୍ବାବଲୋକନ ପାଇଁ ବ୍ୟବହୃତ ହୁଏ, ଏବଂ ଏହା ମଧ୍ୟ ଡିଫଲ୍ଟ ବିନ୍ୟାସ ଫାଇଲ ଅଟେ |
`ol` `online` ର ସଂକ୍ଷିପ୍ତକରଣ, ଯାହା ଅନଲାଇନ୍ ପରିବେଶକୁ ପ୍ରତିନିଧିତ୍ୱ କରେ, ଅଫିସିଆଲ୍ ରିଲିଜ୍ ପାଇଁ ବ୍ୟବହୃତ ହୁଏ, ଏବଂ `npm` କରିବା ସମୟରେ ମଧ୍ୟ ଡିଫଲ୍ଟ ବିନ୍ୟାସ ଫାଇଲ ଅଟେ `-n`

ଆପଣ ଅନ୍ୟ ବିନ୍ୟାସ ଫାଇଲଗୁଡିକ ମଧ୍ୟ ସୃଷ୍ଟି କରିପାରିବେ, ବିନ୍ୟାସ ଫାଇଲ ନାମ ନିର୍ଦ୍ଦିଷ୍ଟ କରିବାକୁ ନିର୍ଦ୍ଦେଶନାମାରେ ବ୍ୟବହାର `--htm_conf` :

ଉଦାହରଣ ସ୍ଵରୁପ:
```
i18n.site --htm_conf yourConfig --save
```

ଏଠାରେ ଅଦ୍ୟତନ ପ୍ରକାଶନ ସଂସ୍କରଣ ସଂଖ୍ୟାକୁ ସୂଚିତ କରେ `--save`

#### <a rel=id href="#npm" id="npm"></a> Npmjs.com ରେ ବିଷୟବସ୍ତୁ ପ୍ରକାଶ କରନ୍ତୁ |

ବିଷୟବସ୍ତୁ ପ୍ରକାଶନ [npmjs.com](//npmjs.com) ହେଉଛି ପରାମର୍ଶିତ ଡିଫଲ୍ଟ ସମାଧାନ ( [ଫ୍ରଣ୍ଟ-ଏଣ୍ଡ ଉଚ୍ଚ ଉପଲବ୍ଧତା](/i18n.site/feature#ha) ଦେଖନ୍ତୁ) |

##### ରିଲିଜ୍ npm login &

ସଂସ୍ଥାପନ କରନ୍ତୁ `nodejs` ଲଗ୍ ଇନ୍ କରିବାକୁ `npm login` ବ୍ୟବହାର କରନ୍ତୁ |

ସଂପାଦନ କରନ୍ତୁ `md/.i18n/htm/ol.yml` `i18n.site` ରେ `v: //unpkg.com/i18n.site` ନିଜ ନିଜ `npm` ପ୍ୟାକେଜ୍ ନାମ) ରେ ପରିବର୍ତ୍ତନ କରନ୍ତୁ |

କେବଳ ଅଣସଂରକ୍ଷିତ ପ୍ୟାକେଜ୍ ନାମ ବ୍ୟବହାର କରନ୍ତୁ ୱେବସାଇଟ୍ ଡୋମେନ୍ ନାମକୁ ବ୍ୟବହାର କରିବା ଏକ ଭଲ ପସନ୍ଦ [npmjs.com](//npmjs.com)

`npm` ଉପରେ ଆଧାର କରି `/.v` କରିବାବେଳେ `i18n.site` `v:` ମୂଲ୍ୟର ଉପସର୍ଗ ଭାବରେ **`//unpkg.com/` ବ୍ୟବହାର କରିବାକୁ ନିଶ୍ଚିତ ହୁଅନ୍ତୁ** |

ଅନୁବାଦ ଏବଂ ପ୍ରକାଶନ କରିବା ପାଇଁ `i18n.site --npm` କିମ୍ବା `i18n.site -n` ଚଲାନ୍ତୁ `md`

ଯଦି ଆପଣ ପ୍ରକାଶନ ପାଇଁ ଏକ ନିରନ୍ତର ଏକୀକରଣ ପରିବେଶ ବ୍ୟବହାର କରନ୍ତି, ତେବେ ଏହାକୁ ସଂସ୍ଥାପନ କରିବାର କ is ଣସି ଆବଶ୍ୟକତା ନାହିଁ `nodejs` କେବଳ ଲଗ୍ ଇନ୍ ଏବଂ ପ୍ରକାଶିତ ଅନୁମତି `~/.npmrc` ପରିବେଶକୁ କପି କରନ୍ତୁ |

ଯଦି ଆପଣ ପ୍ୟାକେଜ୍ ନାମକୁ `v:` ରେ ପରିବର୍ତ୍ତନ କରନ୍ତି `ol.yml` ଦୟାକରି **ପ୍ରଥମେ `.i18n/v/ol` ବିଲୋପ କରିବାକୁ ନିଶ୍ଚିତ କରନ୍ତୁ** ଏବଂ ତାପରେ ଏହାକୁ ପ୍ରକାଶ କରନ୍ତୁ!

##### ପ୍ରକ୍ସି ସର୍ଭର npm ଦ୍ୱାରା ପ୍ରକାଶିତ |

ଯଦି ଚାଇନାର ମଧ୍ୟପ୍ରଦେଶର ଉପଭୋକ୍ତାମାନେ ନେଟୱର୍କ ସମସ୍ୟାର ସମ୍ମୁଖୀନ ହୁଅନ୍ତି ଏବଂ `npm` ପ୍ରକାଶ କରିବାକୁ ଅସମର୍ଥ ହୁଅନ୍ତି, ସେମାନେ ପ୍ରକ୍ସି ସର୍ଭରକୁ ବିନ୍ୟାସ କରିବାକୁ ପରିବେଶ ଭେରିଏବଲ୍ `https_proxy` ସେଟ୍ କରିପାରିବେ |

ତୁମର ପ୍ରକ୍ସି ସର୍ଭର ପୋର୍ଟ `7890` ତୁମେ ଲେଖି ପାରିବ:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### ଆତ୍ମ-ହୋଷ୍ଟ ବିଷୟବସ୍ତୁ |

ଯଦି ଆପଣ ବିଷୟବସ୍ତୁକୁ ସ୍ୱ-ହୋଷ୍ଟ କରିବାକୁ ଚାହାଁନ୍ତି, ପ୍ରଥମେ, ସଂପାଦନ `md/.i18n/htm/ol.yml` ଏବଂ `v: //unpkg.com/i18n.site` ଆପଣଙ୍କର URL ଉପସର୍ଗରେ ପରିବର୍ତ୍ତନ କରନ୍ତୁ, ଯେପରିକି `v: //i18n-v.xxx.com` |

ଡିରେକ୍ଟୋରୀ ପ୍ରବେଶ କରନ୍ତୁ ଏବଂ ଚଲାନ୍ତୁ `md`

```
i18n.site --htm_conf ol --save
```

କିମ୍ବା ସଂକ୍ଷିପ୍ତକରଣ

```
i18n.site -c ol -s
```

ତାପରେ, `v:` ରେ ଥିବା URL ଉପସର୍ଗ ପଥରେ ଥିବା ଡିରେକ୍ଟୋରୀରେ ଥିବା ବିଷୟବସ୍ତୁକୁ ବିନ୍ୟାସ କରନ୍ତୁ `md/out/ol/v`

ଶେଷରେ, **`1s` `/.v` ସମାପ୍ତ ହେଉଥିବା ପଥର କ୍ୟାଚ୍ ସମୟକୁ ବିନ୍ୟାସ କରନ୍ତୁ** , ନଚେତ୍ ନୂତନ ପ୍ରକାଶିତ ବିଷୟବସ୍ତୁକୁ ତୁରନ୍ତ ପ୍ରବେଶ କରାଯାଇପାରିବ ନାହିଁ |

ଅନାବଶ୍ୟକ ଅନୁରୋଧକୁ ହ୍ରାସ କରିବା ପାଇଁ ଅନ୍ୟ ପଥ ପାଇଁ କ୍ୟାଚ୍ ସମୟ ଏକ ବର୍ଷ କିମ୍ବା ତା’ଠାରୁ ଅଧିକ ସେଟ୍ ହୋଇପାରେ |

##### S3 କୁ ହୋଷ୍ଟ ବିଷୟବସ୍ତୁ |

+ host- ହୋଷ୍ଟ ବିଷୟବସ୍ତୁ ପାଇଁ, ଆପଣଙ୍କର ନିଜ ସର୍ଭର ବ୍ୟବହାର କରିବା `CDN` , ଅନ୍ୟ ଏକ ସାଧାରଣ ବିକଳ୍ପ ହେଉଛି ବ୍ୟବହାର କରିବା `S3`

ଆପଣ [rclone](https://rclone.org) `S3` ବ୍ୟବହାର କରିପାରିବେ, ତାପରେ ନିମ୍ନରେ ଥିବା ସ୍କ୍ରିପ୍ଟକୁ ଅନୁସରଣ କରନ୍ତୁ ଏବଂ ସଂଶୋଧନ କରନ୍ତୁ, ଏବଂ ଆପଣ ପ୍ରତ୍ୟେକ ଥର ପ୍ରକାଶନ ସମୟରେ ବର୍ଦ୍ଧିତ ପରିବର୍ତ୍ତନଗୁଡ଼ିକୁ କପି କରନ୍ତୁ `S3`

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

ବିନ୍ୟାସ କରିବାକୁ ମନେରଖ `CDN` ଯାହା ଦ୍ # ାରା `/.v` ରେ ଶେଷ ହେଉଥିବା ପଥର କ୍ୟାଚ୍ ସମୟ ହେଉଛି `1s` , ନଚେତ୍ ତୁମେ ତୁରନ୍ତ ନୂତନ ପ୍ରକାଶିତ ବିଷୟବସ୍ତୁକୁ ପ୍ରବେଶ କରିବାକୁ ସମର୍ଥ ହେବ ନାହିଁ |

`out/ol/htm` , ଚାଇନାର ମେଳା ବିକ୍ରେତାମାନଙ୍କର ଅବଜେକ୍ଟ ଷ୍ଟୋରେଜ୍ ବ୍ୟବହାର `CDN` +

ଏକକ ପୃଷ୍ଠା ପ୍ରୟୋଗଗୁଡ଼ିକ ସହିତ ଖାପ ଖୁଆଇବା ପାଇଁ ପଥକୁ ପୁନ r ଲିଖନ କରିବା ପାଇଁ ଆପଣ ଏଜ୍ କମ୍ପ୍ୟୁଟିଂ ବ୍ୟବହାର କରିପାରିବେ, ଯେପରିକି [Baidu ସ୍ମାର୍ଟ `CDN`](//cloud.baidu.com/product/cdn.html) ଏହା ଏହିପରି ବିନ୍ୟାସ ହୋଇପାରିବ:

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```

ଏଥିସହ, ମଧ୍ୟପ୍ରଦେଶର ଚାଇନାର କ୍ଲାଉଡ୍ ବିକ୍ରେତାଙ୍କ ବିଦେଶୀ ଟ୍ରାଫିକ୍ ଚାର୍ଜ ଅପେକ୍ଷାକୃତ ମହଙ୍ଗା, ଯଦି ଆପଣ ଖର୍ଚ୍ଚକୁ ଅପ୍ଟିମାଇଜ୍ କରିବାକୁ ଚାହାଁନ୍ତି, ତେବେ ଆପଣ [ହୁଆୱାଇ କ୍ଲାଉଡ୍ ର ମାଗଣା ଭ ograph ଗୋଳିକ DNS](https://support.huaweicloud.com/usermanual-dns/dns_usermanual_0041.html) ଏବଂ [Cloudflare for SaaS](https://developers.cloudflare.com/cloudflare-for-platforms/cloudflare-for-saas) ର କଷ୍ଟମ୍ ଡୋମେନ୍ ନାମ (ନିମ୍ନରେ ଦେଖାଯାଇଥିବା ପରି) ବ୍ୟବହାର କରିପାରିବେ | ଟ୍ରାଫିକ୍ ଡାଇଭର୍ସନ - ମଧ୍ୟପ୍ରଦେଶ ଚାଇନା Baidu Cloud ରେ `CDN` ଆନ୍ତର୍ଜାତୀୟ ଟ୍ରାଫିକ୍ cloudflare ଯାଏ |

ଏହି ନିୟୋଜନ ଅପ୍ଟିମାଇଜେସନ୍ ସମାଧାନଗୁଡିକ ଅଧିକ ଜଟିଳ ଏବଂ ଭବିଷ୍ୟତରେ ପୃଥକ ଅଧ୍ୟାୟରେ ଉପସ୍ଥାପିତ ହେବ |

#### ଜେନେରିକ୍ ଡୋମେନ୍ ନାମ ପୁନ ir ନିର୍ଦ୍ଦେଶ |

ଯଦି ତୁମେ ତୁମର ମୁଖ୍ୟ ୱେବସାଇଟ୍ ଭାବରେ ଏକ ୱେବସାଇଟ୍ ସୃଷ୍ଟି କରିବାକୁ ବ୍ୟବହାର `i18n.site` , ତୁମେ ସାଧାରଣତ pan ପ୍ୟାନ୍-ଡୋମେନ୍ ପୁନ ir ନିର୍ଦ୍ଦେଶକୁ ବିନ୍ୟାସ କରିବାକୁ ପଡିବ, ଅର୍ଥାତ୍ `*.xxx.com` ( `www.xxx.com` ଅନ୍ତର୍ଭୂକ୍ତ କରି) ପ୍ରବେଶକୁ ପୁନ ir ନିର୍ଦ୍ଦେଶ କର `xxx.com`

ଆଲିବାବା କ୍ଲାଉଡ୍ ର `EdgeScript` ( [ଇଂରାଜୀ ଡକ୍ୟୁମେଣ୍ଟ୍](https://www.alibabacloud.com/help/en/cdn/developer-reference/how-edgescript-works) / [ଚାଇନିଜ୍ ଡକ୍ୟୁମେଣ୍ଟ୍](https://help.aliyun.com/zh/cdn/developer-reference/edgescript) ) ସାହାଯ୍ୟରେ ଏହି ଆବଶ୍ୟକତା ହାସଲ ହୋଇପାରିବ `CDN`

[ଆଲିବାବା CDN](https://cdn.console.aliyun.com/domain/list) ରେ ଏକ ଡୋମେନ୍ ନାମ ଯୋଡନ୍ତୁ ଏବଂ `*.xxx.com` ନାମକୁ ଆଲିବାବା କ୍ଲାଉଡ୍ `CDN` କୁ ସୂଚିତ କରନ୍ତୁ `CNAME`

<img alt="" src="https://p.3ti.site/1721122000.avif">

ଉଦାହରଣ ସ୍ୱରୂପ, ଉପର ଚିତ୍ରରେ ପ୍ୟାନ୍-ଡୋମେନ୍ ନାମ ପୁନ ir ନିର୍ଦ୍ଦେଶ `*.i18n.site` ନିମ୍ନଲିଖିତ ଅଟେ:

```
rewrite(concat('https://i18n.site/',$uri), 'redirect',301)
```

#### Nginx ସହିତ ନିୟୋଜନ କରନ୍ତୁ |

ଦୟାକରି `server` ନିମ୍ନଲିଖିତ ପରି ଏକ ସଂରଚନା ଯୋଡନ୍ତୁ nginx ଯେଉଁଠାରେ `/root/i18n/md/out/ol/htm` ଦୟାକରି ଏହାକୁ ନିଜ ପ୍ରୋଜେକ୍ଟର ପଥରେ ପରିବର୍ତ୍ତନ କରନ୍ତୁ `out/ol/htm`

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```

### ଡିରେକ୍ଟୋରୀ ସଂରଚନା

#### ଜନସାଧାରଣ

ୱେବସାଇଟ୍ ର ଷ୍ଟାଟିକ୍ ଫାଇଲ୍, ଯେପରିକି `favicon.ico` `robots.txt` , ଇତ୍ୟାଦି |

ଏଠାରେ ଥିବା ଆଇକନ୍ ଫାଇଲଗୁଡିକ ସୃଷ୍ଟି ହୋଇପାରିବ [realfavicongenerator.net](https://realfavicongenerator.net)

#### .i18n

ଡିରେକ୍ଟୋରୀରେ `i18n.site` ର ବିନ୍ୟାସ ଫାଇଲ, ଅନୁବାଦ କ୍ୟାଚ୍ ଇତ୍ୟାଦି ଅଛି `.i18n` ବିବରଣୀ ପାଇଁ ପରବର୍ତ୍ତୀ ଅଧ୍ୟାୟ ["ବିନ୍ୟାସ"](/i18n.site/conf) ଦେଖନ୍ତୁ |

#### en

ବିନ୍ୟାସ ଫାଇଲରେ `fromTo` `en` `.i18n/conf.yml` ଉତ୍ସ ଭାଷା ଡିରେକ୍ଟୋରୀ |

```yaml
i18n:
  fromTo:
    en: zh
```
