# Усталяваць &

## Маркер Канфігурацыі

`i18n.site` Убудаваны `i18` інструмент перакладу, [пстрыкніце тут `i18` каб наладзіць маркер доступу](/i18/use) .

## Усталяваць

```sh
bash <(curl -sS https://i.i18n.site) i18n.site
```

### Паведамленне Кантэнту

`i18n.site` [аднастаронкавую архітэктуру прыкладання](https://developer.mozilla.org/docs/Glossary/SPA) , а старонка ўваходу на вэб-сайт і змест вэб-сайта разгортваюцца незалежна.

Пасля выканання прыведзенага вышэй перакладу ў `md/out/dev` будуць створаны каталогі `htm` і `v` .

Тут `dev` паказвае, што ён створаны на аснове файла `.i18n/htm/dev.yml` .

`dev` Пад зместам :

`htm` Пад каталогам знаходзіцца старонка ўваходу на сайт.

`v` Каталог змяшчае змест вэб-сайта з нумарам версіі.

Лакальны папярэдні прагляд скапіруе ўсе файлы ў каталог `out/dev/v/0.1.0` незалежна ад нумара версіі.

Для афіцыйнага выпуску змененыя файлы будуць скапіяваны ў каталог з новым нумарам версіі.

#### Выкарыстоўвайце -C, Каб Вызначыць Файл Канфігурацыі

Розныя файлы канфігурацыі будуць ствараць адпаведныя каталогі ў каталогу `out`

Напрыклад, `.i18n/htm/ol.yml` створыць каталог `out/ol` .

`dev.yml` і `ol.yml` - канфігурацыі па змаўчанні.

`dev` - гэта абрэвіятура `development` , якая ўяўляе асяроддзе распрацоўкі, выкарыстоўваецца для лакальнага папярэдняга прагляду, а таксама з'яўляецца файлам канфігурацыі па змаўчанні.
`ol` абрэвіятура `online` , якая прадстаўляе сеткавае асяроддзе, выкарыстоўваецца для афіцыйнага выпуску, а таксама з'яўляецца канфігурацыйным файлам пры публікацыі ў `npm` з выкарыстаннем параметра каманднага радка `-n` .

Вы таксама можаце стварыць іншыя файлы канфігурацыі `--htm_conf` каб указаць імя файла канфігурацыі.

напрыклад:
```
i18n.site --htm_conf yourConfig --save
```

Тут `--save` паказвае нумар версіі абнаўлення.

#### <a rel=id href="#npm" id="npm"></a> Публікуйце кантэнт на npmjs.com

Публікацыя кантэнту [npmjs.com](//npmjs.com) рэкамендаваным рашэнні па змаўчанні (гл. [Высокая даступнасць інтэрфейсу](/i18n.site/feature#ha) ).

##### npm login & Вызваленне

Усталюйце `nodejs` выкарыстоўвайце `npm login` для ўваходу.

Рэдагаваць `md/.i18n/htm/ol.yml` змяніце `i18n.site` у `v: //unpkg.com/i18n.site` на свой `npm` імя пакета.

Проста выкарыстоўвайце незанятае імя пакета ў якасці імя пакета [npmjs.com](//npmjs.com)

Пры публікацыі на аснове пакета `npm` **абавязкова выкарыстоўвайце `//unpkg.com/`** у якасці прэфікса значэння `v:` `i18n.site` пад гэтым прэфіксам `/.v` быў спецыяльна аптымізаваны для своечасовага прагляду новых выпускаў.

Запусціце `i18n.site --npm` або `i18n.site -n` у каталогу `md` , каб перакласці і апублікаваць.

Калі вы выкарыстоўваеце асяроддзе бесперапыннай інтэграцыі для публікацыі, няма неабходнасці ўсталёўваць яго `nodejs` Проста скапіруйце дазвол `~/.npmrc` для ўваходу ў сістэму і апублікаваны ў асяроддзе.

Калі вы зменіце назву пакета ў `v:` у `ol.yml` , **не забудзьцеся спачатку выдаліць `.i18n/v/ol` ,** а потым апублікаваць яго.

##### Проксі-Сервер, Апублікаваны npm

Калі карыстальнікі ў мацерыковым Кітаі сутыкнуліся з праблемамі сеткі і не могуць апублікаваць пакет `npm` , яны могуць усталяваць зменную асяроддзя `https_proxy` , каб наладзіць проксі-сервер.

Мяркуючы, што ваш порт проксі-сервера `7890` вы можаце напісаць:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### Самастойны Кантэнт

Калі вы хочаце самастойна размясціць змесціва, спачатку адрэдагуйце `md/.i18n/htm/ol.yml` і змяніце `v: //unpkg.com/i18n.site` на прэфікс URL-адраса, напрыклад `v: //i18n-v.xxx.com` .

Увайдзіце ў каталог `md`

```
i18n.site --htm_conf ol --save
```

або абрэвіятура

```
i18n.site -c ol -s
```

Затым наладзьце змесціва ў каталогу `md/out/ol/v` на прэфікс URL, усталяваны ў `v:` .

Нарэшце, **наладзьце час кэшавання шляху, які заканчваецца `1s` `/.v`**

Час захавання ў кэшы для іншых шляхоў можа быць устаноўлены на адзін год або больш, каб паменшыць непатрэбныя запыты.

##### Размясціце Кантэнт На s3

Для самастойнага размяшчэння кантэнту, у дадатак + выкарыстання ўласнага сервера, `CDN` выкарыстоўваць `S3`

Вы можаце выкарыстоўваць сервер [rclone](https://rclone.org) Login `S3` , затым звяртацца да сцэнарыя ніжэй і капіраваць толькі паступовыя змены пры `S3` публікацыі.

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

Не забудзьце наладзіць `CDN` каб час кэшавання шляху, які заканчваецца на `/.v` быў роўны `1s` , інакш вы не зможаце неадкладна атрымаць доступ да толькі што выпушчанага змесціва.

#### Аптымізацыя Разгортвання Вэб-Сайта Ў Кітаі

Калі вы хочаце атрымаць лепшую прадукцыйнасць даступнасці ў сеткавым асяроддзі мацерыковага Кітая, спачатку [зарэгіструйце даменнае імя](//beian.aliyun.com) .

Затым выкарыстоўвайце сховішча `out/ol/htm` у мацерыковым `CDN` +

Вы можаце выкарыстоўваць краявыя вылічэнні, каб перапісаць шлях для адаптацыі да аднастаронкавых прыкладанняў, такіх як [Baidu Smart Cloud `CDN`](//cloud.baidu.com/product/cdn.html) Яго можна наладзіць так:

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```

Паколькі запісы `MX` і `CNAME` запісы не могуць суіснаваць, калі вы хочаце атрымліваць электронныя лісты з даменнымі імёнамі, вам трэба супрацоўнічаць са скрыптам [cname_flatten](https://github.com/i18n-site/lib/tree/main/cname_flatten) , каб звесці `CNAME` у `A` запіс.

#### Універсальнае Перанакіраванне Даменнага Імя

Калі вы выкарыстоўваеце `i18n.site` стварыць вэб-сайт у якасці асноўнага вэб-сайта, вам звычайна трэба наладзіць пан-даменнае перанакіраванне, гэта значыць перанакіраваць `*.xxx.com` (уключаючы `www.xxx.com` ) доступ да `xxx.com` .

Гэта патрабаванне можа быць выканана з дапамогай Alibaba Cloud `CDN` `EdgeScript` ( [дакумент на англійскай мове](https://www.alibabacloud.com/help/en/cdn/developer-reference/how-edgescript-works) / [дакумент на кітайскай мове](https://help.aliyun.com/zh/cdn/developer-reference/edgescript) )

Дадайце даменнае імя ў [Alibaba CDN](https://cdn.console.aliyun.com/domain/list) і накіруйце `*.xxx.com` імя на `CNAME` Alibaba Cloud `CDN` .

#### Разгарнуць З Дапамогай nginx

Калі ласка, дадайце канфігурацыю, падобную да `server` nginx дзе `/root/i18n/md/out/ol/htm` змяніце яе на шлях вашага ўласнага праекта `out/ol/htm` :

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```

### Структура Каталога

#### Грамадскасць

Статычныя файлы вэб-сайта, такія як `favicon.ico` `robots.txt` і г.д.

Файлы значкоў тут можна стварыць з дапамогай [realfavicongenerator.net](https://realfavicongenerator.net)

#### .i18n

`.i18n` У каталогу знаходзяцца файлы канфігурацыі `i18n.site` , кэш перакладаў і г.д. Падрабязнасці глядзіце ў наступным раздзеле [«Канфігурацыя»](/i18n.site/conf) .

#### Ан

Каталог зыходнай мовы, які `.i18n/conf.yml` `fromTo` `en` у канфігурацыйным файле

```yaml
i18n:
  fromTo:
    en: zh
```

Калі ласка, звярніцеся да канфігурацыі перакладу [i18](/i18/use)
