## Нууц Үггүй Нэвтрэх Тохиргоог Хийнэ Үү

Хэдэн минутын дараа серверүүд бүгд идэвхжсэн.

Эхний хийх зүйл бол локал машины `~/.ssh/config` -г засварлаж IP нууц үггүй нэвтрэхийг тохируулахын тулд `ssh-copy-id root@c2` ашиглана уу.

<img alt="" src="https://i-01.eu.org/1718287198.avif">

Та эхлүүлэх тохиргоог хурдан дуусгахын тулд энэ [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) өөрчилж болно.

Нууц үггүй тохиргоо хийж дууссаны дараа та үүнийг багцаар нь ашиглах боломжтой бөгөөд энэ нь тохиромжтой [pdsh](https://github.com/chaos/pdsh)

Эхнийх нь ipv6-г гараар идэвхжүүлж, шалгах явдал юм

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

<img alt="" src="https://i-01.eu.org/1718287969.avif">

Дараа нь нэг товшилтоор системийг ubuntu 24 болгон сайжруулж, түгээмэл хэрэглэгддэг программ хангамжийг суулгана уу.

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 нь хэлийг хятад хэл рүү, цагийн бүсийг Бээжин рүү шилжүүлнэ гэсэн үг.

Анхдагчаар суулгасан программ хангамж нь , [direnv](https://direnv.net/) гэх мэт миний хувийн хэрэглээний дадал зуршилд нийцүүлэн тохируулагдсан [mise](https://github.com/jdx/mise) Шаардлагатай бол та [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) агуулахыг хуулж, шаардлагатай бол өөрчлөх боломжтой.

Энэ алхам нь удаан хугацаа шаардагддаг бөгөөд хагас цаг хүлээсний дараа түгээмэл хэрэглэгддэг програм хангамжийг шинэчлэх, суулгах ажил амжилттай болно. Та үүнийг дараах тушаалаар баталгаажуулж болно.

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## mosh Нь Ssh-Г Хурдасгадаг

Европоос Хятад руу чиглэсэн сүлжээ тогтвортой `ssh` байна. Хурдлахыг зөвлөж байна [mosh](//mosh.org)

Дараах байдлаар локал машины (голчлон орчны хувьсагч `PATH` ) санд `c0` скрипт үүсгэ `~/.bin`

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

Үүсгэсэний дараа гүйцэтгэх зөвшөөрөл болон зөөлөн холбоос `c1` `c2` ​​.

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

Цаашид та шууд `c0` `c1` оруулах боломжтой ба `mosh` өндөр хурдны шууд холболтыг ашиглаж болно [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

## Аврах Системийн Дэлгэрэнгүй Мэдээлэл

Аврах системээ оруулна уу 22 Порт руу нэвтрэх [tcping](https://github.com/paradiseduo/tcping) болсны дараа та 2-3 минутын турш түүнд нэвтрэх боломжгүй болно.

```
tcping ip 22 -c 9999
```

Портын хүртээмжийг хянах боломжтой.

### Сэргээх Хормын Хувилбарыг Оруулсны Дараа Хатуу Диск Рүү Хэрхэн Хандах Вэ &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

Жишээлбэл, та хормын хувилбарыг харж болно

```
snapper -c etc list
```

Хормын хувилбарын файлын өөрчлөлтийг харах

```
snapper -c etc status 5..6
```

Тодорхой ялгаатай текстийг харах

```
snapper -c etc diff 3..5
```

Хормын хувилбарыг сэргээх: Жишээлбэл, 5-р агшин зургийн дараах бүх өөрчлөлтийг буцаах

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### Бүртгэлийн Тогтвортой Байдлыг Идэвхжүүлэх

Аврах системийн дагуу бүртгэлийг үргэлжлүүлэхийг идэвхжүүлэхийн тулд танд тохирох лавлахыг гараар үүсгэж, зарим нэмэлт алхмуудыг хийх шаардлагатай, учир нь танд системийн үйлчилгээ байхгүй байж магадгүй юм. Та дараах алхмуудыг туршиж үзэж болно.

Эхлээд chroot, дараа нь

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

Ubuntu системээ дахин ачаална уу: Дээрх өөрчлөлтүүдийг хийсний дараа та аврах системийн оронд машинаа хэвийн болон Ubuntu системдээ дахин асаах хэрэгтэй.

Ийм байдлаар systemd-journald шинэ тохиргооноос эхэлж, логуудыг хатуу диск дээр байнга хадгалж эхэлнэ.

Дээрх үйлдлүүд нь аврах систем дээр хийгддэг тул эдгээр өөрчлөлтүүд нь системийг дахин ачаалж, аврах горимд байхаа больсны дараа л хүчин төгөлдөр болно гэдгийг санаарай.

Энэ нь та одоо байгаа логуудад гарсан өөрчлөлтүүдийг шууд харахгүй, харин үүний дараа үүсгэсэн бүх бүртгэлүүд хэвээр үлдэнэ гэсэн үг.

Ингэснээр та ssh үйлчилгээний бүртгэлийг харж болно

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## Хэцүү, Төвөгтэй Өвчний Талаархи Тэмдэглэл

### Хачирхалтай Асуудлыг Шийд: Сервер ssh Чадахгүй Байна

Би өмнө нь автомат эмхэтгэл болон байршуулах скрипт хийхийн тулд github үйлдлийг ашигласан боловч байршуулах үйлчилгээ нь ssh байж болохгүй гэдгийг олж мэдсэн.

Аврах системд нэвтэрч, бүртгэлийг харах нь зөвшөөрлийн асуудал юм:

`Missing privilege separation directory: /run/sshd`

Та дараах командыг ашиглан энэ санг үүсгэж болно

```
chown root:root /
systemd-tmpfiles --create
```

Давтан шалгалтын дараа, учир `1001` github action tar хэрэглэгч `uid`

tar Савыг `uid` хийх `1001` `--no-same-owner`

Үүний үр дүнд rsync нь үндсэн директорийн хэрэглэгчийн бүлгийг өөрчлөх бөгөөд дараа нь та дахин эхлүүлсний дараа ssh руу нэвтрэх боломжгүй болно.

Декомпресс скриптийг дараах байдлаар өөрчил:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
