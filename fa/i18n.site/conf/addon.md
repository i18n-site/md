# پلاگین

`.i18n/conf.yml` در آن می‌توان پیکربندی پلاگین‌ها کرد، مانند:

```yml
addon:
  - i18n.addon/toc
```

## پلاگین‌های رسمی

* `i18n.addon/toc` : فهرست راهنما
  بر اساس `TOC`، فهرست راهنمای `json` ایجاد می‌شود، به صورت پیش‌فرض فعال است

* `i18n.addon/mouse` : اثرات ماوس

## قرارداد نام فایل

پلاگین‌ها بسته‌های `npm` هستند.

بسته مرتبط با `i18n.addon/toc` در بالا [https://www.npmjs.com/package/@i18n.addon/toc](https://www.npmjs.com/package/@i18n.addon/toc) است.

این پلاگین به طور پیش‌فرض از آخرین نسخه استفاده می‌کند و به‌روزرسانی‌ها را هفتگی بررسی می‌کند.

اگر مایل به ثابت نگه داشتن نسخه هستید، می‌توانید `i18n.addon/toc@0.1.3` بنویسید.

دستور ترجمه `i18n.site` فایل‌های قراردادی بسته افزونه را نصب کرده و سپس اجرا می‌کند.

اسامی فایل‌های قراردادی به شرح زیر است

### htmIndex.js

`htmIndex.js` در انتهای `.i18n/htm/index.js` تزریق می‌شود.

که در آن `__CONF__` با نام پیکربندی فعلی (مثل `dev` یا `ol`) جایگزین می‌شود.

### afterTran.js

پس از اتمام ترجمه فراخوانی می‌شود و پارامترهای ارسالی به شرح زیر است.

* `lang_li` : لیست زبان‌ها، زبان اول زبان مبدأ است
* `changed` : لیست فایل‌های تغییر یافته
* `root` : دایرۀ اصلی پروژه

مقدار بازگشتی یک فرهنگ لغت است، مانند

```json
{
  file:{
    //  path: txt, for example :
    // "_.json": "[]"
  }
}
```

`file` : لیست فایل‌های خروجی، `path` : مسیر فایل، `txt` : محتوای فایل

## توابع داخلی

زمان اجرای داخلی `js` بر اساس توسعه二级 [boa](https://github.com/boa-dev/boa) ساخته شده است و توابع داخلی به شرح زیر می‌باشند:

* `wPath(path, txt)` : نوشتن در فایل
* `rTxt(path)` : خواندن فایل متنی
* `rBin(path)` : خواندن فایل باینری
* `rDir(dirpath)` : خواندن دایرکتوری، مقدار بازگشتی یک آرایه است: لیست دایرکتوری‌ها، لیست فایل‌ها

## راهنمای توسعه

توسعه افزونه‌ها می‌تواند به مرجع [https://github.com/i18n-site/addon](https://github.com/i18n-site/addon) مراجعه کند
