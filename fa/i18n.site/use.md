# نصب کنید &

## نشانه‌پیکربندی

`i18n.site` دارای یک ابزار ترجمه `i18` است، لطفاً [اینجا را کلیک کنید تا به سند `i18` برای پیکربندی نشانه دسترسی پیدا کنید](/i18/use) .

## نصب کنید

```sh
bash <(curl -sS https://i.i18n.site) i18n.site
```

## پروژه نمایشی

بیایید با یک پروژه نمایشی شروع کنیم تا نحوه استفاده از `i18n.site` را یاد بگیریم.

ابتدا مخزن دمو را کلون کنید و دستور زیر را اجرا کنید:

```
git clone https://github.com/i18n-site/demo.i18n.site.git md
git clone https://github.com/i18n-site/demo.i18n.site.docker.git docker
```

کاربران در سرزمین اصلی چین می‌توانند:

```
git clone https://atomgit.com/i18n/demo.i18n.site.git md
git clone https://atomgit.com/i18n/demo.i18n.site.docker.git docker
```

نام دایرکتوری کلون کد `demo.i18n.site` باید `md` باشد تا پیش‌نمایش محلی با `docker` تسهیل شود.

### انتشار محتوا

`i18n.site` معماری [برنامه تک‌صفحه‌ای](https://developer.mozilla.org/docs/Glossary/SPA) را اتخاذ می‌کند و صفحه ورودی وب‌سایت و محتوای وب‌سایت به طور مستقل مستقر می‌شوند.

پس از اجرای ترجمه فوق، دایرکتوری‌های `htm` و `v` تحت دایرکتوری `md/out/dev` ایجاد می‌شوند.

در اینجا، `dev` به این معنی است که بر اساس فایل پیکربندی `.i18n/htm/dev.yml` ساخته شده است.

دایرکتوری `dev`:

دایرکتوری `htm` صفحه ورودی وب‌سایت است.

دایرکتوری `v` حاوی محتوای وب‌سایت با شماره نسخه است.

پیش‌نمایش محلی به شماره نسخه اهمیتی ندارد و همه فایل‌ها را در دایرکتوری `out/dev/v/0.1.0` کپی می‌کند.

برای انتشار رسمی، فایل‌های تغییر یافته در دایرکتوری شماره نسخه جدید کپی می‌شوند.

#### از -c برای تعیین فایل پیکربندی استفاده کنید

فایل‌های پیکربندی مختلف دایرکتوری‌های مربوطه را در دایرکتوری `out` ایجاد می‌کنند.

مثلاً، `.i18n/htm/ol.yml` دایرکتوری `out/ol` را ایجاد می‌کند.

`dev.yml` و `ol.yml` تنظیمات پیش‌فرض هستند.

`dev` مخفف `development` است که محیط توسعه را نشان می‌دهد و برای پیش‌نمایش محلی استفاده می‌شود، همچنین فایل پیکربندی پیش‌فرض است.
`ol` مخفف `online` است که نشان‌دهنده محیط آنلاین است و برای انتشار رسمی استفاده می‌شود، همچنین فایل پیکربندی پیش‌فرض برای استفاده از دستور `-n` در `npm` است.

همچنین می‌توانید فایل‌های پیکربندی دیگری ایجاد کنید، از `--htm_conf` در خط فرمان برای تعیین نام فایل پیکربندی استفاده کنید

مثلاً:
```
i18n.site --htm_conf yourConfig --save
```

در اینجا `--save` نشان‌دهنده به‌روزرسانی شماره نسخه است.

انتشار محتوا [به](/1) [npmjs.com](/0)

##### ورود به npm و انتشار

`nodejs` را نصب کنید و با `npm login` وارد شوید.

`md/.i18n/htm/ol.yml` را ویرایش کنید و `i18n.site` در `v: //unpkg.com/i18n.site` را به نام بسته `npm` خود تغییر دهید.

فقط از نام بسته خالی استفاده کنید [npmjs.com](//npmjs.com) و استفاده از نام دامنه وب‌سایت به عنوان نام بسته انتخاب خوبی است.

هنگام انتشار بر اساس بسته `npm` `i18n.site` **حتماً از `//unpkg.com/` برای پیشوند 1 استفاده** `/.v` `v:`

برای ترجمه و انتشار، در دایرکتوری `md` دستور `i18n.site --npm` یا `i18n.site -n` را اجرا کنید.

اگر از یک محیط یکپارچه‌سازی مداوم برای انتشار استفاده می‌کنید، نیازی به نصب `nodejs` نیست. فقط مجوزهای ورود به سیستم و انتشار `~/.npmrc` را در محیط کپی کنید.

اگر نام بسته `v:` در `ol.yml` را تغییر دادید، **لطفاً ابتدا `.i18n/v/ol` را حذف کنید** و سپس منتشر کنید.

##### سرور پروکسی منتشر شده توسط npm

اگر کاربران در سرزمین اصلی چین با مشکلات شبکه مواجه شوند و نتوانند بسته `npm` را منتشر کنند، می‌توانند متغیر محیطی `https_proxy` را برای پیکربندی سرور پروکسی تنظیم کنند.

با فرض اینکه پورت سرور پروکسی شما `7890` باشد، می‌توانید به این صورت بنویسید:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### محتوای خود میزبان

اگر میخواهید محتوا را خود میزبانی کنید، ابتدا `md/.i18n/htm/ol.yml` را ویرایش کنید و `v: //unpkg.com/i18n.site` به پیشوند URL خود، مانند `v: //i18n-v.xxx.com` تغییر دهید.

به دایرکتوری `md` بروید و اجرا کنید

```
i18n.site --htm_conf ol --save
```

یا مخفف

```
i18n.site -c ol -s
```

سپس، محتوای موجود در دایرکتوری `md/out/ol/v` را در مسیر پیشوند URL تنظیم شده در `v:` پیکربندی کنید.

در نهایت، **زمان کش مسیری که به `/.v` ختم می‌شود را به `1s` تنظیم کنید**، در غیر این صورت نمی‌توان به محتوای تازه منتشر شده بلافاصله دسترسی پیدا کرد.

زمان کش برای مسیرهای دیگر را می‌توان روی یک سال یا بیشتر تنظیم کرد تا درخواست‌های غیرضروری کاهش یابد.

##### میزبانی محتوا در S3

برای خود میزبان کردن محتوا، علاوه بر استفاده از سرور خود، می‌توان از `S3`+`CDN` استفاده کرد.

برای ورود به سرور `S3` می توانید `S3` [rclone](https://rclone.org)

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

به یاد داشته باشید که `CDN` را طوری پیکربندی کنید که زمان کش مسیری که به `/.v` ختم می‌شود `1s` باشد، در غیر این صورت نمی‌توان به محتوای تازه منتشر شده بلافاصله دسترسی پیدا کرد.

#### بهینه‌سازی استقرار وب‌سایت در سرزمین اصلی چین

اگر می‌خواهید عملکرد دسترسی بهتری در محیط شبکه سرزمین اصلی چین به دست آورید، لطفاً ابتدا [یک نام دامنه ثبت کنید](//beian.aliyun.com) .

سپس، از ذخیره‌سازی اشیاء فروشندگان ابری در سرزمین اصلی چین استفاده کنید و محتوای زیر `out/ol/htm` را با `CDN` مستقر کنید.

می توانید [`CDN`](//cloud.baidu.com/product/cdn.html) محاسبات لبه برای بازنویسی مسیر برای انطباق با برنامه های تک صفحه ای استفاده کنید

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```
![](https://p.3ti.site/1721121273.avif)

از آنجایی که رکورد `MX` و رکورد `CNAME` نمی‌توانند همزمان وجود داشته باشند، اگر می‌خواهید ایمیل‌های نام دامنه را همزمان دریافت کنید، باید با [cname_flatten](https://github.com/i18n-site/lib/tree/main/cname_flatten) `CNAME` تا رکورد `A` همکاری کنید.

علاوه بر این، از آنجایی که هزینه‌های ترافیک خارج از کشور فروشندگان ابری در سرزمین اصلی چین نسبتاً گران است، اگر می‌خواهید هزینه‌ها را بهینه کنید، می‌توانید از [DNS وضوح جغرافیایی رایگان هواوی](https://support.huaweicloud.com/usermanual-dns/dns_usermanual_0041.html) و نام دامنه سفارشی [Cloudflare for SaaS](https://developers.cloudflare.com/cloudflare-for-platforms/cloudflare-for-saas) (همانطور که در زیر نشان داده شده است) استفاده کنید تا انحراف ترافیک──ترافیک در سرزمین اصلی چین باید به ابر `CDN` و ترافیک بین‌المللی به cloudflare هدایت شود.

![](https://p.3ti.site/1721119788.avif)

این راه‌حل‌های بهینه‌سازی استقرار پیچیده‌تر هستند و در فصل‌های جداگانه در آینده معرفی خواهند شد.

#### تغییر مسیر نام دامنه عمومی

اگر `i18n.site` برای ایجاد یک وب‌سایت به عنوان وب‌سایت اصلی خود استفاده می‌کنید، معمولاً باید تغییر مسیر پان‌دامنه را پیکربندی کنید، یعنی دسترسی به `*.xxx.com` (از جمله `www.xxx.com`) را به `xxx.com` تغییر دهید.

این نیاز را می‌توان با کمک Alibaba Cloud `CDN` `EdgeScript` ( [سند انگلیسی](https://www.alibabacloud.com/help/en/cdn/developer-reference/how-edgescript-works) / [سند چینی](https://help.aliyun.com/zh/cdn/developer-reference/edgescript)) به دست آورد

نام دامنه را در [Alibaba CDN](https://cdn.console.aliyun.com/domain/list) اضافه کنید و نام دامنه `CNAME` `*.xxx.com` `CDN` را تنظیم کنید.

![](https://p.3ti.site/1721122000.avif)

به عنوان مثال، پیکربندی تغییر مسیر پان‌دامنه `*.i18n.site` در تصویر بالا به شرح زیر است:

```
rewrite(concat('https://i18n.site/',$uri), 'redirect',301)
```

![](https://p.3ti.site/1721121934.avif)

#### استقرار با nginx

لطفاً یک پیکربندی شبیه به زیر در پاراگراف `server` اضافه کنید nginx و `/root/i18n/md/out/ol/htm` را به مسیر پروژه `out/ol/htm` خود تغییر دهید

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```

### ساختار دایرکتوری

#### `public`

فایل‌های ثابت وب‌سایت، مانند `favicon.ico`، `robots.txt` و غیره.

این فایل‌های نماد را می‌توان با [realfavicongenerator.net](https://realfavicongenerator.net) ایجاد کرد.

#### `.i18n`

در زیر دایرکتوری `.i18n` فایل‌های پیکربندی، حافظه پنهان ترجمه و غیره `i18n.site` قرار دارند. برای جزئیات به فصل بعدی ["پیکربندی"](/i18n.site/conf) مراجعه کنید.

#### `en`

فهرست راهنمای زبان مبدأ، مربوط به فایل پیکربندی `en` از `fromTo` در `.i18n/conf.yml`

```yaml
i18n:
  fromTo:
    en: zh
```

لطفاً به پیکربندی ترجمه مراجعه کنید [i18](/i18/use)
