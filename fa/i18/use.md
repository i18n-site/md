# نصب و استفاده کنید

## ویندوز ابتدا git bash را نصب کنید

windows [برای دانلود و نصب اینجا کلیک `git bash`](https://git-scm.com/download/win) .

عملیات بعدی را در `git bash`

## نصب

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### پیکربندی نشانه ترجمه

برای کپی کردن توکن کلیک [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

ایجاد کنید `~/.config/i18n.site.yml` محتوای کپی شده را در آن قرار دهید، محتوا به شرح زیر است:

```
token: YOUR_API_TOKEN
```

علاوه بر این، برای پرداخت باید یک کارت اعتباری [i18n.site/payBill](//i18n.site/payBill) کنید (بدون نیاز به شارژ مجدد، وب سایت به طور خودکار هزینه ها را با توجه به استفاده کسر می کند، [برای قیمت به صفحه اصلی مراجعه کنید](/#price) ).

## استفاده کنید

### پروژه نمایشی

لطفاً برای یادگیری پیکربندی ترجمه به پروژه آزمایشی مراجعه `i18` [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

کاربران در چین می توانند کلون کنند [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

پس از شبیه سازی، وارد دایرکتوری شوید و ترجمه را اجرا کنید `i18`

### ساختار دایرکتوری

ساختار دایرکتوری انبار قالب به شرح زیر است

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

فهرست `en`

### ترجمه را اجرا کنید

دایرکتوری را وارد کنید و برای ترجمه اجرا `i18` .

### فایل ها را به مخزن اضافه کنید

علاوه بر ترجمه، برنامه فایل های زیر را نیز تولید می کند، لطفا آنها را به مخزن اضافه کنید.

```
.i18n/hash
.i18n/cache/.gitignore
```

در میان آنها : محتوای `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

این بدان معناست که همه فایل‌های موجود در `.i18n/cache/` را نادیده بگیرید (به جز `.i18n/cache/.gitignore` ).

اگر نرم افزار کنترل نسخه شما نیست `git` لطفاً آن را مطابق این پیکربندی نادیده بگیرید.

## فایل پیکربندی

این فایل پیکربندی `i18` ابزار ترجمه خط فرمان `.i18n/conf.yml`

مطالب به شرح زیر است:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### زبان مبدأ &

در فایل پیکربندی، زیردستان `fromTo`

زبان مبدأ است `en` `zh ja ko de fr` زبان مقصد ترجمه است.

کد زبان را ببینید [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

به عنوان مثال، اگر می خواهید چینی را به انگلیسی ترجمه کنید، این خط را بازنویسی کنید `zh: en`

اگر می‌خواهید به همه زبان‌های پشتیبانی‌شده ترجمه کنید، لطفاً بعد از آن را خالی بگذارید `:` مثلا

```
i18n:
  fromTo:
    en:
```

### نادیده گرفتن فایل

به طور پیش فرض، تمام فایل هایی که با `.md` و `.yml` شروع می شوند در فهرست زبان مبدا ترجمه می شوند.

اگر می خواهید فایل های خاصی را نادیده بگیرید و آنها را ترجمه نکنید (مانند پیش نویس های ناتمام)، می توانید از تنظیمات `ignore` استفاده کنید.

`ignore` نحوی شبیه به `.gitignore` استفاده می کند [globset](https://docs.rs/globset/latest/globset/#syntax)

به عنوان مثال، در فایل پیکربندی بالا به این معنی `_* ` که فایل هایی که با `_` شروع می شوند، ترجمه نمی شوند.

## قوانین ترجمه

### ویراستاران ترجمه نباید خطوطی را اضافه یا حذف کنند

ترجمه قابل ویرایش است. متن اصلی را اصلاح کنید و دوباره آن را به صورت ماشینی ترجمه کنید، تغییرات دستی ترجمه بازنویسی نخواهد شد (اگر این پاراگراف از متن اصلی اصلاح نشده باشد).

اما توجه داشته باشید که خطوط ترجمه و متن اصلی باید یک به یک مطابقت داشته باشند. یعنی هنگام کامپایل ترجمه سطر اضافه یا حذف نکنید. در غیر این صورت باعث سردرگمی در حافظه پنهان ویرایش ترجمه می شود.

اگر مشکلی پیش آمد، لطفاً [برای راه حل ها به سؤالات متداول مراجعه کنید.](/i18/qa#H1)

### `YAML` ترجمه کردن

ابزار خط فرمان تمام فایل‌هایی که با `.yml` در فهرست فایل زبان مبدأ ختم می‌شوند را پیدا می‌کند و آنها را ترجمه می‌کند.

* توجه داشته باشید که پسوند نام فایل باید `.yml` (نه `.yaml` ).

این ابزار فقط مقادیر فرهنگ لغت را به `.yml` ترجمه می کند، نه کلیدهای فرهنگ لغت.

مثلا `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

به عنوان ترجمه خواهد شد `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

ترجمه `YAML` را نیز می توان به صورت دستی تغییر داد (اما کلیدها یا خطوط را در ترجمه اضافه یا حذف نکنید).

بر اساس `YAML` ، شما به راحتی می توانید راه حل های بین المللی برای زبان های برنامه نویسی مختلف بسازید.

## استفاده پیشرفته

### زیرمجموعه ترجمه

تا زمانی که ایجاد `.i18n/conf.yml` (نیازی نیست هر بار از قالب پروژه آزمایشی شروع کنید)، `i18` خوب کار می کند.

ابزار خط فرمان، پیکربندی `.i18n/conf.yml`

پروژه هایی که [monorepo](//monorepo.tools) معماری استفاده می کنند می توانند فایل های زبان را به زیر شاخه ها تقسیم کنند.

<img alt="" src="https://p.3ti.site/1719910016.avif">

### دایرکتوری نصب سفارشی

به طور پیش فرض در `/usr/local/bin`

`/usr/local/bin` مجوز نوشتن نداشته باشد در `~/.bin` نصب خواهد شد.

تنظیم متغیرهای محیطی شما می توانید دایرکتوری نصب : تعریف کنید `TO`

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
