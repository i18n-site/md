# نصب و استفاده کنید

## ویندوز ابتدا git bash را نصب کنید

لطفاً [برای دانلود و نصب `git bash` اینجا را کلیک کنید](https://git-scm.com/download/win)

عملیات بعدی را در `git bash` اجرا کنید.

## نصب کنید

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### پیکربندی نشانه ترجمه

برای کپی کردن توکن کلیک [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="width:400px">

`~/.config/i18n.site.yml` را ایجاد کنید و محتوای کپی شده را در آن قرار دهید. محتوا به شرح زیر است:

```
token: YOUR_API_TOKEN
```

علاوه بر این، برای پرداخت باید یک کارت اعتباری [i18n.site/payBill](//i18n.site/payBill) کنید (بدون نیاز به شارژ مجدد، وب سایت به طور خودکار هزینه ها را با توجه به استفاده کسر می کند، [برای قیمت به صفحه اصلی مراجعه کنید](/#price) ).

## استفاده کنید

### پروژه نمایشی

لطفاً برای یادگیری پیکربندی ترجمه `i18` به پروژه آزمایشی مراجعه کنید [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

کاربران در چین می توانند کلون کنند [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

پس از کلونینگ وارد دایرکتوری شده و `i18` را اجرا کنید تا ترجمه کامل شود.

### ساختار دایرکتوری

ساختار دایرکتوری انبار قالب به شرح زیر است:

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

فایل‌های نمایشی ترجمه شده در دایرکتوری `en` فقط یک نمونه هستند و قابل حذف هستند.

### ترجمه را اجرا کنید

دایرکتوری را وارد کنید و `i18` را برای ترجمه اجرا کنید.

程序 علاوه بر ترجمه، پوشه `.i18n/data` را نیز تولید می‌کند، لطفاً آن را به مخزن اضافه کنید.

پس از ترجمه فایل‌های جدید، در این دایرکتوری فایل داده‌های جدیدی ایجاد می‌شود، به خاطر بسپارید که `git add .` را اضافه کنید.

## فایل پیکربندی

`.i18n/conf.yml` فایل پیکربندی ابزار ترجمه خط فرمان `i18` است

مطالب به شرح زیر است:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### زبان مبدأ &

در فایل پیکربندی، تابع `fromTo`:

`en` زبان مبدأ، `zh ja ko de fr` زبان مقصد ترجمه است.

کد زبان را ببینید [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

به عنوان مثال، اگر می‌خواهید چینی را به انگلیسی ترجمه کنید، این خط را `zh: en` بازنویسی کنید.

اگر می‌خواهید به همه زبانهای پشتیبانی شده ترجمه کنید، لطفاً بعد از `:` آن را خالی بگذارید. برای مثال:

```
i18n:
  fromTo:
    en:
```

شما می‌توانید `fromTo` مختلف را برای زیرشاخه‌ها/فایل‌ها پیکربندی کنید:

```
i18n:
  fromTo:
    en:
  path:
    blog:
      fromTo:
        zh:
    blog/your_file_name.md:
      fromTo:
        ja:
```

در این جدول پیکربندی، زبان مبدأ ترجمه کاتالوگ `blog` `zh` و زبان مبدأ ترجمه `blog/your_file_name.md` `ja` است.

### نادیده گرفتن فایل

به طور پیش‌فرض، تمام فایل‌هایی که با `.md` و `.yml` در فهرست زبان مبدأ شروع می‌شوند، ترجمه خواهند شد.

اگر می‌خواهید فایل‌های خاصی را نادیده بگیرید و آنها را ترجمه نکنید (مانند پیش‌نویس‌های ناتمام)، می‌توانید آن را با فیلد `ignore` پیکربندی کنید.

`ignore` از دستور `.gitignore` استفاده می‌کند [globset](https://docs.rs/globset/latest/globset/#syntax).

به عنوان مثال، `_*` در فایل پیکربندی بالا به این معنی است که فایل‌هایی که با `_` شروع می‌شوند ترجمه نمی‌شوند.

## قوانین ترجمه

### ویراستاران ترجمه نباید خطوطی را اضافه یا حذف کنند

ترجمه قابل ویرایش است. متن اصلی را اصلاح کنید و دوباره آن را به صورت ماشینی ترجمه کنید، تغییرات دستی ترجمه بازنویسی نمی‌شود (اگر این پاراگراف از متن اصلی اصلاح نشده باشد).

اما توجه داشته باشید که خطوط ترجمه و متن اصلی باید یک به یک مطابقت داشته باشند. یعنی هنگام کامپایل ترجمه سطر اضافه یا حذف نکنید. در غیر این صورت باعث سردرگمی در حافظه پنهان ویرایش ترجمه می‌شود.

اگر مشکلی پیش آمد، لطفاً [به سوالات متداول مراجعه کنید](/i18/qa#H1)

### `YAML` ترجمه

ابزار خط فرمان همه فایلهایی که با `.yml` ختم می‌شوند در فهرست فایل زبان مبدأ پیدا کرده و آنها را ترجمه می‌کند.

* توجه داشته باشید که پسوند نام فایل باید `.yml` باشد (نه `.yaml`).

این ابزار فقط مقادیر فرهنگ لغت را به `.yml` ترجمه می‌کند، نه کلیدهای فرهنگ لغت.

به عنوان مثال `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

`i18n/zh/i18n.yml` ترجمه خواهد شد

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

ترجمه `YAML` را نیز می توان به صورت دستی تغییر داد (اما کلیدها یا خطوط را در ترجمه اضافه یا حذف نکنید).

بر اساس ترجمه `YAML` ، می توانید به راحتی راه حل های بین المللی برای زبان های برنامه نویسی مختلف بسازید.

## زیرمجموعه ترجمه

### زیرمجموعه ترجمه

تا زمانی که `.i18n/conf.yml` ایجاد شود (نیازی نیست هر بار از قالب پروژه آزمایشی شروع کنید)، `i18` به خوبی کار خواهد کرد.

ابزار خط فرمان `.i18n/conf.yml` پیکربندی را در همه زیر شاخه ها پیدا کرده و آنها را ترجمه می کند.

پروژه هایی که [monorepo](//monorepo.tools) معماری استفاده می کنند می توانند فایل های زبان را به زیر شاخه ها تقسیم کنند.

![](https://p.3ti.site/1719910016.avif)

### دایرکتوری نصب سفارشی

به طور پیش فرض روی `/usr/local/bin` نصب می شود.

اگر `/usr/local/bin` مجوز نوشتن نداشته باشد روی `~/.bin` نصب می شود.

تنظیم متغیر محیطی `TO` می تواند دایرکتوری نصب را تعریف کند، به عنوان مثال :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
