# Versi Proyek

Njupuk proyek demo minangka conto:

`en/demo2/v` yaiku nomer versi proyek saiki, sing bakal ditampilake ing sisih tengen jeneng proyek ing garis garis sidebar.

<img src="https://p.3ti.site/1721290486.avif" width="320px">

`en/` minangka kode basa sing cocog karo basa sumber terjemahan sing `.i18n/conf.yml` .

Yen basa sumber sampeyan dudu basa Inggris, file `v` kudu diselehake ing direktori proyek basa sumber sampeyan. 

Kemampuan kanggo browsing versi sejarah dokumen lagi dikembangake.

Disaranake mung ngowahi nomer versi dokumen nalika ngeculake update utama (kayata `v1` , `v2` ) kanggo ngindhari kaca-kaca sing diindeks mesin telusur amarga kakehan nomer versi.

## Gunakake File `v` Kanggo Pamisah Indeks File Kanggo Macem-Macem Proyek

Ing proyek demo, saliyane kanggo `en/demo2/v` , sampeyan uga bisa ndeleng manawa isi direktori `en/blog` lan `en/demo1` kosong `v` file.

Kosong `v` ora bakal ditampilake ing garis garis sidebar, nanging anggere file `v` ana, indeks independen bakal digawe kanggo file ing direktori lan subdirektori.

Kanthi pamisah indeks proyek sing beda-beda, sampeyan bisa ngindhari akses alon amarga ngemot indeks kabeh file ing kabeh situs bebarengan.

[https://unpkg.com/i18n.site/en/blog.json](https://unpkg.com/i18n.site/en/blog.json) : ing proyek demo `blog`
