## Куллану

### Демо Проекты

Зинһар `i18` тәрҗемә конфигурациясен өйрәнү өчен демо проектына мөрәҗәгать итегез [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

Кытайда кулланучылар клон ясый ала [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

Клонлаштырганнан соң, каталогка керегез һәм тәрҗемәне тәмамлагыз `i18`

### Белешмәлек Структурасы

Шаблон склад каталог структурасы түбәндәгечә

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` тәрҗемә ителгән демо файллар бар, алар бары тик мисал һәм бетерелергә мөмкин!

### Тәрҗемә Эшләгез

Белешмәлекне кертегез һәм тәрҗемә итегез `i18`

### Резервуарга Файллар Өстәгез

Тәрҗемә итүгә өстәп, программа шулай ук ​​түбәндәге файлларны чыгарачак, зинһар, аларны складка өстәгез.

```
.i18n/hash
.i18n/cache/.gitignore
```

Алар : эчтәлеге түбәндәгечә `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

Димәк, `.i18n/cache/` барлык файлларны санга сукмагыз ( `.i18n/cache/.gitignore` дан кала)!

Әгәр дә сезнең версия белән идарә итү программасы булмаса `git` зинһар, аны бу конфигурация буенча санга сукмагыз!

## Конфигурация Файлы

`.i18n/conf.yml` `i18` линия тәрҗемә коралының конфигурация файлы

Эчтәлеге түбәндәгечә:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Чыганак Теле Тәрҗемә Теле &

Конфигурация файлында, кул астында эшләүчеләр `fromTo`

`en` чыганак теле, `zh ja ko de fr` тәрҗемә итүнең максатлы теле!

Тел коды [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

Әйтик, кытай телен инглиз теленә тәрҗемә итәсегез килсә, бу юлны яңадан языгыз `zh: en`

Барлык телләргә дә тәрҗемә итәсегез килсә, зинһар, буш калдырыгыз `:` Мәсәлән

```
i18n:
  fromTo:
    en:
```

### Файлны Санга Сукмагыз

Килешү буенча, барлык чыганаклар һәм `.yml` тел каталогындагы тәрҗемә ителәчәк `.md`

Әгәр дә сез кайбер файлларны санга сукмыйсыз һәм аларны тәрҗемә итмисез икән (мәсәлән, тәмамланмаган шашка), сез кыр кыры конфигурациясен куллана аласыз `ignore`

`.gitignore` охшаган синтаксис [globset](https://docs.rs/globset/latest/globset/#syntax) `ignore`

Мәсәлән, югарыдагы конфигурация файлында `_` белән башланган файллар тәрҗемә ителмәячәк дигән сүз `_* `

## Тәрҗемә Кагыйдәләре

### Тәрҗемә Редакторлары Сызыклар Өстәргә Яки Бетерергә Тиеш Түгел

Тәрҗемә редакцияләнә. Оригиналь текстны үзгәртегез һәм аны яңадан тәрҗемә итегез, тәрҗемәгә кул белән үзгәртүләр язылмаячак (әгәр төп текстның бу абзацы үзгәртелмәгән булса).

Ләкин зинһар, онытмагыз, тәрҗемә юллары һәм оригиналь текст бер-берсенә туры килергә тиеш. Ягъни, тәрҗемәне төзегәндә сызыклар өстәмәгез яки бетермәгез. Otherwiseгыйсә, ул тәрҗемә редакцияләү кэшында буталчык тудырачак.

Әгәр дә берәр нәрсә дөрес булмаса, [чишелешләр өчен Сорауларга мөрәҗәгать итегез.](/i18/qa#H1)

### Тәрҗемә Итү `YAML`

Команда сызыгы барлык файлларны чыганак тел файллары каталогында табачак һәм тәрҗемә итәчәк `.yml`

* Игътибар итегез, файл исеме суффиксы булырга тиеш ( `.yaml` түгел) `.yml`

Бу корал сүзлек кыйммәтләрен генә тәрҗемә итә `.yml`

Мәсәлән `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

кебек тәрҗемә ителәчәк `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

Тәрҗемә `YAML` шулай ук ​​кул белән үзгәртелергә мөмкин (ләкин тәрҗемәдә ачкычларны яки сызыкларны кушмагыз).

`YAML` нигезләнеп, сез төрле программалаштыру телләре өчен халыкара чишелешләр төзи аласыз.

## Алдынгы Куллану

### Тәрҗемә Бүлекчәсе

`.i18n/conf.yml` ясаган вакытта (демо проект шаблоныннан башларга кирәк түгел), `i18` яхшы эшләячәк.

Команда сызыгы `.i18n/conf.yml` барлык бүлекчәләрдә табылачак һәм тәрҗемә ителәчәк!

[monorepo](//monorepo.tools) кулланган проектлар тел файлларын бүлекчәләргә бүләргә мөмкин!

### Махсус Урнаштыру Каталогы

Килешү буенча урнаштырылачак `/usr/local/bin`

`/usr/local/bin` язу рөхсәте булмаса, ул `~/.bin` урнаштырылачак.

Әйләнә-тирә үзгәрүчәннәрне куегыз `TO` мәсәлән, урнаштыру каталогын билгели аласыз :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
