# .i18n/conf.yml

`i18n.site` өчен профиль `.i18n/conf.yml` .

[`i18`](/i18) , `ignore:` һәм `i18n:` көйләүләреннән кала, конфигурация файллары түбәндәгечә:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
addon:
  - i18n.addon/toc
```

Алар арасында `upload` - `ext:` конфигурация пункты бастырганда `.md` генә йөкләнәчәк дигән сүз.

## Югары Навигация Диңгезе

Баш битнең башындагы навигация менюсына туры килгән `nav:` конфигурация параметрлары.

<img src="//p.3ti.site/1721051426.avif" style="width:320px">

Алар арасында `i18n: home` `en/i18n.yml` `home: Home` туры килә.

`en/i18n.yml` күп телләргә тәрҗемә ителәчәк, мәсәлән `zh/i18n.yml` .

<img src="//p.3ti.site/1721051689.avif" style="width:320px">

Тәрҗемә тәмамлангач, сез `yml` тәрҗемә кыйммәтен үзгәртә аласыз, ләкин `yml` тәрҗемә ачкычын кушмагыз яки бетермәгез.

### `use: Toc` , Бер Файл Шаблоны (План Белән)

`nav` :

```
  - i18n: home
    use: Toc
    url: /
```

`use: Toc` - `Markdown` шаблонны күрсәтүче `Toc` шаблон ярдәмендә күрсәтүне аңлата.

`TOC` - кыскарту `Table of Contents` Бу шаблон күрсәтелгәндә, бу `Markdown` файлның сызыгы кырыйда күрсәтеләчәк.

`url:` файл юлын күрсәтә ( `/` төп каталогка туры килә `/README.md` бу файл `Markdown` баш хәреф префиксы һәм кечкенә хәреф суффиксын таләп итә).

### `use: Md` , Бер Файл Шаблоны (План Юк)

`Md` шаблон һәм `Toc` шаблон бер үк, икесе дә бер `Markdown` файл күрсәтү өчен кулланыла. Ләкин `Md` шаблон кыр сызыгында планны күрсәтми.

Сез югарыдагы конфигурациядәге `use: Toc` `use: Md` үзгәртә аласыз, `md` каталогта `i18n.site` эшләтеп җибәрә аласыз, аннары баш биттәге үзгәрешләрне күзәтү өчен үсешне карау URL-ка керә аласыз.

### Конфигурация Юлы Булмаган Килеш Йөкләү

Әгәр дә билгеле бер юлга керсәләр һәм аның юл префиксы `nav:` белән конфигурацияләнмәсә, юлга туры килгән `MarkDown` файл килешү буенча йөкләнәчәк һәм `Md` шаблон ярдәмендә күрсәтеләчәк.

Мәсәлән, `/test` рөхсәтле булса, һәм `nav:` бу юлсыз конфигурацияләнсә, һәм бит теле инглиз булса ( `en` код), `/en/test.md` килешү буенча йөкләнәчәк һәм `Md` шаблон ярдәмендә күрсәтеләчәк.

`/en/test.md` бу файл булмаса, килешү буенча `404` бит күрсәтеләчәк.

### `use: Doc` , Күп Файллы Шаблон

Конфигурация файлында:

```
  - i18n: blog
    use: Doc
```

Шаблонны күрсәтү өчен `Doc` куллануны күрсәтә.

`Doc` шаблон бер яки берничә проект өчен документ схемаларын булдыру өчен берничә `MarkDown` интеграцияләүне хуплый.

#### Бер Проект (Берничә Файл)

Aboveгарыдагы `blog` - `Doc` пунктлы режим.

```
  - i18n: blog
    url: blog
    use: Doc
```

##### url Буш Булганда, Ул i18n Кыйммәтенә Туры Килми

Әгәр дә `url` язылмаган булса, `url` бәясе `i18n` дефолт. Бу кагыйдә башка шаблоннар өчен дә көченә керә.

Aboveгарыдагы язу ысулы `url: blog` тигез, һәм аңа туры килгән файл `en/blog/TOC` .

#### Берничә Проект

`.i18n/conf.yml` `i18n:doc` конфигурациясе - күп проектлы режим.

```
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
```

<img src="https://p.3ti.site/1721275191.avif" width="320px">

Монда, `menu: NB demo1,demo2` , тамчы менюны күрсәтү өчен `NB` шаблон куллануны аңлата.

`NB` , бу `Name Breif` кыскартылуы, тамчы меню проектның исемен һәм лозунгын күрсәтә ала дигән сүз.

`NB` аннан соң аңа бирелгән `demo1,demo2` параметр белән бара.
Игътибар итегез : ** `demo1,demo2` `,` кадәр һәм аннан соң ** урыннар булырга тиеш түгел.

Aboveгарыдагы параметрлар өчен тиешле каталог индексы файлы:

* `en/demo1/TOC`
* `en/demo2/TOC`

#### TOC Эчтәлек Индексы Таблицасы

`i18n.site` демо складында `js` плагин `.i18n/hook/after.tran/TOC.js` эшләячәк, `json` каталог схемасын булдыру өчен `TOC` шаблон конфигурациясенә туры килгән `doc` каталог индекс файлын уку өчен.

`doc` шаблон куллансагыз, бу плагин булырга тиеш.

Әгәр дә сез `i18n.site` проектны буш папкадан башласагыз, `.i18n` демо проектны сезнең каталогка күчерергә онытмагыз.

`Doc` шаблон ясалган `json` нигезендә эчтәлек таблицасын күрсәтәчәк.

##### ТОКны Күпләп Ничек Күчерергә?

`TOC` файл чыганак тел каталогына урнаштырылырга тиеш.

Мәсәлән, чыганак теле кытай булса, өстә `TOC` - `zh/blog/TOC` .

Чыганак теле үзгәртелсә, проекттагы билгеле бер телнең `TOC` файлын бүтән телгә күчерергә кирәк.

Сез түбәндәге боерыкларга мөрәҗәгать итә аласыз:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

Зинһар, югарыдагы боерыктагы `en/` һәм `zh/` тел кодыгызга үзгәртегез.
