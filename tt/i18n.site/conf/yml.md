# .i18n / conf.yml

Конфигурация файлы `.i18n/conf.yml` `i18n.site`

`ignore:` Һәм `i18n:` көйләүләреннән кала [`i18`](/i18) конфигурация файл түбәндәгечә:

```yaml
upload:
  ext:
    - md
nav:
  - i18n: home
    use: Toc
    url: /
  - i18n: doc
    menu: NB demo1,demo2
    use: Doc
  - i18n: blog
    use: Doc
```

Алар арасында `ext:` конфигурация пункты - бастырганда `.md` йөкләнәчәк дигән сүз `upload`

## Югары Навигация Диңгезе

баш битнең башындагы навигация менюсына туры килгән конфигурация параметрлары `nav:`

### , Бер Файл Шаблоны (План Белән) `use: Toc`

: `nav`

```
  - i18n: home
    use: Toc
    url: /
```

бер `Markdown` күрсәтүче `Toc` рендерингын куллануны аңлата `use: Toc`

`TOC` `Table of Contents` кыскартуы. Бу шаблон күрсәтелгәндә, бу `Markdown` файлның схемасы кырыйда күрсәтеләчәк.

`Markdown` файл юлын күрсәтә ( `/` төп каталогка туры килә `/README.md` `url:` бу файл исеме баш хәреф префиксы һәм кечкенә хәреф суффиксы таләп итә).

### , Бер Файл Шаблоны (План Юк) `use: Md`

Шаблон `Md` `Toc` бертигез, икесе дә бер `Markdown` файл күрсәтү өчен кулланыла. Ләкин `Md` шаблон кырыйдагы планны күрсәтми.

Сез югарыдагы конфигурациядә `use: Toc` үзгәртә аласыз, яңадан `i18n.site` `md` каталогында `use: Md` , аннары баш биттәге үзгәрешләрне күзәтү өчен үсешне карау URL'ына керегез.

### Конфигурация Юлы Булмаган Килеш Йөкләү

Әгәр дә билгеле бер юлның юл префиксы конфигурацияләнмәгән булса `nav:` юлга туры килгән `MarkDown` файл килешү буенча йөкләнәчәк һәм `Md` шаблон ярдәмендә күрсәтеләчәк.

Мисал өчен, сез `/test` , һәм `nav:` бу юлсыз конфигурацияләнсә, һәм бит теле инглиз (код `en` ) булса, шаблон килешү буенча йөкләнәчәк `/en/test.md` һәм кулланыла `Md` .

Әгәр `/en/test.md` бу файл булмаса, килешү `404` бите күрсәтеләчәк!

### Күп Файллы Шаблон `use: Doc`

Конфигурация файлында:

```
  - i18n: blog
    use: Doc
```

Шаблонны күрсәтү өчен куллануны күрсәтә `Doc`

Шаблон бер яки берничә проект өчен документ схемаларын булдыру өчен берничә `MarkDown` интеграцияләүне хуплый `Doc`

#### Бер Проект (Берничә Файл)

aboveгарыда `Doc` проектның бер проектлы режимы `blog`

```
  - i18n: blog
    url: blog
    use: Doc
```

##### url Буш Булганда, Ул i18n Кыйммәтенә Туры Килми

Әгәр дә сез язмасагыз `url` `i18n` кыйммәтенә `url` . Бу кагыйдә башка шаблоннар өчен дә көченә керә.

Aboveгарыдагы язу ысулы булганга тигез `url: blog` Һәм аңа туры килгән файл `en/blog/TOC` .

#### TOC Эчтәлек Индексы Таблицасы

Каталог индексы файлын `doc` `json` конфигурациясенә туры килә `TOC` уку өчен, демо складында `.i18n/hook/after.tran/TOC.js` `js` плагинны башкарачак `i18n.site`

Әгәр дә сез шаблон куллансагыз `doc` бу плагин булырга тиеш!

`i18n.site` буш папкадан башлап җибәрсәгез, демо проектындагы `.i18n` сезнең каталогка күчерергә онытмагыз!

`Doc` ясалган `json` нигезендә эчтәлек таблицасын күрсәтәчәк.

##### ТОКны Күпләп Ничек Күчерергә?

Файл чыганак тел каталогына урнаштырылырга тиеш `TOC`

Мәсәлән, чыганак теле кытай булса, өстә `zh/blog/TOC` `TOC`

Чыганак теле үзгәртелсә, проекттагы `TOC` билгеле бер телдә бүтән телгә күчерергә кирәк.

Сез түбәндәге боерыкларга мөрәҗәгать итә аласыз:

```
rsync -av --remove-source-files --include='*/' \
--include='TOC' --exclude='*' en/ zh/
```

Зинһар, үзгәртегез һәм югарыдагы `zh/` сезнең тел кодыгызга `en/`
