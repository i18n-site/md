## ssh Тизләтә

Европадан Кытайга кадәр челтәр тотрыклы түгел `ssh` Тизләнергә килегез [mosh](//mosh.org)

`~/.bin` машинаның `c0` скриптын ясагыз (нигездә әйләнә-тирә үзгәрүченең `PATH` ) каталогына түбәндәгечә:

```bash
#! / usr / bin / env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

Иҗаттан соң башкарыла торган рөхсәтләр һәм йомшак сылтама бирегез `c1` `c2` .

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

Киләчәктә сез турыдан- `c1` керә аласыз `c0` һәм сез `mosh` югары тизлектә туры тоташуны куллана аласыз [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

## Коткару Системасы Детальләре

Коткару 2-3 [tcping](https://github.com/paradiseduo/tcping) 22

```
tcping ip 22 -c 9999
```

Портка керү мөмкинлеген контрольдә тотарга мөмкин.

### Коткаруга Кергәннән Соң Ничек Каты Дискка Керергә &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

Сез скриншотны карый аласыз, мәсәлән

```
snapper -c etc list
```

Файл үзгәрүләрен карау

```
snapper -c etc status 5..6
```

Конкрет дифф текстны карау

```
snapper -c etc diff 3..5
```

Скриншотны торгызу: Мәсәлән, 5-нче скриншоттан соң барлык үзгәрешләрне кире кайтару

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### Бүрәнә Ныклыгын Рөхсәт Итегез

Коткару системасы астында бүрәнә ныклыгын булдыру сезгә тиешле каталогны кулдан ясарга һәм өстәмә адымнар ясарга тиеш, чөнки сезнең системалы хезмәт юк. Менә сез адымнар ясый аласыз:

Башта череп, аннары

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

Сезнең Ubuntu системасына кабызыгыз: aboveгарыдагы үзгәрешләрне ясаганнан соң, коткару системасы урынына машинаны гадәттә һәм Ubuntu системасына яңадан кертергә кирәк.

Шул рәвешле, systemd-journald яңа конфигурациядән башланачак һәм каты дискта бүрәнәләрне саклый башлый.

Онытмагыз, югарыдагы операцияләр коткару системасында башкарыла, шуңа күрә бу үзгәрешләр система яңадан эшләнгәннән соң һәм коткару режимында булмаганда гына көченә керәчәк.

Димәк, сез булган бүрәнәләргә үзгәрешләр кертелмәячәк, ләкин шуннан соң барлыкка килгән бүрәнәләр дәвам итәчәк.

Шулай итеп сез ssh хезмәтенең журналларын карый аласыз

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## Авыр Һәм Катлаулы Авырулар Турында Искәрмәләр

### Сәер Проблеманы Чишегез: Сервер ssh Эшли Алмый

Мин моңа кадәр автоматик компиляция һәм урнаштыру скрипты ясау өчен github хәрәкәтен кулландым, ләкин урнаштыру хезмәтенең ssh була алмавын таптым.

Коткару системасына керү һәм бүрәнәләрне карау - рөхсәт проблемасы:

`Missing privilege separation directory: /run/sshd`

Сез бу каталогны түбәндәге боерык белән ясый аласыз

```
chown root:root /
systemd-tmpfiles --create
```

`1001` тикшерүләрдән соң, чөнки tar йөкләгән кулланучы `uid` github action

Чыгару параметрларсыз эшләнергә тиеш tar Ул `1001` кулланучы өчен `uid` декомпрессияләнәчәк `--no-same-owner`

Нәтиҗәдә, rsync тамыр каталогының кулланучылар төркемен үзгәртәчәк, һәм сез яңадан башланганнан соң ssh-ка керә алмассыз.

Декомпрессия сценарийын түбәндәгечә үзгәртегез:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
