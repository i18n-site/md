## इस्तेमाल

### डेमो परियोजना

`i18` के विन्यास जानय लेल कृपया डेमो प्रोजेक्ट देखू [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

चीन में यूजर क्लोन क सकैत छथि [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

क्लोनिंग के बाद, डाइरेक्टरी दर्ज करू आ अनुवाद पूरा करबाक लेल `i18`

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

### अनुवाद चलाउ

निर्देशिका दर्ज करू आ अनुवाद करबाक लेल चलाउ `i18`

### भंडार मे फाइल जोड़ू

अनुवाद के अलावा, प्रोग्राम निम्नलिखित फाइल सेहो उत्पन्न करत, कृपया ओकरा भंडार मे जोड़ू.

```
.i18n/hash
.i18n/cache/.gitignore
```

ओहि मे के सामग्री निम्नलिखित : `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

एकर मतलब अछि `.i18n/cache/.gitignore` `.i18n/cache/`

जँ अहाँक संस्करण नियंत्रण सॉफ्टवेयर नहि अछि `git` कृपया एहि विन्यासक अनुसार एकरा अनदेखी करू.

## विन्यास फाइल

`.i18n/conf.yml` ई `i18` लाइन अनुवाद उपकरण क' विन्यास फाइल अछि

सामग्री निम्नलिखित अछि : १.

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### स्रोत भाषा अनुवाद भाषा &

विन्यास फाइल मे, के अधीनस्थ `fromTo`

`en` `zh ja ko de fr`

भाषा कोड देखू [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

जेना चीनी के अंग्रेजी में अनुवाद करय चाहैत छी त एहि पाँति के फेर स लिखू `zh: en`

जँ अहाँ सभ समर्थित भाषा मे अनुवाद करय चाहैत छी त' कृपया `:` उदाहरण लेल

```
i18n:
  fromTo:
    en:
```

### फाइल के अनदेखी करू

डिफ़ॉल्ट रूप सँ, स्रोत भाषा निर्देशिका मे `.md` आओर `.yml` सँ शुरू होमय बला सभ फाइल अनुवाद कएल जाएत.

यदि अहाँ किछु फाइल केँ अनदेखी करय चाहैत छी आ ओकर अनुवाद नहि करय चाहैत छी (जेना अधूरा ड्राफ्ट), त' अहाँ `ignore`

`.gitignore` के समान वाक्य रचना [globset](https://docs.rs/globset/latest/globset/#syntax) प्रयोग करैत अछि `ignore`

उदाहरण क लेल, `_` विन्यास फाइल मे `_* `

## अनुवाद के नियम

### अनुवाद संपादक केँ पाँति नहि जोड़बाक चाही वा मेटाओल नहि जेबाक चाही

अनुवाद संपादन योग्य अछि। मूल पाठ क॑ संशोधित करी क॑ ओकरा फेर स॑ मशीन-अनुवाद करी क॑ अनुवाद म॑ मैनुअल संशोधन क॑ ओवरराइट नै करलऽ जैतै (जदि मूल पाठ केरऽ ई पैराग्राफ म॑ संशोधन नै करलऽ गेलऽ छै) ।

मुदा ध्यान राखब जे अनुवादक पाँति आ मूल पाठ एकसँ एक संग मेल खाए पड़त। अर्थात अनुवादक संकलन करबा काल पाँति नहि जोड़ब वा मेटाब । नै तँ अनुवाद संपादन कैशमे भ्रम उत्पन्न करत।

यदि किछ गलत भ गेल छै, त [समाधान कें लेल कृपया पूछल जाय वाला सवाल कें देखूं.](/i18/qa#H1)

### `Yaml` भाषांतर केनाइ

कमांड लाइन टूल स्रोत भाषा फाइल डायरेक्टरी मे `.yml`

* ध्यान राखू जे फाइल नाम प्रत्यय `.yml` ( `.yaml` नहि ) हेबाक चाही ।

टूल केवल शब्दकोश मान के अनुवाद करैत अछि `.yml`

उदाहरण लेल `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

अनुवाद कयल जायत जेना `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML`

अनुवाद `YAML` आधार पर, अहाँ विभिन्न प्रोग्रामिंग भाषा के लेल अंतर्राष्ट्रीय समाधान आसानी स बना सकैत छी |

## उन्नत प्रयोग

### अनुवाद उपनिर्देशिका

जाबे तक अहाँ `.i18n/conf.yml` (हर बेर डेमो प्रोजेक्ट टेम्पलेट स शुरू करबाक जरूरत नै), `i18` ठीक काज करत।

कमांड लाइन टूल सभ उपनिर्देशिका मे `.i18n/conf.yml` खोजत आओर एकर अनुवाद करत.

जे परियोजना [monorepo](//monorepo.tools)

### कस्टम स्थापना निर्देशिका

ई डिफ़ॉल्ट रूपसँ `/usr/local/bin`

जँ लिखबाक अनुमति नहि अछि `/usr/local/bin` एकरा `~/.bin` पर इंस्टॉल कएल जाएत.

वातावरण चर सेट : `TO`

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
