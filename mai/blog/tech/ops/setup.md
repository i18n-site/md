## पासवर्ड-मुक्त लॉगिन कॉन्फ़िगर करू

किछु मिनटक बाद सर्वर सब सक्रिय भ गेल।

पहिल बात IP जे स्थानीय मशीन कें `~/.ssh/config` कें संपादित करूं आ एकटा उपनाम सेट करूं, आ फेर पासवर्ड-मुक्त लॉगिन कें कॉन्फ़िगर करय कें लेल `ssh-copy-id root@c2` उपयोग करूं.

![](https://i-01.eu.org/1718287198.avif)

आरंभीकरण विन्यास कें तेजी सं पूरा करय कें लेल अहां अइ [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) संशोधित कयर सकय छी.

पासवर्ड-मुक्त विन्यास पूरा भेलाक बाद, अहां [pdsh](https://github.com/chaos/pdsh) उपयोग क सकय छी एकरा बैच मे संचालित कएल जा सकैत अछि, जे सुविधाजनक अछि.

पहिल अछि मैन्युअल रूप स ipv6 कए सक्षम करब आ परीक्षण करब

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

![](https://i-01.eu.org/1718287969.avif)

तखन एक क्लिक सं सिस्टम कें ubuntu 24 मे अपग्रेड करू आ आमतौर पर उपयोग कैल जाय वाला सॉफ्टवेयर इंस्टॉल करू.

```bash

```

CN=1 के मतलब छै भाषा चीनी आरू समय क्षेत्र क॑ बीजिंग म॑ बदलना ।

डिफ़ॉल्ट रूप सं इंस्टॉल कएल गेल [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) हमर व्यक्तिगत उपयोगक आदतिक अनुसार कॉन्फ़िगर कएल गेल अछि, [direnv](https://direnv.net/) [mise](https://github.com/jdx/mise)

एहि कदम मे बहुत समय लगैत अछि आधा घंटा इंतजार केलाक बाद आमतौर पर उपयोग कएल जाए वाला सॉफ्टवेयर के अपग्रेड आ इंस्टॉलेशन सफल भ जाएत. अहां एकर पुष्टि निम्नलिखित कमांड सं क सकय छी:

## मोश ssh के गति बढ़ाबैत अछि

यूरोप स चीन तक के नेटवर्क स्थिर नै अछि एकर उपयोग `ssh` गति बढ़ेबाक लेल कयल जाइत अछि [mosh](//mosh.org)

स्थानीय निर्देशिका `~/.bin` (मुख्य रूप स वातावरण चर `PATH` मे जोडल गेल) मे `c0` स्क्रिप्ट निम्नलिखित रूप स बनाउ:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

निर्माण क' बाद, एक्जीक्यूटिव अनुमति आओर सॉफ्ट-लिंक `c1` आओर `c2` प्रदान करू.

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

भविष्य में, अहाँ सीधा `c0` आ `c1` दर्ज क `mosh` हाई-स्पीड डायरेक्ट कनेक्शन के उपयोग क सकैत छी आ दर्ज क सकैत छी [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

![](https://i-01.eu.org/1718300753.avif)

तखन विभाजन केँ परिवर्तित करबाक लेल निम्नलिखित स्क्रिप्ट चलाउ

```bash
nohup bash -c "bash <(curl -s https://raw.githubusercontent.com/i18n-ops/ops/main/contabo/ext4_btrfs.sh) >/tmp/nohup.out 2>&1 && reboot" &

```

रूपांतरण सफल भेलाक बाद स्क्रिप्ट स्वचालित रूप सँ सिस्टम केँ पुनः आरंभ क' देत.

### btrfs संपीड़न अनुपात देखें

ई डिफ़ॉल्ट रूप सँ स्वचालित रूप सँ संपीड़ित भ' जायत.

```bash
if ! command -v compsize &>/dev/null; then
apt install -y  btrfs-compsize
fi
compsize -x /
```

### मैनुअल संपीड़न

यदि अहां पूर्णतावादी छी त अहां मैन्युअल रूप सं सेहो पूरा डिस्क संपीड़न क सकय छी कमांड निम्नलिखित अछि.

```bash
btrfs filesystem defragment -r -czstd /
```

जेना कि नीचा देल गेल आकृति मे देखाउल गेल अछि, अहां देख सकैत छी जे नव सिस्टम मे मैनुअल संपीड़न क बाद लगभग 100एमबी बेसी संपीड़ित भ गेल ।

## बचाव प्रणाली के विवरण

बचाव प्रणाली मे प्रवेश 22 पोर्ट [tcping](https://github.com/paradiseduo/tcping) सुलभ भेलाक बाद, अहाँ एकरा सेट पासवर्ड सँ 2-3 मिनट धरि एक्सेस नहि क' सकब बस एक क्षण प्रतीक्षा करू आ ई ठीक भ' जायत.

```
tcping ip 22 -c 9999
```

बंदरगाह कें सुलभता कें निगरानी कैल जा सकय छै.

### रेस्क्यू में प्रवेश के बाद हार्ड ड्राइव तक कोना पहुंचल जाय Restore Snapshot &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

अहाँ स्नैपशॉट देख सकैत छी, उदाहरण लेल

```
snapper -c etc list
```

स्नैपशॉट फाइल परिवर्तन देखू

```
snapper -c etc status 5..6
```

विशिष्ट diff पाठ देखें

```
snapper -c etc diff 3..5
```

स्नैपशॉट कें पुनर्स्थापित करू: उदाहरण कें लेल, स्नैपशॉट 5 कें बाद सबटा परिवर्तन कें पूर्ववत करू

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### लॉग स्थायित्व सक्षम करू

बचाव प्रणाली कें तहत लॉग स्थायित्व कें सक्षम करय कें लेल अहां कें मैन्युअल रूप सं संबंधित निर्देशिका बनावा आ किछ अतिरिक्त चरणक कें प्रदर्शन करनाय आवश्यक छै, कियाकि अहां कें पास चल रहल सिस्टमडी सेवा नहि भ सकय छै. एहि ठाम एहन चरण अछि जकरा अहां कोशिश क सकय छी:

पहिने chroot, तखन

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

अपन उबंटू सिस्टम मे रिबूट करू : उपरोक्त परिवर्तन केलाक बाद, अहाँ के अपन मशीन के सामान्य रूप सं आ रेस्क्यू सिस्टम के बजाय अपन उबंटू सिस्टम मे रिबूट करय पड़त.

एहि तरहें, systemd-journald नव विन्यास सं शुरू करत आओर हार्ड डिस्क पर लॉग कें लगातार संग्रहण शुरू करत.

ध्यान राखू जे उपरोक्त ऑपरेशन बचाव प्रणाली पर कैल जायत छै, अइ कें लेल ओ परिवर्तन तखन तइक प्रभावी नहि होयत जखन तइक सिस्टम रिबूट नहि भ जायत आ आब बचाव मोड मे नहि रहत.

एकरऽ मतलब छै कि आपने क॑ तुरंत मौजूदा लॉग म॑ कोनो बदलाव नै देखै ल॑ मिलतै जे लगातार होय जाय छै, लेकिन एकरऽ बाद जेनरेट करलऽ गेलऽ सब लॉग स्थायी होय जैतै ।

एहि तरहेँ अहाँ ssh सेवाक लॉग देखि सकैत छी

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

### अजीब समस्याक समाधान करू: सर्वर ssh नहि क' सकैत अछि

हम पहिने एकटा स्वचालित संकलन आ परिनियोजन स्क्रिप्ट बनेबाक लेल github क्रियाक प्रयोग केने रही, मुदा पाओल गेल जे परिनियोजन सेवा ssh नहि भ' सकैत अछि.

बचाव प्रणाली मे प्रवेश करनाय आ लॉग देखनाय अनुमति कें मुद्दा छै:

`Missing privilege separation directory: /run/sshd`

अहाँ निम्नलिखित कमांड सँ ई निर्देशिका बना सकैत छी

```
chown root:root /
systemd-tmpfiles --create
```

बेर-बेर जांच के बाद एकर कारण अछि github action tar के यूजर `uid` `1001` अछि

tar होबाक चाही जँ कोनो पैरामीटर `--no-same-owner` नहि अछि , एकरा `1001` उपयोगकर्ताक लेल `uid` पर डिकम्प्रेस कएल जाएत

परिणामस्वरूप, rsync रूट डायरेक्टरी क' प्रयोक्ता समूह बदलत, आओर फेर अहाँ पुनः आरंभ करबाक बाद ssh मे लॉग इन नहि क' सकब.

डिकम्प्रेसन स्क्रिप्ट केँ निम्नलिखित रूप मे संशोधित करू:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
