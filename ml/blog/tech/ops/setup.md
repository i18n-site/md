# സെർവർ ആരംഭിക്കുക (BTRFS പാർട്ടീഷൻ ഉപയോഗിക്കുക, സ്നാപ്പർ ഷെഡ്യൂൾ ചെയ്ത സ്നാപ്പ്ഷോട്ട്)

## contabo വാങ്ങൽ സെർവർ

നീണ്ട ചരിത്രവും (20 വർഷത്തിലേറെ) നിരവധി ഉപയോക്താക്കളും (200,000-ത്തിലധികം) ഉള്ള ജർമ്മനിയിലെ വിലകുറഞ്ഞ VPS ദാതാവാണ് contabo, അതിനാൽ ഓടിപ്പോകുന്നതിനെക്കുറിച്ച് വിഷമിക്കേണ്ടതില്ല.

VPS [contabo.com](//contabo.com) യൂറോ വില തിരഞ്ഞെടുക്കാൻ ഓർക്കുക, അത് വിലകുറഞ്ഞതാണ്.

![](https://i-01.eu.org/1718285782.avif)

കോൺടാക്ബോയ്ക്ക് ഒറ്റത്തവണ ഇൻസ്റ്റാളേഷൻ ഫീസ് ഉണ്ട്.
എന്നിരുന്നാലും, എല്ലാ മാസവും കുറച്ച് ദിവസങ്ങൾ ഉണ്ട്, അവിടെ ഇൻസ്റ്റാളേഷൻ ഫീസ് സൗജന്യമാണ്, കൂടാതെ പ്രമോഷണൽ ഇമെയിലുകൾ ശ്രദ്ധിക്കുക.

യൂറോപ്പിലെ ഫ്രീ ഏരിയ ഫീസ് കൂടാതെ, മറ്റ് പ്രദേശങ്ങളിൽ വില വർദ്ധനവിൻ്റെ വ്യത്യസ്ത തലങ്ങളുണ്ട്.

എന്തായാലും ഞാൻ ഒരു യൂറോപ്യൻ സെർവർ തിരഞ്ഞെടുത്തു, ക്ലൗഡ്ഫ്ലെയർ ഉപയോഗിച്ചതിന് ശേഷം, സെർവറുകൾ തമ്മിൽ വലിയ വ്യത്യാസമില്ല.

അന്തിമ കോൺഫിഗറേഷൻ ഇനിപ്പറയുന്നതാണ്, 16GB മെമ്മറി, 6CPU, 200 GB NVMe, ഒരു മാസം 9.50 യൂറോ, 74.43 RMB ന് തുല്യമാണ്.

![](https://i-01.eu.org/1718286008.avif)

**ഈ വില ആലിബാബ ക്ലൗഡിൻ്റെയും ആമസോണിൻ്റെയും വിലയുടെ ഏതാണ്ട് പത്തിലൊന്നാണ്** .

![](https://i-01.eu.org/1718356956.avif)

പ്രകടനവും പാരാമീറ്ററുകളും പ്രവർത്തിപ്പിക്കാൻ ഞാൻ [Yet-Another-Bench-Script](https://github.com/masonr/yet-another-bench-script) :

```
curl -sL yabs.sh | bash -s -- -s https://www.vpsbenchmarks.com/yabs/upload
```

മോഡൽ `AMD EPYC 9224 24-Core Processor` , സിംഗിൾ-കോർ പെർഫോമൻസ് സ്കോർ 1285 CPU
, മൾട്ടി-കോർ സ്കോർ 4277 കൂടുതൽ ഫലങ്ങൾ കാണുക [vpsbenchmarks.com](https://www.vpsbenchmarks.com/yabs/contabo-6c-16gb-3fb7d8)

ഞാൻ ഒരേ സമയം മൂന്ന് സെർവറുകൾ വാങ്ങുകയും ഒരേ സമയം അവ സജീവമാക്കുകയും ചെയ്തു, ഇത് സെർവറുകൾ ഒരേ കമ്പ്യൂട്ടർ മുറിയിലാണെന്ന് ഉറപ്പാക്കുന്നു, ഇത് ഭാവിയിൽ ഉയർന്ന ലഭ്യത സൃഷ്ടിക്കുന്നത് എളുപ്പമാക്കുന്നു.

അതിൻ്റെ സെർവറുകൾ ഇടയ്ക്കിടെ ഹാംഗ് അപ്പ് ചെയ്യുന്നു, ഉയർന്ന ലഭ്യതയിലൂടെ വെബ്സൈറ്റ് പ്രവേശനക്ഷമത ഉറപ്പാക്കുന്നു.

പുതിയ ഉപയോക്താക്കൾ വാങ്ങുമ്പോൾ യഥാർത്ഥ നാമം പ്രാമാണീകരിക്കുന്നതിന് അവരുടെ ഐഡി കാർഡുകൾ സമർപ്പിക്കേണ്ടതുണ്ട്.

## പാസ്വേഡ് രഹിത ലോഗിൻ കോൺഫിഗർ ചെയ്യുക

കുറച്ച് മിനിറ്റുകൾക്ക് ശേഷം, സെർവറുകൾ എല്ലാം സജീവമായി.

അപരനാമം സജ്ജീകരിക്കുന്നതിന് ലോക്കൽ മെഷീൻ്റെ `~/.ssh/config` എഡിറ്റ് ചെയ്യുക, തുടർന്ന് പാസ്വേഡ് രഹിത ലോഗിൻ കോൺഫിഗർ ചെയ്യുന്നതിന് `ssh-copy-id root@c2` ഉപയോഗിക്കുക IP ആദ്യ കാര്യം.

![](https://i-01.eu.org/1718287198.avif)

ഇനിഷ്യലൈസേഷൻ കോൺഫിഗറേഷൻ വേഗത്തിൽ പൂർത്തിയാക്കാൻ നിങ്ങൾക്ക് ഈ [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) പരിഷ്കരിക്കാനാകും.

പാസ്വേഡ് രഹിത കോൺഫിഗറേഷൻ പൂർത്തിയാക്കിയ ശേഷം, നിങ്ങൾക്ക് ഇത് ഉപയോഗിക്കാൻ കഴിയും [pdsh](https://github.com/chaos/pdsh) അത് ബാച്ചുകളിൽ പ്രവർത്തിക്കാം.

ആദ്യത്തേത് ipv6 സ്വമേധയാ പ്രവർത്തനക്ഷമമാക്കുകയും പരീക്ഷിക്കുകയും ചെയ്യുക എന്നതാണ്

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

![](https://i-01.eu.org/1718287969.avif)

തുടർന്ന് ഒറ്റ ക്ലിക്കിൽ സിസ്റ്റം ubuntu 24 ലേക്ക് അപ്ഗ്രേഡ് ചെയ്ത് സാധാരണയായി ഉപയോഗിക്കുന്ന സോഫ്റ്റ്വെയർ ഇൻസ്റ്റാൾ ചെയ്യുക.

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 എന്നാൽ ഭാഷ ചൈനീസ് ഭാഷയിലേക്കും സമയ മേഖല ബീജിംഗിലേക്കും മാറ്റുക എന്നാണ്.

ഡിഫോൾട്ടായി ഇൻസ്റ്റാൾ [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) [direnv](https://direnv.net/) , [mise](https://github.com/jdx/mise)

ഈ ഘട്ടം അര മണിക്കൂർ കാത്തിരുന്ന ശേഷം, സാധാരണയായി ഉപയോഗിക്കുന്ന സോഫ്റ്റ്വെയറിൻ്റെ നവീകരണവും ഇൻസ്റ്റാളേഷനും വിജയകരമാകും. ഇനിപ്പറയുന്ന കമാൻഡ് ഉപയോഗിച്ച് നിങ്ങൾക്ക് ഇത് സ്ഥിരീകരിക്കാൻ കഴിയും:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## മോഷ് ssh വേഗത്തിലാക്കുന്നു

യൂറോപ്പിൽ നിന്ന് ചൈനയിലേക്കുള്ള നെറ്റ്വർക്ക് `ssh` വേഗത്തിലാക്കാൻ [mosh](//mosh.org) ശുപാർശ ചെയ്യുന്നു.

ലോക്കൽ ഡയറക്ടറി `~/.bin` ൽ `c0` സ്ക്രിപ്റ്റുകൾ സൃഷ്ടിക്കുക (പ്രധാനമായും എൻവയോൺമെൻ്റ് വേരിയബിൾ `PATH` ലേക്ക് ചേർത്തത്) ഇനിപ്പറയുന്ന രീതിയിൽ:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

സൃഷ്ടിച്ച ശേഷം, എക്സിക്യൂട്ടബിൾ അനുമതികളും സോഫ്റ്റ്-ലിങ്കും `c1` , `c2` നൽകുക.

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

ഭാവിയിൽ, `mosh` ഹൈ-സ്പീഡ് ഡയറക്ട് കണക്ഷൻ ഉപയോഗിക്കുന്നതിന് നിങ്ങൾക്ക് നേരിട്ട് `c0` , `c1` നൽകാം [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

## പാർട്ടീഷൻ ഫോർമാറ്റ് ext4-ൽ നിന്ന് btrfs-ലേക്ക് പരിവർത്തനം ചെയ്യുക

[btrfs](https://wikipedia.org/wiki/Btrfs) എളുപ്പമുള്ള ബാക്കപ്പിനായി ഡിസ്ക് സ്നാപ്പ്ഷോട്ടുകൾ എടുക്കാം.

ആദ്യം റെസ്ക്യൂ സിസ്റ്റത്തിലേക്ക് പ്രവേശിക്കുക

![](https://i-01.eu.org/1718300448.avif)

ഡെബിയൻ റെസ്ക്യൂ തിരഞ്ഞെടുക്കുക

![](https://i-01.eu.org/1718300753.avif)

സെർവർ പുനരാരംഭിക്കുന്നതിനായി കാത്തിരിക്കുക

ssh വിരലടയാള മാറ്റങ്ങൾ ഉണ്ടാകും, അത് വൃത്തിയാക്കാൻ `ssh-keygen -R ip` ഉപയോഗിക്കുക

പാർട്ടീഷൻ പരിവർത്തനം ചെയ്യാൻ ഇനിപ്പറയുന്ന സ്ക്രിപ്റ്റ് പ്രവർത്തിപ്പിക്കുക

```bash
nohup bash -c "bash <(curl -s https://raw.githubusercontent.com/i18n-ops/ops/main/contabo/ext4_btrfs.sh) >/tmp/nohup.out 2>&1 && reboot" &
sleep 1 && tail -f /tmp/nohup.out
```

പരിവർത്തനം വിജയിച്ചതിന് ശേഷം സ്ക്രിപ്റ്റ് യാന്ത്രികമായി സിസ്റ്റം പുനരാരംഭിക്കും.

### btrfs കംപ്രഷൻ അനുപാതം കാണുക

കംപ്രഷൻ അനുപാതം പരിശോധിക്കുന്നതിനുള്ള കമാൻഡ് സ്വയമേവ കംപ്രസ്സുചെയ്യും:

```bash
if ! command -v compsize &>/dev/null; then
apt install -y  btrfs-compsize
fi
compsize -x /
```

### മാനുവൽ കംപ്രഷൻ

നിങ്ങൾ ഒരു പെർഫെക്ഷനിസ്റ്റ് ആണെങ്കിൽ, നിങ്ങൾക്ക് സ്വമേധയാ ഒരു പൂർണ്ണ ഡിസ്ക് കംപ്രഷൻ നടത്താം:

```bash
btrfs filesystem defragment -r -czstd /
```

ചുവടെയുള്ള ചിത്രത്തിൽ കാണിച്ചിരിക്കുന്നതുപോലെ, പുതിയ സിസ്റ്റത്തിൽ മാനുവൽ കംപ്രഷൻ ചെയ്ത ശേഷം, ഏകദേശം 100MB കൂടുതൽ കംപ്രസ് ചെയ്തതായി നിങ്ങൾക്ക് കാണാൻ കഴിയും.

![](https://i-01.eu.org/2023/12/8NWlr4c.webp)

### സ്നാപ്പർ ഉപയോഗിച്ച് ഒരു Btrfs സ്നാപ്പ്ഷോട്ട് ബാക്കപ്പ് സൃഷ്ടിക്കുക

Btrfs ബാക്കപ്പിനുള്ള മികച്ച [snapper](https://manual.siduction.org/sys-admin-btrfs-snapper_en.html) .

ഇനിഷ്യലൈസേഷൻ സ്ക്രിപ്റ്റ് `~/ubuntu` ഡയറക്ടറി സൃഷ്ടിക്കും, ഇനിപ്പറയുന്ന കമാൻഡ് പ്രവർത്തിപ്പിക്കുക:

```
~/ubuntu/snapper/init.sh
```

സ്ക്രിപ്റ്റ് ഉള്ളടക്കത്തിൽ നിങ്ങളെ കാണാം [github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh](https://github.com/i18n-ops/ubuntu/blob/main/snapper/init.sh)

സ്നാപ്പ്ഷോട്ടുകൾ സജ്ജീകരിച്ചിരിക്കുന്ന ഉപവോള്യങ്ങൾ കാണുന്നതിന് `snapper list-configs` ഉപയോഗിക്കുക.

![](https://i-01.eu.org/1718302689.avif)

ഒരു ഉപവോള്യത്തിൻ്റെ സ്നാപ്പ്ഷോട്ട് കാണുന്നതിന് `snapper -c mnt list` ഉപയോഗിക്കുക

`/etc/snapper/configs/mnt` ഈ ഉപവോള്യത്തിൻ്റെ നിർദ്ദിഷ്ട ബാക്കപ്പ് ക്രമീകരണങ്ങൾ കാണാൻ കഴിയും

തുടർന്ന്, പുനഃസ്ഥാപിക്കാൻ Snapshots ക്ലിക്ക് ചെയ്യുക.

![](https://i-01.eu.org/1718304063.avif)

എന്നിരുന്നാലും, ഈ സ്നാപ്പ്ഷോട്ടിൻ്റെ ഗ്രാനുലാരിറ്റി വളരെ വലുതാണ്, ഒരിക്കൽ പുനഃസ്ഥാപിച്ചാൽ, പുതുതായി ചേർത്ത ഡാറ്റ പൂർണ്ണമായും നഷ്ടപ്പെടും, അതിനാൽ ദയവായി ഇത് ജാഗ്രതയോടെ ഉപയോഗിക്കുക.

സാധാരണയായി btrfs സ്നാപ്പ്ഷോട്ടുകൾ ഉപയോഗിക്കുന്നത് കൂടുതൽ സൗകര്യപ്രദമാണ്.

## റെസ്ക്യൂ സിസ്റ്റം വിശദാംശങ്ങൾ

പോർട്ട് [tcping](https://github.com/paradiseduo/tcping) ആക്സസ് 22 ശേഷം, 2-3 മിനിറ്റ് നേരത്തേക്ക് നിങ്ങൾക്ക് അത് ആക്സസ് ചെയ്യാൻ കഴിയില്ല.

```
tcping ip 22 -c 9999
```

പോർട്ട് പ്രവേശനക്ഷമത നിരീക്ഷിക്കാൻ കഴിയും.

### സ്നാപ്പ്ഷോട്ട് പുനഃസ്ഥാപിക്കുക & പ്രവേശിച്ച ശേഷം ഹാർഡ് ഡ്രൈവ് എങ്ങനെ ആക്സസ് ചെയ്യാം

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

നിങ്ങൾക്ക് സ്നാപ്പ്ഷോട്ട് കാണാൻ കഴിയും, ഉദാഹരണത്തിന്

```
snapper -c etc list
```

സ്നാപ്പ്ഷോട്ട് ഫയൽ മാറ്റങ്ങൾ കാണുക

```
snapper -c etc status 5..6
```

നിർദ്ദിഷ്ട ഡിഫ് ടെക്സ്റ്റ് കാണുക

```
snapper -c etc diff 3..5
```

സ്നാപ്പ്ഷോട്ട് പുനഃസ്ഥാപിക്കുക: ഉദാഹരണത്തിന്, സ്നാപ്പ്ഷോട്ട് 5-ന് ശേഷമുള്ള എല്ലാ മാറ്റങ്ങളും പഴയപടിയാക്കുക

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### ലോഗ് പെർസിസ്റ്റൻസ് പ്രവർത്തനക്ഷമമാക്കുക

ഒരു റെസ്ക്യൂ സിസ്റ്റത്തിനു കീഴിലുള്ള ലോഗ് പെർസിസ്റ്റൻസ് പ്രവർത്തനക്ഷമമാക്കുന്നതിന്, നിങ്ങൾക്ക് ഒരു systemd സേവനം പ്രവർത്തിക്കാത്തതിനാൽ, നിങ്ങൾ സ്വയം അനുബന്ധ ഡയറക്ടറി സൃഷ്ടിക്കുകയും ചില അധിക ഘട്ടങ്ങൾ നടപ്പിലാക്കുകയും ചെയ്യേണ്ടതുണ്ട്. നിങ്ങൾക്ക് ശ്രമിക്കാവുന്ന ഘട്ടങ്ങൾ ഇതാ:

ആദ്യം chroot, പിന്നെ

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

നിങ്ങളുടെ ഉബുണ്ടു സിസ്റ്റത്തിലേക്ക് റീബൂട്ട് ചെയ്യുക: മുകളിൽ പറഞ്ഞ മാറ്റങ്ങൾ വരുത്തിയ ശേഷം, റെസ്ക്യൂ സിസ്റ്റത്തിന് പകരം നിങ്ങളുടെ മെഷീൻ സാധാരണ രീതിയിലും ഉബുണ്ടു സിസ്റ്റത്തിലും റീബൂട്ട് ചെയ്യേണ്ടതുണ്ട്.

ഈ രീതിയിൽ, systemd-journald പുതിയ കോൺഫിഗറേഷനിൽ ആരംഭിക്കുകയും ഹാർഡ് ഡിസ്കിൽ ലോഗുകൾ സ്ഥിരമായി സൂക്ഷിക്കാൻ തുടങ്ങുകയും ചെയ്യും.

മേൽപ്പറഞ്ഞ പ്രവർത്തനങ്ങൾ ഒരു റെസ്ക്യൂ സിസ്റ്റത്തിലാണ് നടത്തുന്നത് എന്ന കാര്യം ഓർക്കുക, അതിനാൽ സിസ്റ്റം റീബൂട്ട് ചെയ്യുന്നതുവരെ ആ മാറ്റങ്ങൾ പ്രാബല്യത്തിൽ വരില്ല, ഇനി റെസ്ക്യൂ മോഡിൽ ഇല്ല.

ഇതിനർത്ഥം, നിലവിലുള്ള ലോഗുകളിൽ സ്ഥിരമായി മാറുന്ന മാറ്റങ്ങളൊന്നും നിങ്ങൾ ഉടനടി കാണില്ല, എന്നാൽ അതിനുശേഷം സൃഷ്ടിക്കുന്ന എല്ലാ ലോഗുകളും നിലനിൽക്കും.

ഇതുവഴി നിങ്ങൾക്ക് ssh സേവനത്തിൻ്റെ ലോഗുകൾ കാണാനാകും

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## ബുദ്ധിമുട്ടുള്ളതും സങ്കീർണ്ണവുമായ രോഗങ്ങളെക്കുറിച്ചുള്ള കുറിപ്പുകൾ

### വിചിത്രമായ പ്രശ്നം പരിഹരിക്കുക: സെർവറിന് ssh ചെയ്യാൻ കഴിയില്ല

മുമ്പ് ഒരു ഓട്ടോമാറ്റിക് കംപൈലേഷനും വിന്യാസ സ്ക്രിപ്റ്റും നിർമ്മിക്കാൻ ഞാൻ github ആക്ഷൻ ഉപയോഗിച്ചു, എന്നാൽ വിന്യാസ സേവനം ssh ആകാൻ കഴിയില്ലെന്ന് കണ്ടെത്തി.

റെസ്ക്യൂ സിസ്റ്റത്തിൽ പ്രവേശിക്കുന്നതും ലോഗുകൾ നോക്കുന്നതും ഒരു അനുമതി പ്രശ്നമാണ്:

`Missing privilege separation directory: /run/sshd`

ഇനിപ്പറയുന്ന കമാൻഡ് ഉപയോഗിച്ച് നിങ്ങൾക്ക് ഈ ഡയറക്ടറി സൃഷ്ടിക്കാൻ കഴിയും

```
chown root:root /
systemd-tmpfiles --create
```

ആവർത്തിച്ചുള്ള അന്വേഷണങ്ങൾക്ക് ശേഷം, tar github action `uid` `1001` ആണ്

`--no-same-owner` പാരാമീറ്റർ ഇല്ലെങ്കിൽ, അത് `1001` ഉപയോക്താക്കൾക്ക് `uid` ആയി വിഘടിപ്പിക്കും tar

തൽഫലമായി, rsync റൂട്ട് ഡയറക്ടറിയുടെ ഉപയോക്തൃ ഗ്രൂപ്പിനെ മാറ്റും, തുടർന്ന് പുനരാരംഭിച്ചതിന് ശേഷം നിങ്ങൾക്ക് ssh-ലേക്ക് ലോഗിൻ ചെയ്യാൻ കഴിയില്ല.

ഡീകംപ്രഷൻ സ്ക്രിപ്റ്റ് ഇനിപ്പറയുന്ന രീതിയിൽ പരിഷ്ക്കരിക്കുക:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
