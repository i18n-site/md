# ഇൻസ്റ്റാൾ ചെയ്ത് ഉപയോഗിക്കുക

## വിൻഡോസ് ആദ്യം git bash ഇൻസ്റ്റാൾ ചെയ്യുക

windows സിസ്റ്റം, [ആദ്യം ഡൗൺലോഡ് ചെയ്ത് ഇൻസ്റ്റാൾ ചെയ്യാൻ ഇവിടെ ക്ലിക്ക് `git bash`](https://git-scm.com/download/win) .

ൽ തുടർന്നുള്ള പ്രവർത്തനങ്ങൾ പ്രവർത്തിപ്പിക്കുക `git bash`

## ഇൻസ്റ്റാൾ ചെയ്യുക

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### വിവർത്തന ടോക്കൺ കോൺഫിഗർ ചെയ്യുക

ടോക്കൺ പകർത്താൻ [i18n.site/token](//i18n.site/token)

<img src="https://p.3ti.site/1719911689.avif" style="max-width:400px">

സൃഷ്‌ടിക്കുക `~/.config/i18n.site.yml` പകർത്തിയ ഉള്ളടക്കം അതിൽ ഒട്ടിക്കുക, ഉള്ളടക്കം ഇപ്രകാരമാണ്:

```
token: YOUR_API_TOKEN
```

കൂടാതെ, പേയ്‌മെൻ്റിനായി നിങ്ങൾ ഒരു ക്രെഡിറ്റ് കാർഡ് [i18n.site/payBill](//i18n.site/payBill) (റീചാർജ് ആവശ്യമില്ല, ഉപയോഗത്തിനനുസരിച്ച് വെബ്‌സൈറ്റ് സ്വയമേവ ഫീസ് കുറയ്ക്കും, [വിലനിർണ്ണയത്തിനായി ഹോംപേജ് കാണുക](/#price) ).

## ഉപയോഗിക്കുക

### ഡെമോ പ്രോജക്റ്റ്

വിവർത്തനത്തിൻ്റെ കോൺഫിഗറേഷൻ പഠിക്കാൻ `i18` ഡെമോ പ്രോജക്റ്റ് പരിശോധിക്കുക [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18)

ചൈനയിലെ ഉപയോക്താക്കൾക്ക് ക്ലോൺ ചെയ്യാൻ കഴിയും [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

ക്ലോണിംഗിന് ശേഷം, വിവർത്തനം പൂർത്തിയാക്കാൻ ഡയറക്‌ടറിയിൽ പ്രവേശിച്ച് പ്രവർത്തിപ്പിക്കുക `i18`

### ഡയറക്ടറി ഘടന

ടെംപ്ലേറ്റ് വെയർഹൗസ് ഡയറക്ടറി ഘടന ഇപ്രകാരമാണ്

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

`en` എന്ന ഡയറക്‌ടറിയിൽ വിവർത്തനം ചെയ്‌ത ഡെമോ ഫയലുകൾ അടങ്ങിയിരിക്കുന്നു, അവ ഒരു ഉദാഹരണം മാത്രമാണ്, അവ ഇല്ലാതാക്കാൻ കഴിയും.

### വിവർത്തനം പ്രവർത്തിപ്പിക്കുക

വിവർത്തനം ചെയ്യാൻ ഡയറക്ടറി നൽകി പ്രവർത്തിപ്പിക്കുക `i18`

### റിപ്പോസിറ്ററിയിലേക്ക് ഫയലുകൾ ചേർക്കുക

വിവർത്തനത്തിന് പുറമേ, പ്രോഗ്രാം ഇനിപ്പറയുന്ന ഫയലുകളും സൃഷ്ടിക്കും, ദയവായി അവ ശേഖരത്തിലേക്ക് ചേർക്കുക.

```
.i18n/hash
.i18n/cache/.gitignore
```

അവയിൽ : ഉള്ളടക്കം ഇപ്രകാരമാണ് `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

`.i18n/cache/` എല്ലാ ഫയലുകളും അവഗണിക്കുക എന്നാണ് ഇതിനർത്ഥം ( `.i18n/cache/.gitignore` ഒഴികെ).

നിങ്ങളുടെ പതിപ്പ് നിയന്ത്രണ സോഫ്‌റ്റ്‌വെയർ ഇല്ലെങ്കിൽ `git` ഈ കോൺഫിഗറേഷൻ അനുസരിച്ച് അത് അവഗണിക്കുക.

## കോൺഫിഗറേഷൻ ഫയൽ

`.i18n/conf.yml` ഇത് `i18` ലൈൻ വിവർത്തന ഉപകരണത്തിൻ്റെ കോൺഫിഗറേഷൻ ഫയലാണ്

ഉള്ളടക്കം ഇപ്രകാരമാണ്:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### ഉറവിട ഭാഷ &

കോൺഫിഗറേഷൻ ഫയലിൽ, `fromTo`

`en` എന്നത് ഉറവിട ഭാഷയാണ്, `zh ja ko de fr` എന്നത് വിവർത്തനത്തിൻ്റെ ലക്ഷ്യ ഭാഷയാണ്.

ഭാഷാ കോഡ് കാണുക [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

ഉദാഹരണത്തിന്, നിങ്ങൾക്ക് ചൈനീസ് ഭാഷ ഇംഗ്ലീഷിലേക്ക് വിവർത്തനം ചെയ്യണമെങ്കിൽ, ഈ വരി വീണ്ടും എഴുതുക `zh: en`

പിന്തുണയ്‌ക്കുന്ന എല്ലാ ഭാഷകളിലേക്കും നിങ്ങൾക്ക് വിവർത്തനം ചെയ്യണമെങ്കിൽ, ശേഷം ശൂന്യമായി വിടുക `:` ഉദാഹരണത്തിന്

```
i18n:
  fromTo:
    en:
```

### ഫയൽ അവഗണിക്കുക

സ്ഥിരസ്ഥിതിയായി, ഉറവിട ഭാഷാ ഡയറക്‌ടറിയിലെ `.md` , `.yml` എന്നിവയിൽ ആരംഭിക്കുന്ന എല്ലാ ഫയലുകളും വിവർത്തനം ചെയ്യപ്പെടും.

നിങ്ങൾക്ക് ചില ഫയലുകൾ അവഗണിക്കുകയും അവ വിവർത്തനം ചെയ്യാതിരിക്കുകയും ചെയ്യണമെങ്കിൽ (പൂർത്തിയാകാത്ത ഡ്രാഫ്റ്റുകൾ പോലെ), നിങ്ങൾക്ക് `ignore` കോൺഫിഗറേഷൻ ഉപയോഗിക്കാം.

`ignore` `.gitignore` സമാനമായ ഒരു വാക്യഘടന ഉപയോഗിക്കുന്നു [globset](https://docs.rs/globset/latest/globset/#syntax)

ഉദാഹരണത്തിന്, മുകളിലെ കോൺഫിഗറേഷൻ ഫയലിൽ `_` എന്നതിൽ ആരംഭിക്കുന്ന ഫയലുകൾ വിവർത്തനം ചെയ്യില്ല `_* ` .

## വിവർത്തന നിയമങ്ങൾ

### വിവർത്തന എഡിറ്റർമാർ വരികൾ ചേർക്കാനോ ഇല്ലാതാക്കാനോ പാടില്ല

വിവർത്തനം എഡിറ്റ് ചെയ്യാവുന്നതാണ്. ഒറിജിനൽ ടെക്‌സ്‌റ്റ് പരിഷ്‌കരിച്ച് അത് വീണ്ടും മെഷീൻ വിവർത്തനം ചെയ്യുക, വിവർത്തനത്തിലേക്കുള്ള മാനുവൽ പരിഷ്‌ക്കരണങ്ങൾ തിരുത്തിയെഴുതപ്പെടില്ല (യഥാർത്ഥ വാചകത്തിൻ്റെ ഈ ഖണ്ഡിക പരിഷ്‌കരിച്ചിട്ടില്ലെങ്കിൽ).

എന്നാൽ വിവർത്തനത്തിൻ്റെ വരികളും ഒറിജിനൽ ടെക്‌സ്‌റ്റും ഒന്നുമായി പൊരുത്തപ്പെടണം എന്നത് ദയവായി ശ്രദ്ധിക്കുക. അതായത്, വിവർത്തനം കംപൈൽ ചെയ്യുമ്പോൾ വരികൾ ചേർക്കുകയോ ഇല്ലാതാക്കുകയോ ചെയ്യരുത്. അല്ലെങ്കിൽ, ഇത് വിവർത്തന എഡിറ്റിംഗ് കാഷെയിൽ ആശയക്കുഴപ്പമുണ്ടാക്കും.

എന്തെങ്കിലും തെറ്റ് സംഭവിക്കുകയാണെങ്കിൽ, [പരിഹാരങ്ങൾക്കായി FAQ റഫർ ചെയ്യുക.](/i18/qa#H1)

### `YAML`

കമാൻഡ് ലൈൻ ടൂൾ സോഴ്സ് ലാംഗ്വേജ് ഫയൽ ഡയറക്‌ടറിയിൽ `.yml` എല്ലാ ഫയലുകളും കണ്ടെത്തി വിവർത്തനം ചെയ്യും.

* ഫയലിൻ്റെ പേര് സഫിക്സ് ( `.yaml` അല്ല) ആയിരിക്കണം എന്നത് ശ്രദ്ധിക്കുക `.yml`

`.yml` നിഘണ്ടു മൂല്യങ്ങൾ മാത്രമാണ് ഉപകരണം വിവർത്തനം ചെയ്യുന്നത്, നിഘണ്ടു കീകളല്ല.

ഉദാഹരണത്തിന് `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

എന്ന് പരിഭാഷപ്പെടുത്തും `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

`YAML` എന്നതിൻ്റെ വിവർത്തനം സ്വമേധയാ പരിഷ്കരിക്കാനും കഴിയും (എന്നാൽ വിവർത്തനത്തിൽ കീകളോ വരികളോ ചേർക്കുകയോ ഇല്ലാതാക്കുകയോ ചെയ്യരുത്).

`YAML` അടിസ്ഥാനമാക്കി, നിങ്ങൾക്ക് വിവിധ പ്രോഗ്രാമിംഗ് ഭാഷകൾക്കായി അന്താരാഷ്ട്ര പരിഹാരങ്ങൾ എളുപ്പത്തിൽ നിർമ്മിക്കാൻ കഴിയും.

## വിപുലമായ ഉപയോഗം

### വിവർത്തന ഉപഡയറക്‌ടറി

നിങ്ങൾ `.i18n/conf.yml` (ഓരോ തവണയും ഡെമോ പ്രോജക്റ്റ് ടെംപ്ലേറ്റിൽ നിന്ന് ആരംഭിക്കേണ്ടതില്ല), `i18` നന്നായി പ്രവർത്തിക്കും.

കമാൻഡ് ലൈൻ ടൂൾ എല്ലാ ഉപഡയറക്‌ടറികളിലും `.i18n/conf.yml` കണ്ടെത്തി അത് വിവർത്തനം ചെയ്യും.

### കസ്റ്റം ഇൻസ്റ്റലേഷൻ ഡയറക്ടറി

ഇത് സ്ഥിരസ്ഥിതിയായി ഇൻസ്റ്റാൾ ചെയ്യപ്പെടും `/usr/local/bin`

`/usr/local/bin` `~/.bin`

എൻവയോൺമെൻ്റ് വേരിയബിളുകൾ സജ്ജമാക്കുക `TO` ഉദാഹരണത്തിന്, നിങ്ങൾക്ക് ഇൻസ്റ്റലേഷൻ ഡയറക്ടറി നിർവചിക്കാം :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
