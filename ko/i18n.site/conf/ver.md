# 프로젝트 버전

데모 프로젝트를 예시로 들어보겠습니다

`en/demo2/v`는 프로젝트의 현재 버전 번호로, 사이드바의 개요에서 프로젝트 이름 오른쪽에 표시됩니다.

<img src="https://p.3ti.site/1721290486.avif" width="320px">

`en/`은 `.i18n/conf.yml` 설정에서 번역의 원본 언어에 해당하는 언어 코드입니다.

원본 언어가 영어가 아닌 경우, `v` 파일은 원본 언어의 프로젝트 디렉토리에 위치해야 합니다.

문서의 이전 버전을 브라우저하는 기능은 현재 개발 중입니다.

버전 번호가 너무 많아지면 검색 엔진의 색인이 복잡해질 수 있으므로, 중요한 업데이트가 발행될 때만 (예: `v1`, `v2`) 문서의 버전 번호를 수정하는 것이 좋습니다.

## 빈 `v` 파일을 사용하여 다른 프로젝트의 파일 인덱스를 나눕니다

데모 프로젝트에서는 `en/demo2/v` 외에도 `en/blog`와 `en/demo1` 디렉토리에 빈 `v` 파일이 존재합니다.

빈 `v` 파일은 사이드바 개요에 표시되지 않지만, `v` 파일이 존재하는 경우 해당 디렉토리와 하위 디렉토리의 파일에 대해 독립적인 인덱스가 생성됩니다.

다른 프로젝트의 인덱스를 나누면 전체 사이트의 모든 파일 인덱스를 한 번에 로드할 때 발생할 수 있는 접근 속도 저하를 방지할 수 있습니다.

예를 들어, 데모 프로젝트에서 `blog`에 해당하는 인덱스 파일은 다음과 같습니다: [https://unpkg.com/i18n.site/en/blog.json](https://unpkg.com/i18n.site/en/blog.json)
