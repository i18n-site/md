## 서론

인터넷은 물리적 공간의 거리를 缺乏하지만, 언어의 차이는 여전히 인류의 통합을 방해하고 있습니다.

브라우저는 번역 기능을 내장하고 있지만, 검색 엔진은 여전히 다양한 언어로의 쿼리를 지원하지 않습니다.

소셜 미디어와 이메일을 통해 대중은 자신의 모국어 정보 소스에 집중하는 데 익숙해졌습니다.

정보의 폭발과 글로벌 시장에서, 부족한 관심을 끌기 위해서는 **다국어 지원이 기본적인 요소입니다**.

개인 오픈 소스 프로젝트라 할지라도, 더 넓은 청중에게 영향을 미치고자 한다면 처음부터 국제적인 기술 선택이 필요합니다.

### <a rel=id href="#i18" id="i18"></a> i18 : MarkDown 명령줄 번역 도구

`Markdown` 과 `YAML`을 다양한 언어로 번역하는 명령줄 도구 ([소스 코드](https://github.com/i18n-site/rust/tree/main/i18))입니다.

`Markdown`의 형식을 완벽하게 유지하며, 파일의 변경 사항을 인식하여 변경된 파일만 번역합니다.

번역은 편집 가능합니다.

원본 텍스트를 수정하고 다시 기계 번역하면, 번역에 대한 수동 수정 사항이 덮어쓰여지지 않습니다(원문의 이 단락이 수정되지 않은 경우).

우리는 `UNIX`의 철학 "모든 것이 파일"을 따르며, 복잡하고 번거로운 기업 수준의 솔루션 없이도 수백 가지 언어의 번역을 관리할 수 있습니다.

가장 친숙한 도구를 사용하여 `Markdown` 편집할 수 있으며(단락을 추가하거나 삭제할 수는 없음) 버전 제어를 수행하는 데 가장 친숙한 방법을 사용할 수 있습니다.

언어 파일의 코드 베이스를 오픈 소스로 만들고, `Pull Request` 프로세스를 통해 글로벌 사용자가 번역의 지속적인 최적화에 참여할 수 있습니다. [github](//github.com) 등의 오픈 소스 커뮤니티와 원활하게 연결됩니다.

[→ 사용 가이드는 프로젝트 문서를 참조하세요](/i18).

### i18n.site : 다국어 정적 웹사이트 생성기

다국어 정적 웹사이트를 생성하는 명령줄 도구 ([소스 코드](https://github.com/i18n-site/rust/tree/main/i18n-site))입니다.

순수하게 정적이며, 읽기 경험을 최적화하며, [i18](#i18) 번역과 통합되어 프로젝트 문서 사이트를 구축하는 데 최적입니다.

기본 프론트엔드 프레임워크는 모듈식 플러그인 아키텍처를 채택하여 필요에 따라 백엔드 기능을 통합할 수 있습니다.

이 웹사이트는 이 프레임워크를 기반으로 개발되었으며, 사용자, 결제 등 다양한 기능을 포함하고 있습니다. ([소스 코드](/i18n.site/src)) 자세한 튜토리얼은 나중에 작성될 예정입니다.
