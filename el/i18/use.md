# Εγκατάσταση και Χρήση

## Για τα Windows, εγκαταστήστε πρώτα το Git Bash

Στο σύστημα Windows, [κάντε κλικ εδώ για να κατεβάσετε και να εγκαταστήσετε πρώτα το `git bash`](https://git-scm.com/download/win).

Εκτελέστε τις επόμενες ενέργειες στο `git bash`.

## Εγκατάσταση

```sh
bash <(curl -sS https://i.i18n.site) i18
```

### Ρύθμιση του διακριτικού μετάφρασης

Προσέξτε το [i18n.site/token](//i18n.site/token) και κ copiate το διακριτικό

## Χρήση

### Έργο Επίδειξης

Ανατρέξτε στο παράδειγμα έργου [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18) για να μάθετε πώς να διαμορφώσετε τη μετάφραση του `i18`.

Οι χρήστες στην Κίνα μπορούν να клωνοποιήσουν το [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

Μετά την κλωνοποίηση, μεταβείτε στον κατάλογο και εκτελέστε το `i18` για να ολοκληρώσετε τη μετάφραση.

### Δομή καταλόγου

Η δομή του καταλόγου της αποθήκης προτύπων είναι η εξής:

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

Τα αρχεία επίδειξης στον κατάλογο `en` είναι απλώς παραδείγματα και μπορούν να διαγραφούν.

### Εκτέλεση της μετάφρασης

Μεταβείτε στον κατάλογο και εκτελέστε το `i18` για να ξεκινήσει η μετάφραση.

### Προσθήκη αρχείων στο αποθετήριο

Το πρόγραμμα, εκτός από τη μετάφραση, θα δημιουργήσει επίσης τα ακόλουθα αρχεία, τα οποία πρέπει να προσθέσετε στο αποθετήριο.

```
.i18n/hash
.i18n/cache/.gitignore
```

Μεταξύ αυτών, το περιεχόμενο του αρχείου `.i18n/cache/.gitignore` είναι το εξής:

```
**/*
!**/.gitignore
```

Αυτό σημαίνει ότι αγνοούνται όλα τα αρχεία στον κατάλογο `.i18n/cache/` (με εξαίρεση το `.i18n/cache/.gitignore`).

Αν το σύστημα ελέγχου εκδόσεων που χρησιμοποιείτε δεν είναι το `git`, ακολουθήστε αυτή τη διαμόρφωση για να αγνοήσετε τα αρχεία.

## Αρχείο διαμόρφωσης

Το αρχείο `.i18n/conf.yml` είναι το αρχείο διαμόρφωσης της γραμμής εντολών του εργαλείου μετάφρασης `i18`

Το περιεχόμενο του είναι ως εξής:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Γλώσσα πηγής & γλώσσα μετάφρασης

Στο αρχείο διαμόρφωσης, οι ρυθμίσεις του `fromTo` είναι:

`en` είναι η γλώσσα πηγής, ενώ `zh ja ko de fr` είναι οι γλώσσες προορισμού της μετάφρασης.

Οι κώδικες γλώσσας βρίσκονται στο [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE).

Για παράδειγμα, αν θέλετε να μεταφράσετε τα Κινέζικα σε Αγγλικά, γράψτε `zh: en`.

Αν θέλετε να μεταφράσετε σε όλες τις υποστηριζόμενες γλώσσες, αφήστε το πεδίο μετά το `:` κενό, όπως:

```
i18n:
  fromTo:
    en:
```

Μπορείτε να ρυθμίσετε διαφορετικές παραμέτρους `fromTo` για διαφορετικούς υποκαταλόγους/αρχεία. Μια επίδειξη είναι γραμμένη ως:

```
i18n:
  fromTo:
    en:
  path:
    blog:
      fromTo:
        zh:
    blog/your_file_name.md:
      fromTo:
        ja:
```

Σε αυτόν τον πίνακα διαμόρφωσης, η γλώσσα πηγής για τον κατάλογο `blog` είναι `zh`, ενώ για το αρχείο `blog/your_file_name.md` είναι `ja`.

### Αγνόηση αρχείων

Από προεπιλογή, όλα τα αρχεία με κατάληξη `.md` και `.yml` στον κατάλογο της γλώσσας πηγής θα μεταφραστούν.

Αν θέλετε να αγνοήσετε ορισμένα αρχεία και να μην τα μεταφράσετε (π.χ. προσωρινά έγγραφα), μπορείτε να το κάνετε μέσω της ρύθμισης `ignore`.

Το `ignore` χρησιμοποιεί τη γλώσσα [globset](https://docs.rs/globset/latest/globset/#syntax) παρόμοια με το `.gitignore`.

Για παράδειγμα, το `_*` σημαίνει ότι αγνοούνται όλα τα αρχεία που ξεκινούν με `_`.

## Κανόνες μετάφρασης

### Οι συντάκτες μετάφρασης δεν πρέπει να προσθέτουν ή να αφαιρούν γραμμές

Η μετάφραση μπορεί να επεξεργαστεί χειροκίνητα. Όταν τροποποιείτε το αρχικό κείμενο και εκτελείτε ξανά τη μηχανική μετάφραση, οι χειροκINETικές τροποποιήσεις δεν θα αντικατασταθούν (αν δεν έχει τροποποιηθεί η συγκεκριμένη παράγραφος του αρχικού κειμένου).

Ωστόσο, σημειώστε ότι οι γραμμές της μετάφρασης και του αρχικού κειμένου πρέπει να αντιστοιχούν ακριβώς. Δηλαδή, μην προσθέτετε ή αφαιρείτε γραμμές κατά τη σύνταξη της μετάφρασης, διαφορετικά θα προκαλέσετε σύγχυση στην κρυφή μνήμη επεξεργασίας της μετάφρασης.

Αν προκύψουν προβλήματα, ανατρέξτε στις [συχνές ερωτήσεις για λύσεις](/i18/qa#H1)

### Μεταφράσεις `YAML`

Το εργαλείο γραμμής εντολών θα βρει όλα τα αρχεία με κατάληξη `.yml` στον κατάλογο της γλώσσας πηγής και θα τα μεταφράσει.

* Σημειώστε ότι η κατάληξη του αρχείου πρέπει να είναι `.yml` (όχι `.yaml`).

Το εργαλείο μεταφράζει μόνο τις τιμές των λεξικών σε `.yml`, όχι τα κλειδιά των λεξικών.

Για παράδειγμα, `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

θα μεταφραστεί ως `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

Η μετάφραση του `YAML` μπορεί επίσης να τροποποιηθεί χειροκίνητα (αλλά μην προσθέτετε ή διαγράφετε κλειδιά ή γραμμές στη μετάφραση).

Με βάση τη μετάφραση `YAML`, μπορείτε εύκολα να δημιουργήσετε διεθνείς λύσεις για διάφορες γλώσσες προγραμματισμού.

## Μετάφραση υποκαatalogου

### Υποκατάλογος Μετάφρασης

Εφόσον δημιουργείται `.i18n/conf.yml` (δεν χρειάζεται να ξεκινάτε από το πρότυπο έργου επίδειξης κάθε φορά), το `i18` θα λειτουργεί σωστά.

Το εργαλείο γραμμής εντολών θα βρει τις διαμορφώσεις `.i18n/conf.yml` σε όλους τους υποκαταλόγους και θα τις μεταφράσει.

Για έργα που χρησιμοποιούν την αρχιτεκτονική [monorepo](//monorepo.tools)

![](https://p.3ti.site/1719910016.avif)

### Προσαρμοσμένος Κατάλογος Εγκατάστασης

Θα εγκατασταθεί στο `/usr/local/bin` ως προεπιλογή.

Εάν δεν έχετε άδεια εγγραφής στο `/usr/local/bin`, θα εγκατασταθεί στο `~/.bin`.

Η ρύθμιση της μεταβλητής περιβάλλοντος `TO` μπορεί να ορίσει τον κατάλογο εγκατάστασης, για παράδειγμα:

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
