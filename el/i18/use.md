## Χρήση

### Έργο Επίδειξης

Ανατρέξτε στο έργο [github.com/i18n-site/demo.i18](//github.com/i18n-site/demo.i18) για να μάθετε τη διαμόρφωση της μετάφρασης `i18`

Οι χρήστες στην Κίνα μπορούν να κλωνοποιήσουν [atomgit.com/i18n/demo.i18](//atomgit.com/i18n/demo.i18)

Μετά την κλωνοποίηση, εισαγάγετε τον κατάλογο και `i18` την ολοκλήρωση της μετάφρασης.

### Δομή Καταλόγου

Η δομή του καταλόγου αποθήκης προτύπων έχει ως εξής

```
┌── .i18n
│  └── conf.yml
└── en
   ├── _IgnoreDemoFile.md
   ├── i18n.yml
   └── README.md
```

Ο κατάλογος `en` περιέχει τα μεταφρασμένα αρχεία επίδειξης, τα οποία είναι απλώς ένα παράδειγμα και μπορούν να διαγραφούν.

### Εκτέλεση Μετάφρασης

Εισαγάγετε τον κατάλογο και `i18` για μετάφραση.

### Προσθέστε Αρχεία Στο Αποθετήριο

Εκτός από τη μετάφραση, το πρόγραμμα θα δημιουργήσει επίσης τα ακόλουθα αρχεία, προσθέστε τα στο αποθετήριο.

```
.i18n/hash
.i18n/cache/.gitignore
```

Μεταξύ : , το περιεχόμενο του `.i18n/cache/.gitignore`

```
**/*
!**/.gitignore
```

Αυτό σημαίνει ότι αγνοήστε όλα τα αρχεία στον `.i18n/cache/` (εκτός από `.i18n/cache/.gitignore` ).

Εάν το λογισμικό ελέγχου έκδοσης δεν είναι `git` αγνοήστε το σύμφωνα με αυτήν τη διαμόρφωση.

## Αρχείο Διαμόρφωσης

`.i18n/conf.yml` το αρχείο διαμόρφωσης του εργαλείου μετάφρασης `i18` line

Το περιεχόμενο έχει ως εξής:

```
i18n:
  fromTo:
    en: zh ja ko de fr
    # en:

ignore:
  - _*
```

### Γλώσσα Πηγής &

Στο αρχείο ρυθμίσεων, οι υφιστάμενοι του `fromTo`

`en` είναι η γλώσσα πηγής, `zh ja ko de fr` είναι η γλώσσα-στόχος της μετάφρασης.

Κωδικός γλώσσας βλ [i18n.site/i18/LANG_CODE](https://i18n.site/i18/LANG_CODE)

Για παράδειγμα, εάν θέλετε να μεταφράσετε τα κινέζικα στα αγγλικά, ξαναγράψτε αυτή τη γραμμή `zh: en`

Εάν θέλετε να μεταφράσετε σε όλες τις υποστηριζόμενες γλώσσες, αφήστε το κενό μετά το `:` για παράδειγμα

```
i18n:
  fromTo:
    en:
```

### Αγνοήστε Το Αρχείο

Από προεπιλογή, όλα τα αρχεία που ξεκινούν με `.md` και `.yml` στον κατάλογο της γλώσσας πηγής θα μεταφραστούν.

Εάν θέλετε να αγνοήσετε ορισμένα αρχεία και να μην τα μεταφράσετε (όπως ημιτελή πρόχειρα), μπορείτε να χρησιμοποιήσετε τη διαμόρφωση του πεδίου `ignore`

Χρησιμοποιεί σύνταξη παρόμοια με [globset](https://docs.rs/globset/latest/globset/#syntax) `.gitignore` `ignore`

Για παράδειγμα, στο παραπάνω αρχείο διαμόρφωσης `_* ` ότι τα αρχεία που ξεκινούν με `_` δεν θα μεταφραστούν.

## Κανόνες Μετάφρασης

### Οι Συντάκτες Μετάφρασης Δεν Πρέπει Να Προσθέτουν Ή Να Διαγράφουν Γραμμές

Η μετάφραση είναι επεξεργάσιμη. Τροποποιήστε το αρχικό κείμενο και μεταφράστε το ξανά αυτόματα, οι μη αυτόματες τροποποιήσεις στη μετάφραση δεν θα αντικατασταθούν (εάν αυτή η παράγραφος του αρχικού κειμένου δεν έχει τροποποιηθεί).

Σημειώστε όμως ότι οι γραμμές της μετάφρασης και του αρχικού κειμένου πρέπει να αντιστοιχούν μία προς μία. Δηλαδή, μην προσθέτετε ή διαγράφετε γραμμές κατά τη σύνταξη της μετάφρασης. Διαφορετικά, θα προκαλέσει σύγχυση στην κρυφή μνήμη επεξεργασίας μετάφρασης.

Εάν κάτι πάει στραβά, ανατρέξτε στις [Συνήθεις Ερωτήσεις για λύσεις.](/i18/qa#H1)

### `YAML` Μεταφράζω

Το εργαλείο γραμμής εντολών θα βρει όλα τα αρχεία που τελειώνουν με `.yml` στον κατάλογο αρχείων της γλώσσας πηγής και θα τα μεταφράσει.

* Σημειώστε ότι το επίθημα ονόματος αρχείου πρέπει `.yaml` είναι `.yml`

Το εργαλείο μεταφράζει μόνο τις τιμές του λεξικού σε `.yml` όχι τα κλειδιά του λεξικού.

για παράδειγμα `i18n/en/i18n.yml`

```
apiToken: API Token
defaultToken: Default Token
```

θα μεταφραστεί ως `i18n/zh/i18n.yml`

```
apiToken: 接口令牌
defaultToken: 默认令牌
```

Μπορείτε επίσης να τροποποιήσετε τη μετάφραση του `YAML`

Με βάση `YAML` Translation, μπορείτε εύκολα να δημιουργήσετε διεθνείς λύσεις για διάφορες γλώσσες προγραμματισμού.

## Προηγμένη Χρήση

### Υποκατάλογος Μετάφρασης

Εφόσον δημιουργείτε `i18` `.i18n/conf.yml`

Το εργαλείο γραμμής εντολών θα βρει `.i18n/conf.yml` διαμόρφωση σε όλους τους υποκαταλόγους και θα τη μεταφράσει.

Τα έργα που χρησιμοποιούν την αρχιτεκτονική [monorepo](//monorepo.tools)

<img alt="" src="https://p.3ti.site/1719910016.avif">

### Προσαρμοσμένος Κατάλογος Εγκατάστασης

Θα εγκατασταθεί στο `/usr/local/bin`

Εάν δεν έχει άδεια εγγραφής `/usr/local/bin` θα εγκατασταθεί στο `~/.bin` .

Ορισμός μεταβλητών `TO` Μπορείτε να ορίσετε τον κατάλογο εγκατάστασης, για παράδειγμα :

```sh
TO=/bin sudo bash <(curl -sS https://i.i18n.site) i18
```
