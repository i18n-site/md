# Πρόσθετο

Οι προσθήκες μπορούν να ρυθμιστούν στο αρχείο `.i18n/conf.yml`, όπως:

```yml
addon:
  - i18n.addon/toc
```

## Επίσημες Προσθήκες

* `i18n.addon/toc` : Κατάλογος ευρετηρίων
  Δημιουργία ευρετηρίου καταλόγου `json` με βάση το `TOC`, ενεργοποιημένο από προεπιλογή

* `i18n.addon/mouse` : Εφέ ποντικιού

## Σύμβαση Ονοματοδοσίας Αρχείων

Οι προσθήκες είναι πακέτα του `npm`.

Το πακέτο που αντιστοιχεί στο `i18n.addon/toc` είναι [https://www.npmjs.com/package/@i18n.addon/toc](https://www.npmjs.com/package/@i18n.addon/toc).

Οι προσθήκες χρησιμοποιούν από προεπιλογή την πιο πρόσφατη έκδοση και ελέγχουν ενημερώσεις εβδομαδιαία.

Αν επιθυμείτε να κλειδώσετε μια έκδοση, μπορείτε να γράψετε `i18n.addon/toc@0.1.3`.

Η εντολή μετάφρασης `i18n.site` θα εγκαταστήσει το αρχείο σύμβασης του πακέτου προσθήκης και θα το εκτελέσει.

Τα ονόματα των αρχείων σύμβασης είναι τα εξής

### htmIndex.js

`htmIndex.js` θα εγχυθεί στο τέλος του `.i18n/htm/index.js`.

Το `__CONF__` θα αντικατασταθεί με το όνομα της τρέχουσας διαμόρφωσης (π.χ. `dev` ή `ol`).

### afterTran.js

θα καλείται μετά την ολοκλήρωση της μετάφρασης, με τις παρακάτω παραμέτρους.

* `lang_li` : Λίστα γλωσσών, η πρώτη γλώσσα είναι η γλώσσα της πηγής
* `changed` : Αλλαγμένα αρχεία
* `root` : Ρίζα του έργου

Η επιστρεφόμενη τιμή είναι ένα λεξικό, π.χ

```json
{
  file:{
    //  path: txt, for example :
    // "_.json": "[]"
  }
}
```

`file` : Λίστα αρχείων εξόδου, `path` : Διαδρομή αρχείου, `txt` : Περιεχόμενο αρχείου.

## Ενσωματωμένες Συνάρτησεις

Ο ενσωματωμένος `js` χρόνος εκτέλεσης, βασισμένος στη δευτερεύουσα ανάπτυξη του [boa](https://github.com/boa-dev/boa), περιλαμβάνει τις εξής ενσωματωμένες συναρτήσεις:

* `wPath(path, txt)` : Γράψε στο αρχείο
* `rTxt(path)` : Διάβασε το αρχείο κειμένου
* `rBin(path)` : Διάβασε το δυαδικό αρχείο
* `rDir(dirpath)` : Διάβασε τον κατάλογο, η επιστρεφόμενη τιμή είναι μια λίστα : Κατάλογοι, Αρχεία
