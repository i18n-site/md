# Εγκαταστήστε &

## Διαμόρφωση Token

Το `i18n.site` περιέχει ενσωματωμένο εργαλείο μετάφρασης `i18`, παρακαλώ [κάντε κλικ εδώ για να ανατρέξετε στο έγγραφο διαμόρφωσης του token `i18`](/i18/use).

## Εγκατάσταση

```sh
bash <(curl -sS https://i.i18n.site) i18n.site
```

## Έργο Επίδειξης

Ας ξεκινήσουμε με το έργο επίδειξης για να μάθουμε πώς να χρησιμοποιούμε το `i18n.site`.

Πρώτα, κλωνοποιήστε το αποθετήριο επίδειξης και εκτελέστε την εντολή ως εξής:

```
git clone https://github.com/i18n-site/demo.i18n.site.git md
git clone https://github.com/i18n-site/demo.i18n.site.docker.git docker
```

Οι χρήστες στην ηπειρωτική Κίνα μπορούν:

```
git clone https://atomgit.com/i18n/demo.i18n.site.git md
git clone https://atomgit.com/i18n/demo.i18n.site.docker.git docker
```

Το όνομα του καταλόγου του κλώνου του αποθετηρίου `demo.i18n.site` πρέπει να είναι `md`, για να διευκολύνεται η τοπική προεπισκόπηση με το `docker`.

### Δημοσίευση Περιεχομένου

Το `i18n.site` χρησιμοποιεί αρχιτεκτονική [single-page application](https://developer.mozilla.org/docs/Glossary/SPA) και η σελίδα εισόδου του ιστότοπου και το περιεχόμενο του ιστότοπου αναπτύσσονται ανεξάρτητα.

Μετά την εκτέλεση της μετάφρασης, θα δημιουργηθούν οι καταλόγοι `htm` και `v` στον κατάλογο `md/out/dev`.

Εδώ, `dev` σημαίνει ότι έχει χτιστεί με βάση το αρχείο διαμόρφωσης `.i18n/htm/dev.yml`.

`dev` κατάλογος:

Ο κατάλογος `htm` είναι η σελίδα εισόδου του ιστότοπου.

Ο κατάλογος `v` περιέχει το περιεχόμενο του ιστότοπου με αριθμό έκδοσης.

Η τοπική προεπισκόπηση δεν χρειάζεται να ανησυχεί για τον αριθμό έκδοσης και θα αντιγράψει όλα τα αρχεία στον κατάλογο `out/dev/v/0.1.0`.

Για επίσημη κυκλοφορία, τα τροποποιημένα αρχεία θα αντιγραφούν στον νέο κατάλογο αριθμών έκδοσης.

#### Χρησιμοποιήστε -c για να καθορίσετε το αρχείο διαμόρφωσης

Διαφορετικά αρχεία διαμόρφωσης θα δημιουργήσουν αντίστοιχους καταλόγους στον κατάλογο `out`.

Για παράδειγμα, το αρχείο `.i18n/htm/ol.yml` θα δημιουργήσει τον κατάλογο `out/ol`.

`dev.yml` και `ol.yml` είναι οι προεπιλεγμένες διαμορφώσεις.

`dev` είναι η συντομογραφία του `development`, που υποδεικνύει το περιβάλλον ανάπτυξης, χρησιμοποιείται για τοπική προεπισκόπηση και είναι επίσης το προεπιλεγμένο αρχείο διαμόρφωσης.
`ol` είναι η συντομογραφία του `online`, που υποδεικνύει το διαδικτυακό περιβάλλον, χρησιμοποιείται για επίσημη κυκλοφορία και είναι επίσης το προεπιλεγμένο αρχείο διαμόρφωσης όταν χρησιμοποιείται η γραμμή εντολών `-n` για δημοσίευση στο `npm`.

Μπορείτε επίσης να δημιουργήσετε άλλα αρχεία διαμόρφωσης. Χρησιμοποιήστε `--htm_conf` στη γραμμή εντολών για να καθορίσετε το όνομα του αρχείου διαμόρφωσης που θα χρησιμοποιήσετε:

π.χ.:
```
i18n.site --htm_conf yourConfig --save
```

Εδώ, `--save` σημαίνει την ενημέρωση του αριθμού έκδοσης της δημοσίευσης.

##### npm login & Δημοσίευση

Εγκαταστήστε το `nodejs` και συνδεθείτε με `npm login`.

Επεξεργαστείτε `md/.i18n/htm/ol.yml` και αλλάξτε `i18n.site` σε `v: //unpkg.com/i18n.site` στο δικό σας όνομα πακέτου `npm` .

Χρησιμοποιήστε απλώς το μη κατειλημμένο όνομα πακέτου στο [npmjs.com](//npmjs.com). Η χρήση του ονόματος τομέα του ιστότοπου ως ονόματος πακέτου είναι μια καλή επιλογή.

Κατά τη δημοσίευση με βάση το πακέτο `npm`, **βεβαιωθείτε ότι χρησιμοποιείτε `//unpkg.com/` ως πρόθεμα του `v:`**, το `i18n.site` έχει βελτιστοποιήσει ειδικά τον χρόνο προσωρινής αποθήκευσης του `/.v` κάτω από αυτό το πρόθεμα για να επιτύχετε την άμεση προβολή των νέων εκδόσεων.

Εκτελέστε `i18n.site --npm` ή `i18n.site -n` στον κατάλογο `md` για μετάφραση και δημοσίευση.

Εάν χρησιμοποιείτε περιβάλλον συνεχούς ενοποίησης για δημοσίευση, δεν χρειάζεται να εγκαταστήσετε το `nodejs`. Απλώς αντιγράψτε τα δικαιώματα σύνδεσης και δημοσίευσης από το `~/.npmrc` στο περιβάλλον.

Εάν τροποποιήσετε το όνομα πακέτου `v:` στο `ol.yml`, **βεβαιωθείτε ότι πρώτα διαγράψετε το `.i18n/v/ol`** και μετά δημοσιεύσετε.

##### Διακομιστής Μεσολάβησης για Δημοσίευση npm

Εάν οι χρήστες στην ηπειρωτική Κίνα αντιμετωπίσουν προβλήματα δικτύου και δεν μπορούν να δημοσιεύσουν πακέτα `npm`, μπορούν να ρυθμίσουν τη μεταβλητή περιβάλλοντος `https_proxy` για να ρυθμίσουν τις παραμέτρους του διακομιστή μεσολάβησης.

Υποθέτοντας ότι η θύρα του διακομιστή μεσολάβησης είναι `7890`, μπορείτε να γράψετε:

```
https_proxy=http://127.0.0.1:7890 i18n.site -n
```

#### Αυτο-Φιλοξενία Περιεχομένου

Εάν θέλετε να φιλοξενήσετε μόνοι σας το περιεχόμενο, επεξεργαστείτε πρώτα `md/.i18n/htm/ol.yml` και αλλάξτε `v: //unpkg.com/i18n.site` στο πρόθεμα URL σας, όπως `v: //i18n-v.xxx.com` .

Εισέλθετε στον κατάλογο `md` και εκτελέστε:

```
i18n.site --htm_conf ol --save
```

ή συντομογραφία

```
i18n.site -c ol -s
```

Στη συνέχεια, διαμορφώστε το περιεχόμενο στον κατάλογο `md/out/ol/v` στη διαδρομή του προθέματος URL που έχει οριστεί στο `v:`.

Τέλος, **διαμορφώστε τον χρόνο προσωρινής αποθήκευσης της διαδρομής που τελειώνει σε `/.v` σε `1s`**, διαφορετικά δεν είναι δυνατή η άμεση πρόσβαση στο περιεχόμενο που κυκλοφόρησε πρόσφατα.

Ο χρόνος προσωρινής αποθήκευσης για άλλες διαδρομές μπορεί να οριστεί σε ένα έτος ή περισσότερο για να μειωθούν τα περιττά αιτήματα.

##### Φιλοξενία Περιεχομένου στο S3

Για να φιλοξενήσετε το περιεχόμενο μόνοι σας, εκτός από τη χρήση του δικού σας διακομιστή, μια άλλη κοινή επιλογή είναι να χρησιμοποιήσετε `CDN` `S3` +

Μπορείτε να χρησιμοποιήσετε το [rclone](https://rclone.org) για να συνδεθείτε στον διακομιστή `S3`, στη συνέχεια να ανατρέξετε και να τροποποιήσετε το ακόλουθο σενάριο και να αντιγράψετε μόνο τις σταδιακές αλλαγές στο `S3` για κάθε έκδοση.

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

Θυμηθείτε να διαμορφώσετε `CDN` έτσι ώστε ο χρόνος προσωρινής αποθήκευσης της διαδρομής που τελειώνει σε `/.v` να είναι `1s`, διαφορετικά δεν είναι δυνατή η άμεση πρόσβαση στο περιεχόμενο που κυκλοφόρησε πρόσφατα.

#### Βελτιστοποίηση της ανάπτυξης ιστοσελίδων στην ηπειρωτική Κίνα

Για να επιτύχετε καλύτερη απόδοση στην προσβασιμότητα από το δίκτυο της ηπειρωτικής Κίνας, παρακαλώ [καταχωρήστε πρώτα το όνομα τομέα](//beian.aliyun.com).

Στη συνέχεια, χρησιμοποιήστε την αποθήκευση αντικειμένων από πάροχους cloud στην ηπειρωτική Κίνα μαζί με το `CDN` για να αναπτύξετε το περιεχόμενο του `out/ol/htm`.

Μπορείτε να χρησιμοποιήσετε την edge computing για να αναγράψετε τη διαδρομή για να προσαρμόσετε σε εφαρμογές μίας σελίδας, π.χ. το [Baidu Smart Cloud `CDN`](//cloud.baidu.com/product/cdn.html) μπορεί να ρυθμιστεί ως εξής:

```js
var uri=r.uri,p=uri.lastIndexOf('.');
if(
  p<0 || !'|avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml|'.includes('|'+uri.slice(p+1)+'|')
){
  r.uri='/index.html'
}
r.respHeader(()=>{
var t = [];
r.rawHeadersOut.forEach((i)=>{
    var out = r.headersOut;
    var key = i[0].toLowerCase();
    if(key.startsWith('x-')||key.startsWith('ohc-')){
        delete out[key]
    }
    out['Cache-Control']='max-age='+9e5;
    ['Content-MD5','Age','Expires','Last-Modified'].forEach((i)=>delete out[i])
})
})
```
![](https://p.3ti.site/1721121273.avif)

Λόγω του ότι οι εγγραφές `MX` και `CNAME` δεν μπορούν να συνυπάρχουν, αν επιθυμείτε να λαμβάνετε ταχυδρομικά μηνύματα με το όνομα τομέα ταυτόχρονα, θα πρέπει να συνδυάσετε το [cname_flatten](https://github.com/i18n-site/lib/tree/main/cname_flatten) για να μετατρέψετε το `CNAME` σε `A`.

Επιπλέον, λόγω του ότι οι χρεώσεις για την κυκλοφορία στο εξωτερικό από τους πάροχους cloud στην ηπειρωτική Κίνα είναι υψηλές, για να βελτιστοποιήσετε το κόστος, μπορείτε να συνδυάσετε τη [δωρεάν γεωγραφική ανάλυση του Huawei DNS](https://support.huaweicloud.com/usermanual-dns/dns_usermanual_0041.html) και το προσαρμοσμένο όνομα τομέα του [Cloudflare for SaaS](https://developers.cloudflare.com/cloudflare-for-platforms/cloudflare-for-saas) (όπως φαίνεται στην εικόνα), για να επιτύχετε εκτροπή της κυκλοφορίας — η κυκλοφορία από την ηπειρωτική Κίνα να περνά από το `CDN` της Baidu, και η διεθνής κυκλοφορία να περνά από το Cloudflare.

![](https://p.3ti.site/1721119788.avif)

Αυτές οι στρατηγικές βελτιστοποίησης ανάπτυξης είναι αρκετά σύνθετες και θα παρουσιαστούν σε μελλοντικά κεφάλαια με λεπτομέρεια.

#### Προσαρμογή της γενικής ανακατεύθυνσης τομέα

Αν χρησιμοποιείτε το `i18n.site` για να δημιουργήσετε τον κύριο ιστότοπό σας, συχνά θα πρέπει να ρυθμίσετε την ανακατεύθυνση του γενικού τομέα, δηλαδή να ανακατευθύνετε την πρόσβαση από το `*.xxx.com` (περιλαμβανομένου του `www.xxx.com`) στον τομέα `xxx.com`.

Αυτή η απαίτηση μπορεί να επιτευχθεί με τη χρήση του `EdgeScript` από το Alibaba Cloud `CDN` ( [Αγγλική τεκμηρίωση](https://www.alibabacloud.com/help/en/cdn/developer-reference/how-edgescript-works) / [Κινέζικη τεκμηρίωση](https://help.aliyun.com/zh/cdn/developer-reference/edgescript))

Προσθέστε το όνομα τομέα στο [Alibaba CDN](https://cdn.console.aliyun.com/domain/list) και κατευθύνετε το `*.xxx.com` στο `CNAME` του Alibaba Cloud `CDN`.

![](https://p.3ti.site/1721122000.avif)

Για παράδειγμα, η ρύθμιση ανακατεύθυνσης του γενικού τομέα `*.i18n.site` είναι η εξής:

```
rewrite(concat('https://i18n.site/',$uri), 'redirect',301)
```

![](https://p.3ti.site/1721121934.avif)

#### Ανάπτυξη με το nginx

Προσθέστε τη διαμόρφωση `out/ol/htm` στον κατάλογο `/root/i18n/md/out/ol/htm` στην παράγραφο `server` του nginx:

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```

### Δομή καταλόγου

#### `public`

Τα στατικά αρχεία του ιστοτόπου, όπως `favicon.ico`, `robots.txt` κ.λπ.

Τα αρχεία εικονιδίων μπορούν να δημιουργηθούν χρησιμοποιώντας το [realfavicongenerator.net](https://realfavicongenerator.net).

#### `.i18n`

Στον κατάλογο `.i18n` υπάρχουν τα αρχεία διαμόρφωσης, η προσωρινή μνήμη μεταφράσεων κ.λπ. του `i18n.site`. Για περισσότερες λεπτομέρειες, δείτε το επόμενο κεφάλαιο [『Ρύθμιση』](/i18n.site/conf).

#### `en`

Κατάλογος της πηγαίας γλώσσας, που αντιστοιχεί στο `en` από το `fromTo` στο αρχείο `.i18n/conf.yml`

```yaml
i18n:
  fromTo:
    en: zh
```

Για τη διαμόρφωση της μετάφρασης, ανατρέξτε στο [i18](/i18/use).
