## Διακριτικό Διαμόρφωσης

Το εργαλείο μετάφρασης `i18` είναι ενσωματωμένο, κάντε κλικ εδώ `i18n.site` [να ανατρέξετε στο έγγραφο `i18` για να διαμορφώσετε το διακριτικό πρόσβασης](/i18/use) .

## Εγκαθιστώ

```sh
bash <(curl -sS https://i.i18n.site) i18n.site
```

## Έργο Επίδειξης

Ας ξεκινήσουμε με ένα έργο επίδειξης και ας μάθουμε πώς να το χρησιμοποιήσουμε `i18n.site`

```
git clone https://github.com/i18n-site/demo.i18n.site.git md
git clone https://github.com/i18n-site/demo.i18n.site.docker.git docker
```

Οι χρήστες στην ηπειρωτική Κίνα μπορούν:

### Δημοσιεύστε Περιεχόμενο

`i18n.site` μια αρχιτεκτονική [εφαρμογής μιας σελίδας](https://developer.mozilla.org/docs/Glossary/SPA) και η σελίδα εισόδου του ιστότοπου και το περιεχόμενο του ιστότοπου αναπτύσσονται ανεξάρτητα.

Μετά την εκτέλεση της παραπάνω μετάφρασης, οι κατάλογοι `htm` και `v` θα δημιουργηθούν στον κατάλογο `md/out/dev`

Εδώ, `dev` υποδηλώνει ότι είναι χτισμένο με βάση το αρχείο `.i18n/htm/dev.yml` .

Κάτω από : περιεχόμενο `dev`

Κάτω `htm` τον κατάλογο βρίσκεται η σελίδα εισόδου του ιστότοπου.

`v` κατάλογος περιέχει το περιεχόμενο του ιστότοπου με τον αριθμό έκδοσης.

Η τοπική προεπισκόπηση θα αντιγράψει όλα τα αρχεία στον κατάλογο `out/dev/v/0.1.0` ανεξάρτητα από τον αριθμό έκδοσης.

Για επίσημη κυκλοφορία, τα τροποποιημένα αρχεία θα αντιγραφούν στον νέο κατάλογο αριθμών έκδοσης.

#### Χρησιμοποιήστε -c Για Να Καθορίσετε Το Αρχείο Διαμόρφωσης

Διαφορετικά αρχεία διαμόρφωσης θα δημιουργήσουν αντίστοιχους καταλόγους κάτω από τον κατάλογο `out`

Για παράδειγμα, `.i18n/htm/ol.yml` θα δημιουργήσει έναν `out/ol` κατάλογο.

`dev.yml` και `ol.yml` είναι οι προεπιλεγμένες διαμορφώσεις.

`dev` είναι η συντομογραφία του `development` , που αντιπροσωπεύει το περιβάλλον ανάπτυξης, χρησιμοποιείται για τοπική προεπισκόπηση και είναι επίσης το προεπιλεγμένο αρχείο διαμόρφωσης.
`ol` η συντομογραφία του `online` , που αντιπροσωπεύει το διαδικτυακό περιβάλλον, χρησιμοποιείται για επίσημη κυκλοφορία και είναι επίσης το προεπιλεγμένο αρχείο διαμόρφωσης κατά τη δημοσίευση σε `npm` χρησιμοποιώντας την παράμετρο γραμμής εντολών `-n` .

Μπορείτε επίσης να δημιουργήσετε άλλα αρχεία διαμόρφωσης Χρησιμοποιήστε το `--htm_conf`

για παράδειγμα:
```
i18n.site --htm_conf yourConfig --save
```

`--save` δηλώνει τον αριθμό έκδοσης της ενημέρωσης.

##### Διακομιστής Μεσολάβησης Που Δημοσιεύτηκε Από Το npm

Εάν οι χρήστες στην ηπειρωτική Κίνα αντιμετωπίσουν προβλήματα δικτύου και `https_proxy` μπορούν να δημοσιεύσουν το πακέτο `npm`

Υποθέτοντας ότι η θύρα διακομιστή μεσολάβησής σας είναι `7890` μπορείτε να γράψετε:

```

```

#### Αυτοφιλοξενούμενο Περιεχόμενο

Εάν θέλετε να φιλοξενήσετε μόνοι σας το περιεχόμενο, πρώτα επεξεργαστείτε `md/.i18n/htm/ol.yml` και αλλάξτε `v: //unpkg.com/i18n.site` στο πρόθεμα URL, όπως `v: //i18n-v.xxx.com` .

Εισαγάγετε τον κατάλογο `md`

```
i18n.site --htm_conf ol --save
```

ή συντομογραφία

```
i18n.site -c ol -s
```

Στη συνέχεια, ρυθμίστε το περιεχόμενο στον `v:` `md/out/ol/v`

Τέλος, **ρυθμίστε τον χρόνο προσωρινής αποθήκευσης της διαδρομής που `/.v` σε `1s`** , διαφορετικά δεν είναι δυνατή η άμεση πρόσβαση στο περιεχόμενο που κυκλοφόρησε πρόσφατα.

Ο χρόνος προσωρινής αποθήκευσης για άλλες διαδρομές μπορεί να οριστεί σε ένα έτος ή περισσότερο για να μειωθούν τα περιττά αιτήματα.

##### Φιλοξενήστε Περιεχόμενο Στο s3

Για να φιλοξενήσετε μόνοι σας περιεχόμενο, εκτός από τη χρήση του δικού σας διακομιστή, μια άλλη κοινή επιλογή είναι να χρησιμοποιήσετε `CDN` + `S3`

Μπορείτε να `S3` τον `S3` [rclone](https://rclone.org)

```bash
i18n.site -c ol -s
s3=your-s3
bucket=your-bucket
ver=$(head -1 .i18n/v/ol/v.hash | cut -c 2-)
rclone copy --overwrite-dir out/ol/htm/v/$ver $s3:/$bucket/$ver
rclone copy out/ol/v/.v "$s3:/$bucket/"
```

Θυμηθείτε να ρυθμίσετε τις παραμέτρους `CDN` έτσι ώστε ο χρόνος προσωρινής αποθήκευσης της διαδρομής που τελειώνει σε `/.v` να είναι `1s` , διαφορετικά δεν θα μπορείτε να έχετε άμεση πρόσβαση στο περιεχόμενο που κυκλοφόρησε πρόσφατα.

#### Αναπτύξτε Με Το nginx

Προσθέστε μια διαμόρφωση παρόμοια με την ακόλουθη στην `server` nginx όπου `/root/i18n/md/out/ol/htm` παρακαλούμε αλλάξτε τη στη διαδρομή του δικού σας έργου `out/ol/htm` :

```
location / {
  root  /root/i18n/md/out/ol/htm;
  add_header Cache-Control "max-age=9999999";
  if ($uri !~* \.(avif|css|html|ico|js|json|png|svg|txt|webmanifest|xml)$) {
     rewrite ^ /index.html last;
  }
}
```
