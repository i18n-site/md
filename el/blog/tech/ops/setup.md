## Διαμόρφωση Σύνδεσης Χωρίς Κωδικό Πρόσβασης

Μετά από λίγα λεπτά, οι διακομιστές ενεργοποιήθηκαν όλοι.

`ssh-copy-id root@c2` πρώτο πράγμα IP να επεξεργαστείτε τα τοπικά μηχανήματα `~/.ssh/config`

<img alt="" src="https://i-01.eu.org/1718287198.avif">

Μπορείτε να τροποποιήσετε αυτό το [ssh-copy-id.sh](https://github.com/i18n-ops/ubuntu/blob/main/ssh-copy-id.sh) για να ολοκληρώσετε τη διαμόρφωση αρχικοποίησης πιο γρήγορα.

Αφού ολοκληρωθεί η διαμόρφωση χωρίς κωδικό πρόσβασης, μπορείτε να το χρησιμοποιήσετε [pdsh](https://github.com/chaos/pdsh)

Το πρώτο είναι να ενεργοποιήσετε χειροκίνητα το ipv6 και να δοκιμάσετε

```bash
pdsh -w 'c0 c1 c2' 'bash -i -c enable_ipv6'
pdsh -w 'c0 c1 c2' 'curl -s 6.ipw.cn;echo'
```

<img alt="" src="https://i-01.eu.org/1718287969.avif">

Στη συνέχεια, αναβαθμίστε το σύστημα σε ubuntu 24 με ένα κλικ και εγκαταστήστε το λογισμικό που χρησιμοποιείται συνήθως.

```bash
pdsh -w 'c0 c1 c2' 'curl -s https://raw.githubusercontent.com/i18n-ops/ubuntu/main/boot.sh | CN=1 bash'
```

CN=1 σημαίνει αλλαγή γλώσσας στα κινέζικα και ζώνη ώρας στο Πεκίνο.

Το λογισμικό που είναι εγκατεστημένο από προεπιλογή έχει ρυθμιστεί σύμφωνα με τις προσωπικές μου συνήθειες χρήσης, όπως [mise](https://github.com/jdx/mise) [direnv](https://direnv.net/) , κ.λπ. Εάν είναι απαραίτητο, μπορείτε να αντιγράψετε την αποθήκη [i18n-ops/ubuntu](https://github.com/i18n-ops/ubuntu) και να την τροποποιήσετε όπως απαιτείται.

Αυτό το βήμα διαρκεί πολύ Μετά από μισή ώρα αναμονής, η αναβάθμιση και η εγκατάσταση του συνήθως χρησιμοποιούμενου λογισμικού θα είναι επιτυχής. Μπορείτε να το επιβεβαιώσετε με την ακόλουθη εντολή:

```bash
pdsh -w 'c0 c1 c2' 'cat /etc/issue'
```

## Το mosh Επιταχύνει Το ssh

Το δίκτυο από την Ευρώπη προς την Κίνα δεν είναι σταθερό. Συνιστάται να `ssh` χρησιμοποιήσετε [mosh](//mosh.org)

Δημιουργήστε ένα σενάριο `c0` στον `~/.bin` του τοπικού μηχανήματος (κυρίως προστέθηκε στη μεταβλητή περιβάλλοντος `PATH` ) ως εξής:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

Μετά τη δημιουργία, παραχωρήστε εκτελέσιμα δικαιώματα και soft-link `c1` `c2` .

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

Στο μέλλον, μπορείτε να εισάγετε απευθείας `c0` `c1` και μπορείτε να χρησιμοποιήσετε απευθείας σύνδεση υψηλής ταχύτητας `mosh` και να εισαγάγετε [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html)

Στη συνέχεια, εκτελέστε το ακόλουθο σενάριο για να μετατρέψετε το διαμέρισμα

## Λεπτομέρειες Συστήματος Διάσωσης

Εισαγάγετε το σύστημα 22 Όταν η θύρα είναι [tcping](https://github.com/paradiseduo/tcping) προσβάσιμη, δεν θα μπορείτε να έχετε πρόσβαση με τον καθορισμένο κωδικό πρόσβασης για 2-3 λεπτά.

```
tcping ip 22 -c 9999
```

Η προσβασιμότητα του λιμένα μπορεί να παρακολουθηθεί.

### Πώς Να Αποκτήσετε Πρόσβαση Στο Σκληρό Δίσκο Μετά Την Είσοδο Στο Στιγμιότυπο Restore &

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

Μπορείτε να δείτε το στιγμιότυπο, για παράδειγμα

```
snapper -c etc list
```

Προβολή αλλαγών αρχείου στιγμιότυπου

```
snapper -c etc status 5..6
```

Προβολή συγκεκριμένων διαφορετικών κειμένων

```
snapper -c etc diff 3..5
```

Επαναφορά στιγμιότυπου: Για παράδειγμα, αναιρέστε όλες τις αλλαγές μετά το στιγμιότυπο 5

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### Ενεργοποίηση Επιμονής Καταγραφής

Η ενεργοποίηση της διατήρησης καταγραφής σε ένα σύστημα διάσωσης απαιτεί να δημιουργήσετε με μη αυτόματο τρόπο τον αντίστοιχο κατάλογο και να εκτελέσετε ορισμένα πρόσθετα βήματα, επειδή ενδέχεται να μην εκτελείτε μια υπηρεσία systemd. Ακολουθούν τα βήματα που μπορείτε να δοκιμάσετε:

Πρώτα chroot, μετά

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

Επανεκκίνηση στο σύστημά σας Ubuntu: Αφού κάνετε τις παραπάνω αλλαγές, πρέπει να επανεκκινήσετε το μηχάνημά σας κανονικά και στο σύστημα Ubuntu αντί για το σύστημα διάσωσης.

Με αυτόν τον τρόπο, το systemd-journald θα ξεκινήσει με τη νέα διαμόρφωση και θα αρχίσει να αποθηκεύει τα αρχεία καταγραφής επίμονα στον σκληρό δίσκο.

Λάβετε υπόψη ότι οι παραπάνω λειτουργίες εκτελούνται σε ένα σύστημα διάσωσης, επομένως αυτές οι αλλαγές θα τεθούν σε ισχύ μόνο όταν το σύστημα επανεκκινηθεί και δεν είναι πλέον σε λειτουργία διάσωσης.

Αυτό σημαίνει ότι δεν θα δείτε αμέσως τυχόν αλλαγές στα υπάρχοντα αρχεία καταγραφής να γίνονται μόνιμες, αλλά όλα τα αρχεία καταγραφής που δημιουργούνται μετά από αυτό θα διατηρηθούν.

Με αυτόν τον τρόπο μπορείτε να δείτε τα αρχεία καταγραφής της υπηρεσίας ssh

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## Σημειώσεις Για Δύσκολες Και Περίπλοκες Ασθένειες

### Λύστε Το Περίεργο Πρόβλημα: Ο Διακομιστής Δεν Μπορεί Να ssh

Χρησιμοποίησα την ενέργεια github για να δημιουργήσω ένα σενάριο αυτόματης μεταγλώττισης και ανάπτυξης στο παρελθόν, αλλά διαπίστωσα ότι η υπηρεσία ανάπτυξης δεν μπορούσε να ssh.

Η είσοδος στο σύστημα διάσωσης και η εξέταση των αρχείων καταγραφής είναι ένα ζήτημα άδειας:

`Missing privilege separation directory: /run/sshd`

Μπορείτε να δημιουργήσετε αυτόν τον κατάλογο με την ακόλουθη εντολή

```
chown root:root /
systemd-tmpfiles --create
```

Μετά από επανειλημμένες έρευνες, ο χρήστης που ανέβασε tar `uid` `1001` github action

tar αποσυσκευασία πρέπει να γίνει χωρίς παραμέτρους `--no-same-owner` θα αποσυμπιεστεί σε `uid` για `1001` χρήστη

Ως αποτέλεσμα, το rsync θα αλλάξει την ομάδα χρηστών του ριζικού καταλόγου και, στη συνέχεια, δεν θα μπορείτε να συνδεθείτε στο ssh μετά την επανεκκίνηση.

Τροποποιήστε το σενάριο αποσυμπίεσης ως εξής:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
