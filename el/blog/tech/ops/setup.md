## Το mosh επιταχύνει τη σύνδεση ssh

Το δίκτυο από την Ευρώπη προς την Κίνα δεν είναι σταθερό. Συνιστάται η χρήση [mosh](//mosh.org) `ssh` .

Δημιουργήστε το σενάριο `c0` στον κατάλογο `~/.bin` (ο οποίος θα προστεθεί στη μεταβλητή περιβάλλοντος `PATH`) ως εξής:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
arg=/root/.tmux_default
else
arg=${@:1:$#}
fi

vps=$(basename $0)
set -ex
mosh $vps -p 60000:60009 $arg
```

Στο μέλλον, μπορείτε να εισάγετε απευθείας `c0` ή `c1` για να συνδεθείτε γρήγορα στο TMUX μέσω του Mosh.

```bash
chmod +x c0
ln -s c0 c1
ln -s c0 c2
```

Στη συνέχεια, μπορείτε να εισάγετε απευθείας `c0` και `c1` για να χρησιμοποιήσετε το `mosh` για γρήγορη σύνδεση και να εισέλθετε στο [tmux](https://www.ruanyifeng.com/blog/2019/10/tmux.html).

## Μπορείτε να παρακολουθήσετε την προσβασιμότητα της θύρας

Εισαγάγετε το σύστημα διάσωσης. Μόλις η θύρα 22 είναι προσβάσιμη μέσω [tcping](https://github.com/paradiseduo/tcping), θα υπάρξει μια περίοδος 2-3 λεπτών κατά την οποία δεν θα μπορείτε να συνδεθείτε με τον καθορισμένο κωδικό πρόσβασης. Παρακαλώ περιμένετε.

```
tcping ip 22 -c 9999
```

Μπορείτε να δείτε το στιγμιότυπο, για παράδειγμα:

### Προβολή αλλαγών αρχείων στιγμιότυπου

```
mkdir -p /mnt/os
mount  /dev/sda3 /mnt/os -t btrfs -o "defaults,ssd,discard,noatime,compress=zstd:3,space_cache=v2"
mount --bind /dev /mnt/os/dev
mount --bind /proc /mnt/os/proc
mount --bind /sys /mnt/os/sys
mount --bind /run /mnt/os/run
chroot /mnt/os zsh -c "mount -a && dbus-daemon --system 2>/dev/null; exec zsh"
```

Προβολή συγκεκριμένων διαφορών κειμένων

```
snapper -c etc list
```

Εμφάνιση των αλλαγών στα αρχεία του στιγμιότυπου

```
snapper -c etc status 5..6
```

Ενεργοποίηση επιμονής καταγραφής

```
snapper -c etc diff 3..5
```

Επαναφορά στιγμιότυπου: Για παράδειγμα, αναιρέστε όλες τις αλλαγές μετά το στιγμιότυπο 5

```
snapper -c etc undochange 1..0
snapper -c disk undochange 1..0
```

### Πρώτα εκτελέστε chroot, μετά:

Επανεκκινήστε στο σύστημά σας Ubuntu: Αφού κάνετε τις παραπάνω αλλαγές, πρέπει να επανεκκινήσετε το μηχάνημά σας κανονικά και να μεταβείτε στο σύστημα Ubuntu αντί για το σύστημα διάσωσης

Πρώτα εκτελέστε chroot, μετά

```
mkdir -p /var/log/journal
chown root:systemd-journal /var/log/journal
chmod 2755 /var/log/journal
sed -i 's/^#\?\s*Storage=.*/Storage=persistent/' /etc/systemd/journald.conf
```

Λάβετε υπόψη ότι οι παραπάνω λειτουργίες εκτελούνται σε ένα σύστημα διάσωσης, επομένως αυτές οι αλλαγές δεν θα τεθούν σε ισχύ έως ότου επανεκκινηθεί το σύστημα και δεν βρίσκεται πλέον σε λειτουργία διάσωσης.

Αυτό σημαίνει ότι δεν θα δείτε αμέσως αλλαγές στα υπάρχοντα αρχεία καταγραφής που γίνονται μόνιμες, αλλά όλα τα αρχεία καταγραφής που δημιουργούνται μετά από αυτό θα διατηρηθούν.

Με αυτόν τον τρόπο μπορείτε να δείτε τα αρχεία καταγραφής της υπηρεσίας SSH.

Σημειώσεις για δύσκολες και περίπλοκες ασθένειες.

Λύση του περίεργου προβλήματος: Ο διακομιστής δεν μπορεί να συνδεθεί μέσω SSH

```
journalctl --directory=/var/log/journal  _SYSTEMD_UNIT=ssh.service
```

## Σημειώσεις για δύσκολες και περίπλοκες προβλήματα

### Η είσοδος στο σύστημα διάσωσης και η εξέταση των αρχείων καταγραφής έδειξαν ότι υπήρχε πρόβλημα άδειας:

Μπορείτε να δημιουργήσετε αυτόν τον κατάλογο με την ακόλουθη εντολή:

Η είσοδος στο σύστημα διάσωσης και η εξέταση των αρχείων καταγραφής είναι ένα ζήτημα άδειας:

`Missing privilege separation directory: /run/sshd`

Μπορείτε να δημιουργήσετε αυτόν τον κατάλογο με την ακόλουθη εντολή

```
chown root:root /
systemd-tmpfiles --create
```

Μετά από επανειλημμένες έρευνες, διαπίστωσα ότι ο χρήστης `uid` του tar στο github action είναι `1001`

Η αποσυσκευασία πρέπει να γίνει χωρίς την παράμετρο `--no-same-owner`, αλλιώς θα αποσυμπιεστεί στο `uid` `1001`

Ως αποτέλεσμα, το rsync θα αλλάξει την ομάδα χρηστών του ριζικού καταλόγου και, στη συνέχεια, δεν θα μπορείτε να συνδεθείτε στο ssh μετά την επανεκκίνηση.

Τροποποιήστε το σενάριο αποσυμπίεσης ως εξής:

```bash
tar --no-same-owner -xvf $TZT
rm $TZT

if [ -f "init.sh" ]; then
  ./init.sh
  rm init.sh
fi

rsync --remove-source-files -av . /
```
